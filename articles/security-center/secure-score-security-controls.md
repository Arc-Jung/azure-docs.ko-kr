# <a name="enhanced-secure-score-preview-in-azure-security-center"></a>Azure 보안 센터에서 향상된 보안 점수(미리 보기)

## <a name="introduction-to-secure-score"></a>보안 점수 소개

Azure 보안 센터에는 현재 보안 상황을 이해하고 보안을 효율적이고 효과적으로 개선하는 데 도움이 되는 두 가지 주요 목표가 있습니다. 이러한 목표를 달성할 수 있는 보안 센터의 핵심 측면은 안전한 점수입니다.

보안 센터는 보안 문제에 대한 리소스, 구독 및 조직을 지속적으로 평가합니다. 그런 다음 모든 결과를 단일 점수로 집계하여 현재 보안 상황을 한눈에 알 수 있습니다. 점수를 사용하여 조직의 보안 노력과 프로젝트를 추적합니다. 

보안 센터의 보안 점수 페이지에는 다음이 포함됩니다.

- **점수** - 보안 점수는 백분율 값으로 표시되지만 기본 값도 명확합니다.

    [![기본 숫자가 명확한 백분율 값으로 표시된 보안 점수도 명확합니다.](media/secure-score-security-controls/secure-score-with-percentage.png)](media/secure-score-security-controls/secure-score-with-percentage.png#lightbox)

- **보안 제어** - 각 컨트롤은 관련 보안 권장 사항의 논리적 그룹이며 취약한 공격 표면을 반영합니다. 컨트롤은 이러한 권장 사항을 구현하는 데 도움이 되는 지침이 있는 일련의 보안 권장 사항입니다. 점수는 컨트롤 내의 단일 리소스에 대한 *모든* 권장 사항을 수정할 때만 향상됩니다.

    조직에서 각 개별 공격 표면을 얼마나 잘 보호하고 있는지 즉시 확인하려면 각 보안 제어에 대한 점수를 검토합니다.

    자세한 내용은 [보안 점수가 아래 계산 방법을 참조하세요.](secure-score-security-controls.md#how-the-secure-score-is-calculated) 


>[!TIP]
> 이전 버전의 보안 센터는 권장 수준에서 점수를 부여했습니다. 이제 컨트롤 내의 단일 리소스에 대한 *모든* 권장 사항을 수정하는 경우에만 점수가 향상됩니다. 따라서 리소스의 보안을 개선했을 때만 점수가 향상됩니다.
> 이 향상된 버전은 아직 미리 보기 이지만 이전 보안 점수 환경Azure Portal에서 옵션으로 사용할 수 있습니다. 


## <a name="locating-your-secure-score"></a>보안 점수 찾기

보안 센터는 점수를 눈에 띄게 표시합니다: 개요 페이지에 가장 먼저 표시되는 것입니다. 전용 보안 점수 페이지를 클릭하면 구독별로 점수가 세분화된 것을 볼 수 있습니다. 단일 구독을 클릭하여 우선 순위가 높은 권장 사항의 자세한 목록과 수정이 구독 점수에 미치는 잠재적 영향을 확인합니다.

## <a name="how-the-secure-score-is-calculated"></a>보안 점수 계산 방법 

전체 보안 점수에 대한 각 보안 컨트롤의 기여도는 권장 사항 페이지에 명확하게 표시됩니다.

[![향상된 보안 점수는 보안 제어를 도입합니다.](media/secure-score-security-controls/security-controls.png)](media/secure-score-security-controls/security-controls.png#lightbox)

보안 제어에 대한 가능한 모든 포인트를 얻으려면 모든 리소스가 보안 제어 내의 모든 보안 권장 사항을 준수해야 합니다. 예를 들어 보안 센터에는 관리 포트를 보호하는 방법에 대한 여러 권장 사항이 있습니다. 과거에는 관련 및 상호 의존적인 권장 사항 중 일부를 수정하는 동시에 다른 권장 사항을 해결하지 않은 상태로 두면 보안 점수가 향상될 수 있습니다. 객관적으로 살펴보면 모든 것을 해결할 때까지 보안이 개선되지 않았다고 주장하기가 쉽습니다. 지금, 당신은 당신의 보안 점수에 차이를 만들기 위해 그들 모두를 수정해야합니다.

예를 들어 "시스템 업데이트 적용"이라는 보안 제어에는 최대 점수가 6개있으며, 도구 설명에서 컨트롤의 잠재적 증가 값에 대해 확인할 수 있습니다.

[![보안 제어 "시스템 업데이트 적용"](media/secure-score-security-controls/apply-system-updates-control.png)](media/secure-score-security-controls/apply-system-updates-control.png#lightbox)

위의 스크린샷에서 보안 제어 "시스템 업데이트 적용"의 가능성은 "2%(1포인트)"를 보여줍니다. 즉, 이 컨트롤의 모든 권장 사항을 수정하면 점수가 2% 증가합니다(이 경우 1점). 간단히 하기 위해 권장 사항 목록의 "증가 가능성" 열의 값은 정수로 반올림됩니다. 도구 설명은 정확한 값을 보여 줍니다.

* **최대 점수** - 컨트롤 내의 모든 권장 사항을 완료하여 얻을 수 있는 최대 점수입니다. 컨트롤의 최대 점수는 해당 컨트롤의 상대적 유의를 나타냅니다. 최대 점수 값을 사용하여 먼저 작업할 문제를 분류합니다. 
* **잠재적 증가** - 컨트롤 내에서 사용할 수 있는 나머지 점입니다. 이러한 점수를 보안 점수에 추가하려면 컨트롤의 모든 권장 사항을 수정합니다. 위의 예에서 컨트롤에 대해 표시된 한 점은 실제로 0.96포인트입니다.
* **현재 점수** - 이 컨트롤의 현재 점수입니다. 각 컨트롤은 총 점수에 기여합니다. 이 예제에서 컨트롤은 합계에 5.04포인트를 기여합니다.

## <a name="improving-your-secure-score"></a>보안 점수 향상

보안 점수를 향상하려면 권장 사항 목록에서 보안 권장 사항을 수정합니다. 각 리소스에 대해 수동으로 또는 **빠른 수정을** 사용하여 각 권장 사항을 수동으로 수정할 수 있습니다! 옵션(사용 가능한 경우)을 사용하여 리소스 그룹에 권장 사항에 대한 수정을 신속하게 적용할 수 있습니다. 자세한 내용은 [권장 사항 수정을](security-center-remediate-recommendations.md)참조하십시오.

>[!IMPORTANT]
> 기본 제공 권장 사항만 보안 점수에 영향을 미칩니다.

## <a name="secure-score-faq"></a>보안 점수 자주 묻는 질문

### <a name="why-has-my-secure-score-gone-down"></a>보안 점수가 내려간 이유는 무엇입니까?
보안 센터는 점수 계산 방식의 변경 사항을 포함하는 향상된 보안 점수(현재 미리 보기 상태)로 전환했습니다. 이제 점수를 받으려면 리소스에 대한 모든 권장 사항을 해결해야 합니다. 점수도 0-10의 척도로 바뀌었습니다.

### <a name="if-i-address-only-three-out-of-four-recommendations-in-a-security-control-will-my-secure-score-change"></a>보안 제어에서 4가지 권장 사항 중 3개만 해결하면 보안 점수가 변경됩니다.
아니요. 단일 리소스에 대한 모든 권장 사항을 수정할 때까지 변경되지 않습니다. 컨트롤의 최대 점수를 얻으려면 모든 리소스에 대해 모든 권장 사항을 수정해야 합니다.

### <a name="is-the-previous-experience-of-the-secure-score-still-available"></a>보안 점수의 이전 환경은 여전히 사용할 수 있습니까? 
예. 잠시 동안 그들은 전환을 완화하기 위해 나란히 실행됩니다. 이전 모델이 시간에 따라 단계적으로 폐지될 것으로 예상합니다. 

### <a name="if-a-recommendation-isnt-applicable-to-me-and-i-disable-it-in-the-policy-will-my-security-control-be-fulfilled-and-my-secure-score-updated"></a>권장 사항이 적용되지 않고 정책에서 권장 사항이 비활성화된 경우 보안 제어가 충족되고 보안 점수가 업데이트되나요?
예. 사용자 환경에서 권장 사항이 적용되지 않는 경우 권장 사항을 사용하지 않도록 설정하는 것이 좋습니다. 특정 권장 사항을 사용하지 않도록 설정하는 방법에 대한 지침은 [보안 정책 사용 안 함](https://docs.microsoft.com/azure/security-center/tutorial-security-policy#disable-security-policies)참조를 참조하십시오.

### <a name="if-a-security-control-offers-me-zero-points-towards-my-secure-score-should-i-ignore-it"></a>보안 컨트롤이 내 보안 점수에 대해 0점을 제공하는 경우 무시해야 합니까?
경우에 따라 컨트롤 최대 점수가 0보다 크지만 영향은 0입니다. 리소스 고정을 위한 증분 점수가 무시할 수 있으면 0으로 반올림됩니다. 이러한 권장 사항은 여전히 보안 이 향상된 기능을 제공하기 때문에 무시하지 마십시오. 유일한 예외는 "추가 모범 사례" 컨트롤입니다. 이러한 권장 사항을 수정하면 점수가 증가하지는 않지만 전반적인 보안이 향상됩니다.
