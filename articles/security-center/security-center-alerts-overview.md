# <a name="security-alerts-in-azure-security-center"></a>Azure Security Center의 보안 경고

Azure 보안 센터에는 다양한 리소스 유형에 대한 다양한 경고가 있습니다. 보안 센터는 Azure에 배포된 리소스와 온-프레미스 및 하이브리드 클라우드 환경에 배포된 리소스에 대한 경고를 생성합니다.

보안 경고는 고급 검색에 의해 트리거되며 Azure 보안 센터의 표준 계층에서만 사용할 수 있습니다. 평가판을 사용할 수 있습니다. [보안 정책](security-center-pricing.md)의 가격 책정 계층 선택에서 업그레이드할 수 있습니다. 가격 책정에 대한 자세한 내용은 [보안 센터 페이지](https://azure.microsoft.com/pricing/details/security-center/) 를 방문하세요.

## <a name="responding-to-todays-threats"></a>오늘날의 위협에 <a name="respond-threats"> </a> 대응

지난 20년 동안 위협 환경에 중요한 변경 내용이 있었습니다. 과거에는 일반적으로 "할 수 있는 일"을 보는 데 주로 관심이 있는 개별 공격자에 의한 웹 사이트 훼손에 대해서만 걱정해야 했습니다. 오늘날의 공격자는 훨씬 더 정교하고 조직적입니다. 특정 금융 및 전략적 목표가 있는 경우가 많습니다. 또한 국가 또는 조직 범죄로 자금 지원을 받으므로 더 많은 사용 가능한 리소스가 있습니다.

이러한 변화하는 현실은 공격자 계급에서 전례없는 수준의 전문성을 이끌었습니다. 공격자는 더 이상 웹 손상에 관심이 없습니다. 그들은 이제 정보, 금융 계좌 및 개인 데이터를 훔치는 데 관심이 있습니다 . 재무 목표를 가진 공격자보다 더욱 걱정되는 것은 인프라 및 사용자에게 해를 입히기 위해 네트워크를 위반하는 공격자입니다.

이에 대한 응답으로 조직은 종종 알려진 공격 서명을 검색하여 엔터프라이즈 경계 또는 엔드포인트 방어에 집중하는 다양한 지점 솔루션을 배포합니다. 이러한 솔루션은 보안 분석가에게 심사 및 조사를 요구하는 대용량의 낮은 신뢰 경고를 생성하는 경향이 있습니다. 대부분의 조직은 이러한 경고에 응답하는 데 필요한 시간 및 전문 지식이 부족하므로 많은 경고가 해결되지 않습니다.  

또한 공격자는 많은 서명 기반 방어를 파괴하고 클라우드 [환경에 적응하기 위해](https://azure.microsoft.com/blog/detecting-threats-with-azure-security-center/)방법을 발전시켰습니다. 새로운 위협 요소를 보다 신속하게 식별하고 감지 및 대응을 신속하게 처리하기 위해 새로운 접근 방식이 필요합니다.

## <a name="what-are-security-alerts-and-security-incidents"></a>보안 경고 및 보안 인시던트란 무엇입니까? 

**경고는** 보안 센터에서 리소스에 대한 위협을 감지할 때 생성하는 알림입니다. 보안 센터는 문제를 신속하게 조사하는 데 필요한 정보와 함께 경고의 우선 순위를 지정하고 나열합니다. Security Center는 공격을 해결하는 방법에 대한 권장 사항도 제공합니다.

**보안 인시던트는** 각 경고를 개별적으로 나열하는 대신 관련 경고의 모음입니다. 보안 센터는 [클라우드 스마트 경고 상관 관계를](security-center-alerts-cloud-smart.md) 사용하여 다양한 경고 및 낮은 충실도 신호를 보안 인시던트와 상호 연관시킵니다.

보안 센터는 인시던트를 사용하여 공격 캠페인과 모든 관련 경고에 대한 단일 보기를 제공합니다. 이 보기를 사용하면 공격자가 수행한 작업과 영향을 받은 리소스를 신속하게 파악할 수 있습니다. 자세한 내용은 [클라우드 스마트 경고 상관 관계를](security-center-alerts-cloud-smart.md)참조하십시오.



## <a name="how-does-security-center-detect-threats"></a>보안 센터는 위협을 어떻게 감지합니까? <a name="detect-threats"> </a>

Microsoft 보안 연구원은 지속적으로 위협을 지켜보고 있습니다. 클라우드 및 온-프레미스에서 Microsoft의 글로벌 입지로 인해 광범위한 원격 분석 집합에 액세스할 수 있습니다. 광범위하고 다양한 데이터 집합 컬렉션을 통해 온-프레미스 소비자 및 엔터프라이즈 제품뿐만 아니라 온라인 서비스 전반에서 새로운 공격 패턴과 추세를 발견할 수 있습니다. 결과적으로 보안 센터는 공격자가 새롭고 더욱 정교한 악용을 릴리스하는 동안 신속하게 해당 감지 알고리즘을 업데이트할 수 있습니다. 이 방법을 통해 빠르게 움직이는 위협 환경을 지속적으로 관리할 수 있습니다.

실제 위협을 감지하고 거짓 긍정을 줄이기 위해 보안 센터는 Azure 리소스 및 네트워크에서 로그 데이터를 수집, 분석 및 통합합니다. 또한 방화벽 및 엔드포인트 보호 솔루션과 같은 연결된 파트너 솔루션과도 작동합니다. 보안 센터는 이 정보를 분석하여 여러 소스의 정보를 상호 연관하여 위협을 식별합니다.

![보안 센터 데이터 수집 및 프레젠테이션](./media/security-center-alerts-overview/security-center-detection-capabilities.png)

보안 센터는 서명 기반 방식을 뛰어 넘는 고급 보안 분석을 사용합니다. 수동 접근 방법을 사용하고 공격의 발전을 예측하여 식별할 수 없는 위협을 감지하도록 전체 클라우드 패브릭에 대한 이벤트를 평가하는 데 빅 데이터 및 [기계 학습](https://azure.microsoft.com/blog/machine-learning-in-azure-security-center/) 기술의 돌파구를 활용합니다. 이러한 보안 분석은 다음과 같습니다.

* **통합 위협 인텔리전스**: Microsoft는 엄청난 양의 글로벌 위협 인텔리전스를 보유하고 있습니다. 원격 분석은 Azure, Office 365, Microsoft CRM online, Microsoft Dynamics AX, outlook.com, MSN.com, Microsoft DCU(Digital Crimes Unit) 및 MSRC(Microsoft 보안 대응 센터)와 같은 여러 출처에서 유입됩니다. 연구원은 또한 주요 클라우드 서비스 공급자 간에 공유되는 위협 인텔리전스 정보를 수신하고 다른 제3자로부터 피드합니다. Azure Security Center는 이 정보를 사용하여 알려진 부정 행위자의 위협을 경고할 수 있습니다.

* **행동 분석**: 행동 분석은 데이터를 분석하고 알려진 패턴의 컬렉션과 비교하는 기술입니다. 그러나 이러한 패턴은 단순한 서명이 아닙니다. 대량 데이터 세트에 적용되는 복잡한 기계 학습 알고리즘을 통해 결정됩니다. 또한 전문 분석가가 악의적인 행동을 신중하게 분석하여 결정합니다. Azure Security Center는 동작 분석을 사용하여 가상 머신 로그, 가상 네트워크 디바이스 로그, 패브릭 로그, 크래시 덤프 및 기타 원본에 대한 분석에 따라 손상된 리소스를 식별할 수 있습니다.

* **변칙 검색**: Azure Security Center는 변칙 검색을 사용하여 위협을 식별합니다. 대규모 데이터 집합에서 파생된 알려진 패턴에 따라 달라지는 행동 분석과 달리 변칙 검색은 보다 "개인화"되어 배포와 관련된 기준에 중점을 둡니다. 배포에 대한 정상적인 작동을 확인하기 위해 기계 학습이 적용되고 보안 이벤트를 표시할 수 있는 이상값 조건을 정의하는 규칙이 생성됩니다.

## <a name="how-are-alerts-classified"></a>경고는 어떻게 분류되나요?

보안 센터는 각 경고에 참석하는 순서의 우선 순위를 지정하여 리소스가 손상될 때 즉시 받을 수 있도록 경고에 심각도를 할당합니다. 심각도는 보안 센터가 찾기 또는 경고를 발행하는 데 사용되는 분석 및 경고로 이어진 활동의 배후에 악의적인 의도가 있다는 신뢰 수준에 얼마나 확신을 가지고 있는지에 따라 다집니다.

> [!NOTE]
> 경고 심각도는 01-01-2019 이전의 포털 및 REST API 버전에서 다르게 표시됩니다. 이전 버전의 API를 사용하는 경우 아래에 설명된 일관된 환경을 위해 업그레이드하십시오.

- **높음:** 리소스가 손상될 가능성이 높습니다. 지금 즉시 리소스를 살펴보아야 합니다. Security Center는 경고 실행에 사용되는 악의적 의도와 결과 둘 다에서 신뢰성이 높습니다. 자격 증명 훔치기에 많이 사용되는 Mimikatz처럼 알려진 악의적 도구 실행을 검색하는 경고를 예로 들 수 있습니다.
- **중간 크기:** 의심스러운 활동일 수 있는 리소스가 손상되었음을 나타낼 수 있습니다.
분석 또는 검색에 대한 보안 센터의 신뢰도는 중간이며 악의적인 의도에 대한 신뢰도는 중간에서 높음입니다. 이러한 검색은 일반적으로 기계 학습 또는 변칙 기반 검색입니다. 예를 들어 비정상적인 위치에서 로그인을 시도합니다.
- **낮음:** 이는 양성 양성 또는 차단된 공격일 수 있습니다.
   * Security Center는 신뢰도가 높지 않기 때문에 의도가 악의적이고 작업은 악의적이지 않을 수 있습니다. 예를 들어 로그 지우기는 공격자가 흔적을 숨기려 시도할 때 발생할 수도 있지만, 대부분은 관리자가 수행하는 일상적인 작업입니다.
   * 보안 센터는 일반적으로 공격이 차단된 시기를 알려주지 않습니다. 
- **정보:** 보안 인시던트를 드릴다운하거나 특정 경고 ID로 REST API를 사용하는 경우에만 정보 경고가 표시됩니다. 일반적으로 한 인시던트는 여러 경고로 구성되며, 그 중 일부는 오직 정보 제공만을 위해 단독으로 표시될 수 있지만 다른 경고의 컨텍스트에서는 자세히 살펴볼 필요가 있습니다. 

## <a name="continuous-monitoring-and-assessments"></a>지속적인 모니터링 및 평가

Azure 보안 센터는 Microsoft 전역에서 보안 연구 및 데이터 과학 팀이 위협 환경의 변화를 지속적으로 모니터링할 수 있다는 이점을 누릴 수 있습니다. 다음 이니셔티브가 포함됩니다.

* **위협 인텔리전스 모니터링**: 위협 인텔리전스에는 기존 또는 새로운 위협에 대한 메커니즘, 지표, 시사점 및 실행 가능한 조언이 포함됩니다. 이 정보는 보안 커뮤니티에서 공유되고 Microsoft는 내부 및 외부 소스에서 위협 인텔리전스 피드를 지속적으로 모니터링합니다.
* **신호 공유**: Microsoft의 광범위한 클라우드 및 온-프레미스 서비스, 서버 및 클라이언트 엔드포인트 장치 포트폴리오의 보안 팀의 인사이트가 공유되고 분석됩니다.
* **Microsoft 보안 전문가**: 과학 수사 및 웹 공격 감지와 같은 특수 보안 필드에서 활동하는 Microsoft 팀과의 지속적인 참여입니다.
* **탐지 튜닝**: 알고리즘은 실제 고객 데이터 세트에 대해 실행되며 보안 연구원은 고객과 협력하여 결과를 검증합니다. 기계 학습 알고리즘을 구체화하기 위해 참 및 거짓 긍정이 사용됩니다.

이러한 결합된 노력은 새롭고 향상된 탐지로 절정을 이수하며, 즉각적으로 이점을 얻을 수 있으므로 취할 수 있는 조치는 없습니다.
