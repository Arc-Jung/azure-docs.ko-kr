# <a name="enterprise-integration-with-as2"></a>AS2와 엔터프라이즈 통합
논리 앱에서 엔터프라이즈 기능을 사용하려면 먼저 규약을 만들어야 합니다.

## <a name="prerequisites"></a>필수 조건
* Azure 구독에 [통합 계정](../logic-apps/logic-apps-enterprise-integration-accounts.md)을 정의해야 합니다.  
* 통합 계정에 이미 둘 이상의 [파트너](logic-apps-enterprise-integration-partners.md)가 정의되어 있어야 합니다.  

> [!NOTE]
> 규약을 만들 때 규약 파일의 내용이 규약 유형과 일치해야 합니다.    

통합 계정을 만들고 파트너를 추가한 후에 다음 섹션의 절차에 따라 규약을 만들 수 있습니다.  

## <a name="create-an-agreement"></a>규약 만들기

1. [Azure Portal](http://portal.azure.com "Azure Portal")에 로그인합니다.  
2. **추가 서비스**를 선택하고 필터 검색 상자에서 **통합**을 입력한 다음 결과 목록에서 **통합 계정**을 선택합니다.

 ![결과 목록에서 [통합 계정]을 선택합니다.](./media/logic-apps-enterprise-integration-agreements/overview-1.png)    
3. 인증서를 추가할 통합 계정을 선택합니다.

 ![통합 계정 선택](./media/logic-apps-enterprise-integration-overview/overview-3.png)  
4. **규약** 타일을 선택합니다. 타일이 표시되지 않으면 추가합니다.

 ![[규약] 타일을 선택합니다.](./media/logic-apps-enterprise-integration-agreements/agreement-1.png)   
5. **규약** 블레이드에서 **추가**를 선택합니다.

 !["추가" 선택](./media/logic-apps-enterprise-integration-agreements/agreement-2.png)  
6. 규약 이름을 입력하고, **규약 유형** 목록에서 **AS2**를 선택하고, **호스트 파트너**, **호스트 ID**, **게스트 파트너** 및 **게스트 ID** 목록에서 해당 정보를 입력합니다.

 ![규약 이름을 입력합니다.](./media/logic-apps-enterprise-integration-agreements/agreement-3.png)  

 다음 표에서는 **추가** 대화 상자의 속성에 대해 설명합니다.

    | 속성 | 설명 |
    | --- | --- |
    | 호스트 파트너 | 규약에는 호스트와 게스트 파트너가 모두 필요합니다. 호스트 파트너는 규약을 구성하는 조직을 나타냅니다. |
    | 호스트 ID | 호스트 파트너에 대한 식별자입니다. |
    | 게스트 파트너 | 규약에는 호스트와 게스트 파트너가 모두 필요합니다. 게스트 파트너는 호스트 파트너와 함께 일하는 조직을 나타냅니다. |
    | 게스트 ID | 게스트 파트너에 대한 식별자입니다. |
    | 수신 설정 | 규약을 통해 받는 모든 메시지에 적용되는 속성입니다. |
    | 송신 설정 | 규약을 통해 보낸 모든 메시지에 적용 되는 속성입니다. |

7. 이 규약을 통해 받은 메시지를 처리하는 방법을 구성하려면 다음을 수행합니다.

 a. **설정 받기**를 선택합니다.

 b. 필요에 따라 **메시지 속성 재정의** 확인란을 선택하여 들어오는 메시지의 속성을 재정의할 수 있습니다.

 c. 들어오는 모든 메시지를 서명하도록 요구하려면 **메시지 서명 필요** 확인란을 선택합니다. 이 옵션을 선택하면 **인증서** 목록에서 **게스트 파트너 공용 인증서**를 선택하여 메시지 서명의 유효성을 검사합니다.

 d. 들어오는 모든 메시지를 암호화하도록 요구하려면 **메시지 암호화 필요** 확인란을 선택합니다. 이 옵션을 선택하면 **인증서** 목록에서 **호스트 파트너 개인 인증서**를 선택하여 들어오는 메시지의 암호를 해독합니다.

 e. 메시지를 압축하도록 요구하려면 **메시지 압축 필요** 확인란을 선택합니다.    

 f. 받은 메시지에 대한 동기 MDN(메시지 처리 알림)을 보내려면 **MDN 보내기** 확인란을 선택합니다.

 g. 받은 메시지에 대한 서명된 MDN을 보내려면 **서명된 MDN 보내기** 확인란을 선택합니다.

 h. 받은 메시지에 대한 비동기 MDN을 보내려면 **비동기 MDN 보내기** 확인란을 선택합니다.

 !["설정 받기" 속성 설정](./media/logic-apps-enterprise-integration-agreements/agreement-4.png)  

 다음 표에서는 **설정 받기** 속성에 대해 설명합니다.  

 | 속성 | 설명 |
 | --- | --- |
 | 메시지 속성 재정의 | 받은 메시지의 속성을 재정의할 수 있음을 나타냅니다. |
 | 메시지 서명 필요 | 메시지를 디지털로 서명하도록 요구합니다. 서명 확인을 위한 게스트 파트너 공용 인증서를 구성합니다.  |
 | 메시지 암호화 필요 | 메시지를 암호화하도록 요구합니다. 암호화되지 않은 메시지는 거부됩니다. 메시지 암호를 해독하기 위한 호스트 파트너 개인 인증서를 구성합니다.  |
 | 메시지 압축 필요 | 메시지를 압축하도록 요구합니다. 압축되지 않은 메시지는 거부됩니다. |
 | MDN 텍스트 | 메시지 보낸 사람에게 보낼 기본 MDN(메시지 처리 알림)입니다. |
 | MDN 전송 | MDN을 보내도록 요구합니다. |
 | 서명된 MDN 전송 | MDN을 서명하도록 요구합니다. |
 | MIC 알고리즘 | |
 | 비동기 MDN 전송 | 메시지를 비동기적으로 보내도록 요구합니다. |
 | URL | MDN을 보낼 URL입니다. |

8. 이 규약을 통해 보낸 메시지를 처리하는 방법을 구성하려면 다음을 수행합니다.

 a. **설정 보내기**를 선택합니다.  

 b. 파트너에게 서명된 메시지를 보내려면 **메시지 서명 사용** 확인란을 선택합니다. 이 옵션을 선택하면 **MIC 알고리즘** 목록과 **인증서** 목록에서 각각 **호스트 파트너 개인 인증서 MIC 알고리즘** 및 **호스트 파트너 개인 인증서**를 선택하여 메시지를 서명합니다.

 c. 파트너에게 암호화된 메시지를 보내려면 ** 메시지 암호화 사용 ** 확인란을 선택합니다. 이 옵션을 선택하면 **암호화 알고리즘** 목록과 **인증서** 목록에서 각각 **게스트 파트너 공용 인증서 알고리즘** 및 **게스트 파트너 공용 인증서**를 선택하여 메시지를 암호화합니다.

 d. 메시지를 압축하려면 **메시지 압축 사용** 확인란을 선택합니다.

 e. HTTP content-type 헤더를 한 줄로 펼치려면 **HTTP 헤더 펼침** 확인란을 선택합니다.

 f. 보낸 메시지에 대한 동기 MDN을 받으려면 **MDN 요청** 확인란을 선택합니다.

 g. 보낸 메시지에 대한 서명된 MDN을 받으려면 **서명된 MDN 요청** 확인란을 선택합니다.

 h. 보낸 메시지에 대한 비동기 MDN을 받으려면 **비동기 MDN 요청** 확인란을 선택합니다. 이 옵션을 선택하면 MDN을 보낼 URL을 입력합니다.  

 i. NRR(수신 거부 없음)을 요구하려면 **NRR 사용** 확인란을 선택합니다.

 j. **확인**을 선택합니다.

 !["설정 보내기" 속성 설정](./media/logic-apps-enterprise-integration-agreements/agreement-5.png)  

 다음 표에서는 **설정 보내기** 속성에 대해 설명합니다.  

 | 속성 | 설명 |
 | --- | --- |
 | 메시지 서명 사용 | 규약에서 보낸 모든 메시지를 서명하도록 요구합니다. |
 | MIC 알고리즘 | 메시지에 서명하는 데 사용할 알고리즘입니다. 메시지에 서명하기 위한 호스트 파트너 개인 인증서 MIC 알고리즘을 구성합니다. |
 | 인증서 | 메시지에 서명하는 데 사용할 인증서입니다. 메시지에 서명하기 위한 호스트 파트너 개인 인증서를 구성합니다. |
 | 메시지 암호화 사용 | 이 규약에서 보낸 모든 메시지를 암호화하도록 요구합니다. 메시지를 암호화하기 위한 게스트 파트너 공용 인증서 알고리즘을 구성합니다. |
 | 암호화 알고리즘 | 메시지 암호화에 사용할 암호화 알고리즘입니다. 메시지를 암호화하기 위한 게스트 파트너 공용 인증서를 구성합니다. |
 | 인증서 | 메시지를 암호화하는 데 사용할 인증서입니다. 메시지를 암호화하기 위한 게스트 파트너 개인 인증서를 구성합니다. |
 | 메시지 압축 사용 | 이 규약에서 보낸 모든 메시지를 압축하도록 요구합니다. |
 | HTTP 헤더 펼침 | HTTP content-type 헤더를 한 줄에 배치합니다. |
 | MDN 요청 | 이 규약에서 보낸 모든 메시지에 대한 MDN을 요구합니다. |
 | 서명된 MDN 요청 | 이 규약으로 보내는 모든 MDN을 서명하도록 요구합니다. |
 | 비동기 MDN 요청 | 이 규약으로 비동기 MDN을 보내도록 요구합니다. |
 | URL | MDN을 보낼 URL입니다. |
 | NRR 사용 | 지정된 대로 데이터를 받았다는 증명 정보를 제공하는 통신 특성인 NRR(수신 거부 없음)을 요구합니다. |
