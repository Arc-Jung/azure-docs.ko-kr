# <a name="firewall-tables-overview"></a>방화벽 테이블 개요

방화벽 테이블 사설 클라우드 리소스 간에 네트워크 트래픽을 필터링 하는 규칙을 나열 합니다. VLAN 또는 서브넷에 적용할 수 있습니다. 규칙에는 다음 원본 네트워크 또는 IP 주소 및 대상 네트워크 또는 IP 주소 간의 네트워크 트래픽을 제어합니다.

## <a name="firewall-rules"></a>방화벽 규칙

다음 표에서 방화벽 규칙의 매개 변수를 설명 합니다.

| 자산 | 세부 정보 |
| ---------| --------|
| **Name** | 방화벽 규칙 및 용도 고유 하 게 식별 하는 이름입니다. |
| **우선 순위** | 100 ~ 100이 가장 높은 우선 순위를 사용 하 여 4096 사이의 숫자입니다. 규칙은 우선 순위에 따라 처리됩니다. 일치 하는 규칙에서 트래픽을 상태가 되 면 규칙 처리가 중지 됩니다. 결과적으로, 우선 순위가 낮은 우선 순위가 더 높은 규칙과 동일한 특성을 포함 하는 존재 하는 모든 규칙은 처리 되지 않습니다.  충돌 하는 규칙을 발생 하지 않도록 주의 합니다. |
| **상태 추적** | 상태 비저장 (사설 클라우드, 인터넷 또는 VPN) 추적 수 또는 상태 저장 (공용 IP).  |
| **프로토콜** | 옵션에는 모든 TCP 또는 UDP 포함 됩니다. ICMP가 필요한 경우 사용 합니다. |
| **방향** | 규칙이 인바운드 또는 아웃바운드 트래픽에 적용되는지 여부입니다. |
| **작업** | 허용 또는 거부 규칙에 정의 된 트래픽 유형에 대 한 합니다. |
| **원본** | IP 주소, classless inter-domain routing (CIDR) 블록 (예: 10.0.0.0/24), 또는 Any  범위를 지정 하 서비스 태그 또는 응용 프로그램 보안 그룹을 사용 하면 더 적은 보안 규칙을 만들 수 있습니다. |
| **원본 포트** | 트래픽은 네트워크에서 발생 하는 포트입니다.  개별 포트 또는 포트 443에서 8000 8080 등의 범위를 지정할 수 있습니다. 범위를 지정하면 더 적은 보안 규칙을 만들어도 됩니다. |
| **대상** | IP 주소, classless inter-domain routing (CIDR) 블록 (예: 10.0.0.0/24), 또는 Any  범위를 지정 하 서비스 태그 또는 응용 프로그램 보안 그룹을 사용 하면 더 적은 보안 규칙을 만들 수 있습니다.  |
| **대상 포트** | 네트워크 트래픽을 이동 하는 포트입니다.  개별 포트 또는 포트 443에서 8000 8080 등의 범위를 지정할 수 있습니다. 범위를 지정하면 더 적은 보안 규칙을 만들어도 됩니다.|

### <a name="stateless"></a>상태 비저장

규칙을 상태 비저장 개별 패킷의 살펴봅니다 및 규칙을 기준으로 필터링 합니다.  
추가 규칙을 반대 방향으로 트래픽 흐름에 대해 필요할 수 있습니다.  다음 사항 간의 트래픽에 대 한 상태 비저장 규칙을 사용 합니다.

* 사설 클라우드의 서브넷
* 온-프레미스 서브넷과 사설 클라우드 서브넷
* 사설 클라우드에서 인터넷 트래픽

### <a name="stateful"></a>상태 저장

 상태 저장 규칙은 통과 하는 연결을 인식 합니다. 기존 연결에 대한 흐름 레코드가 만들어집니다. 통신은 흐름 레코드의 연결 상태에 따라 허용 또는 거부됩니다.  인터넷에서 트래픽 필터링 하도록 공용 IP 주소에 대 한이 규칙 유형을 사용 합니다.
