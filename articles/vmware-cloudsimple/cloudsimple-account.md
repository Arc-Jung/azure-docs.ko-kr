# <a name="account-management-overview"></a>계정 관리 개요

CloudSimple 서비스를 만들면 CloudSimple에서 계정을 만듭니다.  계정은 Azure 구독에 연결 된 서비스가 위치한입니다.  사용 하 여 모든 사용자 **소유자** 하 고 **참가자** CloudSimple 포털에 액세스할 수 있는 구독에는 역할입니다.  Azure 구독 ID 및 CloudSimple 서비스와 연결 된 테 넌 트 ID에서 검색 되는 [계정 페이지](https://docs.azure.cloudsimple.com/account/)합니다.

## <a name="additional-alert-emails"></a>경고 전자 메일 추가

경고를 받을 CloudSimple에서 전자 메일 Id를 구성할 수 있습니다.

* 서비스와 관련 된
* 자동 처리

## <a name="cloudsimple-operator-access"></a>CloudSimple 연산자 액세스

서비스 운영 담당자에 대 한 CloudSimple 포털에 대 한 액세스를 제어할 수 있습니다.  서비스 운영 담당자 지원 티켓을 제출할 때에 포털에 로그인 합니다.  서비스 작업은 보고 된 문제를 해결 되며 수행 된 작업 감사 로그에 사용할 수 있습니다.

## <a name="users"></a>사용자

권한이 있는 모든 사용자 **소유자** 하 고 **참가자** 구독의 역할에에서 CloudSimple 포털에 액세스할 수 있습니다.  포털에 액세스 하는 경우 사용자 CloudSimple 계정에 만들어집니다.  계정 페이지에서 특정 사용자에 대 한 CloudSimple 포털에 대 한 액세스를 비활성화할 수 있습니다.
