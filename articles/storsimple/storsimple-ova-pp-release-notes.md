# StorSimple 가상 배열 릴리스 정보(미리 보기) 

## 개요

다음 릴리스 정보는 Microsoft Azure StorSimple 가상 배열(StorSimple 온-프레미스 가상 장치 또는 StorSimple 가상 장치라고도 함) 2015년 12월 공개 미리 보기 릴리스에 대한 중대한 미해결 문제를 확인합니다.

릴리스 정보는 계속 업데이트되며, 해결 방법이 필요한 중대한 문제가 발견되면 해당 내용이 추가됩니다. StorSimple 가상 장치를 배포하기 전에 릴리스 정보에 포함된 정보를 주의 깊게 검토하세요.

>[AZURE.IMPORTANT]
>
>- StorSimple 가상 배열은 미기 보기 중이며 평가 및 배포 계획 용도로 사용됩니다. 프로덕션 환경에서는 이 미리 보기를 설치하도록 지원되지 않습니다. 
>
>- StorSimple 가상 배열에서 문제가 발생하는 경우 [StorSimple MSDN 포럼](https://social.msdn.microsoft.com/Forums/home?forum=StorSimple)에 문제를 게시해 주세요.

다음 표에서 이 릴리스의 알려진 문제를 간략하게 설명합니다.

| 번호| 기능 | 문제 | 해결 방법/설명 |
|----|---------|-------|---------------------|                                                                                                                                                                                                                                                      
| **1.** | 업데이트 | 미리 보기 릴리스에서 만든 가상 장치는 지원되는 일반 사용 가능 버전으로 업데이트될 수 없습니다.| 이러한 가상 장치는 재해 복구(DR) 워크플로를 사용하여 일반 사용 가능 릴리스로 장애 조치되어야 합니다.|
| **2.** | 계층화된 볼륨 또는 공유 | StorSimple 계층화된 볼륨에서 응용 프로그램에 대한 바이트 범위 잠금은 지원되지 않습니다. 바이트 범위 잠금을 사용하도록 설정하면 StorSimple 계층화가 실행되지 않습니다. | 권고 조치:<ul><li>응용 프로그램 논리에서 바이트 범위 잠금을 해제합니다.</li><li> 이 응용 프로그램에 대한 데이터를 계층화된 볼륨이 아닌 로컬로 고정된 볼륨에 넣도록 선택합니다.</li>*경고*: 로컬로 고정된 볼륨 사용 및 바이트 범위 잠금이 활성화되면 로컬로 고정된 볼륨은 복원이 완료되기 전이라도 온라인 상태가 될 수 있으니 주의해야 합니다. 이러한 경우, 복원이 진행 중이면, 복원이 완료될 때까지 반드시 기다려야 합니다.|
| **3.** | 계층화된 공유 | 큰 파일로 작업하면 계층화가 매우 느려질 수 있습니다. | 큰 파일로 작업하는 경우에는 제일 큰 파일이 공유 크기의 3% 미만이 되도록 해야 합니다.|
| **4.** | 로컬로 고정된 볼륨 또는 공유 | 로컬로 고정된 볼륨 또는 공유의 사용 가능한 용량은 프로비전된 용량의 85-90%입니다. 10-15% 범위는 메타데이터를 위해 따로 유지됩니다. | 원하는 만큼 사용 가능한 용량을 확보하려면 그보다 큰 볼륨을 프로비전해야 합니다. 예를 들어, 1TB 사용 가능한 용량을 위해서는 1.15TB 볼륨을 프로비전해야 합니다.|
| **5.** | 로컬 웹 UI | Internet Explorer 보안 강화 기능(IE ESC)을 사용하도록 설정하면, 일부 로컬 웹 UI 페이지(예: **문제 해결** 또는 **유지 관리** )가 제대로 작동하지 않을 수 있습니다. 해당 페이지의 단추도 작동하지 않을 수 있습니다. | Internet Explorer의 보안 강화 기능을 해제하십시오.|
| **6.** | 로컬 웹 UI | 이번 릴리스에서 로컬 웹 UI에 대한 지역화는 지원되지 않습니다. | 이후 릴리스에서 구현될 예정입니다.|
| **7.** | 로컬 웹 UI | Hyper-V 가상 컴퓨터에서 웹 UI의 네트워크 인터페이스는 10Gbps 인터페이스로 표시됩니다. | 이러한 동작은 Hyper-V를 반영합니다. Hyper-V는 가상 네트워크 어댑터에 10Gbps를 항상 표시합니다.|                                                                                                 
| **8.** | 재해 복구 | 파일 서버에 대한 재해 복구는 원본 장치의 도메인과 같은 도메인에만 수행할 수 있습니다. 다른 도메인의 대상 장치로 재해 복구는 이번 릴리스에서 지원되지 않습니다. | 이후 릴리스에서 구현될 예정입니다.|
| **9.**| 공유에 사용된 용량| 공유에 데이터가 전혀 없어도 공유 사용량이 표시될 수 있습니다. 이것은 공유에 사용된 용량에 메타데이터가 포함되기 때문입니다. | |                                                                                                                                                                                                                                                         
| **10.** | 표준 시간대 설정| 등록된 장치에 대해 표준 시간대가 변경되면, 로컬 웹 UI를 통해 실행되는 진단 테스트에 변경 내용이 반영되지 않습니다. | 이 문제는 향후 릴리스에서 해결될 것입니다.|
| **11.** | 네트워크 설정 | 고정 IP 주소가 웹 UI를 통해 네트워크 인터페이스에 대해 구성되면 IP 변경이 발생합니다. 하지만, DNS 서버 IP도 IPv6 사이트 로컬 주소로 수정됩니다. | 이 문제는 향후 릴리스에서 해결될 것입니다.|
| **12.** | Azure PowerShell | 이번 릴리스에서는 Azure PowerShell을 통해 StorSimple 가상 장치를 관리할 수 없습니다. | 가상 장치에 대한 모든 관리는 Azure 클래식 포털 및 로컬 웹 UI를 통해 수행됩니다.|
| **13.** | 작업 | 작업 진행률은 세부적이지 않습니다. 작업 진행률은 0에서 100%로 바로 급증할 수 있습니다. | 이것은 향후 릴리스에서 해결될 것입니다.|
| **14.** | 프로비전된 데이터 디스크 | 특정한 크기의 데이터 디스크를 프로비전하고 해당 StorSimple 가상 장치를 만들고 나면, 데이터 크기를 확장하거나 축소하지 말아야 합니다. 이런 작업을 시도하면 장치의 로컬 계층에 있는 모든 데이터가 손실될 수 있습니다. | |
| **15.** | 도움말 서랍 | 도움말 서랍 항목은 사용할 수 없습니다.| 모든 설명서는 Microsoft 다운로드 센터와 Azure 설명서 사이트를 통해 제공됩니다. |
