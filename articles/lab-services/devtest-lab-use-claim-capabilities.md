# <a name="use-claim-capabilities-in-azure-devtest-labs"></a>Azure DevTest 랩에서 클레임 기능 사용
Azure DevTest Labs 서비스는 개발자와 테스터의 효율성과 효율성을 향상시킵니다. 이 문서에서는 Azure DevTest Labs에서 가상 컴퓨터를 클레임하거나 클레임 해제하는 기능에 중점을 둡니다. 또한 이 기능은 사용자 환경을 개선하는 다양한 방법을 나열합니다. 이 기능이 사용될 수 있는 다양한 시나리오를 살펴보기 전에 **클레임이** 무엇이고 어떻게 작동하는지 살펴보겠습니다.

## <a name="claimable-machines"></a>클레임 가능한 컴퓨터
클레임 가능한 컴퓨터는 소유자가 없는 랩에서 만든 VM(가상 컴퓨터)입니다. 컴퓨터가 클레임되면 사용자는 해당 VM에 대한 모든 옵션을 사용할 수 있습니다. 사용자가 컴퓨터를 클레임하면 몇 가지 변경 사항이 만들어집니다. VM은 **청구 가능한 가상 시스템** 목록에서 Azure 포털의 내 가상 **컴퓨터** 목록으로 이동됩니다. 

사용자는 VM에 연결하여 아티팩트를 사용자 지정하거나, 컴퓨터를 다시 시작, 중지 또는 클레임 해제할 수 있습니다. VM을 청구할 수 있도록 하는 방법에는 몇 가지가 있습니다.

- 컴퓨터를 만들고 클레임 가능한 풀로 이동하도록 클레임 을 해제합니다. 
- [고급 설정을](https://azure.microsoft.com/updates/azure-devtest-labs-claim-lab-vms-from-a-shared-pool/)사용하여 VM을 만들고 공유 풀에 배치합니다.

클레임/청구 취소 기능을 효과적으로 사용할 수 있는 두 가지 경우가 있습니다. 첫 번째 경우는 더 많은 사전 생각과 계획이 필요하며 제대로 설계되고 실행되어야 합니다. 그리고, 두 번째는 더 상황입니다. 다음은 다른 사례의 몇 가지 예입니다.

## <a name="designed-use-of-claimable-machines"></a>청구 가능한 기계의 설계 사용

- **소프트웨어 개발/테스트:** 개발자 나 테스터는 구성 된 컴퓨터를 준비 하 고 청구 되지 않은 상태로 함으로써 생산성을 높일 수 있습니다. 다양한 구성, 필요한 도구 및 최신 코드를 사용하여 VM 집합을 사용하면 컴퓨터를 설정하는 데 시간을 할애하지 않고도 VM을 청구하고 작업을 시작할 수 있습니다. VM을 클레임하기 전에 컴퓨터는 프로비저닝되지만 덜 자주 사용되는 컴퓨터를 사용하는 데 드는 비용을 최소화하는 종료됩니다. VM이 필요한 경우 사용자는 단순히 컴퓨터를 시작하는 VM을 주장합니다. 새 VM을 만드는 것이 더 쉽고 저렴하기 때문에 클레임 해제 옵션은 이 경우 유용하지 않습니다.
- **교실/실습실:** 학생이 Azure 포털을 사용하여 컴퓨터에 즉시 연결할 수 있도록 클래스 또는 랩에 대해 VM을 미리 구성해야 합니다.  학생이 VM을 주장하면 랩은 아무도 동일한 컴퓨터를 주장할 수 없도록 합니다. 이 프로세스를 자동화하면 지정된 환경이 있는 필요한 수의 컴퓨터를 사용할 수 있습니다. 학생이 나타나지 않거나 늦게 실행되는 경우 최소한의 비용으로 세션이 끝날 때까지 청구되지 않은 컴퓨터를 계속 사용할 수 있습니다. VM이 이전 사용자가 완료될 때 알 수 없는 상태이기 때문에 클레임 해제 옵션은 이 시나리오에서 효과적이지 않습니다.
- **데모:** 랩의 컴퓨터가 특정 환경으로 설정된 데모에 컴퓨터를 사용합니다. 이 기능은 여러 사람이 동시에 또는 회의와 같은 임의의 시간에 데모를 제공할 수 있는 경우에 유용합니다. 데모가 컴퓨터의 상태를 변경해서는 안 되므로 사용자가 다음 데모를 위해 VM을 클레임 풀로 되돌릴 수 있으므로 클레임 해제 옵션이 유용할 수 있습니다. 청구되지 않은 컴퓨터가 프로비저닝해제되고 최소한의 비용이 발생하므로 VM은 더 오랜 기간 동안 랩에 남아 있을 수 있습니다.
- **임시/계약 근로자:** 사용자가 컴퓨터를 사용하도록 허용합니다. 떠날 때 데이터 손실 없이 VM을 클레임 풀로 반환합니다. VM을 청구하지 않은 경우 다른 사용자가 VM을 클레임하고 컴퓨터를 계속 하거나 추가 정보를 검토할 수 있습니다.
- **일반적으로:** 특정 종지에서 전용 원본을 자동으로 구성하고 배포하는 기능은 다양한 상황에서 유용합니다. 클레임/클레임 해제 기능을 사용하면 설정된 구성을 사용하여 클레임되지 않은 상태에서 VM을 빌드하는 자동화된 프로세스를 통해 사용자가 효율성을 높이는 데 도움이 되는 여러 가지 상황이 있습니다. 구성에는 사용자의 필요에 따라 다른 운영 체제, 언어, 디스크 또는 [기타 소프트웨어(아티팩트)가](devtest-lab-artifact-author.md) 포함될 수 있습니다. 랩에서 VM을 클레임할 수 있으므로 랩 사용자는 컴퓨터를 구성하는 데 시간과 노력을 들이지 않고도 제대로 구성된 시스템을 얻을 수 있습니다. 랩 관리자는 VM의 클레임 상태를 사용하여 생성된 컴퓨터 수를 개선하고, 컴퓨터를 정리하고, 구성의 우선 순위를 결정할 수 있습니다. [이미지 팩터리는](image-factory-create.md) 여러 랩에 대한 VM 및 이미지를 빌드하는 자동화된 프로세스의 좋은 예입니다. 스크립트는 적절한 변경사항으로 다음 상황을 실행하도록 수정하거나 사용자 지정 시스템을 만들기 위한 참조로 사용할 수 있습니다.

## <a name="situational-use-of-claimable-machines"></a>청구 가능한 기계의 상황적 사용

- 사용자가 컴퓨터의 제어를 한 컴퓨터에서 다른 컴퓨터로 전달할 수 있도록 허용하는 클레임/청구 취소 기능을 사용하여 다음에 누가 컴퓨터를 선택할지 명시적으로 알 필요가 없습니다.
- 특정 컴퓨터 구성이 버그를 재현할 수 있는 시나리오의 개발, 테스트 및 디버깅을 통해 다른 개발자가 컴퓨터를 클레임하고 작업을 계속할 수 있도록 컴퓨터를 청구 해제할 수 있습니다. 이 기능은 더 많은 사람들이 세계의 다른 지역에서 원격으로 작업할 때 특히 유용합니다. 
- 팀 멤버는 단일 환경에서 작업할 수 있습니다. 예를 들어 자동화할 수 없는 복잡한 환경을 수동으로 설정하거나 이미지와 같은 단일 입력에 대해서만 수정을 처리할 수 있는 리소스를 만들 수 있습니다. 과거에는 전용 컴퓨터를 가동하고 실행하여 이 문제를 처리했습니다. 클레임 가능한 기능은 기본 제공 사용자 액세스 제어 및 사용 가능한 경우 시각적 식별을 통해 수동 프로세스에 대한 개선입니다. 클레임되지 않은 경우 VM은 비용을 줄이기 위해 프로비저닝해제됩니다.
- VM에 연결된 데이터 디스크가 있습니다. 최대 1TB의 데이터를 사용하여 각 디스크를 복사하거나 복제할 필요 없이 많은 양의 데이터를 전달할 수 있습니다. VM은 처음에 많은 양의 데이터를 가진 연결된 디스크로 만들어집니다.  그러면 모든 사용자가 컴퓨터를 클레임하고 데이터에 액세스할 수 있습니다. 완료되면 VM을 해제하여 다른 사용자가 컴퓨터에 허용하도록 허용합니다.

청구 가능한 기계를 사용하는 데는 몇 가지 주의 사항이 있으며, 가장 일반적으로 기계에 대한 액세스 권한을 얻는 데 사용됩니다. 컴퓨터가 도메인에 가입되어 있는 경우 컴퓨터의 클레임 사용자는 이미 액세스 권한이 부여되어 있어야 하며, 일반적으로 VM이 생성될 때 랩의 모든 사용자를 포함하는 그룹에 대한 액세스 권한을 부여하여 수행됩니다. 컴퓨터가 도메인에 가입되어 있지 않은 경우 공용 리포지토리의 **VM 암호 재설정** 아티팩트를 실행하여 사용자를 관리자로 추가해야 합니다.  아티팩트는 기기를 시작하거나 클레임한 후에도 적용할 수 있습니다.
