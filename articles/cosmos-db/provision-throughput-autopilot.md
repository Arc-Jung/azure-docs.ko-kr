# <a name="create-azure-cosmos-containers-and-databases-in-autopilot-mode-preview"></a>Autopilot 모드(미리 보기)에서 Azure Cosmos 컨테이너 및 데이터베이스 만들기

Azure Cosmos DB를 사용하여 수동 또는 Autopilot 모드로 컨테이너의 처리량을 프로비전할 수 있습니다. 이 문서에서는 Autopilot 모드의 이점과 사용 사례에 대해 설명합니다.

> [!NOTE]
> 자동 조종 모드는 현재 공개 미리 보기에서 사용할 수 있습니다. 새 [데이터베이스 및 컨테이너에](#create-a-database-or-a-container-with-autopilot-mode) 대해서만 자동 조종 장치를 사용하도록 설정할 수 있습니다. 기존 컨테이너 및 데이터베이스에는 사용할 수 없습니다.

처리량의 수동 프로비저닝 외에도 이제 자동 조종 모드에서 Azure Cosmos 컨테이너를 구성할 수 있습니다. 자동 조종 모드로 구성된 컨테이너 및 데이터베이스는 **전 세계적으로 워크로드의 가용성, 대기 시간, 처리량 또는 성능에 영향을 주지 않고 애플리케이션 요구에 따라 프로비저닝된 처리량을 자동으로 즉시 조정합니다.**

자동 조종 모드로 컨테이너 및 데이터베이스를 구성할 때 초과할 `Tmax` 최대 처리량을 지정해야 합니다. 그런 다음 컨테이너는 처리량을 `0.1*Tmax < T < Tmax`조정하여 . 즉, 컨테이너와 데이터베이스는 구성한 최대 처리량 값의 최저 10%에서 구성된 최대 처리량 값까지 워크로드 요구 사항에 따라 즉시 확장됩니다. 언제든지 자동 조종 데이터베이스`Tmax`또는 컨테이너에서 최대 처리량() 설정을 변경할 수 있습니다. 자동 조종 장치 옵션을 사용하면 컨테이너 또는 데이터베이스당 400RU/s의 최소 처리량을 더 이상 적용할 수 없습니다.

자동 조종 장치를 미리 볼 때 컨테이너 또는 데이터베이스에서 지정된 최대 처리량에 대해 시스템은 계산된 저장소 제한 내에서 작동할 수 있습니다. 저장소 제한을 초과하면 최대 처리량이 자동으로 더 높은 값으로 조정됩니다. 자동 조종 모드로 데이터베이스 수준 처리량을 사용하는 경우 데이터베이스 내에서 허용되는 `0.001*TMax`컨테이너 수는 다음과 같이 계산됩니다. 예를 들어 20,000개의 자동 조종 장치 RU/s를 프로비전하는 경우 데이터베이스에 20개의 컨테이너가 있을 수 있습니다.

## <a name="benefits-of-autopilot-mode"></a>오토파일럿 모드의 장점

자동 조종 모드로 구성된 Azure Cosmos 컨테이너에는 다음과 같은 이점이 있습니다.

* **간단한 :** 자동 조종 모드의 컨테이너는 다양한 컨테이너에 대해 프로비저닝된 처리량(R)과 용량을 수동으로 관리하는 복잡성을 제거합니다.

* **확장 가능:** 오토파일럿 모드의 컨테이너는 필요에 따라 프로비저닝된 처리량 용량을 원활하게 조정합니다. 클라이언트 연결, 응용 프로그램에 는 중단이 없으며 기존 SLA에 영향을 주지 않습니다.

* **비용 효율적인:** 자동 조종 모드로 구성된 컨테이너를 사용하는 경우 워크로드에 필요한 리소스에 대해서만 시간당 비용을 지불합니다.

* **고가용성:** 오토파일럿 모드의 컨테이너는 데이터 내구성과 고가용성을 보장하기 위해 전 세계적으로 분산된 내결함성 백엔드와 동일하게 사용됩니다.

## <a name="use-cases-of-autopilot-mode"></a>오토파일럿 모드의 사용 사례

자동 조종 모드로 구성된 Azure Cosmos 컨테이너의 사용 사례는 다음과 같습니다.

* **가변 워크로드:** 1시간에서 몇 시간, 매일 몇 시간 또는 1년에 여러 번 사용하는 가볍게 사용되는 응용 프로그램을 실행하는 경우. 예를 들어 인적 자원, 예산 책정 및 운영 보고를 위한 응용 프로그램이 있습니다. 이러한 시나리오의 경우 자동 조종 모드로 구성된 컨테이너를 사용할 수 있으며 더 이상 피크 또는 평균 용량에 대해 수동으로 프로비전할 필요가 없습니다.

* **예측할 수 없는 워크로드:** 하루 종일 데이터베이스 사용량이 있지만 예측하기 어려운 최대 작업인 워크로드를 실행하는 경우 예를 들어 일기 예보가 변경될 때 활동이 급증하는 트래픽 사이트가 포함됩니다. 자동 조종 모드로 구성된 컨테이너는 응용 프로그램의 최대 부하 요구를 충족하도록 용량을 조정하고 활동의 급증이 끝나면 다시 축소합니다.

* **새로운 응용 프로그램:** 새 응용 프로그램을 배포하는 데 필요한 프로비저닝 처리량(예: 필요한 R)의 양에 대해 잘 모르는 경우 자동 조종 모드로 구성된 컨테이너를 사용하면 애플리케이션의 용량 요구 사항과 요구 사항에 맞게 자동으로 확장할 수 있습니다.

* **자주 사용되지 않는 응용 프로그램:** 볼륨이 낮은 응용 프로그램/ 웹/블로그 사이트와 같이 하루 또는 주 또는 한 달에 몇 시간 동안만 사용되는 응용 프로그램이 있는 경우

* **데이터베이스 개발 및 테스트:** 작업 시간 동안 컨테이너를 사용하는 개발자가 있지만 밤이나 주말에는 컨테이너가 필요하지 않은 경우. 자동 조종 모드로 구성된 컨테이너를 사용하면 사용하지 않을 때 최소로 축소됩니다.

* **예약된 프로덕션 워크로드/쿼리:** 단일 컨테이너에 일련의 예약된 요청/작업/쿼리가 있고 절대 낮은 처리량으로 실행하려는 유휴 기간이 있는 경우 이제 쉽게 실행할 수 있습니다. 예약된 쿼리/요청이 자동 조종 모드로 구성된 컨테이너에 제출되면 필요에 따라 자동으로 확장되고 작업을 실행합니다.

이전 문제에 대한 해결 에는 구현에 엄청난 시간이 필요할 뿐만 아니라 구성이나 코드가 복잡해지며 이를 해결하기 위해 수동 작업이 자주 필요합니다. 자동 조종 모드는 이러한 문제에 대해 더 이상 걱정할 필요가 없도록 위의 시나리오를 즉시 사용할 수 있습니다.

## <a name="comparison--containers-configured-in-manual-mode-vs-autopilot-mode"></a>비교 – 수동 모드와 자동 조종 모드로 구성된 컨테이너

|  | 수동 모드로 구성된 컨테이너  | 자동 조종 모드로 구성된 컨테이너 |
|---------|---------|---------|
| **프로비저닝된 처리량** | 수동으로 프로비전되었습니다. | 워크로드 사용 패턴에 따라 자동으로 즉각적으로 확장됩니다. |
| **요청/작업의 속도 제한 (429)**  | 소비가 프로비저닝 된 용량을 초과하는 경우 발생할 수 있습니다. | 사용한 처리량이 자동 조종 모드로 선택한 최대 처리량 내에 있으면 발생하지 않습니다.   |
| **용량 계획** |  필요한 처리량의 초기 용량 계획 및 프로비저닝을 수행해야 합니다. |    용량 계획에 대해 걱정할 필요가 없습니다. 시스템은 자동으로 용량 계획 및 용량 관리를 처리합니다. |
| **가격 책정** | 시간당 RU/s를 수동으로 프로비전했습니다. | 단일 쓰기 지역 계정의 경우 시간당 자동 조종 장치 RU/s를 사용하여 시간단위로 사용한 처리량에 대해 비용을 지불합니다. <br/><br/>쓰기 영역이 여러 개인 계정의 경우 자동 조종 장치에 대한 추가 요금이 부과되지 않습니다. 시간당 동일한 다중 마스터 RU/s를 사용하여 시간단위로 사용되는 처리량에 대해 비용을 지불합니다. |
| **워크로드 유형에 가장 적합** |  예측 가능하고 안정적인 워크로드|   예측할 수 없는 가변 워크로드  |

## <a name="create-a-database-or-a-container-with-autopilot-mode"></a>자동 조종 모드로 데이터베이스 또는 컨테이너 만들기

Azure 포털을 통해 새 데이터베이스 또는 컨테이너를 만들 때 새 데이터베이스 또는 컨테이너에 대해 자동 조종 장치를 구성할 수 있습니다. 다음 단계를 사용하여 새 데이터베이스 또는 컨테이너를 만들고, 자동 조종 장치를 사용하도록 설정하고, 최대 처리량(RU/s)을 지정합니다.

1. [Azure 포털](https://portal.azure.com) 또는 Azure [Cosmos DB 탐색기에서 로그인합니다.](https://cosmos.azure.com/)

1. Azure Cosmos DB 계정으로 이동하여 **데이터 탐색기** 탭을 엽니다.

1. **새 컨테이너를 선택합니다.** 데이터베이스, 컨테이너 및 파티션 키의 이름을 입력합니다. **처리량**에서 **자동 조종 장치** 옵션을 선택하고 자동 조종 장치 옵션을 사용할 때 데이터베이스 또는 컨테이너가 초과할 수 없는 최대 처리량(RU/s)을 선택합니다.

   ![컨테이너 만들기 및 자동 조종 장치 처리량 구성](./media/provision-throughput-autopilot/create-container-autopilot-mode.png)

1. **확인**을 선택합니다.

**데이터베이스 처리량 프로비저닝** 옵션을 선택하여 자동 조종 모드를 사용하여 공유 처리량 데이터베이스를 만들 수 있습니다.

## <a name="throughput-and-storage-limits-for-autopilot"></a><a id="autopilot-limits"></a>오토파일럿의 처리량 및 스토리지 제한

다음 표에서는 자동 조종 모드의 다양한 옵션에 대한 최대 값 및 저장소 제한을 보여 주며 다음과 같은 옵션을 보여 주며 다음과 같은 옵션을 보여 주실 수 있습니다.

|최대 처리량 제한  |최대 스토리지 제한  |
|---------|---------|
|4000 RU / s  |   50GB    |
|20,000 RU/s  |  200GB  |
|100,000 RU/s    |  1TB   |
|500,000 RU/s    |  5TB  |
