# <a name="use-cases-for-built-in-analytics-with-azure-cosmos-db"></a>Azure Cosmos DB를 사용하여 기본 제공 분석을 위한 사용 사례

Azure Cosmos DB의 아파치 스파크와 함께 기본 제공 분석을 사용하여 다음 사용 사례를 달성할 수 있습니다.

## <a name="htap-scenarios"></a>HTAP 시나리오

Azure Cosmos DB의 기본 제공 분석을 사용하여 다음을 수행할 수 있습니다.

* 트랜잭션 처리 중 사기 를 감지합니다.
* 쇼핑객이 전자 상거래 매장을 탐색할 때 권장 사항을 제공합니다.
* 작업자에게 중요한 제조 장비의 임박한 실패를 경고합니다.
* 운영 데이터에 직접 실시간 분석을 내장하여 빠르고 실행 가능한 통찰력을 창출합니다.

Azure Cosmos DB는 기본적으로 내장된 아파치 스파크를 사용하여 하이브리드 트랜잭션/분석 처리(HTAP) 시나리오를 지원합니다. Azure Cosmos DB는 기존 시스템과 함께 제공되는 운영 및 분석 분리를 제거합니다.

## <a name="globally-distributed-data-lake-without-requiring-any-etl"></a>ETL없이 전 세계적으로 분산된 데이터 레이크

기본적으로 내장된 아파치 스파크를 통해 Azure Cosmos DB는 단일 시스템 이미지를 제공하는 전역분산 데이터 레이크를 구축하는 빠르고 간단하며 확장 가능한 방법을 제공합니다. 전 세계적으로 분산된 멀티 모델 데이터는 값비싼 ETL 파이프라인에 투자할 필요가 없으며 온디맨드 규모로 확장하여 데이터 팀이 데이터 세트를 분석하는 방식에 혁명을 일으키고 있습니다.

## <a name="time-series-analytics-over-historic-data"></a>과거 데이터에 대한 타임계 분석

경우에 따라 과거에 완료된 이벤트에 대한 특정 시점과 같이 데이터를 기반으로 질문에 답해야 할 수도 있습니다. 예를 들어 특정 날짜에 CRM 활동 상태 수를 가져옵니다. 일주일 전에 보고서를 실행한 경우 상태 수는 해당 시점의 각 활동의 상태에 따라 표시됩니다. 오늘 같은 보고서를 실행하면 상태상태가 현재와 같은 활동의 수를 제공 할 것입니다, 이는 지난 주 이후 변경 되었을 수 있습니다, 그들은 오픈에서 닫기자신의 수명 주기를 통해 이동으로. 따라서 사례의 수명 주기의 각 단계에서 스냅숏을 보고해야 합니다.

Azure Cosmos DB를 사용하면 사용자는 시간 이동의 개념을 구현하여 데이터에 대한 분석을 소급하여 쿼리하고 실행하고 데이터가 기록의 특정 시점에서 어떻게 보였는지 물어볼 수 있습니다. 즉, 사용자는 데이터의 현재 보기와 기록 보기를 모두 쉽게 보고 분석을 실행할 수 있습니다.

## <a name="globally-distributed-machine-learning-and-ai"></a>전 세계적으로 분산된 머신 러닝 및 AI

기업이 빠르게 증가하는 데이터 볼륨과 다양한 데이터 유형 및 형식에 맞서고 있는 가운데, 전 세계 페타바이트 규모에서 더 깊고 정확한 통찰력을 얻을 수 있는 능력은 거의 불가능해지고 있습니다. 기본적으로 내장된 아파치 스파크를 갖춘 Azure Cosmos DB는 광범위한 기계 학습 알고리즘 라이브러리를 제공하는 전 세계적으로 분산된 분석 플랫폼을 제공합니다. 대화형 Jupyter 노트북을 사용하여 모델을 빌드하고 학습하고 클러스터 관리 기능을 만들 수 있습니다. 이러한 기능을 통해 주문형으로 고도로 조정되고 자동 탄성 스파크 클러스터를 프로비전할 수 있습니다.

## <a name="deep-learning-on-multi-model-globally-distributed-data"></a>전 세계적으로 분산된 다중 모델에 대한 딥 러닝

딥 러닝은 데이터 볼륨과 복잡성이 계속 증가함에 따라 빅 데이터 예측 분석 솔루션을 제공하는 이상적인 방법입니다. 딥 러닝을 통해 기업은 비정형 및 반정형 데이터의 힘을 활용하여 AI, 자연어 처리 등과 같은 기술을 활용하는 사용 사례를 제공할 수 있습니다.

## <a name="reporting-integrating-with-power-apps-power-bi"></a>보고(전원 앱, Power BI와 통합)

Power BI는 셀프 서비스 비즈니스 인텔리전스 기능을 갖춘 대화형 시각화를 제공하여 최종 사용자가 자체적으로 보고서 및 대시보드를 만들 수 있도록 합니다. 내장 된 스파크 커넥터를 사용 하 여 Azure 코스모스 DB에서 아파치 스파크 클러스터에 전원 BI 데스크톱을 연결할 수 있습니다. 이 커넥터를 사용하면 직접 쿼리를 사용하여 Azure Cosmos DB의 아파치 스파크로 처리를 오프로드할 수 있습니다.

## <a name="iot-analytics-at-global-scale"></a>글로벌 규모의 IoT 분석

증가하는 네트워크 센서에서 생성된 데이터는 이전에 불투명한 시스템 및 프로세스에 대한 전례 없는 가시성을 제공합니다. 핵심은 IoT 장치가 전 세계에 배포되는 위치에 관계없이 이 급류에서 실행 가능한 통찰력을 찾는 것입니다. Azure Cosmos DB를 사용하면 IOT 회사가 전 세계 어디서나 실시간으로 고속 센서 및 시간계 데이터를 분석할 수 있습니다. 이를 통해 상호 연결된 세계의 진정한 가치를 활용하여 향상된 고객 경험, 운영 효율성 및 새로운 수익 기회를 제공할 수 있습니다.

## <a name="stream-processing-and-event-analytics"></a>스트림 처리 및 이벤트 분석 

로그 파일에서 센서 데이터에 이르기까지 기업은 점점 더 데이터의 "스트림"에 대처할 필요성이 증가하고 있습니다. 이 데이터는 여러 소스에서 동시에 안정적인 스트림에 도착합니다. 이러한 데이터 스트림을 디스크에 저장하고 소급하여 분석하는 것이 가능하지만, 데이터가 도착할 때 처리하고 조치를 처리하는 것이 현명하거나 중요할 수 있습니다. 예를 들어 금융 거래와 관련된 데이터 스트림을 실시간으로 처리하여 잠재적으로 사기성 거래를 식별하고 거부할 수 있습니다.

## <a name="interactive-analytics"></a>대화형 분석

미리 정의된 쿼리를 실행하여 판매, 생산성 또는 주가에 대한 정적 대시보드를 만들 수 있을 뿐만 아니라 대화형으로 데이터를 탐색할 수 있습니다. 대화형 분석을 사용하면 질문을 하거나, 결과를 보고, 응답을 기반으로 초기 질문을 변경하거나, 결과를 자세히 파악할 수 있습니다. Azure Cosmos DB의 아파치 스파크는 신속하게 응답하고 적응하여 대화형 쿼리를 지원합니다.

## <a name="data-exploration-using-jupyter-notebooks"></a>Jupyter 노트북을 사용한 데이터 탐색

새 데이터 집합이 있는 경우 실행 중인 모델 및 테스트에 대해 자세히 알아보기 전에 데이터를 검사해야 합니다. 즉, 예비 데이터 분석을 수행해야 합니다. 데이터 탐색은 여러 가지 결정을 내릴 수 있습니다. 예를 들어 데이터에 사용하기에 적합한 방법, 데이터가 특정 모델링 가정을 충족하는지 여부, 데이터를 정리해야 하는지 여부, 재구성 등과 같은 세부 정보를 찾을 수 있습니다. Azure Cosmos DB의 기본 내장 Jupyter 노트북과 아파치 스파크를 사용하면 트랜잭션 및 분석 데이터에 대해 빠르고 효과적인 탐색 데이터 분석을 수행할 수 있습니다.
