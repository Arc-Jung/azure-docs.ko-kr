# <a name="network-security-group-flow-logs--version-2--upcoming-changes"></a>네트워크 보안 그룹 흐름 로그 - 버전 2 - 예정된 변경

네트워크 보안 그룹 흐름 로그 형식이 2018년 9월 15일부터 변경될 예정입니다. 추가되는 필드는 흐름 상태에 대한 정보, 증분 바이트 수 및 각 방향으로 전송되는 패킷을 제공합니다. 흐름 로그를 구문 분석하는 애플리케이션은 이 변경의 영향을 받게 되므로, 그에 따라 업데이트해야 합니다. 이 문서에서는 변경을 자세히 설명합니다.

## <a name="what-is-changing"></a>변경되는 내용

네트워크 보안 그룹 흐름 로그의 버전은 "버전": 1에서 "버전": 2로 변경될 예정이며 로그의 *flowTuples* 속성에 추가 필드가 추가될 것입니다. 형식에 대한 세부 정보는 [버전 2에서 흐름이 모델링되는 방식](#how-is-a-flow-modeled-in-version-2)에 제공됩니다.

### <a name="version-1-flow-tuple-format"></a>버전 1 흐름 튜플 형식

```
"1493763938,185.170.185.105,10.2.0.4,35370,23,T,I,D"
```

### <a name="version-2-flow-tuple-format"></a>버전 2 흐름 튜플 형식

```
"1493763938,185.170.185.105,10.2.0.4,35370,23,T,I,A,C,1,103,1,186"
```

## <a name="when-will-this-change-take-place"></a>이 변경은 언제 수행되나요?

이 변경은 미국 중서부 지역에서 **2018년 9월 15일**부터 적용됩니다. 공용 클라우드 지역의 변경 롤아웃은 2018년 11월 31일까지 완료되며, 그 이후에는 버전 2 로그만 사용할 수 있게 됩니다.

## <a name="am-i-affected-by-the-change"></a>나도 변경의 영향을 받나요?

네트워크 보안 그룹 흐름 로그 데이터를 처리하는 애플리케이션을 빌드하거나 사용하는 경우 이 변경의 영향을 받을 수 있습니다.

### <a name="if-i-use-traffic-analytics"></a>트래픽 분석을 사용하는 경우는 영향을 받나요?

 아니요. 트래픽 분석은 버전 1에서 버전 2 흐름 로깅으로 전환하는 동안 영향을 받지 않습니다. 곧 제공될 개선 기능은 버전 2 흐름 로그에 제공되는 추가 세션 및 대역폭 정보를 활용합니다.

### <a name="if-im-using-third-party-tooling"></a>타사 도구를 사용하는 경우는 영향을 받나요?

로그 형식의 변경 정보는 모든 [Network Watcher 파트너](https://azure.microsoft.com/services/network-watcher)에게 미리 전달되었습니다. 세부 정보에 대해서는 공급업체에 문의하세요.

### <a name="if-i-have-built-a-custom-application-or-integration"></a>사용자 지정 애플리케이션 또는 통합을 구축한 경우는 영향을 받나요?

**예**. NSG 흐름 로그를 새 형식으로 처리하도록 응용 프로그램을 수정해야 합니다.

## <a name="how-is-a-flow-modeled-in-version-2"></a>버전 2에서 흐름은 어떤 방식으로 모델링되나요?

로그의 버전 2에는 흐름 상태가 도입되었습니다. 흐름이 시작되면 흐름 상태 *B*가 기록됩니다. 흐름 상태 *C* 및 흐름 상태 *E*는 각각 흐름의 연속 및 흐름 종료를 나타내는 상태입니다. *C* 및 *E* 상태 둘 다 트래픽 대역폭 정보를 포함합니다.

**예제**: 185.170.185.105:35370과 10.2.0.4:23 간 TCP 대화의 흐름 튜플:

"1493763938,185.170.185.105,10.2.0.4,35370,23,T,I,A,B,,,," "1493695838,185.170.185.105,10.2.0.4,35370,23,T,I,A,C,1021,588096,8005,4610880" "1493696138,185.170.185.105,10.2.0.4,35370,23,T,I,A,E,52,29952,47,27072"

### <a name="how-does-the-format-differ-from-version-1"></a>이 형식은 버전 1과 어떻게 다른가요?

버전 1에서는 흐름이 설정되거나 설정되려고 시도될 때마다 흐름 튜플 ["1493763938,185.170.185.105,10.2.0.4,35370,23,T,I,D"]가 생성됩니다(사례 거부).

### <a name="how-are-bytes-and-packets-accounted-for"></a>바이트 및 패킷은 어떤 방식으로 고려되나요?

연속 *C* 및 종료 *E* 흐름 상태의 경우, 바이트 및 패킷 수는 이전 흐름 튜플 레코드 시간으로부터의 집계 개수입니다. 이전 예제 대화를 참조할 때 전송된 패킷의 총 수는 1021+52+8005+47 = 9125가 됩니다. 전송된 바이트의 총 수는 588096+29952+4610880+27072 = 5256000입니다.

### <a name="if-my-application-doesnt-understand-the-traffic-bandwidth-fields-how-does-version-2-affect-the-application"></a>애플리케이션이 트래픽 대역폭 필드를 이해하지 못하면 버전 2가 애플리케이션에 어떻게 영향을 미치나요?

버전 1 네트워크 보안 그룹 흐름 로깅을 사용하는 애플리케이션은 일반적으로 고유 흐름 수를 계산합니다. 흐름 상태를 고려하도록 구문 분석이 변경되지 않으면 보고된 흐름 수가 부정확하게 증가하는 것을 볼 수 있습니다. *C* 및 *E* 흐름 상태에서 흐름 튜플을 무시하면 고유 흐름을 계산할 수 있습니다.

## <a name="can-i-control-the-version-format-i-receive"></a>받는 버전 형식을 제어할 수 있나요?

 아니요. 흐름 로그를 사용하거나 사용하지 않도록 설정해도 로그의 출력 형식에는 영향을 주지 않습니다. 버전 1에서 버전 2 로그로의 변경은 지역별로 진행됩니다. 지역이 버전 1에서 버전 2로 업그레이드되면 NSG 흐름 로그가 이러한 형식 둘 다로 표시될 수 있습니다. 롤아웃이 완료되면 버전 2 로그만 사용할 수 있습니다.

## <a name="while-the-change-occurs-can-i-see-both-formats-for-the-same-network-security-group"></a>변경이 발생하는 동안 동일한 네트워크 보안 그룹에 대해 이러한 형식을 둘 다 볼 수 있나요?

예. 한 지역 내에서 mac 주소를 기준으로 전환이 수행됩니다. 네트워크 보안 그룹을 여러 컴퓨터에 적용할 수 있으므로 두 형식의 로그가 스토리지에 기록된 것을 볼 수 있습니다. 로그는 버전 1 또는 버전 2가 됩니다.

## <a name="will-i-see-duplicate-data"></a>중복된 데이터를 볼 수 있나요?

형식 간에 흐름 로깅 데이터가 중복되는 경우는 없습니다. 흐름은 변경이 발생하는 동안 버전 1 또는 버전 2 형식으로 기록됩니다.

## <a name="how-can-i-test-the-new-format-ahead-of-time"></a>새 형식을 미리 테스트하려면 어떻게 해야 하나요?

예. 솔루션 테스트에 사용하기 위해 동일한 버전 2 흐름 로그 파일을 [다운로드](https://aka.ms/nsgflowlogsv2blobsample)할 수 있습니다.

## <a name="are-there-changes-to-configuration-and-management-of-network-security-group-flow-logging"></a>네트워크 보안 그룹 흐름 로깅의 구성 및 관리가 변경되나요?

 아니요. 동일한 Azure Active Directory 테넌트를 공유하는 구독에서 Azure Storage 계정에 대한 지원이 올 초에 [릴리스](https://azure.microsoft.com/blog/new-azure-network-watcher-integrations-and-network-security-group-flow-logging-updates/)되었습니다. 이러한 변경으로 인해 네트워크 보안 그룹 흐름 로그를 관리할 때 필요한 스토리지 계정 수가 줄어들 수 있습니다.

## <a name="questions-or-feedback"></a>질문이나 의견이 있으신가요?

네트워크 보안 그룹 흐름 로그 및 Network Watcher 사용에 대한 귀하의 경험을 들려주셔요. 온라인 또는 이메일(AzureNetworkWatcher@microsoft.com)로 의견이나 제안 사항을 제공할 수 있습니다.
