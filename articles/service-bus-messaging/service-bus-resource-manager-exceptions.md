# <a name="service-bus-resource-manager-exceptions"></a>서비스 버스 리소스 관리자 예외

이 문서에서는 템플릿 또는 직접 호출을 통해 Azure 리소스 관리자를 사용하여 Azure Service Bus와 상호 작용할 때 생성된 예외를 나열합니다.

> [!IMPORTANT]
> 이 문서는 자주 업데이트됩니다. 업데이트를 다시 확인하시기 바랍니다.

다음은 Azure 리소스 관리자를 통해 표시되는 다양한 예외/오류입니다.

## <a name="error-bad-request"></a>오류: 잘못된 요청

"잘못된 요청"은 리소스 관리자가 받은 요청이 유효성 검사에 실패했으나 있음을 의미합니다.

| 오류 코드 | 오류 하위 코드 | 오류 메시지 | 설명 | 권장 |
| ---------- | ------------- | ------------- | ----------- | -------------- |
| 잘못된 요청 | 40,000 | 서브 코드 = 40000. 네임스페이스 '네임스페이스 *이름'이 '기본' 계층을* 사용하므로 큐를 만들 때 속성 *'속성 이름'을* 설정할 수 없습니다. 이 작업은 '표준' 또는 '프리미엄' 계층에서만 지원됩니다. | Azure 서비스 버스 기본 계층에서 아래 속성을 설정하거나 업데이트할 수 없습니다. <ul> <li> RequiresDuplicateDetection </li> <li> AutoDeleteOnIdle </li> <li>RequiresSession</li> <li>DefaultMessageTimeToLive </li> <li> DuplicateDetectionHistoryTimeWindow </li> <li> 인에이블익스프레스 </li> <li> ForwardTo </li> <li> 토픽 </li> </ul> | 이 기능을 사용하려면 기본 계층에서 표준 또는 프리미엄 계층으로 업그레이드하는 것이 좋습니다. |
| 잘못된 요청 | 40,000 | 서브 코드 = 40000. 기존 큐(또는 토픽)의 '필요 중복 감지' 속성에 대한 값은 변경할 수 없습니다. | 엔터티를 만들 때 중복 검색을 활성화/사용하지 않도록 설정해야 합니다. 생성 후에는 중복 검색 구성 매개 변수를 변경할 수 없습니다. | 이전에 만든 큐/토픽에서 중복 검색을 활성화하려면 중복 검색이 있는 새 큐/토픽을 만든 다음 원래 큐에서 새 큐/토픽으로 전달할 수 있습니다. |
| 잘못된 요청 | 40,000 | 서브 코드 = 40000. 지정된 값 16384가 잘못되었습니다. 속성 'MaxSizeInMegabytes'는 다음 값 중 하나여야 합니다: 1024;2048;3072;4096;5120. | MaxSizeInMega바이트 값이 잘못되었습니다. | MaxSizeInMega바이트가 1024, 2048, 3072, 4096, 5120 중 하나인지 확인합니다. |
| 잘못된 요청 | 40,000 | 서브 코드 = 40000. 큐/토픽의 분할은 변경할 수 없습니다. | 엔터티에 대해 분할을 변경할 수 없습니다. | 새 엔터티(큐 또는 토픽)를 만들고 파티션을 사용하도록 설정합니다. | 
| 잘못된 요청 | none | 네임스페이스 *'네임스페이스 이름'이* 없습니다. | Azure 구독 내에 네임스페이스가 없습니다. | 이 오류를 해결하려면 아래를 시도하십시오. <ul> <li> Azure 구독이 올바른지 확인합니다. </li> <li> 네임스페이스가 있는지 확인합니다. </li> <li> 네임스페이스 이름이 올바른지 확인합니다(맞춤법 오류 또는 null 문자열 없음). </li> </ul> | 
| 잘못된 요청 | 40400 | 서브 코드 = 40400. 자동 전달 대상 엔터티가 없습니다. | 자동 전달 대상 엔터티의 대상이 없습니다. | 원본을 생성하기 전에 대상 엔터티(큐 또는 토픽)가 있어야 합니다. 대상 엔터티를 만든 후 다시 시도합니다. |
| 잘못된 요청 | 40,000 | 서브 코드 = 40000. 제공된 잠금 시간이 허용되는 최대 '5'분을 초과합니다. | 메시지를 잠글 수 있는 시간은 최소 1분(최소)에서 5분(최대) 사이여야 합니다. | 제공된 잠금 시간이 1분에서 5분 사이인지 확인합니다. |
| 잘못된 요청 | 40,000 | 서브 코드 = 40000. 지연된 지속 속성 및 필요 중복 검색 속성을 함께 사용할 수 없습니다. | 중복 검색이 활성화된 엔터티는 영구적이어야 하므로 지속성을 지연할 수 없습니다. | [중복 검색에](duplicate-detection.md) 대해 자세히 알아보기 |
| 잘못된 요청 | 40,000 | 서브 코드 = 40000. 기존 큐의 요구 세션 속성에 대한 값은 변경할 수 없습니다. | 엔터티를 만들 때 세션에 대한 지원을 사용하도록 설정해야 합니다. 생성된 후에는 기존 엔터티(큐 또는 구독)에서 세션을 활성화/비활성화할 수 없습니다. | "요구 세션" 속성을 사용하도록 설정한 새 큐(또는 구독)를 삭제하고 다시 만듭니다. |
| 잘못된 요청 | 40,000 | 서브 코드 = 40000. 'URI_PATH'에는 서비스 버스에서 허용되지 않는 문자가 포함되어 있습니다. 엔터티 세그먼트에는 문자, 숫자, 마침표(.), 하이픈(-) 및 밑줄(_)만 포함될 수 있습니다. | 엔터티 세그먼트에는 문자, 숫자, 마침표(.), 하이픈(-) 및 밑줄(_)만 포함될 수 있습니다. 다른 문자로 인해 요청이 실패합니다. | URI 경로에 잘못된 문자가 없는지 확인합니다. |


## <a name="error-code-429"></a>오류 코드: 429

HTTP와 마찬가지로 "오류 코드 429"는 "너무 많은 요청"을 나타냅니다. 이는 해당 리소스에 대한 너무 많은 요청(또는 충돌하는 작업으로 인해)으로 인해 특정 리소스(네임스페이스)가 제한되고 있음을 의미합니다.

| 오류 코드 | 오류 하위 코드 | 오류 메시지 | 설명 | 권장 |
| ---------- | ------------- | ------------- | ----------- | -------------- |
| 429 | 50004 | 하위 코드 =50004. 네임스페이스가 제한되고 있기 *your namespace* 때문에 요청이 종료되었습니다. | 이 오류 조건은 들어오는 요청 수가 리소스의 제한을 초과하면 발생합니다. | 몇 초 동안 기다렸다가 다시 시도하십시오. <br/> <br/> [할당량](service-bus-quotas.md) 및 [Azure 리소스 관리자 요청 한도에](../azure-resource-manager/management/request-limits-and-throttling.md) 대해 자세히 알아보기|
| 429 | 40901 | 하위 코드 =40901. 충돌하는 또 다른 작업이 진행 중입니다. | 동일한 리소스/엔터티에서 다른 충돌하는 작업이 진행 중입니다. | 다시 시도하기 전에 현재 진행 중인 작업이 완료될 때까지 기다립니다. |
| 429 | 40900 | 하위 코드 =40900. 충돌. 리소스의 현재 상태에서 허용되지 않는 작업을 요청하고 있습니다. | 이 조건은 동일한 엔터티(큐, 토픽, 구독 또는 규칙)에서 동시에 작업을 수행하기 위해 여러 요청을 할 때 발생할 수 있습니다. | 몇 초 동안 기다렸다가 다시 시도하십시오. |
| 429 | 40901 | 다른 요청과 충돌하는 엔터티 *'엔터티 이름'에* 대한 요청 | 동일한 리소스/엔터티에서 다른 충돌하는 작업이 진행 중입니다. | 다시 시도하기 전에 이전 작업이 완료될 때까지 기다립니다. |
| 429 | 40901 | 엔터티 *'엔터티 이름'에*대한 또 다른 업데이트 요청이 진행 중입니다. | 동일한 리소스/엔터티에서 다른 충돌하는 작업이 진행 중입니다. | 다시 시도하기 전에 이전 작업이 완료될 때까지 기다립니다. |
| 429 | none | 리소스 충돌이 발생했습니다. 충돌하는 또 다른 작업이 진행 중일 수 있습니다. 실패한 작업에 대한 다시 시도인 경우 백그라운드 정리는 여전히 보류 중입니다. 나중에 다시 시도하십시오. | 동일한 엔터티에 대해 보류 중인 작업이 있을 때 이 조건이 적중할 수 있습니다. | 다시 시도하기 전에 이전 작업이 완료될 때까지 기다립니다. |


## <a name="error-code-not-found"></a>오류 코드: 찾을 수 없습니다.

이 오류 클래스는 리소스를 찾을 수 했음을 나타냅니다.

| 오류 코드 | 오류 하위 코드 | 오류 메시지 | 설명 | 권장 |
| ---------- | ------------- | ------------- | ----------- | -------------- |
| 찾을 수 없음 | none | 엔터티 *'엔터티 이름'을* 찾을 수 없습니다. | 작업을 찾을 수 없는 엔터티입니다. | 엔터티가 있는지 확인하고 작업을 다시 시도합니다. |
| 찾을 수 없음 | none | 찾을 수 없음. 작업이 없습니다. | 수행하려는 작업이 없습니다. | 작업을 확인하고 다시 시도하십시오. |
| 찾을 수 없음 | none | 들어오는 요청은 네임스페이스 정책 put 요청으로 인식되지 않습니다. | 들어오는 요청 본문이 null이므로 put 요청으로 실행할 수 없습니다. | null이 아닌지 확인하려면 요청 본문을 확인하십시오. | 
| 찾을 수 없음 | none | 메시징 엔터티 *'엔터티 이름'을* 찾을 수 없습니다. | 작업을 실행하려는 엔터티를 찾을 수 없습니다. | 엔터티가 있는지 확인하고 작업을 다시 시도하십시오. |

## <a name="error-code-internal-server-error"></a>오류 코드: 내부 서버 오류

이 오류 클래스는 내부 서버 오류가 있음을 나타냅니다.

| 오류 코드 | 오류 하위 코드 | 오류 메시지 | 설명 | 권장 |
| ---------- | ------------- | ------------- | ----------- | -------------- |
| 내부 서버 오류 | 50000 | 하위 코드 =50000. 내부 서버 오류| 여러 가지 이유로 발생할 수 있습니다. 증상 중 일부는 - <ul> <li> 클라이언트 요청/본문이 손상되어 오류가 발생합니다. </li> <li> 클라이언트 요청이 서비스의 처리 문제로 인해 시간 시간이 지났습니다. </li> </ul> | 이 문제를 해결하려면 <ul> <li> 요청 매개 변수가 null 또는 잘못된 형식이 아닌지 확인합니다. </li> <li> 요청을 다시 시도하십시오. </li> </ul> |

## <a name="error-code-unauthorized"></a>오류 코드: 무단

이 오류 클래스는 명령을 실행할 수 있는 권한 부여가 없는 경우를 나타냅니다.

| 오류 코드 | 오류 하위 코드 | 오류 메시지 | 설명 | 권장 |
| ---------- | ------------- | ------------- | ----------- | -------------- |
| 권한 없음 | none | 보조 네임스페이스에서 잘못된 작업입니다. 보조 네임스페이스는 읽기 전용입니다. | 이 작업은 읽기 전용 네임스페이스로 설정된 보조 네임스페이스에 대해 수행되었습니다. | 기본 네임스페이스에 대해 명령을 다시 시도합니다. [보조 네임스페이스에](service-bus-geo-dr.md) 대해 자세히 알아보기 |
| 권한 없음 | none | 누락된 토큰: 권한 부여 헤더를 찾을 수 없습니다. | 이 오류는 권한 부여에 null 또는 잘못된 값이 있을 때 발생합니다. | 권한 부여 헤더에 언급된 토큰 값이 null이 아닌지 확인합니다. |
