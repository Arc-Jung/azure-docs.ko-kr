# <a name="orchestration-mode-preview"></a>오케스트레이션 모드(미리 보기)

가상 시스템 규모 집합은 플랫폼 관리 가상 시스템의 논리적 그룹을 제공합니다. 확장 집합을 사용하면 가상 컴퓨터 구성 모델을 만들고 CPU 또는 메모리 로드에 따라 추가 인스턴스를 자동으로 추가 또는 제거하고 최신 OS 버전으로 자동으로 업그레이드합니다. 일반적으로 배율 집합을 사용하면 스케일 집합 생성 시 제공된 VM 구성 모델을 사용하여 가상 컴퓨터를 만들 수 있으며, 축척 집합은 구성 모델을 기반으로 암시적으로 생성된 가상 시스템만 관리할 수 있습니다.

축척 집합 오케스트레이션 모드(미리 보기)를 사용하면 축척 집합이 축척 집합 구성 모델 외부에서 명시적으로 생성된 가상 컴퓨터를 오케스트레이션할지 또는 축척 집합을 기반으로 암시적으로 생성된 가상 시스템 인스턴스를 오케스트레이션할지 여부를 선택할 수 있습니다. 구성 모델. 또한 확장 집합 오케스트레이션 모드를 사용하면 장애 도메인 및 가용성 영역에 이러한 VM을 배포하여 고가용성을 위해 VM 인프라를 설계할 수 있습니다.


가상 컴퓨터 규모 집합은 2가지 오케스트레이션 모드를 지원합니다.

- ScaleSetVM - 스케일 세트에 추가된 가상 컴퓨터 인스턴스는 축척 집합 구성 모델을 기반으로 합니다. 가상 시스템 인스턴스 수명 주기-생성, 업데이트, 삭제-확장 집합에 의해 관리 됩니다.
- VM(가상 시스템) – 스케일 집합 외부에서 생성된 가상 시스템은 스케일 집합에 명시적으로 추가할 수 있습니다. 
 

> [!IMPORTANT]
> 오케스트레이션 모드는 축척 세트를 만들 때 정의되며 나중에 변경하거나 업데이트할 수 없습니다. 
> 
> 가상 시스템 규모 집합의 이 기능은 현재 공개 미리 보기상태입니다.
> 이 미리 보기 버전은 서비스 수준 계약 없이 제공되며 프로덕션 워크로드에는 사용하지 않는 것이 좋습니다. 특정 기능이 지원되지 않거나 기능이 제한될 수 있습니다. 
> 자세한 내용은 [Microsoft Azure Preview에 대한 추가 사용 약관](https://azure.microsoft.com/support/legal/preview-supplemental-terms/)을 참조하세요.


## <a name="orchestration-modes"></a>오케스트레이션 모드

|                             | "오케스트레이션 모드": "VM" (가상 머신) | "오케스트레이션 모드": "ScaleSetVM" (가상 머신스케일세트VM) |
|-----------------------------|--------------------------------------------|--------------------------------------------------------------|
| VM 구성 모델      | None                                       | 필수 |
| 배율 조정 세트에 새 VM 추가  | VM을 만들 때 확장 집합에 VM이 명시적으로 추가됩니다. | VM은 VM 구성 모델, 인스턴스 수 및 자동 크기 조정 규칙에 따라 축척 집합에 암시적으로 생성되고 추가됩니다. | |
| VM 삭제                   | VM은 개별적으로 삭제해야 하며 VM이 있는 경우 배율 집합은 삭제되지 않습니다. | VM은 개별적으로 삭제할 수 있으며 축척 세트를 삭제하면 모든 VM 인스턴스가 삭제됩니다.  |
| VM 연결/분리           | 지원되지 않음                              | 지원되지 않음 |
| 인스턴스 수명 주기(삭제를 통한 생성) | VM과 해당 아티팩트(예: 디스크 및 NIC)는 독립적으로 관리할 수 있습니다. | 인스턴스및 해당 아티팩트(예: 디스크 및 NIC)는 인스턴스를 만드는 축척 집합 인스턴스에 암시적입니다. 스케일 세트 외부에서 분리하거나 별도로 관리할 수 없습니다. |
| 장애 도메인               | 장애 도메인을 정의할 수 있습니다. 지역 지원 기준 2개 또는 3개, 가용성 영역에 대해 5개 입니다. | 1에서 5까지가는 장애 도메인을 정의 할 수 있습니다. |
| 업데이트 도메인              | 업데이트 도메인이 장애 도메인에 자동으로 매핑됩니다. | 업데이트 도메인이 장애 도메인에 자동으로 매핑됩니다. |
| 가용성 영역          | 하나의 가용성 영역에서 지역 배포 또는 VM 지원 | 지역 배포 또는 여러 가용성 영역을 지원합니다. 영역 균형 전략을 정의할 수 있습니다. |
| 자동 크기 조정                   | 지원되지 않음                              | 지원됨 |
| OS 업그레이드                  | 지원되지 않음                              | 지원됨 |
| 모델 업데이트               | 지원되지 않음                              | 지원됨 |
| 인스턴스 제어            | 전체 VM 제어. VM에는 Azure VM 관리 기능(예: Azure Policy, Azure 백업 및 Azure 사이트 복구)의 전체 범위를 지원하는 정규화된 URI가 있습니다. | VM은 스케일 집합의 종속 리소스입니다. 인스턴스는 축척 집합을 통해서만 관리를 위해 액세스할 수 있습니다. |
| 인스턴스 모델              | Microsoft.Compute/가상 머신 모델 정의. | 마이크로소프트.계산/가상머신스케일셋/가상머신 모델 정의. |
| 용량                    | 빈 축척 집합을 만들 수 있습니다. 스케일 세트에 최대 200개의 VM을 추가할 수 있습니다. | 배율 집합은 인스턴스 수 0 - 1000으로 정의할 수 있습니다. |
| 이동                        | 지원됨                                  | 지원됨 |
| 단일 배치 그룹 == 거짓 | 지원되지 않음                          | 지원됨 |
