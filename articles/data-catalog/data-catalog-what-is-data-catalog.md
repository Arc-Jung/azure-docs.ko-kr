# Azure 데이터 카탈로그란?

Microsoft**Azure 데이터 카탈로그**는 등록 시스템과 엔터프라이즈 데이터 원본에 대한 검색 시스템 역할을 하는 완전히 관리되는 클라우드 서비스입니다. **Azure 데이터 카탈로그**는 분석가에서부터 데이터 과학자 및 개발자에 이르는 모든 사용자가 데이터 입력을 등록, 검색, 이해 및 소비할 수 있도록 지원하는 성능을 제공합니다.

## 문제 설명-동기 및 개요

일반적으로 엔터프라이즈 데이터 원본을 검색하는 일은 기초 지식에 기반한 유기적인 프로세스였습니다. 이것은 자사의 정보 자산에서 최고의 가치를 얻고자 하는 회사에게 수 많은 과제를 안겨 줍니다.

-	사용자는 데이터 원본이 다른 프로세스의 일부로 접촉되지 않는 한 데이터 원본이 있다는 사실을 알지 못합니다. 데이터 원본이 등록되는 중앙 위치가 없습니다.
-	사용자가 데이터 원본의 위치를 알지 못하면 클라이언트 응용 프로그램을 사용하여 데이터에 연결할 수 없습니다. 데이터 소비 환경에서는 사용자가 연결 문자열이나 경로를 알고 있어야 합니다.
-	사용자가 데이터 원본의 위치를 알지 못하면 데이터의 의도된 용도를 알 수 없습니다. 데이터 원본과 설명서가 다른 위치에 있으며 다른 환경을 통해 소비됩니다.
-	사용자는 정보 자산에 대한 질문이 있으면 해당 데이터를 담당하는 전문가나 팀을 찾아 오프라인으로 참가하도록 해야 합니다. 이 때 용도에 대한 전문가의 관점을 가진 데이터와 데이터 간의 명시적 연결은 없습니다.
-  사용자가 데이터 원본에 대한 액세스 권한을 요청하는 프로세스를 이해하지 않는 한, 데이터 원본 및 해당 설명서를 검색해도 필요한 데이터에 액세스할 수 없습니다.

이러한 과제는 데이터의 소비자가 직면하고 있는 반면에 정보 자산의 생성 및 유지 관리를 책임지는 사용자는 자신만의 과제에 직면하고 있습니다.

-	데이터 원본에 설명이 포함된 메타데이터로 주석을 추가하는 것은 종종 시간 낭비입니다. 클라이언트 응용 프로그램은 일반적으로 데이터 원본에 저장된 설명을 무시합니다.
-	데이터 원본에 대한 설명서 생성은 종종 시간 낭비입니다. 설명서를 데이터 원본과 동기화시키는 것은 지속적인 책임이며, 설명서는 대개 오래된 것으로 간주되기 때문에 사용자가 이를 신뢰하지 않습니다.
- 데이터 원본에 대한 액세스를 제한하고 데이터 소비자에게 액세스를 요청하는 방법을 알고 있는지 확인하는 것이 지속적인 문제입니다.

데이터 원본에 대한 설명서 생성 및 유지 관리는 복잡하고 시간 소모적입니다. 이 설명서를 데이터 원본을 사용하는 모든 이가 사용할 수 있도록 만들어야 하는 과제는 이보다 더 복잡하고 시간 소모적입니다.

이 모든 과제가 합쳐지면 엔터프라이즈 데이터의 사용과 이해를 독려 및 홍보하고자 하는 회사에게 큰 장애물이 됩니다.

## 서비스 설명

**Azure 데이터 카탈로그**는 기업에서 관리하는 데이터를 필요로 하는 사용자가 기존의 정보 자산을 쉽게 발견하고 이해할 수 있게 지원함으로써 이 문제를 해결하고 기업이 기존 정보 자산으로부터 최고의 가치를 얻어낼 수 있도록 설계합니다.

**Azure 데이터 카탈로그**는 데이터 원본을 등록할 수 있는 클라우드 기반의 서비스를 제공합니다. 데이터가 기존 위치에 남게 되지만 메타데이터의 복사본은 해당 데이터 원본의 위치에 대한 참조와 함께 **Azure 데이터 카탈로그**에 추가됩니다. 이 메타데이터는 또한 인덱싱되므로 각 데이터의 원본을 검색을 통해 쉽게 찾을 수 있으며 검색한 사용자가 이해할 수 있습니다.

일단 데이터 소스가 등록되면 등록을 실행하는 사용자나 엔터프라이즈 내 다른 사용자에 의해 메타데이터를 보강할 수 있습니다. 모든 사용자는 설명, 태그 또는 설명서와 데이터 원본 액세스 요청과 같은 기타 메타데이터를 제공하여 데이터 원본에 주석을 달 수 있습니다. 이 설명이 포함된 메타데이터는 데이터 원본으로부터 등록된 구조적 메타데이터(예: 열 이름과 데이터 형식)를 보완하여 검색과 이해를 쉽게 만듭니다.

데이터 원본과 데이터 원본의 용도를 찾고 이해하는 것이 원본을 등록하는 주요 목적입니다. 기업 사용자가 자신의 노력(비즈니스 인텔리전스, 응용 프로그램 개발, 데이터 과학 또는 적절한 데이터가 필요한 기타 모든 작업)을 위해 데이터가 필요할 때는 **Azure 데이터 카탈로그** 검색 환경을 사용하여 요구에 일치하는 데이터를 빠르게 찾고 데이터를 파악하여 목적에 부합하는지 평가하며, 선택한 도구에서 데이터 원본을 열어서 해당 데이터를 소비할 수 있습니다.

## 데이터 원본 등록

데이터 원본 등록은 **Azure 데이터 카탈로그** 데이터 원본 등록 도구를 사용하여 실행됩니다. 이 응용 프로그램은 **Azure 데이터 카탈로그** 포털에서 다운로드할 수 있습니다.

등록 프로세스는 다음 세 가지 단계로 구성됩니다.

1.	데이터 원본에 연결 - 사용자가 데이터 원본 및 자격 증명을 지정하여 SQL Server 인스턴스와 같은 데이터 원본에 연결합니다.
2.	등록하려는 개체 선택 - 사용자가 **Azure 데이터 카탈로그**와 등록해야 하는 지정된 위치에 있는 개체를 선택합니다. 이 서버의 모든 데이터베이스에 테이블의 전체 집합 또는 특히 선택한 테이블 및 뷰 하위 집합을 수 있습니다.
3.	등록 완료 - 사용자가 프로세스를 완료하고 데이터 원본 등록 도구가 데이터 원본에서 구조적 메타데이터를 추출한 다음, 해당 메타데이터를 **Azure 데이터 카탈로그** 클라우드 서비스로 전송합니다.

> [AZURE.NOTE]미리 보기의 경우 **Azure 데이터 카탈로그**는 현재 다음 데이터 원본 및 자산 형식을 지원합니다.

- SQL Server 테이블
- SQL Server 뷰
- Oracle 데이터베이스 테이블
- Oracle 데이터베이스 뷰
- SQL Server Analysis Services 다차원 차원
- SQL Server Analysis Services 다차원 측정값
- SQL Server Analysis Services 다차원 KPI
- SQL Server Analysis Services 테이블 형식 테이블
- SQL Server Reporting Services 보고서
- Azure 저장소 Blob
- Azure 저장소 디렉터리

**Azure 데이터 카탈로그** 미리보기 중에 추가 데이터 원본과 자산 형식이 추가됩니다.

> [AZURE.IMPORTANT]데이터 원본 등록 도구에서 “미리보기 포함"을 선택하지 않는 한 **Azure 데이터 카탈로그**에 데이터 원본을 등록해도 데이터 원본으로부터 데이터가 복사되지 않습니다. 등록은 데이터가 아닌 데이터 원본 메타데이터를 복사합니다. 메타데이터의 예로는 열의 이름 및 데이터 형식과 기타 데이터 원본 속성과 함께 테이블 이름 및 기타 데이터 원본 개체가 포함됩니다. 메타데이터에는 또한 데이터 원본 위치가 포함되므로 **Azure 데이터 카탈로그**를 사용하여 데이터 원본을 발견한 사용자가 해당 데이터 원본에 연결할 수 있습니다. “미리보기 포함”을 선택했다면 데이터 원본 등록 도구가 **Azure 데이터 카탈로그** 포털에서 데이터 원본을 발견하는 사용자들에게 표시될 작은 레코드 집합을 **Azure 데이터 카탈로그**에도 복사합니다.

## 데이터 원본 메타데이터 보강

일단 등록이 완료되면 데이터 원본을 검색 및 소비할 수 있지만 **Azure 데이터 카탈로그**의 진정한 가치는 데이터 원본으로부터 추출한 구조적 메타데이터와 동일한 환경에서 설명이 포함된 비즈니스 메타데이터를 보유하고 있다는 사실입니다. 이 추가 메타데이터는 다음 세 가지 큰 혜택을 제공합니다.

-	등록된 데이터 원본을 보다 쉽게 검색할 수 있습니다. 사용자가 제공한 메타데이터가 **Azure 데이터 카탈로그** 검색 인덱스에 추가됩니다. 그러면 사용자가 원래 데이터 원본에는 없을 수도 있는 용어와 개념을 사용하여 데이터를 검색할 수 있습니다. 예를 들어, 고객 데이터가 포함된 데이터베이스 테이블의 이름이 "tbl\_c45"인 경우 "Customer"라는 친숙한 이름을 제공하면 고객 데이터를 찾는 사용자가 훨씬 쉽게 이를 찾을 수 있습니다. 마찬가지로, 보고서, 대시보드 또는 이 데이터를 사용하는 프로세스의 이름이 포함된 설명을 제공하면 다운스트림 아티팩트를 검색 용어로 사용하는 사용자들이 데이터 원본을 보다 쉽게 찾을 수 있습니다.
-	등록된 데이터 원본은 일단 검색되면 보다 쉽게 이해됩니다. 사용자가 제공한 메타데이터는 주석이 추가된 데이터 원본을 확인하는 모든 **Azure 데이터 카탈로그** 사용자에게 제시되며, 이는 추가 컨텍스트 및 정보의 제공에 도움이 됩니다. 대부분의 데이터 원본은 보통 의미 있는 설명이나 설명서를 포함하지 않으며, 이를 포함하는 데이터 원본은 일반적으로 기술적인 DBA나 데이터베이스 개발자 대상에 중점을 두고 있습니다. **Azure 데이터 카탈로그** 내에서 대상 그룹에 적합한 설명과 태그로 데이터 원본을 보강함으로써 사용자들은 데이터를 검색하는 이들이 세부 정보와 의도하는 용도를 이해할 수 있도록 지원할 수 있습니다.
-  등록된 각 데이터 원본이 요청 액세스 정보를 포함할 수 있으므로, 사용자가 기존 프로세스를 쉽게 이해하고 수행하여 데이터 원본 및 해당 데이터에 대한 액세스를 요청할 수 있습니다.

> [AZURE.NOTE]각 **Azure 데이터 카탈로그** 사용자는 데이터 자산과 특성에 대해 자신만의 태그와 설명을 추가할 수 있습니다. **Azure 데이터 카탈로그**는 각 주석의 값과 원본을 추적하며 사용자와 주석이 추가된 날짜를 표시합니다. 메타데이터에 대한 이 크라우드소싱 접근 방식을 통해 데이터 및 그 용도에 대한 관점을 가진 모든 사용자가 전체적인 사용자 커뮤니티와 의견 및 리소스를 공유할 수 있습니다.

## 탐색, 검색 및 이해

**Azure 데이터 카탈로그**에 데이터 원본을 등록 및 보강하는 목표는 사용자들이 전사적으로 원본을 검색, 이해 및 사용할 수 있게 하기 위해서입니다. **Azure 데이터 카탈로그** 포털은 이 프로세스에 대한 기본 도구입니다.

**Azure 데이터 카탈로그** 포털에서는 데이터 탐색 및 찾기: 검색 및 필터링을 위한 두 가지 메커니즘을 제공합니다.

데이터 원본에 대한 **Azure 데이터 카탈로그**를 검색하려면, **Azure 데이터 카탈로그** 포털의 검색 상자에 검색 용어를 입력하면 됩니다. 그러면 포털이 검색 용어와 일치하는 등록된 각 데이터 원본에 대한 타일을 표시합니다. 이 타일에는 다른 높은 수준의 정보와 함께 데이터 원본에 할당된 이름, 설명 및 태그가 포함됩니다.

**Azure 데이터 카탈로그**의 내용을 필터링하려면, **Azure 데이터 카탈로그** 포털에 표시된 필터를 하나 이상 선택하면 됩니다. 그러면 포털에 표시된 타일이 지정된 필터 조건과 일치하는 타일로 제한됩니다. 검색 없이 데이터 원본을 필터링하거나 검색 결과를 필터링할 수 있습니다.

데이터 원본에 대한 보다 완전한 정보를 확인하고 현재 작업에 적절한지 파악하려면 데이터 원본에 대한 타일을 클릭하기만 하면 됩니다. 그러면 속성 창이 표시되고 모든 메타데이터가 포함됩니다.

속성 창 위에 추가 단추가 생깁니다.

1.	미리 보기: 이 단추를 선택하면 데이터 원본 등록 중에 미리 보기가 선택된 경우 데이터 원본으로부터 미리 보기 레코드의 정적인 집합이 표시됩니다.
2.	스키마: 이 단추를 선택하면 열 이름 및 데이터 형식, 그리고 **Azure 데이터 카탈로그** 내 모든 열 수준의 메타데이터를 포함한 데이터 원본 스키마를 표시합니다.

> [AZURE.NOTE]**검색** 환경은 **소비** 환경뿐만 아니라 **보강** 경험을 위한 진입점이 될 수 있음을 기억하십시오. **Azure 데이터 카탈로그**가 가져오는 크라우드소싱 접근 방식이란 등록된 데이터 원본을 발견하는 모든 사용자가 발견한 데이터에 사용 이외에도 데이터에 대한 자신의 의견을 공유할 수 있다는 뜻입니다.

## 데이터 원본 메타데이터 제거

데이터 원본을 등록한 후에는 **Azure 데이터 카탈로그**로부터 데이터 원본 참조를 제거해야 하는 경우도 있습니다. 이것은 변화하는 요구 사항이나 사용 중지되는 원본 시스템이 원인일 수 있습니다. 이유에 상관 없이 **Azure 데이터 카탈로그**는 선택하여 삭제하기만 하면 데이터 원본을 쉽게 제거하여 더 이상 검색 및 소비되지 않게 할 수 있습니다.

> [AZURE.IMPORTANT]**Azure 데이터 카탈로그**에서 데이터 원본 삭제는 **Azure 데이터 카탈로그** 서비스에 저장된 메타데이터만 삭제합니다. 원래 데이터 원본이 어떤 방식으로 영향을 받지 않습니다.

## 데이터 원본 사용

데이터 검색의 궁극적인 목표는 사용자가 필요로 하는 데이터를 찾고 선택한 데이터 도구에서 사용하기 위함입니다. Azure 데이터 카탈로그의 데이터 소비 환경은 두 가지 방법으로 기능을 활성화합니다.

1.	**Azure 데이터 카탈로그**에서 직접 지원하는 클라이언트 응용 프로그램의 경우, 사용자들은 포털에서 찾을 수 있는 데이터 원본 타일 내의 **Open In(열기)** 메뉴를 클릭할 수 있습니다. 그러면 클라이언트 응용 프로그램이 선택한 데이터 원본에 대한 연결과 함께 시작됩니다.
2.	모든 클라이언트 응용 프로그램의 경우 사용자는 선택한 데이터 원본에 대해 속성 창에 표시되는 연결 정보를 사용할 수 있습니다. 이 정보에는 데이터 연결에 필요한 모든 세부 정보(서버 이름, 데이터베이스 이름 및 개체 이름)가 포함되며, 클라이언트 도구의 연결 환경에 복사할 수 있습니다. 데이터 원본에 대해 요청 액세스 세부 정보를 제공한 경우, 이 정보는 연결 세부 정보 옆에 표시됩니다.

> [AZURE.NOTE]Azure 데이터 카탈로그의 미리 보기의 경우 Microsoft Excel 및 SQL Server Reporting Services 보고서 관리자만 **Open In(열기)** 메뉴에서 직접 지원되고 사용 가능합니다.
