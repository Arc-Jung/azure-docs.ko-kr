# 논리 앱 모니터링

[논리 앱을 만든](app-service-logic-create-a-logic-app.md) 후에 Azure 포털에서 전체 실행 기록을 확인할 수 있습니다. 기록을 보려면 **찾아보기**를 선택하고 **논리 앱**을 선택합니다. 구독의 논리 앱 목록이 모두 표시됩니다. 모든 논리 앱을 선택할 수 있으며 **사용** 또는 **사용 안 함**으로 설정할 수 있습니다. 논리 앱 **사용**은 트리거 이벤트에 대한 응답으로 논리 앱을 실행하도록 트리거함을 의미합니다. 논리 앱 **사용 안 함**은 이벤트에 대한 응답에서 실행되지 않습니다.

![개요](./media/app-service-logic-monitor-your-logic-apps/overview.png)

논리 앱에 대한 블레이드가 표시되는 경우 유용한 다음 두 개의 섹션이 있습니다.

- **요약** - 최신 상태를 보여 주며 논리 앱 편집을 위한 진입점입니다.
- **모든 실행** - 이 논리 앱의 실행 목록을 보여 줍니다.

## 앱의 실행 보기

![모든 실행](./media/app-service-logic-monitor-your-logic-apps/allruns.png)

이 실행 목록은 특정 실행의 **시작 시간**, **실행 식별자**(REST API 호출 시 사용할 수 있음) 및 **기간**을 보여 줍니다. 행을 선택하면 해당 실행의 세부 정보가 표시됩니다.

세부 정보 블레이드는 실행에 포함된 모든 작업의 실행 시간 및 순서 그래프를 보여 줍니다. 다음에는 실행된 모든 작업의 전체 목록이 있습니다.

![실행 및 동작](./media/app-service-logic-monitor-your-logic-apps/runandaction.png)

끝으로, 특정 동작의 **입력** 및 **출력** 섹션에서 해당 동작으로 전달되고 동작에서 수신된 모든 데이터를 확인할 수 있습니다. 전체 내용을 보려면 링크를 선택합니다.(또한 링크를 복사하여 콘텐츠를 다운로드 할 수 있습니다)

다른 중요한 정보는 **추적 ID**입니다. 이 식별자는 모든 작업 호출의 헤더에 전달됩니다. 고유한 서비스 내에 로깅이 있는 경우 추적 ID를 로깅하는 것이 좋으며, 그런 다음 고유한 로그와 이 식별자를 상호 참조할 수 있습니다.

## 트리거 기록 보기 

폴링 트리거는 일정한 간격으로 API를 확인하지만 응답에 따라 실행을 시작하지 않을 수도 있습니다.(예: `200`은 실행, `202`는 실행 안 함을 의미함) 트리거 기록을 통해 발생하지만 논리 앱을 실행하지 않는 모든 호출을 확인할 수 있습니다.(`202`이 응답)

![트리거 기록](./media/app-service-logic-monitor-your-logic-apps/triggerhistory.png)

각 트리거는 **발생**하거나 발생하지 않았거나 혹은 일종의 오류가 있는지(**실패**했는지) 사용자가 볼 수 있습니다. 트리거가 실패한 이유를 조사하려면 **출력** 링크를 클릭합니다. 발생한 경우 **실행** 링크를 선택하여 발생한 후의 상황을 확인합니다.

*푸시* 트리거의 경우 여기서 시작한 실행 시간을 확인하지 *않습니다*. 대신 *콜백 등록*을 확인합니다. 논리 앱을 다시 호출하기 위해 등록하는 경우입니다. 푸시 트리거가 작동하지 않은 경우 등록(출력에서 볼 수 있음)에 문제가 있을 수 있지만, 작동하는 경우 해당 API를 구체적으로 조사해야 합니다.

## Azure 진단 사용

워크플로에 대한 런타임 데이터를 검색 및 저장하는 데 진단 정보를 사용할 수 있습니다. 논리 앱 블레이드에서 **모니터링** 차트로 아래로 스크롤하고 **진단을 활성화하려면 여기를 클릭**을 선택합니다. 그런 다음 논리 앱의 영역에서 저장소 계정에 대해 구성하고 논리 앱의 **로그** 또는 **메트릭**을 구독할 수 있습니다. **로그**는 실행, 작업, 이벤트가 시작 또는 완료될 때마다 이벤트를 전송합니다. **메트릭**은 기간 내에 실행이 몇 번이나 실행되는지에 대한 집계된 데이터를 제공합니다.

## 버전 관리 사용

UI에서 현재 사용 불가능한 추가 기능(출시 예정)이 있지만 [REST API](https://msdn.microsoft.com/library/azure/mt643788.aspx)를 사용하여 사용 가능합니다. 논리 앱의 정의를 업데이트하면 이전 버전의 정의가 저장됩니다. 진행 중인 실행이 있는 경우 해당 실행이 실행을 시작할 때 존재한 논리 앱의 버전을 참조하기 때문입니다. 실행이 진행되는 동안에는 실행 정의를 변경할 수 없습니다. 버전 기록 REST API를 통해 이 정보에 액세스할 수 있습니다.
