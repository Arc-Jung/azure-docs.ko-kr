# Microsoft Azure 앱 서비스의 엔터프라이즈 커넥터
Microsoft Azure 앱 서비스(또는 줄여서 앱 서비스)에는 SAP 및 Marketo와 같이 엔터프라이즈 시스템에서 사용할 수 있는 여러 커넥터가 포함되어 있습니다. 이러한 커넥터를 사용하면 엔터프라이즈 시스템에 쉽게 연결하고 다양한 작업을 완료할 수 있습니다.

이러한 엔터프라이즈 커넥터는 "트리거" 또는 "동작" 기능을 제공합니다. 트리거는 MongoDb에서의 항목 업데이트와 같이 특정 이벤트를 기반으로 하는 새 인스턴스를 시작합니다. MongoDB에서 항목을 업데이트한 후 MongoDB에서 항목을 가져오는 경우와 같이 작업은 결과입니다.


## 엔터프라이즈 커넥터란 무엇인가요?
엔터프라이즈 커넥터는 온-프레미스 및 클라우드 기반 엔터프라이즈 시스템에 연결할 수 있는 기존 "API 앱"이며, 여기에는 다음이 포함됩니다.

커넥터 | 설명
--- | ---
Marketo | **동작** 커넥터입니다. Marketo에 연결하고 잠재 고객 생성/업데이트, 잠재 고객 변경 가져오기, 캠페인 예약, 캠페인 요청, 잠재 고객 가져오기, 캠페인/목록 정보 가져오기, 목록에 잠재 고객 추가, 목록에서 잠재 고객 제거 등의 다양한 작업을 수행합니다.
MongoDB | **트리거** 및 **동작** 커넥터입니다. Mongo 서버 데이터베이스 또는 온-프레미스 Mongo 서버에 연결합니다. MongoDB 컬렉션에서 항목의 생성, 업데이트, 가져오기 및 삭제와 같은 다양한 작업을 수행할 수 있습니다. 
QuickBooks | **동작** 커넥터입니다. QuickBooks에 연결하고 Intuit QuickBooks에서 응용 프로그램을 사용하여 고객, 항목, 송장 등과 같은 다양한 엔터티 생성, 업데이트, 읽기, 삭제 및 쿼리 작업을 수행할 수 있습니다.
Salesforce |**동작** 커넥터입니다. SalesForce에 연결하고 Salesforce 계정에서 계정, 잠재 고객, 기회, 사례 등의 다양한 엔터티를 관리합니다. 다른 엔터티에 대해 생성, 업데이트, 삽입, 쿼리 및 삭제와 같은 다양한 작업을 수행할 수 있습니다. 
SugarCRM | **동작** 커넥터입니다. SugarCRM 온라인에 연결하고 응용 프로그램을 사용하여 계정, 연락처, 제품 등과 같은 다양한 유형의 모듈 생성, 업데이트, 읽기 및 삭제 작업을 수행할 수 있습니다.
SAP | **동작** 커넥터입니다. 온-프레미스 SAP 서버에 연결합니다. RFC, BAPI 및 tRFC를 호출하며 IDOC도 보냅니다.


이러한 커넥터를 사용하여 엔터프라이즈 시스템의 다양한 작업을 완료할 수 있습니다. 예를 들어 QuickBooks 커넥터를 사용하여 QuickBooks 내에서 새 고객 추가, 송장 업데이트 또는 항목 삭제와 같은 일반적인 작업을 수행할 수 있습니다.

원하는 개수만큼 커넥터를 손쉽게 만들 수 있습니다. 또한 여러 시나리오 또는 워크플로 내에서 단일 커넥터를 다시 사용할 수도 있습니다.

예를 들어 10개의 제품 항목을 구매하려는 고객을 회의에서 만난다고 가정해 보겠습니다. 모바일 장치에서, 직접 만든 QuickBooks 커넥터를 사용하는 모바일 앱을 열고 고객 세부 정보를 추가하고 송장을 만들 수 있습니다.

회의에 참석하고 있는 동안 같은 비즈니스 위치에 있는 고객이 기존 송장을 지급하려고 합니다. 직원은 사용자가 직접 만든 동일한 QuickBooks 커넥터를 사용하는 응용 프로그램을 열고 고객 정보를 검색하고 송장을 업데이트하여 새 잔액을 반영합니다.

이는 다른 워크플로이지만, 동일한 QuickBooks 커넥터를 사용합니다. 어떠한 코드를 작성하지 않고도 이러한 커넥터를 생성 및 사용할 수 있습니다. 이제 시작하겠습니다.


## 시작하기 위한 요구 사항
en 엔터프라이즈를 사용하는 커넥터를 만들 경우 몇 가지 필요한 리소스가 있습니다. 이러한 항목을 커넥터 내에서 사용하려면 먼저 사용자가 해당 항목을 만들어야 합니다. 이러한 요구 사항에는 다음이 포함됩니다.

요구 사항 | 설명
--- | ---
서비스 버스 네임스페이스 및 해당 키 값 | 온-프레미스 SAP 또는 MongoDB를 사용할 경우 서비스 버스 네임스페이스 및 해당 키 값이 필요합니다. 온-프레미스 시스템에 연결되지 않은 경우 서비스 버스 네임스페이스가 필요하지 않습니다.<br/><br/>[서비스 버스 네임스페이스 만들기](http://msdn.microsoft.com/library/azure/hh690931.aspx)
온-프레미스 하이브리드 연결 관리자 | 온-프레미스 시스템에 연결할 경우 온-프레미스 시스템에 하이브리드 연결 관리자를 설치합니다. 온-프레미스 커넥터를 만들 때 커넥터 속성 또는 설정에서 다운로드를 사용할 수 있습니다. 
시스템별 속성 | 엔터프라이즈 시스템을 사용할 경우 필요한 시스템별 속성(예: 사용자 이름 및 암호)이 있습니다. 


## 커넥터 만들기
Azure 포털을 사용하여 커넥터를 만들 수 있습니다.

### Azure 포털에서 커넥터 만들기
Azure 포털에서 논리 앱, 웹앱 또는 모바일 앱을 만들 때 엔터프라이즈 커넥터를 만들 수 있습니다. 또는 자체 블레이드를 사용하여 만들 수 있습니다. 두 가지 방법 모두 쉬우므로 요구 사항 또는 선호도에 따라 선택하면 됩니다. 일부 사용자는 특정 속성이 있는 모든 커넥터를 먼저 만드는 것을 선호합니다. 그런 다음 논리, 웹 또는 모바일 앱을 만든 후 만든 커넥터를 추가합니다.

다음 단계에서는 커넥터 블레이드를 사용하여 엔터프라이즈 커넥터를 만듭니다.

1. Azure 포털 시작 보드(홈페이지)에서 **마켓플레이스**를 선택합니다. **API 앱**에 기존의 모든 커넥터가 나열됩니다. 특정 커넥터를 **검색**할 수도 있습니다.
2. 커넥터를 선택합니다. 새 블레이드에서 **만들기**를 선택합니다. 
3. 다음과 같은 속성을 입력합니다. 

	속성 | 설명
--- | ---
이름 | 커넥터의 이름을 입력합니다. 예를 들어 이름을 *SAPConnector*, *SalesForceGetAccounts* 또는 *QuickBooksGetItems*로 지정할 수 있습니다.
패키지 설정 | *SAP User Name* 또는 *SugarCRM Server URL*과 같은 엔터프라이즈 시스템 설정을 입력합니다. 이 항목의 [엔터프라이즈 시스템별 속성](#AddProperties)을 참조하세요. 
앱 서비스 계획 | 요금제를 나열합니다. 필요한 리소스가 더 많거나 적은 경우 변경할 수 있습니다.
가격 책정 계층 | Azure 구독 내에서 가격 책정 범주를 나열하는 읽기 전용 속성입니다. 
리소스 그룹 | 새 그룹을 만들거나 기존 그룹을 사용합니다. 논리 앱, 웹앱 및 모바일 앱에 대한 모든 API 앱과 커넥터가 동일한 리소스 그룹에 속해야 합니다. <br/><br/>이 속성은 [리소스 그룹 사용](../resource-group-overview.md)에서 설명합니다. 
구독 | 현재 구독을 나열하는 읽기 전용 속성입니다.
위치 | Azure 서비스를 호스트하는 지리적 위치입니다. 
시작 보드에 추가 | 시작 보드(홈페이지)에 커넥터를 추가하려면 이 속성을 선택합니다.


	**<a name="AddProperties"></a>엔터프라이즈 시스템별 속성**

> [AZURE.IMPORTANT]모든 커넥터에는 해당 엔터프라이즈 시스템에 고유한 속성이 있습니다. SAP에 연결할 경우 SAP 관련 속성을 입력합니다. Salesforce에 연결할 경우에는 Salesforce 관련 속성을 입력하는 식으로 해당하는 고유 속성을 입력합니다. 다음 표에는 필수 엔터프라이즈 시스템 속성이 나와 있습니다.
	
	Enterprise System | Required Properties
--- | ---
Marketo | <ul><li>끝점</li><li>공급자 이름</li></ul>
MongoDB| <ul><li>연결 문자열</li><li>호스트</li><li>포트</li><li>사용자 이름</li><li>암호</li><li>데이터베이스</li><li>SSL 암호화 사용</li><li>온-프레미스: 클라우드 기반인 경우 False를 입력합니다. MongoDb 시스템이 온-프레미스인 경우 True를 입력하고 다음 속성도 입력합니다.<ul><li>공유 액세스 키 이름</li><li>서비스 버스 네임스페이스</li><li>릴레이 경로</li><li>키 전송</li></ul></li></ul>
QuickBooks | <ul><li>회사 ID</li><li>공급자 이름</li></ul>
SAP | <ul><li>호스트 이름</li><li>언어</li><li>사용자 이름</li><li>암호</li><li>시스템 번호</li><li>서비스 버스 연결 문자열</li><li>RFC 이름</li><li>TRFC 이름</li><li>BAPI 이름</li><li>IDOC 이름</li></ul>
Salesforce | <ul><li>공급자 이름</li><li>인스턴스</li><li>버전</li><li>엔터티(쉼표로 구분된 값)</li></ul>
SugarCRM | <ul><li>서버 URL</li><li>공급자 이름</li><li>모듈 이름</li></ul>

4. **만들기**를 선택합니다.


## 온-프레미스 하이브리드 연결 관리자를 설치합니다.
SAP와 같이 온-프레미스 시스템에 연결하는 엔터프라이즈 커넥터를 만든 후 하이브리드 연결 관리자를 설치합니다.

1. 온-프레미스 엔터프라이즈 시스템에서 Azure 관리 포털을 열고 엔터프라이즈 커넥터를 선택합니다. 
2. 블레이드에서 **하이브리드 연결**을 선택합니다. 
3. **기본 게이트웨이 구성 문자열** 값을 복사합니다. 
4. **다운로드 및 구성**을 선택합니다. 설치하는 동안 복사한 게이트웨이 구성 문자열을 붙여넣고 설치를 계속합니다. 
5. 연결을 확인하려면 엔터프라이즈 커넥터 블레이드를 엽니다. 상태에 **연결됨**이 표시됩니다. 

[온-프레미스 SAP 서버와 통합](app-service-logic-integrate-with-an-on-premise-SAP-server.md)에 예제가 나와 있습니다.


## API 앱 모니터링
Azure 관리 포털에서 엔터프라이즈 API 앱을 엽니다. **작업** 섹션에서 다양한 관리 작업을 볼 수 있습니다. 예를 들어 다음을 수행할 수 있습니다.

- 정보 및 오류 이벤트 보기
- 작업자 프로세스(w3wp)의 메모리 사용 및 스레드 수 보기
- 응용 프로그램 및 웹 서버 로그 보기\

자세한 내용은 [논리 앱 모니터링](app-service-logic-monitor-your-logic-apps.md)을 참조하세요.


## 응용 프로그램에 커넥터 추가 
Microsoft Azure 앱 서비스는 이러한 커넥터를 사용할 수 있는 다양한 응용 프로그램 종류를 표시합니다. 새 커넥터를 만들거나 논리 앱, 모바일 앱 또는 웹앱에 기존 커넥터를 추가할 수 있습니다.

앱 내에서 간단히 갤러리의 커넥터를 선택하면 해당 커넥터가 앱에 자동으로 추가됩니다.

다음 단계에서는 논리 앱, 모바일 앱 또는 웹앱에 엔터프라이즈 커넥터를 추가합니다.

1. Azure 포털 시작 보드(홈페이지)에서 **마켓플레이스**로 이동하여 논리 앱, 모바일 앱 또는 웹앱을 검색합니다.

	새 앱을 만드는 경우 논리 앱, 모바일 앱 또는 웹앱을 검색합니다. 앱을 선택하고 새 블레이드에서 **만들기**를 선택합니다. [논리 앱 만들기](app-service-logic-create-a-logic-app.md)에 단계가 나열되어 있습니다.

2. 앱을 열고 **트리거 및 동작**을 선택합니다.

3. 만든 커넥터를 **갤러리**에서 추가합니다. 그러면 해당 커넥터가 앱에 자동으로 추가됩니다.
4. **확인**을 선택하여 변경 내용을 저장합니다. 


## 보안
일부 엔터프라이즈 커넥터는 OAuth 보안을 사용합니다. 커넥터를 앱에 추가하는 경우 로그인 계정으로 엔터프라이즈 시스템에 연결하여 커넥터에 **권한을 부여**하고 사용 조건에 동의합니다. 이렇게 하면 앱 및 커넥터가 로그인 계정을 사용하여 시스템에 인증합니다.
