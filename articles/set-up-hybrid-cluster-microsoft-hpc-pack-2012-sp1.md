>Microsoft HPC Pack을 사용하여 하이브리드 계산 클러스터 설정 </h1 >  <p >이 자습서에서는 Microsoft HPC Pack 2012 R2 및 Azure를 사용하여 작은 하이브리드 HPC(고성능 컴퓨팅) 클러스터를 설정하는 방법을 보여 줍니다. 클러스터는 온-프레미스 헤드 노드(Windows Server 운영 체제와 HPC 팩이 실행되는 컴퓨터) 및 Azure 클라우드 서비스에 작업자 역할 인스턴스로 배포되는 일부 계산 노드로 구성됩니다. 그런 다음 하이브리드 클러스터에서 계산 작업을 실행할 수 있습니다. </p >  <p > <img src="./media/set-up-hybrid-cluster-microsoft-hpc-pack-2012-sp1/hybrid _cluster _overview.png" alt="하이브리드 HPC 클러스터" / > </p >  <p >이 자습서에서는 Azure의 확장 가능한 주문형 계산 리소스를 사용하여 계산이 많이 사용되는 응용 프로그램을 실행하는 방법을 보여 줍니다. 이 방법을 클러스터 "클라우드로 버스트"라고도 합니다. </p >  <p >이 자습서는 이전에 계산 클러스터나 HPC Pack을 사용한 경험이 없다고 가정합니다. 데모를 위해 하이브리드 계산 클러스터를 신속하게 배포하도록 도와주는 역할만 합니다. 하이브리드 HPC Pack 클러스터를 프로덕션 환경에 대규모로 배포하는 단계와 고려 사항은  <a href="http://go.microsoft.com/fwlink/p/?LinkID=200493" >자세한 지침 </a >을 참조하세요. </p >  <blockquote >  <p >[WACOM.NOTE] Azure는 각 작업에 적합한  <a href="http://go.microsoft.com/fwlink/p/?LinkId=389844" >다양한 크기 </a >를 계산 리소스에 제공합니다. 예를 들어 A8 및 A9 인스턴스는 특정 HPC 응용 프로그램에 필요한 낮은 대기 시간, 높은 처리량의 응용 프로그램 네트워크 액세스와 고성능을 결합합니다. 자세한 내용은  <a href="http://go.microsoft.com/fwlink/p/?Linkid=328042" >A8 및 A9 계산 집약적인 인스턴스 정보 </a >를 참조하세요. </p >  </blockquote >  <p >이 자습서에서는 다음 기본 단계를 단계별로 안내합니다. <blockquote >  <p >[WACOM.NOTE] 이 자습서를 완료하려면 Azure 계정이 필요합니다. 계정이 없는 경우 몇 분 만에 무료 평가판 계정을 만들 수 있습니다. 자세한 내용은  <a >Windows Server 2012 R2 또는 Windows Server 2012 버전이 실행되는 온-프레미스 컴퓨터. 이 컴퓨터가 HPC 클러스터의 헤드 노드가 됩니다. Windows Server 설치와 동일한 언어의 설치 파일을 선택합니다. </p > </li >  <li > <p >헤드 노드에서 로컬 관리자 권한을 가진 도메인 계정 </p > </li >  <li > <p >포트 443의 헤드 노드와 Azure 간 TCP 연결 </p > </li >  </ul >  <h2 id="BKMK _DeployHN" >헤드 >을 클릭합니다. </p >  <p > <img src="./media/set-up-hybrid-cluster-microsoft-hpc-pack-2012-sp1/config _hpc2.png" alt="네트워크 구성" / > </p > </li >  <li > <p >네트워크 구성 마법사에서  <strong >모든 노드가 엔터프라이즈 네트워크에만 있음
