# <a name="interpreting-your-scorecard"></a>성과 기록표 해석

성과 기록표 탭에는 테스트의 집계 및 분석 결과가 포함됩니다. 각 테스트에는 자체 성과 기록표가 있습니다. 성과 기록표는 네트워킹 요구 사항에 대한 데이터 기반 결과를 제공하기 위해 측정 결과의 빠르고 의미 있는 요약을 제공합니다. 인터넷 분석기는 분석을 처리하므로 의사 결정에 집중할 수 있습니다.

성과 기록표 탭은 인터넷 분석기 리소스 메뉴에서 찾을 수 있습니다. 


## <a name="filters"></a>필터

* ***테스트:*** 결과를 보려는 테스트를 선택합니다 - 각 테스트에는 자체 성과 기록표가 있습니다. 대부분의 경우 24시간 이내에 테스트 데이터가 나타나면 분석을 완료할 수 있습니다. 
* ***기간 & 종료 날짜:*** 스코어카드는 매일 3개의 스코어카드가 생성되며, 각 스코어카드는 이전 24시간 전(일), 7일 전(주), 30일 전(월)의 다른 집계 기간을 반영합니다. "종료 날짜" 필터를 사용하여 보려는 기간의 마지막 날을 선택합니다. 
* ***국가:*** 최종 사용자가 있는 각 국가에 대해 성과 기록표가 생성됩니다. 전역 필터에는 모든 최종 사용자가 포함됩니다.

## <a name="measurement-count"></a>측정 수

측정 횟수는 분석의 신뢰도에 영향을 미칩니다. 카운트가 높을수록 결과가 더 정확해집니다. 최소한 테스트는 엔드포인트당 최소 100개의 측정을 목표로 해야 합니다. 측정 수가 너무 적으면 응용 프로그램에서 더 자주 실행하도록 JavaScript 클라이언트를 구성하십시오. 끝점 A와 B의 측정 수는 작은 차이가 예상되고 괜찮지만 매우 유사해야 합니다. 큰 차이가 있는 경우 결과를 신뢰할 수 없습니다.

## <a name="percentiles"></a>백분위수

밀리초 단위로 측정되는 대기 시간은 인터넷에서 소스와 대상 간의 속도를 측정하는 데 널리 적용된 메트릭입니다. 지연 시간 데이터는 일반적으로 분산되지 않습니다(즉, "Bell Curve"를 따르지 않음) 산술 평균과 같은 통계를 사용할 때 발생하는 큰 대기 시간 값의 "긴 꼬리"가 있기 때문입니다. 대안으로 백분위수는 데이터를 분석하는 "배포없는" 방법을 제공합니다. 예를 들어 중앙값 또는 50번째 백분위수는 분포의 중간을 요약합니다 . 75번째 백분위수 값은 분포의 모든 값의 75% 보다 크다는 것을 의미합니다. 인터넷 분석기는 P50, P75 및 P95로 약어백분위수(p50)를 말합니다.

인터넷 분석기 백분위수는 _샘플 메트릭입니다._ 이는 실제 _인구 메트릭과_대조적입니다. 예를 들어 서던 캘리포니아 대학교와 Microsoft 학생 간의 일일 실제 인구 평균 대기 시간은 해당 기간 동안 의 모든 요청의 평균 대기 시간 값입니다. 실제로 모든 요청의 값을 측정하는 것은 비실용적이므로 합리적으로 큰 샘플이 실제 모집단을 대표한다고 가정합니다.

분석 목적을 위해 P50(중앙값)은 대기 시간 분포에 대한 예상 값으로 유용합니다. P95와 같은 높은 백분위수는 최악의 경우 대기 시간이 얼마나 높은지 식별하는 데 유용합니다. 일반적으로 고객 대기 시간을 이해하는 데 관심이 있다면 P50은 집중해야 할 올바른 메트릭입니다. 최악의 실적을 내는 고객의 성능을 이해하는 데 관심이 있다면 P95가 중점을 두어야 합니다. P75는 이 둘 사이의 균형입니다.


## <a name="deltas"></a>델타

델타는 끝점 A에 대한 메트릭 값의 차이이며, B. 델타는 A보다 B의 이점을 보여주기 위해 계산되지만 음수 값은 B의 성능이 더 나빠졌음을 나타냅니다. 델타는 절대(예: 10밀리초) 또는 상대(5%)일 수 있습니다.

## <a name="confidence-interval"></a>예측 

신뢰 구간(CI)은 중앙값, P75 또는 평균과 같은 모집단 메트릭을 포함할 확률이 있는 값의 범위입니다. 우리는 95% CI를 사용하는 일반적인 통계 규칙을 따릅니다.

인터넷 분석기의 경우 표본 메트릭이 실제 모집단 메트릭에 매우 근접할 가능성이 있으므로 신뢰도 간격이 좁습니다. 신뢰 구간이 넓기 때문에 샘플 메트릭이 실제 모집단 메트릭을 반영한다는 확신이 줄어듭니다. CI를 개선하는 가장 좋은 방법은 측정 수를 늘리는 것입니다.

## <a name="time-series"></a>시계열 

타임계는 시간에 따라 메트릭이 어떻게 변하는지 보여줍니다. 인터넷에는 피크 트래픽 기간, 평일-주말 인구 차이 및 공휴일과 같은 성능에 영향을 미치는 많은 시간적 요인이 있습니다.
