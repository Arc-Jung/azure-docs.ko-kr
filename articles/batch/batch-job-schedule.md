# <a name="schedule-jobs-for-efficiency"></a>효율성을 위한 작업 예약

일괄 처리 작업을 예약하면 다른 작업에 종속성이 있는 작업을 고려하면서 먼저 실행하려는 작업의 우선 순위를 지정할 수 있습니다. 작업을 예약하면 최소한의 리소스를 사용할 수 있습니다. 노드는 필요하지 않을 때 해제할 수 있으며, 다른 작업에 종속된 작업은 워크플로를 최적화하는 시간에 맞춰 스폰됩니다. 한 번에 하나의 작업만 실행됩니다. 새 새 는 이전 이 완료될 때까지 시작되지 않습니다. 작업을 자동 완성으로 설정할 수 있습니다. 

## <a name="benefit-of-job-scheduling"></a>작업 스케줄링의 이점

작업 예약의 이점은 작업 생성 일정을 지정할 수 있다는 것입니다. 작업 관리자 작업을 사용하여 예약하는 작업은 작업과 연결됩니다. 작업 관리자 작업은 작업에 대한 작업을 만듭니다. 이렇게 하려면 작업 관리자 작업이 Batch 계정으로 인증해야 합니다. AZ_BATCH_AUTHENTICATION_TOKEN 액세스 토큰을 사용합니다. 토큰은 작업의 나머지 부분에 대한 액세스를 허용합니다. 

## <a name="use-the-portal-to-schedule-a-job"></a>포털을 사용하여 작업 예약

   1. [Azure 포털에](https://portal.azure.com/)로그인합니다.

   2. 작업을 예약할 Batch 계정을 선택합니다.

   3. 새 작업 일정을 작성하고 **기본 양식을**작성하려면 **추가를** 선택합니다.



![작업 예약][1]

**작업 일정 ID**: 이 작업 일정에 대한 고유 식별자입니다.

**표시 이름**: 작업의 표시 이름은 고유할 필요는 없지만 최대 길이는 1024자입니다.

**실행 하지 마십시오:** 작업이 실행 될 가장 빠른 시간을 지정 합니다. 이 설정을 설정하지 않으면 일정이 즉시 작업을 실행할 준비가 됩니다.

**실행 안**: 여기에 설정한 시간 이후에 작업이 실행되지 않습니다. 시간을 지정하지 않으면 명시적으로 종료할 때까지 활성 상태로 유지되는 되풀이 작업 일정을 만듭니다.

**되풀이 간격**: 작업 사이의 시간을 지정할 수 있습니다. 예약된 한 번에 하나의 작업만 가질 수 있으므로 작업 일정에 따라 새 작업을 만들 시간이지만 이전 작업이 계속 실행 중인 경우 Batch 서비스는 이전 작업이 완료될 때까지 새 작업을 만들지 않습니다.  

**시작 창**: 일정이 작업을 만들어야 한다는 것을 나타내는 시간부터 완료될 때까지 시간 간격을 지정합니다. 창 에서 현재 작업이 완료되지 않으면 다음 작업이 시작되지 않습니다.

기본 양식의 맨 아래에 작업을 실행할 풀을 지정합니다. 풀 ID 정보를 찾으려면 **업데이트를**선택합니다. 

![풀 지정][2]


**풀 ID**: 작업을 실행할 풀을 식별합니다.

**작업 구성 작업**: **업데이트를** 선택하여 작업 관리자 작업과 작업 준비 및 릴리스 작업의 이름을 지정합니다.

**우선 순위**: 작업에 우선 순위를 부여합니다.

**최대 벽 시계 시간**: 작업이 실행할 수 있는 최대 시간을 설정합니다. 시간 프레임 내에 완료되지 않으면 일괄 처리가 작업을 종료합니다. 이 설정을 설정하지 않으면 작업에 대한 시간 제한이 없습니다.

**최대 작업 재시도 횟수**: 작업을 최대 4회까지 다시 시도할 수 있는 횟수를 지정합니다. 이는 API 호출이 가질 수 있는 재시도 횟수와 동일하지 않습니다.

**모든 작업이 완료되면**: 기본값은 작업이 없습니다.

**작업이 실패하는 경우**: 기본값은 작업이 아닙니다. 재시도 수가 소진되거나 작업을 시작할 때 오류가 있는 경우 작업이 실패합니다. 

**저장을**선택한 후 왼쪽 탐색에서 **작업 일정으로** 이동하면 **실행 정보를**선택하여 작업 실행을 추적할 수 있습니다.
