# <a name="copying-applications-and-data-to-pool-nodes"></a>풀 노드에 애플리케이션 및 데이터 복사

Azure Batch는 데이터 및 응용 프로그램을 태스크에서 사용할 수 있도록 계산 노드에 데이터 및 응용 프로그램을 가져오는 여러 가지 방법을 지원 합니다. 데이터와 응용 프로그램은 전체 작업을 실행 해야 할 수 있으므로 모든 노드에 설치 해야 합니다. 특정 작업에만 필요 하거나 작업에 대해 설치 해야 하지만 모든 노드에 있을 필요는 없습니다. Batch는 이러한 각 시나리오에 대 한 도구를 포함 합니다.

- **풀 시작 태스크 리소스 파일**: 풀의 모든 노드에 설치 해야 하는 응용 프로그램 또는 데이터의 경우 설치 명령을 수행 하기 위해 응용 프로그램 패키지 또는 시작 작업의 리소스 파일 컬렉션과 함께이 메서드를 사용 합니다.  

예: 
- 시작 태스크 명령줄을 사용 하 여 응용 프로그램 이동 또는 설치

- Azure storage 계정의 특정 파일 또는 컨테이너 목록을 지정 합니다. 자세한 내용은 [REST 설명서에서 # resourcefile 추가](https://docs.microsoft.com/rest/api/batchservice/pool/add#resourcefile) 를 참조 하세요.

- 풀에서 실행 되는 모든 작업은 먼저 MyApplication과 함께 설치 해야 하는 nuget.exe를 실행 합니다. 이 메커니즘을 사용 하는 경우 시작 작업의 **대기 대기** 속성을 **true**로 설정 해야 합니다. 자세한 내용은 [REST 설명서의 # starttask 추가](https://docs.microsoft.com/rest/api/batchservice/pool/add#starttask)를 참조 하세요.

- 풀의 **응용 프로그램 패키지 참조** : 풀의 모든 노드에 설치 해야 하는 응용 프로그램 또는 데이터 응용 프로그램 패키지와 연결 된 설치 명령은 없지만 시작 작업을 사용 하 여 설치 명령을 실행할 수 있습니다. 응용 프로그램에 설치가 필요 하지 않거나 많은 수의 파일로 구성 된 경우이 방법을 사용할 수 있습니다. 응용 프로그램 패키지는 많은 수의 파일 참조를 작은 페이로드로 결합 하므로 많은 수의 파일에 적합 합니다. 하나 100 이상의 개별 리소스 파일을 하나의 작업에 포함 하려고 하면 일괄 처리 서비스가 단일 작업에 대 한 내부 시스템 제한에 대해 발생할 수 있습니다. 또한 동일한 응용 프로그램의 여러 버전을 포함 하 고 둘 중 하나를 선택 해야 하는 엄격한 버전 관리 요구 사항이 있는 경우 응용 프로그램 패키지를 사용 합니다. 자세한 내용은 [Batch 응용 프로그램 패키지를 사용 하 여 계산 노드에 응용 프로그램 배포](https://docs.microsoft.com/azure/batch/batch-application-packages)를 참조 하세요.

- 작업 **준비 태스크 리소스 파일**: 작업을 실행 하기 위해 설치 해야 하지만 전체 풀에는 설치 하지 않아도 되는 응용 프로그램 또는 데이터입니다. 예를 들어, 풀에 다양 한 유형의 작업이 있고 한 작업 유형만 MyApplication을 실행 해야 하는 경우 설치 단계를 작업 준비 태스크에 배치 하는 것이 좋습니다. 작업 준비 태스크에 대 한 자세한 내용은 [Batch 계산 노드에서 작업 준비 및 작업 릴리스 태스크 실행](https://azure.microsoft.com/documentation/articles/batch-job-prep-release/)을 참조 하세요.

- **작업 리소스 파일**: 응용 프로그램이 나 데이터가 개별 작업에만 관련 된 경우에 해당 합니다. 예를 들어 5 개의 작업이 있으며 각 작업은 서로 다른 파일을 처리 한 다음 blob storage에 출력을 기록 합니다.  이 경우 각 작업에 고유한 입력 파일이 있기 때문에 **작업 리소스 파일** 컬렉션에 입력 파일을 지정 해야 합니다.

## <a name="determine-the-scope-required-of-a-file"></a>파일에 필요한 범위 확인

풀, 작업 또는 태스크에 필요한 파일의 범위를 결정 해야 합니다. 풀로 범위가 지정 된 파일은 풀 응용 프로그램 패키지 또는 시작 태스크를 사용 해야 합니다. 작업 범위를 지정 하는 파일은 작업 준비 태스크를 사용 해야 합니다. 풀 또는 작업 수준으로 범위가 지정 된 파일의 좋은 예는 응용 프로그램입니다. 작업 범위를 지정 하는 파일은 작업 리소스 파일을 사용 해야 합니다.

### <a name="other-ways-to-get-data-onto-batch-compute-nodes"></a>일괄 처리 계산 노드에 데이터를 가져오는 다른 방법

일괄 처리 REST API에 공식적으로 통합 되지 않은 일괄 처리 계산 노드로 데이터를 가져오는 다른 방법이 있습니다. Azure Batch 노드를 제어 하 고 사용자 지정 실행 파일을 실행할 수 있기 때문에 일괄 처리 노드가 대상에 연결 되어 있고 해당 원본에 대 한 자격 증명이 Azure Batch 노드에 있는 한, 원하는 수의 사용자 지정 소스에서 데이터를 끌어올 수 있습니다. 몇 가지 일반적인 예는 다음과 같습니다.

- SQL에서 데이터 다운로드
- 다른 웹 서비스/사용자 지정 위치에서 데이터 다운로드
- 네트워크 공유 매핑

### <a name="azure-storage"></a>Azure 스토리지

Blob storage에서 확장성 목표를 다운로드 합니다. Azure storage 파일 공유 확장성 목표는 단일 blob의 경우와 동일 합니다. 크기는 필요한 노드 및 풀의 수에 영향을 줍니다.
