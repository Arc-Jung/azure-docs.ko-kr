# <a name="copy-applications-and-data-to-pool-nodes"></a>풀 노드에 애플리케이션 및 데이터 복사

Azure Batch는 태스크에서 사용할 수 있도록 데이터 및 응용 프로그램을 계산 노드로 가져오는 여러 가지 방법을 지원 합니다.

선택 하는 방법은 파일 또는 응용 프로그램의 범위에 따라 달라질 수 있습니다. 데이터와 응용 프로그램은 전체 작업을 실행 해야 할 수 있으므로 모든 노드에 설치 해야 합니다. 일부 파일 또는 응용 프로그램은 특정 작업에만 필요할 수 있습니다. 다른 항목은 작업에 대해 설치 해야 하지만 모든 노드에 있을 필요는 없습니다. Batch에는 이러한 각 시나리오에 대한 도구가 있습니다.

## <a name="determine-the-scope-required-of-a-file"></a>파일의 필요 범위 결정

풀, 작업 또는 태스크에 필요한 파일의 범위를 결정해야 합니다. 풀로 범위가 지정된 파일은 풀 애플리케이션 패키지 또는 시작 태스크를 사용해야 합니다. 작업으로 범위가 지정된 파일은 작업 준비 태스크를 사용해야 합니다. 풀 또는 작업 수준으로 범위가 지정된 파일의 좋은 예는 애플리케이션입니다. 태스크로 범위가 지정된 파일은 태스크 리소스 파일을 사용해야 합니다.

## <a name="pool-start-task-resource-files"></a>풀 시작 태스크 리소스 파일

풀의 모든 노드에 설치 해야 하는 응용 프로그램 또는 데이터의 경우 풀 시작 작업 리소스 파일을 사용 합니다. 설치 명령을 수행 하기 위해 [응용 프로그램 패키지](batch-application-packages.md) 또는 시작 작업의 리소스 파일 컬렉션과 함께이 메서드를 사용 합니다.  

예를 들어 시작 태스크 명령줄을 사용 하 여 응용 프로그램을 이동 하거나 설치할 수 있습니다. 또한 Azure storage 계정에서 파일 또는 컨테이너의 목록을 지정할 수 있습니다. 자세한 내용은 [REST 설명서에서 # ResourceFile 추가](/rest/api/batchservice/pool/add#resourcefile)를 참조 하세요.

풀에서 실행 되는 모든 작업에서 .msi 파일을 사용 하 여 먼저 설치 해야 하는 응용 프로그램 (.exe)을 실행 하는 경우 시작 작업의 **대기 대기** 속성을 **true** 로 설정 해야 합니다. 자세한 내용은 [REST 설명서에서 # StartTask 추가](/rest/api/batchservice/pool/add#starttask)를 참조 하세요.

## <a name="application-package-references"></a>애플리케이션 패키지 참조

풀의 모든 노드에 설치 해야 하는 응용 프로그램 또는 데이터의 경우 [응용 프로그램 패키지](batch-application-packages.md)를 사용 하는 것이 좋습니다. 애플리케이션 패키지에 연결된 설치 명령은 없으나 시작 태스크를 사용하여 모든 설치 명령을 실행할 수 있습니다. 애플리케이션이 설치가 필요하지 않거나 많은 파일로 구성된 경우 이 메서드를 사용할 수 있습니다.

응용 프로그램 패키지는 많은 파일 참조를 작은 페이로드로 결합할 수 있으므로 많은 수의 파일이 있는 경우 유용 합니다. 100개가 넘는 개별 리소스 파일을 한 태스크에 포함하려는 경우, Batch 서비스가 단일 태스크에 대한 내부 시스템 제한에 대한 답이 될 수 있습니다. 응용 프로그램 패키지는 동일한 응용 프로그램의 다양 한 버전이 있고 이러한 응용 프로그램 중 하나를 선택 해야 하는 경우에도 유용 합니다.

## <a name="extensions"></a>확장

[확장](create-pool-extensions.md) 은 Batch 계산 노드에서 사후 프로 비전 구성 및 설치를 용이 하 게 하는 작은 응용 프로그램입니다. 풀을 만들 때 지원 되는 확장을 선택 하 여 프로 비전 될 때 계산 노드에 설치할 수 있습니다. 그런 다음 확장에서 원하는 작업을 수행할 수 있습니다.

## <a name="job-preparation-task-resource-files"></a>작업 준비 태스크 리소스 파일

작업을 실행 하기 위해 설치 해야 하지만 전체 풀에 설치 하지 않아도 되는 응용 프로그램 또는 데이터의 경우 [작업 준비 작업 리소스 파일](./batch-job-prep-release.md)을 사용 하는 것이 좋습니다.

예를 들어 풀에 다양 한 유형의 작업이 있고 실행 하기 위해 하나의 작업 유형만 .msi 파일이 필요한 경우 설치 단계를 작업 준비 태스크에 배치 하는 것이 좋습니다.

## <a name="task-resource-files"></a>태스크 리소스 파일

작업 리소스 파일은 응용 프로그램 또는 데이터가 개별 작업에만 관련 된 경우에 적합 합니다.

예를 들어, 각각 다른 파일을 처리 하 고 blob storage에 출력을 기록 하는 5 개의 작업이 있을 수 있습니다 .이 경우 각 작업에 고유한 입력 파일이 있기 때문에 작업 리소스 파일 컬렉션에 입력 파일을 지정 해야 합니다.

## <a name="additional-ways-to-get-data-onto-nodes"></a>노드에 데이터를 가져오는 추가 방법

Azure Batch 노드를 제어 하 고 사용자 지정 실행 파일을 실행할 수 있기 때문에 원하는 수의 사용자 지정 소스에서 데이터를 끌어올 수 있습니다. 일괄 처리 노드가 대상에 연결 되어 있고 노드의 해당 원본에 대 한 자격 증명이 있는지 확인 합니다.

일괄 처리 노드에 데이터를 전송 하는 몇 가지 방법의 예는 다음과 같습니다.

- SQL에서 데이터 다운로드
- 다른 웹 서비스/사용자 지정 위치에서 데이터 다운로드
- 네트워크 공유 매핑

## <a name="azure-storage"></a>Azure Storage

Blob storage는 확장성 목표를 다운로드 한다는 점에 유의 하세요. Azure Storage 파일 공유 확장성 목표는 단일 Blob의 경우와 동일합니다. 크기는 필요한 노드 및 풀의 수에 영향을 줍니다.
