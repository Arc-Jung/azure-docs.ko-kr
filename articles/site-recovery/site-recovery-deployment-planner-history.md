# <a name="azure-site-recovery-deployment-planner-version-history"></a>Azure 사이트 복구 배포 플래너 버전 기록

이 문서에서는 Azure 사이트 복구 배포 플래너의 모든 버전에 대한 기록과 함께 각 릴리스 날짜의 알려진 제한 사항및 수정 사항을 제공합니다.

## <a name="version-251"></a>버전 2.51

**릴리스 날짜: 2019년 8월 22일**

**수정:**

- 배포 플래너 버전 2.5에서 비용 권장 문제를 수정했습니다.

## <a name="version-25"></a>버전 2.5

**릴리스 날짜: 2019년 7월 29일**

**수정:**

- VMware 가상 컴퓨터 및 물리적 컴퓨터의 경우 관리되는 디스크에 대한 복제를 기반으로 하는 것이 좋습니다.
- 윈도우에 대 한 추가 지원 10 (x64), 윈도우 8.1 (x64), 윈도우 8 (x64), 윈도우 7 (x64) SP1 이상

## <a name="version-24"></a>버전 2.4

**릴리스 날짜: 2019년 4월 17일**

**수정:**

- 특히 지역화 기반 오류를 처리할 때 운영 체제 호환성이 향상되었습니다.
- 호환성 검사 목록에 최대 20Mbps의 데이터 변경 률(변동)이 있는 VM을 추가했습니다.
- 향상된 오류 메시지
- vCenter 6.7에 대한 지원이 추가되었습니다.
- Windows 서버 2019 및 레드 햇 엔터프라이즈 리눅스 (RHEL) 워크 스테이션에 대 한 지원을 추가 했습니다.



## <a name="version-23"></a>버전 2.3

**릴리스 날짜: 2018년 12월 3일**

**수정:**

- 배포 플래너가 제공된 대상 위치 및 구독으로 보고서를 생성하지 못하는 문제를 수정했습니다.

## <a name="version-22"></a>버전 2.2 

**릴리스 날짜: 2018년 4월 25일**

**수정:**

- GetVMList 작업:
  - 지정된 폴더가 없는 경우 GetVMList가 실패하는 문제를 수정했습니다. 이제 기본 디렉토리를 만들거나 outputfile 매개 변수에 지정된 디렉토리를 만듭니다.
  - GetVMList에 대한 자세한 실패 이유를 추가했습니다.
- 배포 플래너 보고서의 호환되는 VM 시트에 VM 형식 정보를 열로 추가했습니다.
- 하이퍼-V에서 Azure 재해 복구까지:
  - 프로파일링에서 공유 VHD 및 PassThrough 디스크가 있는 제외된 VM입니다. Startprofiling 작업에는 콘솔에서 제외된 VM 목록이 표시됩니다.
  - 호환되지 않는 VM 목록에 64개 이상의 디스크가 있는 VM을 추가했습니다.
  - 초기 복제(IR) 및 델타 복제(DR) 압축 계수를 업데이트했습니다.
  - SMB 저장소에 대한 제한된 지원이 추가되었습니다.

## <a name="version-21"></a>버전 2.1

**릴리스 날짜: 2018년 1월 3일**

**수정:**

- Excel 보고서를 업데이트했습니다.
- GetThroughput 작업에서 버그를 수정했습니다.
- 보고서를 프로파일로 작성하거나 생성하는 VM 수를 제한하는 옵션이 추가되었습니다. 기본 한도는 1,000VM입니다.
- VM웨어에서 Azure 재해 복구까지:
  - 호환되지 않는 테이블에 Windows Server 2016 VM 문제가 해결되었습니다. 
  - 확장 가능한 펌웨어 인터페이스(EFI) Windows VM에 대한 호환성 메시지가 업데이트되었습니다.
- VM웨어를 Azure및 하이퍼-V에서 Azure로 업데이트했으며, VM당 VM 데이터 변동 제한을 업데이트했습니다. 
- VM 목록 파일 구문 분석의 안정성이 향상되었습니다.

## <a name="version-201"></a>버전 2.0.1

**릴리스 날짜: 2017년 12월 7일**

**수정:**

- 네트워크 대역폭을 최적화하기 위한 권장 사항이 추가되었습니다.

## <a name="version-20"></a>버전 2.0

**릴리스 날짜: 2017년 11월 28일**

**수정:**

- Azure 재해 복구에 Hyper-V에 대한 지원이 추가되었습니다.
- 추가 비용 계산기.
- VM웨어에 대한 OS 버전 검사를 Azure 재해 복구에 추가하여 VM이 보호에 호환되는지 호환되지 않는지 확인합니다. 이 도구는 해당 VM에 대한 vCenter 서버에서 반환되는 OS 버전 문자열을 사용합니다. VMware에서 VM을 만드는 동안 사용자가 선택한 게스트 운영 체제 버전입니다.

**알려진 제한 사항:**

- 하이퍼-V에서 Azure 재해 복구의 경우 이름이 다음과 같은 `,` `"`이름의 `[` `]`VM: 에 대한 및 ``` ` ``` 지원되지 않습니다. 프로파일이 지정되면 보고서 생성이 실패하거나 잘못된 결과가 발생합니다.
- VMware에서 Azure 재해 복구에 대 한 쉼표가 포함 된 VM 지원 되지 않습니다. 프로파일이 지정되면 보고서 생성이 실패하거나 잘못된 결과가 발생합니다.

## <a name="version-131"></a>버전 1.3.1

## <a name="version-13"></a>버전 1.3

**릴리스 날짜: 2017년 5월 9일**

**수정:**

- 보고서 생성시 관리 디스크에 대한 지원이 추가되었습니다. 단일 저장소 계정에 배치할 수 있는 VM 수는 장애 조치/테스트 장애 조치(Failover)에 대해 관리 디스크를 선택한 경우에 따라 계산됩니다.

## <a name="version-12"></a>버전 1.2

**릴리스 날짜: 2017년 4월 7일**

**수정:**

- 추가된 부팅 유형(BIOS 또는 EFI)은 각 VM에 대해 VM이 보호에 호환되는지 호환되지 않는지 확인합니다.
- 호환되는 VM 및 호환되지 않는 VM 워크시트에서 각 가상 컴퓨터에 대한 OS 유형 정보를 추가했습니다.
- 미국 정부 및 중국 Microsoft Azure 지역에 대한 GetThroughput 작업에 대한 지원이 추가되었습니다.
- vCenter 및 ESXi 서버에 대한 몇 가지 추가 필수 요건이 추가되었습니다.
- 로캘 설정을 영어 이외의 것으로 설정하면 잘못된 보고서가 생성되던 문제를 수정했습니다.

## <a name="version-11"></a>버전 1.1

**릴리스 날짜: 2017년 3월 9일**

**수정:**

- 다양한 vCenter ESXi 호스트에서 이름이나 IP 주소가 같은 VM이 두 개 이상 있는 경우 VM 프로파일링을 방지하지 못하는 문제를 수정했습니다.
- 호환되는 VM 및 호환되지 않는 VM 워크시트에 대해 복사 및 검색을 비활성화하는 문제를 수정했습니다.

## <a name="version-10"></a>버전 1.0

**릴리스 날짜: 2017년 2월 23일**

**알려진 제한 사항:**

- Azure 재해 복구 시나리오에 대한 VMware에 대해서만 지원합니다. 하이퍼-V에서 Azure 재해 복구 시나리오의 경우 [하이퍼-V 용량 플래너 도구를](./site-recovery-capacity-planning-for-hyper-v-replication.md)사용합니다.
- 미국 정부 및 중국 Microsoft Azure 지역에 대한 GetThroughput 작업을 지원하지 않습니다.
- vCenter 서버에 다양한 ESXi 호스트에서 이름이나 IP 주소가 같은 두 개 이상의 VM이 있는 경우 이 도구는 VM을 프로파일화할 수 없습니다.
이 버전에서 도구는 VMListFile에서 중복된 VM 이름 또는 IP 주소에 대한 프로파일링을 건너뜁니다. 해결 방법은 VCenter 서버 대신 ESXi 호스트를 사용하여 VM을 프로파일링하는 것입니다. 각 ESXi 호스트에 대해 하나의 인스턴스를 실행해야 합니다.
