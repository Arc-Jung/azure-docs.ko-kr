#탄력적인 확장 용어집
다음은 Azure SQL 데이터베이스의 탄력적인 확장 기능에 대해 정의된 용어입니다.

![Elastic Scale terms][1]

**데이터베이스**: Azure SQL 데이터베이스입니다. 

**데이터 종속 라우팅**: 특정 분할 키가 있는 경우 응용 프로그램에서 분할된 데이터베이스에 연결할 수 있는 기능입니다. **다중 분할된 데이터베이스 쿼리**와 비교됩니다.

**전역 분할된 데이터베이스 맵**: **분할 집합** 내에서 분할 키와 각 분할된 데이터베이스 간의 매핑 집합입니다. GSM은 **분할된 데이터베이스 맵 관리자**에 저장됩니다. **로컬 분할된 데이터베이스 맵**을 비교합니다.

**목록 분할된 데이터베이스 맵**: 분할 키가 개별적으로 매핑되는 분할된 데이터베이스 맵입니다. **범위 분할된 데이터베이스 맵**과 비교됩니다.   

**로컬 분할된 데이터베이스 맵**: 분할된 데이터베이스에 저장되는, 로컬 분할된 데이터베이스 맵은 분할된 데이터베이스에 상주하는 shardlet에 대한 매핑을 포함합니다.

**다중 분할된 데이터베이스 쿼리**: 여러 분할된 데이터베이스에 대해 쿼리를 실행하는 기능이 UNION ALL 의미 체계(또는 "팬아웃 쿼리")를 사용하여 반환됩니다. **데이터 종속 라우팅**과 비교됩니다.

**범위 분할된 데이터베이스 맵**: 분할된 데이터베이스 배포 전략이 인접 값의 여러 범위를 기반으로 하는 분할된 데이터베이스 맵입니다. 

**참조 테이블**: 분할되지 않았지만 분할된 데이터베이스에 걸쳐 복제되는 테이블입니다. 

**분할된 데이터베이스**: 분할된 데이터 집합의 데이터를 저장하는 Azure SQL 데이터베이스입니다. 

**SE(분할된 데이터베이스 탄력성)**: **수평적 크기 조정** 및 **수직적 크기 조정** 모두를 수행하는 기능입니다.

**분할된 테이블**: 분할된 테이블, 즉, 그 데이터가 분할 키 값에 따라 분할된 데이터베이스에 걸쳐 분산됩니다. 

**분할 키**: 분할된 데이터베이스 간에 데이터를 배포하는 방법을 결정하는 열 값입니다. 값 형식은 int, bigint, varbinary 또는 고유 식별자 중 하나일 수 있습니다. 

**분할 집합**: 분할된 데이터베이스 맵 관리자에서 동일한 분할된 데이터베이스 맵 특성을 가진 분할된 데이터베이스의 컬렉션입니다.  

**Shardlet**: 분할된 데이터베이스에서 분할 키의 단일 값과 연결된 모든 데이터입니다. Shardlet은 분할된 테이블을 재배포하는 경우 데이터 이동 가능한 가장 작은 단위입니다. 

**분할된 데이터베이스 맵**: 분할 키와 각 분할된 데이터베이스 간의 매핑 집합입니다.

**분할된 데이터베이스 맵 관리자**: 분할된 데이터베이스 맵, 분할된 데이터베이스 위치 및 하나 이상의 분할된 데이터베이스 집합에 대한 매핑을 포함하는 데이터 저장소 및 관리 개체입니다.


##동사

**수평적 크기 조정**: 분할된 데이터베이스 맵에서 분할된 데이터베이스를 추가 또는 제거하여 분할된 데이터베이스의 컬렉션을 확장(또는 축소)하는 동작입니다.

**병합**: 두 개의 분할된 데이터베이스에서 한 분할된 데이터베이스로 shardlet을 이동하고 그에 따라 분할된 데이터베이스 맵을 업데이트하는 동작입니다.

**Shardlet 이동**: 단일 shardlet을 여러 분할된 데이터베이스로 이동하는 동작입니다. 

**분할된 데이터베이스**: 분할 키를 기반으로 여러 데이터베이스 간에 동일하게 구조화된 데이터를 수평적으로 분할하는 동작입니다.

**분할**: 한 분할된 데이터베이스에서 다른(일반적으로 새로운) 분할된 데이터베이스로 몇 개의 shardlet을 이동하는 동작입니다. 분할 키는 사용자가 분할 지점으로 제공합니다.

**수직적 크기 조정**: 개별 분할된 데이터베이스의 성능 수준을 확장(또는 축소)하는 동작입니다. 예를 들어, 분할된 데이터베이스를 Standard에서 Premium으로 변경합니다(성능상의 이유로 필요한 경우).
