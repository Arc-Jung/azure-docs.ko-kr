# <a name="gain-insight-into-global-regional-and-local-threats"></a>전역, 지역 및 로컬 위협에 대한 인사이트 확보

온-프레미스 관리 콘솔의 사이트 맵은 사용자의 비즈니스 토폴로지를 반영 하는 지리적 및 논리적 세그먼트로 네트워크를 분할 하 여 전체 보안 범위를 달성할 수 있도록 도와줍니다.

- **지리적 기능 수준**: 사이트는 맵에 표시 된 지리적 위치에 따라 그룹화 된 여러 장치를 반영 합니다. 기본적으로 Azure Defender for IoT는 세계 지도를 제공 합니다. 조직이 나 비즈니스 구조를 반영 하도록 지도를 업데이트 합니다. 예를 들어 특정 국가, 구/군/시 또는 산업 캠퍼스의 사이트를 반영 하는 지도를 사용 합니다. 지도에서 사이트 색이 변경 되 면 SOC 팀은 기능에서 중요 한 시스템 상태를 나타냅니다.

  지도는 대화형 이며 각 사이트를 열고이 사이트의 정보에 살펴보기 수 있습니다.

- **글로벌 논리 계층**: 비즈니스 단위는 특정 산업에 따라 엔터프라이즈를 논리적 세그먼트로 분할 하는 방법입니다. 이렇게 하면 비즈니스 토폴로지가 맵에 반영 됩니다.

  예를 들어 투명 효과 팩터리, 플라스틱 팩터리 및 자동차 팩터리를 포함 하는 글로벌 회사는 세 개의 다른 사업부로 관리할 수 있습니다. 위치에 있는 물리적 사이트에는 세 가지 투명 효과 생산 선, 플라스틱 생산 라인 및 트럭 엔진 생산 선이 포함 됩니다. 따라서이 사이트에는 세 가지 사업부 모두의 담당자가 있습니다.

- **지리적 영역 수준**: 글로벌 엔터프라이즈를 지리적 지역으로 나누는 지역을 만듭니다. 예를 들어 설명한 회사는 북아메리카, 서유럽 및 동유럽 지역을 사용할 수 있습니다. 북아메리카에는 세 가지 사업부의 팩터리가 있습니다. 서유럽에는 자동차 팩터리 및 유리 팩터리가 있고 동유럽에는 플라스틱 팩터리가 있습니다.

- **로컬 논리 세그먼트 수준**: 영역은 기능 영역 또는 생산 라인 등을 정의 하는 사이트 내의 논리적 세그먼트입니다. 영역 작업을 통해 영역 정의와 관련 된 보안 정책을 적용할 수 있습니다. 예를 들어 5 개 생산 라인을 포함 하는 사이트는 5 개의 영역으로 분할 될 수 있습니다.

- **로컬 뷰 수준**: 단일 센서 설치의 로컬 보기는 연결 된 장치의 운영 및 보안 상태에 대 한 통찰력을 제공 합니다.

## <a name="work-with-site-map-views"></a>사이트 맵 보기 작업

온-프레미스 관리 콘솔은 상황에 맞는 맵에서 산업 네트워크의 전체 보기를 제공 합니다. 일반 지도 보기는 각 사이트의 지리적 위치를 사용 하 여 조직의 글로벌 지도를 표시 합니다.

:::image type="content" source="media/how-to-work-with-maps/enterprise.png" alt-text="엔터프라이즈 맵 보기의 스크린샷":::

### <a name="color-coded-map-views"></a>색으로 구분 된 지도 보기

**녹색**: 보안 이벤트 수가 Defender for IoT가 시스템에 대해 정의한 임계값 보다 작습니다. 어떤 조치가 필요하지 않습니다.

**노랑**: 보안 이벤트 수는 Defender for IoT가 시스템에 대해 정의한 임계값과 같습니다. 이벤트를 조사 하는 것이 좋습니다.  

**빨강**: 보안 이벤트 수가 Defender for IoT가 시스템에 대해 정의한 임계값을 초과 합니다. 즉각적인 조치를 취합니다.

### <a name="risk-level-map-views"></a>위험 수준 지도 보기

**위험 평가**: 위험 평가 보기에는 사이트 위험에 대 한 정보가 표시 됩니다. 위험 정보는 완화의 우선 순위를 지정 하 고도로 지도를 빌드하여 보안 개선을 계획 합니다.

**인시던트 응답**: 기업 전체의 각 사이트에서 승인 되지 않은 모든 경고를 중앙 집중식으로 확인 합니다. 특정 사이트에서 검색 된 경고를 드릴 다운 하 고 관리할 수 있습니다.

:::image type="content" source="media/how-to-work-with-maps/incident-responses.png" alt-text="인시던트 응답이 있는 Enterprise 맵 보기의 스크린샷":::

**악의적인 활동**: 맬웨어가 검색 된 경우 사이트는 빨강으로 표시 됩니다. 이는 즉각적인 조치를 취해야 함을 나타냅니다.

:::image type="content" source="media/how-to-work-with-maps/malicious-activity.png" alt-text="악의적인 활동을 포함 하는 엔터프라이즈 맵 보기의 스크린샷":::

**운영 경고**: ot 시스템에 대 한이 지도 보기는 PLC 중지, 펌웨어 업로드 및 프로그램 업로드와 같은 운영 인시던트가 발생할 수 있는 모든 시스템을 더 잘 이해할 수 있도록 합니다.

:::image type="content" source="media/how-to-work-with-maps/operational-alerts.png" alt-text="운영 경고를 포함 하는 엔터프라이즈 맵 보기의 스크린샷":::

지도 보기를 선택 하려면:

1. 맵에서 **기본 보기** 를 선택 합니다.
2. 보기를 선택합니다.

:::image type="content" source="media/how-to-work-with-maps/map-views.png" alt-text="사이트 맵 기본 보기의 스크린샷":::

## <a name="update-the-site-map-image"></a>사이트 맵 이미지 업데이트

IoT 용 Defender는 기본 세계 지도를 제공 합니다. 예를 들어 국가 지도 나 도시 지도를 반영 하도록 변경할 수 있습니다. 

맵을 바꾸려면 다음을 수행 합니다.

1. 왼쪽 창에서 **시스템 설정** 을 선택 합니다.

2. **사이트 맵 변경** 을 선택 하 고 그래픽 파일을 업로드 하 여 기본 맵을 바꿉니다.
