# <a name="work-with-device-notifications"></a>디바이스 알림 작업

알림은 주의가 필요한 네트워크 작업에 대 한 정보를 제공 하며,이 작업을 처리 하는 데 필요한 권장 사항도 제공 합니다. 예를 들어 다음에 대 한 알림을 받을 수 있습니다.

- 비활성 장치입니다. 장치가 더 이상 네트워크에 속해 있지 않으면 제거할 수 있습니다. 장치가 비활성 상태 이면 (예: 네트워크에서 실수로 연결이 끊어져) 장치를 다시 연결 하 고 알림을 해제 합니다.

- 현재 MAC 주소로 식별 되는 장치에서 IP 주소가 검색 되었습니다. 장치에 대 한 IP 주소를 부여 하 여 응답 합니다.

알림에 응답 하면 장치 맵, 장치 인벤토리, 데이터 마이닝 쿼리 및 보고서에 제공 되는 정보가 향상 됩니다. 또한 합법적인 네트워크 변경에 대 한 통찰력 및 잠재적인 네트워크 구성을 제공 합니다.

**알림 및 경고**

네트워크 활동에 대 한 알림을 수신 하는 것 외에도 *경고가* 표시 될 수 있습니다. 알림은 위협을 제공 하지 않는 네트워크 변경 또는 확인 되지 않은 장치 속성에 대 한 정보를 제공 합니다. 경고는 네트워크의 위협에 노출 될 수 있는 네트워크 편차 및 변경에 대 한 정보를 제공 합니다.

알림을 보려면:

1. 콘솔의 왼쪽 메뉴 창에서 **장치 맵** 을 선택 합니다.

2. **알림** 아이콘을 선택 합니다. 아이콘 위의 빨간색 숫자는 알림 수를 나타냅니다.

   :::image type="content" source="media/how-to-enrich-asset-information/notifications-alert-screenshot.png" alt-text="알림 아이콘.":::

   **알림** 창에 센서가 감지한 모든 알림이 표시 됩니다.

   :::image type="content" source="media/how-to-enrich-asset-information/notification-screen.png" alt-text="알림.":::

## <a name="filter-the-notifications-window"></a>알림 창 필터링

검색 필터를 사용 하 여 관심 있는 알림을 표시할 수 있습니다.

| 필터링 기준 | Description |
|--|--|
| 유형별로 필터링 | 특정 관심 영역을 포함 하는 알림을 봅니다. 예를 들어 비활성 장치에 대 한 알림만 표시 합니다. |
| 날짜 범위로 필터링 | 특정 시간 범위를 포함 하는 알림을 표시 합니다. 예를 들어 지난 주에만 전송 된 알림을 봅니다. |
| 특정 정보 검색 | 특정 알림을 검색 합니다. |

## <a name="notification-events-and-responses"></a>알림 이벤트 및 응답

다음 표에서는이를 처리 하는 옵션과 함께 수신할 수 있는 알림 이벤트 유형을 설명 합니다. 권장 값을 사용 하 여 장치 정보를 업데이트 하거나 알림을 해제할 수 있습니다. 알림을 해제 하면 장치 정보가 권장 된 정보로 업데이트 되지 않습니다. 트래픽이 다시 검색 되 면 알림이 다시 전송 됩니다.

| 알림 이벤트 유형 | Description | 응답 |
|--|--|--|
| 새 IP 검색 됨 | 새 IP 주소가 장치와 연결 됩니다. 5 개의 시나리오가 검색 될 수 있습니다. <br /><br /> 추가 IP 주소가 장치와 연결 되어 있습니다. 이 장치는 기존 MAC 주소와도 연결 되어 있습니다.<br /><br /> 기존 MAC 주소를 사용 하는 장치에 대 한 새 IP 주소가 검색 되었습니다. 현재 장치는 IP 주소를 사용 하 여 통신 하지 않습니다.<br /> <br /> NetBIOS 이름을 사용 하는 장치에 대 한 새 IP 주소가 검색 되었습니다. <br /><br /> IP 주소가 MAC 주소와 연결 된 장치에 대 한 관리 인터페이스로 검색 되었습니다. <br /><br /> 가상 IP 주소를 사용 하는 장치에 대 한 새 IP 주소가 검색 되었습니다. | **추가 IP를 장치** (병합 장치)로 설정 <br /> <br />**기존 IP 바꾸기** <br /> <br /> **해제**<br /> 알림을 제거 합니다. |
| 비활성 장치 | 장치에서 60 일 넘게 트래픽이 검색 되지 않았습니다. | **삭제** <br /> 이 장치가 네트워크의 일부가 아니면 제거 합니다. <br /><br />**해제** <br /> 장치가 네트워크에 속해 있는 경우 알림을 제거 합니다. 장치가 비활성 상태 이면 (예: 네트워크에서 실수로 연결이 끊어져) 알림을 해제 하 고 장치를 다시 연결 합니다. |
| 새 OT 장치 | 서브넷에 ICS 서브넷에 정의 되지 않은 OT 장치가 포함 되어 있습니다. <br /><br /> 하나 이상의 OT 장치를 포함 하는 각 서브넷은 ICS 서브넷으로 정의할 수 있습니다. 이는 지도에서 OT와 IT 장치를 구분 하는 데 도움이 됩니다. | **ICS 서브넷으로 설정** <br /> <br /> **해제** <br />장치가 서브넷의 일부가 아닌 경우 알림을 제거 합니다. |
| 구성 된 서브넷 없음 | 현재 네트워크에 구성 된 서브넷이 없습니다. <br /><br /> 지도에서 더 나은 표시를 위해 서브넷을 구성 하 고, OT 및 IT 장치를 구분 하는 기능을 구성 합니다. | **서브넷 구성을 열고** 서브넷을 구성 합니다. <br /><br />**해제** <br /> 알림을 제거 합니다. |
| 운영 체제 변경 | 하나 이상의 새 운영 체제가 장치와 연결 되어 있습니다. | 장치와 연결할 새 OS의 이름을 선택 합니다.<br /><br /> **해제** <br /> 알림을 제거 합니다. |
| 새 서브넷 | 새 서브넷이 검색 되었습니다. | **Learn**<br />서브넷을 자동으로 추가 합니다.<br />**서브넷 구성 열기**<br />누락 된 모든 서브넷 정보를 추가 합니다.<br />**해제**<br />알림을 제거 합니다. |
| 장치 유형 변경 | 새 장치 유형이 장치와 연결 되어 있습니다. | **{...} (으)로 설정**<br />새 형식을 장치와 연결 합니다.<br />**해제**<br />알림을 제거 합니다. |

## <a name="respond-to-many-notifications-simultaneously"></a>동시에 여러 알림에 응답

여러 알림을 동시에 처리 해야 할 수도 있습니다. 예를 들면 다음과 같습니다.

- 대량 네트워크 서버 집합으로 OS를 업그레이드 한 경우 업그레이드 된 모든 서버에 대 한 새 서버 버전을 학습할 센서에 지시할 수 있습니다. 

- 특정 줄의 장치 그룹이 단계적으로 진행 되 고 더 이상 활성 상태가 아닌 경우 콘솔에서 이러한 장치를 제거 하도록 센서에 지시할 수 있습니다.

새로 검색 된 정보를 여러 장치에 적용 하거나 무시 하도록 센서에 지시할 수 있습니다.   

알림을 표시 하 고 알림을 처리 하려면:

1. **유형별 필터링, 날짜 범위** 옵션 또는 **모두 선택** 옵션을 사용 합니다. 필요에 따라 알림의 선택을 취소 합니다.

2. [ **학습**]을 선택 하 여 선택한 장치에 새로 검색 된 정보를 적용 하도록 센서에 지시 합니다. 또는 **해제** 를 선택 하 여 새로 검색 된 정보를 무시 하도록 센서에 지시 합니다. 동시에 배우고 해제할 수 있는 알림 수는 개별적으로 처리 해야 하는 알림 수와 함께 표시 됩니다.

**새 ip** 와 구성 된 **서브넷** 은 동시에 처리할 수 없습니다. 수동 확인이 필요 합니다.
