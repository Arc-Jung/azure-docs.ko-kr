# <a name="defender-for-iot-glossary"></a>Defender for IoT 용어집

이 용어집에서는 IoT 플랫폼용 Azure Defender에 대 한 중요 한 용어 및 개념에 대해 간략하게 설명 합니다. **자세한 정보** 링크를 선택 하 여 용어집의 관련 용어로 이동 합니다. 이를 통해 제품 도구를 보다 신속 하 게 학습 하 고 사용할 수 있습니다.

> [!Note]
> `(AL)`이름에이 나열 된 용어는 에이전트 기반 장치 빌더 용어입니다. 

<a name="glossary-a"></a>

## <a name="a"></a>A

| 용어 | 설명 | 자세한 정보 |
|--|--|--|
| **액세스 그룹** | 액세스 그룹 규칙을 만들어 대기업의 사용자 액세스 요구 사항을 지원 합니다.<br /><br />규칙을 사용 하면 관련 사업부, 지역, 사이트 및 영역에서 특정 사용자 역할에 대해 IoT 온-프레미스 관리 콘솔용 Defender에 대 한 보기 및 구성 액세스를 제어할 수 있습니다.<br /><br />예를 들어 Active Directory 그룹의 보안 분석가가 유럽 서 부 자동차 데이터에 액세스할 수 있지만 아프리카의 데이터에는 액세스할 수 없도록 합니다. | **[온-프레미스 관리 콘솔](#o)** <br /><br />**[비즈니스 단위](#b)** |
| **액세스 토큰** | IoT REST API 용 Defender에 액세스 하기 위한 액세스 토큰을 생성 합니다. | **[API](#glossary-a)** |
| **경고 이벤트 승인** | 검색 된 이벤트에 대해 한 번만 경고를 숨기도록 IoT에 대해 Defender를 지시 합니다. 이벤트가 다시 검색 되 면 경고가 다시 트리거됩니다. | **[](#glossary-a) <br /> 경고 <br /> [경고 이벤트 알아보기](#l) <br /> <br /> [경고 이벤트 음소거](#m)** |
| **경고** | 승인 된 네트워크 동작, 네트워크 이상 또는 의심 스러운 네트워크 활동 및 트래픽과의 편차와 관련 하 여 IoT 엔진의 Defender가 트리거하는 메시지입니다. | **[전달 규칙](#f) <br /> <br /> [제외 규칙](#e) <br /> <br /> [시스템 알림](#s)** |
| **경고 주석** | 보안 분석가 및 관리자가 경고 메시지에서 수행 하는 설명입니다. 예를 들어 경고 주석은 수행할 완화 작업 또는 이벤트와 관련 하 여 연락할 개인의 이름에 대 한 지침을 제공할 수 있습니다.<br /><br />경고를 검토 하는 사용자는 이벤트 상태를 가장 잘 반영 하는 설명 또는 주석을 선택 하거나 경고를 조사 하는 데 필요한 단계를 선택할 수 있습니다. | **[경고](#glossary-a)** |
| **변칙 엔진** | 비정상적인 M2M (컴퓨터 간) 통신 및 동작을 검색 하는 IoT 용 Defender 엔진입니다. 예를 들어, 엔진은 과도 한 SMB 로그인 시도를 검색할 수 있습니다. 변칙 경고는 이러한 이벤트가 검색 될 때 트리거됩니다. | **[IoT 엔진에 대 한 Defender](#d)** |
| **API** | 외부 시스템에서 IoT 용 Defender에 의해 검색 된 데이터에 액세스 하 고 SSL 연결을 통해 외부 REST API를 사용 하 여 작업을 수행할 수 있습니다. | **[액세스 토큰](#glossary-a)** |
| **공격 벡터 보고서** | 악용 가능 끝점의 취약점 체인에 대 한 실시간 그래픽 표현입니다.<br /><br />보고서를 사용 하면 공격 순서에서 완화 작업의 효과를 평가 하 여를 확인할 수 있습니다. 예를 들어 공격 체인을 중단 하 여 시스템 업그레이드가 공격자의 경로를 중단 하는지 여부 또는 대체 공격 경로가 유지 되는지 여부를 평가할 수 있습니다. 이는 재구성 및 완화 작업의 우선 순위를 정하는 것입니다. | **[위험 평가 보고서](#r)** |

## <a name="d"></a>D

| 용어 | 설명 | 자세한 정보 |
|--|--|--|
| **데이터 마이닝** | 네트워크 장치에 대 한 포괄적이 고 세부적인 보고서 생성:<br /><br />- **SOC 인시던트 응답**: 즉각적인 인시던트 대응을 처리할 수 있도록 실시간으로 보고 합니다. 예를 들어 보고서는 패치가 필요할 수 있는 장치를 나열할 수 있습니다.<br /><br />- **법적** 고가: 조사 보고서에 대 한 기록 데이터를 기반으로 하는 보고서입니다.<br /><br />- **IT 네트워크 무결성**: 전반적인 네트워크 보안을 개선 하는 데 도움이 되는 보고서입니다. 예를 들어 보고서는 약한 인증 자격 증명이 있는 장치를 나열할 수 있습니다.<br /><br />- **표시 유형**: 네트워크의 모든 기본 매개 변수를 볼 수 있는 모든 쿼리 항목을 포함 하는 보고서입니다.<br /><br />볼 수 있는 읽기 전용 사용자에 대 한 데이터 마이닝 보고서를 저장 합니다. | **[](#b) <br /> 기준선 <br /> [보고서](#r)** |
| **IoT 엔진에 대 한 Defender** | IoT 용 Defender의 셀프 학습 분석 엔진은 서명을 업데이트 하거나 규칙을 정의할 필요가 없습니다. 엔진은 ICS 특정 동작 분석과 데이터 과학을 사용 하 여 비정상, 맬웨어, 운영 문제, 프로토콜 위반 및 기준 네트워크 작업의 편차에 대 한 네트워크 트래픽을 지속적으로 분석 합니다.<br /><br />엔진에서 편차를 검색 하면 경고가 트리거됩니다. **경고 화면 또는** siem에서 경고를 보고 관리할 수 있습니다. | **[경고](#glossary-a)** |
| **IoT 플랫폼용 Defender** | Iot 센서 및 온-프레미스 관리 콘솔에 대 한 Defender에 설치 된 IoT 용 Defender 솔루션 | **[](#s) <br /> 센서 <br /> [온-프레미스 관리 콘솔](#o)** |
| **장치 맵** | IoT 용 Defender에서 감지한 네트워크 장치를 그래픽으로 표현한 것입니다. 장치 간 연결 및 각 장치에 대 한 정보를 표시 합니다. 맵을 사용 하 여 다음을 수행 합니다.<br /><br />-중요 한 장치 정보를 검색 하 고 제어 합니다.<br /><br />-네트워크 조각을 분석 합니다.<br /><br />-장치 세부 정보 및 요약을 내보냅니다. | **[Purdue 계층 그룹](#p)** |
| **장치 인벤토리-센서** | 장치 인벤토리는 IoT 용 Defender에서 검색 한 광범위 한 장치 특성을 표시 합니다. 옵션은 다음에서 사용할 수 있습니다.<br /><br />-표시 된 정보를 필터링 합니다.<br /><br />-이 정보를 CSV 파일로 내보냅니다.<br /><br />-Windows 레지스트리 세부 정보를 가져옵니다. | **[그룹](#g)** <br /><br />**[장치 인벤토리-온-프레미스 관리 콘솔](#d)** |
| **장치 인벤토리-온-프레미스 관리 콘솔** | 연결 된 센서의 장치 정보는 장치 인벤토리에 있는 온-프레미스 관리 콘솔에서 볼 수 있습니다. 그러면 온-프레미스 관리 콘솔의 사용자에 게 모든 네트워크 정보에 대 한 포괄적인 보기가 제공 됩니다. | **[장치 인벤토리-센서](#d) <br /> <br /> [장치 인벤토리-데이터 통합자](#d)** |
| **장치 인벤토리-데이터 통합자** | 온-프레미스 관리 콘솔의 데이터 통합 기능을 사용 하면 다른 엔터프라이즈 리소스의 정보를 사용 하 여 장치 인벤토리의 데이터를 향상 시킬 수 있습니다. 예제 리소스는 CMDBs, DNS, 방화벽 및 Web Api입니다. | **[장치 인벤토리-온-프레미스 관리 콘솔](#d)** |
| **장치** 쌍 `(AL)` | 장치 쌍은 메타 데이터, 구성 및 조건을 포함 하 여 장치 상태 정보를 저장 하는 JSON 문서입니다. | [모듈 쌍](#m) <br /> <br />[보안 모듈 쌍](#s) |
