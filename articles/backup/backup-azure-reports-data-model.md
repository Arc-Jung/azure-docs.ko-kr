# <a name="data-model-for-azure-backup-diagnostics-events"></a>Azure 백업 진단 이벤트에 대한 데이터 모델

## <a name="coreazurebackup"></a>코어Azure백업

이 표에서는 볼트 및 백업 항목과 같은 핵심 백업 엔터티에 대한 정보를 제공합니다.

| **필드**                         | **데이터 유형** | **설명**                                              |
| --------------------------------- | ------------- | ------------------------------------------------------------ |
| ResourceId                        | 텍스트          | 수집되는 데이터에 대한 리소스 식별자(예: 예를 들어 복구 서비스 볼트 리소스 ID입니다. |
| OperationName                     | 텍스트          | 이 필드는 현재 작업(BackupItem, BackupItemAssociation 또는 보호된 컨테이너)의 이름을 나타냅니다. |
| Category                          | 텍스트          | 이 필드는 Azure Monitor 로그에 푸시된 진단 데이터의 범주를 나타냅니다. 예를 들어, 코어Azure백업. |
| 에이전트 버전                      | 텍스트          | 에이전트 백업 또는 보호 에이전트의 버전 번호(SC DPM 및 MABS의 경우) |
| AzureBackupAgentVersion           | 텍스트          | 백업 관리 서버의 Azure 백업 에이전트 버전 |
| AzureDataCenter                   | 텍스트          | 볼트가 있는 데이터 센터                       |
| 백업항목앱버전              | 텍스트          | 백업 항목의 응용 프로그램 버전                       |
| BackupItemFriendlyName            | 텍스트          | 백업 항목의 이름                             |
| BackupItemName                    | 텍스트          | 백업 항목의 이름                                      |
| 백업항목보호상태         | 텍스트          | 백업 항목의 보호 상태                          |
| 백업항목프론트엔드사이즈            | 텍스트          | 백업 항목의 프런트 엔드 크기                            |
| BackupItemType                    | 텍스트          | 백업 항목의 유형입니다. 예: VM, 파일 폴더             |
| 백업항목유니크Id                | 텍스트          | 백업 항목의 고유 식별자                         |
| 백업 관리 서버 유형        | 텍스트          | MABS, SC DPM에서와 같이 백업 관리 서버의 유형     |
| 백업 관리서버유니크Id    | 텍스트          | 백업 관리 서버를 고유하게 식별하는 필드      |
| BackupManagementType              | 텍스트          | 백업 작업을 수행하는 서버에 대한 공급자 유형입니다. 예를 들어, IaaSVM, 파일 폴더 |
| 백업관리서버이름        | 텍스트          | 백업 관리 서버 이름                         |
| 백업 관리서버OSVersion   | 텍스트          | 백업 관리 서버의 OS 버전                   |
| 백업 관리서버버전     | 텍스트          | 백업 관리 서버 버전                      |
| 최신 복구 지점 위치       | 텍스트          | 백업 항목의 최신 복구 지점 위치    |
| 최신 복구 포인트 타임           | DateTime      | 백업 항목의 최신 복구 지점의 날짜 시간   |
| 가장 오래된 복구 지점위치       | 텍스트          | 백업 항목의 가장 오래된 복구 지점 위치    |
| 가장 오래된 리커버리포인트타임           | DateTime      | 백업 항목의 최신 복구 지점의 날짜 시간   |
| PolicyUniqueId                    | 텍스트          | 정책을 식별하는 고유 ID                             |
| 보호된 컨테이너우호적이름    | 텍스트          | 보호된 서버의 친숙한 이름                        |
| 보호된 컨테이너위치        | 텍스트          | 보호된 컨테이너가 온-프레미스 또는 Azure에 있는지 여부 |
| 보호된 컨테이너 이름            | 텍스트          | 보호된 컨테이너의 이름                            |
| 보호된 컨테이너 OSType          | 텍스트          | 보호된 컨테이너의 OS 유형                          |
| 보호된 컨테이너 OSVersion       | 텍스트          | 보호된 컨테이너의 OS 버전                        |
| 보호된 컨테이너보호상태 | 텍스트          | 보호된 컨테이너의 보호 상태                  |
| 보호된 컨테이너 유형            | 텍스트          | 보호된 컨테이너가 서버인지 컨테이너인지 여부  |
| 보호된 컨테이너고유 ID        | 텍스트          | DPM, MABS를 사용하여 백업된 VM을 제외한 모든 컨테이너에 대해 보호된 컨테이너를 식별하는 데 사용되는 고유 ID |
| 보호된 컨테이너 워크로드 유형    | 텍스트          | 백업된 보호된 컨테이너의 유형입니다. 예를 들어, IaaSVMContainer |
| ProtectionGroupName               | 텍스트          | 보호 그룹의 이름 백업 항목은 SC DPM 및 MABS(해당하는 경우)에 대해 보호됩니다. |
| ResourceGroupName                 | 텍스트          | 수집되는 데이터에 대한 리소스 리소스 그룹(예: 복구 서비스 자격 증명 모음) |
| schemaVersion                     | 텍스트          | 이 필드는 스키마의 현재 버전을 나타내며 **V2입니다.** |
| 보조 백업보호 상태    | 텍스트          | 백업 항목에 대해 보조 보호가 활성화되어 있는지 여부  |
| 시스템 상태                             | 텍스트          | 백업 항목 개체의 상태입니다. 예를 들어 활성, 삭제됨 |
| StorageReplicationType            | 텍스트          | 볼트에 대한 저장소 복제 유형입니다. 예를 들어, 지오레디던트 |
| SubscriptionId                    | 텍스트          | 데이터가 수집되는 리소스의 구독 식별자(예: 복구 서비스 자격 증명 모음) |
| VaultName                         | 텍스트          | 자격 증명 모음의 이름                                            |
| VaultTags                         | 텍스트          | 볼트 리소스와 연결된 태그                    |
| 볼트유니크                     | 텍스트          | 볼트의 고유 식별자                             |
| SourceSystem                      | 텍스트          | 현재 데이터의 소스 시스템 - Azure                  |

| **필드**                      | **데이터 유형** | **설명**                                              |

| 경고 코드                      | 텍스트          | 경고 유형을 고유하게 식별하는 코드                     |

| AlertType                      | 텍스트          | 경고 유형입니다. 예를 들어, 백업                           |
| AlertUniqueId                  | 텍스트          | 생성된 경고의 고유 식별자                    |
| 백업항목유니크Id             | 텍스트          | 경고와 연결된 백업 항목의 고유 식별자 |

| BackupManagementType           | 텍스트          | 백업 작업을 수행하는 서버에 대한 공급자 유형(예: IaaSVM, FileFolder) |
| 카운트오브경고통합      | Number        | 통합 경고인 경우 통합된 경고 수  |
| 보호된 컨테이너고유 ID     | 텍스트          | 경고와 연결된 보호된 서버의 고유 식별자 |
| 권장 작업              | 텍스트          | 경고를 해결하는 데 권장되는 작업                      |
| schemaVersion                  | 텍스트          | 현재 버전의 스키마(예: **V2)**            |
| 시스템 상태                          | 텍스트          | 경고 개체의 현재 상태(예: Active, Deleted) |
| 스토리지유니드                | 텍스트          | 저장소 엔터티를 식별하는 데 사용되는 고유 ID                |
| 볼트유니크                  | 텍스트          | 경고와 관련된 볼트를 식별하는 데 사용되는 고유 ID    |
| SourceSystem                   | 텍스트          | 현재 데이터의 원본 시스템(Azure)                    |

| SourceSystem                   | 텍스트          | 현재 데이터의 원본 시스템(Azure)                    |

| 작업 시작날짜 시간               | DateTime      | 작업 실행이 시작된 날짜 및 시간                       |
| JobStatus                      | 텍스트          | 완료된 작업의 상태(예: Completed, Failed)   |
| JobUniqueId                    | 텍스트          | 작업을 식별하는 고유 ID                                |
| 보호된 컨테이너고유 ID     | 텍스트          | 경고와 연결된 보호된 서버의 고유 식별자 |

| 복구작업위치            | 텍스트          | 복구 중인 복구 지점이 저장된 위치 |

| schemaVersion                  | 텍스트          | 현재 버전의 스키마(예: **V2)**            |

| 볼트유니크                  | 텍스트          | 경고와 연결된 보호된 볼트의 고유 식별자 |
