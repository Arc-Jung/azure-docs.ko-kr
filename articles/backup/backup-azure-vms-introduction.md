# Azure 가상 컴퓨터 백업-소개

이 섹션에서는 Azure 가상 컴퓨터를 보호 하기 위해 Microsoft Azure 백업을 사용 하 여 소개를 제공 합니다. 이 문서를 읽어보면에 대해 알아봅니다.

- Azure는 방법과 가상 컴퓨터 백업 작동
- Azure 가상 컴퓨터를 백업 하는 절차
- 부드러운 백업 경험을 달성 하기 위해 필수 구성 요소
- 일반적인 오류가 발생 하 고 이들을 처리 하는 방법
- 지원 되지않는 시나리오 및 제품 변경 사항에 영향을 하는 방법의 목록

자세한 내용을 보려면 Azure 가상 컴퓨터에 대 한 신속 하 게, 참조는 [가상 컴퓨터 설명서](https://azure.microsoft.com/documentation/services/virtual-machines/).

## Azure 가상 컴퓨터를 백업 하는 이유
클라우드 컴퓨팅 수 있도록 응용 프로그램에는 Microsoft Azure 가상 컴퓨터를 개발 하는 이유는 확장 가능 하 고 항상 사용 가능한 환경 – 실행할 수 있습니다. 이러한 Azure 가상 컴퓨터에서 생성 된 데이터는 중요 한 그리고 안전 하 게 보관에 대 한 백업이 필요 합니다. 백업에서 복원할 데이터를 필요로 하는 일반적인 시나리오입니다.

- 우발적 이거나 악의적인 파일 삭제
- 패치를 업데이트 하는 동안 가상 컴퓨터의 손상
- 전체 가상 컴퓨터의 실수나 악의적인 삭제

서로 다른 두가지 방법에서 이러한 가상 컴퓨터에서 데이터가 백업 수 있습니다.:

- 개별 가상 컴퓨터 내에서 데이터 소스는 백업
- 전체 가상 컴퓨터를 백업

백업 전체 가상 컴퓨터의 되므로 널리 사용 되는 것이 훨씬 더 간단 하 게 관리 및 전체 응용 프로그램 및 운영 체제의 쉽게 복원 용이 하 게도 합니다. 게스트에 데이터 백업이 나 전체 가상 컴퓨터의 백업에 대 한 azure 백업을 사용할 수 있습니다.

가상 컴퓨터 백업에 대 한 Azure Backup을 사용 하는 비즈니스 이점 다음과 같습니다.

- 가상 컴퓨터에 대 한 백업 및 복구 워크플로 자동화
- 되도록 응용 프로그램에 일관 된 백업을 일관성 있는 상태에서 데이터 시작을 복구 합니다.
- 가상 컴퓨터 백업 하는 동안 관련 된 중단 되지 않습니다.
- Windows 또는 Linux 가상 컴퓨터를 백업할 수 있습니다.
- 복구 지점이 있는 Azure 백업 자격 증명 모음에 쉽게 복원에 사용할 수 있습니다.
- 자동 정리 및 가비지 수집 이전 복구 지점입니다.

## Azure 가상 컴퓨터 백업 어떻게 작동 합니까?
가상 컴퓨터를 백업 하려면 먼저 데이터의 지정 시간 스냅숏이 필요 합니다. Azure 백업 서비스는 예약된 된 시간에 백업 작업을 시작 하 고 스냅숏을 예비 내선 번호를 트리거합니다. 예비 내선 번호 일관성을 달성 하기 위해 게스트에서 VSS 서비스와 조정 및 일관성에 도달한 후에 Azure 저장소 서비스의 blob 스냅숏 API를 호출 합니다. 이 절차에 따라 종료 하지 않고도 가상 컴퓨터의 디스크의 일관 된 스냅숏의 얻을에 수행 됩니다.

스냅숏이, 후 데이터는 백업 자격 증명 모음에 Azure 백업 서비스에 의해 전달 됩니다. 서비스를 식별 하 고 백업 저장소를 효율적으로 만드는 – 마지막 백업에서 변경 된 블록만 전송 담당 합니다. 데이터 전송이 완료 되 면에 스냅숏을 제거 되 고 복구 지점이 만들어집니다. 이 복구 지점은 Azure 관리 포털에서 볼 수 있습니다.

![Azure 가상 컴퓨터 백업 아키텍처](./media/backup-azure-vms-introduction/vmbackup-architecture.png)

>[AZURE.NOTE]Linux 가상 컴퓨터에만 파일에 일관 된 백업은 가능한입니다.

## 계산 인스턴스를 보호
Azure 백업을 사용 하 여 한를 백업 하는 azure 가상 컴퓨터에 적용 됩니다. [Azure 백업 가격](http://azure.microsoft.com/pricing/details/backup/). 기준으로 인스턴스 보호 계산 되는 *실제* "리소스"디스크를 제외 하 고 – 가상 컴퓨터에 있는 모든 데이터의 합계인 가상 컴퓨터의 크기입니다.  *하지* 요금이 청구 데이터 디스크에 저장 된 실제 데이터에 있지만 가상 컴퓨터에 연결 된 각 데이터 디스크에 대해 지원 되는 최대 크기에 기반 합니다. 마찬가지로, 백업 저장소 bill 각 복구 지점에 있는 실제 데이터의 합계인 Azure 백업을 사용 하 여 저장 된 데이터의 양에 기반 합니다.

최대 크기가 각각 1TB의 두 추가 데이터 디스크가 있는 A2 표준 크기의 가상 컴퓨터는 걸릴 예. 다음 표에서 각이 디스크에 저장 된 실제 데이터 제공 됩니다.

|디스크 유형|최대 크기|실제 데이터 표시|
|---------|--------|------|
| OS 디스크 | 1023GB | 17GB |
| 로컬 디스크 / 리소스 디스크 | 135 G B | 5GB (백업에 포함 되지) |
| 데이터 디스크 1 |	1023GB | 30GB |
| 데이터 디스크 2 | 1023GB | 0 G B |

 *실제* g B + 30 17GB + 0 = 47 g B에이 경우 가상 컴퓨터의 크기입니다. 이것은 월별 청구의 기반이 되는 보호 된 인스턴스 크기 됩니다. 가상 컴퓨터의 데이터 양이 증가 함에 따라 보호 된 인스턴스 크기 또한 대금 청구에 사용 되는 적절 하 게 변경 됩니다.

요금 청구는 첫번째 성공적인 백업 완료 될 때까지 시작 되지 않습니다. 이 시점에 저장소 및 보호 된 인스턴스 모두에 대 한 청구를 시작 합니다. 요금 청구는으로 계속 *Azure 백업을 사용 하 여 저장 된 데이터를 백업 하는 모든* 가상 컴퓨터용입니다. 보호 중지 작업을 수행 중인 백업 데이터를 보존 하는 경우 요금 청구를 중지 하지 않습니다. 지정 된 가상 컴퓨터에 대 한 청구는 보호가 중지 된 경우에 중단 됩니다 *및* 모든 백업 데이터가 삭제 됩니다. (보호 중지 되었습니다) 하는 경우 현재 백업 작업이 없는 경우 마지막으로 성공한 백업 당시의 가상 컴퓨터의 크기는 월별 청구의 기반이 되는 보호 된 인스턴스 크기가 됩니다.

## 필수 조건
### 1. 백업 자격 증명 모음
Azure 가상 컴퓨터를 백업 하려면 백업 자격 증명을 만들려면 먼저 지정 해야 합니다. 자격 증명 모음에는 모든 백업 및 시간에 따라 생성 된 복구 지점을 저장 하는 엔터티입니다. 자격 증명 모음에는 또한 백업 되 고 있는 가상 컴퓨터에 적용 될 백업 정책을 포함 합니다.

아래 이미지에서는 다양 한 Azure 백업 엔터티 간의 관계를 보여줍니다. ![Azure 백업 엔터티 및 관계](./media/backup-azure-vms-introduction/vault-policy-vm.png)

### 백업 자격 증명을 만들려면

1. [관리 포털](http://manage.windowsazure.com/)에 로그인합니다.

2. 클릭 하 여 **새** > **데이터 서비스** > **복구 서비스** > **백업 자격 증명 모음** > **빨리 만들기**. 조직 계정과 연결 된 여러 구독이 있는 경우에 백업 저장소와 연결할 올바른 구독이 선택 합니다. 각 Azure 구독에서 보호 되 고 가상 컴퓨터를 구성 하도록 여러 백업 자격 증명 모음을 사용할 수 있습니다.

3. **이름**에 자격 증명 모음을 식별하기 위한 이름을 입력합니다. 이는 각 구독에 대해 고유해야 합니다.

4. **지역**에서 자격 증명 모음에 대한 지리적 지역을 선택합니다. 자격 증명 모음 보호 하려는 가상 컴퓨터와 동일한 지역에 있어야 하는 참고 합니다. 가상 컴퓨터를 서로 다른 지역에 있는 경우 각 구성 파일에서 자격 증명 모음을 만듭니다. 백업 데이터를 저장할 저장소 계정을 지정할 필요가 없습니다 – 백업 자격 증명 및 Azure 백업 서비스는이 자동으로 처리 합니다. ![백업 자격 증명 모음 만들기](./media/backup-azure-vms-introduction/backup_vaultcreate.png)

    >[AZURE.NOTE]가상 컴퓨터 사용 하 여 백업을 Azure 백업 서비스는 에서만 지원 영역을 선택 합니다. 목록을 확인 [영역 지원](http://azure.microsoft.com/regions/#services). 원하는 영역을 없는 경우 지원 되는 오늘날 자격 증명 모음을 만드는 동안 드롭다운 목록에 표시 되지 않습니다.

5. 클릭할 **자격 증명 모음 만들기**. 백업 자격 증명 모음을 만드는 데 시간이 걸릴 수 있습니다. 포털의 맨 아래에 상태 알림을 모니터링 합니다. ![알림 메시지 자격 증명 모음 만들기](./media/backup-azure-vms-introduction/creating-vault.png)

6. 메시지 확인 자격 증명 모음 성공적으로 만들었습니다을 활성으로 복구 서비스 페이지에 나열 됩니다. ![백업 자격 증명 모음 목록](./media/backup-azure-vms-introduction/backup_vaultslist.png)

7. 갑니다. 백업 자격 증명 모음을 클릭 하는 **빠른 시작** 페이지, 여기서 Azure 가상 컴퓨터의 백업에 대 한 지침이 표시 됩니다. ![대시보드 페이지에서 가상 컴퓨터 백업 지침](./media/backup-azure-vms-introduction/vmbackup-instructions.png)

    >[AZURE.NOTE]자격 증명 모음 만들어진 직후 적절 한 저장소 중복 옵션을 선택 했는지 확인 합니다. 에 대해 자세히 알아보십시오 [백업 자격 증명 모음에서 저장소 중복 옵션을 설정][vault-storage-redundancy].

### 2. VM 에이전트
시작 하기 전에 Azure 가상 컴퓨터를 백업 하려면 가상 컴퓨터에 Azure VM Agent가 올바르게 설치 되었는지 확인 합니다. 가상 컴퓨터를 백업 하기 위해 Azure 백업 서비스는 확장명이 VM 에이전트를 설치 합니다. VM 에이전트에서 가상 컴퓨터를 만들 때 선택적 구성 요소 이기 때문에 가상 컴퓨터가 프로 비전 된 전에 VM 에이전트에 대 한 확인란을 선택 되어있는지 확인 해야 합니다.

에 대해 알아봅니다는 [VM 에이전트](https://go.microsoft.com/fwLink/?LinkID=390493&clcid=0x409) 및 [설치 하는 방법](http://azure.microsoft.com/blog/2014/04/15/vm-agent-and-extensions-part-2/).

>[AZURE.NOTE]Azure에 온-프레미스 데이터 센터에서 가상 컴퓨터를 마이그레이션할 계획 이라면 다운로드 하 고 마이그레이션 프로세스에서 시작 하기 전에 VM 에이전트 MSI 설치를 확인 합니다. 가상 컴퓨터를 보호 하 여 Azure 사이트 복구를 사용 하 여 Azure에도 적용 됩니다.

## 미리 보기 중 제한 사항

- 5 개 이상의 디스크가 있는 가상 컴퓨터의 백업 지원 되지 않습니다.
- 프리미엄 저장소를 사용 하 여 가상 컴퓨터의 백업 지원 되지 않습니다.
- 백업 여러 Nic를 사용 하 여 가상 컴퓨터의 또는 부하 분산 구성에서 지원 되지 않습니다.
- 복원 하는 동안 기존 가상 컴퓨터를 교체 하는 것은 지원 되지 않습니다. 먼저 기존 가상 컴퓨터 및 모든 관련된 디스크를 삭제 하 고 백업에서 데이터를 복원 합니다.
- Azure 사이트 복구를 사용 하 여 복원 하는 가상 컴퓨터의 백업 지원 되지 않습니다.
- 영역 간 백업 및 복원 지원 되지 않습니다.
- 가상 컴퓨터 사용 하 여 백업을 Azure 백업 서비스는 에서만 지원 영역을 선택 합니다. 목록을 확인 [영역 지원](http://azure.microsoft.com/regions/#services). 원하는 영역을 없는 경우 지원 되는 오늘날 자격 증명 모음을 만드는 동안 드롭다운 목록에 표시 되지 않습니다.
- Azure 백업 서비스를 사용 하 여 가상 컴퓨터 백업은 선택 하는 운영 체제 버전에 대해서만 지원만 됩니다.
  - **Linux**: azure 보증 배포판의 목록은 사용 가능한 [여기](../virtual-machines-linux-endorsed-distributions.md). 또한 다른 가져오기-Your-소유-Linux 배포판 VM 에이전트는 가상 컴퓨터에서 사용할 수 있는 정상적으로 작동 해야 합니다.
  - **Windows Server**: Windows Server 2008 r 2 보다 오래 된 버전 지원 되지 않습니다.

모든 기능에 참조를 포함 하려는 경우 [의견 보내기](http://aka.ms/azurebackup_feedback).
