# Azure DPM 백업 소개

이 문서에서는 Microsoft Azure 백업을 사용하여 System Center Data Protection Manager(DPM) 서버와 워크로드를 보호하는 방법을 소개합니다. 이 문서를 읽어 보면 다음을 이해하게 됩니다.

- Azure DPM 서버 백업 작동 방식
- 원활한 백업 경험을 위한 필수 조건
- 발생하는 일반적인 오류 및 오류를 처리하는 방법
- 지원되는 시나리오

System Center DPM 백업 파일 및 애플리케이션 데이터 DPM에 백업된 데이터는 테이프나 디스크에 저장하거나 Microsoft Azure 백업을 사용하여 Azure에 백업할 수 있습니다. DPM 은 Azure 백업과 다음과 같이 상호작용합니다.

- **물리적 서버 또는 온-프레미스 가상 컴퓨터로 배포하는 DPM** — DPM을 물리적 서버 또는 온-프레미스 Hyper-V 가상 컴퓨터로 배포하는 경우, 디스크나 테이프 백업에 더해 데이터를 Azure 백업 자격 증명 모음에 백업할 수 있습니다.
- **Azure 가상 컴퓨터로 배포하는 DPM** — System Center 2012 R2 업데이트 3부터 DPM을 Azure 가상 컴퓨터로 배포할 수 있습니다. DPM을 Azure 가상 컴퓨터로 배포하는 경우, 데이터를 DPM Azure 가상 컴퓨터에 연결된 Azure 디스크에 백업하거나 데이터 스토리지를 Azure 백업 자격 증명 모음에 백업하여 오프로드할 수 있습니다.

## DPM 서버를 백업하는 이유

DPM 서버 백업에 Azure 백업을 사용할 경우의 비즈니스 이점은 다음과 같습니다.

- 온-프레미스 DPM 배포의 경우, 테이프에 대한 장기 배포 대신 Azure 백업을 사용할 수 있습니다.
- Azure로 DPM을 배포하는 경우, Azure 백업을 사용하면 Azure 디스크에서 저장소를 오프로드할 수 있어 오래된 데이터를 Azure 백업에 저장하고 새 데이터를 디스크에 저장하여 강화할 수 있습니다.

## DPM 서버 백업 작동 방식
가상 컴퓨터를 백업하려면 먼저 데이터의 지정 시간 스냅숏이 필요합니다. Azure 백업 서비스는 예약된 시간에 백업 작업을 시작하고 스냅숏을 만드는 백업 확장을 트리거합니다. 백업 확장은 일관성을 유지하기 위해 게스트 내 VSS 서비스와 조정되고, 일관성이 달성되면 Azure 저장소 서비스의 Blob 스냅숏 API를 호출합니다. 이 작업은 종료하지 않고 가상 컴퓨터의 일관된 디스크 스냅숏을 가져오기 위한 것입니다.

스냅숏이 생성된 후 Azure 백업 서비스가 백업 자격 증명 모음으로 데이터를 전송합니다. 서비스는 마지막 백업에서 변경된 블록만 식별하고 전송하여 백업 자격 증명 모음 및 네트워크를 효율적으로 만듭니다. 데이터 전송이 완료되면 스냅숏이 제거되고 복구 지점이 생성됩니다. 이 복구 지점은 Azure 관리 포털에서 확인할 수 있습니다.

>[AZURE.NOTE]Linux 가상 컴퓨터의 경우 파일 일치 백업만 가능합니다.

## 필수 조건
DPM 데이터를 백업하기 위해 다음과 같이 Azure 백업을 준비합니다.

1. **백업 자격 증명 모음 만들기** — Azure 백업 콘솔에 자격 증명 모음을 만듭니다.
2. **저장소 자격 증명 다운로드** — Azure 백업에서 자격 증명 모음에 대해 만든 관리 인증서를 업로드합니다.
3. **Azure 백업 에이전트 설치 및 서버 등록** — Azure 백업에서 각 DPM 서버에 에이전트를 설치하고 백업 자격 증명 모음에 DPM 서버를 등록합니다.

### 백업 자격 증명 모음 만들기
Azure 가상 컴퓨터를 백업하려면 먼저 백업 자격 증명 모음을 만들어야 합니다. 자격 증명 모음은 모든 백업과 시간에 따라 생성된 복구 지점을 저장하는 엔터티입니다. 자격 증명 모음에는 백업 중인 가상 컴퓨터에 적용할 백업 정책도 포함됩니다.

1. [관리 포털](http://manage.windowsazure.com/)에 로그인합니다.
2. **새로 만들기** > **데이터 서비스** > **복구 서비스** > **백업 자격 증명 모음** > **빠른 생성**을 클릭합니다. 조직 계정과 연결된 구독이 여러 개인 경우 백업 자격 증명 모음과 연결할 올바른 계정을 선택합니다. 각 Azure 구독에서 여러 개의 백업 자격 증명 모음을 사용하여 보호 중인 데이터를 구성할 수 있습니다.
3. **이름**에 자격 증명 모음을 식별하기 위한 이름을 입력합니다. 이는 각 구독에 대해 고유해야 합니다.
4. **지역**에서 자격 증명 모음에 대한 지리적 지역을 선택합니다. 자격 증명 모음은 보호하려는 가상 컴퓨터와 동일한 지역에 있어야 합니다. 가상 컴퓨터가 여러 지역에 있는 경우 각 지역에 자격 증명 모음을 만듭니다. 백업 데이터를 저장하기 위해 저장소 계정을 지정할 필요는 없습니다. 백업 자격 증명 모음 및 Azure 백업 서비스가 자동으로 처리합니다.
    > [AZURE.NOTE][지원되는 하위 지역](http://azure.microsoft.com/regions/#services)

5. **구독**에 백업 저장소와 함께 사용할 Azure 구독을 입력합니다.
6. **자격 증명 모음 만들기**를 클릭합니다. ![백업 자격 증명 모음 만들기](./media/backup-azure-dpm-introduction/backup_vaultcreate.png)

    백업 자격 증명 모음을 만드는 데 시간이 걸릴 수 있습니다. 포털의 맨 아래에서 상태 알림을 모니터링합니다. ![자격 증명 모음 알림 메시지 만들기](./media/backup-azure-dpm-introduction/creating-vault.png)

    자격 증명 모음이 성공적으로 만들어졌으며 복구 서비스 페이지에 **활성** 상태로 나열됨을 확인하는 메시지가 나타납니다.

    ![백업 자격 증명 모음 목록](./media/backup-azure-dpm-introduction/backup_vaultslist.png)

7. 백업 자격 증명을 클릭하여 DPM 서버 백업 지침이 표시되는 **빠른 시작** 페이지로 이동합니다. ![대시보드 페이지의 가상 컴퓨터 백업 지침](./media/backup-azure-dpm-introduction/vmbackup-instructions.png)

    > [AZURE.NOTE]자격 증명 모음이 생성된 후 즉시 적절한 저장소 중복 옵션을 선택해야 합니다. [백업 자격 증명 모음에서 저장소 중복 옵션 설정](http://azure.microsoft.com/documentation/articles/backup-azure-backup-create-vault/#azure-backup---storage-redundancy-options)에 대해 자세히 알아보세요.

### 저장소 자격 증명 다운로드
1. **복구 서비스**를 클릭한 후 백업 자격 증명 모음을 클릭합니다. **빠른 시작** 페이지에서 **저장소 자격 증명 다운로드**를 클릭하여 자격 증명 파일을 다운로드하고 안전한 위치에 저장합니다. 자격 증명을 편집할 수는 없으므로 위치를 열 필요가 없습니다. 보안상의 이유로 파일의 키는 48시간 후에 만료됩니다.

2. 파일을 Azure 백업 자격 증명 모음에 등록하고자 하는 DPM 서버가 쉽게 액세스할 수 있는 안전한 위치에 복사합니다. Azure 백업 에이전트를 설치할 때 해당 파일을 선택해야 합니다.

### Azure 백업 에이전트 설치 및 서버 등록
에이전트 설치 파일을 다운로드하고 백업하고자 하는 데이터를 포함하는 각 DPM 서버에서 이를 실행합니다. 에이전트는 **Azure Download Center**에 저장되며, 고유의 설치 프로세스가 있습니다. 설치를 실행하면 에이전트가 설치되고 DPM 서버가 자격 증명 모음에 등록됩니다. 다음 사항에 유의하세요.

- 에이전트를 설치하려면 DPM 서버의 관리 권한이 필요합니다.
- 복수의 DPM 서버에 설치하려면 설치 파일을 공유된 네트워크 리소스에 배치하거나 그룹 정책 또는 System Center Configuration Manager와 같은 관리 제품을 사용하여 에이전트를 설치할 수 있습니다.
- 설치 후 DPM 서버를 다시 시작할 필요가 없습니다.

#### 백업 에이전트를 설치하고 서버를 등록하려면

1. Azure 백업 자격 증명 모음의 **빠른 시작** 페이지에서 **Azure 백업 에이전트 다운로드** 아래에 있는 **Windows Server 또는 System Center Data Protection Manager 또는 Windows 클라이언트용**을 클릭합니다. 실행하고자 하는 애플리케이션을 DPM 서버에 다운로드합니다.
2. 설치 파일 **MARSAgentInstaller.exe**를 실행합니다. 서비스 규약에 동의하고 누락된 필수 구성 요소 소프트웨어 설치를 선택합니다.
3. **설정 설치** 페이지에서 **설치 폴더** 및 **캐시 위치**를 선택합니다.

    기본 캐시 위치 폴더는 <system drive>:\Program Files\Azure Backup Agent입니다. 캐시 위치에서 설치 프로세서는 **Azure 백업 에이전트** 폴더에 **Scratch**라는 이름의 폴더를 만듭니다. 캐시 위치에는 최소한 2.5GB(또는 Azure에 백업할 데이터 크기의 10%)의 사용 가능한 공간이 있어야 합니다. 로컬 시스템 관리자 및 관리자 그룹의 구성원만 캐시 디렉터리에 액세스하여 거비스 거부 공격을 방지할 수 있습니다.

4. **프록시 구성** 페이지에서 에이전트에 대한 사용자 지정 프록시 설정을 설정하여 Azure에 연결합니다. 설정을 구성하지 않으면 DPM 서버의 기본 인터넷 액세스 설정이 사용됩니다. 인증이 필요한 프록시 서버를 사용하는 경우 이 페이지에 세부 사항을 입력해야 합니다.
5. **Microsoft 업데이트 옵트인** 페이지에서 업데이트를 사용하기를 권장합니다. 서버가 이미 자동 업데이트를 사용하도록 되어 있으면 이 단계를 건너뜁니다. Microsoft 업데이트 설정은 모든 Microsoft 제품 업데이트에 대한 것이며 Azure 백업 에이전트 단독이 아닙니다.
6. **설치** 페이지가 표시됩니다. 설치는 필수 소프트웨어의 설치 여부 및 설정 완료 여부를 확인합니다. 완료되면 Azure 백업 에이전트가 성공적으로 설치되었다는 메시지를 받게 됩니다. 이 시점에서 업데이트 확인을 선택할 수 있습니다. 업데이트 확인을 실행하도록 하는 것을 권장합니다.
7. **계속 등록**을 클릭하여 서버를 자격 증명 모음에 등록합니다.
8. **자격 증명 모음 식별** 페이지에서 Azure 백업 자격 증명 모음에 생성한 자격 증명 모음 등록 파일을 선택합니다.
9. **암호화 설정** 페이지에서 암호 상세 사항을 지정하거나 암호를 자동으로 생성합니다.
10. 클립보드에 복사를 클릭한 후 암호 생성을 클릭합니다. 암호가 클립보드에 복사되었다는 메시지를 받게 됩니다. 메모장을 열고 클립보드에서 암호를 붙여넣기한 후 파일을 저장하거나 파일을 인쇄하고 바로 잠그는 것을 추천합니다. 등록을 클릭하여 백업 자격 증명 모음에 DPM 서버를 등록합니다.

    > [AZURE.TIP]
11. **Register**를 클릭합니다.

    등록이 완료되면 Azure 백업의 사용할 수 있는 DPM 콘솔이 표시됩니다.

    Azure 백업은 언제나 지정하거나 자동 생성되는 암호(영숫자 문자열)로 소스에서 데이터를 암호화합니다.
    >[AZURE.NOTE]Azure 백업은 결코 암호를 보존하지 않으므로 이를 잊으면 데이터를 복원하거나 복구할 수 없습니다. 키를 외부 위치에 저장하는 것을 적극 권장합니다.

암호를 지정하고 **완료**를 클릭한 경우, 프로덕션 서버를 백업 자격 증명 모음에 등록하는 데 에이전트에 대해서 몇 초 정도 걸립니다. 자격 증명 모음 등록이 완료되는 즉시 요약 **서버 등록** 페이지가 표시됩니다.

## 요구 사항(및 제한 사항)

- DPM은 물리적 서버 또는 System Center 2012 SP1 또는 System Center 2012 R2에 설치된 Hyper-V 가상 컴퓨터로 실행할 수 있습니다. 최소 System Center 2012 R2 DPM 2012 R2 업데이트 롤업 3에서 실행되는 Azure 가상 컴퓨터 또는 최소 System Center 2012 R2 업데이트 롤업 5에서 실행되는 VMWare의 Windows 가상 컴퓨터로도 실행할 수 있습니다.
- DPM을 System Center 2012 SP1에서 실행 중인 경우, System Center Data Protection Manager SP1용 업데이트 롤업 2를 설치해야 합니다. Azure 백업 에이전트를 설치하기 전에 필수입니다.
- DPM 서버는 Windows PowerShell 및 .Net Framework 4.5가 설치되어 있어야 합니다.
- DPM은 대부분의 워크로드를 Azure 백업에 백업할 수 있습니다. 지원되는 전체 목록은 아래의 Azure 백업 지원 항목을 참조하십시오.
- Azure 백업에 저장된 데이터는 “테이프에 복사” 옵션으로 복구할 수 없습니다.
- Azure 백업 기능을 사용할 수 있는 Azure 계정이 필요합니다. 계정이 없는 경우 몇 분 만에 무료 평가판 계정을 만들 수 있습니다. [Azure 백업 가격 정책](https://azure.microsoft.com/pricing/details/backup/)을 읽어보십시오.
- Azure 백업을 사용하려면 백업하고자 하는 서버에 Azure 백업 에이전트를 설치해야 합니다. 각 서버에는 캐시 위치로 최소한 2.5GB의 사용 가능한 로컬 저장소 공간이 있어야 합니다(캐시 위치로 15GB의 사용 가능한 로컬 저장소 공간이 있는 것이 바람직합니다).
- 데이터는 Azure 자격 증명 모음 저장소에 저장됩니다. Azure 백업 자격 증명 모음에 백업할 수 있는 데이터의 양에는 제한이 없지만 데이터 소스(예를 들면 가상 컴퓨터 또는 데이터베이스)의 크기는 1.65TB를 초과해서는 안 됩니다.

다음 파일 형식은 Azure에 대한 백업을 지원합니다.

- 암호화(전체 백업에만 해당)
- 압축(증분 백업 지원)
- 스파스(증분 백업 지원)
- 압축 및 스파스(스파스로 처리됨)

다음은 지원하지 않습니다.

- 대/소문자 구분 파일 시스템의 서버는 지원하지 않습니다.
- 하드 링크(건너뜀)
- 재분석 지점(건너뜀)
- 암호화 및 압축(건너뜀)
- 암호화 및 스파스(건너뜀)
- 압축된 스트림
- 스파스 스트림

>[AZURE.NOTE]System Center 2012 DPM SP1부터는 DPM으로 보호되는 워크로드를 Microsoft Azure 백업을 사용하여 Azure에 백업할 수 있습니다.
