# <a name="glossary-of-terms-for-custom-vision-service"></a>Custom Vision Service에 대한 용어집

Custom Vision Service 및 해당 의미에서 사용된 몇 가지 용어는 다음과 같습니다.

## <a name="classifier"></a>분류자

분류자는 몇 가지 학습 이미지를 사용하여 Custom Vision Service로 빌드하는 모델입니다. 새 분류자의 학습을 완료하면 앱에 추가할 수 있는 평가 엔드포인트(HTTPS)를 받습니다. 빌드한 각 분류자는 자체 프로젝트에 있으며, 로그인하면 모든 프로젝트를 볼 수 있습니다.

## <a name="domain"></a>도메인

프로젝트를 만들 때 해당 프로젝트에 대해 "도메인"를 선택합니다. 도메인은 이미지에서 개체의 특정 형식에 대한 분류자를 최적화합니다. 예를 들어 시나리오가 당근 케이크 이미지와 애플 파이 이미지를 분류하는 경우 “식품” 도메인을 선택합니다. 어떤 도메인을 선택할지 잘 모를 경우 "일반" 도메인을 선택합니다.

- **식품 도메인입니다.** 식당 메뉴에 표시되는 식사에 최적화되었습니다. 개별 과일 또는 채소를 인식하는 데 최적화되지 않았습니다. 개별 과일이나 채소 사진을 분류하려면 해당 용도의 일반 도메인을 사용합니다.
- **랜드마크 도메인입니다.** 인식 가능한 자연적 및 인공적인 랜드마크에 최적화되었습니다. 이 도메인은 랜드마크 앞에서 포즈를 취한 한 무리의 사람들로 인해 랜드마크가 약간 가려진다해도 사진에 명확히 표시되는 경우 가장 잘 작동합니다.
- **소매 도메인입니다.** 쇼핑 카탈로그 또는 쇼핑 웹 사이트에서 이미지를 분류하는 데 최적화되었습니다. 드레스, 바지, 셔츠 등을 분류할 때 높은 정밀도를 원하는 경우 소매 도메인을 사용합니다.
- **성인 도메인입니다.** 성인 콘텐츠와 비성인 콘텐츠 간 정의 향상에 최적화되었습니다. 예를 들어 수영복을 입은 사람의 이미지를 차단하려는 경우 이 도메인을 사용하여 해당 작업을 수행하는 사용자 지정 분류자를 빌드할 수 있습니다.
- **일반 도메인입니다.** 다양한 이미지 분류 작업에 적합합니다.

**소형 도메인**에 의해 생성된 모델은 반복 내보내기 기능을 사용하여 내보낼 수 있습니다. 모바일 장치의 실시간 분류 제약 조건에 최적화되었습니다. 소형 도메인을 사용하여 빌드한 분류자는 교육 데이터의 양이 동일한 표준 도메인보다 정확도가 살짝 떨어질 수 있습니다. 단점은 거의 실시간으로 로컬에서 실행할 정도로 작다는 것입니다. 

## <a name="training-image"></a>훈련 이미지

고정밀 분류자를 만들려면 Custom Vision Service는 몇 가지 학습 이미지가 필요합니다. 교육 이미지는 Custom Vision Service으로 분류하려는 이미지의 사진입니다. 예를 들어 오렌지를 분류하려면 Custom Vision Service가 오렌지를 인식할 수 있는 분류자를 만들 수 있도록 Custom Vision Service에 여러 오렌지 이미지를 업로드해야 합니다. 태그당 최소 30개 이미지를 사용하는 것이 좋습니다.

## <a name="iteration"></a>반복

분류자를 학습 또는 재학습할 때마다 모델의 새로운 반복을 만듭니다. 시간에 따른 진행률을 비교할 수 있도록 지난 몇 차례 반복을 유지합니다. 더 이상 유용하지 않은 반복은 삭제해도 됩니다. 반복 삭제가 영구적으로 적용되면 해당 반복에 고유한 모든 이미지 또는 변경 내용도 삭제합니다. 

## <a name="workspace"></a>작업 영역

작업 영역에는 모든 학습 이미지가 포함되며, 제거된 또는 추가된 이미지 같은 가장 최근의 반복에서 모든 변경 내용을 반영합니다. 분류자를 학습하는 경우 작업 영역에 있는 이미지를 사용하여 분류자의 새로운 반복을 만듭니다.

## <a name="tags"></a>태그들

태그를 사용하여 학습 이미지의 개체에 레이블을 지정합니다. 개 및 조랑말을 식별하는 분류자를 만드는 경우 개를 포함하는 이미지에 "개" 태그를, 조랑말을 포함하는 이미지에는 "조랑말" 태그를, 개 및 조랑말 모두를 포함하는 이미지에는 “개” 및 "조랑말" 태그 둘 다를 배치합니다.

## <a name="evaluation"></a>평가

분류자를 학습하면 자동 생성된 https 엔드포인트를 사용하여 평가에 대한 모든 이미지를 제출할 수 있습니다. 분류자는 신뢰도의 순서대로 예측된 태그 집합을 반환합니다.

## <a name="predictions"></a>예측

분류자가 분류할 새 이미지를 적용하므로 사용자에 대한 이미지를 저장합니다. 잘못 예측된 이미지에 올바르게 태그를 지정하여 분류자의 정밀도를 향상하려면 이러한 이미지를 사용할 수 있습니다. 그런 다음, 분류자를 다시 학습하려면 이러한 새 이미지를 사용할 수 있습니다.

## <a name="precision"></a>자릿수

이미지를 분류할 때 분류자가 이미지를 올바르게 분류할 가능성은 어느 정도인가요? 분류자(개와 망아지)를 학습하는 데 사용되는 모든 이미지 중에서 모델이 맞은 백분율은 어느 정도인가요? 100개 이미지 중 올바른 태그가 99개이면 정밀도는 99%가 됩니다.

## <a name="recall"></a>회수

올바르게 분류되었어야 하는 모든 이미지 중에서 분류자가 올바르게 식별한 이미지는 몇 개였나요? 100% 회수는 분류자의 학습에 사용된 이미지에서 38개의 개 이미지가 있는 경우 분류자가 개 38마리를 찾은 것을 의미합니다.

## <a name="settings"></a>설정

프로젝트 수준 설정 및 사용자 수준 설정의 두 가지 유형의 설정이 있습니다.

- 프로젝트 수준 설정은 다음과 같습니다. 
  
  프로젝트 수준 설정은 프로젝트 또는 분류자에 적용됩니다. 다음과 같습니다.

   - 프로젝트 도메인
   - 프로젝트 이름
   - 프로젝트 설명
   - Usage:
      - 학습 이미지 수
      - 만든 태그 수
      - 만든 반복 횟수

- 사용자 수준 설정은 다음과 같습니다. 
   - 구독 키: 학습에 대한 구독 키 및 평가/예측에 대한 구독 키입니다.
   - Usage:
      - 만든 프로젝트 수
      - 평가/예측 API 호출 수입니다.
