# <a name="select-a-domain-for-a-custom-vision-project"></a>사용자 지정 비전 프로젝트의 도메인 선택

사용자 지정 비전 프로젝트의 설정 블레이드에서 프로젝트의 도메인을 선택할 수 있습니다. 시나리오에 가장 가까운 도메인을 선택합니다.

## <a name="image-classification"></a>이미지 분류

|도메인|목적|
|---|---|
|__일반__| 광범위한 이미지 분류 작업에 최적화되었습니다. 다른 도메인이 적절하지 않거나 선택할 도메인을 잘 모르겠으면 일반 도메인을 선택합니다.|
|__Food__|식당 메뉴에 표시되는 것 같은 음식 사진에 최적화되었습니다. 개별 과일이나 채소 사진을 분류하려면 음식 도메인을 사용합니다.|
|__랜드마크__|인식 가능한 자연적 및 인공적인 랜드마크에 최적화되었습니다. 이 도메인은 랜드마크가 사진에서 명확하게 표시될 때 가장 효과적입니다. 앞에 서 있는 사람이 랜드마크를 약간 가리고 있는 경우에도 도메인이 작동합니다.|
|__소매__|쇼핑 카탈로그 또는 쇼핑 웹 사이트에 있는 이미지에 최적화되었습니다. 드레스, 바지 및 셔츠를 높은 정밀도로 분류하려는 경우 이 도메인을 사용합니다.|
|__압축 도메인__| 에지 디바이스에서 실시간 분류의 제약 조건에 최적화되었습니다.|

## <a name="object-detection"></a>개체 감지

|도메인|목적|
|---|---|
|__일반__| 광범위한 개체 감지 작업에 맞게 최적화됩니다. 다른 도메인이 적절하지 않거나 선택할 도메인을 잘 모르겠으면 일반 도메인을 선택합니다.|
|__로고__|이미지에서 브랜드 로고를 찾도록 최적화됩니다.|
|__선반에 있는 제품__|선반에서 제품을 감지하고 분류하는 데 최적화되어 있습니다.|
|__압축 도메인__| 에지 디바이스에서 실시간 물체 감지의 제약 조건에 최적화되었습니다.|

## <a name="compact-domains"></a>압축 도메인

압축 도메인에서 생성된 모델을 로컬에서 실행하기 위해 내보낼 수 있습니다. 모델 성능은 선택한 도메인에 따라 다릅니다. 아래 표에서, 우리는 인텔 데스크탑 CPU와 엔비디아 GPU에 \[모델\]크기와 추론 시간을보고 1 . 

> [!NOTE]
> 이 숫자에는 전처리 및 후처리 시간이 포함되지 않습니다.

|Task|도메인|모델 크기|CPU 추론 시간|GPU 추론 시간|
|---|---|---|---|---|
|분류|일반(압축)|5MB|13ms|5ms|
|개체 감지|일반(압축)|45 MB|35 ms|5ms|
|개체 감지|일반(컴팩트) [S1]|14 MB|27 ms|7ms|

## <a name="vaidk-vision-ai-dev-kit"></a>VAIDK (비전 AI 개발 키트)

컴팩트 도메인을 선택하면 "기본 플랫폼"과 "비전 AI 개발 키트"를 구분할 수 있는 추가 옵션 "내보내기 기능"이 제공됩니다.

_내보내기 기능에서_ 두 가지 옵션은 다음과 같습니다.

- 기본 플랫폼(텐서플로우, 코어ML, ONNX 등)
- 비전 AI 개발 키트.

_비전 AI 개발자 키트가_ _일반,_ _랜드마크_및 _소매를_ 선택하지만 _식품_ 소형 도메인은 이미지 분류에 사용할 수 있지만 _일반(컴팩트)_ 및 _일반(소형) [S1]은_ 개체 감지에 사용할 수 있습니다.

>[!NOTE]
>개체 검색을 위한 __일반(소형)__ 도메인에는 특별한 후처리 논리가 필요합니다. 자세한 내용은 내보낸 zip 패키지의 예제 스크립트를 참조하십시오. 후처리 논리가 없는 모델이 필요한 경우 __일반(컴팩트) [S1]을__사용합니다.

>[!IMPORTANT]
>내보낸 모델이 클라우드의 예측 API와 정확히 동일한 결과를 제공한다는 보장은 없습니다. 실행 중인 플랫폼 또는 전처리 구현의 약간의 차이로 인해 모델 출력에 더 큰 차이가 발생할 수 있습니다. 전처리 논리에 대한 자세한 내용은 [이 문서를](python-tutorial.md)참조하십시오.

\[1\] 인텔 제온 E5-2690 CPU와 엔비디아 테슬라 M60
