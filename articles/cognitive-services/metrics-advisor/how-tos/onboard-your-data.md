# <a name="how-to-onboard-your-metric-data-to-metrics-advisor"></a>방법: 메트릭 데이터를 메트릭 Advisor에 등록

## <a name="avoid-loading-partial-data"></a>부분 데이터 로드 방지

일부 데이터는 메트릭 관리자와 데이터 원본에 저장 된 데이터의 불일치 때문에 발생 합니다. 이는 메트릭 Advisor가 데이터 가져오기를 완료 한 후 데이터 원본이 업데이트 된 경우에 발생할 수 있습니다. 메트릭 관리자는 지정 된 데이터 소스에서 데이터를 한 번만 가져옵니다.

예를 들어 메트릭이 모니터링을 위해 메트릭 관리자에 등록 된 경우입니다. 메트릭 관리자가 타임 스탬프 A에서 메트릭 데이터를 성공적으로 가져와 해당 데이터에 대 한 변칙 검색을 수행 합니다. 그러나 데이터를 수집 한 후 특정 타임 스탬프 A의 메트릭 데이터를 새로 고친 경우 새 데이터 값을 검색할 수 없습니다.

나중에 설명 하는 기록 데이터를 [백필](manage-data-feeds.md#backfill-your-data-feed) 하 여 불일치를 완화할 수 있지만 이러한 시간에 대 한 경고가 이미 트리거된 경우에는 새로운 변칙 경고를 트리거하지 않습니다. 이 프로세스는 시스템에 추가 작업을 추가할 수 있으며 자동이 아닙니다.

부분 데이터를 로드 하지 않으려면 다음과 같은 두 가지 방법을 사용 하는 것이 좋습니다.

* 하나의 트랜잭션으로 데이터 생성:

    동일한 타임 스탬프의 모든 차원 조합에 대 한 메트릭 값이 한 트랜잭션의 데이터 원본에 저장 되어 있는지 확인 합니다. 위의 예에서는 모든 데이터 원본의 데이터가 준비 될 때까지 기다린 후 한 번의 트랜잭션으로 메트릭 관리자에 로드 합니다. 메트릭 관리자는 데이터가 성공적으로 검색 되거나 부분적으로 검색 될 때까지 데이터 피드를 정기적으로 폴링할 수 있습니다.

* 수집 **시간 오프셋** 매개 변수에 적절 한 값을 설정 하 여 데이터 수집을 지연 합니다.

    데이터 피드의 수집 **시간 오프셋** 매개 변수를 설정 하 여 데이터가 완전히 준비 될 때까지 수집을 지연 시킵니다. 이는 Azure Table Storage와 같은 트랜잭션을 지원 하지 않는 일부 데이터 원본에 유용할 수 있습니다. 자세한 내용은 [고급 설정](manage-data-feeds.md#advanced-settings) 을 참조 하세요.

## <a name="add-a-data-feed-using-the-web-based-workspace"></a>웹 기반 작업 영역을 사용 하 여 데이터 피드 추가

메트릭 관리자 포털에 로그인 하 고 작업 영역을 선택한 후 **시작**을 클릭 합니다. 그런 다음 작업 영역의 기본 페이지에서 왼쪽 메뉴에 있는 **데이터 피드 추가** 를 클릭 합니다.

### <a name="add-connection-settings"></a>연결 설정 추가

다음으로 시계열 데이터 원본에 연결 하기 위한 매개 변수 집합을 입력 합니다. 
* **원본 유형**: 시계열 데이터가 저장되는 데이터 원본의 유형입니다.
* **세분성**: 시계열 데이터에서 연속 된 데이터 요소 사이의 간격입니다. 현재 메트릭 관리자는 매년, 매월, 매주, 매일, 매시간 및 사용자 지정을 지원 합니다. 사용자 지정 옵션에서 지 원하는 가장 낮은 간격은 60 초입니다.
  * **초**: *GranularityName* 가 *사용자 지정*으로 설정 된 시간 (초)입니다.
* 데이터 **수집 이후 데이터 수집 (UTC)**: 데이터 수집에 대 한 기준 시작 시간입니다. *startOffsetInSeconds* 는 데이터 일관성을 위해 오프셋을 추가 하는 데 자주 사용 됩니다.

다음에는 데이터 원본에 대 한 연결 정보와 데이터를 필요한 스키마로 변환 하는 데 사용 되는 사용자 지정 쿼리를 지정 해야 합니다. 다른 필드 및 다른 유형의 데이터 원본에 연결 하는 방법에 대 한 자세한 내용은 [다른 데이터 원본에서 데이터 피드 추가](../data-feeds-from-different-sources.md)를 참조 하세요.

### <a name="verify-and-get-schema"></a>스키마 확인 및 가져오기

연결 문자열과 쿼리 문자열이 설정 된 후 **확인 및 스키마 가져오기** 를 선택 하 여 연결을 확인 하 고 쿼리를 실행 하 여 데이터 원본에서 데이터 스키마를 가져옵니다. 일반적으로 데이터 원본 연결에 따라 몇 초 정도 걸립니다. 이 단계에서 오류가 발생하는 경우 다음을 확인합니다.

* 연결 문자열과 쿼리가 올바른지 여부
* 방화벽 설정이 있는 경우 Metrics Advisor 인스턴스가 데이터 원본에 연결할 수 있는지 여부

### <a name="schema-configuration"></a>스키마 구성

데이터 스키마를 로드 한 후에는 해당 필드를 선택 합니다.

데이터 요소의 타임 스탬프를 생략 하면 데이터 요소가 대신 수집 때 메트릭 관리자가 타임 스탬프를 사용 합니다. 각 데이터 피드에 대해 최대 하나의 열을 타임 스탬프로 지정할 수 있습니다. 열을 타임 스탬프로 지정할 수 없다는 메시지가 표시 되는 경우 쿼리 또는 데이터 원본을 확인 하 고 쿼리 결과에 여러 타임 스탬프가 있는지 확인 합니다 (미리 보기 데이터에만 해당). 데이터 수집을 수행 하는 경우 메트릭 관리자는 지정 된 원본에서 시계열 데이터의 단일 청크 (예: 세분성에 따라 1 일, 1 시간)만 사용할 수 있습니다.

|선택 영역  |설명  |참고  |
|---------|---------|---------|
| **표시 이름** | 원래 열 이름 대신 작업 영역에 표시 되는 이름입니다. | |
|**Timestamp**     | 데이터 요소의 타임스탬프. 생략하면 Metrics Advisor가 데이터 요소가 수집된 시간의 타임스탬프를 대신 사용합니다. 각 데이터 피드에 대해 최대 하나의 열을 타임 스탬프로 지정할 수 있습니다.        | 선택 사항입니다. 최대 하나의 열에 지정해야 합니다. **열을 타임 스탬프 오류로 지정할 수 없는** 경우 쿼리 또는 데이터 원본에서 중복 타임 스탬프를 확인 합니다.      |
|**측정값**     |  데이터 피드의 숫자 값. 각 데이터 피드에 대해 여러 측정값을 지정할 수 있지만 하나 이상의 열을 측정값으로 선택 해야 합니다.        | 하나 이상의 열에 지정해야 합니다.        |
|**크기**     | 범주별 값. 여러 값의 조합이 특정 단일 차원 시계열(예: 국가, 언어, 테넌트)을 식별합니다. 0 개 이상의 열을 차원으로 선택할 수 있습니다. 참고: 문자열이 아닌 열을 차원으로 선택할 때는 주의 해야 합니다. | 선택 사항입니다.        |
|**무시**     | 선택된 열을 무시하세요.        | 선택 사항입니다. 아래 텍스트를 참조 하세요.       |

열을 무시 하려면 쿼리 또는 데이터 원본을 업데이트 하 여 해당 열을 제외 하는 것이 좋습니다. 열 **무시** 를 사용 하 여 열을 무시 한 다음 특정 열에서 **무시할** 수도 있습니다. 열이 차원 이어야 하며 실수로 *무시*된 것으로 설정 된 경우 메트릭 Advisor는 일부 데이터를 수집 수 있습니다. 예를 들어 쿼리의 데이터가 아래와 같다고 가정 합니다.

| 행 ID | 타임스탬프 | 국가 | 언어 | Income |
| --- | --- | --- | --- | --- |
| 1 | 2019/11/10 | 중국 | ZH-CN-CN | 10000 |
| 2 | 2019/11/10 | 중국 | KO-KR | 1000 |
| 3 | 2019/11/10 | US | ZH-CN-CN | 12000 |
| 4 | 2019/11/11 | US | KO-KR | 23000 |
| ... | ...| ... | ... | ... |

*Country* 가 차원이 고 *언어가* *무시*된 것으로 설정 된 경우 첫 번째와 두 번째 행은 동일한 차원을 갖습니다. 메트릭 관리자는 두 행의 값 중 하나를 임의로 사용 합니다. 이 경우에는 메트릭 관리자가 행을 집계 하지 않습니다.

### <a name="automatic-roll-up-settings"></a>자동 롤업 설정

> [!IMPORTANT]
> 근본 원인 분석 및 기타 진단 기능을 사용 하려면 **자동 롤업 설정을** 구성 해야 합니다. 사용 하도록 설정 하면 자동 롤업 설정을 변경할 수 없습니다.

메트릭 관리자는 수집 하는 동안 각 차원에서 집계 (예: SUM, MAX, MIN)를 자동으로 수행 하 고 루트 사례 분석 및 기타 진단 기능에 사용 될 계층 구조를 작성 합니다. 

다음과 같은 시나리오를 고려해 보세요.

* *데이터에 대 한 롤업 분석을 포함할 필요가 없습니다.*

    메트릭 관리자 롤업을 사용할 필요는 없습니다.

* *데이터가 이미 롤오버 되었으며 차원 값은 NULL 또는 빈 값 (기본값), NULL만, 기타로 표시 됩니다.*

    이 옵션은 행이 이미 합산 되었으므로 메트릭 관리자가 데이터를 롤업할 필요가 없음을 의미 합니다. 예를 들어 *NULL만*선택 하는 경우 아래 예제의 두 번째 데이터 행은 모든 국가 *및 언어의*집계로 표시 됩니다. *Country* 의 값이 비어 있는 네 번째 데이터 행은 불완전 한 데이터를 나타낼 수 있는 일반 행으로 표시 됩니다.
    
    | 국가 | 언어 | Income |
    |---------|----------|--------|
    | 중국   | ZH-CN-CN    | 10000  |
    | N  | KO-KR    | 999999 |
    | US      | KO-KR    | 12000  |
    |         | KO-KR    | 5,000   |

* *Sum/Max/Min/Avg/Count를 계산 하 고이를 표시 하 여 데이터를 롤업 하는 메트릭 Advisor가 필요 합니다. <some string>*

    Cosmos DB 또는 Azure Blob Storage와 같은 일부 데이터 원본은 *group by* 또는 *cube*와 같은 특정 계산을 지원 하지 않습니다. 메트릭 관리자는 수집 중에 자동으로 데이터 큐브를 생성 하는 롤오버 옵션을 제공 합니다.
    이 옵션은 메트릭 관리자가 선택한 알고리즘을 사용 하 여 롤업을 계산 하 고 지정 된 문자열을 사용 하 여 메트릭 관리자의 롤업을 표시 해야 함을 의미 합니다. 이렇게 하면 데이터 원본의 데이터가 변경 되지 않습니다.
    예를 들어 차원 (국가, 지역)과 함께 판매 메트릭을 나타내는 시계열 집합이 있다고 가정 합니다. 지정 된 타임 스탬프에 대해 다음과 같이 표시 될 수 있습니다.


    | 국가       | 지역           | Sales |
    |---------------|------------------|-------|
    | Canada        | Alberta          | 100   |
    | Canada        | British Columbia | 500   |
    | 미국 | Montana          | 100   |


    *합계*를 사용 하 여 자동 롤오버를 사용 하도록 설정 하면 메트릭 관리자는 차원 조합을 계산 하 고 데이터 수집 중 메트릭의 합계를 계산 합니다. 결과는 다음과 같을 수 있습니다.

    | 국가       | 지역           | Sales |
    | ------------ | --------------- | ---- |
    | Canada        | Alberta          | 100   |
    | NULL          | Alberta          | 100   |
    | Canada        | British Columbia | 500   |
    | NULL          | British Columbia | 500   |
    | 미국 | Montana          | 100   |
    | NULL          | Montana          | 100   |
    | NULL          | NULL             | 700   |
    | Canada        | NULL             | 600   |
    | 미국 | NULL             | 100   |

    `(Country=Canada, Region=NULL, Sales=600)` 는 캐나다 (모든 지역)의 판매량 합계가 600 임을 의미 합니다.

    SQL 언어의 변환은 다음과 같습니다.

    ```mssql
    SELECT
        dimension_1,
        dimension_2,
        ...
        dimension_n,
        sum (metrics_1) AS metrics_1,
        sum (metrics_2) AS metrics_2,
        ...
        sum (metrics_n) AS metrics_n
    FROM
        each_timestamp_data
    GROUP BY
        CUBE (dimension_1, dimension_2, ..., dimension_n);
    ```

    자동 롤업 기능을 사용 하기 전에 다음을 고려 하세요.

    * *합계* 를 사용 하 여 데이터를 집계 하려면 메트릭이 각 차원에 추가 되어 있는지 확인 합니다. 다음은 *비 가산적* 메트릭에 대 한 몇 가지 예입니다.
      * 분수 기반 메트릭입니다. 여기에는 비율, 백분율 등이 포함 됩니다. 예를 들어, 전체 국가의 실업률 요금을 계산 하기 위해 각 상태의 실업률 rate를 추가 해서는 안 됩니다.
      * 차원에서 겹칩니다. 예를 들어, 스포츠와 같은 사람들의 수를 계산 하기 위해 각 스포츠에 사용자 수를 추가 하면 안 됩니다. 두 항목 사이에 겹치는 부분이 있으므로 한 사람이 여러 스포츠를 사용할 수 있습니다.
    * 전체 시스템의 상태를 확인 하기 위해 큐브의 크기는 제한 됩니다. 현재 제한은 100만입니다. 데이터가이 제한을 초과 하는 경우 해당 타임 스탬프에 대 한 수집은 실패 합니다.

## <a name="advanced-settings"></a>고급 설정

수집 오프셋 또는 동시성 지정과 같이 사용자 지정 된 방식으로 데이터 수집을 사용 하도록 설정 하는 몇 가지 고급 설정이 있습니다. 자세한 내용은 데이터 피드 관리 문서의 [고급 설정](manage-data-feeds.md#advanced-settings) 섹션을 참조 하세요.

## <a name="specify-a-name-for-the-data-feed-and-check-the-ingestion-progress"></a>데이터 피드의 이름을 지정 하 고 수집 진행률을 확인 합니다.
 
작업 영역에 표시될 데이터 피드의 사용자 지정 이름을 지정합니다. 그런 다음 **제출**을 클릭 합니다. 데이터 피드 세부 정보 페이지에서 수집 진행률 표시줄을 사용 하 여 상태 정보를 볼 수 있습니다.

:::image type="content" source="../media/datafeeds/ingestion-progress.png" alt-text="수집 진행률 표시줄" lightbox="../media/datafeeds/ingestion-progress.png":::


수집 오류 세부 정보를 확인 하려면: 

1. **자세한 정보 표시**를 클릭 합니다.
2. **상태** 를 클릭 한 다음 **실패** 또는 **오류**를 선택 합니다.
3. 실패 한 수집을 마우스로 가리키고 표시 되는 세부 정보 메시지를 확인 합니다.

:::image type="content" source="../media/datafeeds/check-failed-ingestion.png" alt-text="실패 한 수집 확인":::

*실패* 상태는 나중에이 데이터 원본에 대 한 수집을 다시 시도 함을 나타냅니다.
*오류* 상태는 메트릭 관리자가 데이터 원본에 대해 다시 시도 하지 않음을 나타냅니다. 데이터를 다시 로드 하려면 백필/다시 로드를 수동으로 트리거해야 합니다.

**새로 고침 진행률**을 클릭 하 여 수집 진행률을 다시 로드할 수도 있습니다. 데이터 수집이 완료 된 후에는 메트릭을 클릭 하 고 변칙 검색 결과를 확인할 수 있습니다.
