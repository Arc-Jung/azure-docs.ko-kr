# <a name="add-confirmations-to-a-command-in-a-custom-commands-preview-application"></a>사용자 지정 명령 미리 보기 응용 프로그램의 명령에 확인 추가

이 문서에서는 사용자 지정 명령 미리 보기 앱에서 명령에 대 한 확인을 만드는 방법에 대해 알아봅니다.

## <a name="prerequisites"></a>사전 요구 사항

다음 문서의 단계를 완료 합니다.
> [!div class="checklist"]
> * [빠른 시작: 사용자 지정 명령 미리 보기 앱 만들기](./quickstart-custom-speech-commands-create-new.md)
> * [빠른 시작: 매개 변수를 사용 하 여 사용자 지정 명령 미리 보기 앱 만들기](./quickstart-custom-speech-commands-create-parameters.md)

## <a name="create-a-setalarm-command"></a>SetAlarm 명령 만들기

확인을 시연 하려면 경보를 설정 하는 새 명령을 만듭니다.

1. [Speech Studio](https://speech.microsoft.com/)에서 사용자가 만든 사용자 지정 명령 미리 보기 앱을 엽니다.
1. 새 **Setalarm** 명령을 만듭니다.
1. 다음 구성이 포함 된 **DateTime** 매개 변수를 추가 합니다.

   | 설정                           | 제안 값                     |  설명                 |
   | --------------------------------- | -----------------------------------------------------| ------------|
   | **이름**                              | **DateTime**                                | 매개 변수에 대 한 설명이 포함 된 이름입니다.                                |
   | **필수**                          | 선택                                 | 명령을 완료 하기 전에이 매개 변수의 값이 필요한 지 여부를 나타내는 확인란 |
   | **필수 매개 변수에 대 한 응답**   | **간단한 편집기-시간 >**                              | 이 매개 변수 값을 알 수 없는 경우 요청 하는 프롬프트입니다. |
   | **형식**                              | **DateTime**                                | 숫자, 문자열, 날짜/시간 또는 지리와 같은 매개 변수의 유형입니다.   |
   | **날짜 기본값**                     | 날짜가 누락 된 경우 오늘 날짜를 사용 합니다.            | 사용자가 제공 하지 않을 경우 사용할 변수의 기본값입니다.  |  
   | **시간 기본값**                     | 시간이 없는 경우 하루 시작을 사용 합니다.     |  사용자가 제공 하지 않는 경우 사용할 변수의 기본값입니다.|

1. 몇 가지 예제 문장을 추가 합니다.
   
    ```
    set an alarm for {DateTime}
    set alarm {DateTime}
    alarm for {DateTime}
   ```

1. 완료 규칙을 추가 하 여 결과를 확인 합니다. 다음 구성을 사용합니다.

   | 설정    | 제안 값                               |설명                                     |
   | ---------- | ------------------------------------------------------- |-----|
   | **규칙 이름**  | **경보 설정**                                               |    규칙의 용도를 설명 하는 이름입니다. |
   | **actions**    | **음성 응답 보내기-> OK, 경보가 set for {DateTime}**    |규칙 조건이 참인 경우 수행할 동작입니다.

## <a name="try-it-out"></a>기능 직접 사용해 보기

1. 오른쪽 창 맨 위에서 **학습** 을 선택 합니다.

1. 학습을 완료 한 후 **테스트**를 선택 하 고 다음 상호 작용을 시도 합니다.
    - 입력: 내일 정오에 경보 설정
    - 출력: 정상, 2020-05-02 12:00:00에 대해 설정 되는 경보
    - 입력: 경보 설정
    - 출력: 시간
    - 입력: 오후 5 시
    - 출력: 정상, 2020-05-01 17:00:00에 대해 설정 되는 경보

## <a name="add-interaction-rules-for-the-confirmation"></a>확인에 대 한 상호 작용 규칙 추가

상호 작용 규칙을 추가 하 여 확인을 만듭니다.

1. **Setalarm** 명령의 가운데 창에서 **추가** 를 선택한 다음 **상호 작용 규칙**  >  **확인 명령**을 선택 합니다.

    이 규칙은 사용자에 게 알람의 날짜 및 시간을 확인 하 라는 메시지를 표시 합니다. 다음 설정을 사용합니다.

   | 설정               | 제안 값                                                                  | 설명                                        |
   | --------------------- | -------------------------------------------------------------------------------- | -------------------------------------------------- |
   | **규칙 이름**             | **날짜 시간 확인**                                                                | 규칙의 용도를 설명 하는 이름입니다.          |
   | **조건**            | **필수 매개 변수-> DateTime**                                                    | 규칙을 실행할 수 있는 시기를 결정 하는 조건    |   
   | **actions**               | **음성 응답 보내기-{DateTime}에 대 한 경보를 설정 하 시겠습니까 >?**     | 규칙 조건이 참인 경우 수행할 동작입니다. |
   | **수준과**          | **사용자의 확인 필요**                                                 | 다음 턴의 기대                      |
   | **실행 후 상태**  | **사용자 입력 대기**                                                            | 다시 설정한 후 사용자에 대 한 상태                  |
  
      > [!div class="mx-imgBorder"]
      > ![필수 매개 변수 응답 만들기](media/custom-speech-commands/add-validation-set-temperature.png)

1. 수락 된 확인에 대 한 상호 작용 규칙을 추가 합니다 (사용자가 "예" 라고 함). 다음 구성을 사용합니다.

   | 설정               | 제안 값                                                                  | 설명                                        |
   | --------------------- | -------------------------------------------------------------------------------- | -------------------------------------------------- |
   | **규칙 이름**             | **수락 된 확인**                                                            | 규칙의 용도를 설명 하는 이름입니다.          |
   | **조건**            | **확인이 성공 했습니다 & 필수 매개 변수-> DateTime**                      | 규칙을 실행할 수 있는 시기를 결정 하는 조건    |   
   | **실행 후 상태** | **완료 규칙 실행**                                                          | 턴 후의 사용자 상태                   |

1. 거부 된 확인에 대 한 상호 작용 규칙을 추가 합니다 (사용자 "아니요"). 다음 구성을 사용합니다.

   | 설정               | 제안 값                                                                  | 설명                                        |
   | --------------------- | -------------------------------------------------------------------------------- | -------------------------------------------------- |
   | **규칙 이름**             | **거부 확인**                                                                   | 규칙의 용도를 설명 하는 이름입니다.          |
   | **조건**            | **필수 매개 변수 > DateTime & 확인이 거부 되었습니다.**                               | 규칙을 실행할 수 있는 시기를 결정 하는 조건    |   
   | **actions**               | **매개 변수 값 지우기-> DateTime & 음성 응답을 보냅니다. > 문제는 발생 하지 않습니다.**  | 규칙 조건이 참인 경우 수행할 동작입니다. |
   | **실행 후의 상태** | **입력 대기**                                                                   | 턴 후의 사용자 상태                   |
   | **수준과**          | **사용자 > DateTime에서 매개 변수 입력이 필요 합니다.**                           | 다음 턴의 기대                      |

## <a name="try-out-the-changes"></a>변경 내용 시도

1. **학습**을 선택합니다.

1. 교육이 완료 되 면 **테스트**를 선택 하 고 다음 상호 작용을 시도 합니다.

    - 입력: 내일 정오에 경보 설정
    - 출력: 2020-05-02 12:00:00에 대 한 경보를 설정 하 시겠습니까?
    - 입력: 아니요
    - 출력: 문제가 없으며 어떤 시간 입니까?
    - 입력: 오후 5 시
    - 출력: "2020-05-01 17:00:00에 대 한 경보를 설정 하 시겠습니까?"
    - 입력: 예
    - 출력: 정상, 2020-05-01 17:00:00에 대해 설정 되는 경보
