# <a name="how-to-add-a-confirmation-to-a-custom-command-preview"></a>방법: 사용자 지정 명령에 확인 추가 (미리 보기)

이 문서에서는 명령에 확인을 추가 하는 방법에 대해 알아봅니다.

## <a name="prerequisites"></a>사전 요구 사항

다음 문서의 단계를 완료 해야 합니다.
> [!div class="checklist"]
> *  [빠른 시작: 사용자 지정 명령 만들기 (미리 보기)](./quickstart-custom-speech-commands-create-new.md)
> * [빠른 시작: 매개 변수를 사용 하 여 사용자 지정 명령 만들기 (미리 보기)](./quickstart-custom-speech-commands-create-parameters.md)

## <a name="create-a-setalarm-command"></a>SetAlarm 명령 만들기

확인을 보여 주기 위해 사용자에 게 경보를 설정 하도록 허용 하는 새 명령을 만들어 보겠습니다.

1. [Speech Studio](https://speech.microsoft.com/)에서 이전에 만든 사용자 지정 명령 응용 프로그램을 엽니다.
1. 새 명령을 `SetAlarm`만듭니다.
1. 다음 구성으로 이라는 `DateTime` 매개 변수를 추가 합니다.

   | 설정                           | 제안 값                     |  Description                 |
   | --------------------------------- | -----------------------------------------------------| ------------|
   | 속성                              | DateTime                                | 매개 변수에 대 한 설명이 포함 된 이름입니다.                                |
   | 필수                          | checked                                 | 명령을 완료 하기 전에이 매개 변수의 값이 필요한 지 여부를 나타내는 확인란 |
   | 필수 매개 변수에 대 한 응답   | 간단한 편집기-시간 >                              | 이 매개 변수 값을 알 수 없는 경우 요청 하는 프롬프트입니다. |
   | 형식                              | DateTime                                | 숫자, 문자열, 날짜/시간 또는 지리와 같은 매개 변수의 형식입니다.   |
   | 날짜 기본값                     | 오늘 날짜를 누락 하는 경우            | 사용자가 제공 하지 않을 경우 사용할 변수의 기본값입니다.  |  
   | 시간 기본값                     | 시간이 누락 된 경우 하루 시작 사용     |  사용자가 제공 하지 않을 경우 사용할 변수의 기본값입니다.|

1. 몇 가지 예제 문장을 추가 합니다.
   
    ```
    set an alarm for {DateTime}
    set alarm {DateTime}
    alarm for {DateTime}
   ```

1. 완료 규칙을 추가 하 여 결과를 확인 합니다.

   | 설정    | 제안 값                               |설명                                     |
   | ---------- | ------------------------------------------------------- |-----|
   | 규칙 이름  | 경보 설정                                               |    규칙의 용도를 설명 하는 이름입니다. |
   | 동작    | 음성 응답 보내기-정상, {DateTime} "에 대 한 경보 집합    |규칙 조건이 참인 경우 수행할 동작입니다.

## <a name="try-it-out"></a>기능 직접 사용해 보기

1. 오른쪽 `Train` 창 위에 있는 아이콘을 선택 합니다.

1. 교육이 완료 되 면를 `Test`선택 합니다.
    - 입력: 내일 정오에 경보 설정
    - 출력: 정상, 2020-05-02 12:00:00에 대해 설정 되는 경보
    - 입력: 경보 설정
    - 출력: 시간
    - 입력: 오후 5 시
    - 출력: 정상, 2020-05-01 17:00:00에 대해 설정 되는 경보

## <a name="add-the-advanced-rules-for-confirmation"></a>확인에 대 한 고급 규칙 추가

확인은 상호 작용 규칙을 추가 하 여 달성 됩니다.

1. 기존 `SetAlarm` `+Add` 명령의 가운데 창에 선택 항목에 따라 **상호 작용 규칙** 을 추가 하 고 **상호 작용 규칙** -> **확인 명령**을 선택 합니다.

    이 규칙은 사용자에 게 알람의 날짜 및 시간을 확인 하 고 다음 턴에 확인 (예/아니요)을 예상 하도록 요청 합니다.

   | 설정               | 제안 값                                                                  | 설명                                        |
   | --------------------- | -------------------------------------------------------------------------------- | -------------------------------------------------- |
   | 규칙 이름             | 날짜 시간 확인                                                                | 규칙의 용도를 설명 하는 이름입니다.          |
   | 조건            | 필수 매개 변수-> DateTime                                                    | 규칙을 실행할 수 있는 시기를 결정 하는 조건    |   
   | 동작               | 음성 응답 보내기-{DateTime}에 대 한 경보를 설정 하 시겠습니까 >?     | 규칙 조건이 참인 경우 수행할 동작입니다. |
   | 수준과          | 사용자의 확인 필요                                                 | 다음 턴의 기대                      |
   | 실행 후 상태  | 사용자 입력 대기                                                            | 다시 설정한 후 사용자에 대 한 상태                  |
  
      > [!div class="mx-imgBorder"]
      > ![필수 매개 변수 응답 만들기](media/custom-speech-commands/add-validation-set-temperature.png)

1. 성공 확인을 처리 하는 또 다른 상호 작용 규칙 추가 (사용자가 예를)

   | 설정               | 제안 값                                                                  | 설명                                        |
   | --------------------- | -------------------------------------------------------------------------------- | -------------------------------------------------- |
   | 규칙 이름             | 수락 된 확인                                                            | 규칙의 용도를 설명 하는 이름입니다.          |
   | 조건            | 확인이 성공 했습니다 & 필수 매개 변수-> DateTime                      | 규칙을 실행할 수 있는 시기를 결정 하는 조건    |   
   | 실행 후 상태 | 완료 규칙 실행                                                          | 턴 후의 사용자 상태                   |

1. 확인 거부를 처리 하는 고급 규칙을 추가 합니다 (사용자가 언급 하지 않음).

   | 설정               | 제안 값                                                                  | 설명                                        |
   | --------------------- | -------------------------------------------------------------------------------- | -------------------------------------------------- |
   | 규칙 이름             | 거부 확인                                                                   | 규칙의 용도를 설명 하는 이름입니다.          |
   | 조건            | 필수 매개 변수 > DateTime & 확인이 거부 되었습니다.                               | 규칙을 실행할 수 있는 시기를 결정 하는 조건    |   
   | 동작               | 매개 변수 값 지우기-> DateTime & 음성 응답을 보냅니다. > 문제는 발생 하지 않습니다.  | 규칙 조건이 참인 경우 수행할 동작입니다. |
   | 실행 후의 상태 | 입력 대기                                                                   | 턴 후의 사용자 상태                   |
   | 수준과          | 사용자의 매개 변수 입력 필요-> DateTime                           | 다음 턴의 기대                      |

## <a name="try-out-the-changes"></a>변경 내용 시도

을 `Train`선택 하 고 학습 완료를 기다린 `Test`후를 선택 합니다.

- 입력: 내일 정오에 경보 설정
- 출력: 2020-05-02 12:00:00에 대 한 경보를 설정 하 시겠습니까?
- 입력: 아니요
- 출력: 문제가 없으며 어떤 시간 입니까?
- 입력: 오후 5 시
- 출력: "2020-05-01 17:00:00에 대 한 경보를 설정 하 시겠습니까?"
- 입력: 예
- 출력: 정상, 2020-05-01 17:00:00에 대해 설정 되는 경보
