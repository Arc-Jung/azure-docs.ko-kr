# <a name="how-to-add-a-confirmation-to-a-custom-command-preview"></a>방법: 사용자 지정 명령에 확인 추가(미리 보기)

이 문서에서는 명령에 확인을 추가하는 방법을 배웁니다.

## <a name="prerequisites"></a>사전 요구 사항

다음 문서의 단계를 완료해야 합니다.

- [빠른 시작: 사용자 지정 명령 만들기(미리 보기)](./quickstart-custom-speech-commands-create-new.md)
- [빠른 시작: 매개 변수를 사용하여 사용자 지정 명령 만들기(미리 보기)](./quickstart-custom-speech-commands-create-parameters.md)

## <a name="create-a-setalarm-command"></a>SetAlarm 명령 만들기

유효성 검사를 보여 주기 위해 사용자가 경보를 설정할 수 있도록 하는 새 명령을 만들어 보겠습니다.

1. [음성 스튜디오에서](https://speech.microsoft.com/) 이전에 만든 사용자 지정 명령 응용 프로그램을 엽니다.
1. 새 명령 **설정 알람** 만들기
1. DateTime이라는 매개 변수 추가

   | 설정           | 제안 값                                          | 설명                                                                                      |
   | ----------------- | ---------------------------------------------------------| ------------------------------------------------------------------------------------------------ |
   | 이름              | DateTime                                                 | 명령 매개 변수에 대한 설명 이름                                                    |
   | 필수          | true                                                     | 명령을 완료하기 전에 이 매개 변수의 값이 필요한지 여부를 나타내는 확인란 |
   | 응답 템플릿 | "- 몇 시?"                                           | 알 수 없는 경우 이 매개 변수의 값을 묻는 프롬프트                              |
   | Type              | DateTime                                                 | 숫자, 문자열 또는 날짜 시간과 같은 매개 변수 유형                                      |
   | 날짜 기본값     | 오늘 날짜가 없는 경우                             |                                                                                                  |
   | 시간 기본값     | 시간의 사용 이없는 경우 하루의 시작                      |                                                                                                  | 

1. 일부 샘플 문장 추가
   
   ```
    set an alarm for {DateTime}
    set alarm {DateTime}
    alarm for {DateTime}
   ```

1. 완료 규칙을 추가하여 결과 확인

   | 설정    | 제안 값                                         | 설명                                        |
   | ---------- | ------------------------------------------------------- | -------------------------------------------------- |
   | 규칙 이름  | 알람 설정                                               | 규칙의 목적을 설명하는 이름          |
   | 동작    | SpeechResponse - "- 확인, {DateTime}에 대한 알람 설정"       | 규칙 조건이 true일 때 취할 작업 |

## <a name="try-it-out"></a>체험

테스트 패널을 선택하고 몇 가지 상호 작용을 시도합니다.

- 입력: 정오에 내일 알람 설정
- 출력: "확인, 알람 설정 12/06/2019 12:00:00"

- 입력: 알람 설정
- 출력: "몇 시입니까?"
- 입력: 오후 5시
- 출력: "확인, 알람 설정 12/05/2019 17:00:00"

## <a name="add-the-advanced-rules-for-confirmation"></a>확인을 위한 고급 규칙 추가

1. 확인을 위해 고급 규칙을 추가합니다. 

    이 규칙은 사용자에게 알람의 날짜와 시간을 확인하도록 요청하며 다음 턴에 대한 확인(예/아니요)을 예상하고 있습니다.

   | 설정               | 제안 값                                                                  | 설명                                        |
   | --------------------- | -------------------------------------------------------------------------------- | -------------------------------------------------- |
   | 규칙 이름             | 날짜 시간 확인                                                                | 규칙의 목적을 설명하는 이름          |
   | 조건            | 필수 매개 변수 - 날짜 시간                                                    | 규칙을 실행할 수 있는 시기를 결정하는 조건    |   
   | 동작               | SpeechResponse - "- {DateTime}에 대한 알람을 설정하시겠습니까?"       | 규칙 조건이 true일 때 취할 작업 |
   | 실행 후 상태 | 입력 대기                                                                   | 턴 후 사용자에 대한 상태                  |
   | 기대          | 확인                                                                     | 다음 턴에 대한 기대                      |

1. 성공적인 확인을 처리하기 위해 고급 규칙을 추가합니다(사용자가 예라고 말)

   | 설정               | 제안 값                                                                  | 설명                                        |
   | --------------------- | -------------------------------------------------------------------------------- | -------------------------------------------------- |
   | 규칙 이름             | 수락된 확인                                                            | 규칙의 목적을 설명하는 이름          |
   | 조건            | 성공확인 & 필수 매개 변수 - DateTime                           | 규칙을 실행할 수 있는 시기를 결정하는 조건    |   
   | 실행 후 상태 | 완료 준비 완료                                                             | 턴 후 사용자의 상태                   |

1. 확인이 거부된 경우를 처리하기 위해 고급 규칙을 추가합니다(사용자가 아니오라고 말했습니다).

   | 설정               | 제안 값                                                                  | 설명                                        |
   | --------------------- | -------------------------------------------------------------------------------- | -------------------------------------------------- |
   | 규칙 이름             | 확인 거부됨                                                                   | 규칙의 목적을 설명하는 이름          |
   | 조건            | 거부확인 & 필수 매개 변수 - DateTime                               | 규칙을 실행할 수 있는 시기를 결정하는 조건    |   
   | 동작               | ClearParameter - 날짜 시간 & 음성 응답 - "문제 없음, 그 때 몇 시?"     | 규칙 조건이 true일 때 취할 작업 |
   | 실행 후 상태 | 입력 대기                                                                   | 턴 후 사용자의 상태                   |
   | 기대          | 엘리미알매개 변수 - 날짜 시간                                                      | 다음 턴에 대한 기대                      |

## <a name="try-it-out"></a>체험

테스트 패널을 선택하고 몇 가지 상호 작용을 시도합니다.

- 입력: 정오에 내일 알람 설정
- 출력: "2019년 12월 07일 12:00:00에 알람을 설정하시겠습니까?"
- 입력: 아니오
- 출력 : "문제 없습니다, 그 때 몇 시?"
- 입력: 오후 5시
- 출력: "2019년 12월 06일 17:00:00에 알람을 설정하시겠습니까?"
- 입력: 예
- 출력: "확인, 알람 설정 12/06/2019 17:00:00"
