# <a name="custom-neural-voice-gating-overview"></a>사용자 지정 신경망 제어 개요

사용자 지정 신경망을 시작 하는 프로세스에 대해 자세히 알아보세요.

## <a name="commitment-to-responsible-innovation"></a>책임 혁신에 대 한 약정

Microsoft는 책임이 있는 AI 디자인에 대 한 Microsoft의 일환으로 사용자 지정 신경망을 사용 하 여 고객을 안내 하는 일련의 자료를 제공 했습니다. 여기에서 설명 하는 지침과 정보는 [AI를 담당](https://www.microsoft.com/AI/our-approach-to-ai) 하는 Microsoft의 원칙을 기준으로 합니다.

### <a name="guidance-for-deploying-custom-neural-voice"></a>사용자 지정 신경망 배포에 대 한 지침

- [책임이 있는 배포에 대 한 지침](concepts-guidelines-responsible-deployment-synthetic.md): 연구를 기반으로 하는 최고 권장 사항
- [음성 인재에 대 한 공개](https://aka.ms/disclosure-voice-talent): 음성 인재에 대해 알고 있어야 하는 기술에 대해 알고 있어야 합니다.
- [공개 디자인](concepts-disclosure-guidelines.md): 사용자에 게 가상 음성이 사용 되는 시기를 알 수 있도록 환경을 디자인 하는 방법 및 서비스를 신뢰 합니다.

### <a name="why-custom-neural-voice-is-a-gated-technology"></a>사용자 지정 신경망이 제어 되는 기술

개인 및 사회의 권리를 보호 하 고, 사용자 컴퓨터의 투명 한 상호 작용을 촉진 하 고, 유해한 deepfakes 및 잘못 된 콘텐츠가 확산 되는 것을 방지 하는 것입니다. 이러한 이유로 사용자 지정 신경망 사용을 제어 했습니다. 고객은 응용 프로그램을 검토 한 후에만 기술에 액세스 하 고 윤리 원칙에 따라 사용 하기 위해 최선을 다하고 있습니다.

### <a name="our-gating-process"></a>Microsoft의 제어 프로세스

사용자 지정 신경망에 액세스 하려면 온라인 흡입구 폼을 작성 하 여 시작 해야 합니다. [여기](https://aka.ms/custom-neural-intake-form)에서 응용 프로그램을 시작 합니다.

사용자 지정 신경망 서비스에 대 한 액세스는이 제어 된 미리 보기 중에 제한 된 수의 고객을 지원 하기 위한 자격 기준, 심사 프로세스 및 가용성에 따라 Microsoft의 유일한 판단을 받습니다.

응용 프로그램 프로세스의 일부로 음성 [인재에 대 한 공개](https://aka.ms/disclosure-voice-talent)공유를 포함 하 여 음성 글꼴을 만들기 전에 음성 인재에서 명시적으로 작성 된 권한을 얻으려면 커밋해야 합니다. 또한 음성 글꼴을 배포할 때 구현은 서비스의 [가상 특성](concepts-disclosure-guidelines.md) 을 사용자에 게 공개 하 고, 서비스 약관에 Microsoft 가상 음성 서비스에 대 한 특성을 제공 하 고, 피드백 채널을 지 원하는 것에 동의 해야 합니다. 이렇게 하면 서비스 사용자가 문제를 보고 하 고 세부 정보를 Microsoft와 공유할 수 있습니다. 사용 약관에 대 한 자세한 내용은 [여기](https://aka.ms/custom-neural-code-of-conduct)를 참조 하세요.
