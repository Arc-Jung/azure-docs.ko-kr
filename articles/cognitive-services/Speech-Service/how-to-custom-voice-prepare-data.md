# <a name="prepare-data-to-create-a-custom-voice"></a>사용자 지정 음성을 만들려면 데이터 준비

응용 프로그램에 대 한 사용자 지정 음성 만드는 준비 된 경우 첫 번째 단계 오디오 녹음/녹화 및 음성 모델을 학습 하려면 연결 된 스크립트를 수집 하는 것입니다. 음성 녹음/녹화에 맞게 조정 된 고유한 음성을 만들려면이 데이터를 사용 하는 서비스입니다. 음성 학습 한 후 응용 프로그램에서 음성 통합을 시작할 수 있습니다.

작은 양의 개념 증명을 만들려면 데이터를 사용 하 여 시작할 수 있습니다. 그러나 사용자가 제공한 더 많은 데이터, 사용자 지정 음성 사운드를 더 자연스럽 게 합니다. 고유한 음성 모델을 학습 시킬 수를 하기 전에 해야 오디오 녹음 및 연결 된 텍스트를 기록 합니다. 이 페이지에서는 데이터 형식, 사용 하는 방법 및 각를 관리 하는 방법을 살펴보겠습니다.

## <a name="data-types"></a>데이터 형식

음성 학습 데이터 집합을 오디오 녹음 및 연결 된 기록을 사용 하 여 텍스트 파일을 포함합니다. 각 오디오 파일 (단일 문장을 단일 설정 대화 시스템용) 단일 utterance를 포함 하 고 15 초 보다 작은 수 해야 합니다.

경우에 따라 오른쪽 데이터 집합 준비 되지 않았 및 약식 또는 정식 또는 기록 하지 않고 사용 가능한 오디오 파일을 사용 하 여 사용자 지정 음성 교육을 테스트 하려고 합니다. 길이 발언에 오디오 세그먼트를 사용 하 여 성적 증명서를 준비 하는 데 유용한 도구 (베타)를 제공 하 고는 [일괄 처리 기록 API](batch-transcription.md)합니다.

이 표에서 데이터 형식 및 각 사용법 텍스트 음성 변환 사용자 지정 음성 모델을 만듭니다.

| 데이터 형식 | 설명 | 사용 시기 | 필요한 추가 서비스 | 모델 학습에 대 한 수량 | 로케일 |
| --------- | ----------- | ----------- | --------------------------- | ----------------------------- | --------- |
| **개별 길이 발언 + 일치 하는 대 본** | (.Zip) 파일의 컬렉션 오디오 (.wav)으로 개별 길이 발언. 15 초 이하의 서식이 지정 된 대 본 (.txt)을 사용 하 여 쌍을 이루는 각 오디오 파일 이어야 합니다. | 대 본을 일치 하는 전문 기록 | 학습을 위해 준비 합니다. | EN-US 및 ZH-CN 하드 필요 하지 않습니다. 두 개 2,000 개 이상의 다른 로캘에 대 한 고유 길이 발언. | 모든 사용자 지정 음성 로캘 |
| **긴 오디오 + 기록 (베타)** | Long, 되지 오디오 파일 (20 초 보다 긴)의 컬렉션 (.zip) 모든 음성된 단어를 포함 하는 기록을 (.txt)을 이룹니다. | 오디오 파일 및 일치 하는 내용 있지만 길이 발언을 분할 되지 됩니다. | (일괄 처리 기록 사용)를 구분 합니다.<br>오디오 형식 변환이 필요 합니다. | EN-US 및 ZH-CN 하드 필요 하지 않습니다. | `en-US` 및 `zh-CN` |
| **오디오 전용 (베타)** | 기록 없이 오디오 파일의 컬렉션 (.zip). | 오디오 파일을 기록 하지 않고 사용할 수 있습니다. | 조각화 + 기록 생성 (일괄 처리 기록 사용).<br>오디오 형식 변환이 필요 합니다.| 에 대 한 하드 않아도 `en-US` 고 `zh-CN`입니다. | `en-US` 및 `zh-CN` |

파일은 형식에 따라 데이터 집합으로 그룹화하고 zip 파일로 업로드해야 합니다. 각 데이터 집합은 단일 데이터 형식만 포함할 수 있습니다.

> [!NOTE]
> 구독 당 가져올 허용 하는 데이터 집합의 최대 수는 10 개의.zip 파일이 무료로 (F0) 구독 사용자 및 표준 구독 (S0) 사용자의 경우 500.

## <a name="individual-utterances--matching-transcript"></a>개별 길이 발언 + 일치 하는 대 본

개별 길이 발언 및 두 가지 방법으로 일치 하는 대 본의 녹화/녹음을 준비할 수 있습니다. 하거나 스크립트를 작성 하 고 음성 재능 읽은 또는 공개적으로 사용 가능한 오디오를 사용 하 고 텍스트를 기록 합니다. 후자의 경우 오디오 파일에서 "음" 및 기타 추임새, 더듬거림, 중얼거리는 말이나 틀린 발음 등을 편집합니다.

좋은 음성 글꼴을 생성 하려면 고성능 마이크를 사용 하 여 자동 방에 녹음/녹화를 만듭니다. 일관 된 볼륨을 읽기 속도, 피치 및 음성의 표현 mannerisms 말하기는 필수적입니다.

> [!TIP]
> 프로덕션용으로 음성을 만들려면 전문적인 녹음 스튜디오에서 성우가 녹음하도록 하는 것이 좋습니다. 자세한 내용은 [사용자 지정 음성에 대한 음성 샘플을 녹음하는 방법](record-custom-voice-samples.md)을 참조하세요.

### <a name="audio-files"></a>오디오 파일

각 오디오 파일 (단일 문장을 단일 설정 대화 시스템의) 단일 utterance를 15 초 보다 작거나 포함 되어야 합니다. 모든 파일은 동일한 음성 언어 여야 합니다. 영 bi 이중을 제외 하 고, 다중 언어 사용자 지정 텍스트 음성 변환 음성 지원 되지 않습니다. 각 오디오 파일에는 파일 이름 확장명이.wav를 사용 하 여 고유한 숫자 파일 이름이 있어야 합니다.

오디오를 준비 하는 경우 다음이 지침을 따릅니다.

| 자산 | 값 |
| -------- | ----- |
| 파일 형식 | .Zip 파일을 그룹화 한 RIFF (.wav) |
| 샘플링 레이트 | 16,000 Hz 이상 |
| 샘플 형식 | PCM, 16비트 |
| 파일 이름 | 숫자,.wav 확장명을 사용 하 여 선택 합니다. 중복 되는 파일 이름은 허용 되지 않습니다. |
| 오디오 길이 | 15 초 보다 짧은 |
| 보관 형식 | .zip |
| 최대 보관 크기 | 200MB |

> [!NOTE]
> Hz 16,000 보다 낮은 샘플링 주기를 사용 하 여.wav 파일은 거부 됩니다. .Zip 파일을 서로 다른 샘플링을 사용 하 여.wav 파일에 있으면만 해당 같거나 높은 16,000 Hz 보다 가져오게 됩니다. 현재, 포털은 최대 200MB의 .zip 보관 파일을 가져옵니다. 그러나 여러 보관 파일을 업로드할 수 있습니다.

### <a name="transcripts"></a>스크립트(Transcript)

기록 파일이 일반 텍스트 파일입니다. 에 기록을 준비 하려면 이러한 지침을 따르세요.

| 자산 | 값 |
| -------- | ----- |
| 파일 형식 | 일반 텍스트(.txt) |
| 인코딩 형식 | ANSI/ASCII, utf-8, u t F-8-BOM, u t F-16-LE, 또는 u t F-16-이어야 합니다. ZH-CN, ANSI/ASCII 및 u t F-8 인코딩을 지원 되지 않습니다. |
| 줄당 발언의 # | **하나의** -기록 파일의 각 줄 뒤에 해당 기록, 오디오 파일 중 하나의 이름이 포함 되어야 합니다. 파일 이름과 전사는 탭(\t)으로 구분 해야 합니다. |
| 최대 파일 크기 | 50MB |

다음은 기록을 구성 방법을 utterance 여 utterance 하나.txt 파일에서의 예:

```
0000000001[tab] This is the waistline, and it's falling.
0000000002[tab] We have trouble scoring.
0000000003[tab] It was Janet Maslin.
```
것은 기록을 해당 하는 오디오의 100% 정확 하 게 기록 합니다. 오류를 기록에서 학습 하는 동안 품질 손실을 소개 합니다.

> [!TIP]
> 텍스트 음성 변환 음성을 선택 길이 발언 (또는 스크립트를 작성)를 고려 하는 프로덕션을 빌드할 때에 음성 검사와 효율성 계정입니다. 원하는 결과를 얻는 데 문제가 있으세요? [사용자 지정 음성 문의](mailto:speechsupport@microsoft.com) 팀이 찾기 미국에 대 한 자세한 내용은 참조 하세요.

## <a name="long-audio--transcript-beta"></a>긴 오디오 + 기록 (베타)

경우에 따라 있습니다 수 있는 세분화 되지 오디오를 사용할 수 있습니다. 긴 오디오 파일을 분할 하 고 기록을 만들 수 있도록 사용자 지정 음성 포털을 통해 서비스 (베타)를 제공 합니다. 유지 따라서에서이 서비스 요금이 음성-텍스트 구독 사용 현황을 포함 합니다.

> [!NOTE]
> 장기 오디오 조각화 서비스는 음성-텍스트를 지 원하는 표준 구독 (S0) 사용자의 일괄 처리 기록 기능을 활용 합니다. 조각화를 처리 하는 동안 오디오 파일과 기록을 전송할 사용자 지정 음성 서비스에 데이터에 대 한 정확도 향상 시킬 수 있도록 인식 모델을 구체화 합니다. 이 프로세스 중 데이터가 유지 됩니다. 조각화 완료 되 면 분할 길이 발언만 및 해당 매핑 내용 다운로드 및 학습에 저장 됩니다.

### <a name="audio-files"></a>오디오 파일

조각화에 대 한 오디오를 준비 하는 경우 다음이 지침을 따릅니다.

| 자산 | 값 |
| -------- | ----- |
| 파일 형식 | 적어도 16 khz-16 비트 PCM 또는. mp3 이상 256kbps에서 그룹화 된.zip 파일의 비트 전송률을 사용 하 여 샘플링 비율이 RIFF (.wav) |
| 파일 이름 | ASCII 문자에만 해당 합니다. 이름에 유니코드 문자가 실패 (중국어 문자 또는 기호 같은 예를 들어, "-"). 중복 이름은 허용 되지 않습니다. |
| 오디오 길이 | 20 개 더 이상 시간 (초) |
| 보관 형식 | .zip |
| 최대 보관 크기 | 200MB |

모든 오디오 파일을 zip 파일로 그룹화 할 수 해야 합니다. 하나의 오디오 zip에.wav 파일 및. mp3 파일을 저장 하려면 확인 있지만 zip 파일에 없는 하위 폴더를 사용할 수 있습니다. 예를 들어, 이름은 'kingstory.wav', 45 초 길이의 다른 하나의 명명 된 'queenstory.mp3', 200 초 길이의 모든 하위 폴더 제외 오디오 파일을 포함 한 zip 파일을 업로드할 수 있습니다. 모든. mp3 파일 처리 후.wav 형식으로 변환 됩니다.

### <a name="transcripts"></a>스크립트(Transcript)

이 표에 나열 된 사양에 성적 증명서를 준비 되어야 합니다. 각 오디오 파일 기록을 일치 해야 합니다.

| 자산 | 값 |
| -------- | ----- |
| 파일 형식 | 일반 텍스트 (.txt).zip으로 그룹화 |
| 파일 이름 | 일치 하는 오디오 파일과 같은 이름을 사용 하 여 |
| 인코딩 형식 | U t F-8-BOM만 |
| 줄당 발언의 # | 제한 없음 |
| 최대 파일 크기 | 50M |

이 데이터 형식의 모든 기록 파일을 zip 파일을 그룹화 할 수 해야 합니다. 없는 하위 폴더는 zip 파일에 허용 됩니다. 예를 들어, 'kingstory.wav' 45 초 라는 오디오 파일을 포함 한 zip 파일 업로드 하 고 다른 하나의 명명 된 'queenstory.mp3', 200 초 시간. 두 대 본, 하나의 명명 된 'kingstory.txt'는 다른 하나의 'queenstory.txt'를 포함 하는 다른 zip 파일을 업로드 해야 합니다. 각 일반 텍스트 파일 내에서 일치 하는 오디오에 대 한 올바른 전체 기록을 제공 합니다.

데이터 집합이 성공적으로 업로드 되 면 할 수 있습니다 오디오 파일 길이 발언 제공 대 본을 기반으로 분할 합니다. 분할 된 표현 및 일치 하는 기록 데이터 집합을 다운로드 하 여 확인할 수 있습니다. 고유 Id는 분할 된 표현에 자동으로 할당 됩니다. 제공한 성적 증명서는 100% 정확한 없는지 확인 하는 것입니다. 성적 증명서에 오류 오디오 구분 하는 동안 정확도 줄이고 더 나중에 제공 되는 교육 단계에 품질 손실을 제공할 수 있습니다.

## <a name="audio-only-beta"></a>오디오 전용 (베타)

기록에 대 한 오디오 녹음/녹화에 없는 경우 사용 합니다 **오디오 전용** 데이터를 업로드 하는 옵션입니다. 시스템 오디오 파일을 기록 하 고 분할 수 있습니다. 염두에,이 서비스는 음성-텍스트 구독 사용 현황에 대해 계산 됩니다.

오디오를 준비 하는 경우 다음이 지침을 따릅니다.

> [!NOTE]
> 장기 오디오 조각화 서비스는 음성-텍스트를 지 원하는 표준 구독 (S0) 사용자의 일괄 처리 기록 기능을 활용 합니다.

| 파일 이름 | ASCII 문자에만 해당 합니다. 이름에 유니코드 문자가 실패 (중국어 문자 또는 기호 같은 예를 들어, "-"). 중복 이름을 허용 하지 않았습니다. |
| 오디오 길이 | 20 개 더 이상 시간 (초) |
| 보관 형식 | .zip |
| 최대 보관 크기 | 200MB |

모든 오디오 파일을 zip 파일로 그룹화 할 수 해야 합니다. 없는 하위 폴더는 zip 파일에 허용 됩니다. 데이터 집합 업로드가 완료 되 면 할 수 있습니다 음성 일괄 처리 기록 서비스에 따라 표현으로 오디오 파일을 분할 합니다. 고유 Id는 분할 된 표현에 자동으로 할당 됩니다. 대 본을 일치 하는 음성 인식을 통해 생성 됩니다. 모든. mp3 파일 처리 후.wav 형식으로 변환 됩니다. 분할 된 표현 및 일치 하는 기록 데이터 집합을 다운로드 하 여 확인할 수 있습니다.
