# <a name="prepare-data-to-create-a-custom-voice"></a>사용자 지정 음성을 만들기 위해 데이터 준비

응용 프로그램에 대한 사용자 지정 텍스트 음성 음성을 만들 준비가 되면 첫 번째 단계는 음성 모델 교육을 시작하기 위해 오디오 녹음 및 관련 스크립트를 수집하는 것입니다. 음성 서비스는 이 데이터를 사용하여 녹음의 음성과 일치하도록 조정된 고유한 음성을 만듭니다. 음성을 학습한 후 응용 프로그램에서 음성 합성을 시작할 수 있습니다.

소량의 데이터로 시작하여 개념 증명을 만들 수 있습니다. 그러나 데이터를 많이 제공할수록 사용자 지정 음성이 더 자연스럽게 들립니다. 텍스트 음성 변환 모델을 학습하려면 오디오 녹음 및 관련 텍스트 기록이 필요합니다. 이 페이지에서는 데이터 형식, 데이터 형식사용 방법 및 각 데이터를 관리하는 방법을 검토합니다.

## <a name="data-types"></a>데이터 형식

음성 학습 데이터 집합에는 오디오 녹음 및 관련 기록이 포함된 텍스트 파일이 포함됩니다. 각 오디오 파일에는 단일 발언(대화 시스템의 경우 한 문장 또는 한 번의 회전)이 포함되어야 하며 길이가 15초 미만이어야 합니다.

경우에 따라 올바른 데이터 집합이 준비되지 않았을 수 있으며 성적 증명서유무에 관계없이 짧거나 긴 오디오 파일로 사용자 지정 음성 교육을 테스트하려고 할 수 있습니다. 우리는 당신이 발언으로 오디오를 분할하고 [배치 전사 API를](batch-transcription.md)사용하여 성적 증명서를 준비하는 데 도움이 도구 (베타)를 제공합니다.

이 표에는 데이터 형식과 각 형식이 사용자 지정 텍스트 음성 변환 모델을 만드는 데 사용되는 방법을 나열합니다.

| 데이터 형식 | 설명 | 사용 시기 | 추가 서비스 필요 | 모델 학습수량 | 로캘 |
| --------- | ----------- | ----------- | --------------------------- | ----------------------------- | --------- |
| **개별 발언 + 일치하는 성적 증명서** | 개별 발언으로 오디오 파일(.wav)의 컬렉션(.zip)입니다. 각 오디오 파일의 길이는 15초 이하여서 형식이 지정된 자막(.txt)과 페어링되어야 합니다. | 일치하는 성적 증명서와 전문 녹음 | 교육 준비 완료. | en-US 및 zh-CN에 대한 하드 요구 사항은 없습니다. 다른 로캘에 대한 2,000개 이상의 뚜렷한 발언. | [모든 사용자 지정 음성 로캘](language-support.md#customization) |
| **긴 오디오 + 성적 증명서 (베타)** | 모든 음성 단어가 포함된 성적 증명서(.txt)와 페어링된 길고 세분화되지 않은 오디오 파일(20초 이상)의 컬렉션입니다. | 오디오 파일과 일치하는 성적 증명서가 있지만 발언으로 분할되지는 않습니다. | 세분화(배치 전사 사용).<br>필요한 경우 오디오 포맷 변환. | 하드 요구 사항 없음  | [모든 사용자 지정 음성 로캘](language-support.md#customization) |
| **오디오 전용(베타)** | 성적 증명서가없는 오디오 파일의 컬렉션 (.zip). | 성적 증명서 없이 오디오 파일만 사용할 수 있습니다. | 세분화 + 전사 생성(배치 전사 사용).<br>필요한 경우 오디오 포맷 변환.| 하드 요구 사항 없음 | [모든 사용자 지정 음성 로캘](language-support.md#customization) |

파일은 유형별로 데이터 집합으로 그룹화하고 zip 파일로 업로드해야 합니다. 각 데이터 집합에는 단일 데이터 형식만 포함될 수 있습니다.

> [!NOTE]
> 구독당 가져올 수 있는 최대 데이터 집합 수는 무료 구독(F0) 사용자의 경우 10.zip 파일, 표준 구독(S0) 사용자의 경우 500개입니다.

## <a name="individual-utterances--matching-transcript"></a>개별 발언 + 일치하는 성적 증명서

개별 발언의 레코딩과 일치하는 성적표는 두 가지 방법으로 준비할 수 있습니다. 스크립트를 작성하고 음성 재능으로 읽거나 공개적으로 사용 가능한 오디오를 사용하여 텍스트로 기록합니다. 후자의 경우 오디오 파일에서 "음" 및 기타 추임새, 더듬거림, 중얼거리는 말이나 틀린 발음 등을 편집합니다.

좋은 음성 글꼴을 생성하려면 고품질 마이크가 있는 조용한 방에서 녹음을 만듭니다. 일관된 볼륨, 말하기 속도, 말하기 피치 및 표현 매너리즘은 필수적입니다.

> [!TIP]
> 프로덕션용으로 음성을 만들려면 전문적인 녹음 스튜디오에서 성우가 녹음하도록 하는 것이 좋습니다. 자세한 내용은 [사용자 지정 음성에 대한 음성 샘플을 녹음하는 방법](record-custom-voice-samples.md)을 참조하세요.

### <a name="audio-files"></a>오디오 파일

각 오디오 파일에는 15초 미만의 단일 발언(한 문장 또는 대화 시스템의 한 번 회전)이 포함되어야 합니다. 모든 파일은 동일한 음성 언어여야 합니다. 중국어-영어 이중 언어를 제외한 다국어 사용자 지정 텍스트 음성은 지원되지 않습니다. 각 오디오 파일에는 파일 이름 확장자 .wav가 있는 고유한 숫자 파일 이름이 있어야 합니다.

오디오를 준비할 때 다음 지침을 따르십시오.

| 속성 | 값 |
| -------- | ----- |
| 파일 형식 | .zip 파일로 그룹화된 RIFF(.wav) |
| 샘플링 레이트 | 최소 16,000Hz |
| 샘플 형식 | PCM, 16비트 |
| 파일 이름 | 숫자, .wav 확장. 중복 된 파일 이름이 허용되지 않습니다. |
| 오디오 길이 | 15초 미만 |
| 보관 형식 | .zip |
| 최대 보관 크기 | 2048 MB |

> [!NOTE]
> 샘플링 속도가 16,000Hz보다 낮은 .wav 파일은 거부됩니다. .zip 파일에 샘플 속도가 다른 .wav 파일이 포함되어 있으면 16,000Hz이상과 같거나 그 이상인 파일만 가져옵니다. 현재, 포털은 최대 200MB의 .zip 보관 파일을 가져옵니다. 그러나 여러 보관 파일을 업로드할 수 있습니다.

### <a name="transcripts"></a>스크립트(Transcript)

전사 파일은 일반 텍스트 파일입니다. 다음 지침을 사용하여 전사를 준비합니다.

| 속성 | 값 |
| -------- | ----- |
| 파일 형식 | 일반 텍스트(.txt) |
| 인코딩 형식 | ANSI/ASCII, UTF-8, UTF-8-BOM, UTF-16-LE 또는 UTF-16-BE. zh-CN의 경우 ANSI/ASCII 및 UTF-8 인코딩은 지원되지 않습니다. |
| 줄당 발언의 # | **하나** - 전사 파일의 각 줄에는 오디오 파일 중 하나의 이름이 포함되어야 하고 그 다음에 해당 전사가 뒤따릅니다. 파일 이름과 전사는 탭(\t)으로 구분 해야 합니다. |
| 최대 파일 크기 | 2048 MB |

다음은 하나의 .txt 파일에서 발언으로 성적 증명서를 구성하는 방법의 예입니다.

```
0000000001[tab] This is the waistline, and it's falling.
0000000002[tab] We have trouble scoring.
0000000003[tab] It was Janet Maslin.
```
성적 증명서는 해당 오디오의 100 % 정확한 전사하는 것이 중요합니다. 성적 증명서의 오류는 훈련 중에 품질 손실을 소개합니다.

> [!TIP]
> 프로덕션 텍스트 음성 음성 을 작성할 때 음성 적용 범위와 효율성을 모두 고려한 발언(또는 스크립트 작성)을 선택합니다. 원하는 결과를 얻는 데 문제가 있으세요? [사용자 지정 음성](mailto:speechsupport@microsoft.com) 팀에 문의하여 상담에 대해 자세히 알아보십시오.

## <a name="long-audio--transcript-beta"></a>긴 오디오 + 성적 증명서 (베타)

경우에 따라 분할된 오디오를 사용할 수 없을 수도 있습니다. 우리는 당신이 긴 오디오 파일을 분할하고 전사를 만들 수 있도록 사용자 정의 음성 포털을 통해 서비스 (베타)를 제공합니다. 이 서비스는 음성-텍스트 구독 사용에 대해 요금이 부과됩니다.

> [!NOTE]
> 긴 오디오 세분화 서비스는 표준 구독(S0) 사용자만 지원하는 음성-텍스트의 일괄 전사 기능을 활용합니다. 세분화를 처리하는 동안 오디오 파일과 자막이 사용자 지정 음성 서비스로 전송되어 인식 모델을 구체화하여 데이터에 대한 정확도를 향상시킬 수 있습니다. 이 프로세스 중에는 데이터가 보존되지 않습니다. 세분화가 완료되면 표현이 분할되고 매핑 성적증명서만 다운로드 및 학습에 저장됩니다.

### <a name="audio-files"></a>오디오 파일

분할을 위해 오디오를 준비할 때 다음 지침을 따르십시오.

| 속성 | 값 |
| -------- | ----- |
| 파일 형식 | PCM 또는 .mp3에서 최소 16khz-16비트의 샘플링 속도를 가진 RIFF(.wav)는 .zip 파일로 그룹화되어 256KBps 이상의 비트 레이트로 분류됩니다. |
| 파일 이름 | ASCII 및 유니코드 문자가 지원됩니다. 중복 된 이름은 허용되지 않습니다. |
| 오디오 길이 | 20초 보다 길다 |
| 보관 형식 | .zip |
| 최대 보관 크기 | 2048 MB |

모든 오디오 파일은 zip 파일로 그룹화되어야 합니다. .wav 파일과 .mp3 파일을 하나의 오디오 지퍼에 넣는 것은 괜찮습니다. 예를 들어 'kingstory.wav', 45초 길이의 오디오 파일, 200초 길이의 'queenstory.mp3'라는 다른 오디오가 포함된 zip 파일을 업로드할 수 있습니다. 모든 .mp3 파일은 처리 후 .wav 형식으로 변환됩니다.

### <a name="transcripts"></a>스크립트(Transcript)

성적 증명서는 이 표에 나열된 사양에 맞게 준비되어야 합니다. 각 오디오 파일은 성적 증명서와 일치해야 합니다.

| 속성 | 값 |
| -------- | ----- |
| 파일 형식 | 일반 텍스트(.txt)는 .zip으로 그룹화됨 |
| 파일 이름 | 일치하는 오디오 파일과 동일한 이름 사용 |
| 인코딩 형식 | UTF-8-BOM 전용 |
| 줄당 발언의 # | 제한 없음 |
| 최대 파일 크기 | 2048 MB |

이 데이터 형식의 모든 성적 증명서 파일은 zip 파일로 그룹화되어야 합니다. 예를 들어 'kingstory.wav'라는 오디오 파일, 45초 길이, 200초 길이의 'queenstory.mp3'라는 오디오 파일이 포함된 zip 파일을 업로드했습니다. 'kingstory.txt'라는 이름의 두 개의 성적 증명서가 포함된 다른 zip 파일을 업로드해야 합니다. 각 일반 텍스트 파일 내에서 일치하는 오디오에 대한 전체 올바른 기록을 제공합니다.

데이터 집합이 성공적으로 업로드되면 제공된 성적 증명서에 따라 오디오 파일을 발언으로 분할하는 데 도움이 됩니다. 데이터 집합을 다운로드하여 분할된 발언과 일치하는 성적표를 확인할 수 있습니다. 고유 아이디는 분할된 발언에 자동으로 할당됩니다. 제공한 성적증명서가 100% 정확한지 확인하는 것이 중요합니다. 성적 증명서의 오류는 오디오 세분화 중에 정확도를 감소시키고 나중에 오는 교육 단계에서 품질 손실을 더 유발할 수 있습니다.

## <a name="audio-only-beta"></a>오디오 전용(베타)

오디오 녹음에 대한 기록이 없는 경우 **오디오 전용** 옵션을 사용하여 데이터를 업로드합니다. 우리의 시스템은 당신이 분할하고 오디오 파일을 전사 하는 데 도움이 수 있습니다. 이 서비스는 음성-텍스트 구독 사용에 계산됩니다.

오디오를 준비할 때 다음 지침을 따르십시오.

> [!NOTE]
> 긴 오디오 세분화 서비스는 표준 구독(S0) 사용자만 지원하는 음성-텍스트의 일괄 전사 기능을 활용합니다.

| 속성 | 값 |
| -------- | ----- |
| 파일 형식 | PCM 또는 .mp3에서 최소 16khz-16비트의 샘플링 속도를 가진 RIFF(.wav)는 .zip 파일로 그룹화되어 256KBps 이상의 비트 레이트로 분류됩니다. |
| 파일 이름 | ASCII 및 유니코드 문자가 지원됩니다. 중복 된 이름이 허용되지 않습니다. |
| 오디오 길이 | 20초 보다 길다 |
| 보관 형식 | .zip |
| 최대 보관 크기 | 2048 MB |

모든 오디오 파일은 zip 파일로 그룹화되어야 합니다. 데이터 집합이 성공적으로 업로드되면 음성 배치 전사 서비스를 기반으로 오디오 파일을 발언으로 분할할 수 있습니다. 고유 아이디는 분할된 발언에 자동으로 할당됩니다. 일치하는 성적 증명서는 음성 인식을 통해 생성됩니다. 모든 .mp3 파일은 처리 후 .wav 형식으로 변환됩니다. 데이터 집합을 다운로드하여 분할된 발언과 일치하는 성적표를 확인할 수 있습니다.
