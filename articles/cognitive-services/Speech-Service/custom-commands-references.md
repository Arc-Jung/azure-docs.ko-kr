# <a name="custom-commands-concepts-and-definitions"></a>사용자 지정 명령 개념 및 정의

이 문서는 사용자 지정 명령 응용 프로그램의 개념 및 정의에 대 한 참조로 사용 됩니다.

## <a name="commands-configuration"></a>명령 구성
명령은 사용자 지정 명령 응용 프로그램의 기본 구성 요소입니다. 명령은 사용자가 정의한 특정 태스크를 완료 하는 데 필요한 구성 집합입니다.

### <a name="example-sentences"></a>예제 문장
예 길이 발언는 사용자가 특정 명령을 트리거할 수 있는 설정 된 예제입니다. 길이 발언 샘플만 제공 해야 하 고 완전 한 목록은 제공 하지 않아야 합니다. 

### <a name="parameters"></a>매개 변수
매개 변수는 작업을 완료 하는 명령에 필요한 정보입니다. 복잡 한 시나리오에서 매개 변수를 사용 하 여 사용자 지정 작업을 트리거하는 조건을 정의할 수도 있습니다.

### <a name="completion-rules"></a>완료 규칙
완료 규칙은 명령을 수행할 준비가 된 후 (예: 규칙의 모든 조건이 충족 되는 경우) 실행 되는 일련의 규칙입니다.

### <a name="interaction-rules"></a>상호 작용 규칙
상호 작용 규칙은 보다 구체적 이거나 복잡 한 상황을 처리 하는 추가 규칙입니다. 유효성 검사를 더 추가 하거나 확인 또는 1 단계 수정과 같은 고급 기능을 구성할 수 있습니다. 사용자 고유의 사용자 지정 상호 작용 규칙을 빌드할 수도 있습니다.

## <a name="parameters-configuration"></a>매개 변수 구성

매개 변수는 명령에서 작업을 완료 하는 데 필요한 정보입니다. 복잡 한 시나리오에서 매개 변수를 사용 하 여 사용자 지정 작업을 트리거하는 조건을 정의할 수도 있습니다.

### <a name="name"></a>이름
매개 변수는 name 속성으로 식별 됩니다. 매개 변수에 대 한 설명이 포함 된 이름을 항상 지정 해야 합니다. 조건, 음성 응답 또는 기타 작업을 생성 하는 경우와 같이 여러 섹션에서 매개 변수를 참조할 수 있습니다.
 
### <a name="isglobal"></a>IsGlobal
이 확인란은이 매개 변수의 범위가 응용 프로그램의 모든 명령에서 공유 되는지 여부를 나타냅니다. 매개 변수가 전역 이면 모든 명령 범위에서 해당 값을 제공할 수 있습니다. 값을 할당 한 후에는 모든 명령에서 값을 참조할 수 있습니다. 

### <a name="required"></a>필요한 공간
이 확인란은 명령 처리 또는 완료에이 매개 변수의 값이 필요한 지 여부를 나타냅니다. 매개 변수가 필수로 표시 된 경우 사용자에 게 값을 제공 하 라는 메시지를 표시 하려면 응답을 구성 해야 합니다.

### <a name="type"></a>형식
사용자 지정 명령은 다음과 같은 매개 변수 유형을 지원 합니다.

* DateTime
* Geography
* number
* String

이러한 모든 매개 변수 형식은 Azure Portal에서 구성할 수 있는 기본 값 구성을 지원 합니다.

### <a name="configuration"></a>Configuration
구성은 형식 문자열에 대해서만 정의 된 매개 변수 속성입니다. 지원되는 값은 다음과 같습니다.

* **없음**.
* **전체 입력 수락**: 사용 하도록 설정 하면 매개 변수가 모든 입력 utterance을 허용 합니다. 이 옵션은 사용자에 게 전체 utterance 매개 변수가 필요한 경우에 유용 합니다. 예는 우편 주소입니다.
* **외부 카탈로그에서 미리 정의 된 입력 값 허용**:이 값은 다양 한 값을 가정할 수 있는 매개 변수를 구성 하는 데 사용 됩니다. 예를 들어 판매 카탈로그가 있습니다. 이 경우 카탈로그는 외부 웹 끝점에서 호스팅되며 독립적으로 구성할 수 있습니다.
* **내부 카탈로그에서 미리 정의 된 입력 값 허용**:이 값은 몇 가지 값을 가정할 수 있는 매개 변수를 구성 하는 데 사용 됩니다. 이 경우 Speech Studio에서 값을 구성 해야 합니다.


### <a name="validation"></a>유효성 검사
유효성 검사는 매개 변수 값에 대 한 제약 조건을 구성할 수 있도록 하는 특정 매개 변수 형식에 적용할 수 있는 구문입니다. 현재 사용자 지정 명령은 다음 매개 변수 형식에 대 한 유효성 검사를 지원 합니다.

* DateTime
* number

## <a name="rules-configuration"></a>규칙 구성
사용자 지정 명령의 규칙은 충족 될 경우 일련의 *작업*을 실행 하는 *조건* 집합으로 정의 됩니다. 규칙을 사용 하 여 다음 번에 *실행 후 상태* 와 *기대치* 를 구성할 수도 있습니다.

### <a name="types"></a>형식
사용자 지정 명령은 다음 규칙 범주를 지원 합니다.

* **완료 규칙**: 이러한 규칙은 명령 처리 시 실행 되어야 합니다. 조건을 충족 하는이 섹션에서 구성 된 모든 규칙이 실행 됩니다. 
* **상호 작용 규칙**: 이러한 규칙을 사용 하 여 추가 사용자 지정 유효성 검사, 확인 및 1 단계 수정을 구성 하거나 다른 사용자 지정 대화 논리를 수행할 수 있습니다. 상호 작용 규칙은 처리의 각 턴에서 평가 되며 완료 규칙을 트리거하는 데 사용할 수 있습니다.

규칙의 일부로 구성 된 다른 작업은 제작 포털에 표시 되는 순서 대로 실행 됩니다.

### <a name="conditions"></a>조건
조건은 규칙을 실행 하기 위해 충족 해야 하는 요구 사항입니다. 규칙 조건은 다음과 같은 형식일 수 있습니다.

* **매개 변수 값이 같음**: 구성 된 매개 변수의 값이 특정 값과 같습니다.
* **매개 변수 값 없음**: 구성 된 매개 변수는 값을 가질 수 없습니다.
* **필수 매개 변수**: 구성 된 매개 변수에 값이 있습니다.
* **모든 필수 매개 변수**: 필수로 표시 된 모든 매개 변수에 값이 있습니다.
* **업데이트 된 매개 변수**: 현재 입력 (utterance 또는 활동)을 처리 한 결과로 하나 이상의 매개 변수 값이 업데이트 되었습니다.
* **확인이 성공 했습니다**. 입력 utterance 또는 활동이 성공적으로 확인 되었습니다 (예).
* **확인이 거부 되었습니다**. 입력 utterance 또는 활동에 확인이 실패 했습니다 (no).
* **이전 명령을 업데이트 해야**함:이 상태는 업데이트와 함께 부정 확인을 catch 하려는 경우 인스턴스에 사용 됩니다. 내부적으로이 조건은 대화 엔진이 이전 턴과 동일 하 고 사용자가 업데이트를 사용 하 여 응답 한 부정적 확인을 감지 하는 경우에 대해 구성 됩니다.

### <a name="actions"></a>동작
* **음성 응답 보내기**: 음성 응답을 다시 클라이언트로 보냅니다.
* **매개 변수 값 업데이트**: 명령 매개 변수의 값을 지정 된 값으로 업데이트 합니다.
* **Clear parameter value**: command 매개 변수 값을 지웁니다.
* **웹 끝점 호출**: 웹 끝점을 호출 합니다.
* **클라이언트에 작업 보내기**: 클라이언트에 사용자 지정 작업을 보냅니다.

### <a name="expectations"></a>수준과
기대치는 다음 사용자 입력의 처리에 대 한 힌트를 구성 하는 데 사용 됩니다. 다음과 같은 형식이 지원됩니다.

* **사용자에 게 확인 필요**:이 예상은 다음 사용자 입력에 대 한 확인 (예/아니요)을 응용 프로그램에 예상 하도록 지정 합니다.
* **사용자의 매개 변수 입력 필요**:이 예상은 응용 프로그램에서 사용자 입력을 필요로 하는 명령 매개 변수를 하나 이상 지정 합니다.

### <a name="post-execution-state"></a>실행 후 상태
실행 후 상태는 현재 입력 (utterance 또는 활동)을 처리 한 후 대화 상자 상태입니다. 다음과 같은 유형입니다.

* **명령이 완료 됨**: 명령을 완료 하면 명령에 대 한 추가 규칙이 처리 되지 않습니다.
* **완료 규칙 실행**: 유효한 모든 완료 규칙을 실행 합니다.
* **사용자 입력 대기**: 다음 사용자 입력을 기다립니다.
