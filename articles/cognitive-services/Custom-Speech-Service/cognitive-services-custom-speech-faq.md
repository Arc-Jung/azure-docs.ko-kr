# <a name="custom-speech-service-frequently-asked-questions"></a>Custom Speech Service 질문과 대답

[!INCLUDE [Deprecation note](../../../includes/cognitive-services-custom-speech-deprecation-note.md)] 

여기 FAQ에서 질문에 대한 답변을 찾을 수 없는 경우에는 [StackOverflow](https://stackoverflow.com/questions/tagged/project-oxford+or+microsoft-cognitive)와 [UserVoice](https://cognitive.uservoice.com/)에서 Custom Speech Service 커뮤니티에 문의해 보세요.

## <a name="general"></a>일반

**질문**: 데이터 세트나 모델의 처리가 완료되면 어떻게 알 수 있나요?

**대답**: 현재 처리 완료를 확인하는 방법은 테이블에 있는 모델이나 데이터 세트의 상태를 확인하는 것뿐입니다.
처리가 완료되면 "준비" 상태가 됩니다.
이메일 알림 등의 통신 처리 상태를 위해 향상된 메서드를 작업 중입니다.

**질문**: 모델을 한 번에 여러 개 만들 수 있나요?

**대답**: 컬렉션에 포함할 수 있는 모델의 수에는 제한이 없지만 각 페이지에서는 모델을 한 번에 하나씩만 만들 수 있습니다.
예를 들어, 현재 프로세스 단계인 언어 모델이 있으면 언어 모델 생성 프로세스를 시작할 수 없습니다.
단 음향 모델과 언어 모델은 동시에 처리할 수 있습니다. 

**질문**: 모델을 잘못 만든 경우 데이터 가져오기 또는 모델 만들기를 진행 중인 경우 어떻게 취소하나요? 

**대답**: 현재는 음향 적응 또는 언어 적응 프로세스를 롤백할 수 없습니다.
가져온 데이터는 가져오기가 완료된 후 삭제할 수 있습니다.

**질문**: 검색 및 받아쓰기 모델과 대화형 모델의 차이는 무엇인가요?

**대답**: Custom Speech Service에서 선택할 수 있는 기준 음향 및 언어 모델은 두 가지입니다.
검색 쿼리 또는 받아쓰기. Microsoft 대화형 AM은 대화 스타일로 통용되는 음성을 인식하는 데 적합합니다.
이런 유형의 음성은 일반적으로 콜센터나 회의에서처럼 다른 사람을 대상으로 합니다.

**질문**: 기존 모델(모델 스태킹)을 업데이트할 수 있나요?

**대답**: 기존 모델을 새 데이터로 업데이트하는 기능은 제공되지 않습니다.
새 데이터 집합이 있고 기존 모델을 사용자 지정하려는 경우 새 데이터 및 사용한 이전 데이터 집합으로 다시 적용해야 합니다.
이전 및 새 데이터 집합은 단일 .zip(음향 데이터인 경우) 또는 .txt 파일(언어 데이터인 경우)로 결합되어야 합니다. 적용이 완료되면 새로 업데이트된 몯레은 새 엔드포인트를 가져오도록 다시 배포되어야 합니다.

**질문**: 기본값보다 더 많은 수의 작업을 동시에 수행해야 하는 경우에는 어떻게 해야 하나요? 

**대답**: 동시 요청을 5개씩(배율 단위) 추가하여 모델을 강화할 수 있습니다. 각 배율 단위는 모델에서 5개의 오디오 스트림을 동시에 처리할 수 있음을 보장합니다. 최대 100 배율 단위(또는 동시 요청 500건)를 구입할 수 있습니다.

더 큰 규모가 필요한 경우 문의해 주세요.

**질문**: 모델을 다운로드하여 로컬에서 실행할 수 있나요?

**대답**: 모델을 다운로드하여 로컬에서 실행할 수는 없습니다.

**질문**: 요청은 기록되나요?

**대답**: 배포 작성 중에 추적을 해제하도록 선택할 수 있습니다. 그러면 추적 해제 시점부터는 오디오나 전사가 기록되지 않습니다. 그렇지 않으면 일반적으로 Azure에서 보안 저장소에 요청이 기록됩니다. 그 밖의 개인 정보 보호 문제로 인해 Custom Speech Service를 사용할 수 없는 경우에는 문의해 주세요.

## <a name="importing-data"></a>데이터 가져오기

**질문**: 데이터 세트의 크기 한도는 얼마인가요? 그 이유는 

**대답**: 현재는 HTTP 업로드용 파일 크기 제한으로 인해 데이터 세트 크기가 2GB로 제한됩니다. 

**질문**: 더 큰 텍스트 파일을 업로드하기 위해 텍스트 파일을 압축할 수 있나요? 

**대답**: 아니요, 현재는 압축되지 않은 텍스트 파일만 허용됩니다.

**질문**: 데이터 보고서에 발화 실패가 기록되어 있습니다. 문제가 되나요?

**대답**: 정상적으로 가져오지 못한 발화가 몇 개뿐이라면 문제가 되지 않습니다.
음향 데이터 집합이나 언어 데이터 집합에서 대부분의 음성을(예: >95%) 가져온 경우에는 데이터 집합을 사용할 수 있습니다. 단, 음성이 실패한 이유를 이해하고 문제 해결을 시도하는 것이 좋습니다.
가장 일반적인 문제(예: 서식 오류)는 쉽게 해결할 수 있습니다. 

## <a name="creating-am"></a>AM 만들기

**질문**: 음향 데이터는 어느 정도나 필요한가요?

**대답**: 처음에는 30분에서 1시간 분량의 음향 데이터를 준비하는 것이 좋습니다.

**질문**: 어떤 종류의 데이터를 수집해야 하나요?

**대답**: 애플리케이션 시나리오 및 사용 사례와 최대한 유사한 데이터를 수집해야 합니다.
즉, 데이터 컬렉션은 장치, 환경 및 화자 유형과 관련하여 대상 애플리케이션 및 사용자와 일치해야 합니다. 일반적으로 최대한 광범위한 화자의 데이터를 수집해야 합니다. 

**질문**: 데이터는 어떻게 수집해야 하나요? 

**대답**: 독립 실행형 데이터 수집 애플리케이션을 만들거나 기존 오디오 녹음 소프트웨어를 사용하면 됩니다.
오디오 데이터를 기록하고 사용하는 애플리케이션 버전을 만들 수도 있습니다. 

**질문**: 적응 데이터를 직접 전사해야 하나요? 

**대답**: 예, 데이터를 전사해야 합니다. 직접 전사하거나 전문적인 전사 서비스를 사용할 수 있습니다. 전문 전사기를 사용하는 경우도 있고 크라우드소싱을 사용하는 경우도 있습니다. 요청에 따라 전사 서비스를 추천할 수도 있습니다.

**질문**: 사용자 지정 음향 모델을 만들려면 시간이 얼마나 걸리나요?

**대답**: 사용자 지정 음향 모델을 만들기 위한 처리 시간은 음향 데이터 세트의 길이와 비슷합니다.
따라서 5시간 길이의 데이터 집합에서 생성된 사용자 지정 음향 모델을 처리하려면 5시간 정도가 소요됩니다. 

## <a name="offline-testing"></a>오프라인 테스트

**질문**: 사용자 지정 언어 모델을 사용하여 사용자 지정 음향 모델의 오프라인 테스트를 수행할 수 있나요?

**대답**: 예, 오프라인 테스트를 설정할 때 드롭다운에서 사용자 지정 언어 모델을 선택하면 됩니다.

**질문**: 사용자 지정 음향 모델을 사용하여 사용자 지정 언어 모델의 오프라인 테스트를 수행할 수 있나요?

**대답**: 예, 오프라인 테스트를 설정할 때 드롭다운 메뉴에서 사용자 지정 음향 모델을 선택하면 됩니다.

**질문**: 단어 오류 비율이란 무엇이며 어떻게 계산되나요?

**대답**: 단어 오류 비율은 음성 인식용 평가 메트릭입니다. 삽입, 삭제 및 대체를 포함하는 총 오류 수를 참조 전사에 포함된 총 단어 수로 나누어 계산됩니다.

**질문**: 사용자 지정 모델의 테스트 결과를 확인했는데, 해당 수치가 양호한 수준인지는 어떻게 알 수 있나요?

**대답**: 결과에는 기준 모델과 사용자 지정 모델을 비교한 내용이 표시됩니다.
사용자 지정이 빛을 발하려면 기준 모델보다 우수해야 합니다.

**질문**: 개선 여부 확인을 위해 베이스 모델의 WER을 확인하려면 어떻게 해야 하나요? 

**대답**: 오프라인 테스트 결과에는 기준 모델의 정확도와 사용자 지정 모델의 정확도, 그리고 기준 모델에 비해 사용자 지정 모델에서 개선된 부분이 표시됩니다.

## <a name="creating-lm"></a>LM 만들기

**질문**: 텍스트 데이터는 어느 정도나 업로드해야 하나요?

**대답**: 애플리케이션에서 사용되는 어휘/구문과 시작 언어 모델의 어휘/구문 간 차이 정도에 따라 업로드해야 하는 텍스트 데이터의 양도 달라지므로, 정확한 답변을 제공해 드리기는 어렵습니다. 모든 새 단어에 대해 예제를 가능한 한 많이 제공하는 것이 유용합니다. 애플리케이션에서 사용되는 일반적인 문구의 경우 언어 데이터의 문구도 포함하면 유용합니다. 이러한 단어도 경청하도록 시스템에 지시하기 때문입니다.
언어 데이터 집합에 적어도 100개, 일반적으로 수백 개 또는 그 이상의 음성이 있는 것이 일반적입니다.
또한 특정 유형*의 쿼리가 다른 쿼리 유형보다 일반적일 것으로 예상되는 경우 데이터 집합에 일반적인 쿼리의 복사본을 여러 개 삽입할 수 있습니다.

**질문**: 단어 목록만 업로드할 수 있나요?

**대답**: 단어 목록을 업로드하면 단어가 어휘에 포함되기는 하지만 해당 단어가 일반적으로 사용되는 방식을 시스템이 학습하지는 않습니다.
전체 또는 부분 음성(사용자가 말하려는 문장이나 문구)을 제공하면 언어 모델이 새 단어를 학습하고 이 단어가 어떻게 사용되는지 학습할 수 있습니다. 사용자 지정 언어 모델은 시스템에 새 단어를 포함시키는 것뿐만 아니라 애플리케이션에서 알려진 단어가 나타날 가능성을 조정하는 데에도 적합합니다. 전체 음성을 제공하면 시스템 학습 성능이 좋아집니다.
