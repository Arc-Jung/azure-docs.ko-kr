# <a name="glossary"></a>용어

[!INCLUDE [Deprecation note](../../../includes/cognitive-services-custom-speech-deprecation-note.md)]

## <a name="a"></a>A

### <a name="acoustic-model"></a>음향 모델

음향 모델은 지정된 언어로 된 오디오의 짧은 조각을 많은 음소 또는 소리 단위의 하나로 레이블을 지정하는 분류자입니다. 예를 들어 "speech"라는 단어는 4개의 음소 "s p iy ch"로 구성됩니다. 이러한 분류는 대략 초당 100번 만들어집니다.

## <a name="b"></a>b

## <a name="c"></a>C

### <a name="conversational-model"></a>대화형 모델

대화 스타일로 통용되는 음성을 인식하는 데 적합한 모델입니다. Microsoft Conversational AM은 일반적으로 다른 사람을 겨냥하는 음성에 맞게 조정됩니다.

## <a name="d"></a>D

### <a name="deployment"></a>배포

조정된 사용자 지정 모델이 서비스가 되어서 URI를 노출하는 프로세스

## <a name="i"></a>I

### <a name="inverse-text-normalization"></a>역 텍스트 정규화

"원시" 서식이 지정되지 않은 텍스트를 서식이 지정된 텍스트로 다시 변환하는 프로세스(즉, 대/소문자 및 문장 부호가 있는)를 ITN(역 텍스트 정규화)이라고 합니다.

## <a name="j"></a>J

## <a name="k"></a>K

## <a name="l"></a>L

### <a name="language-model"></a>언어 모델

언어 모델은 단어의 시퀀스에 대한 확률 분포입니다. 언어 모델은 시스템이 단어 시퀀스 자체의 발생 가능성에 따라 비슷하게 들리는 단어의 시퀀스 중에서 결정하는 데 도움을 줍니다.

## <a name="m"></a>M

## <a name="n"></a>N

### <a name="normalization"></a>정규화

정규화(텍스트): 전사 등의 결과 텍스트를 시스템에서 읽을 수 있는 모호하지 않은 표준 형식으로 변환하는 과정입니다.

## <a name="s"></a>S

### <a name="search-and-dictate-model"></a>검색 및 받아쓰기 모델

명령 처리에 적합한 음향 모델. Microsoft 검색 및 받아쓰기 AM은 명령처럼 애플리케이션 또는 장치에서 지시하는 음성에 적합합니다.

### <a name="subscription-key"></a>구독 키

구독 키는 Custom Speech Service 모델을 호출하기 위해 쿼리 문자열 매개 변수로 지정해야 하는 문자열입니다. 구독 키는 [Azure Portal](https://portal.azure.com/#create/Microsoft.CognitiveServices/apitype/CustomSpeech)에서 가져오며 한 번 가져온 후에는 Custom Speech Service 포털의 "내 구독"에서 찾을 수 있습니다.

전사: 오디오 .wav 파일을 처리하면 생성되는 텍스트입니다.
