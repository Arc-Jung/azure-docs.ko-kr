# <a name="azure-cognitive-services-support-and-feedback-options"></a>Azure 코그너티브 서비스 지원 및 피드백 옵션

Azure Cognitive 서비스의 기능을 살펴보기 시작했습니다. 응용 프로그램에서 새 기능을 구현하고 있을 수 있습니다. 또는 서비스를 사용한 후 서비스를 개선하는 방법에 대한 제안이 있습니까? 다음은 지원을 받고, 최신 상태를 유지하고, 피드백을 제공하고, 코그너티브 서비스에 대한 버그를 보고할 수 있는 옵션입니다.

## <a name="get-support"></a>지원 받기

### <a name="create-an-azure-support-request"></a>Azure 지원 요청 만들기

<div class='icon is-large'>
    <img alt='Azure support' src='https://docs.microsoft.com/media/logos/logo_azure.svg'>
</div>

지금 클라우드를 시작하려는 개발자든 비즈니스에 중요한 전략적 애플리케이션을 배포하려는 대규모 조직이든 관계없이 가장 적합한 [Azure 지원 옵션 및 플랜 선택](https://azure.microsoft.com/support/plans)의 범위를 탐색합니다. Azure 고객은 Azure Portal에서 지원 요청을 만들고 관리할 수 있습니다.

* [Azure Portal](https://ms.portal.azure.com/#blade/Microsoft_Azure_Support/HelpAndSupportBlade/overview)
* [미국 정부의 Azure Portal](https://portal.azure.us)

### <a name="search"></a>검색

<div class='icon is-large'>
    <img alt='Search' src='https://docs.microsoft.com/media/common/i_search.svg'>
</div>

더 빠른 결과를 얻으려면 [스택 오버플로,](https://stackoverflow.com/)Microsoft [문서](https://docs.microsoft.com/azure/cognitive-services/)또는 [GitHub 코드 샘플과](https://github.com/azure-samples)같은 관련 사이트에 대한 검색을 수행합니다. 예를 `site:` 들어 [즐겨 찾는 검색 엔진에서](https://bing.com)쿼리 표기와 같은 사용 어표식을 사용합니다.

```
{search keywords} site:stackoverflow.com
```

검색의 컨텍스트는 어디에 `{search keywords}` 있습니까? 다음과 같은 범위의 검색을 적절하게 사용하는 것이 좋습니다.

 - 스택 오버플로우:`site:stackoverflow.com`
 - 마이크로 소프트 문서 :`site:docs.microsoft.com`
 - GitHub 샘플:`site:github.com/azure-samples`

### <a name="post-a-question-on-stack-overflow"></a>Stack Overflow에 질문 게시

<div class='icon is-large'>
    <img alt='Stack Overflow' src='https://docs.microsoft.com/media/logos/logo_stackoverflow.svg'>
</div>

스택 오버플로의 검색 상자에서 문제에 대한 답을 찾을 수 없는 경우 새 질문을 제출합니다. Stack Overflow는 개발 관련 질문에 대한 기본 설정 채널입니다. 스택 오버플로 커뮤니티의 구성원과 Microsoft 팀 구성원이 직접 문제를 해결하는 데 관여하는 곳입니다.

 질문을 작성할 때 다음 태그 중 하나를 사용합니다.

 - [마이크로소프트 코그너티브](https://stackoverflow.com/questions/tagged/microsoft-cognitive)
 - [Azure Cognitive Services](https://stackoverflow.com/questions/tagged/azure-cognitive-services)

> [!TIP]
> Stack Overflow의 다음 게시물에는 질문을 작성하고 소스 코드를 추가하는 방법에 대한 팁이 포함되어 있습니다. 다음 지침을 따르면 커뮤니티 구성원이 질문을 신속하게 평가하고 답변할 수 있는 기회를 늘릴 수 있습니다.
> * [좋은 질문을 하는 방법](https://stackoverflow.com/help/how-to-ask)
> * [재현 가능한 최소한의 예제를 만드는 방법은 무엇입니까?](https://stackoverflow.com/help/minimal-reproducible-example)

<div class='icon is-large'>
    <img alt='GitHub' src='https://docs.microsoft.com/media/logos/logo_octokitty.svg'>
</div>

다음은 개발자가 문제를 게시하거나 끌어오기 요청을 만들 수 있는 세 가지 유형의 리포지토리입니다. 문제를 만들려면 각 GitHub 리포지토리의 *문제* 탭을 사용하고 *새 문제를*선택합니다. 수정을 제안하려면 파일을 직접 편집하여 끌어오기 요청을 제출합니다. 그러면 끌어오기 요청은 리포지토리 소유자가 이를 볼 수 있을 때까지 *끌어오기 요청* 탭 아래에 있습니다.
