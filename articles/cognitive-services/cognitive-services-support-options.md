# <a name="azure-cognitive-services-support-and-feedback-options"></a>Azure Cognitive Services 지원 및 사용자 의견 옵션

Azure Cognitive Services의 기능을 탐색 하는 것을 시작 하 고 있나요? 응용 프로그램에서 새로운 기능을 구현 하는 경우를 가정해 볼 수 있습니다. 또는 서비스를 사용한 후 개선 하는 방법에 대 한 제안이 있나요? 다음은 지원을 받고, 최신 상태를 유지 하 고, 피드백을 제공 하 고, Cognitive Services에 대 한 버그를 보고할 수 있는 옵션입니다.

## <a name="get-support"></a>지원 받기

### <a name="create-an-azure-support-request"></a>Azure 지원 요청 만들기

<div class='icon is-large'>
    <img alt='Azure support' src='https://docs.microsoft.com/media/logos/logo_azure.svg'>
</div>

지금 클라우드를 시작하려는 개발자든 비즈니스에 중요한 전략적 애플리케이션을 배포하려는 대규모 조직이든 관계없이 가장 적합한 [Azure 지원 옵션 및 플랜 선택](https://azure.microsoft.com/support/plans)의 범위를 탐색합니다. Azure 고객은 Azure Portal에서 지원 요청을 만들고 관리할 수 있습니다.

* [Azure Portal](https://ms.portal.azure.com/#blade/Microsoft_Azure_Support/HelpAndSupportBlade/overview)
* [미국 정부의 Azure Portal](https://portal.azure.us)

### <a name="search"></a>검색

<div class='icon is-large'>
    <img alt='Search' src='https://docs.microsoft.com/media/common/i_search.svg'>
</div>

더 빠른 결과를 위해 [Stack Overflow](https://stackoverflow.com/), [Microsoft docs](https://docs.microsoft.com/azure/cognitive-services/)또는 [GitHub 코드 샘플과](https://github.com/azure-samples)같은 관련 사이트에 대 한 검색을 수행 합니다. 자주 사용 하는 `site:` [검색 엔진](https://bing.com)에서 쿼리 표기법을 사용 합니다. 예를 들면 다음과 같습니다.

```
{search keywords} site:stackoverflow.com
```

여기서 `{search keywords}` 는 검색의 컨텍스트입니다. 이러한 범위 지정 검색을 적절 하 게 사용 하는 것이 좋습니다.

 - Stack Overflow: `site:stackoverflow.com`
 - Microsoft Docs: `site:docs.microsoft.com`
 - GitHub 샘플: `site:github.com/azure-samples`

### <a name="post-a-question-on-stack-overflow"></a>Stack Overflow에 질문 게시

<div class='icon is-large'>
    <img alt='Stack Overflow' src='https://docs.microsoft.com/media/logos/logo_stackoverflow.svg'>
</div>

Stack Overflow의 검색 상자에서 문제에 대 한 답변을 찾을 수 없는 경우 새 질문을 제출 합니다. Stack Overflow는 개발 관련 질문에 대한 기본 설정 채널입니다. Stack Overflow 커뮤니티 및 Microsoft 팀 구성원의 구성원이 문제를 해결 하는 데 직접 관여 하는 위치입니다.

[여기](https://go.microsoft.com/fwlink/?linkid=2126464)에 질문을 게시 합니다. 질문에 적절 한 태그를 사용 하 여 질문을 확인 합니다.

> [!TIP]
> Stack Overflow의 다음 게시물에는 질문을 작성하고 소스 코드를 추가하는 방법에 대한 팁이 포함되어 있습니다. 다음 지침을 따르면 커뮤니티 구성원이 질문을 신속하게 평가하고 답변할 수 있는 기회를 늘릴 수 있습니다.
> * [좋은 질문을 하는 방법](https://stackoverflow.com/help/how-to-ask)
> * [재현 가능한 최소 예제를 만드는 방법](https://stackoverflow.com/help/minimal-reproducible-example)

## <a name="report-bugs"></a>버그 보고

### <a name="create-a-github-issue-or-pull-request"></a>GitHub 문제 또는 끌어오기 요청 만들기

<div class='icon is-large'>
    <img alt='GitHub' src='https://docs.microsoft.com/media/logos/logo_octokitty.svg'>
</div>

다음은 개발자가 문제를 게시 하거나 끌어오기 요청을 만들 수 있는 세 가지 유형의 리포지토리입니다. 문제를 만들려면 해당 GitHub 리포지토리의 *문제* 탭을 사용 하 고 *새 문제*를 선택 합니다. 수정을 제안 하려면 파일을 직접 편집 하 여 끌어오기 요청을 제출 합니다. 그런 다음 끌어오기 요청은 리포지토리 소유자가 볼 수 있을 때까지 *끌어오기 요청* 탭에 그대로 둡니다.
