이 빠른 시작에서는 Anomaly Detector 서비스 및 cURL을 사용하여 시계열 데이터의 일괄 처리에 대한 변칙을 검색하는 방법을 알아봅니다.

Anomaly Detector 개념에 대한 간략한 설명은 [개요](../../overview.md) 문서를 참조하세요.

## <a name="prerequisites"></a>필수 구성 요소

- Azure 구독 - [체험 구독 만들기](https://azure.microsoft.com/free/cognitive-services)
- Azure 구독을 보유한 후에는 Azure Portal에서 <a href="https://ms.portal.azure.com/#create/Microsoft.CognitiveServicesAnomalyDetector"  title="Anomaly Detector 리소스 만들기"  target="_blank">Anomaly Detector 리소스 </a>를 만들어 키와 엔드포인트를 가져옵니다. 배포될 때까지 기다렸다가 **리소스로 이동** 단추를 선택합니다.
    - REST API를 사용하려면 사용자가 만든 리소스의 키 및 엔드포인트 주소가 필요합니다. 
    평가판 가격 책정 계층(`F0`)을 통해 서비스를 사용해보고, 나중에 프로덕션용 유료 계층으로 업그레이드할 수 있습니다.

## <a name="detect-anomalies-for-an-entire-series"></a>전체 시리즈에 대한 변칙 검색

명령 프롬프트에서 다음 명령을 실행합니다. 명령에 다음 값을 삽입해야 합니다.
- 변칙 탐지기 서비스 구독 키입니다.
- 변칙 탐지기 엔드포인트 주소입니다. 
- 비정상 상태를 테스트할 시계열 데이터의 유효한 JSON 파일입니다. 사용자 고유의 파일이 없는 경우 [요청 본문 샘플](https://westus2.dev.cognitive.microsoft.com/docs/services/AnomalyDetector/operations/post-timeseries-entire-detect)에서 sample.json 파일을 만들 수 있습니다.

```curl
curl -v -X POST "https://{endpointresourcename.cognitive.microsoft.com}/anomalydetector/v1.0/timeseries/entire/detect"
-H "Content-Type: application/json"
-H "Ocp-Apim-Subscription-Key: {subscription key}"
-d "@{path_to_file.json}" 
```

모든 값이 채워진 예는 다음과 같습니다.

```curl
curl -v -X POST "https://my-resource-name.cognitiveservices.azure.com/anomalydetector/v1.0/timeseries/entire/detect" -H  "Content-Type: application/json" -H "Ocp-Apim-Subscription-Key:1111112222222ed333333ab333333333" -d "@test.json"
```
