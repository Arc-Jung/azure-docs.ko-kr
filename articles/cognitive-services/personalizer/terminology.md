# <a name="terminology"></a>용어

개인화는 강화 학습에서 용어를 사용합니다. 이러한 용어는 Azure 포털 및 API에서 사용됩니다.

## <a name="conceptual-terminology"></a>개념 용어

* **학습 루프**: 개인 설정의 이점을 누릴 수 있는 응용 프로그램의 모든 부분에 대해 _학습 루프라고_하는 Personalr 리소스를 만듭니다. 개인에 맞게 설정하려는 환경이 둘 이상인 경우 각각에 대한 반복을 만듭니다.

* **모델**: Personalizer 모델은 사용자 행동에 대해 학습된 모든 데이터를 캡처하고, 순위 및 보상 호출에 보내는 인수의 조합에서 교육 데이터를 얻고, 학습 정책에 의해 결정된 교육 동작을 캡처합니다.

## <a name="personalizer-configuration"></a>개인 설정

개인 설정은 Azure [포털에서](https://portal.azure.com)구성됩니다.

* **보상**: 보상 대기 시간, 기본 보상 및 보상 집계 정책에 대한 기본 값을 구성합니다.

* **탐색**: 탐색에 사용할 순위 호출의 백분율을 구성합니다.

* **모델 업데이트 빈도**: 모델을 다시 학습하는 빈도입니다.

* **데이터 보존**: 저장할 데이터의 일 수 이는 학습 루프를 개선하는 데 사용되는 오프라인 평가에 영향을 미칠 수 있습니다.

## <a name="use-rank-and-reward-apis"></a>랭크 및 리워드 API 사용

* **순위**: 기능 및 컨텍스트 기능이 있는 작업을 감안할 때 탐색 또는 악용을 사용하여 상위 액션(콘텐츠 항목)을 반환합니다.

    * **조치**: 액션은 제품 이나 프로모션 과 같은 콘텐츠 항목에서 선택할 수 있습니다. 개인 설정은 순위 API를 통해 사용자에게 표시할 최상위 액션(반환된 보상 액션 ID)을 선택합니다.

    * **컨텍스트**: 보다 정확한 순위를 제공하려면 컨텍스트에 대한 정보를 제공합니다.
        * 사용자
        * 사용하는 디바이스
        * 현재 시간입니다.
        * 현재 상황에 대한 다른 데이터
        * 사용자 또는 컨텍스트에 대한 기록 데이터

        특정 애플리케이션에 다른 컨텍스트 정보가 있을 수 있습니다.

    * **[특징](concepts-features.md)**: 콘텐츠 항목 또는 사용자 컨텍스트에 대한 정보 단위입니다. 집계된 기능만 사용해야 합니다. 특정 시간, 사용자 아이디 또는 기타 집계되지 않은 데이터를 기능으로 사용하지 마십시오.

        * _작업 기능은_ 콘텐츠에 대한 메타데이터입니다.
        * _컨텍스트 기능은_ 콘텐츠가 표시되는 컨텍스트에 대한 메타데이터입니다.

* **탐색**: Personalizer 서비스는 최상의 작업을 반환하는 대신 사용자에 대해 다른 작업을 선택하는 시기를 탐색합니다. Personalizer 서비스는 드리프트와 정체를 방지하고 검색을 통해 진행 중인 사용자 동작에 적응할 수 있습니다.

* **악용**: Personalizer 서비스는 현재 모델을 사용하여 과거 데이터를 기반으로 최상의 작업을 결정합니다.

* **실험 기간:** 개인설정 서비스가 해당 이벤트에 대한 랭크 호출이 발생한 순간부터 보상을 기다리는 시간입니다.

* **비활성 이벤트**: 비활성 이벤트는 Rank라고 하는 이벤트이지만 클라이언트 응용 프로그램 결정으로 인해 사용자가 결과를 볼 수 있는지 확실하지 않습니다. 비활성 이벤트를 사용하면 맞춤 설정 결과를 만들고 저장한 다음, 나중에 기계 학습 모델에 영향을 주지 않고 삭제하도록 결정할 수 있습니다.


* **보상**: 사용자가 랭크 API의 반환 된 보상 액션 ID에 0에서 1 사이의 점수로 어떻게 반응했는지 측정합니다. 0~1 값은 선택이 맞춤 설정의 비즈니스 목표를 달성하는 데 도움이 되는 정도에 기반한 비즈니스 논리에 따라 설정됩니다. 학습 루프는 이 보상을 개별 사용자 기록으로 저장하지 않습니다.

## <a name="offline-evaluations"></a>오프라인 평가

* **평가**: 오프라인 평가는 루프의 데이터를 기반으로 루프에 가장 적합한 학습 정책을 결정합니다.

* **학습 정책**: Personalizer가 모든 이벤트에서 모델을 학습하는 방법은 기계 학습 알고리즘의 작동 방식에 영향을 주는 일부 매개 변수에 의해 결정됩니다. 새 학습 루프는 중간 정도의 성능을 얻을 수 있는 기본 **학습 정책으로**시작됩니다. [평가를](concepts-offline-evaluation.md)실행할 때 , Personalizer는 루프의 사용 사례에 특별히 최적화된 새 학습 정책을 만듭니다. 개인 설정은 평가 중에 생성된 각 특정 루프에 최적화된 정책을 통해 훨씬 더 나은 성능을 발휘합니다. 학습 정책의 _이름은_ Azure 포털의 개인 설정에 대한 모델 및 학습 설정의 학습 **설정입니다.**
