# SQL 데이터베이스 탄력적 풀(Preview) 만들기 및 관리

> [AZURE.SELECTOR]
- [Azure portal](sql-database-elastic-pool-portal.md)
- [PowerShell](sql-database-elastic-pool-powershell.md)

이 문서는 [Azure 포털](https://portal.azure.com)에서 탄력적 풀을 만드는 방법을 보여줍니다.

탄력적 풀을 통해 많은 수의 데이터베이스를 만들고, 유지 관리하며, 성능과 비용을 관리하는 작업을 간소화할 수 있습니다.
 

> [AZURE.NOTE]탄력적 풀은 현재 Preview이며, SQL Database V12 서버와만 사용할 수 있습니다.

 


## 필수 조건

탄력적 풀을 만들려면 다음 사항이 필요합니다.

- Azure 구독! Azure 구독이 필요할 경우 이 페이지 위쪽에서 무료 평가판을 클릭하고 되돌아와 이 문서를 완료합니다.
- Azure SQL 데이터베이스 버전 12 이상 V12 서버가 없을 경우 다음 절차에 따라 만들 수 있습니다. [첫 Azure SQL 데이터베이스 만들기](sql-database-get-started.md).



## 탄력적 풀 만들기

서버에 새 풀을 추가하여 탄력적 풀을 만듭니다. 서버에 여러 풀을 추가할 수 있지만, 각 풀마다 하나의 서버만 연결할 수 있습니다. 또한 서버에 있는 데이터베이스 전체 또는 일부를 풀에 추가할 수 있습니다.


1.	풀에 추가하려는 데이터베이스가 포함된 SQL 데이터베이스 V12 서버를 선택합니다.
2.	**SQL Server** 블레이드 위쪽에서 **풀 추가**를 선택하여 풀을 만듭니다.

   ![탄력적 풀 만들기][1]

## 탄력적 풀 구성

가격 책정 계층을 설정하고, 데이터베이스를 추가하며, 풀의 성능 특성을 구성하여 풀을 구성합니다.

* **풀 추가** 명령을 선택할 때 **PREVIEW 약관**을 선택하고 **Preview 약관** 블레이드를 완료하여 Preview 약관에 동의해야 합니다. 각 구독에 대해 한 번만 약관에 동의하면 됩니다.*

   ![탄력적 풀 구성][2]


### 가격 책정 계층

탄력적 풀의 가격 책정 계층은 SQL 데이터베이스의 서비스 계층과 다소 유사합니다. 가격 책정 계층에 따라 풀에 있는 탄력적 데이터베이스에서 사용 가능한 기능과 각 데이터베이스에 사용 가능한 최대 DTU 수(DTU MAX) 및 저장소(GB)가 결정됩니다.

> [AZURE.NOTE]Preview 는 현재 **표준** 가격 책정 계층으로 한정되어 있습니다.

| 가격 책정 계층 | 데이터베이스당 DTU MAX |
| :--- | :--- |
| 표준 | 데이터베이스당 100 MAX DTU |

### 데이터베이스 추가

언제든지 풀에 포함시킬 특정 데이터베이스를 선택할 수 있습니다. 새 풀을 만들 때, Azure는 풀에 포함될 경우 유리한 데이터베이스를 권장하며 포함하도록 표시합니다. 서버에서 사용할 수 있는 모든 데이터베이스를 추가하거나 또는 필요할 경우 초기 목록에서 데이터베이스를 지울 수 있습니다.

   ![데이터베이스 추가][5]

풀에 추가할 데이터베이스를 선택하려면 다음 조건이 충족되어야 합니다.

- 풀에 데이터베이스를 추가할 공간이 있어야 합니다(데이터베이스의 최대 수가 이미 포함된 경우 추가 불가능). 더 자세히 말하자면, 풀에 데이터베이스당 DTU 보장을 지키기에 충분한 DTU가 있어야 합니다(예를 들어, 그룹의 DTU 보장이 400이고 각 데이터베이스의 DTU 보장이 10이라면, 풀에 허용되는 데이터베이스의 최대 수는 40(400 DTU/데이터베이스당 보장되는 10 DTU = 최대 데이터베이스 수 40)).
- 데이터베이스에서 사용하는 현재 기능을 풀에서 사용할 수 있어야 합니다. 


### 성능 구성

풀과 풀에 있는 탄력적 데이터베이스 모두의 성능 매개 변수를 설정하여 풀의 성능을 구성합니다. **탄력적 데이터베이스 설정**은 풀에 있는 모든 데이터베이스에 적용된다는 점에 유의하세요.

   ![탄력적 풀 구성][3]

풀의 성능을 정의하는 매개 변수는 세 개이며 사용자가 설정할 수 있고, 풀의 DTU 보장, 풀에 있는 탄력적 데이터베이스의 DTU MIN 및 DTU MAX입니다. 다음 표에 각각이 설명되어 있으며 설정 방법에 대한 몇 가지 지침이 있습니다.

| 성능 매개 변수 | 설명 |
| :--- | :--- |
| **POOL DTU/GB** - 풀의 DTU 보장 | 풀의 DTU 보장은 풀에 있는 모든 데이터베이스에서 사용 가능하며 공유되는 DTU의 보장 수치입니다. <br> 현재 이 항목은 200, 400, 800 또는 1200으로 설정할 수 있습니다. <br> 그룹에 대한 DTU 보장의 특정 크기는 해당 그룹의 과거 DTU 사용률을 고려하여 프로비저닝해야 합니다. 또는 원하는 데이터베이스당 DTU 보장과 동시 활성 데이터베이스의 사용률을 기준으로 이 크기를 설정할 수도 있습니다. 풀에 대한 DTU 보장은 또한 풀에 사용할 수 있는 저장소의 양과도 관련이 있으며, 풀에 할당하는 DTU 하나마다 1GB의 데이터베이스 저장소가 소비됩니다(1 DTU = 저장소 1G). <br> **풀의 DTU 보장을 어떻게 설정해야 합니까?** <br>풀의 DTU 보장은 최소한 ([데이터베이스 수] x [데이터베이스당 평균 DTU 사용률]) 이상으로 설정해야 합니다. |
| **DTU MIN** - 각 데이터베이스당 DTU 보장 | 데이터베이스당 DTU 보장은 풀에 있는 단일 데이터베이스에 보장되는 DTU 수입니다. 현재, 이 보장 항목은 0, 10, 20 또는 50 DTU로 설정하거나, 그룹에 있는 데이터베이스에 대해 보장을 제공하지 않도록 선택할 수 있습니다(DTU MIN=0). <br> **데이터베이스당 DTU 보장을 어떻게 설정해야 합니까?** <br>데이터베이스당 DTU 보장(DTU MIN)은 최소한 ([데이터베이스당 평균 사용률]) 이상으로 설정해야 합니다. 데이터베이스당 DTU 보장은 풀에 있는 모든 데이터베이스에 대한 DTU 보장을 설정하는 전역 설정입니다. |
| **DTU MAX** - 데이터베이스당 DTU 최대값 | 데이터베이스당 DTU MAX는 풀에 있는 단일 데이터베이스가 사용할 수 있는 최대 DTU 수입니다. 데이터베이스당 DTU 최대값은 데이터베이스에서 발생할 수 있는 버스트 또는 스파이크를 처리하기에 충분히 높게 설정해야 합니다. 이 최대값은 시스템 한도까지 설정할 수 있으며, 이는 풀의 가격 책정 계층(표준의 경우 100 DTU)에 따라 달라집니다. 이 최대값의 특정 크기가 그룹 내 데이터베이스의 최고 사용률 패턴을 수용할 수 있어야 합니다. 풀은 일반적으로 모든 데이터베이스가 동시에 최대로 사용되지 않을 경우 데이터베이스에 대해 핫 및 콜드 사용률 패턴을 가정하므로 그룹에 대한 일정 수준의 오버커밋이 예상됩니다.<br> **데이터베이스당 DTU 최대값을 얼마로 설정해야 합니까?** <br> 데이터베이스당 DTU MAX 또는 DTU 최대값을 ([데이터베이스 최고 사용률])로 설정합니다. 예를 들어, 데이터베이스당 최고 사용률이 50 DTU이며 그룹 내 100개의 데이터베이스 중 20%만이 동시에 최대로 사용되는 것으로 가정합니다. 데이터베이스당 DTU 최대값이 50 DTU로 설정된 경우, 그룹을 5배로 오버커밋하고 그룹의 DTU 보장을 1,000 DTU로 설정하는 것이 합당할 것입니다. 또한 DTU 최대값은 데이터베이스에 대한 리소스 보장이 아니며, 사용 가능할 때 도달될 수 있는 DTU 최고 한도입니다. |


## 탄력적 풀에 데이터베이스 추가

풀을 만든 후에 풀에 데이터베이스를 추가하거나 풀에서 데이터베이스를 제거할 수 있습니다.


## 탄력적 풀 모니터링 및 관리

탄력적 풀을 만든 후에 기존 풀의 목록을 검색하고 원하는 풀을 선택하여 포털 내의 풀을 모니터링하고 관리할 수 있습니다.

풀을 만든 후에 다음과 같은 작업을 수행할 수 있습니다.

- **풀 구성**을 선택하여 풀 DTU 및 데이터베이스당 DTU 설정을 변경합니다.
- **작업 만들기**를 선택하고 탄력적 작업을 만들어 풀에 있는 데이터베이스를 관리합니다. 탄력적 작업을 통해 개수에 관계없이 풀에 있는 데이터베이스에 대해 T-SQL 스크립트를 실행할 수 있습니다. 자세한 내용은 [탄력적 데이터베이스 작업 개요](sql-database-elastic-jobs-overview.md)를 참조하세요.
- **작업 관리**를 선택하여 기존 탄력적 작업을 관리합니다.



![탄력적 풀 모니터링][8]




![탄력적 풀 모니터링][4]

기존 풀을 선택하면 해당 풀의 리소스 사용률을 볼 수 있습니다. **리소스 사용률** 차트를 클릭하면 차트 및 설정 경고를 사용자 지정할 수 있는 **메트릭** 블레이드가 열립니다.


![리소스 사용률][6]

**차트 편집**을 클릭하여 풀에 대한 원격 분석 데이터를 손쉽게 볼 수 있도록 매개 변수를 추가합니다.


![차트 편집][7]
