# Azure 포털에서 SQL 데이터베이스에 확장성 있는 탄력적 데이터베이스 풀 만들기

> [AZURE.SELECTOR]
- [Azure portal](sql-database-elastic-pool-portal.md)
- [C#](sql-database-elastic-pool-csharp.md)
- [PowerShell](sql-database-elastic-pool-powershell.md)

이 문서에서는 Azure 포털을 사용하여 확장성 있는 [탄력적 데이터베이스 풀](sql-database-elastic-pool.md)을 만드는 방법을 보여줍니다. 탄력적 데이터베이스 풀이 있는 SQL 데이터베이스는 여러 데이터베이스 간에 공유되는 관리 및 리소스를 단순화합니다.

> [AZURE.NOTE] 탄력적 데이터베이스 풀은 현재 미리 보기 상태이며, SQL 데이터베이스 V12 서버에서만 사용할 수 있습니다. SQL 데이터베이스 V11 서버가 있는 경우 한 단계에서 [PowerShell을 사용하여 V12로 업그레이드 및 풀 만들기](sql-database-upgrade-server-powershell.md)를 할 수 있습니다.


시작하기 전에 SQL 데이터베이스 V12 서버에 데이터베이스가 있어야 합니다. 데이터베이스가 없는 경우 [첫 Azure SQL 데이터베이스 만들기](sql-database-get-started.md)를 참조하여 5분 이내에 하나를 만드세요. 또는 SQL 데이터베이스 V11 서버가 이미 있는 경우 [포털에서 V12 업그레이드](sql-database-v12-upgrade.md)한 다음 다시 돌아와서 풀을 만들려면 다음이 지침에 따르세요.


## 1단계: 새 풀 만들기

서버에 새 풀을 추가하여 탄력적 데이터베이스 풀을 만듭니다. 서버에 풀을 여러 개 추가할 수 있지만 다른 서버에 속하는 데이터베이스를 동일한 풀에 추가할 수 없습니다.


1. [Azure 포털](http://portal.azure.com/)에서 **SQL Server**를 클릭합니다.
2. 풀에 추가하려는 데이터베이스가 포함된 서버를 선택합니다.
3. **새 풀**을 클릭합니다.(또는, 권장되는 풀이 있다는 메시지가 표시되는 경우 클릭하면 서버의 데이터베이스에 최적화된 풀을 간편하게 검토하고 만들 수 있습니다.) 권장 사항에 대한 자세한 내용은 아래에 제공됩니다.


    ![서버에 풀 추가](./media/sql-database-elastic-pool-portal/new-pool.png)


4. **탄력적 데이터베이스 풀** 블레이드에서 기본 이름을 그대로 두거나 새 풀의 이름을 입력합니다.

    ![탄력적 풀 구성](./media/sql-database-elastic-pool-portal/configure-elastic-pool.png)


### 권장되는 탄력적 데이터베이스 풀

SQL 데이터베이스 서버로 이동하면 서버에 대해 권장되는 탄력적 데이터베이스 풀이 있다는 메시지를 볼 수도 있습니다(V12에만 해당).

### 권장 사항이 표시되는 이유

SQL 데이터베이스 서비스는 사용 기록을 평가하고 단일 데이터베이스 사용보다 비용 효율적인 경우 하나 이상의 탄력적 데이터베이스 풀을 권장합니다.

![권장되는 풀](./media/sql-database-elastic-pool-portal/recommended-pool.png)

각 권장 사항은 풀에 가장 적합한 서버 데이터베이스의 고유 하위 집합으로 구성됩니다.

각각의 풀 권장 사항은 다음을 포함합니다.

- 풀에 대한 가격 책정 계층(Basic, Standard 또는 Premium)
- 적절한 풀 eDTU 양
- 탄력적 데이터베이스 최소/최대 eDTU 설정
- 권장 데이터베이스 목록

서비스는 탄력적 데이터베이스 풀을 권장할 때 최근 30일간의 원격 분석을 고려합니다. 데이터베이스가 탄력적 데이터베이스 풀의 후보로 간주되려면 7일 이상 존재해야 합니다. 이미 탄력적 데이터베이스 풀에 있는 데이터베이스는 탄력적 데이터베이스 풀 권장 사항에 대한 후보로 간주되지 않습니다.

서비스는 리소스 요구와 각 서비스 계층에 있는 단일 데이터베이스를 동일한 계층의 탄력적 데이터베이스 풀로 이동하는 비용 효율성을 평가합니다. 예를 들어 서버의 모든 Standard 데이터베이스는 표준 탄력적 풀에 적합한지 평가됩니다. 즉, 서비스는 Standard 데이터베이스를 Premium 풀로 이동하는 경우와 같은 계층 간 권장 사항을 제공하지 않습니다.


### 권장되는 풀 만들기

1. 권장되는 풀 목록을 보려면 메시지를 클릭합니다.
1. 자세한 권장 사항을 보려면 풀을 클릭합니다.
2. 권장 사항의 모든 설정을 검토하고 그대로 적용하거나 특정 비즈니스 요구 사항에 맞도록 권장 사항을 편집합니다.


## 2단계: 가격 책정 계층 선택

풀의 가격 책정 계층에 따라 풀에 있는 탄력적 데이터베이스에서 사용 가능한 기능과 각 데이터베이스에서 사용 가능한 최대 eDTU 수(eDTU MAX) 및 저장소(GB)가 결정됩니다. 자세한 내용은 [서비스 계층](sql-database-service-tiers.md#Service-tiers-for-elastic-database-pools)을 참조하세요.

>[AZURE.NOTE] 현재 미리 보기 상태에서는 탄력적 데이터베이스 풀의 가격 책정 계층을 만든 후 변경할 수 없습니다. 기존 탄력적 풀의 가격 책정 계층을 변경하려면 원하는 가격 책정 계층에서 탄력적 풀을 새로 만들고 탄력적 데이터베이스를 이 새로운 풀로 마이그레이션합니다.

   ![가격 책정 계층 선택](./media/sql-database-elastic-pool-portal/pricing-tier.png)

### 풀 구성

가격 책정 계층을 설정한 후에, 데이터베이스를 추가할 풀의 구성을 클릭하고, 풀 eDTU 및 저장소(풀 GB), 그리고 풀의 탄력적 데이터베이스에 대한 최대 및 최소 eDTU를 설정할 위치를 설정합니다.

## 3단계: 풀에 데이터베이스 추가

언제든지 풀에 데이터베이스를 추가하거나 제거할 수 있습니다.

1. 풀을 만드는 동안 **풀 구성** 블레이드의 **데이터베이스 추가**를 클릭합니다.
2. 풀에 추가하려는 데이터베이스를 선택합니다.

    ![데이터베이스 추가](./media/sql-database-elastic-pool-portal/add-databases.png)

    풀에 추가할 데이터베이스를 선택하려면 다음 조건이 충족되어야 합니다.

    - 풀에 데이터베이스를 추가할 공간이 있어야 합니다(데이터베이스의 최대 수가 이미 포함된 경우 추가 불가능). 더 자세히 말하자면, 풀에 데이터베이스당 eDTU 보장을 지키기에 충분한 eDTU가 있어야 합니다(예를 들어 그룹의 eDTU 보장이 400이고 각 데이터베이스의 eDTU 보장이 10이라면 풀에 허용되는 최대 데이터베이스 수는 40(400 eDTU/DB당 보장되는 10 eDTU = 최대 데이터베이스 수 40)임).
    - 데이터베이스에서 사용하는 현재 기능을 풀에서 사용할 수 있어야 합니다.

### 동적 권장 사항

풀에 데이터베이스를 추가한 후에는, 사용자가 선택한 데이터베이스의 기존 사용 정보를 기반으로 권장 사항이 동적으로 생성됩니다. 권장 사항은 eDTU 및 GB 사용 현황 차트는 물론 **풀 구성** 블레이드 상단의 권장 사항 배너에도 표시됩니다. 권장 사항은 특정 데이터베이스에 최적화된 풀을 만드는 데 도움을 주기 위해 제공됩니다.


 ![동적 권장 사항](./media/sql-database-elastic-pool-portal/dynamic-recommendation.png)


## 4단계: 풀의 성능 특성 설정

풀과 풀에 있는 탄력적 데이터베이스 모두의 성능 매개 변수를 설정하여 풀의 성능을 구성합니다. **탄력적 데이터베이스 설정**은 풀에 있는 모든 데이터베이스에 적용된다는 점에 유의하세요.

 ![탄력적 풀 구성](./media/sql-database-elastic-pool-portal/configure-performance.png)

풀의 성능을 정의하며 사용자가 설정할 수 있는 세 개의 매개 변수는 풀의 eDTU 보장, 풀에 있는 탄력적 데이터베이스의 eDTU MIN 및 eDTU MAX입니다. 다음 표에 각각이 설명되어 있으며 설정 방법에 대한 몇 가지 지침이 있습니다. 사용 가능한 특정 값 설정에 대해서는 [탄력적 데이터베이스 풀 참조](sql-database-elastic-pool-reference.md)를 참조하세요.

| 성능 매개 변수 | 설명 |
| :--- | :--- |
| **풀 eDTU** - 풀의 eDTU 보장 | 풀의 eDTU 보장은 풀에 있는 모든 데이터베이스에서 사용 가능하며 공유되는 eDTU의 보장 수치입니다. <br> 그룹의 eDTU 보장에 대한 특정 크기는 해당 그룹의 기록 eDTU 사용률을 고려하여 프로비전해야 합니다. 또는 원하는 데이터베이스당 eDTU 보장과 동시 활성 데이터베이스의 사용률을 기준으로 이 크기를 설정할 수 있습니다. 풀의 eDTU 보장은 풀에 사용할 수 있는 저장소 양과도 관련이 있으며, 풀에 할당하는 eDTU마다 고정된 양의 데이터베이스 저장소가 제공됩니다. <br> **풀의 eDTU 보장을 어떻게 설정해야 하나요?** <br>최소한 풀의 eDTU 보장을 ([데이터베이스 수] x [데이터베이스당 평균 DTU 사용률]) 이상으로 설정해야 합니다. |
| **eDTU MIN** - 각 데이터베이스의 eDTU 보장 | 데이터베이스당 eDTU 보장은 풀에 있는 단일 데이터베이스에 보장되는 eDTU 수입니다. <br> **데이터베이스당 eDTU 보장을 어떻게 설정해야 하나요?** <br>일반적으로 데이터베이스당 eDTU 보장(eDTU MIN)은 0에서 ([데이터베이스당 평균 사용률]) 사이의 값으로 설정됩니다. 데이터베이스당 eDTU 보장은 풀에 있는 모든 데이터베이스에 대한 eDTU 보장을 설정하는 전역 설정입니다. |
| **eDTU MAX** - 데이터베이스당 eDTU 최대값 | 데이터베이스당 eDTU MAX는 풀에 있는 단일 데이터베이스가 사용할 수 있는 최대 eDTU 수입니다. 데이터베이스당 eDTU 최대값은 데이터베이스에서 발생할 수 있는 버스트 또는 스파이크를 처리하기에 충분히 높게 설정해야 합니다. 이 최대값은 시스템 제한까지 설정할 수 있으며, 풀의 가격 책정 계층에 따라 달라집니다(Premium의 경우 1000 eDTU). 이 최대값의 특정 크기가 그룹 내 데이터베이스의 최고 사용률 패턴을 수용할 수 있어야 합니다. 풀은 일반적으로 모든 데이터베이스가 동시에 최대로 사용되지 않을 경우 데이터베이스에 대해 핫 및 콜드 사용률 패턴을 가정하므로 그룹에 대한 일정 수준의 오버커밋이 예상됩니다.<br> **데이터베이스당 eDTU 최대값을 어떻게 설정해야 하나요?** <br> 데이터베이스당 eDTU MAX 또는 eDTU 최대값을 ([데이터베이스 최대 사용률])로 설정합니다. 예를 들어, 데이터베이스당 최고 사용률이 50 DTU이며 그룹 내 100개의 데이터베이스 중 20%만이 동시에 최대로 사용되는 것으로 가정합니다. 데이터베이스당 eDTU 최대값이 50 eDTU로 설정된 경우 풀을 5배만큼 초과 커밋하고 그룹의 eDTU 보장(POOL eDTU)을 1,000 eDTU로 설정하는 것이 적합합니다. 또한 eDTU 최대값은 데이터베이스에 대한 리소스 보장이 아니며, 사용 가능한 경우 도달할 수 있는 eDTU 최고 한도입니다. |


## 풀에 데이터베이스 추가 및 제거

풀이 만들어진 후에 **탄력적 데이터베이스** 페이지에서 데이터베이스를 추가하거나 제거하여 풀에서 기존 데이터베이스를 추가하거나 제거할 수 있습니다.(풀을 찾고 **Essentials**의 **탄력적 데이터베이스** 링크를 클릭합니다)

- **데이터베이스 추가**를 클릭하여 풀에 추가할 수 있는 데이터베이스 목록을 엽니다.

    또는

- 풀에서 제거할 데이터베이스를 선택한 후에 **데이터베이스 제거**를 클릭합니다.

![권장되는 풀](./media/sql-database-elastic-pool-portal/add-remove-databases.png)

## 풀에 새 데이터베이스 만들기

원하는 풀으로 이동하고 **데이터베이스 만들기**를 클릭하여 풀에서 새 데이터베이스를 만듭니다.

SQL 데이터베이스는 올바른 서버 및 풀에 대해 구성되어 있으므로 이름을 입력하고 데이터베이스 옵션을 선택한 다음 **확인**을 클릭하여 새 데이터베이스를 만듭니다.


   ![탄력적 데이터베이스 만들기](./media/sql-database-elastic-pool-portal/create-database.png)



## 탄력적 데이터베이스 풀 모니터링 및 관리

탄력적 데이터베이스 풀을 만든 후 기존 풀 목록을 검색하고 원하는 풀을 선택하여 포털 내의 풀을 모니터링하고 관리할 수 있습니다.

풀을 만든 후에 다음과 같은 작업을 수행할 수 있습니다.

- **풀 구성**을 선택하여 풀 eDTU 및 데이터베이스당 eDTU 설정을 변경합니다.
- **작업 만들기**를 선택하고 탄력적 작업을 만들어 풀에 있는 데이터베이스를 관리합니다. 탄력적인 작업을 통해 개수에 관계없이 풀에 있는 데이터베이스에 대해 Transact-SQL 스크립트를 실행할 수 있습니다. 자세한 내용은 [탄력적 데이터베이스 작업 개요](sql-database-elastic-jobs-overview.md)를 참조하세요.
- **작업 관리**를 선택하여 기존 탄력적 작업을 관리합니다.
- 풀에 새 데이터베이스를 만들려면 **데이터베이스 만들기**를 클릭합니다.
- 풀 eDTU 및 풀당 GB를 조정하려면 풀 설정 링크를 클릭합니다.
- 풀에 데이테베이스를 추가하거나 삭제하려면 데이터베이스 링크를 클릭합니다.
- 풀의 탄력적 데이터베이스에 대한 최대 및 최소 eDTU를 조정하려면 데이터베이스 설정 링크를 클릭합니다. 

![탄력적 풀 모니터링](./media/sql-database-elastic-pool-portal/pool-tasks.png)


기존 풀을 선택하면 해당 풀의 리소스 사용률을 볼 수 있습니다. **리소스 사용률** 차트를 클릭하면 차트 및 설정 경고를 사용자 지정할 수 있는 **메트릭** 블레이드가 열립니다.

![탄력적 풀 모니터링][4] ![리소스 사용률][6]

**차트 편집**을 클릭하여 풀에 대한 원격 분석 데이터를 손쉽게 볼 수 있도록 매개 변수를 추가합니다.


![차트 편집][7]
