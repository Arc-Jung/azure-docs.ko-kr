# 탄력적 데이터베이스 쿼리 – 2015년 6월 미리 보기 

탄력적 데이터베이스 쿼리는 2015년 6월에 미리 보기가 출시된 Azure SQL 데이터베이스의 새로운 기능입니다. 이 미리 보기에는 Azure SQL 데이터베이스에 대한 단일 연결을 사용하여 여러 데이터베이스에 걸쳐 T-SQL 쿼리를 실행하는 기능이 도입되었습니다. 탄력적 데이터베이스 분할된 데이터베이스 맵을 사용하여 정의된 분할된 데이터 집합을 통합된 단일 데이터 저장소로 표시할 수 있습니다. 보고 및 데이터 통합 목적으로 사용되며, 모든 분산 쿼리 처리는 백그라운드에서 수행됩니다. ADO.Net, ODBC, JDBC, Node.js, PHP 등을 포함하여 SQL 데이터베이스와 통신할 수 있는 모든 드라이버를 사용하여 연결할 수 있습니다.

## 탄력적 데이터베이스 쿼리 시나리오

탄력적 데이터베이스 쿼리의 목표는 여러 데이터베이스 또는 분할된 데이터베이스를 통해 하나의 전체적인 T-SQL 쿼리 결과를 얻을 수 있도록 분할된 데이터 계층의 보고 시나리오를 간소화하는 것입니다. T-SQL은 사용자 또는 응용 프로그램에서 직접 작성할 수도 있고 글로벌 쿼리 탄력적 쿼리 데이터베이스에 연결된 도구를 통해 간접적으로 작성할 수도 있습니다. 따라서 탄력적 확장 라이브러리를 사용하여 직접 확장할 수 없는 상용 BI, 보고 또는 데이터 통합 도구, 패키지 소프트웨어에 특히 유용합니다. 또한 SQL Server Management Studio 또는 Visual Studio에서 발급하는 쿼리를 통해 분할된 전체 데이터 컬렉션에 쉽게 액세스할 수 있으며, Entity Framework 또는 기타 ORM 환경에서 투명한 다중 분할 데이터 액세스를 지원합니다. 그림 1은 이 시나리오를 잘 보여 줍니다. 전역 쿼리 탄력적 DB 쿼리는 기존 클라우드 응용 프로그램(탄력적 확장 라이브러리를 사용할 수 있음) 외에도 분할된 데이터 계층에 쿼리를 제출하는 또 다른 방법을 제공합니다.

![탄력적 데이터베이스 쿼리][1]

다음 토폴로지를 통해 시나리오를 이해할 수 있습니다.

-	분할된 데이터 계층(토폴로지 1): 이 데이터 계층은 분할을 목적으로 설계되었습니다. 분할은 (1)탄력적 데이터베이스 클라이언트 라이브러리 또는 (2)자체 분할을 사용하여 수행 및 관리됩니다. 보고의 경우 기능 또는 연결 요구 사항이 탄력적 데이터베이스 클라이언트 라이브러리의 다중 분할 쿼리가 수행할 수 있는 한계를 초과하는 여러 분할된 데이터베이스에 대해 보고서가 계산됩니다. 

-	다중 데이터베이스 설계(토폴로지 2): 종류가 다른 데이터는 서로 다른 데이터베이스에 상주하도록 데이터 계층이 수직적으로 분할됩니다. 보고의 경우 여러 데이터베이스에 분산되어 있지만 보고서 작성 시에는 하나의 전체 결과로 통합되어야 하는 데이터에 대해 보고서가 계산됩니다.

탄력적 데이터베이스 쿼리 미리 보기는 두 방법을 모두 지원할 것입니다. 토폴로지 1을 사용하는 고객은 탄력적 데이터베이스 클라이언트 라이브러리를 사용하여 분할된 데이터베이스를 관리할 경우 기존의 분할된 데이터베이스 맵을 사용할 수 있습니다. 그러므로 자체적으로 데이터베이스를 분할하는 고객은 이 미리 보기 기능을 사용하려면 탄력적 데이터베이스 도구를 사용하여 데이터 계층에 대한 분할된 데이터베이스 맵을 만들어야 합니다. 토폴로지 2를 따르는 고객은 각 데이터베이스에 대해 서로 다른 분할된 데이터베이스 맵을 만들어야 합니다. 이러한 분할된 데이터베이스 맵은 각각 하나의 분할된 데이터베이스만을 가리키게 되며 이후에 분할된 데이터베이스 간 쿼리에 테이블을 노출할 수 있습니다.
