# <a name="time-zones-in-azure-sql-database-managed-instance"></a>Azure SQL 데이터베이스 관리 인스턴스의 표준 시간대

조정된 유니버설 타임(UTC)은 클라우드 솔루션의 데이터 계층에 권장되는 표준 시간대입니다. 또한 Azure SQL Database 관리인스턴스는 날짜 및 시간 값을 저장하고 특정 표준 시간대의 암시적 컨텍스트를 사용하여 날짜 및 시간 함수를 호출하는 기존 응용 프로그램의 요구 사항을 충족하는 표준 시간대를 선택할 수도 있습니다.

[GETDATE()](https://docs.microsoft.com/sql/t-sql/functions/getdate-transact-sql) 또는 CLR 코드와 같은 T-SQL 함수는 인스턴스 수준에서 설정된 표준 시간대를 준수합니다. SQL Server 에이전트 작업은 인스턴스의 표준 시간대에 따라 일정을 따릅니다.

  >[!NOTE]
  > 관리되는 인스턴스는 표준 시간대 설정을 지원하는 Azure SQL Database의 유일한 배포 옵션입니다. 다른 배포 옵션은 항상 UTC를 따릅니다.
UTC가 아닌 표준 시간대에서 날짜 및 시간 정보를 해석해야 하는 경우 단일 및 풀풀된 SQL 데이터베이스에서 [AT TIME ZONE을](https://docs.microsoft.com/sql/t-sql/queries/at-time-zone-transact-sql) 사용합니다.

## <a name="supported-time-zones"></a>지원되는 표준 시간대

지원되는 표준 시간대 집합은 관리되는 인스턴스의 기본 운영 체제에서 상속됩니다. 새 표준 시간대 정의를 얻고 기존 표준 시간대에 대한 변경 사항을 반영하도록 정기적으로 업데이트됩니다.

[일광 절약 시간제/표준 시간대 변경 정책은](https://aka.ms/time) 2010년부터 기록 정확도를 보장합니다.

지원되는 표준 시간대의 이름이 있는 목록은 [sys.time_zone_info](https://docs.microsoft.com/sql/relational-databases/system-catalog-views/sys-time-zone-info-transact-sql) 시스템 보기를 통해 노출됩니다.

## <a name="set-a-time-zone"></a>표준 시간대 설정

관리되는 인스턴스의 표준 시간대는 인스턴스를 만드는 동안에만 설정할 수 있습니다. 기본 표준 시간대는 UTC입니다.

  >[!NOTE]
  > 기존 관리되는 인스턴스의 표준 시간대는 변경할 수 없습니다.

### <a name="set-the-time-zone-through-the-azure-portal"></a>Azure 포털을 통해 표준 시간대 설정

새 인스턴스에 대한 매개변수를 입력할 때 지원되는 표준 시간대 목록에서 표준 시간대를 선택합니다.
  
![인스턴스 를 만드는 동안 표준 시간대 설정](media/sql-database-managed-instance-timezone/01-setting_timezone-during-instance-creation.png)

### <a name="azure-resource-manager-template"></a>Azure Resource Manager 템플릿

[리소스 관리자 템플릿에서](https://aka.ms/sql-mi-create-arm-posh) timezoneId 속성을 지정하여 인스턴스를 만드는 동안 표준 시간대를 설정합니다.

```json
"properties": {
                "administratorLogin": "[parameters('user')]",
                "administratorLoginPassword": "[parameters('pwd')]",
                "subnetId": "[parameters('subnetId')]",
                "storageSizeInGB": 256,
                "vCores": 8,
                "licenseType": "LicenseIncluded",
                "hardwareFamily": "Gen5",
                "collation": "Serbian_Cyrillic_100_CS_AS",
                "timezoneId": "Central European Standard Time"
            },

```

timezoneId 속성에 대 한 지원 되는 값의 목록은이 문서의 끝에 있습니다.

지정하지 않으면 표준 시간대가 UTC로 설정됩니다.

## <a name="check-the-time-zone-of-an-instance"></a>인스턴스의 표준 시간대 확인

[CURRENT_TIMEZONE](https://docs.microsoft.com/sql/t-sql/functions/current-timezone-transact-sql) 함수는 인스턴스의 표준 시간대의 표시 이름을 반환합니다.

## <a name="cross-feature-considerations"></a>기능 간 고려 사항

### <a name="restore-and-import"></a>복원 및 가져오기

백업 파일을 복원하거나 표준 시간대 설정이 다른 서버에서 관리되는 인스턴스 또는 서버로 데이터를 가져올 수 있습니다. 주의해서 그렇게 하십시오. 표준 시간대 설정이 다른 두 SQL Server 인스턴스 간에 데이터를 전송할 때와 마찬가지로 응용 프로그램 동작과 쿼리 및 보고서의 결과를 분석합니다.

### <a name="point-in-time-restore"></a>지정 시간 복원

시간 별 복원을 수행하면 복원 할 시간이 UTC 시간으로 해석됩니다. 이렇게 하면 일광 절약 시간제및 잠재적인 변경으로 인한 모호성을 피할 수 있습니다.

### <a name="auto-failover-groups"></a>자동 장애 조치 그룹

장애 조치 그룹에서 기본 및 보조 인스턴스에서 동일한 표준 시간대를 사용하는 것은 적용되지 않지만 권장됩니다.

  >[!WARNING]
  > 장애 조치 그룹의 기본 및 보조 인스턴스에 대해 동일한 표준 시간대를 사용하는 것이 좋습니다. 특정 드문 사용 사례로 인해 기본 및 보조 인스턴스에서 동일한 표준 시간대를 유지 하지 않습니다. 수동 또는 자동 장애 조치의 경우 보조 인스턴스는 원래 표준 시간대를 유지한다는 점을 이해하는 것이 중요합니다.

## <a name="limitations"></a>제한 사항

- 기존 관리되는 인스턴스의 표준 시간대는 변경할 수 없습니다.
- SQL Server 에이전트 작업에서 시작된 외부 프로세스는 인스턴스의 표준 시간대를 준수하지 않습니다.
