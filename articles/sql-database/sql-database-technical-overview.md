# SQL 데이터베이스 개요

SQL 데이터베이스는 Microsoft SQL Server 엔진을 기반으로 한 클라우드 환경의 관계형 데이터베이스 서비스로, 업무 수행에 중요한 기능이 기본 제공됩니다. SQL 데이터베이스는 클라우드 개발자 및 솔루션 설계자에게 예측 가능한 성능, 확장성, 비즈니스 연속성, 데이터 보호 기능을 제공하며 관리할 필요가 거의 없습니다. 무엇보다도, SQL 데이터베이스는 기존 SQL Server 도구, 라이브러리 및 API를 지원하므로 손쉽게 클라우드로 이동할 수 있습니다. 가상 컴퓨터 또는 인프라를 관리할 필요 없이 SQL 데이터베이스를 [가동시켜 몇 초 내에 실행](sql-database-get-started.md)할 수 있습니다. 프로그래밍 방식 DBA와 비슷한 기능을 사용하여 효율적인 DevOps가 가능하며 자체 관리 기능이 있어서 데이터베이스에 대한 유지 관리가 거의 필요하지 않습니다. 따라서 사용자는 응용 프로그램 개발 및 관리에 집중할 수 있습니다.

SQL 데이터베이스를 사용하면 필요에 따라 높이거나 낮출 수 있는 예측 가능한 성능을 통해 사용 중지 없이 단일 데이터베이스로 [강화](sql-database-service-tiers.md)할 수 있습니다. 반면, [탄력적 데이터베이스](sql-database-elastic-pool.md)를 선택하면 [탄력적 데이터베이스 작업](sql-database-elastic-jobs-overview.md)의 강력한 관리 기능 및 [탄력적 데이터베이스 도구](sql-database-elastic-scale-get-started.md)의 프로그래밍 방식의 기능을 활용하여 사용자가 제어하는 예산 내에서 수천 개의 데이터베이스를 확장할 수 있습니다.

업계를 선도하는 99.99% 가용성 [SLA](http://azure.microsoft.com/support/legal/sla/)는 응용 프로그램의 가동 중지 시간이 거의 없도록 보장합니다. 감사, 복원 및 지역에서 복제를 통한 기본 제공 데이터 보호 기능으로 데이터를 보호할 수 있습니다. SQL 데이터베이스는 최대 35일까지의 모든 트랜잭션 데이터를 지정 시간으로 복원하는 기능을 제공합니다. 또한 사용자가 선택한 Azure 지역에 데이터를 복제할 수 있으며 비즈니스 요구에 맞게 설계된 지리적 재해 복구 정책을 쉽게 구현할 수 있습니다. [최신 Azure SQL 데이터베이스 버전](sql-database-preview-whats-new.md) V12는 SQL Server 엔진과 거의 완벽하게 호환됩니다.

SQL 데이터베이스는 가벼운 데이터베이스 작업부터 대규모 데이터베이스 작업까지 지원되는 기본, 표준 및 프리미엄 [서비스 계층](sql-database-service-tiers.md)이 제공되므로 계층 간에 이동하거나 혁신적인 응용 프로그램 설계를 위해 계층을 혼합할 수도 있습니다. Azure의 성능 및 도달 범위를 바탕으로, Azure 서비스를 SQL 데이터베이스와 조합하여 사용하면 고유한 현대적 응용 프로그램 설계 요구를 충족시키고, 비용 및 리소스를 효율적으로 운용하고, 새로운 비즈니스 기회를 만들 수 있습니다.
