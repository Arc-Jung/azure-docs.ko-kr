# 즉석에서 Azure SQL 데이터베이스 확장

## 앱이 적은 수의 고객에서 모든 사람들로 증가되면 Azure SQL 데이터베이스는 최소한의 앱 가동 중지 시간만으로 즉석에서 확장될 수 있습니다.

Microsoft Azure에서 SQL 데이터베이스를 실행하는 경우 장점 중 하나는 성능을 확장하거나 축소할 수 있고 즉석에서 신속하게 워크로드 요구 변화를 맞출 수 있다는 점입니다. SQL 데이터베이스는 광범위한 범위의 성능 수준을 제공하여 응용 프로그램의 특정 요구를 충족합니다. 또한 앱 사용자에게 예측 가능한 성능 환경을 제공하도록 각 수준에서 성능을 보장합니다.

![scale-on-the-fly](./media/sql-database-scale-on-the-fly/sql-database-scale-on-the-fly.png)

## 앱이 단번에 성공한 경우
SQL 데이터베이스를 사용하면 순식간에 쉽게 크기를 조정할 수 있습니다. 따라서 앱이 단번에 성공하면 한 번 클릭으로 새로운 성능 요구를 수용할 수 있습니다. 또한 가동 중지가 최소로 유지되므로 확장 또는 축소하는 경우 앱이 적중될지를 걱정할 필요가 없습니다.

[설명서 읽기](http://go.microsoft.com/fwlink/?LinkID=787569)

## 예산 내에서  

데이터베이스에 얼마나 많은 비용이 드는지는 앱을 만들려는 경우 고려해야 하는 사항이 아닙니다. 이 때문에 SQL 데이터베이스에 필요한 만큼 지불하는 계획이 있습니다. 기본, 표준 및 프리미엄 성능 옵션을 사용하여 예산에 맞는 성능 옵션을 결정할 수 있습니다. 그리고 앱이 시작되면 언제든지 확장할 수 있습니다.

[설명서 읽기](http://go.microsoft.com/fwlink/?LinkID=787570)

## 가장 빠른 성능 얻기

SQL 데이터베이스는 트랜잭션을 매우 빠르게 실행하고 동시에 실시간 분석 쿼리를 수행하는 첨단 메모리 내 기술을 포함합니다. 기본 제공 관리자는 개발자가 메모리 내 테이블, 인덱스 또는 저장 프로시저를 마이그레이션하는 데 도움이 됩니다. 결과적으로 최종 사용자는 30배 빠른 트랜잭션 및 100배 빠른 쿼리 처리 속도를 경험할 수 있습니다.

[설명서 읽기](http://go.microsoft.com/fwlink/?LinkID=787580)

SQL 데이터베이스에서 제공해야 하는 다른 항목을 보려면 홈 페이지로 이동합니다. [확인해 보세요.](https://azure.microsoft.com/services/sql-database/)
