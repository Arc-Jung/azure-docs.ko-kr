# SQL 데이터베이스 클라이언트 프로그램에 대한 오류 메시지

이 항목에서는 여러 범주의 오류 메시지를 나열합니다. 대부분의 범주는 Azure SQL 데이터베이스에 특정되며 Microsoft SQL Server에 적용되지 않습니다.


클라이언트 프로그램에서 사용자에게 모든 오류에 대해 사용자 지정된 대체 메시지를 제공하는 옵션이 있습니다.


**팁:** 추가 중요도는 *일시적인 오류* 섹션입니다. 이러한 오류는 클라이언트 프로그램에서 설계한 *다시 시도* 논리를 실행하여 작업을 다시 시도하도록 묻는 메시지를 표시해야 합니다.

## 일시적인 오류, 연결 끊김 및 기타 임시 오류

다음 표에서 Azure SQL 데이터베이스를 사용하여 인터넷을 통해 작업하는 동안 발생할 수 있는 연결 끊김 오류 및 다른 일시적인 오류에 대해 설명합니다.

일시적인 오류는 일시적인 장애라고도 합니다. 프로그램이 `SqlException`을(를) 포착하는 경우 프로그램은 `sqlException.Number` 값이 일시적인 오류 섹션에 나열된 값인지 여부를 확인할 수 있습니다. `Number` 값이 일시적인 오류를 나타내는 경우 프로그램은 연결 설정을 다시 시도한 다음 연결을 통해 쿼리를 다시 시도할 수 있습니다. 다시 시도 논리의 코드 예제는 다음을 참조하십시오.


- [클라이언트 개발 및 SQL 데이터베이스에 대한 빠른 시작 코드 샘플](sql-database-develop-quick-start-client-code-samples.md)

- [Azure SQL 데이터베이스에 안정적으로 연결하는 방법](http://msdn.microsoft.com/library/azure/dn864744.aspx)

- 트랜잭션이 너무 오랫동안 열려있었을 수도 있습니다.
- 트랜잭션이 너무 많은 잠금을 보유하고 있을 수도 있습니다.
- 프로그램이 너무 많은 메모리를 사용하고 있을 수도 있습니다.
- 프로그램이 너무 많은 `TempDb` 공간을 사용하고 있을 수도 있습니다.


**팁:** 다음 링크는 이 섹션의 대부분 또는 모든 오류에 적용되는 자세한 정보를 제공합니다.
