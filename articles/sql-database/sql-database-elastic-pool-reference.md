# SQL 데이터베이스 탄력적 데이터베이스 풀 참조(Preview)

수십, 수백 또는 수천 개의 데이터베이스를 다루는 SaaS 개발자라면 탄력적 데이터베이스 풀을 통해 전체 데이터베이스 그룹의 생성, 유지 관리, 성능 및 비용 관리 프로세스를 간소화할 수 있습니다.

이 참조 문서는 탄력적 풀 문서 및 프로그래밍 기능 정보에 대한 링크와 세부 정보를 제공합니다.

## 개요

탄력적 풀은 여러 데이터베이스에서 공유하는 DTU(데이터베이스 처리량 단위) 및 저장소의 컬렉션입니다. 언제든지 탄력적 데이터베이스를 풀에 추가하거나 풀에서 제거할 수 있습니다. 풀에 있는 탄력적 데이터베이스는 필요한 리소스만을 풀에서 사용하므로, 리소스를 필요로 하는 활성 데이터베이스에서 가용 리소스를 확보할 수 있습니다.



## 탄력적 풀을 만들고 관리하기 위한 필수 조건


- 탄력적 풀은 Azure SQL 데이터베이스 V12 서버에서만 사용할 수 있습니다.   
- 탄력적 풀을 위한 PowerShell 및 REST API는 Azure 리소스 관리자(ARM)에서 지원되며, 서비스 관리 명령(RDFE)은 지원되지 않습니다. 
- 탄력적 풀 만들기 및 관리는 [Microsoft Azure 포털](https:portal.azure.com)에서만 지원됩니다. 


## 현재 Preview의 제한 사항

- 탄력적 풀의 현재 Preview에 적용되는 가격 책정 계층은 표준입니다.  
- 탄력적 풀로 데이터베이스를 직접 가져오는 것은 지원되지 않습니다. 독립 실행형 데이터베이스를 가져온 다음 해당 데이터베이스를 풀로 이동할 수 있습니다. 풀 내에서 데이터베이스를 내보내는 것은 지원됩니다.

## 네임스페이스 및 끝점 세부 정보
탄력적 풀은 Microsoft Azure SQL 데이터베이스에 있는 “ElasticPool” 유형의 ARM 리소스입니다.

- **네임스페이스**: Microsoft.Sql/ElasticPool
- REST API 호출에 대한 **보조 끝점**(Azure 리소스 관리자): https://management.azure.com



## 탄력적 데이터베이스 풀 속성

| 속성 | 설명 |
| :-- | :-- |
| creationDate | 풀이 만들어진 날짜입니다. |
| databaseDtuMax | 풀에 있는 단일 데이터베이스에서 사용할 수 있는 최대 DTU 수입니다. 데이터베이스 DTU 최대값은 리소스 보장이 아닙니다. DTU 최대값은 풀에 있는 모든 데이터베이스에 적용됩니다. |
| databaseDtuMin | 풀에 있는 단일 데이터베이스가 보장하는 최소 DTU 수입니다. 데이터베이스 DTU 최소값은 0으로 설정할 수 있습니다. DTU 최소값은 풀에 있는 모든 데이터베이스에 적용됩니다. 풀에 있는 데이터베이스 수와 데이터베이스 DTU 최소값을 곱한 값이 풀 자체의 DTU를 초과할 수 없습니다. |
| DTU | 풀에 있는 모든 데이터베이스에서 공유하는 DTU 수입니다. |
| 버전 | 풀의 서비스 계층입니다. 풀에 있는 모든 데이터베이스에는 버전이 있습니다. |
| elasticPoolId | 풀의 인스턴스의 GUID입니다. |
| elasticPoolName | 풀의 이름입니다. 이름은 상위 서버에 대해 고유합니다. |
| 위치 | 풀이 만들어진 데이터 센터의 위치입니다. |
| state | 구독 청구서 지불이 이행되지 않은 경우 상태가 “비활성화”가 되며 그렇지 않으면 “Ready”가 됩니다. |
| storageMB | 풀의 저장소 한도(단위 MB)입니다. 풀에 있는 단일 데이터베이스는 모두 표준 버전의 저장소 한도(250GB)까지 사용할 수 있지만, 풀에 있는 모든 데이터베이스에서 사용하는 저장소 합계가 해당 풀의 제한을 초과할 수 없습니다. |


## 탄력적 풀 및 탄력적 데이터베이스에 대한 DTU 및 저장소 제한

풀의 저장소 한도는 풀의 DTU 양에 따라 결정됩니다(각 DTY = 스토리지 1GB). 예를 들어 200 DTU의 풀이라면 저장소 한도가 200GB입니다.

| ( 속성 사용) | 기본값 | 유효한 값 |
| :-- | :-- | :-- |
| DTU | 200 | 200, 400, 800, 1200 |
| databaseDtuMax | 100 | 10, 20, 50 100 |
| databaseDtuMin | 0 | 0, 10, 20, 50 |
| storageMB | 200GB* | 200GB, 400GB, 800GB, 1200GB |

*API의 단위는 GB가 아니라 MB

## 작업자 및 세션 제한

탄력적 데이터베이스에 있는 모든 데이터베이스에 대해 지원되는 동시 작업자 및 동시 세션의 최대 수는 풀의 DTU 설정에 따라 결정됩니다.

| DTU | 최대 동시 작업자 | 최대 동시 세션 |
| :-- | :-- | :-- |
| 200 | 400 | 4,800 |
| 400 | 800 | 9,600 |
| 800 | 1,600 | 19,200 |
| 1,200 | 2,400 | 28,800 |


## Azure 리소스 관리자 제한 사항

탄력적 풀을 사용하려면 Azure SQL 데이터베이스 V12 서버가 필요합니다. 서버는 리소스 그룹 내에 위치합니다.

- 각 리소스 그룹에는 최대 800개의 서버가 포함될 수 있습니다.
- 각 서버에는 최대 800개의 탄력적 풀이 포함될 수 있습니다.
- 각 탄력적 풀에는 최대 100개의 데이터베이스가 포함될 수 있습니다.


## 탄력적 풀 작업의 대기 시간

- 데이터베이스당 보장된 DTU(databaseDtuMin) 또는 데이터베이스당 최대 DTU(databaseDtuMax)를 변경하는 작업은 일반적으로 5분 이내에 완료됩니다. 
- 풀의 DTU/저장소 한도(storageMB) 변경은 풀에 있는 모든 데이터베이스에서 사용 중인 공간의 전체 크기에 따라 달라집니다. 변경 시간은 100GB당 평균 90분 이하입니다. 예를 들어, 풀에 있는 모든 데이터베이스에서 사용 중인 총 공간이 200GB일 경우, 풀 DTU/저장소 한도를 변경하는 예상 대기 시간은 3시간 이하입니다.

## 대금 청구 및 가격 정보

탄력적 데이터베이스 풀은 다음 특성에 따라 대금이 청구됩니다.

- 탄력적 풀은 풀에 데이터베이스가 없더라도 생성 시부터 대금이 청구됩니다. 
- 탄력적 풀은 시간당 대금이 청구됩니다. 이는 독립 실행형 데이터베이스의 성능 수준과 같은 계량 빈도입니다. 
- 탄력적 풀이 새로운 크기의 DTU로 조정된 경우, 크기 조정 작업이 완료될 때까지 새 DTU 크기에 따른 대금이 청구되지 않습니다. 이는 독립 실행형 데이터베이스의 성능 수준을 변경하는 것과 동일한 패턴을 따릅니다. 


- 탄력적 풀의 가격은 풀의 DTU 수와 풀에 포함된 데이터베이스 수를 기준으로 책정됩니다.
- 가격은 (풀 DTU 수)x(DTU당 단위 가격) + (데이터베이스 수)x(데이터베이스당 단위 가격)으로 계산됩니다.

탄력적 풀의 단위 DTU 가격은 동일한 서비스 계층에 있는 독립 실행형 데이터베이스의 단위 DTU 가격보다 높습니다. 자세한 내용은 [SQL 데이터베이스 가격](http://azure.microsoft.com/pricing/details/sql-database/)을 참조하세요.
