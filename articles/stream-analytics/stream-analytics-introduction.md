# 스트리밍 데이터 분석에 대한 완전히 관리되는 클라우드 서비스, Azure 스트림 분석 소개

Azure 스트림 분석은 완전히 관리되는 서비스로, 클라우드의 스트리밍 데이터에 대해 대기 시간이 짧고 확장성이 뛰어난 고가용성의 복합 이벤트 처리 기능을 제공합니다. Azure 스트림 분석은 사물 인터넷)Internet of Things)용 빅 데이터 분석 서비스로서 개발자가 데이터 스트림을 기록 레코드 또는 참조 데이터와 쉽게 결합하여 쉽고 빠르게 비즈니스 통찰력을 얻을 수 있도록 합니다.

## 동기 및 개요

현재 방대한 양의 데이터가 유선을 통해 높은 속도로 흐르고 있습니다. 이 스트리밍 데이터를 실시간으로 처리 할 수 있는 조직은 효율성을 획기적으로 개선함으로써 시장에서 차별화를 이룰 수 있습니다. 실시간 스트리밍 분석 시나리오는 금융 서비스 회사에서 제공한 개인 설정된 실시간 주식 거래 분석 및 경고, 실시간 부정 행위 검색, 데이터 및 ID 보호 서비스, 물리적 개체(IoT, 사물 인터넷)에 포함된 센서 및 작동기에 의해 생성된 데이터의 안정적인 수집 및 분석, 웹 클릭 스트림 분석, 특정 기간에 고객이 경험하는 품질이 저하될 때 경고를 발생하는 CRM(고객 관계 관리) 응용 프로그램과 같이 모든 산업 분야에서 찾을 수 있습니다. 기업은 현대의 치열한 경쟁 사회에서 성공을 거두기 위해 이러한 실시간 이벤트 스트림 데이터 분석을 수행하기 위한 가장 유연하고 안정적이며 경제적인 방법을 찾고 있습니다.

스트림 처리 시스템을 구축하는 것은 복잡한 작업입니다. 상관 관계 및 집계와 같은 스트리밍 작업은 효율적으로 구현해야 할 뿐만 아니라 확장 가능하고 내결함성을 갖추어야 합니다. 그 외에도 배포, 디버깅 및 모니터링을 비롯한 추가적인 운영 문제가 산재해 있습니다. 이러한 솔루션의 구축 및 유지 관리와 관련된 비용은 빠르게 증가하고 있습니다. 대기업은 사용자 구축 솔루션을 통해 이러한 고비용 지불 방식을 벗어났지만 중소기업은 진입을 막는 높은 장벽과 고비용 때문에 기회를 놓치는 경우가 많습니다. 특별히 스트리밍 분석을 위해 클라우드에서 관리되는 서비스를 제공 함으로써 스트림 분석은 이러한 문제를 다룹니다.

Azure 포털에서 클릭 몇 번만하면 고객은 SQL 유사 언어로 스트리밍 분석 작업을 작성하여 변환을 지정하고 전반적인 스트리밍 파이프라인의 규모/속도를 모니터링할 수 있습니다. 이 서비스는 초당 몇 킬로바이트에서 기가바이트 이상의 이벤트를 처리할 수 있게 쉽게 확장할 수 있습니다. 오늘날 사용할 수 있는 대부분의 다른 스트리밍 솔루션에서는 고객이 복잡한 사용자 지정 코드를 작성해야 하지만 스트림 분석을 사용하면 간단하고 선언적이며 친숙한 SQL을 작성할 수 있습니다.

스트림 분석은 간단한 필터에서 복잡한 상관 관계에 이르는 다양한 작업을 제공합니다. 기간 이동 집계와 같은 시간 기반 기간 이동 작업을 정의하거나 여러 스트림 간 상관 관계를 파악하여 시퀀스와 같은 패턴을 감지하는 작업, 심지어 현재의 조건을 기록 값 및 모델과 비교하는 작업은 간단한 SQL 유사 스트림 분석 쿼리 언어 연산자 집합을 사용하여 몇 분만에 수행할 수 있습니다. 스트리밍 파이프라인을 지정하는 것은 입력 및 출력을 구성하는 것만큼 간단히 진행되며 원하는 변환을 설명하는 SQL 유사 쿼리를 사용할 수 있습니다. 이를 통해 대부분의 간단한 사용 사례는 충분히 해결되지만 규모가 더 크고 좀 더 복잡한 시나리오는 스트림 분석 구성의 용이성을 통해 큰 이점을 얻을 수 있습니다. 사용자는 원하는 최대 처리량을 달성하기 위해 파이프라인의 각 단계에 투입할 계산 능력을 파악할 수 있습니다.

스트림 분석은 현재 스트림 수집을 위해 Azure 이벤트 허브, 기록 데이터를 위해 Azure Blob 서비스에 직접 연결됩니다. Azure 이벤트 허브의 입력 및 출력 인터페이스 범위를 사용하면 계산의 스트리밍 특성을 손실하지 않고도 스트림 분석을 다른 데이터 소스 및 처리 엔진에 쉽게 통합할 수 있습니다. 데이터는 스트림 분석 기능으로부터 Azure 저장소에 기록된 다음 나중에 일련의 이벤트로 다시 처리되거나 Azure HDInsight를 통해 다른 형식의 배치 분석에서 사용될 수 있습니다. 시간이 중요한 처리 및 언어 통합에 따른 직관적 사양 구현을 위해 고도로 조율된 스트림 분석의 스트리밍 엔진을 개발하는 Microsoft Research 작업이 수년 동안 진행되었습니다. 스트림 분석은 처리 확장을 위해 Hadoop, YARN 및 REEF의 오픈 소스 커뮤니티를 활용하여 구축되었습니다.


## 주요 기능

### 사용 편의성
스트림 분석은 변환을 설명하는 간단하고 선언적인 쿼리 모델을 지원합니다. 사용 편의성을 최적화하기 위해 SQL 변형 기능으로 DSL(Domain-Specific Language)을 선택했으며 고객은 이를 통해 스트리밍 시스템 자체의 기술 복잡성을 느끼지 못하게 됩니다. 스트림 분석 쿼리 언어를 사용하여 조인, 집계, 예측, 필터 등과 같은 일반적인 작업은 물론, 임시 조인, 기간 이동 집계, 임시 필터를 비롯한 임시 함수를 빠르고 쉽게 구현할 수 있습니다.

몇 가지 직관적인 방법으로 SQL 사용자에게 SQL의 의미 체계를 확장하고 있지만 구현 및 도구 사용을 용이하게 하기 위해 여전히 SQL 표준 구문 범위를 벗어나지는 않습니다. 쿼리 언어에 대한 자세한 내용은 “다음 단계” 섹션을 참조하세요.

### 확장성
스트림 분석은 초당 최대 1B의 높은 이벤트 처리량를 다룰 수 있습니다. 다시 말해서 Azure 이벤트 허브와의 통합을 통해 1초에 연결된 장치, 클릭 스트림, 로그 파일에서 발생하는 수백만 개의 이벤트를 수집할 수 있는 솔루션을 구현할 수 있습니다. 이 작업을 수행하기 위해 파티션당 1MB/s를 발생하는 이벤트 허브 분할 기능을 활용합니다. 스트림 분석 기능은 가로 및 세로 축을 따라 이러한 수집된 이벤트를 처리하면서 분할을 지원합니다. 사용자는 쿼리 정의 내에서 계산을 각각이 데이터 병렬 실행 요소로 추가로 분할될 수 있는 많은 수의 논리적 단계로 분할할 수 있습니다. 시간이 지남에 따라 스트림 분석 기능은 이벤트 수집 속도, 처리의 복잡성 및 예상 대기 시간에 따라 자동으로 확장되므로 사용자는 작업을 적절히 사용자 지정할 수 있습니다. 확장성을 구현하는 방법에 자세한 내용은 “다음 단계” 섹션에 있는 링크를 참조하세요.

### 안정성, 반복성 및 빠른 복구
클라우드 관리 서비스인 스트림 분석은 기본 제공 복구 및 검사점 설정 기능을 통해 노드 오류가 발생하는 경우에도 데이터가 손실되지 않도록 돕고 비즈니스 연속성을 제공합니다. 이러한 기능은 스트림 분석의 클라이언트 앵커 모델 아키텍처에 의해 제공됩니다. 이 서비스는 상태를 지속하여 노드 오류로부터의 다시 시작, 다시 계산 및 캐시 출력을 최적화하여 다운스트림 오류를 효율적으로 처리할 수 있게 작성되었습니다.

이 서비스는 내부적으로 상태를 유지하는 기능을 통해 이벤트를 보관할 수 있는 경우 반복 가능한 결과를 제공하고, 향후에도 동일한 결과를 제공하면서 처리를 다시 적용할 수 있습니다. 이 기능을 통해 고객은 특정 시점으로 복구하고 근본 원인 분석 및 가상 분석과 같은 시나리오를 위해 계산을 조사할 수 있습니다. 이러한 기능이 함께 사용되면서 처리 노드 오류로부터 빠르게 복구하여 손실된 상태를 빠르게 다시 처리할 수 있도록 합니다.

### 짧은 대기 시간
관리 되는 서비스의 또 다른 이점은 짧은 대기 시간입니다. 스트림 분석은 현재 종단 간 대기 시간이 1초 미만으로 확인될 수 있게 최적화되었습니다. 따라서 시스템은 높은 처리량으로 계속 처리할 수 있습니다. 통신 모델은 구성된 제한 시간 및 크기 제한에 따라 작동되는 푸시 기반 적응 배치 모델입니다. 따라서 이벤트 및 기타 레코드는 제한에 도달할 때까지 배치로 처리됩니다. 따라서 시스템이 높은 처리량으로 짧은 대기 시간을 제공할 수 있는 것입니다.

### 참조 데이터
스트림 분석은 참조 데이터를 지정하고 사용할 수 있도록 합니다. 이러한 데이터는 기록 데이터일 수도 있고 시간이 지남에 따라 덜 자주 변경되는 데이터일 수도 있습니다. 시스템은 참조 데이터가 다른 이벤트 스트림처럼 실시간으로 수집된 기타 이벤트 스트림에 통합되도록 하여 간편하게 변환합니다. 참조 데이터의 생성 및 사용에 대한 자세한 내용은 “다음 단계” 섹션을 참조하세요.



## Azure 스트림 분석을 선택하는 비즈니스 동기

### 저렴한 비용
클라우드 서비스로서 스트림 분석 서비스는 저비용으로 실시간 분석 솔루션을 유지 관리할 수 있게 최적화되었습니다. 이 서비스는 사용자가 사용량에 따라 비용을 지불하도록 구축되었습니다. 사용량은 처리된 이벤트의 용량과 해당 스트리밍 분석 작업을 처리하기 위해 클러스터 내에 프로비전된 계산 능력의 양에 따라 결정됩니다.

### 빠른 구현
스트림 분석을 통해 하드웨어 비용이나 기타 선불 투자 없이, 시간이 많이 드는 설치 또는 설정 없이도 확장성이 뛰어난 실시간 이벤트 처리 솔루션을 보유할 수 있습니다. Microsoft Azure는 클라우드 서비스 공급자로서 사용자를 위해 모든 사항을 처리합니다. 단순히 Azure 포털을 통해 서비스에 가입하기만 하면 몇 분 안에 작동하고 실행할 수 있습니다. 포털의 친숙한 사용자 인터페이스는 입력 소스를 구성 및 테스트하고, 출력 저장소를 구성 및 테스트하는 단계별 빠른 시작 가이드를 안내합니다. 쉬운 쿼리 편집기는 자동 완성 및 권장 기능을 제공합니다.

### 빠른 개발
스트림 분석은 분산 시스템 수평 확장을 위한 분석 기능을 개발할 때의 마찰과 복잡성을 줄이는 데 도움을 줍니다. 개발자는 SQL 기반 쿼리 언어로 원하는 변환을 간단히 설명하고, 시스템은 확장성, 성능 및 복원력을 위해 작업 분산을 자동으로 처리하여 대부분의 스트림 처리 솔루션에 필요한 절차적 프로그래밍의 필요성을 없애줍니다. 스트림 분석은 기본 제공된 기능과 친숙한 포털의 사용을 통해, 고급 개발자들이 인프라 조달, 지속적인 유지 관리 및 규모 조정 등에 대해 걱정하지 않고도 실시간 처리를 빠르고 쉽게 수행할 수 있도록 지원합니다.

## 시나리오 및 사용 사례

### IoT(Internet of Things)를 위한 실시간 분석
장치가 점점 더 똑똑해지고 더 많은 장치가 통신 기능을 사용하여 작성되면서 이러한 장치에서 생성 및 수집한 데이터로 수행할 수 있는 작업은 상용 및 소비자 영역에서 계속 늘어나고 있습니다. 많은 양의 데이터를 사용할 수 있게 되면서 데이터를 신속하게 결합 및 처리하여 주변 환경과 주기적으로 사용하는 장치를 종합적으로 분석할 수 있을 것입니다. 자동 판매기 예를 통해 정식 IoT 시나리오를 설명할 수 있습니다.

자동 판매기는 시스템으로 수집될 수 있도록 제품 재고, 상태, 온도 등의 데이터를 주기적으로 현장 게이트웨이(자동 판매기에 IP 기능이 없는 경우) 또는 클라우드 게이트웨이(IP 기능이 있는 경우)로 전송합니다. 들어오는 데이터 스트림이 처리 및 변환되므로 계산된 출력이 게이트웨이를 통해 장치로 즉시 다시 공급되어 장치에서 해당 조치가 수행될 수 있게 됩니다. 예를 들어 컴퓨터가 과열되면 사용자가 개입하지 않고도 장치가 자동으로 재부팅되어 펌웨어 업그레이드가 수행되어야 할 수 있습니다. 또한 기술 지원 담당자 업무가 이벤트에 따라 자동으로 예약될 수 있게 처리된 출력에 따라 다른 경고 및 알림이 트리거될 수도 있습니다.

더 많은 데이터가 수집 및 처리되면서 시스템에서 확인된 패턴을 통해 개발하고 학습하는 데 기계 학습이 사용될 수도 있습니다. 예를 들어 시스템에 공급되는 실시간 이벤트 스트림을 기반으로 기계 정비 시기를 예측하거나 문제가 발생할 가능성이 있을 때 기술 지원 담당자의 예방 유지 관리 업무를 예약할 수 있습니다.

이와 같이 장치가 처리 시스템에 정보를 보내고 실시간으로 처리된 결과에 따라 작업을 수행하는 패턴은 IoT 사용 사례에서 일반적으로 볼 수 있습니다. 이러한 기타 시나리오에는 커넥티드 카, 클릭 동향 분석 및 시설 관리가 포함됩니다. 스트림 분석을 사용하여 Azure 이벤트 허브에서 데이터를 수집하고, 데이터를 처리한 후 이벤트 허브에 바로 다시 전송하여 구독된 장치가 해당 조치를 수행하도록 함으로써 짧은 대기 시간과 높은 처리량을 구현할 수 있습니다.


![빅 데이터 분석 및 사물인터넷: 사물인터넷용 Azure 스트림 분석 실시간 분석.][img.stream.analytics.scenario1]


### 대시보드를 통한 원격 분석 및 로그 분석
장치 컴퓨터 및 응용 프로그램 용량이 증가하면서 비즈니스를 운영하기 위해 대기업은 일반적으로 거의 실시간으로 풍부한 분석 자료를 만들어 변화하는 비즈니스 요구를 모니터링하고 대처하는 데 이러한 기능을 활용합니다. 정식 원격 분석/로그 분석 시나리오는 온라인 서비스 또는 응용 프로그램 예제를 사용하여 설명할 수 있지만 응용 프로그램이나 장치 원격 분석을 수집하고 보고하는 기업들에서 공통적인 패턴이 확인됩니다.

응용 프로그램 또는 서비스는 정기적으로 상태 데이터를 수집합니다. 특정 시점의 응용 프로그램이나 인프라의 현재 상태를 나타내는 데이터, 사용자 요청 로그 및 응용 프로그램 내에서 수행되는 동작이나 활동을 나타내는 기타 데이터가 수집됩니다. 데이터는 추가 처리를 위해 blob 또는 다른 유형의 데이터 저장소에 저장됩니다.

기록 분석을 위해 Blob 또는 기타 유형의 저장소에 데이터를 저장하는 것 외에도 실시간 대시보딩을 위한 최근 추세에 발맞춰, 고객은 조치를 취해야 할 때 대시보드 및/또는 알림 형식으로 최종 사용자에게 즉시 제공될 수 있게 들어오는 데이터 스트림을 직접 처리하고 변환하는 방식을 모색하고 있습니다. 예를 들어 서비스 사이트가 작동 중단되면 조사를 시작하고 문제를 신속하게 해결할 수 있도록 운영 담당자에게 알림이 제공될 수 있습니다. 이러한 몇 가지 여러 사용 사례에서 사용자는 일반적으로 실시간 스트림 분석을 통해 수집된 데이터를 처리한 후에 업데이트된 데이터 집합을 기반으로 구축된 실시간 대시보드를 모니터링합니다.
 
![Azure 스트림 분석: 대시보드를 통한 스트림 데이터 원격 분석 및 로그 분석][img.stream.analytics.scenario2]

### 추가 처리를 위해 이벤트 보관
기업에서 신속하고 기민한 업무 처리에 대한 기대는 계속 높아지고 있습니다. 기업 및 개발자는 이제 보다 높은 민첩성에 대한 요구를 충족하기 위해 사용하기 쉬운 클라우드 서비스를 선택하고 있으며 시스템에서 생성하는 연속 데이터 스트림을 거의 실시간으로 수집하고 처리할 수 있는 플랫폼을 찾고 있습니다. 오늘날 이러한 고객들은 데이터 저장소 쓰기에 소요되는 대기 시간을 줄이도록 최적화된 관리 서비스를 이용하지 못함으로써 비즈니스를 위한 가치 있는 통찰력을 제공할 수 있는 중요한 데이터를 얻지 못합니다. 정식 이벤트 보관 시나리오는 다음과 같이 설명할 수 있습니다.

전 세계적으로 분산된 다양한 장치 및 플랫폼의 데이터가 중앙 집중화된 데이터 수집기에 푸시됩니다. 데이터가 중앙 위치에 도달하면 개인적으로 식별이 가능한 정보, 지리적 태깅 추가, IP 조회 수행 등의 일부 상태 비저장 변환이 수행됩니다. 그런 다음 변환된 데이터는 오프라인 처리를 위해 HDInsight 또는 기타 도구에서 쉽게 사용될 수 있는 방식으로 Blob 저장소에 보관됩니다.

![추후 스트리밍 처리를 위한 Azure 스트림 분석 이벤트 보관][img.stream.analytics.scenario3]
