# <a name="filter-based-feature-selection"></a>필터 기반 기능 선택

이 문서에서는 Azure 기계 학습 디자이너(미리 보기)에서 필터 기반 기능 선택 모듈을 사용하는 방법을 설명합니다. 이 모듈은 예측 능력이 가장 큰 입력 데이터 집합의 열을 식별하는 데 도움이 됩니다. 

일반적으로 *기능 선택은* 지정된 출력이 주어지면 입력에 통계 테스트를 적용하는 프로세스를 말합니다. 목표는 출력을 더 예측하는 열을 결정하는 것입니다. 필터 기반 기능 선택 모듈은 선택할 수 있는 여러 기능 선택 알고리즘을 제공합니다. 이 모듈에는 Pearson 상관 관계 및 카제곱 값과 같은 상관 방지 방법이 포함되어 있습니다. 

필터 기반 기능 선택 모듈을 사용하는 경우 데이터 집합을 제공하고 레이블 또는 종속 변수가 포함된 열을 식별합니다. 그런 다음 피쳐 중요도 측정에 사용할 단일 방법을 지정합니다.

모듈은 예측 전력에 따라 순위가 매겨진 최상의 피처 열을 포함하는 데이터 집합을 출력합니다. 또한 선택한 메트릭에서 피처 이름과 점수를 출력합니다.  

## <a name="what-filter-based-feature-selection-is"></a>필터 기반 피쳐 선택은  

선택한 메트릭을 사용하여 관련없는 특성을 찾기 때문에 기능 선택을 위한 이 모듈을 "필터 기반"이라고 합니다. 그런 다음 모델에서 중복 열을 필터링합니다. 데이터에 맞는 단일 통계 측정값을 선택하면 모듈에서 각 피처 열에 대한 점수를 계산합니다. 열은 피처 점수에 따라 순위가 반환됩니다. 

적절한 기능을 선택하면 분류 정확도와 효율성을 개선할 수 있습니다. 

일반적으로 점수가 가장 좋은 열만 사용하여 예측 모델을 작성합니다. 피처 선택 점수가 낮은 열은 데이터 집합에 남아 모델을 작성할 때 무시될 수 있습니다.  

## <a name="how-to-choose-a-feature-selection-metric"></a>피처 선택 메트릭을 선택하는 방법

필터 기반 기능 선택 모듈은 각 열의 정보 값을 평가하기 위한 다양한 메트릭을 제공합니다. 이 섹션에서는 각 메트릭과 해당 메트릭이 적용되는 방식에 대한 일반적인 설명을 제공합니다. [기술 노트](#technical-notes) 및 각 모듈 구성 [지침에서](#how-to-configure-filter-based-feature-selection) 각 메트릭을 사용하기 위한 추가 요구 사항을 찾을 수 있습니다.

-   **피어슨 상관 관계**  

    Pearson의 상관 통계 또는 Pearson의 상관 계수는 통계 모델에서도 `r` 값으로 알려져 있습니다. 이 방법에서는 임의의 두 변수에 대해 상관 관계의 강도를 나타내는 값이 반환됩니다.

    두 변수의 공 분산을 구한 다음 표준 편차의 곱으로 나누는 방식으로 피어슨 상관 계수를 계산합니다. 두 변수의 배율 변경은 계수에 영향을 주지 않습니다.  

-   **치 제곱**  

    양방향 카이 제곱 테스트는 예상 값이 실제 결과와 얼마나 비슷한지를 측정하는 통계 방법입니다. 이 방법에서는 변수가 임의로 선택되며, 적절한 독립 변수 샘플에서 도출한다고 가정합니다. 결과적으로 생기는 카이 제곱 통계는 결과와 예상(임의) 결과가 얼마나 다른지를 나타냅니다.  


> [!TIP]
> 사용자 지정 기능 선택 방법에 대해 다른 옵션이 필요한 경우 [R 스크립트 실행](execute-r-script.md) 모듈을 사용합니다. 

## <a name="how-to-configure-filter-based-feature-selection"></a>필터 기반 피쳐 선택을 구성하는 방법

표준 통계 메트릭을 선택합니다. 모듈은 한 쌍의 열(레이블 열과 피쳐 열) 간의 상관 관계를 계산합니다.

1.  파이프라인에 필터 기반 기능 선택 모듈을 추가합니다. 디자이너의 **기능 선택** 범주에서 찾을 수 있습니다.

2. 잠재적인 기능인 두 개 이상의 열을 포함하는 입력 데이터 집합을 연결합니다.  

    열을 분석하고 기능 점수가 생성되도록 하려면 [메타데이터 편집](edit-metadata.md) 모듈을 사용하여 **IsFeature** 특성을 설정합니다. 

    > [!IMPORTANT]
    > 입력으로 제공하는 열이 잠재적인 기능인지 확인합니다. 예를 들어 단일 값을 포함하는 열에는 정보 값이 없습니다.
    >
    > 일부 열이 잘못된 피처를 만드는 경우 열 선택에서 제거할 수 있습니다. [메타데이터 편집](edit-metadata.md) 모듈을 사용하여 **범주형**으로 플래그를 표시할 수도 있습니다. 
3.  **기능 점수 매기기 방법의**경우 점수를 계산하는 데 사용할 다음 설정된 통계 방법 중 하나를 선택합니다.  

    | 방법              | 요구 사항                             |
    | ------------------- | ---------------------------------------- |
    | 피어슨 상관 관계 | 레이블은 텍스트 또는 숫자일 수 있습니다. 피처는 숫자여야 합니다. |
    치 제곱| 레이블 및 피쳐는 텍스트 또는 숫자일 수 있습니다. 두 범주형 열에 대한 피쳐 중요도를 계산하려면 이 방법을 사용합니다.|

    > [!TIP]
    > 선택한 메트릭을 변경하면 다른 모든 선택 항목이 재설정됩니다. 따라서 먼저 이 옵션을 설정해야 합니다.
4.  **피처 열에서만 작동** 옵션을 선택하여 이전에 피처로 표시된 열에 대해서만 점수를 생성합니다. 

    이 옵션을 선택 취소하면 모듈은 **원하는 피처 수에**지정된 열 수까지 조건을 충족하는 열에 대한 점수를 만듭니다.  

5.  **대상 열의**경우 **시작 열 선택기를** 선택하여 이름 또는 인덱스별로 레이블 열을 선택합니다. (인덱스는 한 기초입니다.)  
    통계적 상관 관계를 포함하는 모든 메서드에는 레이블 열이 필요합니다. 레이블 열이나 여러 레이블 열을 선택하지 않으면 모듈에서 디자인 타임 오류를 반환합니다. 

6.  **원하는 피처 수의**경우 결과로 반환할 피쳐 열 수를 입력합니다.  

    - 지정할 수 있는 최소 피처 수는 하나이지만 이 값을 늘리는 것이 좋습니다.  

    - 원하는 기능의 수를 데이터 집합의 열 수보다 더 많이 지정하면 모든 기능이 반환됩니다. 점수가 0인 피처도 반환됩니다.  

    - 피처 열보다 적은 수의 결과 열을 지정하면 피처의 순위가 내림차순 점수로 지정됩니다. 상위 피쳐만 반환됩니다. 

7.  파이프라인을 제출하거나 필터 기반 피처 선택 모듈을 선택한 다음 **선택된 실행을 선택합니다.**


## <a name="results"></a>결과

처리가 완료된 후:

+ 분석된 피처 열과 해당 점수의 전체 목록을 보려면 모듈을 마우스 오른쪽 단추로 클릭하고 **시각화를**선택합니다.  

+ 기능 선택 기준에 따라 데이터 집합을 보려면 모듈을 마우스 오른쪽 단추로 클릭하고 **시각화를**선택합니다. 

데이터 집합에 예상보다 적은 열이 포함된 경우 모듈 설정을 확인합니다. 또한 입력으로 제공되는 열의 데이터 형식도 확인합니다. 예를 들어 원하는 **피처 수를** 1로 설정하면 출력 데이터 집합에는 레이블 열과 순위가 가장 높은 피쳐 열이라는 두 개의 열만 포함됩니다.


##  <a name="technical-notes"></a>기술 정보  

### <a name="implementation-details"></a>구현 세부 정보

숫자 피쳐와 범주형 레이블에서 Pearson 상관 관계를 사용하는 경우 피처 점수는 다음과 같이 계산됩니다.  

1.  범주형 열의 각 수준에 대해 숫자 열의 조건부 평균을 계산합니다.  

2.  조건부 평균의 열을 숫자 열과 상관 관계를 지정합니다.  

### <a name="requirements"></a>요구 사항  

-   **레이블** 또는 **점수** 열로 지정된 열에는 피처 선택 점수를 생성할 수 없습니다.  

-   메서드가 지원하지 않는 데이터 형식의 열과 함께 점수 매기기 메서드를 사용하려고 하면 모듈에서 오류가 발생합니다. 또는 0 점수가 열에 할당됩니다.  

-   열에 논리적(true/false) 값이 포함된 경우 `True = 1` `False = 0`및 로 처리됩니다.  

-   열이 **레이블** 또는 **점수로**지정된 경우 열이 피쳐가 될 수 없습니다.  

### <a name="how-missing-values-are-handled"></a>누락된 값을 처리하는 방법  

-   누락된 값이 모두 있는 열은 대상(레이블) 열로 지정할 수 없습니다.  

-   열에 누락된 값이 포함된 경우 모듈은 열의 점수를 계산할 때 이를 무시합니다.  

-   피처 열로 지정된 열에 누락된 값이 모두 있는 경우 모듈은 0점을 할당합니다.
