# <a name="convert-to-indicator-values"></a>표시기 값으로 변환
이 문서에서는 Azure 기계 학습 디자이너의 모듈에 대해 설명합니다.

Azure 기계 학습 디자이너의 **표시값 변환 모듈을** 사용하여 범주형 값이 포함된 열을 일련의 이진 표시기 열로 변환합니다.  

또한 이 모듈은 표시기 값으로 변환하는 데 사용되는 변환의 정의를 출력합니다. [변환 적용](apply-transformation.md) 모듈을 사용하여 동일한 스키마가 있는 다른 데이터 집합에서 이 변환을 다시 사용할 수 있습니다.

## <a name="how-to-configure-convert-to-indicator-values"></a>표시값으로 변환을 구성하는 방법

1.  **표시기 값으로 변환을** 찾아 파이프라인 초안으로 드래그합니다. 이 모듈은 **데이터 변환** 범주에서 찾을 수 있습니다.
    > [!NOTE]
    > **인디케이터 값으로 변환 모듈** 전에 [메타데이터 편집](edit-metadata.md) 모듈을 사용하여 대상 열을 범주형으로 표시할 수 있습니다.

1. 변환 할 열을 포함하는 데이터 **집합에 표시값 변환** 모듈을 연결합니다. 

1. **열 편집을** 선택하여 하나 이상의 범주형 열을 선택합니다.

1. 새 부울 **열만** 출력하려면 **범주형 열 덮어쓰기** 옵션을 선택합니다. 이 옵션은 기본적으로 해제되어 있습니다.
    

    > [!TIP]
    >  덮어쓰기 옵션을 선택하면 소스 열이 실제로 삭제또는 수정되지 않습니다. 대신 새 열이 생성되어 출력 데이터 집합에 표시되고 원본 열은 작업 영역에서 사용할 수 있습니다. 원본 데이터를 확인해야 하는 경우 언제든지 [열 추가](add-columns.md) 모듈을 사용하여 원본 열을 다시 추가할 수 있습니다.

1. 파이프라인을 제출합니다.

## <a name="results"></a>결과

서버의 고장 확률이 높거나 중간인지 또는 낮은 지 여부를 나타내는 점수가 있는 열이 있다고 가정합니다.  

| 서버 ID입니다. | 실패 점수 |
| --------- | ------------- |
| 10301     | 낮음           |
| 10302     | 중간        |
| 10303     | 높음          |

**표시값으로 변환을**적용하면 디자이너는 레이블의 단일 열을 부울 값을 포함하는 여러 열로 변환합니다.  

| 서버 ID입니다. | 실패 점수 - 낮음 | 실패 점수 - 중간 | 실패 점수 - 높음 |
| --------- | ------------------- | ---------------------- | -------------------- |
| 10301     | 1                   | 0                      | 0                    |
| 10302     | 0                   | 1                      | 0                    |
| 10303     | 0                   | 0                      | 1                    |

변환의 작동 방식은 다음과 같습니다.  

-   위험을 설명하는 **실패 점수** 열에는 세 가지 가능한 값(높음, 보통 및 낮음)만 있고 누락된 값은 없습니다. 따라서 정확히 세 개의 새 열이 만들어집니다.  

-   새 표시기 열은 이 패턴을 사용하여 원본 열의 열 머리글과 값을 기반으로 명명됩니다 * \<>>. \< *  

-   각 서버는 하나의 위험 등급만 가질 수 있기 때문에 정확히 하나의 표시기 열에 1이 있고 다른 모든 표시기 열에는 0이 있어야 합니다.  

이제 세 개의 표시기 열을 기계 학습 모델의 기능으로 사용할 수 있습니다.

모듈은 두 개의 출력을 반환합니다.

- **결과 데이터 집합**: 변환된 표시기 값 열이 있는 데이터 집합입니다. 청소를 위해 선택되지 않은 열도 "통과"됩니다.
- **표시기 값 변환**: 작업 영역에 저장하고 나중에 새 데이터에 적용할 수 있는 표시기 값으로 변환하는 데 사용되는 데이터 변환입니다.

## <a name="apply-a-saved-indicator-values-operation-to-new-data"></a>저장된 표시기 값 작업을 새 데이터에 적용

표시기 값 작업을 자주 반복해야 하는 경우 데이터 조작 단계를 *변환으로* 저장하여 동일한 데이터 집합으로 다시 사용할 수 있습니다. 이 기능은 동일한 스키마가 있는 데이터를 자주 다시 가져온 다음 정리해야 하는 경우에 유용합니다.

1. 파이프라인에 [변환 적용](apply-transformation.md) 모듈을 추가합니다.

1. 정리할 데이터 집합을 추가하고 데이터 집합을 오른쪽 입력 포트에 연결합니다.

1. 디자이너의 왼쪽 창에서 **데이터 변환** 그룹을 확장합니다. 저장된 변환을 찾아 파이프라인으로 드래그합니다.

1. 저장된 변환을 [변환 적용의](apply-transformation.md)왼쪽 입력 포트에 연결합니다.

   저장된 변환을 적용하면 변환할 열을 선택할 수 없습니다. 이는 변환이 정의되어 원래 작업에 지정된 데이터 형식에 자동으로 적용되기 때문입니다.

1. 파이프라인을 제출합니다.
 
## <a name="technical-notes"></a>기술 정보  

이 섹션에는 자주 묻는 질문에 대한 구현 세부 정보, 팁 및 답변이 포함되어 있습니다.

### <a name="usage-tips"></a>사용 팁

-   범주로 표시된 열만 표시기 열로 변환할 수 있습니다. 다음 오류가 표시되면 선택한 열 중 하나가 범주화되지 않은 것일 수 있습니다.  

     Error 0056: 이름 \<열 이름이> 있는 열이 허용된 범주에 속하지 않습니다.  

     기본적으로 대부분의 문자열 열은 문자열 기능으로 처리되므로 [메타데이터 편집을](edit-metadata.md)사용하여 범주형으로 명시적으로 표시해야 합니다.  

-   표시기 열로 변환할 수 있는 열 수에는 제한이 없습니다. 그러나 각 값 열은 여러 표시기 열을 생성할 수 있으므로 한 번에 몇 개의 열만 변환하고 검토할 수 있습니다.  

-   열에 누락된 값이 포함된 경우 이 이름으로 누락된 범주에 대해 별도의 표시기 열이 만들어집니다 * \<>.*  

-   표시기 값으로 변환하는 열에 숫자가 포함된 경우 다른 피쳐 열과 마찬가지로 범주형으로 표시해야 합니다. 이렇게 하면 숫자는 불연속 값으로 처리됩니다. 예를 들어 MPG 값이 25에서 30에 이르는 숫자 열이 있는 경우 각 불연속 값에 대해 새 표시기 열이 만들어집니다.  

    | 계정을       | 고속도로 MPG -25 | 고속도로 MPG -26 | 고속도로 MPG -27 | 고속도로 MPG -28 | 고속도로 MPG -29 | 고속도로 MPG -30 |
    | ---------- | --------------- | --------------- | --------------- | --------------- | --------------- | --------------- |
    | 콘토소 자동차 | 0               | 0               | 0               | 0               | 0               | 1               |

- 데이터 집합에 치수를 너무 많이 추가하지 않도록 합니다. 먼저 열의 값 수를 확인하고 데이터를 적절히 bin 또는 양자화하는 것이 좋습니다.
