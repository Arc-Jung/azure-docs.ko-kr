# <a name="evaluate-recommender"></a>추천 평가

이 문서에서는 Azure 기계 학습 디자이너(미리 보기)에서 추천자 평가 모듈을 사용하는 방법을 설명합니다. 목표는 권장 모델에서 만든 예측의 정확도를 측정하는 것입니다. 이 모듈을 사용하여 다양한 권장 사항을 평가할 수 있습니다.  
  
-   사용자 및 항목에 대해 예측된 평점    
-   사용자에게 권장되는 항목  
  
권장 사항 모델을 사용하여 예측을 만들 면 지원되는 각 예측 유형에 대해 약간 다른 결과가 반환됩니다. 추천 평가 모듈은 점수가 매겨진 데이터 집합의 열 형식에서 예측 의 종류를 추론합니다. 예를 들어, 점수가 매겨지는 데이터 집합에는 다음이 포함될 수 있습니다.

- 사용자 항목 등급 트리플
- 사용자 및 권장 항목

또한 모듈은 수행되는 예측 유형에 따라 적절한 성능 메트릭을 적용합니다. 

  
## <a name="how-to-configure-evaluate-recommender"></a>추천 평가 를 구성하는 방법

추천 자 평가 모듈은 권장 사항 모델을 사용하여 예측 출력을 해당 "접지 진실" 데이터와 비교합니다. 예를 들어 [점수 SVD 추천자](score-svd-recommender.md) 모듈은 추천평가를 사용하여 분석할 수 있는 점수가 매겨지는 데이터 집합을 생성합니다.

### <a name="requirements"></a>요구 사항

추천 평가에는 다음 데이터 집합이 입력으로 필요합니다. 
  
#### <a name="test-dataset"></a>테스트 데이터 세트

테스트 데이터 집합에는 사용자 항목 등급 트리플 형식의 "접지 진실" 데이터가 포함되어 있습니다.  

#### <a name="scored-dataset"></a>점수가 매겨진 데이터 집합입니다.

점수가 매겨짐에 따라 위험 관리 모델에는 권장 모델에서 생성된 예측이 포함됩니다.  
  
이 두 번째 데이터 집합의 열은 채점 프로세스 중에 수행한 예측 의 종류에 따라 달라집니다. 예를 들어, 점수가 매겨지는 데이터 집합에는 다음 중 하나가 포함될 수 있습니다.

- 사용자, 항목 및 사용자가 항목에 대해 제공할 가능성이 있는 등급
- 사용자에게 권장되는 사용자 및 항목 목록 

### <a name="metrics"></a>메트릭

모델의 성능 메트릭은 입력 유형에 따라 생성됩니다. 다음 섹션에서는 자세히 설명합니다.

## <a name="evaluate-predicted-ratings"></a>예측 등급 평가  

예측 된 등급을 평가할 때 점수가 매겨진 데이터 집합(추천 평가에 대한 두 번째 입력)에는 다음 요구 사항을 충족하는 사용자 항목 등급 트리플이 포함되어야 합니다.
  
-   데이터 집합의 첫 번째 열에는 사용자 식별자가 포함되어 있습니다.    
-   두 번째 열에는 항목 식별자가 포함되어 있습니다.  
-   세 번째 열에는 해당 사용자 항목 등급이 포함되어 있습니다.  
  
> [!IMPORTANT] 
> 평가가 성공하려면 열 이름은 `User`각각 `Item`" `Rating`및 및 의 열 이름이어야 합니다.  
  
추천 평가는 "근거 리 지" 데이터 집합의 등급을 점수가 매겨진 데이터 집합의 예측 된 등급과 비교합니다. 그런 다음 평균 절대 오차(MAE)와 루트 평균 제곱 오차(RMSE)를 계산합니다.



## <a name="evaluate-item-recommendations"></a>항목 권장 사항 평가

항목 권장 사항을 평가할 때 각 사용자에 대해 권장되는 항목이 포함된 점수가 매겨짐에 따라 데이터 집합을 사용합니다.
  
-   데이터 집합의 첫 번째 열에는 사용자 식별자가 포함되어야 합니다.    
-   이후의 모든 열에는 항목이 사용자에게 얼마나 관련성이 있는지에 따라 정렬된 해당 권장 항목 식별자가 포함되어야 합니다. 

이 데이터 집합을 연결하기 전에 가장 관련성이 가장 중요한 항목이 먼저 오게 되도록 데이터 집합을 정렬하는 것이 좋습니다.  

> [!IMPORTANT] 
> 추천 평가자가 작동하려면 열 이름이 `User` `Item 1`". `Item 2` `Item 3`  
  
추천 평가는 평균 정규화된 할인 누적 이득(NDCG)을 계산하고 출력 데이터 집합에서 반환합니다.  
  
권장 항목에 대한 실제 "근거 진실"을 알 수 없기 때문에 평가 추천자는 테스트 데이터 집합의 사용자 항목 등급을 NDCG 계산의 이득으로 사용합니다. 평가하려면 추천자 채점 모듈은 테스트 데이터 집합에서 "접지 진실" 등급이 있는 항목에 대한 권장 사항만 생성해야 합니다.
