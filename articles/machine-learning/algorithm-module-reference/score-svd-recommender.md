# <a name="score-svd-recommender"></a>SVD 추천 점수 매기기

이 문서에서는 Azure 기계 학습 디자이너(미리 보기)에서 SVD 권장 점수 모듈을 사용하는 방법을 설명합니다. 이 모듈을 사용하여 단일 값 분해(SVD) 알고리즘을 기반으로 학습된 권장 사항 모델을 사용하여 예측을 만듭니다.

SVD 권장자는 두 가지 종류의 예측을 생성할 수 있습니다.

- [지정된 사용자와 항목의 등급 예측](#prediction-of-ratings)
- [사용자에게 항목을 추천합니다.](#recommendations-for-users)

두 번째 유형의 예측을 만들 때 다음 모드 중 하나에서 작동할 수 있습니다.

- **프로덕션 모드는** 모든 사용자 또는 항목을 고려합니다. 일반적으로 웹 서비스에서 사용됩니다.

  교육 중에 본 사용자뿐만 아니라 신규 사용자에 대한 점수를 만들 수 있습니다. 자세한 내용은 기술 [노트를](#technical-notes)참조하십시오. 

- **평가 모드는** 평가할 수 있는 감소된 사용자 또는 항목 집합에서 작동합니다. 일반적으로 파이프라인 작업 중에 사용됩니다.

SVD 추천 알고리즘에 대한 자세한 내용은 추천 시스템에 대한 연구 용지 [매트릭스 분해 기술을](https://datajobs.com/data-science-repo/Recommender-Systems-[Netflix].pdf)참조하십시오.

## <a name="how-to-configure-score-svd-recommender"></a>점수 SVD 추천을 구성하는 방법

이 모듈은 서로 다른 요구 사항을 가진 두 가지 유형의 예측을 지원합니다. 

###  <a name="prediction-of-ratings"></a>등급 예측

등급을 예측할 때 모델은 학습 데이터를 고려하여 사용자가 특정 항목에 반응하는 방식을 계산합니다. 점수 매기기 입력 데이터는 사용자와 평가할 항목을 모두 제공해야 합니다.

1. 파이프라인에 학습된 권장 사항 모델을 추가하고 **이를 학습된 SVD 권장 사항에 연결합니다.** [SVD 권장 자 학습모듈을](train-SVD-recommender.md) 사용하여 모델을 만들어야 합니다.

2. **추천 예측 종류를**보려면 등급 **예측을**선택합니다. 다른 매개 변수는 필요하지 않습니다.

3. 예측을 수행할 데이터를 추가하고 **데이터 집합에 연결하여 점수를 매길**수 있습니다.

   모델이 등급을 예측하려면 입력 데이터 집합에 사용자 항목 쌍이 포함되어야 합니다.

   데이터 집합에는 첫 번째 및 두 번째 열의 사용자 항목 쌍에 대한 선택적 세 번째 등급 열이 포함될 수 있습니다. 그러나 세 번째 열은 예측 중에 무시됩니다.

4. 파이프라인을 제출합니다.

### <a name="results-for-rating-predictions"></a>등급 예측에 대한 결과 

출력 데이터 집합에는 사용자, 항목 및 각 입력 사용자 및 항목에 대한 예측 등급의 세 개의 열이 포함됩니다.

###  <a name="recommendations-for-users"></a>사용자를 위한 권장 사항 

사용자에 대 한 항목을 추천 하려면 사용자 및 항목의 목록을 입력으로 제공 합니다. 이 데이터에서 모델은 기존 항목 및 사용자에 대한 지식을 사용하여 각 사용자에게 호소할 가능성이 있는 항목 목록을 생성합니다. 반환되는 권장 사항 수를 사용자 지정할 수 있습니다. 또한 권장 사항을 생성하는 데 필요한 이전 권장 사항 수에 대한 임계값을 설정할 수 있습니다.

1. 파이프라인에 학습된 권장 사항 모델을 추가하고 **이를 학습된 SVD 권장 사항에 연결합니다.**  [SVD 권장 자 학습모듈을](train-svd-recommender.md) 사용하여 모델을 만들어야 합니다.

2. 사용자 목록에 대한 항목을 추천하려면 **추천 예측 종류를** 항목 권장 **사항으로**설정합니다.

3. **권장 항목 선택의**경우 프로덕션 환경에서 점수 매기기 모듈을 사용하고 있는지 또는 모델 평가를 위해 사용하고 있는지 를 나타냅니다. 다음 값 중 하나를 선택합니다.

    - **모든 항목에서**: 웹 서비스 또는 프로덕션 환경에서 사용할 파이프라인을 설정하는 경우 이 옵션을 선택합니다.  이 옵션을 사용하면 프로덕션 모드를 사용할 수 *있습니다.* 이 모듈은 교육 중에 보이는 모든 항목의 권장 사항을 제공합니다.

    - **정격 항목에서(모델 평가용)**: 모델을 개발하거나 테스트하는 경우 이 옵션을 선택합니다. 이 옵션을 사용하면 평가 모드를 사용할 수 *있습니다.* 이 모듈은 정격된 입력 데이터 집합의 해당 항목에서만 권장 사항을 만듭니다.
    
    - **등급이 지정되지 않은 항목에서 (사용자에게 새 항목을 제안하기 위해)**: 모듈에서 평가되지 않은 학습 데이터 집합의 해당 항목에서만 권장 사항을 만들려면 이 옵션을 선택합니다. 

4. 예측을 수행할 데이터 집합을 추가하고 데이터 **집합에 연결하여 점수를 매길**수 있습니다.

    - **모든 항목에서**입력 데이터 집합은 하나의 열로 구성되어야 합니다. 여기에는 권장 사항을 만들 사용자의 식별자가 포함되어 있습니다.

      데이터 집합에는 항목 식별자 및 등급의 추가 두 열이 포함될 수 있지만 이 두 열은 무시됩니다. 

    - **정격 항목(모델 평가용)의**경우 입력 데이터 집합은 사용자 항목 쌍으로 구성되어야 합니다. 첫 번째 열에는 사용자 식별자가 포함되어야 합니다. 두 번째 열에는 해당 항목 식별자가 포함되어야 합니다.

      데이터 집합에는 사용자 항목 등급의 세 번째 열이 포함될 수 있지만 이 열은 무시됩니다.

    - **등급이 정격되지 않은 항목(사용자에게 새 항목을 제안하기 위해)의**경우 입력 데이터 집합은 사용자 항목 쌍으로 구성되어야 합니다. 첫 번째 열에는 사용자 식별자가 포함되어야 합니다. 두 번째 열에는 해당 항목 식별자가 포함되어야 합니다.

     데이터 집합에는 사용자 항목 등급의 세 번째 열이 포함될 수 있지만 이 열은 무시됩니다.

5. **사용자에게 추천할 항목의 최대 수**: 각 사용자에 대해 반환할 항목 수를 입력합니다. 기본적으로 모듈은 다섯 가지 항목을 권장합니다.

6. **사용자당 권장 권 풀의 최소 크기**: 필요한 이전 권장 사항 수를 나타내는 값을 입력합니다. 기본적으로 이 매개 변수는 2로 설정되어 있으므로 적어도 두 명의 다른 사용자가 해당 항목을 권장했습니다.

   평가 모드에서 점수를 매기는 경우에만 이 옵션을 사용합니다. **모든 항목에서** 또는 **등급이 지정되지 않은 항목에서 (사용자에게 새 항목을 제안하기 위해)** 옵션을 선택하는 경우 이 옵션을 사용할 수 없습니다.

7.  **등급이 지정되지 않은 항목(사용자에게 새 항목을 제안하기 위해)의**경우 세 번째 입력 포트인 **교육 데이터(Training Data)를**사용하여 예측 결과에서 이미 등급이 지정된 항목을 제거합니다.

    이 필터를 적용하려면 원래 학습 데이터 집합을 입력 포트에 연결합니다.

8. 파이프라인을 제출합니다.

### <a name="results-of-item-recommendation"></a>항목 추천 결과

점수 SVD 추천에 의해 반환 된 점수 데이터 집합각 사용자에 대 한 권장 된 항목을 나열 합니다.

- 첫 번째 열에는 사용자 식별자가 포함되어 있습니다.
- **사용자에게 추천할 최대 항목 수에**대해 설정한 값에 따라 여러 개의 추가 열이 생성됩니다. 각 열에는 식별자별로 권장항목이 포함되어 있습니다. 권장 사항은 사용자 항목 선호도에 따라 정렬됩니다. 선호도가 가장 높은 항목은 항목 **항목 1**열에 넣습니다.

> [!WARNING]
> [추천자 평가](evaluate-recommender.md) 모듈을 사용하여 이 점수가 매겨지지 않은 데이터 집합을 평가할 수 없습니다.


##  <a name="technical-notes"></a>기술 정보

SVD 권장자가 있는 파이프라인이 있고 모델을 프로덕션으로 이동하는 경우 평가 모드에서 추천을 사용하는 것과 프로덕션 모드에서 사용하는 것 사이에는 주요 차이점이 있다는 점에 유의하십시오.

평가 모드에서는 기본적으로 테스트 집합의 *실제 정보*와 대조하여 확인할 수 있는 예측이 필요합니다. 추천자를 평가할 때는 테스트 집합에서 등급이 정된 항목만 예측해야 합니다. 이렇게 하면 예측가능한 값이 제한됩니다.

모델을 운영할 때 일반적으로 예측 모드를 변경하여 최상의 예측을 얻기 위해 가능한 모든 항목을 기반으로 권장 사항을 만듭니다. 이러한 예측의 대부분을 들어, 해당 지상 진실이 없다. 따라서 권장 사항의 정확성은 파이프라인 작업 중과 동일한 방식으로 확인할 수 없습니다.
