# <a name="import-data-module"></a>데이터 가져오기 모듈

이 문서에서는 Azure 기계 학습 디자이너(미리 보기)의 모듈에 대해 설명합니다.

이 모듈을 사용하여 기존 클라우드 데이터 서비스에서 기계 학습 파이프라인에 데이터를 로드합니다. 

> [!Note]
> 이 모듈에서 제공하는 모든 기능은 worksapce 방문 페이지의 **데이터 스토어** 및 **데이터 집합에서** 수행할 수 있습니다. 데이터 모니터링과 같은 추가 기능이 포함된 **데이터스토어** 및 **데이터 집합을** 사용하는 것이 좋습니다. 자세한 내용은 [데이터에 액세스하는 방법](../how-to-access-data.md) 및 데이터 집합 을 [등록하는 방법을](../how-to-create-register-datasets.md) 참조하세요.
> 데이터 집합을 등록한 후 디자이너 인터페이스의 **데이터 집합** -> **내 데이터 집합** 범주에서 찾을 수 있습니다. 이 모듈은 Studio(클래식) 사용자가 익숙한 환경을 위해 예약되어 있습니다. 
>

**데이터 가져오기** 모듈은 다음 소스의 데이터 읽기를 지원합니다.

- HTTP를 통한 URL
- [**데이터 스토어를**](../how-to-access-data.md)통한 Azure 클라우드 스토리지)
    - Azure Blob 컨테이너
    - Azure 파일 공유
    - Azure 데이터 레이크
    - Azure Data Lake Gen2
    - Azure SQL Database
    - 저어 포스트그레SQL    

클라우드 저장소를 사용하기 전에 먼저 Azure 기계 학습 작업 영역에 데이터 스토어를 등록해야 합니다. 자세한 내용은 [데이터에 액세스하는 방법을](../how-to-access-data.md)참조하십시오. 

원하는 데이터를 정의하고 원본에 연결하면 **[Data 가져오기가](./import-data.md)** 포함된 값을 기반으로 각 열의 데이터 형식을 유추하고 데이터를 디자이너 파이프라인에 로드합니다. **가져오기 데이터** 출력은 모든 디자이너 파이프라인에서 사용할 수 있는 데이터 집합입니다.

원본 데이터가 변경되면 데이터 가져오기 [를](./import-data.md)다시 실행하여 데이터 집합을 새로 고치고 새 데이터를 추가할 수 있습니다.

## <a name="how-to-configure-import-data"></a>가져오기 데이터 구성 하는 방법

1. 파이프라인에 **데이터 가져오기** 모듈을 추가합니다. 이 모듈은 디자이너의 **데이터 입력 및 출력** 범주에서 찾을 수 있습니다.

1. 오른쪽 창을 열려면 모듈을 선택합니다.

1. **데이터 원본을**선택하고 데이터 원본 유형을 선택합니다. HTTP 또는 데이터 스토어일 수 있습니다.

    데이터 스토어를 선택하는 경우 Azure Machine Learning 작업 영역에 이미 등록된 기존 데이터 스토어를 선택하거나 새 데이터 스토어를 만들 수 있습니다. 그런 다음 데이터 스토어에서 가져올 데이터 경로를 정의합니다. 경로 가져오기-데이터 경로 **찾아보기를** ![클릭하여 경로를 쉽게 탐색할 수 있습니다.](media/module/import-data-path.png)

1. 포함할 열을 필터링하려면 미리 보기 스키마를 선택합니다. 구문 분석 옵션에서 구분 기호와 같은 고급 설정을 정의할 수도 있습니다.

    ![가져오기 데이터 미리 보기](media/module/import-data.png)

1. **출력**재생확인란, 실행 시간에 출력을 재생성할 모듈을 실행할지 여부를 결정합니다. 

    기본적으로 선택되지 않은 모듈이 이전에 동일한 매개 변수로 실행된 경우 시스템은 마지막 실행의 출력을 재사용하여 런타임을 줄입니다. 

    이 옵션을 선택하면 시스템이 모듈을 다시 실행하여 출력을 재생성합니다. 따라서 저장소의 기본 데이터가 업데이트될 때 이 옵션을 선택하면 최신 데이터를 얻는 데 도움이 될 수 있습니다.


1. 파이프라인을 제출합니다.

    데이터 가져오기가 디자이너에 데이터를 로드하면 데이터가 포함된 값(숫자 또는 범주형)을 기반으로 각 열의 데이터 형식을 유추합니다.

    헤더가 있으면 헤더를 사용하여 출력 데이터 집합의 열 이름을 지정합니다.

    데이터에 기존 열 헤더가 없는 경우 col1 형식 col2를 사용하여 새 열 이름이 생성됩니다,... , 콜른*.

## <a name="results"></a>결과

가져오기가 완료되면 출력 데이터 집합을 클릭하고 **시각화를** 선택하여 데이터를 성공적으로 가져왔는지 확인합니다.

파이프라인이 실행될 때마다 새 데이터 집합을 가져오는 대신 재사용할 데이터를 저장하려면 모듈의 오른쪽 패널의 출력 탭 아래에 있는 **데이터 집합 등록** **아이콘을** 선택합니다. 데이터 집합의 이름을 선택합니다. 저장된 데이터 집합은 저장 시 데이터를 보존하며 파이프라인의 데이터 집합이 변경되더라도 파이프라인이 다시 실행될 때 데이터 집합이 업데이트되지 않습니다. 이 기능은 데이터의 스냅숏을 만들 때 유용할 수 있습니다.

데이터를 가져온 후 모델링 및 분석을 위한 몇 가지 추가 준비가 필요할 수 있습니다.

- [메타데이터 편집을](./edit-metadata.md) 사용하여 열 이름을 변경하거나, 열을 다른 데이터 유형으로 처리하거나, 일부 열이 레이블 또는 피처임을 나타냅니다.

- [데이터 집합에서 열 선택을](./select-columns-in-dataset.md) 사용하여 변환하거나 모델링에 사용할 열의 하위 집합을 선택합니다. 변환되거나 제거된 열은 열 추가 모듈을 사용하여 원래 데이터 집합에 쉽게 다시 가입할 수 [있습니다.](./add-columns.md)  

- [파티션 및 샘플을](./partition-and-sample.md) 사용하여 데이터 집합을 분할하거나 샘플링을 수행하거나 상위 n행을 가져옵니다.
