# <a name="permutation-feature-importance"></a>순열 기능 중요도

이 문서에서는 Azure 기계 학습 디자이너(미리 보기)에서 순열 기능 중요도 모듈을 사용하여 데이터 집합에 대한 기능 중요도 점수 집합을 계산하는 방법을 설명합니다. 이러한 점수를 사용하여 모델에서 사용할 최상의 기능을 결정할 수 있습니다.

이 단원에서 기능 값은 한 번에 하나의 열로 임의로 섞입니다. 모델의 성능은 전후로 측정됩니다. 성능을 측정하기 위해 표준 메트릭 중 하나를 선택할 수 있습니다.

모듈이 반환하는 점수는 순열 후 학습된 모델의 성능 *변화를* 나타냅니다. 중요한 기능은 일반적으로 셔플 프로세스에 더 민감하므로 중요도 점수가 높아집니다. 

이 문서에서는 순열 기능, 이론적 기초 및 기계 학습의 응용 프로그램인 [순열 기능 중요도에](https://blogs.technet.com/b/machinelearning/archive/2015/04/14/permutation-feature-importance.aspx)대한 개요를 제공합니다.  

## <a name="how-to-use-permutation-feature-importance"></a>순열 피쳐 중요도 를 사용하는 방법

기능 점수 집합을 생성하려면 테스트 데이터 집합뿐만 아니라 이미 학습된 모델이 있어야 합니다.  

1.  파이프라인에 순열 피쳐 중요도 모듈을 추가합니다. 이 모듈은 기능 **선택** 범주에서 찾을 수 있습니다. 

2.  학습된 모델을 왼쪽 입력에 연결합니다. 모델은 회귀 모델 또는 분류 모델이어야 합니다.  

3.  오른쪽 입력에서 데이터 집합을 연결합니다. 바람직하게는 모델을 학습하는 데 사용한 데이터 집합과 다른 데이터 집합을 선택합니다. 이 데이터 집합은 학습된 모델을 기반으로 채점하는 데 사용됩니다. 피쳐 값이 변경된 후 모델을 평가하는 데도 사용됩니다.  

4.  **임의 시드의**경우 무작위화를 위해 시드로 사용할 값을 입력합니다. 0(기본값)을 지정하면 시스템 시계에 따라 숫자가 생성됩니다.

     시드 값은 선택 사항이지만 동일한 파이프라인의 실행에서 재현성을 원하는 경우 값을 제공해야 합니다.  

5.  **성능 측정을 위한 메트릭의**경우 순열 후 모델 품질을 계산할 때 사용할 단일 메트릭을 선택합니다.  

     Azure Machine Learning 디자이너는 분류 또는 회귀 모델을 평가하는지 여부에 따라 다음 메트릭을 지원합니다.  

    -   **분류**

        정확도, 정밀도, 리콜  

    -   **회귀**

        정밀도, 회수, 절대 오차 평균, 루트 평균 제곱 오차, 상대 절대 오차, 상대 제곱 오차, 결정 계수  

     이러한 평가 메트릭및 계산 방법에 대한 자세한 설명은 [모델 평가를](evaluate-model.md)참조하십시오.  

6.  파이프라인을 제출합니다.  

7.  모듈은 피처 열 목록과 연관된 점수를 출력합니다. 목록은 점수의 내림차순으로 순위가 매겨지며 순위가 매겨지며 순위가 매겨지다.  


##  <a name="technical-notes"></a>기술 정보

순열 특징 중요도는 한 번에 하나의 열인 각 피처 열의 값을 임의로 변경하여 작동합니다. 그런 다음 모델을 평가합니다. 

모듈에서 제공하는 순위는 [필터 기반 기능 선택에서](filter-based-feature-selection.md)얻은 순위와 다른 경우가 많습니다. 필터 기반 피쳐 선택은 모델을 *작성하기 전에* 점수를 계산합니다. 

그 이유는 순열 특징 중요도가 피쳐와 대상 값 간의 연결을 측정하지 않기 때문입니다. 대신 각 기능이 모델의 예측에 미치는 영향을 캡처합니다.
