# <a name="convert-to-dataset"></a>데이터 세트로 변환

이 문서에서는 Azure 기계 학습 디자이너(미리 보기)에서 데이터 집합으로 변환 모듈을 사용하여 파이프라인에 대한 모든 데이터를 디자이너의 내부 형식으로 변환하는 방법을 설명합니다.
  
대부분의 경우 변환이 필요하지 않습니다. Azure Machine Learning은 데이터에 대한 작업이 수행될 때 데이터를 기본 데이터 집합 형식으로 암시적으로 변환합니다. 

데이터 집합에서 일종의 정규화 또는 정리를 수행하고 변경 내용이 다른 파이프라인에서 사용되는지 확인하려는 경우 데이터를 데이터 집합 형식으로 저장하는 것이 좋습니다.  
  
> [!NOTE]
> 데이터 집합으로 변환하려면 데이터의 형식만 변경됩니다. 작업 영역에 데이터의 새 복사본을 저장하지 않습니다. 데이터 집합을 저장하려면 출력 포트를 두 번 클릭하고 **데이터 집합으로 저장을**선택하고 새 이름을 입력합니다.  
  
## <a name="how-to-use-convert-to-dataset"></a>데이터 집합으로 변환 사용 방법  

데이터 집합으로 변환을 사용하기 전에 [메타데이터 편집](edit-metadata.md) 모듈을 사용하여 데이터 집합을 준비하는 것이 좋습니다. 열 이름을 추가 하거나 변경, 데이터 형식을 조정 하 고 필요에 따라 다른 변경 할 수 있습니다.

1.  파이프라인에 데이터 집합으로 변환 모듈을 추가합니다. 이 모듈은 디자이너의 **데이터 변환** 범주에서 찾을 수 있습니다. 

2. 데이터 집합을 출력하는 모든 모듈에 연결합니다.   

    데이터가 [테이블 형식인](https://docs.microsoft.com/python/api/azureml-core/azureml.data.tabulardataset?view=azure-ml-py)한 데이터 집합으로 변환할 수 있습니다. 여기에는 데이터 [가져오기를](import-data.md)통해 로드된 데이터, [수동으로 데이터 입력을](enter-data-manually.md)통해 생성된 데이터 또는 [변환 적용을](apply-transformation.md)통해 변환된 데이터 집합이 포함됩니다.

3.  **작업** 드롭다운 목록에서 데이터 집합을 저장하기 전에 데이터에 대한 정리를 수행할지 지정할지 지정합니다.  
  
    - **없음**: 데이터를 있는 대로 사용합니다.  
  
    - **SetMissingValue**: 데이터 집합에서 누락된 값으로 특정 값을 설정합니다. 기본 자리 표시자는 물음표 문자(?)이지만 **사용자 지정 누락 값** 옵션을 사용하여 다른 값을 입력할 수 있습니다. 예를 들어 사용자 지정 **누락 값에**대해 **택시를** 입력하면 데이터 집합의 **모든 택시** 인스턴스가 누락된 값으로 변경됩니다.
  
    - **ReplaceValues**: 이 옵션을 사용하여 다른 정확한 값으로 대체할 단일 정확한 값을 지정합니다. **[바꾸기]** 메서드를 설정하여 누락된 값 또는 사용자 지정 값을 바꿀 수 있습니다.

      - **누락**: 입력 데이터 집합에서 누락된 값을 바꾸려면 이 옵션을 선택합니다. **새 값의**경우 누락된 값을 값으로 바꾸기 위해 값을 입력합니다.
      - **사용자 지정**: 입력 데이터 집합에서 사용자 지정 값을 바꾸려면 이 옵션을 선택합니다. **사용자 지정 값의**경우 찾으려는 값을 입력합니다. 예를 들어 데이터에 누락된 `obs` 값의 자리 표시자로 사용되는 문자열이 포함된 경우 을 입력합니다. `obs` **새 값의**경우 원래 문자열을 바꿀 새 값을 입력합니다.
  
    **ReplaceValues** 작업은 정확한 일치에만 적용됩니다. 예를 들어 이러한 문자열은 영향을 `obs.`받지 `obsolete`않습니다.  
 
  
5.  파이프라인을 제출합니다.  

## <a name="results"></a>결과

+  결과 데이터 집합을 새 이름으로 저장하려면 모듈의 오른쪽 패널에서 출력 탭 아래에 있는 **데이터 집합 등록** 아이콘을 **선택합니다.**  
  
## <a name="technical-notes"></a>기술 정보  

-   데이터 집합을 입력으로 하는 모든 모듈은 CSV 파일 또는 TSV 파일에서도 데이터를 사용할 수 있습니다. 모듈 코드를 실행하기 전에 입력이 전처리됩니다. 전처리는 입력에서 데이터 집합으로 변환 모듈을 실행하는 것과 같습니다.  
  
-   SVMLight 형식에서 데이터 집합으로 변환할 수 없습니다.  
  
-   사용자 지정 바꾸기 작업을 지정하는 경우 검색 및 바꾸기 작업이 전체 값에 적용됩니다. 부분 일치는 허용되지 않습니다. 예를 들어 3을 -1 또는 33으로 바꿀 수 있지만 35와 같은 두 자리 숫자에서는 3을 바꿀 수는 없습니다.  
  
-   사용자 지정 바꾸기 작업에서는 열의 현재 데이터 형식을 따르지 않는 문자를 대체 값으로 사용하면 대체가 자동으로 실패합니다.
