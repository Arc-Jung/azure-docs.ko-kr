# <a name="linear-regression-module"></a>선형 회귀 모듈
이 문서에서는 Azure 기계 학습 디자이너(미리 보기)의 모듈에 대해 설명합니다.

이 모듈을 사용하여 파이프라인에 사용할 선형 회귀 모델을 작성합니다.  선형 회귀는 하나 이상의 독립 변수와 숫자 결과 또는 종속 변수 간의 선형 관계를 설정하려고 시도합니다. 

이 모듈을 사용하여 선형 회귀 방법을 정의한 다음 레이블이 지정된 데이터 집합을 사용하여 모델을 학습합니다. 그러면 학습된 모델을 예측에 사용할 수 있습니다.

## <a name="about-linear-regression"></a>선형 회귀 에 대해

선형 회귀는 기계 학습에 채택되어 라인을 피팅하고 오류를 측정하는 많은 새로운 방법으로 강화된 일반적인 통계 방법입니다. 가장 기본적인 의미에서 회귀는 숫자 대상의 예측을 말합니다. 선형 회귀는 기본 예측 작업에 대한 간단한 모델을 원할 때 여전히 좋은 선택입니다. 선형 회귀는 복잡성이 없는 고차원스파스 데이터 세트에서도 잘 작동하는 경향이 있습니다.

Azure 기계 학습은 선형 회귀 외에도 다양한 회귀 모델을 지원합니다. 그러나 "회귀"라는 용어는 느슨하게 해석될 수 있으며 다른 도구에서 제공되는 일부 유형의 회귀는 지원되지 않습니다.

+ 클래식 회귀 문제는 단일 독립 변수와 종속 변수를 포함합니다. 이를 *간단한 회귀라고*합니다.  이 모듈은 간단한 회귀를 지원합니다.

+ *다중 선형 회귀에는* 단일 종속 변수에 기여하는 두 개 이상의 독립 변수가 포함됩니다. 단일 숫자 결과를 예측하는 데 여러 입력이 사용되는 문제를 *다변량 선형 회귀라고도*합니다.

    **선형 회귀** 모듈은 대부분의 다른 회귀 모듈과 마찬가지로 이러한 문제를 해결할 수 있습니다.

+ *다중 레이블 회귀는* 단일 모델 내에서 여러 종속 변수를 예측하는 작업입니다. 예를 들어, 다중 레이블 로지스틱 회귀에서 샘플은 여러 다른 레이블에 할당될 수 있습니다. (단일 클래스 변수 내에서 여러 레벨을 예측하는 작업과 는 다릅니다.)

    이러한 유형의 회귀는 Azure 기계 학습에서 지원되지 않습니다. 여러 변수를 예측하려면 예측하려는 각 출력에 대해 별도의 수강생을 만듭니다.

수년 동안 통계학자들은 회귀를 위한 점점 더 진보된 방법을 개발해 왔습니다. 선형 회귀에서도 마찬가지입니다. 이 모듈은 오류를 측정하고 회귀 선에 맞는 두 가지 방법( 일반 최소 제곱 방법 및 그라데이션 하강)을 지원합니다.

- **그라데이션 하강은** 모델 학습 프로세스의 각 단계에서 오류의 양을 최소화하는 방법입니다. 기울기 하강에는 여러 변형이 있으며, 다양한 학습 문제에 맞게 최적화하기 위해 광범위하게 연구되었습니다. **솔루션 방법에**대해 이 옵션을 선택하는 경우 단계 크기, 학습 속도 등을 제어하는 다양한 매개 변수를 설정할 수 있습니다. 이 옵션은 통합 매개 변수 스윕의 사용도 지원합니다.

- **일반 최소 제곱은** 선형 회귀에서 가장 일반적으로 사용되는 기술 중 하나입니다. 예를 들어, 최소 제곱은 Microsoft Excel의 분석 도구 팩에 사용되는 방법입니다.

    최소자승법은 실제 값에서 예측선까지 거리의 제곱 합으로 오류를 계산하고 제곱 오차를 최소화하여 모델을 적합하게 하는 손실 함수를 나타냅니다. 이 메서드는 입력과 종속 변수 간의 강력한 선형 관계를 가정합니다.

## <a name="configure-linear-regression"></a>선형 회귀 구성

이 모듈은 다른 옵션으로 회귀 모델을 피팅하는 두 가지 방법을 지원합니다.

+ [일반 최소 제곱을 사용하여 회귀 모델에 맞추기](#create-a-regression-model-using-ordinary-least-squares)

    작은 데이터 집합의 경우 일반 최소 제곱을 선택하는 것이 가장 좋습니다. 이것은 Excel과 유사한 결과를 제공해야합니다.
    
+ [온라인 그라데이션 하강을 사용하여 회귀 모델 작성](#create-a-regression-model-using-online-gradient-descent)

    기울기 하강은 더욱 복잡하며 변수의 수에 비해 학습 데이터가 너무 적은 모델에 더욱 적합한 손실 함수입니다.

### <a name="create-a-regression-model-using-ordinary-least-squares"></a>일반 최소 제곱을 사용하여 회귀 모델 작성

1. 선형 **회귀 모델 모듈을** 디자이너의 파이프라인에 추가합니다.

    이 모듈은 기계 **학습** 범주에서 찾을 수 있습니다. **모델 초기화를**확장하고 **회귀를**확장한 다음 **선형 회귀 모델 모듈을** 파이프라인으로 끕니다.

2. **속성** 창에서 **솔루션 메서드** 드롭다운 목록에서 일반 **최소 제곱을 선택합니다.** 이 옵션은 회귀 선을 찾는 데 사용되는 계산 방법을 지정합니다.

3. **L2 정규화 가중치에서 L2 정규화의**가중치로 사용할 값을 입력합니다. 과적합을 방지하려면 0이 아닌 값을 사용하는 것이 좋습니다.

     정규화가 모델 피팅에 미치는 영향에 대한 자세한 내용은 이 문서를 참조하십시오: [기계 학습을 위한 L1 및 L2 정규화](https://msdn.microsoft.com/magazine/dn904675.aspx)

4. 절편에 대한 용어를 보려면 **절전 용어 포함**옵션을 선택합니다.

    회귀 수식을 검토할 필요가 없는 경우 이 옵션을 선택 취소합니다.

5. **난수 시드의**경우 선택적으로 모델에서 사용하는 난수 생성기를 시드하는 값을 입력할 수 있습니다.

    시드 값을 사용하면 동일한 파이프라인의 여러 실행에서 동일한 결과를 유지하려는 경우에 유용합니다. 그렇지 않으면 기본값은 시스템 시계의 값을 사용하는 것입니다.


7. [파이프라인에 모델 학습](./train-model.md) 모듈을 추가하고 레이블이 지정된 데이터 집합을 연결합니다.

8. 파이프라인을 제출합니다.

### <a name="results-for-ordinary-least-squares-model"></a>일반 최소 제곱 모델에 대한 결과

교육이 완료된 후:


+ 예측을 하려면 학습된 모델을 새 값의 데이터 집합과 함께 [모델 점수](./score-model.md) 모듈에 연결합니다. 


### <a name="create-a-regression-model-using-online-gradient-descent"></a>온라인 그라데이션 하강을 사용하여 회귀 모델 작성

1. 선형 **회귀 모델 모듈을** 디자이너의 파이프라인에 추가합니다.

    이 모듈은 기계 **학습** 범주에서 찾을 수 있습니다. **모델 초기화**확장, **회귀**확장 및 **선형 회귀 모델 모듈을** 파이프라인으로 드래그

2. **속성** 창에서 **솔루션 메서드** 드롭다운 목록에서 회귀 선을 찾는 데 사용되는 계산 방법으로 **온라인 그라데이션 하강을** 선택합니다.

3. **트레이너 만들기 모드의**경우 미리 정의된 매개변수 세트로 모델을 학습할지 또는 매개변수 스윕을 사용하여 모델을 최적화할지 여부를 나타냅니다.

    + **단일 매개 변수**: 선형 회귀 네트워크를 구성하는 방법을 알고 있는 경우 특정 값 집합을 인수로 제공할 수 있습니다.
    
    + **매개 변수 범위**: 최상의 매개 변수를 잘 모르고 매개 변수 스윕을 실행하려는 경우 이 옵션을 선택합니다. 반복할 값 범위를 선택하고 [모델 하이퍼매개 변수 조정은](tune-model-hyperparameters.md) 제공한 설정의 가능한 모든 조합을 반복하여 최적의 결과를 생성하는 하이퍼매개 변수를 결정합니다.  

   
4. **학습 속도의**경우 연속 그라데이션 하강 최적화 프로그램에 대한 초기 학습 속도를 지정합니다.

5. **학습 횟수 에포크의**경우 알고리즘이 예제를 통해 반복해야 하는 횟수를 나타내는 값을 입력합니다. 예제 수가 적은 데이터 집합의 경우 이 횟수는 수렴에 도달할 정도로 커야 합니다.

6. **피쳐 정규화**: 모델을 학습하는 데 사용되는 숫자 데이터를 이미 정규화한 경우 이 옵션을 선택 해제할 수 있습니다. 기본적으로 모듈은 모든 숫자 입력을 0에서 1 사이의 범위로 정규화합니다.

    > [!NOTE]
    > 
    > 점수 매기기에 사용되는 새 데이터에 동일한 정규화 방법을 적용해야 합니다.

7. **L2 정규화 가중치에서 L2 정규화의**가중치로 사용할 값을 입력합니다. 과적합을 방지하려면 0이 아닌 값을 사용하는 것이 좋습니다.

    정규화가 모델 피팅에 미치는 영향에 대한 자세한 내용은 이 문서를 참조하십시오: [기계 학습을 위한 L1 및 L2 정규화](https://msdn.microsoft.com/magazine/dn904675.aspx)


9. 반복이 진행됨에 따라 학습 속도를 줄이려면 **학습 속도 감소**옵션을 선택합니다.  

10. **난수 시드의**경우 선택적으로 모델에서 사용하는 난수 생성기를 시드하는 값을 입력할 수 있습니다. 시드 값을 사용하면 동일한 파이프라인의 여러 실행에서 동일한 결과를 유지하려는 경우에 유용합니다.


12. 레이블이 지정된 데이터 집합과 학습 모듈 중 하나를 추가합니다.

    매개변수 스윕을 사용하지 않는 경우 [모델 학습](train-model.md) 모듈을 사용합니다.

13. 파이프라인을 제출합니다.

### <a name="results-for-online-gradient-descent"></a>온라인 그라데이션 하강에 대한 결과

교육이 완료된 후:

+ 예측을 하려면 학습된 모델을 새 입력 데이터와 함께 [모델 점수](./score-model.md) 매기기 모듈에 연결합니다.
