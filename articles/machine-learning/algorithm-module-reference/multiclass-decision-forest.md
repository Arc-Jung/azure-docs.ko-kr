# <a name="multiclass-decision-forest-module"></a>다중 클래스 의사 결정 포리스트 모듈

이 문서에서는 Azure Machine Learning 서비스에 대 한 시각적 인터페이스 (미리 보기)의 모듈을 설명 합니다.

이 모듈을 사용 하 여 기계 학습 모델을 기반으로 만들려는 합니다 *포리스트 의사 결정* 알고리즘입니다. 의사 결정 포리스트는 신속 하 게 일련의 태그가 지정 된 데이터에서 학습 하는 동안 의사 결정 트리를 작성 하는 앙상블 모델입니다.

## <a name="more-about-decision-forests"></a>의사 결정 포리스트 대 한 자세한 정보

의사 결정 포리스트 알고리즘을 앙상블 학습 방법에 대 한 분류 됩니다. 여러 의사 결정 트리를 작성 하 여 알고리즘이 차례로 *투표* 가장 인기 있는 출력 클래스에 있습니다. 투표를 집계에는 각 분류 의사 결정 트리의 포리스트는 레이블 빈도 정규화 되지 않은 히스토그램 출력 양식입니다. 집계 프로세스 이러한 히스토그램을 더하고 "확률" 각 레이블에 대해 가져오려는 결과 정규화 합니다. 높은 예측 신뢰도 있는 트리 앙상블의 최종 결정에 더 높은 가중치가 부여 된 경우

의사 결정 트리는 일반적 비 파라메트릭 모델, 즉 다양 한 배포를 사용 하 여 데이터를 지 않습니다. 각 트리에서 리프 노드 (의사 결정)에 도달할 때까지 트리 구조 수준을 늘립니다 각 클래스에 대해 간단한 테스트 시퀀스를 실행 합니다.

의사 결정 트리 경우 많은 이점이 제공 합니다.

+ 비선형 의사 결정 경계를 나타낼 수 있습니다.
+ 이러한는 학습 및 예측 중에 계산과 메모리 사용량 효율적입니다.
+ 이러한 통합된 기능 선택 및 분류를 수행 합니다.
+ 잡음 기능이 있는 경우 복원이 됩니다.

Azure Machine Learning에서 의사 결정 포리스트 분류자의 의사 결정 트리 앙상블으로 구성 됩니다. 일반적으로 앙상블 모델 검사 및 단일 의사 결정 트리에 비해 정확성 향상을 제공합니다. 자세한 내용은 [의사 결정 트리](http://go.microsoft.com/fwlink/?LinkId=403677)합니다.

## <a name="how-to-configure-multiclass-decision-forest"></a>다중 클래스 의사 결정 포리스트를 구성 하는 방법



1. 추가 된 **다중 클래스 의사 결정 포리스트** 모듈 인터페이스에서 실험을 합니다. 이 모듈에서 찾을 수 있습니다 **Machine Learning**를 **초기화 모델**, 및 **분류**합니다.

2. 열려는 모듈을 두 번 클릭 합니다 **속성** 창입니다.

3. 에 대 한 **메서드를 다시**, 개별 트리를 만드는 데 사용할 메서드를 선택 합니다.  모음 만들기 또는 복제에서 선택할 수 있습니다.

    + **모음**: Bagging 라고 *부트스트랩 집계*합니다. 이 방법에서는 데이터 집합의 원래 크기 될 때까지 대체를 사용 하 여 원래 데이터 집합을 임의로 샘플링 하 여 만든 새 샘플에서 각 트리 증가 합니다. 모델의 출력으로 결합 되어 *투표*, 집계의 양식입니다. 자세한 내용은 부트스트랩 집계에 대 한 Wikipedia 항목을 참조 하세요.

    + **복제**: 복제에서는 각 트리는 정확히 동일한 입력된 데이터에 대해 학습 됩니다. 다양 한 트리를 만드는 임의 분할의 각 트리 노드에 대 한 조건자는 확인 상태로 유지 됩니다.

   

4. 설정 하 여 학습 모델을 지정 합니다 **트레이너 모드 만들기** 옵션입니다.

    + **매개 변수를 단일**: 모델을 구성 하 고 값을 인수로 집합을 제공 하고자 하는 방법을 알고 있는 경우이 옵션을 선택 합니다.


5. **의사 결정 트리의 수**: 앙상블에서 만들 수 있는 의사 결정 트리의 최대 수를 입력 합니다. 추가 의사 결정 트리를 만들면 적용 범위가 확대를 잠재적으로 가져올 수 있지만 학습 시간이 늘어날 수 있습니다.

    이 값에는 또한 학습된 된 모델을 시각화 하는 경우 결과에 표시 되는 트리 수를 제어 합니다. 를 보거나 단일 트리를 인쇄 하려면 값을 1로 설정할 수 있습니다. 그러나이 경우는 하나의 트리만 될 수 있습니다 (초기 매개 변수 집합을 사용 하 여 트리)를 생성 하 고 추가 반복에서 수행 됩니다.

6. **의사 결정 트리의 최대 깊이**: 의사 결정 트리의 최대 깊이를 제한할 숫자를 입력 합니다. 트리의 수준을 늘리면 높아지지만 학습 시간이 더 길어지고 과잉 맞춤이 정밀도 높아질 수 있습니다.

7. **노드당 임의 분할 수가**: 트리의 각 노드를 작성할 때 사용할 분할의 수를 입력 합니다. A *분할* (노드) 트리의 각 수준에서 기능 즉 임의로 구분 됩니다.

8. **리프 노드당 샘플의 최소 수**: 트리의 터미널 노드 (리프)를 만드는 데 필요한 경우의 최소 수를 나타냅니다. 이 값을 늘려 새 규칙 만들기에 대 한 임계값을 늘려야 합니다.

    예를 들어, 단일 사례도 1의 기본값을 사용 하 여 새 규칙을 만드는 발생할 수 있습니다. 값을 5로 늘리면 학습 데이터에 동일한 조건을 만족 하는 사례가 다섯 개 이상 포함 해야 합니다.



10. 레이블이 지정 된 데이터 집합을 지정 하 고 학습 모듈 중 하나를 연결 합니다.

    + 설정 하는 경우 **트레이너 모드 만들기** 를 **단일 매개 변수**를 사용 합니다 [모델 학습](./train-model.md) 모듈입니다.

11. 실험을 실행합니다.

## <a name="results"></a>결과

완료 된 후 교육:

+ 각 노드에 대 한 규칙을 보려면 각 트리 분할으로 드릴 다운 하려면 클릭 합니다.
