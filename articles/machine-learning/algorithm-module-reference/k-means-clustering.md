##  <a name="understanding-k-means-clustering"></a>이해 k-means 클러스터링
 
일반적으로 클러스터링에서는 반복 기술을 데이터 집합의 사례를 그룹화 비슷한 특징을 가진 클러스터로 합니다. 이러한 그룹화는 데이터에서 변칙을 식별 하는 데이터를 탐색 하는 데 고 최종적으로 예측 하는 데 유용 합니다. 클러스터링 모델도 도움이 될 수 있습니다 검색 또는 단순 관찰 하 여 논리적으로 파생 될 수 있습니다 하는 데이터 집합 내에서 관계를 식별 합니다. 이러한 이유로, 클러스터링는 대개 기계 학습 작업의 초기 단계에서 데이터를 탐색 하 고 예기치 않은 상관관계를 검색 합니다.  
  
 K-means 메서드를 사용 하 여 클러스터링 모델을 구성 하는 경우 대상 수를 지정 해야 합니다 *k* 개수를 나타내는 *중심* 모델에서 원하는 합니다. 중심에는 각 클러스터의 특징에는 지점입니다. K-means 알고리즘은 클러스터 내 제곱합을 최소화 하 여 클러스터 중 하나에 들어오는 각 데이터 요소를 할당 합니다. 
 
학습 데이터를 처리할 때 k-means 알고리즘은 각 클러스터에 대 한 시작점을 제공 하 고를 중심의 위치를 반복적으로 구체화 Lloyd의 알고리즘을 적용 하는 중심을 선택 하는 randomnly의 초기 집합으로 시작 합니다. K-means 알고리즘의 구축 및 이러한 조건 중 하나 이상이 부합 하는 경우 클러스터를 구체화를 중지 합니다.  
  
-   중심이 안정화, 즉 개별 지점에 대 한 클러스터 할당이 더 이상 변경 되 고 알고리즘을 솔루션으로 수렴 되었습니다.  
  
-   알고리즘 반복 횟수를 지정 된 실행을 완료 했습니다.  
  
 교육 단계를 완료 한 후 사용 합니다 [클러스터에 데이터 할당](assign-data-to-clusters.md) k-means 알고리즘에서 발견 된 클러스터 중 하나에 새 사례를 할당 하는 모듈입니다. 새로운 사례가 각 클러스터의 중심 사이의 거리를 계산 하 여 클러스터 할당을 수행 합니다. 각 새 사례를 가장 가까운 중심을 사용 하 여 클러스터에 할당 됩니다.  

## <a name="how-to-configure-k-means-clustering"></a>K-means 클러스터링을 구성 하는 방법
  
1.  추가 된 **K-means 클러스터링** 모듈을 실험 합니다.  
  
2.  설정 하 여 학습 모델을 지정 합니다 **트레이너 모드 만들기** 옵션입니다.  
  
    -   **매개 변수를 단일**: 클러스터링 모델에 사용 하려는 정확한 매개 변수를 알고 있으면 특정 값 집합을 인수로 제공할 수 있습니다.  
  
   
  
3.  에 대 한 **수의 중심**를 클러스터로 시작 하는 알고리즘을 원하는 횟수를 입력 합니다.  
  
     클러스터의에이 숫자에 정확 하 게 생성 하기 위해 모델 보장 되지 않습니다. algorithn이이 숫자 데이터 요소를 사용 하 여 시작 하 고 최적의 구성을 찾기까지 반복 합니다.  
  
     
  
4.  속성을 **초기화** 초기 클러스터 구성을 정의 하는 데 사용 되는 알고리즘을 지정 하는 데 사용 됩니다.  
  
    -   **처음 N**: 몇 가지 초기 데이터 요소 개수가 데이터 집합에서 선택 하 고 초기 수단으로 사용 됩니다.  
  
         라고도 합니다 *Forgy 방법이*합니다.  
  
    -   **임의**: 클러스터에서 데이터 요소를 임의로 배치한 알고리즘과 다음 임의로 클러스터 요소의 중심이 되도록 초기 평균 지점 할당을 계산 합니다.  
  
         라고도 합니다 *임의 파티션* 메서드.  
  
    -   **K-Means++**: 이것이 클러스터를 초기화 하기 위한 기본 방법입니다.  
  
         합니다 **k-means + +** 알고리즘 David arthur 2007에서 제안 된 및 표준 불량 클러스터링을 방지 하려면 Sergei Vassilvitskii k-means 알고리즘입니다. **K-평균 + +** 는 초기 클러스터 중심을 선택 하기 위한 다른 방법을 사용 하 여 표준 k-means 특징을 향상 합니다.  
  
    
5.  에 대 한 **임의 숫자 초기값**, 선택적으로 초기화 하는 클러스터에 대 한 초기값으로 사용할 값을 입력 합니다. 이 값 클러스터 선택에 상당한 영향을 미칠 수 있습니다.  
  
    
  
6.  에 대 한 **메트릭**, 또는 새 데이터 요소와 임의로 선택한 중심 클러스터 벡터 사이의 거리를 측정 하는 데 함수를 선택 합니다. Azure Machine Learning에는 다음 클러스터 거리 메트릭을 지원합니다.  
  
    -   **유클리드**: 유클리드 거리를 일반적으로 k-means 클러스터링의 클러스터 산 점도 측정값으로 사용 됩니다. 이 메트릭은 지점과 중심 사이의 평균 거리를 최소화 하기 때문에 좋습니다.
  

  
7.  에 대 한 **반복**, 알고리즘의 선택 항목을 완료 하기 전에 학습 데이터에 대해 반복 해야 하는 횟수를 입력 합니다.  
  
     학습 시간 및 정확도 분산 하려면이 매개 변수를 조정할 수 있습니다.  
  
8.  에 대 한 **할당 레이블 모드**, 레이블 열을 데이터 집합에 있는 경우 처리 방법을 지정 하는 옵션을 선택 합니다.  
  
     K-means 클러스터링을 감독 되지 않은 컴퓨터 학습 방법 이므로 레이블은 선택적입니다. 그러나 데이터 집합에 레이블 열 이미 있으면 해당 값을 사용 하 여 클러스터의 선택 가이드 또는 값 무시 함을 지정할 수 있습니다.  
  
    -   **레이블 열을 무시**: 레이블 열에서 값을 무시 되 고 모델 작성에 사용 되지 않습니다.
  
    -   **누락 된 값 입력**: 레이블 열 값은 클러스터 작성 하는 데 기능으로 사용 됩니다. 모든 행은 레이블을 누락 하는 경우 다른 기능을 사용 하 여 값 귀속 됩니다.  
  
    -   **Center에 가장 가까운에서 덮어쓰기**: 레이블 열 값은 현재 중심에 가장 가까운 요소의 레이블을 사용 하 여 예측 된 레이블 값으로 대체 됩니다.  

8.  옵션을 선택 **기능 정규화**학습 전에 기능을 정규화 할 경우.
  
     데이터 요소는 정규화 학습 하기 전에 정규화를 적용 하는 경우 `[0,1]` MinMaxNormalizer 여 합니다.

10. 모델을 학습 합니다.  
  
    -   설정 하는 경우 **트레이너 모드 만들기** 하 **단일 매개 변수**태그가 지정 된 데이터 집합을 추가 하 고 사용 하 여 모델을 학습 합니다 [클러스터링 모델 학습](train-clustering-model.md) 모듈입니다.
