# <a name="one-vs-all-multiclass"></a>OVA(One-vs-All)  다중 클래스

이 문서에서는 Azure 기계 학습 디자이너(미리 보기)에서 One-vs-All 다중 클래스 모듈을 사용하는 방법을 설명합니다. 목표는 *일대일* 접근 방식을 사용하여 여러 클래스를 예측할 수 있는 분류 모델을 만드는 것입니다.

이 모듈은 결과가 연속 또는 범주 예측 변수에 의존하는 경우 세 개 이상의 가능한 결과를 예측하는 모델을 만드는 데 유용합니다. 이 메서드를 사용하면 여러 출력 클래스가 필요한 문제에 대해서도 이진 분류 방법을 사용할 수 있습니다.

### <a name="more-about-one-versus-all-models"></a>한 대 모든 모델에 대해 자세히 알아보기

일부 분류 알고리즘은 의도적으로 두 개 이상의 클래스를 사용할 수 있도록 허용합니다. 다른 값은 가능한 결과를 두 값 중 하나(이진 또는 2클래스 모델) 중 하나로 제한합니다. 그러나 이진 분류 알고리즘조차도 다양한 전략을 통해 다중 클래스 분류 작업에 맞게 조정할 수 있습니다. 

이 모듈은 각 여러 출력 클래스에 대해 이진 모델을 생성하는 단일 대 모든 메서드를 구현합니다. 모듈은 개별 클래스에 대한 이러한 각 이진 모델을 해당 보체(모델의 다른 모든 클래스)에 대해 이진 분류 문제인 것처럼 평가합니다. 그런 다음 모듈은 이러한 이진 분류기를 실행하고 가장 높은 신뢰도 점수를 가진 예측을 선택하여 예측을 수행합니다.  

본질적으로 모듈은 개별 모델의 앙상블을 만든 다음 결과를 병합하여 모든 클래스를 예측하는 단일 모델을 만듭니다. 모든 이진 분류기는 일대일 모델의 기준으로 사용할 수 있습니다.  

예를 들어 [2클래스 지원 벡터 컴퓨터](two-class-support-vector-machine.md) 모델을 구성하고 One-vs-all 다중 클래스 모듈에 대한 입력으로 제공한다고 가정해 보겠습니다. 모듈은 출력 클래스의 모든 멤버에 대해 2클래스 지원 벡터 머신 모델을 만듭니다. 그런 다음 모든 클래스에 대한 결과를 결합하는 일 대 모든 메서드를 적용합니다.  

## <a name="how-to-configure-the-one-vs-all-multiclass-classifier"></a>일 대 모든 다중 클래스 분류기를 구성하는 방법  

이 모듈은 여러 클래스를 분석하기 위해 이진 분류 모델의 앙상블을 만듭니다. 이 모듈을 사용하려면 *먼저 이진 분류* 모델을 구성하고 학습해야 합니다. 

이진 모델을 One 대 All 다중 클래스 모듈에 연결합니다. 그런 다음 레이블이 지정된 학습 데이터 집합과 함께 [Train 모델을](train-model.md) 사용하여 모델의 앙상블을 학습합니다.

모델을 결합하면 One-vs-All Multiclass가 여러 이진 분류 모델을 만들고 각 클래스에 대한 알고리즘을 최적화한 다음 모델을 병합합니다. 이 모듈은 학습 데이터 집합에 여러 클래스 값이 있을 수 있더라도 이러한 작업을 수행합니다.

1. 디자이너의 파이프라인에 One-vs-All 다중 클래스 모듈을 추가합니다. 기계 학습 - **초기화,** **분류** 범주에서 이 모듈을 찾을 수 있습니다.

   일 대 모든 다중 클래스 분류기에는 자체의 구성 가능한 매개 변수가 없습니다. 모든 사용자 지정은 입력으로 제공되는 이진 분류 모델에서 수행해야 합니다.

2. 파이프라인에 이진 분류 모델을 추가하고 해당 모델을 구성합니다. 예를 들어 [2클래스 지원 벡터 컴퓨터](two-class-support-vector-machine.md) 또는 [2클래스 부스트 의사 결정 트리를](two-class-boosted-decision-tree.md)사용할 수 있습니다.

3. [파이프라인에 학습 모델](train-model.md) 모듈을 추가합니다. 한 대 모든 다중 클래스의 출력인 학습되지 않은 분류기연결을 연결합니다.

4. [Train Model의](train-model.md)다른 입력에서는 여러 클래스 값이 있는 레이블이 지정된 학습 데이터 집합을 연결합니다.

5. 파이프라인을 제출합니다.

## <a name="results"></a>결과

교육이 완료되면 모델을 사용하여 다중 클래스 예측을 수행할 수 있습니다.

또는 레이블이 지정된 유효성 검사 데이터 집합에 대해 교차 유효성 검사를 위해 학습되지 않은 분류기를 [교차 유효성 검사 모델에](cross-validate-model.md) 전달할 수 있습니다.
