# <a name="group-data-into-bins-module"></a>Bin 모듈로 데이터 그룹화

이 문서에서는 Azure Machine Learning 디자이너의 Bin 모듈에 데이터 그룹화를 사용 하 여 숫자를 그룹화 하거나 연속 데이터의 분포를 변경 하는 방법을 설명 합니다.

Bin으로 데이터 그룹화 모듈은 데이터를 범주화하기 위한 여러 옵션을 지원합니다. bin 에지가 설정되는 방법과 값이 bin에 할당되는 방법을 사용자 지정할 수 있습니다. 예를 들어, 다음을 수행할 수 있습니다.  

+ bin 경계로 사용할 일련의 값을 수동으로 입력합니다.  
+ *분위수* 또는 백분위수 순위를 사용하여 bin에 값을 할당합니다.  
+ 값의 짝수 배포를 bin으로 강제합니다.  

## <a name="more-about-binning-and-grouping"></a>범주화 및 그룹화에 대한 자세한 정보

*양자화*라고도 하는 데이터를 *범주화 하거나 그룹화* 하는 것은 기계 학습을 위해 숫자 데이터를 준비 하는 데 중요 한 도구입니다. 다음과 같은 시나리오에서 유용 합니다.

+ 연속 숫자 열에 너무 많은 고유 값이 있어 효과적으로 모델링할 수 없습니다. 따라서 자동으로 또는 그룹에 값을 할당 하 여 더 작은 불연속 범위 집합을 만듭니다.

+ 특정 범위를 나타내는 범주 값으로 숫자 열을 바꾸려고 합니다.

    예를 들어 사용자 인구 통계에 대해 사용자 지정 범위(예: 1-15, 16-22, 23-30 등)를 지정하여 나이 열의 값을 그룹화할 수 있습니다.

+ 데이터 세트는 예상 범위를 크게 벗어난 몇 가지 극단값을 가지며, 이러한 값은 학습된 모델에 큰 영향을 줍니다. 모델의 바이어스를 완화 하기 위해 변 위치 메서드를 사용 하 여 데이터를 균일 한 배포로 변형할 수 있습니다.

    이 메서드를 사용하여 Bin으로 데이터 그룹화 모듈은 대략 동일한 개수의 샘플이 각 bin에 포함되도록 이상적인 bin 위치 및 bin 너비를 결정합니다. 그런 다음 선택 하는 정규화 방법에 따라 bin의 값이 백분위 수로 변환 되거나 bin 숫자로 매핑됩니다.

## <a name="how-to-configure-group-data-into-bins"></a>Bin으로 데이터 그룹화를 구성하는 방법

1. 디자이너에서 파이프라인에 **그룹 데이터를 bin** 모듈에 추가 합니다. 이 모듈은 범주 **데이터 변환**에서 찾을 수 있습니다.

2. 숫자 데이터를 포함하는 데이터 세트를 bin에 연결합니다. 양자화는 숫자 데이터를 포함 하는 열에만 적용할 수 있습니다. 

    데이터 세트에 숫자가 아닌 열이 포함된 경우 [데이터 세트에서 열 선택](select-columns-in-dataset.md) 모듈을 사용하여 작업할 열의 하위 집합을 선택합니다.

3. 범주화 모드를 지정합니다. 범주화 모드는 다른 매개 변수를 결정 하므로 먼저 범주화 **모드** 옵션을 선택 해야 합니다. 지원되는 범주화 유형은 다음과 같습니다.

    - **분위수**: 분위수 메서드는 백분위수 순위에 따라 bin에 값을 할당합니다. 이 메서드를 동일한 높이 범주화 라고도 합니다.

    - **동일 너비**:이 옵션을 사용 하면 총 bin 수를 지정 해야 합니다. 데이터 열의 값은 각 bin이 시작 값과 끝 값 사이의 간격을 동일 하 게 하기 위해 bin에 배치 됩니다. 결과적으로 데이터가 특정 지점을 중심으로 모여 있는 경우 일부 bin에 더 많은 값이 있을 수 있습니다.

    - **사용자 지정 에지**: 각 bin을 시작하는 값을 지정할 수 있습니다. 에지 값은 항상 bin의 하한입니다. 
    
      예를 들어 값을 두 개의 계급로 그룹화 한다고 가정 합니다. 1은 0 보다 큰 값을 가지 며 한 값은 0 보다 작거나 같은 값을 갖습니다. 이 경우 bin 가장자리의 경우 **bin 가장자리를 쉼표로 구분 된 목록**으로 **0** 을 입력 합니다. 모듈의 출력은 1과 2 이며 각 행 값의 bin 인덱스를 나타냅니다. 쉼표로 구분 된 값 목록은 1, 3, 5, 7과 같이 오름차순 이어야 합니다.

4. **변 위치** 및 **동일 너비** 범주화 모드를 사용 하는 경우 **bin 수** 옵션을 사용 하 여 만들려는 bin 또는 *변 위치*의 수를 지정 합니다.

5. **열의**경우 열 선택기를 사용 하 여 원하는 값이 포함 된 열을 선택 합니다. 열은 숫자 데이터 형식이어야 합니다.

    선택한 모든 해당 열에 동일한 범주화 규칙이 적용됩니다. 다른 메서드를 사용 하 여 일부 열을 저장 해야 하는 경우 각 열 집합에 대해 그룹 데이터의 개별 인스턴스를 bin 모듈로 사용 합니다.

    > [!WARNING]
    > 허용 되는 유형이 아닌 열을 선택 하면 런타임 오류가 발생 합니다. 모듈은 허용되지 않는 형식의 열을 찾는 즉시 오류를 반환합니다. 오류가 발생하면 선택한 모든 열을 검토합니다. 오류에 잘못된 열이 모두 나열되지 않습니다.

6. **출력 모드**의 경우 양자화 값을 출력 하는 방법을 지정 합니다.

    + **추가**: 범주화 된 값을 사용 하 여 새 열을 만든 다음 입력 테이블에 추가 합니다.

    + **Inplace**: 데이터 세트의 새 값으로 원본 값을 대체합니다.

    + **ResultOnly**: 결과 열만 반환합니다.

7. **변 위치** 범주화를 선택 하는 경우 **변 위치 정규화** 옵션을 사용 하 여 변 위치로 정렬 하기 전에 값을 정규화 하는 방법을 결정 합니다. 정규화 값은 값을 변환 하지만 최종 bin 수에는 영향을 주지 않습니다.

    지원되는 정규화 형식은 다음과 같습니다.

    + **Percent**: 값은 [0100] 범위 내에서 정규화 됩니다.

    + **PQuantile**: 값은 [0, 1] 범위 내에서 정규화 됩니다.

    + **QuantileIndex**: 값은 [1, bin 수] 범위 내에서 정규화 됩니다.

8. **사용자 지정 가장자리** 옵션을 선택 하는 경우에는 쉼표로 **구분 된 bin** 에 지 텍스트 상자의 목록에서 *bin 가장자리로* 사용할 쉼표로 구분 된 숫자 목록을 입력 합니다. 

    값은 bin을 나누는 지점을 표시 합니다. 예를 들어 하나의 bin 가장자리 값을 입력 하는 경우 두 개의 bin이 생성 됩니다. 두 개의 bin 가장자리 값을 입력 하면 3 개의 bin이 생성 됩니다.

    값은 bin이 생성 된 순서에 따라 내림차순으로 정렬 되어야 합니다.

10. 열을 **범주별로 태그** 지정 옵션을 선택 하 여 양자화 열이 범주 변수로 처리 되도록 지정 합니다.

11. 파이프라인을 제출합니다.

## <a name="results"></a>결과

Bin으로 데이터 그룹화 모듈은 각 요소가 지정된 모드에 따라 범주화된 데이터 세트를 반환합니다. 

또한 범주화 *변환을*반환 합니다. 이 함수는 동일한 범주화 모드 및 매개 변수를 사용 하 여 데이터의 새 샘플을 bin으로 [변환](apply-transformation.md) 모듈에 전달할 수 있습니다.  

> [!TIP]
> 학습 데이터에 대해 범주화를 사용 하는 경우 테스트 및 예측에 사용 하는 데이터에 대해 동일한 범주화 방법을 사용 해야 합니다. 또한 동일한 bin 위치 및 bin 너비를 사용 해야 합니다. 
> 
> 동일한 범주화 방법을 사용 하 여 데이터가 항상 변환 되도록 하려면 유용한 데이터 변환을 저장 하는 것이 좋습니다. 그런 다음, [변환 적용](apply-transformation.md) 모듈을 사용 하 여 다른 데이터 집합에 적용 합니다.
