# <a name="two-class-logistic-regression-module"></a>2 클래스 로지스틱 회귀 모듈도

이 문서에서는 Azure Machine Learning 서비스에 대 한 시각적 인터페이스 (미리 보기)의 모듈을 설명 합니다.

이 모듈을 사용 하 여 두과 두 개의 결과 예측 하는 로지스틱 회귀 모델을 만듭니다. 

로지스틱 회귀는 여러 종류의 문제를 모델링에 사용 되는 유명한 통계 기법입니다. 이 알고리즘은는 *감독 학습* 메서드  따라서 모델의 학습에 결과 이미 포함 하는 데이터 집합을 제공 해야 합니다.  

### <a name="about-logistic-regression"></a>로지스틱 회귀에 대 한  

로지스틱 회귀는 결과의 확률을 예측 하는 데 사용 되 고 분류 작업에 특히 널리 사용 되는 통계는 잘 알려진 방법입니다. 알고리즘으로 데이터를 로지스틱 함수 이벤트 발생 확률을 예측 합니다.
  
이 모듈에서는 이진 또는 이분법 변수의 분류 알고리즘 최적화 됩니다. 여러 결과 분류 해야 할 경우 사용 합니다 [다중 클래스 로지스틱 회귀](./multiclass-logistic-regression.md) 모듈입니다.

##  <a name="how-to-configure"></a>구성 방법  

이 모델을 학습 하려면 레이블 또는 클래스 열이 포함 된 데이터 집합을 제공 해야 합니다. 이 모듈은 이므로 2 클래스 문제에 대 한 레이블 또는 클래스 열 정확히 두 개의 값을 포함 해야 합니다. 

예를 들어, 레이블 열 수 있습니다 수 [응답] "Yes" 또는 "No"의 가능한 값을 사용 하 여 합니다. 또는 "높음" 또는 "낮음"의 가능한 값을 사용 하 여 [신용 위험] 수 있습니다. 
  
1.  추가 된 **2 클래스 로지스틱 회귀** 모듈을 실험 합니다.  
  
2.  설정 하 여 학습 모델을 지정 합니다 **트레이너 모드 만들기** 옵션입니다.  
  
    -   **매개 변수를 단일**: 모델을 구성 하려는 방법를 알고 있으면 특정 값 집합을 인수로 제공할 수 있습니다.  
  
3.  에 대 한 **최적화 허용 오차**, 모델을 최적화 하는 경우 사용할 임계값 값을 지정 합니다. 반복 간의 개선 지정 된 임계값 아래로 떨어지는 경우 알고리즘 솔루션에서 수렴 형 있는 것으로 간주 됩니다 및 교육을 중지 합니다.  
  
4.  에 대 한 **L1 정규화 가중치** 하 고 **L2 정규화 가중치**, L1 및 L2 정규화 매개 변수에 사용할 값을 입력 합니다. 둘 다에 대해 0이 아닌 값이 좋습니다.  
  
     *정규화* 극단적인 계수 값을 사용 하 여 모델을 penalizing 여 과잉 맞춤을 방지 하기 위한 메서드입니다. 정규화는 계수 값 가설의 오류와 연결 된 페널티를 추가 하 여 작동 합니다. 따라서 더 극단적인 계수 값을 사용 하 여 정확한 모델에는 페널티가 되지만 더욱 일반적인 값을 사용 하 여 정확도 떨어지는 모델은 페널티가 적게 부여 합니다.  
  
     L1 및 L2 정규화에 미치는 영향이 달라 하 고 사용 합니다.  
  
    -   L1 차원 데이터로 작업할 때 유용 스파스 모델에 적용할 수 있습니다.  
  
    -   반면, L2 정규화는 스파스가 아닌 데이터에 대 한 것이 좋습니다.  
  
     이 알고리즘은 L1 및 L2 정규화 값의 선형 조합을 지원: 즉, 경우 <code>x = L1</code> 하 고 <code>y = L2</code>, 다음 <code>ax + by = c</code> 정규화 조건의 선형 범위를 정의 합니다.  
  
    > [!NOTE]
    >  L1 및 L2 정규화에 대 한 자세한 하 시겠습니까? 다음 문서에서는 L1 및 L2 정규화는 다양 한 방법 및 로지스틱 회귀 및 신경망 모델에 대 한 코드 샘플을 사용 하 여 모델 맞춤에 미치는 영향에 대해 설명 합니다.  [L1 및 L2 정규화 Machine Learning에 대 한](https://msdn.microsoft.com/magazine/dn904675.aspx)  
    >
    > 로지스틱 회귀 모델에 대 한 다양 한 선형 조합을 L1 및 L2 조건의 고안 되 었어야 합니다: 예를 들어 [탄력적 net 정규화](https://wikipedia.org/wiki/Elastic_net_regularization)합니다. 모델에서 유효한 선형 조합을 정의 하는 이러한 조합을 참조 하는 것이 좋습니다.
      
5.  에 대 한 **L-BFGS의 메모리 크기**에 사용할 메모리의 양을 지정 *L-BFGS* 최적화 합니다.  
  
     L-BFGS는 "제한 된 메모리 Broyden Fletcher-나타냅니다"를 나타냅니다. 최적화 알고리즘 매개 변수 예측에 널리 사용 되는 것입니다. 이 매개 변수는 이전 위치 및 다음 단계 계산을 위해 저장 하는 기울기의 수를 나타냅니다.  
  
     이 최적화 매개 변수는 다음 단계 및 방향을 계산 하는 데 사용 되는 메모리의 양을 제한 합니다. 메모리를 적게 지정할수록 학습 속도가 빨라지지만 정확도 낮아집니다.  
  
6.  에 대 한 **임의 숫자 초기값**를 정수 값을 입력 합니다. 시드 값을 정의 하는 것이 동일한 실험을 여러 번 실행을 재현할 수 결과 하려는 경우 중요 합니다.  
  
  
8. 태그가 지정 된 데이터 집합을 실험에 추가 하 고 연결 중 하나는 [학습 모듈](module-reference.md)합니다.  
  
    -   설정 하는 경우 **트레이너 모드 만들기** 를 **단일 매개 변수**를 사용 합니다 [모델 학습](./train-model.md) 모듈입니다.  
  
9. 실험을 실행합니다.  
  
## <a name="results"></a>결과
