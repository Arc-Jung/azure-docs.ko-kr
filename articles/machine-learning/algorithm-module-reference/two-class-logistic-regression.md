# <a name="two-class-logistic-regression-module"></a>2클래스 로지스틱 회귀 모듈

이 문서에서는 Azure 기계 학습 디자이너(미리 보기)의 모듈에 대해 설명합니다.

이 모듈을 사용하여 두 가지(및 두 개) 결과를 예측하는 데 사용할 수 있는 로지스틱 회귀 모델을 만듭니다. 

로지스틱 회귀는 여러 종류의 문제를 모델링하는 데 사용되는 잘 알려진 통계 기법입니다. 이 알고리즘은 *감독된 학습* 방법입니다.  따라서 모델을 학습하기 위해 결과를 이미 포함하는 데이터 집합을 제공해야 합니다.  

### <a name="about-logistic-regression"></a>물류 회귀에 관하여  

로지스틱 회귀는 결과의 확률을 예측하는 데 사용되는 통계에서 잘 알려진 방법이며 분류 작업에 특히 적합합니다. 알고리즘은 로지스틱 함수에 데이터를 피팅하여 이벤트가 발생할 확률을 예측합니다.
  
이 모듈에서 분류 알고리즘은 이분형 또는 이진 변수에 최적화되어 있습니다. 여러 결과를 분류해야 하는 경우 [다중 클래스 로지스틱 회귀 모듈을](./multiclass-logistic-regression.md) 사용합니다.

##  <a name="how-to-configure"></a>구성 방법  

이 모델을 학습하려면 레이블 또는 클래스 열이 포함된 데이터 집합을 제공해야 합니다. 이 모듈은 두 클래스 문제를 위한 것이기 때문에 레이블 또는 클래스 열에는 정확히 두 개의 값이 포함되어야 합니다. 

예를 들어 레이블 열은 "예" 또는 "아니요"의 가능한 값을 가진 [투표함]일 수 있습니다. 또는 [신용 위험]일 수 있으며 가능한 값은 "높음" 또는 "낮음"일 수 있습니다. 
  
1.  파이프라인에 **2클래스 로지스틱 회귀 모듈을** 추가합니다.  
  
2.  **트레이너 만들기 모드** 만들기 옵션을 설정하여 모델을 학습할 방법을 지정합니다.  
  
    -   **단일 매개 변수**: 모델을 구성하는 방법을 알고 있는 경우 특정 값 집합을 인수로 제공할 수 있습니다.  

    -   **매개 변수 범위**: 최상의 매개 변수를 잘 모르는 경우 모델 하이퍼 매개 변수 조정 모듈을 사용하여 최적의 매개 변수를 찾을 수 [있습니다.](tune-model-hyperparameters.md) 일부 값 범위를 제공하고 트레이너는 여러 설정 조합을 통해 이월하여 최상의 결과를 생성하는 값의 조합을 결정합니다.
  
3.  **최적화 공차의**경우 모델을 최적화할 때 사용할 임계값을 지정합니다. 반복 간의 개선이 지정된 임계값 아래로 떨어지면 알고리즘이 솔루션에 수렴된 것으로 간주되고 학습이 중지됩니다.  
  
4.  **L1 정규화 가중치** 및 **L2 정규화 가중치의**경우 정규화 매개변수 L1 및 L2에 사용할 값을 입력합니다. 둘 다에 0이 아닌 값을 사용하는 것이 좋습니다.  
     *정규화는* 극단적인 계수 값으로 모델을 불이익을 주어 과적합을 방지하는 방법입니다. 정규화는 계수 값과 연관된 페널티를 가설의 오류에 추가하여 작동합니다. 따라서 극단적인 계수 값을 가진 정확한 모델은 더 많은 불이익을 주지만 더 보수적인 값을 가진 덜 정확한 모델은 불이익을 덜 받을 것입니다.  
  
     L1 및 L2 정규화가 미치는 영향과 사용 방법은 여러 가지가 있습니다.  
  
    -   L1은 스파스 모델에 적용할 수 있으며 이는 고차원 데이터로 작업할 때 유용합니다.  
  
    -   한편, L2 정규화는 스파스가 아닌 데이터에 사용하는 것이 좋습니다.  
  
     이 알고리즘은 L1 및 L2 정규화 값의 <code>x = L1</code> <code>y = L2</code> <code>ax + by = c</code> 선형 조합을 지원합니다.  
  
    > [!NOTE]
    >  L1 및 L2 정규화에 대해 자세히 알고 싶으신가요? 다음 문서에서는 L1 및 L2 정규화가 어떻게 다른지, 그리고 로지스틱 회귀 및 신경망 모델에 대한 코드 샘플인 [기계 학습을 위한 L1 및 L2 정규화를](https://msdn.microsoft.com/magazine/dn904675.aspx) 통해 모델 피팅에 미치는 영향에 대해 설명합니다.  
    >
    > L1 및 L2 용어의 다른 선형 조합은 로지스틱 회귀 모델에 대해 고안되었습니다: 예를 들어 [탄성 순 정규화.](https://wikipedia.org/wiki/Elastic_net_regularization) 이러한 조합을 참조하여 모델에 효과적인 선형 조합을 정의하는 것이 좋습니다.
      
5.  **L-BFGS의 메모리 크기에**대해 *L-BFGS* 최적화에 사용할 메모리 양을 지정합니다.  
  
     L-BFGS는 "제한된 메모리 브로이든 - 플레처 - 골드 파브 - 샨노"를 의미합니다. 이것은 매개 변수 예측에 널리 사용되는 최적화 알고리즘입니다. 이 매개 변수는 다음 단계 계산을 위해 저장할 이전 위치 및 기울기의 수를 나타냅니다.  
  
     이 최적화 매개 변수는 다음 단계 및 방향을 컴퓨팅하는 데 사용되는 메모리의 양을 제한합니다. 메모리를 적게 지정할수록 학습 속도가 빨라지지만 정확도는 낮아집니다.  
  
6.  **난수 시드의**경우 정수 값을 입력합니다. 시드 값을 정의하는 것은 동일한 파이프라인의 여러 실행에서 결과를 재현할 수 있도록 하는 경우에 중요합니다.  
  
  
8. 태그가 지정된 데이터 집합을 파이프라인에 추가하고 교육 모듈 중 하나를 [연결합니다.](module-reference.md)  
  
    -   **트레이너 만들기 모드를** 단일 매개 **변수로**설정하면 [학습 모델](./train-model.md) 모듈을 사용합니다.  
  
9. 파이프라인을 제출합니다.  
  
## <a name="results"></a>결과

교육이 완료된 후:
