# <a name="boosted-decision-tree-regression-module"></a>강화된 의사 결정 트리 회귀 모듈

이 문서에서는 Azure 기계 학습 디자이너(미리 보기)의 모듈에 대해 설명합니다.

이 모듈을 사용하여 부스팅을 사용하여 회귀 트리의 앙상블을 만듭니다. *부스팅은* 각 트리가 이전 트리에 종속된다는 것을 의미합니다. 알고리즘은 선행된 나무의 잔류를 피팅하여 학습합니다. 따라서 의사 결정 트리 앙상블이 승격되면 정확도는 개선되지만 적용 범위가 감소할 약간의 위험이 따릅니다.  
  
이 회귀 방법은 감독된 학습 방법이므로 *레이블이 지정된 데이터 집합이*필요합니다. 레이블 열에는 숫자 값이 포함되어야 합니다.  

> [!NOTE]
> 숫자 변수를 사용하는 데이터 집합에만 이 모듈을 사용합니다.  

모델을 정의한 후 [기차 모델을](./train-model.md)사용하여 모델을 학습합니다.

  
## <a name="more-about-boosted-regression-trees"></a>한눈에 보는 부스트 회귀 트리  

승격은 모음 만들기, 임의 포리스트 등과 함께 앙상블 모델을 만드는 여러 기존 방법 중 하나입니다.  Azure 기계 학습에서 향상된 의사 결정 트리는 MART 그라데이션 증폭 알고리즘을 효율적으로 구현합니다. 그라데이션 승격은 회귀 문제에 대한 기계 학습 기술입니다. 미리 정의된 손실 함수를 사용해 각 단계의 오류를 측정하고 다음 단계에서 오류를 수정하는 방식으로 각 회귀 트리를 단계별로 작성합니다. 따라서 예측 모델은 실제로는 더 약한 예측 모델의 앙상블입니다.  
  
회귀 문제에서 부스팅은 단계별로 일련의 나무를 빌드한 다음 임의의 분별 가능한 손실 함수를 사용하여 최적의 트리를 선택합니다.

## <a name="how-to-configure-boosted-decision-tree-regression"></a>향상 된 의사 결정 트리 회귀를 구성 하는 방법

1.  **파이프라인에 향상된 의사 결정 트리 모듈을** 추가합니다. 이 모듈은 **기계 학습,** **초기화,** **회귀** 범주에서 찾을 수 있습니다. 
  
2.  **트레이너 만들기 모드** 만들기 옵션을 설정하여 모델을 학습할 방법을 지정합니다.  
  
    -   **단일 매개 변수**: 모델을 구성하는 방법을 알고 있는 경우 이 옵션을 선택하고 특정 값 집합을 인수로 제공합니다. 
     
    -   **매개 변수 범위**: 최상의 매개 변수를 잘 모르고 매개 변수 스윕을 실행하려는 경우 이 옵션을 선택합니다. 반복할 값 범위를 선택하고 [모델 하이퍼매개 변수 조정은](tune-model-hyperparameters.md) 제공한 설정의 가능한 모든 조합을 반복하여 최적의 결과를 생성하는 하이퍼매개 변수를 결정합니다.    
   
  
3. **트리당 최대 잎 수**: 모든 트리에서 만들 수 있는 터미널 노드(잎)의 최대 수를 나타냅니다.  

    이 값을 늘리면 트리 크기가 커지고 정밀도는 높아질 수 있지만 학습 시간이 더 길어지고 과잉 맞춤이 발생할 수 있습니다.  

4. **리프 노드당 최소 샘플 수**: 트리에서 터미널 노드(리프)를 만드는 데 필요한 최소 케이스 수를 나타냅니다.

    이 값을 늘려 새 규칙을 작성하기 위한 임계값을 늘립니다. 예를 들어, 기본값이 1이면 단일 사례만으로도 새 규칙을 하나 작성할 수 있습니다. 값을 5로 늘리면 학습 데이터에 동일한 조건을 만족하는 사례가 다섯 개 이상 있어야 합니다.

5. **학습 속도**: 학습 하는 동안 단계 크기를 정의 하는 0과 1 사이 숫자를 입력 합니다. 학습 속도에 따라 학습자가 최적의 솔루션에 수렴하는 속도 또는 속도가 결정됩니다. 단계 크기가 너무 크면 최적의 솔루션을 오버슈트할 수 있습니다. 단계 크기가 너무 작으면 최상의 솔루션에 수렴하는 데 교육시간이 더 오래 걸립니다.

6. **구성된 트리 수**: 앙상블에서 작성할 의사 결정 트리의 총 수를 나타냅니다. 더 많은 의사 결정 트리를 만들면 잠재적으로 더 나은 커버리지를 얻을 수 있지만 교육 시간은 늘어입니다.

    이 값은 학습된 모델을 시각화할 때 표시되는 트리 수도 제어합니다. 단일 트리를 보거나 인쇄하려면 값을 1로 설정할 수 있습니다. 그러나 하나의 트리(초기 매개 변수 집합이 있는 트리)만 생성되며 추가 반복은 수행되지 않습니다.

7. **난수 시드**: 임의 시드 값으로 사용할 선택적 비음수 정수를 입력합니다. 시드를 지정하면 동일한 데이터와 매개 변수가 있는 런에서 재현성이 보장됩니다.

    기본적으로 임의 시드는 0으로 설정되어 있으며, 이는 초기 시드 값이 시스템 클럭에서 가져온 것을 의미합니다.
  

9. 학습 데이터 집합 및 교육 모듈 중 하나를 추가합니다.

    - **트레이너 모드 만들기** 옵션을 **단일 매개 변수로**설정하면 [학습 모델](train-model.md) 모듈을 사용합니다.  
  
    

10. 파이프라인을 제출합니다.  
  
## <a name="results"></a>결과

교육이 완료된 후:

+ 점수를 매기기 위해 모델을 사용하려면 모델을 [값 값](./score-model.md)에 연결하여 새 입력 예제에 대한 값을 예측합니다.

+ 학습된 모델의 스냅샷을 저장하려면 **학습된 모델의** 오른쪽 패널에서 **출력 탭을** 선택하고 **데이터 집합 등록 아이콘을** 클릭합니다. 학습된 모델의 복사본은 모듈 트리에 모듈로 저장되며 파이프라인의 연속 실행에는 업데이트되지 않습니다.
