# <a name="clean-missing-data-module"></a>정리 누락 된 데이터 모듈

이 문서에서는 Azure Machine Learning 서비스에 대 한 시각적 인터페이스 (미리 보기)의 모듈을 설명 합니다.

이 모듈을 사용 하 여 제거, 바꾸기 또는 누락 값을 유추 합니다. 

데이터 과학자 종종 누락 값에 대 한 데이터를 확인 하 고 데이터를 수정 하거나 새 값을 삽입 하도록 다양 한 작업을 수행 합니다. 이러한 정리 작업의 목적은 모델을 학습할 때 발생할 수 있는 데이터 누락으로 인 한 문제를 방지 하기 위해 하는 것입니다. 

이 모듈에서는 여러 유형의 누락 된 값을 포함 하 여 "치료"에 대 한 작업을 지원 합니다.

+ 자리 표시자, 평균 또는 기타 값을 사용 하 여 누락 값 대체
+ 값이 누락 된 행과 열을 완전히 제거
+ 통계 방법에 따라 값을 유추


이 모듈을 사용 하 여 원본 데이터 집합을 변경 되지 않습니다. 대신, 후속 워크플로에서 사용할 수 있는 작업 영역에서 새 데이터 집합을 만듭니다. 또한 다시 사용할 수 있도록 정리 된 새 데이터 집합을 저장할 수 있습니다.

또한이 모듈 누락 값을 정리 하는 데 사용 하는 변환의 정의 출력 합니다. 이 변환을 사용 하 여 동일한 스키마에 있는 다른 데이터 집합에서 다시 사용할 수 있습니다 합니다 [변환 적용](./apply-transformation.md) 모듈입니다.

### <a name="replace-missing-values"></a>누락 값 대체  

사용자가 적용 될 때마다 합니다 [누락 데이터 정리](./clean-missing-data.md) 모듈 집합이 한 데이터를 동일한 정리 작업을 선택 하는 모든 열에 적용 됩니다. 따라서 다른 메서드를 사용 하 여 다른 열을 정리 해야 할 경우 모듈의 개별 인스턴스를 사용 합니다.

1.  추가 합니다 [누락 데이터 정리](./clean-missing-data.md) 모듈을 실험에 누락 값이 있는 데이터 집합을 연결 합니다.  
  
2.  에 대 한 **정리할 열**를 변경 하려면 누락 된 값을 포함 하는 열을 선택 합니다. 여러 열을 선택할 수 있지만 선택한 모든 열에서 동일한 대체 메서드를 사용 해야 합니다. 따라서 일반적으로 해야 문자열 열과 숫자 열을 개별적으로 정리 합니다.

    예를 들어 모든 숫자 열에 누락 값을 확인 합니다.

    1. 선택한 열 선택기를 엽니다 **사용 하 여 규칙**합니다.
    2. 에 대 한 **시작**를 선택 **NO 열**합니다.

        모든 열으로도 시작할 수 있으며 다음 열을 제외할 수 있습니다. 처음에 규칙 표시 되지 않는 경우 먼저 클릭할 **모든 열**를 클릭할 수 있지만 **NO 열** 클릭 하 고 **모든 열** 모든 열을 사용 하 여 시작 하 고 필터링 한 후에 다시 (제외)를 기준으로 열 이름, 데이터 형식 또는 열 인덱스입니다.

    3. 에 대 한 **포함**를 선택 **열 형식이** 를 선택 하 고 드롭다운 목록에서 **숫자**, 또는 특정 숫자 형식입니다. 
  
    선택 하는 모든 정리 또는 대체 방법은에 적용할 수 있어야 **모든** 열을 선택에서 합니다. 모든 열의 데이터가 지정 된 작업을 사용 하 여 호환 되지 않으면 모듈 오류 반환 하 고 실험을 중지 합니다.
  
3.  에 대 한 **최소 누락 값 비율**, 수행할 작업에 필요한 누락 값의 최소 수를 지정 합니다.  
  
    와 함께에서이 옵션을 사용 **최대 누락 값 비율** 는 정리 작업을 데이터 집합에 대해 이루어집니다 조건을 정의할 수 있습니다. 값이 누락 된 행이 너무 많거나 너무 적을 경우에 작업을 수행할 수 없습니다. 
  
    나타냅니다를 입력 하는 숫자를 **비율** 열의 모든 값에 누락 값입니다. 기본적으로 **최소 누락 값 비율** 속성이 0으로 설정 됩니다. 즉, 하나의 누락 된 값이 없는 경우에 누락 값을 정리 합니다. 

    > [!WARNING]
    > 지정 된 작업을 적용 하려면 각 열이 조건이 충족 되어야 합니다. 예를 들어 하나의 열에는 실제로 20% 누락 값이 있지만 세 개의 열을 선택한 다음.2 (20%)에 누락 값의 최소 비율을 설정 합니다. 이 경우 열에만 정리 작업은 누락 값이 20% 이상에 적용 됩니다. 따라서 다른 열은 변경할 수 없습니다.
    > 
    > 누락 값이 변경 되었는지 여부를 모를 경우이 옵션을 선택 **누락 값 지표 열 생성**합니다. 열은 각 열에 최소 및 최대 범위에 대 한 지정된 된 조건을 충족 여부를 나타내는 데이터 집합에 추가 됩니다.  
  
4. 에 대 한 **최대 누락 값 비율**, 수행할 작업에 사용할 수 있는 누락 된 값의 최대 수를 지정 합니다.   
  
    행의 30% 이하로 누락 값이 포함 되지만 값으로 유지 하는 경우에 누락 값 대체를 수행 하려는 하는 예를 들어-행의 30% 이상 누락 값이 있으면 됩니다.  
  
    열의 모든 값에 누락 값의 비율로 수를 정의합니다. 기본적으로 **최대 누락 값 비율** 1로 설정 됩니다. 이 열에 값을 100%를 사용할 수 없는 경우에 누락 값 정리 되는 것을 의미 합니다.  
  
   
  
5. 에 대 한 **정리 모드**바꾸기 위한 다음 옵션 중 하나를 선택 하거나 값 누락을 제거 합니다.  
  
  
    + **사용자 지정 대체 값**: 모든 누락 값에 적용 되는 (예: 0 이나 NA) 자리 표시자 값을 지정 하려면이 옵션을 사용 합니다. 대체로 지정 하는 값을 열의 데이터 형식과 호환 되어야 합니다.
  
    + **평균으로 바꾸기**: 열의 평균을 계산 하 고 대체 값으로 평균을 사용 하 여 각 누락 값 열에 대 한 키를 누릅니다.  
  
        Double, 정수는 열 또는 부울 데이터 형식에만 적용 됩니다.  
  
    + **중앙값으로 바꾸기**: 열 중앙값을 계산한 열의 모든 누락 값에 대 한 대체로 중간 값을 사용 합니다.  
  
        정수 또는 Double 데이터 형식의 열에만 적용 됩니다. 
  
    + **최빈값으로 바꾸기**: 열에 대 한 모드를 계산한 다음 열의 모든 누락 값에 대 한 대체 값으로 모드를 사용 합니다.  
  
        정수, Double, 부울 또는 범주 데이터 형식의 열에 적용 됩니다. 
  
    + **전체 행 제거**: 하나 이상의 누락 값이 있는 데이터 집합의 모든 행을 완전히 제거 합니다. 누락 값을 임의로 누락로 간주할 수 하는 경우에 유용 합니다.  
  
    + **전체 열 제거**: 하나 이상의 누락 값이 있는 데이터 집합의 모든 열을 완전히 제거 합니다.  
  
    
  
6. 옵션 **대체 값** 옵션을 선택한 경우 사용할 수 있습니다 **사용자 지정 대체 값**합니다. 열의 모든 누락 값에 대 한 대체 값으로 사용 하 여 새 값을 입력 합니다.  
  
    참고는 정수, Double, 부울 또는 날짜 데이터 형식 열에만이 옵션을 사용할 수 있습니다. 날짜 열에 대 한 대체 값을 입력할 수도 있습니다 100 나노초 단위 숫자로 1/1/0001부터 오전 12 시  
  
7. **누락 값 지표 열 생성**: 출력 열에 값을 정리 하는 누락 값에 대 한 조건을 충족 했는지 여부를 표시 하려는 경우이 옵션을 선택 합니다. 이 옵션은 새 청소 작업을 위해 설정 하 고 정상적으로 작동 하는지 확인 하려는 경우에 특히 유용 합니다.
  
8. 실험을 실행합니다.

### <a name="results"></a>결과

모듈에 두 개의 출력을 반환합니다.  

-   **정리 된 데이터 집합**: 해당 옵션을 선택한 경우 표시기 열을 함께 지정 된 대로 값이 누락 된 선택한 열을 구성 하는 데이터 집합을 처리 합니다.  

    정리는 또한 "전달"에 대 한 중 하나를 선택 하지는 열입니다.  
  
-  **정리 변환**: 데이터 변환이 정리에 사용 되는 작업 영역에 저장 하 고 나중에 새 데이터에 적용할 수 있습니다.

### <a name="apply-a-saved-cleaning-operation-to-new-data"></a>새 데이터에 저장 된 정리 작업을 적용  

정리 작업을 자주 반복 해야 할 경우 여 작성법으로 데이터 정리를 위해 저장 하는 것이 좋습니다는 *변환*, 동일한 데이터 집합을 사용 하 여 다시 사용 합니다. 청소 변환을 저장 자주 다시 가져와야 하 고 다음 동일한 스키마가 있는 데이터를 정리 해야 하는 경우 특히 유용 합니다.  
      
1.  추가 된 [변환 적용](./apply-transformation.md) 모듈을 실험 합니다.  
  
2.  정리 하려는 데이터 집합을 추가 하 고 데이터 집합 오른쪽 입력된 포트에 연결 합니다.  
  
3.  확장 된 **변환** 인터페이스의 왼쪽 창에서 그룹입니다. 저장된 된 변형을 찾아 실험으로 끌어옵니다.  
  
4.  저장 된 변환의 왼쪽된 입력된 포트에 연결 [변환 적용](./apply-transformation.md)합니다. 

    저장 된 변환에 적용할 때는 변환 적용 되는 열을 선택할 수 없습니다. 이것은 변환이 이미 정의 되 고 원래 작업에 지정 된 열에 자동으로 적용 됩니다.

    그러나 숫자 열의 하위 집합에서 변환을 만들었다고 가정해 보십시오. 일치 하는 숫자 열 에서만에서 누락 값이 변경 때문에 오류가 발생 하지 않고이 변환이 혼합된 열 형식 데이터 집합에 적용할 수 있습니다.

6.  실험을 실행합니다.
