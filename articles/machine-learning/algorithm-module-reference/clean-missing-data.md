# <a name="clean-missing-data-module"></a>누락된 데이터 정리 모듈

이 문서에서는 Azure 기계 학습 디자이너(미리 보기)의 모듈에 대해 설명합니다.

이 모듈을 사용하여 누락된 값을 제거, 대체 또는 유추합니다. 

데이터 과학자는 종종 누락된 값이 있는지 확인한 다음 다양한 작업을 수행하여 데이터를 수정하거나 새 값을 삽입합니다. 이러한 청소 작업의 목표는 모델을 학습할 때 발생할 수 있는 누락된 데이터로 인한 문제를 방지하는 것입니다. 

이 모듈은 다음과 같은 "정리" 누락된 값에 대한 여러 유형의 작업을 지원합니다.

+ 누락된 값을 자리 표시자, 평균 또는 기타 값으로 바함
+ 누락된 값이 있는 행과 열을 완전히 제거합니다.
+ 통계 적 방법을 기반으로 값 추론


이 모듈을 사용 해도 원본 데이터 집합이 변경 되지 않습니다. 대신 작업 영역에 후속 워크플로에서 사용할 수 있는 새 데이터 집합을 만듭니다. 새로 정리된 데이터 집합을 저장하여 다시 사용할 수도 있습니다.

또한 이 모듈은 누락된 값을 정리하는 데 사용되는 변환의 정의를 출력합니다. [변환 적용](./apply-transformation.md) 모듈을 사용하여 동일한 스키마를 가진 다른 데이터 집합에서 이 변환을 다시 사용할 수 있습니다.  

## <a name="how-to-use-clean-missing-data"></a>누락된 데이터 정리 사용 방법

이 모듈을 사용하면 정리 작업을 정의할 수 있습니다. 나중에 새 데이터에 적용할 수 있도록 정리 작업을 저장할 수도 있습니다. 청소 프로세스를 만들고 저장하는 방법에 대한 다음 섹션을 참조하십시오. 
 
+ [누락된 값을 바꾸려면](#replace-missing-values)
  
+ [새 데이터에 정리 변환을 적용하려면](#apply-a-saved-cleaning-operation-to-new-data)
 
> [!IMPORTANT]
> 누락된 값을 처리하는 데 사용하는 정리 방법은 결과에 큰 영향을 줄 수 있습니다. 다른 방법을 실험하는 것이 좋습니다. 특정 방법의 사용에 대한 정당성과 결과의 품질을 모두 고려하십시오.

### <a name="replace-missing-values"></a>누락된 값 대체  

[누락된 데이터 정리](./clean-missing-data.md) 모듈을 데이터 집합에 적용할 때마다 선택한 모든 열에 동일한 정리 작업이 적용됩니다. 따라서 다른 방법을 사용하여 다른 열을 청소해야 하는 경우 모듈의 별도 인스턴스를 사용합니다.

1.  [누락된 데이터 정리](./clean-missing-data.md) 모듈을 파이프라인에 추가하고 누락된 값이 있는 데이터 집합을 연결합니다.  
  
2.  **열을 정리하려면**변경하려는 누락된 값이 포함된 열을 선택합니다. 여러 열을 선택할 수 있지만 선택한 모든 열에서 동일한 대체 방법을 사용해야 합니다. 따라서 일반적으로 문자열 열과 숫자 열을 별도로 정리해야 합니다.

    예를 들어 모든 숫자 열에서 누락된 값을 확인하려면 다음을 수행합니다.

    1. **누락된 데이터 정리** 모듈을 선택하고 모듈의 오른쪽 패널에서 **열 편집을** 클릭합니다.

    3. **포함에**대해 드롭다운 목록에서 **열 유형을** 선택한 다음 **숫자를**선택합니다. 
  
    선택한 모든 정리 또는 교체 방법은 선택 영역의 **모든** 열에 적용되어야 합니다. 열의 데이터가 지정된 작업과 호환되지 않는 경우 모듈은 오류를 반환하고 파이프라인을 중지합니다.
  
3.  **누락된 값 비율의**경우 작업을 수행하는 데 필요한 누락된 값의 최소 수를 지정합니다.  
  
    이 옵션을 **최대 누락 값 비율과** 함께 사용하여 데이터 집합에서 정리 작업이 수행되는 조건을 정의합니다. 값이 누락된 행이 너무 많거나 너무 적으면 작업을 수행할 수 없습니다. 
  
    입력한 숫자는 열의 모든 값에 대한 누락된 값의 **비율을** 나타냅니다. 기본적으로 최소 **누락 값 비율** 속성은 0으로 설정됩니다. 즉, 누락된 값이 하나만 있는 경우에도 누락된 값이 정리됩니다. 

    > [!WARNING]
    > 지정된 작업이 적용되려면 이 조건을 각 열에서 충족해야 합니다. 예를 들어 세 개의 열을 선택한 다음 누락된 값의 최소 비율을 .2(20%)로 설정한다고 가정하지만 실제로는 한 열에만 20%의 누락 된 값이 있습니다. 이 경우 정리 작업은 값이 20% 이상 누락된 열에만 적용됩니다. 따라서 다른 열은 변경되지 않습니다.
    > 
    > 누락 값이 변경되었는지 여부를 모르면 **누락 값 지표 열 생성** 옵션을 선택합니다. 각 열이 최소 및 최대 범위에 대한 지정된 기준을 충족하는지 여부를 나타내기 위해 데이터 집합에 열이 추가됩니다.  
  
4. **누락된 값 비율의 최대**경우 작업을 수행할 수 있는 누락된 값의 최대 수를 지정합니다.   
  
    예를 들어 30% 이하의 행에 누락된 값이 포함되어 있는 경우에만 누락된 값 대체를 수행할 수 있지만 행의 30% 이상이 누락된 값이 있는 경우 값을 있는 그대로 두는 것이 좋습니다.  
  
    열의 모든 값에 대한 누락 값의 비율로 수를 정의합니다. 기본적으로 **누락된 최대 값 비율은** 1로 설정됩니다. 즉, 열에 있는 값의 100%가 누락된 경우에도 누락된 값이 정리됩니다.  
  
   
  
5. **정리 모드의**경우 누락된 값을 대체하거나 제거하기 위한 다음 옵션 중 하나를 선택합니다.  
  
  
    + **사용자 지정 대체 값**: 이 옵션을 사용하여 누락된 모든 값에 적용되는 자리 표시자 값(예: 0 또는 NA)을 지정합니다. 대체값으로 지정한 값은 열의 데이터 형식과 호환되어야 합니다.
  
    + **평균으로 바꾸기**: 열 평균을 계산하고 열의 각 누락 된 값에 대한 대체 값으로 mean를 사용합니다.  
  
        Integer, Double 또는 Boolean 데이터 형식의 열에만 적용됩니다.  
  
    + **중앙값으로 바꾸기**: 열 중앙값을 계산하고 중앙값을 열의 누락된 값에 대한 대체값으로 사용합니다.  
  
        Integer 또는 Double 데이터 형식의 열에만 적용됩니다. 
  
    + **모드로 바꾸기**: 열의 모드를 계산하고 열을 열의 모든 누락된 값에 대한 대체 값으로 사용합니다.  
  
        Integer, Double, Boolean 또는 Categorical 데이터 형식의 열에 적용됩니다. 
  
    + **전체 행 제거**: 하나 이상의 누락된 값이 있는 데이터 집합의 행을 완전히 제거합니다. 누락 값이 무작위로 누락된 것으로 간주할 수 있는 경우 이 옵션을 사용하면 유용합니다.  
  
    + **전체 열 제거**: 하나 이상의 누락된 값이 있는 데이터 집합의 열을 완전히 제거합니다.  
  
    
  
6. 옵션 **대체 값을** 선택한 경우 사용자 **지정 대체 값을**사용할 수 있습니다. 열에서 누락된 모든 값에 대한 대체 값으로 사용할 새 값을 입력합니다.  
  
    정수, 더블, 부울 또는 문자열이 있는 열에서만 이 옵션을 사용할 수 있습니다.
  
7. **누락된 값 표시기 열 생성**: 열의 값이 누락된 값 정리 기준을 충족하는지 여부를 표시하려면 이 옵션을 선택합니다. 이 옵션은 새 청소 작업을 설정하고 설계된 대로 작동하는지 확인하려는 경우에 특히 유용합니다.
  
8. 파이프라인을 제출합니다.

### <a name="results"></a>결과

모듈은 두 개의 출력을 반환합니다.  

-   **정리된 데이터 집합**: 해당 옵션을 선택한 경우 누락된 값이 지정된 대로 처리되고 표시기 열이 있는 선택한 열로 구성된 데이터 집합입니다.  

    청소를 위해 선택되지 않은 열도 "통과"됩니다.  
  
-  **변환 정리**: 정리에 사용되는 데이터 변환으로 작업 영역에 저장하고 나중에 새 데이터에 적용할 수 있습니다.

### <a name="apply-a-saved-cleaning-operation-to-new-data"></a>저장된 정리 작업을 새 데이터에 적용  

청소 작업을 자주 반복해야 하는 경우 동일한 데이터 집합으로 재사용하려면 데이터 정리를 위한 레시피를 *변환으로*저장하는 것이 좋습니다. 정리 변환을 저장하는 것은 동일한 스키마가 있는 데이터를 자주 다시 가져온 다음 정리해야 하는 경우에 특히 유용합니다.  
      
1.  파이프라인에 [변환 적용](./apply-transformation.md) 모듈을 추가합니다.  
  
2.  정리할 데이터 집합을 추가하고 데이터 집합을 오른쪽 입력 포트에 연결합니다.  
  
3.  디자이너의 왼쪽 창에서 변환 그룹을 **확장합니다.** 저장된 변환을 찾아 파이프라인으로 드래그합니다.  

4.  저장된 변환을 [변환 적용의](./apply-transformation.md)왼쪽 입력 포트에 연결합니다. 

    저장된 변환을 적용하면 변환이 적용되는 열을 선택할 수 없습니다. 이는 변환이 이미 정의되어 원래 작업에 지정된 열에 자동으로 적용되기 때문입니다.

    그러나 숫자 열의 하위 집합에 변환을 만들었다고 가정합니다. 누락된 값은 일치하는 숫자 열에서만 변경되므로 오류를 발생하지 않고 혼합 열 형식의 데이터 집합에 이 변환을 적용할 수 있습니다.

6.  파이프라인을 제출합니다.
