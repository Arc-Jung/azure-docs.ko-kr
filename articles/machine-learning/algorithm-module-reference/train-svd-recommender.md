# <a name="train-svd-recommender"></a>SVD 추천 학습

이 문서에서는 Azure 기계 학습 디자이너(미리 보기)에서 SVD 권장 사항 학습을 사용하는 방법을 설명합니다. 이 모듈을 사용하여 단일 값 분해(SVD) 알고리즘을 기반으로 권장 모델을 학습합니다.  

Train SVD 추천 모듈은 사용자 항목 등급 트리플의 데이터 집합을 읽습니다. 숙련된 SVD 추천서를 반환합니다. 그런 다음 학습된 모델을 사용하여 [SVD 추천자 점수를](score-svd-recommender.md) 사용하여 등급을 예측하거나 권장 사항을 생성할 수 있습니다.  


  
## <a name="more-about-recommendation-models-and-the-svd-recommender"></a>추천 모델 및 SVD 추천에 대한 자세한 내용은  

추천 시스템의 주요 목적은 시스템의 *사용자에게* 하나 이상의 *항목을* 추천하는 것입니다. 항목의 예로는 영화, 레스토랑, 책 또는 노래가 있습니다. 사용자는 사람, 사용자 그룹 또는 항목 기본 설정을 가진 다른 엔터티일 수 있습니다.  

추천 시스템에는 두 가지 주요 접근 방식이 있습니다. 

+ **콘텐츠 기반** 접근 방식은 사용자와 항목 모두에 대한 기능을 사용합니다. 사용자는 연령 및 성별과 같은 속성으로 설명할 수 있습니다. 항목은 작성자 및 제조업체와 같은 속성으로 설명할 수 있습니다. 소셜 매치메이킹 사이트에서 콘텐츠 기반 추천 시스템의 일반적인 예를 찾을 수 있습니다. 
+ **공동 작업 필터링은** 사용자와 항목의 식별자만 사용합니다. 사용자가 지정한 등급의 (스파스) 매트릭스에서 항목에 이러한 엔터티에 대한 암시적 정보를 가져옵니다. 사용자가 평가한 항목과 동일한 항목을 평가한 다른 사용자로부터 사용자에 대해 알아볼 수 있습니다.  

SVD 권장 자는 사용자 및 항목의 식별자와 사용자가 항목에 대해 지정한 등급 행렬을 사용합니다. 그것은 공동 *추천입니다.* 

SVD 추천자에 대한 자세한 내용은 관련 연구 논문: [추천 시스템에 대한 매트릭스 분해 기법을](https://datajobs.com/data-science-repo/Recommender-Systems-[Netflix].pdf)참조하십시오.


## <a name="how-to-configure-train-svd-recommender"></a>기차 SVD 추천을 구성하는 방법  

### <a name="prepare-data"></a>데이터 준비

모듈을 사용하기 전에 입력 데이터는 권장 사항 모델이 기대하는 형식이어야 합니다. 사용자 항목 등급 트리플의 교육 데이터 집합이 필요합니다.

+ 첫 번째 열에는 사용자 식별자가 포함되어 있습니다.
+ 두 번째 열에는 항목 식별자가 포함되어 있습니다.
+ 세 번째 열에는 사용자 항목 쌍에 대한 등급이 포함되어 있습니다. 등급 값은 숫자 유형이어야 합니다.  

Azure 기계 학습 디자이너의 **영화 등급** 데이터 집합(데이터 **집합** 을 선택한 다음 **샘플)은**예상된 형식을 보여 줍니다.

![영화 평점](media/module/movie-ratings-dataset.png)

이 샘플에서 한 사용자가 여러 영화를 평가한 것을 확인할 수 있습니다. 

### <a name="train-the-model"></a>모델 학습

1.  디자이너의 파이프라인에 SVD 권장 사항 학습 모듈을 추가하고 교육 데이터에 연결합니다.  
   
2.  **요인 수의**경우 권장 요소와 함께 사용할 요소 수를 지정합니다.  
    
    각 요소는 사용자가 항목과 관련된 정도를 측정합니다. 요인의 수는 또한 잠재 인자 공간의 차원이다. 사용자 수와 항목수가 증가함에 따라 더 많은 요인을 설정하는 것이 좋습니다. 그러나 숫자가 너무 크면 성능이 떨어질 수 있습니다.
    
3.  **권장 알고리즘 반복 횟수는** 알고리즘이 입력 데이터를 처리해야 하는 횟수를 나타냅니다. 이 숫자가 높을수록 예측이 더 정확해집니다. 그러나 숫자가 높을수록 교육 속도가 느려집니다. 기본값은 30입니다.

4.  **학습 속도의**경우 학습을 위한 단계 크기를 정의하는 0.0에서 2.0 사이의 숫자를 입력합니다.

    학습 속도는 각 반복에서 단계의 크기를 결정합니다. 단계 크기가 너무 크면 최적의 솔루션을 오버슈트할 수 있습니다. 단계 크기가 너무 작으면 최상의 솔루션을 찾는 데 교육시간이 더 오래 걸립니다. 
  
5.  파이프라인을 제출합니다.
