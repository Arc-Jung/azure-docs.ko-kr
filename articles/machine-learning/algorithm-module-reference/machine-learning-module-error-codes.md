# <a name="exceptions-and-error-codes-for-algorithm--module-reference"></a>예외 및 알고리즘 및 모듈 참조에 대 한 오류 코드

오류 메시지 및 예외 코드 모듈을 사용 하 여 Azure Machine Learning Studio에서 발생할 수 있는 방법을 알아봅니다. 

문제를 해결 하려면 일반적인 원인에 대해 알아보려면이 문서에서 오류를 찾아보십시오. 두 가지 방법으로 Studio의 오류 메시지의 전체 텍스트를 가져올 수 있습니다.  
 
- 링크를 클릭 **출력 로그 보기**, 오른쪽 창에서 아래쪽으로 스크롤합니다. 자세한 오류 메시지는 창의 마지막 두 줄에 표시 됩니다.  
  
- 오류가 있는 모듈을 선택 하 고 빨간색 X를 클릭 합니다. 관련 오류 텍스트에만 표시 됩니다.  
  
오류 메시지 텍스트를 유용한 없으면 보내주세요 컨텍스트 및 원하는 추가 또는 변경에 대 한 정보. 오류 항목에 대 한 피드백을 제출 하거나 방문 합니다 [Azure Machine Learning STUDIO 포럼](https://aka.ms/aml-forum-studio) 질문을 게시 하 고 있습니다.  


## <a name="error-0001"></a>Error 0001  
 데이터 집합의 하나 이상의 지정 된 열을 찾을 수 없습니다 예외가 발생 합니다.  
  
 열 선택 영역을 모듈의 경우 했지만 선택한 열이 입력된 데이터 집합에 없는 경우이 오류가 발생 합니다. 열 이름을 수동으로 입력 한 경우 또는 열 선택기는 실험을 실행 했을 때 데이터 집합에 존재 하지 않는 제안 된 열을 제공 하는 경우이 오류가 발생할 수 있습니다.  
  
**해결 방법:** 이 예외를 throw 하는 모듈을 다시 방문 하 고 열 이름 또는 이름이 올바른지, 그리고 참조 된 열을 모두 수행 있는지를 확인 합니다.  
  
|예외 메시지|  
|------------------------|  
|지정 된 열을 하나 이상 찾을 수 없습니다.|  
|열 이름 또는 인덱스를 사용 하 여 "{0}" 찾을 수 없음|  
|열 이름 또는 인덱스를 사용 하 여 "{0}"에 존재 하지 않습니다"{1}"|

 이 오류는 입력으로 매개 변수를 지정 하 고 값 형식이 예상 되는 형식과 다른 암시적 변환을 수행할 수 없는 경우 Azure Machine Learning에서 발생 합니다.  
  
**해결 방법:** 모듈 요구 사항을 확인 하 고 필요한 값 형식을 결정 (문자열, 정수, double, 등.)  
  
|예외 메시지|  
|------------------------|  
|매개 변수를 구문 분석 하지 못했습니다.|  
|구문 분석 하지 못했습니다 "{0}" 매개 변수|  
|(변환)를 구문 분석 하지 못했습니다 "{0}"매개 변수를 "{1}"|  
|변환 하지 못했습니다. "{0}"에서 매개 변수"{1}"to"{2}"|  
|변환 하지 못했습니다. "{0}"매개 변수 값"{1}"에서"{2}"to"{3}"|  
|값을 변환 하지 못했습니다. "{0}"에서 열"{1}"에서"{2}"to"{3}"형식의 사용"{4}" 제공|  
  

## <a name="error-0003"></a>0003 오류  
 예외가 발생 한 경우 또는 null 이거나 비워 둘 이상의 입력 됩니다.  
  
 입력 또는 모듈에 대 한 매개 변수가 null 이거나 비어 있으면 Azure Machine Learning에서이 오류가 나타납니다.  예를 들어, 매개 변수의 모든 값에 입력 하지 않은 경우이 오류가 발생할 수 있습니다. 누락 값이 있는 데이터 집합 또는 빈 데이터 집합을 선택한 경우에 발생할 수 있습니다.  
  
**해결 방법:**
 
+ 예외를 생성 하는 모듈을 열고 모든 입력이 지정 되었는지 확인 합니다. 필요한 모든 입력을 지정 했는지 확인 합니다. 
+ Azure storage에서 로드 되는 데이터에 액세스할 수 있고 계정 이름이 나 키를 변경 되지 않은 없는지 확인 하십시오.  
+ 누락 값 또는 null에 대 한 입력된 데이터를 확인 합니다.
+ 예상 데이터 형식으로 반환 되는 경우 데이터 원본에 대해 쿼리를 사용 하 여 확인 합니다. 
+ 오타 또는 데이터의 사양에 다른 변경 내용을 확인 합니다.
  
|예외 메시지|  
|------------------------|  
|입력 중 하나 이상이 null 이거나 빈 됩니다.|  
|입력 "{0}"가 null 이거나 비어|  
  

## <a name="error-0004"></a>0004 오류  
 매개 변수는 특정 값 보다 작거나 예외가 발생 합니다.  
  
 메시지의 매개 변수에 필요한 데이터를 처리 하는 모듈에 대 한 경계 값 미만인 경우에 Azure Machine Learning에서이 오류가 나타납니다.  
  
**해결 방법:** 예외를 throw 하는 모듈을 다시 방문 하 고 지정된 된 값 보다 큰 값으로 매개 변수를 수정 합니다.  
  
|예외 메시지|  
|------------------------|  
|매개 변수는 경계 값 보다 커야 합니다.|  
|매개 변수 "{0}" 값 보다 크거나 같아야 합니다. {1}합니다.|  
|매개 변수 "{0}"값이 "{1}" 보다 크거나 같아야 하는 {2}|

**해결 방법:** 예외를 throw 하는 모듈을 다시 방문 하 고 매개 변수를 지정 된 값 보다 크거나를 수정 합니다.  
  
|예외 메시지|  
|------------------------|  
|매개 변수는 경계 값 보다 크거나 이어야 합니다.|  
|매개 변수 "{0}" 값 보다 크거나 같은 이어야 {1}합니다.|  
|매개 변수 "{0}"값이 "{1}" 보다 크거나 같은 야 {2}합니다.|

## <a name="error-0007"></a>0007 오류  
 매개 변수가 특정 값 보다 크면 예외가 발생 합니다.  
  
 모듈에 대 한 속성에서 허용 된 것 보다 큰 값을 지정 하는 경우 Azure Machine Learning에서이 오류가 나타납니다. 예를 들어, 지원 되는 날짜 범위 외부에 있는 데이터를 지정할 수 있습니다 또는 세 개의 열만 사용할 수 있는 경우 다섯 개의 열을 사용 하는 수를 나타낼 수 있습니다. 
 
 이 오류는 두 가지 방법으로 일치 해야 하는 데이터 집합을 지정 하는 경우에 나타날 수 있습니다. 예를 들어, 열 이름을 바꿀 인덱스에서 열을 지정 하는 경우에 제공한 이름의 개수 열 인덱스 수가 일치 해야 합니다. 다른 예로 두 개의 열을 사용 하 여 열 수가 행에 있어야 하는 수학 작업을 수 있습니다. 
  
**해결 방법:**
 
 + 해당 모듈을 열고 모든 숫자 속성 설정을 검토 합니다.
 + 해당 속성에 대 한 모든 매개 변수 값 지원 되는 값 범위 안에 포함 되도록 합니다.
 + 모듈에서 여러 입력을 사용 하는 경우에 동일한 크기의 입력 되는지 확인 합니다.

 + 데이터 집합 또는 데이터 원본 변경 되었는지 여부를 확인 합니다. 경우에 따라 이전 버전의 데이터를 사용 하 여 작동 하는 값을 열 수, 열 데이터 형식 또는 데이터의 크기 변경 된 후 실패 합니다.

## <a name="error-0008"></a>오류 0008

**해결 방법:** 예외를 throw 하는 모듈을 다시 방문 하 고 지정된 된 범위 내에 있는 매개 변수를 수정 합니다.  
  
|예외 메시지|

## <a name="error-0009"></a>0009 오류  
 Azure storage 계정 이름 또는 컨테이너 이름을 잘못 지정 하면 예외가 발생 합니다.  
  
이 오류는 Azure storage 계정에 대 한 매개 변수를 지정 하지만 이름이 나 암호를 확인할 수 없는 경우 Azure Machine Learning Studio에서 발생 합니다. 암호 또는 계정 이름에는 오류는 여러 가지 이유로 발생할 수 있습니다.
 
 + 잘못 된 형식입니다. Machine Learning Studio 사용에 대 한 몇 가지 새로운 계정 형식은 지원 되지 않습니다. 참조 [데이터 가져오기](import-data.md) 세부 정보에 대 한 합니다.
 + 잘못 된 계정 이름 입력
 + 계정이 더 이상 존재합니다.
 + 저장소 계정에 대 한 암호가 잘못 되었거나 변경 되었습니다.
 + 컨테이너 이름을 지정 하지 않은 또는 컨테이너가 존재 하지 않습니다.
 + 파일 경로 (blob 경로)를 완전히 지정 되지 않았습니다.
   
**해결 방법:**

이러한 문제는 종종 계정 이름, 암호 또는 컨테이너 경로 수동으로 입력 하려는 경우에 발생 합니다. 에 대 한 새 마법사를 사용 하는 것이 좋습니다 합니다 [데이터 가져오기](import-data.md) 조회 및 이름 확인 하도록 도와주는 모듈입니다.

또한 계정, 컨테이너 또는 blob 삭제 되었습니다 여부를 확인 합니다. 다른 Azure storage 유틸리티를 사용 하 여는 계정 이름 및 암호를 올바르게 입력 했는지, 그리고 된 컨테이너가 있는지 확인 하십시오. 

Azure Machine Learning에서 일부 최신 계정 유형에 지원 되지 않습니다. 예를 들어, 새 "핫" 또는 "콜드" 저장소 형식은 기계 학습에 사용할 수 없습니다. 클래식 저장소 계정 및 "범용" 정상적으로 작동 하는 대로 생성 된 저장소 계정입니다.

Blob의 전체 경로가 지정 된 경우 경로로 지정 되어 있는지 확인 **컨테이너/blobname**, 컨테이너 및 blob 계정에 있어야 하 고 있습니다.  
  
 경로 선행 슬래시가 없어야 합니다. 예를 들어 **/컨테이너/blob** 올바르지 않으므로 새로로 입력 해야 **blob컨테이너/** 합니다.  

  
|예외 메시지|  
|------------------------|  
|Azure storage 계정 이름 또는 컨테이너 이름이 올바르지 않습니다.|  
|Azure storage 계정 이름 "{0}"또는 컨테이너 이름을"{1}"가 올바르지 않습니다; 형식 컨테이너 blob 컨테이너 이름이 필요 합니다.|  
  

## <a name="error-0010"></a>오류 0010  
 입력된 데이터 집합에 열 이름이 일치 해야 하는데 그렇지 않은 경우 예외가 발생 합니다.  
  
 메시지의 열 인덱스에 두 개의 입력된 데이터 집합의 다른 열 이름이 있으면 Azure Machine Learning에서이 오류가 나타납니다.  
  
**해결 방법:** 사용 하 여 [메타 데이터 편집](edit-metadata.md) 또는 동일한 열 이름이 지정 된 열 인덱스에 대 한 원래 데이터 집합을 수정 합니다.  
  
|예외 메시지|  
|------------------------|  
|입력된 데이터 집합에서 해당 인덱스를 사용 하 여 열 이름이 다릅니다.|  
|열 이름은 열에 대해 동일 하지 않습니다 {0} 입력 데이터 집합 (0부터 시작) ({1} 고 {2} 각각).|  
  

## <a name="error-0011"></a>Error 0011  
 예외가 발생 전달 되 면 열 집합 인수가 데이터 집합 열에 적용 되지 않습니다.  
  
 지정 된 열 선택이 지정된 된 데이터 집합의 열 중 하나를 일치 하지 않는 경우 Azure Machine Learning에서이 오류가 나타납니다.  
  
 열을 선택 하지 않은 하나 이상의 열이 작동 하려면 모듈에 대 한 필요한 경우에이 오류를 가져올 수 있습니다.

**해결 방법:** 디스크 드라이브의 임시 폴더로 추출 하 여 하나 이상의 xml 모듈 정의 파일은 zip 패키지의 루트 폴더에 있는지 확인 합니다. 모든 xml 파일을 zip 패키지를 추출한 폴더에 직접 있어야 합니다. Zip 압축 하도록 선택한 폴더와 동일한 이름을 가진 zip 패키지 내의 하위 폴더를 만들기는이 xml 파일이 포함 된 폴더를 선택 하지 않는 zip 패키지를 만들 때 있는지 확인 합니다.  
  
|예외 메시지|  
|------------------------|  
|지정한 ZIP 파일을 없습니다는 모듈 정의 파일을 (.xml 파일)|  


## <a name="error-0104"></a>0104 오류  
 모듈 정의 파일을 찾을 수 없는 스크립트를 참조할 때 발생 합니다.  
  
 Azure Machine Learning에서이 오류는 사용자 지정 모듈 xml 정의 파일에서 스크립트 파일을 참조 하는 경우 throw 되는 **언어** zip 패키지에 존재 하지 않는 요소입니다. 스크립트 파일 경로에 정의 되어는 **sourceFile** 의 속성을 **언어** 요소입니다. 원본 파일의 경로를 zip 패키지 (모듈 xml 정의 파일을 동일한 위치)의 루트에 상대적입니다. 스크립트 파일의 하위 폴더에 있으면 스크립트 파일에 상대 경로 지정 해야 합니다. 예를 들어에 저장 된 모든 스크립트를 **myScripts** zip 패키지 내의 폴더를 **언어** 요소에이 경로 추가 하는 것이 **sourceFile** 속성 아래. 예를 들면 다음과 같습니다.  
  
 `<Language name="R" sourceFile="myScripts/CustomAddRows.R" entryPoint="CustomAddRows" />`  
  
**해결 방법:** 있는지 확인 값을 **sourceFile** 속성에는 **언어** 요소의 사용자 지정 모듈 xml 정의 올바르고 zip 패키지의 올바른 상대 경로에 있는 소스 파일.  
  
|예외 메시지|  
|------------------------|

|참조 되는 R 스크립트 파일 '{0}' 찾을 수 없습니다. 정의 위치에서 파일에 상대 경로가 올바른지 확인 합니다.|

## <a name="error-0143"></a>0143 오류

**해결 방법:** 유효한 GitHub 리포지토리를 가리키는 URL을 확인 합니다. 다른 사이트 형식은 지원 되지 않습니다.  
  
|예외 메시지|  
|------------------------|

|GitHub URL을 구문 분석할 수 없습니다 (필요 ' \blob\\' 또는 ' \tree\\' 리포지토리 이름 뒤에 오는): {0}|

## <a name="error-1000"></a>1000 오류

|예외 메시지|  
|------------------------|  
|라이브러리 예외입니다.|  
|라이브러리 예외: {0}|  
|{0} 라이브러리 예외: {1}|
