# <a name="get-labels-for-data"></a>데이터 레이블 가져오기

많은 양의 데이터에 레이블을 지정하는 것은 기계 학습 프로젝트에서 골칫거리인 경우가 많습니다. 이미지 분류 또는 개체 감지와 같은 컴퓨터 비전 구성 요소가 있는 ML 프로젝트에는 일반적으로 수천 개의 이미지와 해당 레이블이 필요합니다. 
 
Azure Machine Learning은 레이블 지정 프로젝트를 만들고, 관리하고, 모니터링할 수 있는 중앙 위치를 제공합니다. 레이블 지정 프로젝트를 사용하면 데이터, 레이블 및 팀 멤버를 조정하여 레이블 지정 작업을 더 효율적으로 관리할 수 있습니다. 현재 지원되는 작업은 이미지 분류(다중 레이블 또는 다중 클래스) 및 경계 상자를 사용하는 개체 식별입니다.

Azure는 진행 상황을 추적하고 불완전한 레이블 지정 작업의 큐를 유지 관리합니다. 레이블 지정자에는 Azure 계정이 참여할 필요가 없습니다. MSA(Microsoft 계정) 또는 [Azure Active Directory](https://docs.microsoft.com/azure/active-directory/active-directory-whatis)를 사용하여 인증되면 시간이 허용하는 범위 내에서 레이블을 지정할 수 있습니다. 바로 가기 키를 사용하여 레이블을 할당하고 변경할 수 있습니다. 

프로젝트를 시작 및 중지하고, 사용자와 팀을 추가 및 제거하고, 진행 상황을 모니터링할 수 있습니다. 레이블이 지정된 데이터를 COCO 형식 또는 Azure ML 데이터 세트로 내보낼 수 있습니다. 

이 문서에서는 다음 방법을 설명합니다.

> [!div class="checklist"]
> * 프로젝트 만들기
> * 프로젝트의 데이터 및 구조 지정
> * 프로젝트에서 작업하는 팀 및 사용자 관리
> * 프로젝트 실행 및 모니터링
> * 레이블 내보내기 

## <a name="prerequisites"></a>필수 조건

* 로컬 파일 또는 이미 Azure 스토리지에 레이블을 지정하려는 데이터
* 적용하려는 레이블 세트
* 레이블 지정 지침
* Azure 구독. Azure 구독이 없는 경우 시작하기 전에 체험 계정을 만듭니다. 지금 [Azure Machine Learning 평가판 또는 유료 버전](https://aka.ms/AMLFree)을 사용해 보세요.
* Azure Machine Learning 작업 영역 [Azure Machine Learning 작업 영역 만들기](how-to-manage-workspace.md)를 참조하세요.

## <a name="create-a-labeling-project"></a>레이블 지정 프로젝트 만들기

레이블 지정 프로젝트는 [Azure Machine Learning](https://ml.azure.com/)에서 관리됩니다. **레이블 지정 프로젝트** 페이지에서는 프로젝트, 팀 및 사용자를 관리할 수 있습니다. 프로젝트에는 하나 이상의 팀이 할당되고, 팀에는 한 명 이상의 사용자가 할당됩니다. 

데이터가 이미 Azure Blob 스토리지에 저장된 경우 레이블 지정 프로젝트를 만들기 전에 해당 데이터를 데이터 저장소로 사용할 수 있도록 해야 합니다. 자세한 내용은 [데이터 저장소 만들기 및 등록](https://docs.microsoft.com/azure/machine-learning/service/how-to-access-data#create-and-register-datastores)을 참조하세요. 

프로젝트를 만들려면 **프로젝트 추가**를 선택합니다. 적절한 이름을 지정하고, **레이블 지정 작업 유형**을 선택합니다. 

![레이블 지정 프로젝트 만들기 마법사](media/how-to-create-labeling-projects/labeling-creation-wizard.png)

* 클래스 세트에서 **하나 _이상_** 의 레이블을 이미지에 적용할 수 있는 프로젝트의 경우 **이미지 분류 다중 레이블**을 선택합니다. 예를 들어 개의 사진에는 *dog* 및 *daytime*이라는 레이블이 모두 지정될 수 있습니다.
* 클래스 세트의 **단일 클래스**만 이미지에 적용할 수 있는 프로젝트의 경우 **이미지 분류 다중 클래스**를 선택합니다.
* 클래스를 이미지 내의 개체에 할당하고 개체 주위의 경계 상자를 지정하는 작업을 수행하는 프로젝트의 경우 **개체 식별(경계 상자)** 을 선택합니다.

이동할 준비가 되면 **다음**을 선택합니다.

## <a name="specify-data-to-be-labeled"></a>레이블을 지정할 데이터 지정

데이터가 포함된 데이터 세트를 이미 만든 경우 **기존 데이터 세트 선택** 드롭다운에서 선택할 수 있습니다. 또는 **데이터 세트 만들기**를 선택하여 기존 Azure 데이터 저장소를 선택하거나 로컬 파일을 업로드합니다. 

### <a name="create-a-dataset-from-an-azure-datastore"></a>Azure 데이터 저장소에서 데이터 세트 만들기

많은 사용 사례에서 로컬 파일을 직접 업로드하도록 선택하는 것이 좋지만, 많은 양의 데이터를 전송하는 경우 [Azure Storage Explorer](https://azure.microsoft.com/features/storage-explorer/)가 더 강력하고 빠릅니다. 파일을 이동하는 기본 방법으로 Azure Storage Explorer를 사용하는 것이 좋습니다.

Azure Blob 스토리지에 이미 저장한 데이터에서 데이터 세트를 만들려면 다음을 수행합니다.

1. **데이터 세트 만들기** 및 **데이터 저장소에서**를 선택합니다.
1. 데이터 세트에 대한 **이름**을 할당합니다.
1. **데이터 세트 형식**으로 "파일"을 선택해야 합니다.  
1. 데이터 저장소를 선택합니다. 
1. 데이터가 Blob 스토리지 내의 하위 폴더에 있으면 **찾아보기**를 선택하여 경로를 선택합니다. 
    * 선택한 경로의 하위 폴더에 모든 파일을 포함하려면 경로 뒤에 `/**`를 추가할 수도 있습니다.
    * 현재 컨테이너 및 하위 폴더에 있는 모든 데이터를 포함하려면 `**/*.*`를 사용합니다.
1. 데이터 세트에 대한 설명을 제공합니다.
1. **다음**을 선택합니다. 
1. 세부 정보를 확인합니다. **뒤로**를 선택하여 설정을 수정하거나 **만들기**를 선택하여 데이터 세트를 만들 수 있습니다.

### <a name="create-a-dataset-by-uploading-data"></a>데이터를 업로드하여 데이터 세트 만들기

데이터를 직접 업로드하려면 다음을 수행합니다.

1. **데이터 세트 만들기** 및 **로컬 파일에서**를 선택합니다.
1. 데이터 세트에 대한 **이름**을 할당합니다.
1. **데이터 세트 형식**으로 "파일"을 선택해야 합니다.
1. **고급 설정**을 선택하면 데이터의 데이터 저장소, 컨테이너 및 경로를 사용자 지정할 수 있습니다.
1. **찾아보기**를 선택하여 업로드할 로컬 파일을 선택합니다.
1. 데이터 세트에 대한 설명을 제공합니다.
1. **다음**을 선택합니다. 
1. 세부 정보를 확인합니다. **뒤로**를 선택하여 설정을 수정하거나 **만들기**를 선택하여 데이터 세트를 만들 수 있습니다.

데이터가 Azure ML 작업 영역의 기본 Blob 저장소(`workspaceblobstore`)로 업로드됩니다.

## <a name="specify-label-classes"></a>레이블 클래스 지정

**레이블 클래스** 페이지에서 데이터를 분류하는 데 사용되는 클래스 세트를 지정합니다. 레이블 지정자의 정확도와 속도가 선택하기 쉬운 편의성의 영향을 받으므로 이러한 클래스를 고려해야 합니다. 예를 들어 식물이나 동물의 전체 속과 종을 주석 처리하는 것보다 필드 코드를 사용하거나 속을 축약하는 것이 더 나을 수 있습니다. 

**+** 단추를 사용하여 하나의 새 행을 추가하면서 행당 하나의 레이블을 입력합니다. 3개 또는 4개 이상의 레이블이 있지만 10개 미만인 경우 작업 속도를 높이기 위해 숫자 키를 사용하여 레이블 지정자에 지침을 제공하려면 레이블 앞에 "1: ", "2: " 등의 접두사를 붙입니다. 

## <a name="describe-the-labeling-task"></a>레이블 지정 작업 설명

레이블 지정 작업을 명확하게 설명하는 것이 중요합니다. **레이블 지정 지침** 페이지에서는 외부 사이트에 연결하여 레이블 지정자에 지침을 제공할 수 있습니다. 작업 지향적이고 대상 그룹에 적합한 지침을 유지합니다. 

* 표시되는 레이블은 무엇이고, 이 중에서 어떻게 선택하나요? 참조해야 하는 참조 텍스트가 있나요?
* 적절한 레이블이 없는 경우 어떻게 해야 하나요? 
* 여러 개의 레이블이 적절한 것으로 보이는 경우 어떻게 해야 하나요?
* 레이블에 적용해야 하는 신뢰도 임계값은 무엇인가요? 확실하지 않은 경우 "최고의 추측"을 원하나요?
* 관심 있는 개체가 부분적으로 폐색되거나 겹치는 경우 어떻게 해야 하나요?
* 관심 있는 개체가 이미지의 가장자리에서 잘리는 경우 어떻게 해야 하나요?
* 이미지를 제출한 후 이미지가 잘못된 것으로 생각되면 어떻게 해야 하나요? 

경계 상자와 관련된 다른 중요한 질문은 다음과 같습니다.

* 이 작업에 대한 경계 상자는 어떻게 정의되나요? 개체와 완전히 관련되어 있거나, 가능한 한 가깝게 잘리거나, 어느 정도의 여유 공간이 허용되어야 하나요?? 
* 레이블 지정자에서 경계 상자를 정의해야 하는 주의 사항과 일관성 수준은 무엇인가요?

>[!Note]
> 레이블 지정자는 1-9의 숫자 키가 있는 처음 9개의 레이블 중에서 선택할 수 있습니다. 

## <a name="initialize-the-labeling-project"></a>레이블 지정 프로젝트 초기화

초기화되면 레이블 지정 프로젝트의 일부 측면은 변경할 수 없습니다. 즉, 작업 유형 또는 데이터 세트를 변경할 수 없습니다. 레이블을 수정하고 작업 설명의 URL을 변경할 수 있습니다. 먼저 설정을 신중하게 검토한 후에 프로젝트를 만들어야 합니다. 프로젝트가 제출되면 **레이블 지정** 홈페이지로 돌아갑니다. 이 경우 프로젝트가 **초기화 중**으로 표시됩니다. 이 페이지는 자동으로 새로 고치지 않으므로 일정 시간이 지난 후에 수동으로 새로 고치면 프로젝트가 **만들어짐**으로 표시됩니다. 

## <a name="manage-teams-and-people"></a>팀 및 사용자 관리

레이블 지정 프로젝트는 기본 팀을 가져와서 사용자를 기본 멤버로 추가합니다. 각 레이블 지정 프로젝트는 새 기본 팀을 가져오지만, 팀은 프로젝트 간에 공유할 수 있습니다. 프로젝트에는 둘 이상의 팀이 있을 수 있습니다. **팀** 페이지에서 **팀 추가**를 선택하여 팀을 만듭니다. 

사용자는 **사용자** 페이지에서 관리됩니다. 이메일 주소에서 키가 지정된 사용자는 추가하거나 제거할 수 있습니다. 각 레이블 지정자는 사용하는 Microsoft 계정 또는 Azure Active Directory를 사용하여 인증해야 합니다.  

사용자가 추가되면 하나 이상의 팀에 할당할 수 있습니다. **팀** 페이지로 이동하여 관심 있는 특정 팀을 선택한 다음, 원하는 대로 **사용자 할당** 또는 **사용자 제거**를 사용합니다.

팀의 모든 사용자에게 이메일을 보내려면 팀을 선택하여 **팀 세부 정보** 페이지를 표시합니다. 이 페이지에서 **이메일 팀** 단추를 클릭하면 팀 내 모든 사용자의 주소가 포함된 이메일 편집기가 열립니다.

## <a name="run-and-monitor-the-project"></a>프로젝트 실행 및 모니터링

프로젝트가 초기화되면 Azure에서 프로젝트 실행을 시작합니다. 주 **레이블 지정** 페이지에서 프로젝트를 클릭하면 **프로젝트 세부 정보**로 이동합니다. **대시보드** 탭에 레이블 지정 작업의 진행률이 표시됩니다. 

**데이터** 탭에서 데이터 세트를 보고, 레이블이 지정된 데이터를 검토할 수 있습니다. 잘못된 레이블이 지정된 데이터가 표시되면 이를 **선택**하고 **거부**를 선택할 수 있습니다. 그러면 레이블이 제거되고, 해당 데이터를 레이블이 지정되지 않은 큐에 다시 넣습니다. 

**팀** 탭에서 이 프로젝트에 팀을 할당하거나 할당 취소할 수 있습니다. 

프로젝트를 오프라인 또는 온라인 상태로 전환하려면 프로젝트의 실행 상태를 전환하는 **일시 중지**/**시작** 단추를 선택합니다.

**데이터 레이블 지정**을 선택하여 **프로젝트 세부 정보** 페이지에서 레이블을 데이터에 직접 지정할 수 있습니다. 프로젝트가 실행되는 경우에만 레이블을 데이터에 지정할 수 있습니다. 

## <a name="export-the-labels"></a>레이블 내보내기

기계 학습 실험을 위해 언제든지 레이블 데이터를 내보낼 수 있습니다. 이미지 레이블은 [COCO 형식](http://cocodataset.org/#format-data) 또는 Azure ML 데이터 세트로 내보낼 수 있습니다. 레이블 지정 프로젝트의 **프로젝트 세부 정보** 페이지에서 **내보내기** 단추를 찾을 수 있습니다.

COCO 파일은 **export/coco** 내의 폴더에 있는 Azure ML 작업 영역의 기본 Blob 저장소에 만들어집니다. 내보낸 Azure ML 데이터 세트는 Azure Machine Learning의 **데이터 세트** 섹션 아래에서 액세스할 수 있습니다. 데이터 세트 세부 정보 페이지에는 Python에서 레이블에 액세스할 수 있는 샘플 코드도 있습니다.
