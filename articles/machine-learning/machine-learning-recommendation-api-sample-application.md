# 기계 학습 권장 사항 API의 공통 작업

##목적

이 문서에서는 샘플 응용 프로그램을 통해 Azure 기계 학습 권장 사항 API의 일부 사용법을 보여 줍니다.

이 응용 프로그램은 전체 기능을 포함하거나 모든 API를 사용하지는 않지만 Azure 기계 학습 권장 사항 서비스를 처음 실행하려고 할 때 수행할 몇몇 공통 작업을 보여 줍니다. 

##권장 사항 소개

다음 데이터를 기반으로 권장 사항 모델을 빌드할 때 Azure 기계 학습 권장 사항 서비스를 통해 권장 사항이 사용하도록 설정됩니다.

* 권장하려는 항목의 리포지토리(카탈로그라고도 함)
* 사용자/세션당 항목 사용을 나타내는 데이터(샘플 앱의 일부가 아니라 데이터 취득을 통해 시간이 지남에 따라 취득 가능)

권장 사항 모델이 작성되면 이 모델을 사용하여 선택한 항목 집합(단일 항목일 수 있음)에 따라 사용자에 대해 항목을 예측할 수 있습니다.

위 시나리오를 허용하기 위해 Azure 기계 학습 권장 사항 서비스에서 다음 작업을 수행합니다.

* 모델 만들기 - 모델은 데이터(카탈로그 및 사용)와 예측 모델을 포함할 논리 컨테이너이고, 각 모델 컨테이너는 생성 시 할당되는 고유 ID를 통해 식별되고, 모델 ID라는 이 ID는 대부분 API에서 사용됩니다. 
* 카탈로그 업로드 - 모델 컨테이너를 만든 후에 카탈로그에 연결할 수 있습니다.

참고: 위의 단계('모델 만들기' 및 '카탈로그 업로드')는 일반적으로 모델 수명 주기 동안 한 번 수행됩니다.

* 사용 업로드 - 모델 컨테이너에 사용 데이터를 추가합니다.
* 권장 사항 모델 작성 - 충분한 데이터가 있으면 권장 사항 모델 빌드를 트리거합니다. 이 작업에서는 최신 기계 학습 알고리즘을 사용하여 권장 사항 모델을 만듭니다. 각 빌드는 고유 ID와 연결되고, 이 ID는 일부 API 기능에 필요하므로 사용자가 이 ID를 유지해야 합니다.
* 빌드 프로세스 모니터링 - 권장 사항 모델 빌드는 비동기 작업이고 데이터(카탈로그 및 사용) 크기 및 빌드 매개 변수에 따라 몇 분에서 몇 시간까지 걸릴 수 있습니다. 따라서 빌드를 모니터링해야 합니다. 연결된 해당 빌드가 성공적으로 종료되는 경우에만 권장 사항 모델이 빌드됩니다.
* (선택 사항) 활성 권장 사항 모델 빌드합니다. 이 단계는 모델 컨테이너에서 권장 사항 모델이 두 개 이상 빌드된 경우에만 필요합니다. 활성 빌드(활성 권장 사항 모델)를 지정하지 않고 권장 사항을 가져오는 모든 요청은 자동으로 기본 활성 빌드로 리디렉션됩니다. 

참고: 활성 빌드(권장 사항 모델)는 프로덕션 준비 상태이고 테스트 같은 환경(스테이징이라고도 함)에 머무르는 비활성 권장 사항 모델과 달리 프로덕션 작업용으로 빌드됩니다.

* 권장 사항 가져오기 - 권장 사항 모델이 있으면 선택한 단일 항목 또는 항목 목록에 대한 권장 사항을 트리거할 수 있습니다. 

일반적으로 특정 기간에 대한 권장 사항 가져오기를 호출하고, 그 동안 사용 데이터를 다시 Azure 기계 학습 권장 사항 시스템으로 리디렉션할 수 있고, 이 데이터는 모델에서 지정된 컨테이너에 추가됩니다. 충분한 사용 데이터가 있으면 새 권장 사항 모델을 빌드를 트리거하여 더 실제적인 데이터를 사용할 수 있습니다.

##사용되는 API

응용 프로그램에서는 사용할 수 있는 API의 하위 집합을 통해 Azure 기계 학습 권장 사항 기능의 작은 하위 집합만을 사용합니다. 다음 API는 응용 프로그램으로 보여 줍니다.

* 모델 만들기 - 데이터 및 권장 사항 모델을 포함하는 논리 컨테이너를 만듭니다. 모델은 이름으로 식별되고, 사용자는 같은 이름을 가진 모델을 두 번 만들 수 없습니다.
* 카탈로그 파일 업로드 - 카탈로그 데이터를 업로드합니다.
* 사용 파일 업로드 - 사용 데이터를 업로드합니다.
* 빌드 트리거 - 권장 사항 모델을 만듭니다.
* 빌드 실행 모니터링 - 권장 사항 모델 빌드의 상태를 모니터링합니다.
* 빌드된 권장 사항 모델 선택 - 특정 모델 컨테이너에 대해 기본적으로 사용할 권장 사항 모델을 지정합니다. 이 단계는 둘 이상의 권장 사항 모델이 있고 비활성 빌드를 활성 빌드로 사용하려는 경우에만 필요합니다.
* 권장 사항 가져오기 - 제공된 단일 항목 또는 항목 집합에 따라 권장 항목을 검색합니다. 

API에 대한 자세한 내용은 Microsoft Azure 마켓플레이스 설명서를 확인하세요. 

참고: 모델에는 같거나 업데이트된 카탈로그 및 추가 사용 데이터를 사용하여 각 빌드가 만들어지면서 시간이 지남에 따라(동시가 아님) 여러 빌드가 포함될 수 있습니다.

## 공통 문제

* 샘플을 실행하려면 사용자 이름 및 Microsoft Azure 마켓플레이스 기본 계정 키를 명령줄로 제공해야 합니다.
* 샘플 앱은 연속해서 실행할 수 없음 - 응용 프로그램은 핸들 만들기, 업로드, 빌드 모니터링 및 미리 정의된 모델 이름의 권장 사항 가져오기로 진행되므로 호출 간에 모델 이름을 변경하지 않으면 연속 실행에 실패합니다.
* 데이터가 없는 권장 사항이 반환될 수 있음 - 샘플 앱에서는 매우 작은 카탈로그 및 사용 파일을 사용하므로 만들어진 권장 사항 모델은 카탈로그의 일부 항목에 권장 항목이 없으면 의미가 없습니다.

## 고지 사항
샘플 앱은 프로덕션용으로 실행하도록 제공되지 않고, 카탈로그 및 사용에 제공된 데이터는 매우 작고 의미 있는 권장 사항 모델을 제공하지 않으며, 데모로만 제공됩니다.
