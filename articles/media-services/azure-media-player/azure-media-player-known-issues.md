# <a name="known-issues"></a>알려진 문제 #

현재 릴리스에는 다음과 같은 알려진 문제가 있습니다.

## <a name="azure-media-player"></a>Azure Media Player ##

- 잘못 구성된 인코더는 재생에 문제가 발생할 수 있습니다.
- 타임스탬프가 2^53보다 큰 스트림에는 재생 문제가 있을 수 있습니다.
  - 완화: 90kHz 비디오 및 44.1kHz 오디오 타임스케일 사용
- 사용자 상호 작용없이 모바일 장치에서 자동 재생이 없습니다. 그것은 플랫폼에 의해 차단됩니다.
- 불연속에 가까운 검색은 재생 실패의 원인이 될 수 있습니다.
- 대규모 프레젠테이션을 다운로드하면 UI가 잠글 수 있습니다.
- 프레젠테이션이 종료된 후 동일한 소스를 다시 재생할 수 없습니다.
  - 소스가 종료된 후 소스를 재생하려면 소스를 다시 설정해야 합니다.
- 빈 매니페스트는 플레이어에게 문제를 일으킬 수 있습니다.
  - 이 문제는 라이브 스트림을 시작할 때 발생할 수 있으며 매니페스트에서 충분한 청크가 발견되지 않습니다.
- UI 검색바 외부에서 재생 위치가 표시될 수 있습니다.
- 이벤트 순서가 모든 기술에서 일치하지는 않습니다.
- 버퍼링된 속성은 기술 간에 일관성이 없습니다.
- 네이티브ControlsForTouch는 "개체가 속성 또는 메서드 'setControls'를 지원하지 않습니다"를 피하기 위해 거짓 (기본값)이어야합니다.
- 이제 포스터는 절대 URL이어야 합니다.
- 장치의 고대비 모드를 사용할 때 UI에서 사소한 미적 문제가 발생할 수 있습니다.
- 완전히 디코딩된 문자열에 "%" 또는 "+"를 포함하는 URL에 소스를 설정하는 데 문제가 있을 수 있습니다.

## <a name="ad-insertion"></a>광고 삽입 ##

- 브라우저에 광고 차단기를 설치할 때 광고가 삽입되는 데 문제가 있을 수 있습니다(주문형 또는 실시간).
- 휴대기기에 광고를 재생하는 데 문제가 있을 수 있습니다.

## <a name="azurehtml5js"></a>AzureHtml5JS ##

- 라이브 콘텐츠의 DVR 창에서 콘텐츠가 완료되면 타임라인은 해당 영역을 찾거나 프레젠테이션이 끝날 때까지 계속 증가합니다.
- MSE를 사용하도록 설정 파이어 폭스에서 라이브 프리젠 테이션은 몇 가지 문제가 있습니다
- 오디오 또는 비디오인 에셋은 AzureHtml5JS 기술을 통해서만 재생되지 않습니다.
  - 오디오 또는 비디오 없이 에셋을 재생하려면 [Azure Media Services 탐색기 도구를](https://aka.ms/amse) 사용하여 빈 오디오 또는 비디오를 삽입하여 재생할 수 있습니다.
    - 자동 오디오를 삽입하는 방법에 대한 지침은 [여기에서](https://azure.microsoft.com/documentation/articles/media-services-advanced-encoding-with-mes/#silent_audio) 찾을 수 있습니다.

## <a name="flash"></a>깜박임 ##

- AES 콘텐츠는 Adobe의 캐싱 논리의 버그로 인해 플래시 버전 30.0.0.134에서 재생되지 않습니다. Adobe는 문제를 해결하고 30.0.0.154에서 발표했습니다.
- 기술 및 http 오류(예: 404네트워크 시간 초과)는 플레이어가 다른 기술보다 복구하는 데 더 오래 걸릴 수 있습니다.
- flashSS 기술이 있는 비디오 영역을 클릭하여 플레이어를 재생/일시 중지하지 않습니다.
- 사용자가 Flash를 설치했지만 사이트에 로드할 수 있는 권한을 부여하지 않는 경우 무한 회전이 발생할 수 있습니다. 이는 플레이어가 플러그인이 설치되어 있고 사용할 수 있다고 생각하고 플러그인이 콘텐츠를 실행하고 있다고 생각하기 때문입니다. 자바 스크립트 코드가 전송되었지만 브라우저 설정은 사용자가 플러그인을 허용하는 프롬프트를 수락 할 때까지 플러그인이 실행되지 못하도록 차단했습니다. 모든 브라우저에서 발생할 수 있습니다.  

## <a name="silverlight"></a>Silverlight ##

- 누락된 기능
- 기술 및 http 오류(예: 404네트워크 시간 초과)는 플레이어가 다른 기술보다 복구하는 데 더 오래 걸릴 수 있습니다.
- 실버 라이트와 맥 재생에 사파리와 파이어 `"http://` `https://` 폭스 명시적으로 정의 또는 소스에 대 한 필요.
- 이 기술에 대한 API가 없는 경우 일반적으로 null을 반환합니다.
- 사용자가 Flash를 설치했지만 사이트에 로드할 수 있는 권한을 부여하지 않는 경우 무한 회전이 발생할 수 있습니다. 이는 플레이어가 플러그인이 설치되어 있고 사용할 수 있다고 생각하고 플러그인이 콘텐츠를 실행하고 있다고 생각하기 때문입니다. 자바 스크립트 코드가 전송되었지만 브라우저 설정은 사용자가 플러그인을 허용하는 프롬프트를 수락 할 때까지 플러그인이 실행되지 못하도록 차단했습니다. 모든 브라우저에서 발생할 수 있습니다.  

## <a name="native-html5"></a>네이티브 HTML5 ##

- Html5 테크는 첫 번째 세트 소스에 대한 canplaythrough 이벤트를 보내고 있습니다.
- 이 기술은 브라우저가 구현한 내용만 지원합니다.  이 기술에서는 일부 기능이 누락되었을 수 있습니다.  
- 이 기술에 대한 API가 없는 경우 일반적으로 null을 반환합니다.
- 이 기술에 캡션 및 자막에 문제가 있습니다. 이 기술에서 사용할 수 있거나 볼 수 없을 수도 있습니다.
- HLS/Html5 기술 시나리오에서 대역폭이 제한되어 있으면 비디오 없이 오디오가 재생됩니다.

### <a name="microsoft"></a>Microsoft ###

- IE8 재생은 현재 ECMAScript 5와의 비호환성으로 인해 작동하지 않습니다.
- IE 및 일부 버전의 Edge에서는 단추를 탭하고 선택하거나 F/f 단축키를 사용하여 전체 화면을 입력할 수 없습니다.

## <a name="google"></a>Google ##

- 동일한 매니페스트의 여러 인코딩 프로필에는 Chrome에서 몇 가지 재생 문제가 있으며 권장되지 않습니다.
- 크롬AzureHtml5JS로 HE-AAC를 재생할 수 없습니다. [버그 추적기에](https://bugs.chromium.org/p/chromium/issues/detail?id=534301)대한 세부 정보를 따르십시오.
- Chrome v58을 통해 와이드바인 콘텐츠는 https:// 프로토콜을 통해 로드/재생되어야 하며 그렇지 않으면 재생이 실패합니다.

## <a name="mozilla"></a>Mozilla ##

- 오디오 스트림 스위치는 AzureHtml5JS를 사용할 때 오디오 스트림이 동일한 코덱 개인 데이터를 갖도록 요구합니다. 파이어 폭스 플랫폼이 필요합니다.

## <a name="apple"></a>Apple ##

- Mac의 Safari는 사용자에게 유리하지 않다고 판단될 때 플래시 및 Silverlight와 같은 플러그인을 차단하는 "전원을 절약하기 위해 플러그인 중지"라는 설정으로 기본적으로 절전 모드가 활성화됩니다. 이 블록은 플러그인의 존재, 만 기능을 차단하지 않습니다. 기본 techOrder가 주어지면 재생을 시도할 때 문제가 발생할 수 있습니다.
  - 완화 1: 비디오 플레이어가 '전면 및 중앙'인 경우(문서의 왼쪽 상단 모서리에서 시작하여 3000 x 3000 픽셀 경계 내에서) 계속 재생되어야 합니다.
  - 완화 2: ["azureHtml5JS", "html5"]로 Safari의 기술 주문을 변경합니다. 이러한 완화는 FlashSS 기술에서 사용할 수 있는 모든 기능을 얻지 못하는 것을 의미합니다.
- 실버라이트를 통한 PlayReady 콘텐츠에 Safari에서 재생에 문제가 있을 수 있습니다.
- AES 및 제한된 토큰 콘텐츠는 iOS 및 구형 Android 장치를 사용하여 재생되지 않습니다.
  - 이 시나리오를 달성하려면 프록시를 서비스에 추가해야 합니다.
- iOS 휴대전화의 기본 스킨이 표시됩니다.
- iPhone 재생은 항상 네이티브 플레이어 전체 화면에서 발생합니다.
  - 캡션을 포함한 기능은 이 비인라인 재생에서 유지되지 않을 수 있습니다.
- 라이브 프레젠테이션이 종료되면 iOS 기기가 프레젠테이션을 제대로 종료하지 않습니다.
- iOS는 DVR 기능을 허용하지 않습니다.
- iOS 오디오 스트림 스위치는 iOS 네이티브 플레이어 UI를 통해서만 수행할 수 있으며 오디오 스트림에 동일한 코덱 개인 데이터가 있어야 합니다.
- 이전 버전의 Safari에서는 라이브 스트림을 재생하는 데 문제가 있을 수 있습니다.

## <a name="older-android"></a>이전 안드로이드 ##

- AES 및 제한된 토큰 콘텐츠는 iOS 및 구형 Android 장치를 사용하여 재생되지 않습니다.
  - 이 시나리오를 달성하려면 프록시를 서비스에 추가해야 합니다.
