# <a name="building-an-accessible-application"></a>액세스 가능한 애플리케이션 작성

인터넷 사용자의 20% 이상이 액세스 가능한 웹 응용 프로그램에 대한 필요성을 가지고 있습니다. 따라서 모든 사용자가 쉽게 사용할 수 있도록 응용 프로그램을 설계되었는지 확인하는 것이 중요합니다. 접근성을 완료해야 할 작업 집합으로 생각하기보다는 전체 사용자 환경의 일부로 생각하십시오. 응용 프로그램에 액세스할수록 응용 프로그램을 사용할 수 있는 사람이 많아지게 됩니다. 

맵과 같은 풍부한 대화형 콘텐츠의 경우 다음과 같은 일반적인 접근성 고려 사항이 있습니다.
- 웹 응용 프로그램을 보는 데 어려움이 있는 사용자를 위해 화면 판독기를 지원합니다.
- 마우스, 터치 및 키보드와 같은 웹 응용 프로그램과 상호 작용하고 탐색하는 여러 가지 방법이 있습니다.
- 색상 대비가 서로 혼합되지 않고 서로 구분하기 어려워지도록 합니다. 

Azure Maps 웹 SDK는 다음과 같은 많은 내게 필요한 옵션 기능을 갖추고 있습니다.
- 맵이 이동할 때와 사용자가 컨트롤 또는 팝업에 초점을 맞출 때 화면 판독기 설명입니다.
- 마우스, 터치 및 키보드 지원.
- 도로 맵 스타일에서 접근 가능한 색상 대비 지원.

모든 Microsoft 제품에 대한 전체 내게 필요한 옵션 준수 세부 정보는 [여기에서](https://cloudblogs.microsoft.com/industry-blog/government/2018/09/11/accessibility-conformance-reports/)찾을 수 있습니다. "Azure Maps 웹"을 검색하여 Azure Maps 웹 SDK에 대한 문서를 특별히 찾습니다. 

## <a name="navigating-the-map"></a>지도 탐색

맵을 확대/축소, 패닝, 회전 및 투구할 수 있는 방법에는 여러 가지가 있습니다. 다음은 맵을 탐색하는 모든 다른 방법에 대해 자세히 설명합니다.

**지도 확대/축소**

- 마우스를 사용하여 맵을 두 번 클릭하여 한 레벨을 확대합니다.
- 마우스를 사용하여 휠을 스크롤하여 맵을 확대합니다.
- 터치 스크린을 사용하여 두 손가락으로 맵을 터치하고 함께 꼬집어 축소하거나 손가락을 펴서 확대합니다.
- 터치 스크린을 사용하여 맵을 두 번 탭하여 한 레벨을 확대합니다.
- 맵에 포커스를 맞추면 [플러스 기호()`+``=`또는 같음 기호()'를 사용하여 한 레벨을 확대합니다.
- 맵에 포커스를 맞추면 마이너스 기호,`-`하이픈(하이픈) 또는 밑줄()`_`()을 사용하여 한 레벨을 축소합니다.
- 마우스, 터치 또는 키보드 탭과 줌 컨트롤을 사용하여 / 입력 키.
- `Shift` 버튼을 길게 누르고 왼쪽 마우스 버튼을 지도에서 누르고 드래그하여 맵을 확대할 영역을 그립니다.

**맵 이동**

- 마우스를 사용하여 맵의 왼쪽 마우스 버튼을 누르고 어떤 방향으로든 드래그합니다.
- 터치 스크린을 사용하여 지도를 터치하고 어떤 방향으로든 드래그합니다.
- 맵에 초점을 맞춘 경우 화살표 키를 사용하여 맵을 이동합니다.

**맵 회전**

- 마우스를 사용하여 맵의 오른쪽 마우스 버튼을 누르고 왼쪽 또는 오른쪽으로 드래그합니다. 
- 터치 스크린을 사용하여 두 손가락으로 지도를 터치하고 회전합니다.
- 맵에 포커스를 맞추면 시프트 키와 왼쪽 또는 오른쪽 화살표 키를 사용합니다.
- 마우스, 터치 또는 키보드 탭/입력 키와 함께 회전 컨트롤을 사용합니다.

**지도 피치**

- 마우스를 사용하여 맵의 오른쪽 마우스 버튼을 누르고 위 또는 아래로 드래그합니다. 
- 터치 스크린을 사용하여 두 손가락으로 맵을 터치하고 위아래로 드래그합니다.
- 맵에 초점을 맞추고 시프트 키와 위쪽 또는 아래쪽 화살표 키를 사용합니다. 
- 마우스, 터치 또는 키보드 탭과 피치 컨트롤을 사용하여 / 입력 키.

## <a name="change-the-map-style"></a>맵 스타일 변경

모든 개발자가 응용 프로그램에서 가능한 모든 맵 스타일을 사용할 수 있기를 원하지는 않습니다. 개발자가 맵의 스타일 선택기 컨트롤을 표시하면 사용자는 탭이 있는 마우스, 터치 또는 키보드를 사용하여 맵 스타일을 변경하거나 키를 입력할 수 있습니다. 개발자는 맵 스타일 선택기 컨트롤에서 사용할 맵 스타일을 지정할 수 있습니다. 또한 개발자는 프로그래밍 방식으로 맵 스타일을 설정하고 변경할 수 있습니다.

**고대비 사용**

- 맵 컨트롤이 로드되면 고대비가 활성화되어 있고 브라우저에서 맵을 지원하는지 확인합니다.
- 맵 컨트롤은 장치의 고대비 모드를 모니터링하지 않습니다. 장치 모드가 변경되면 맵이 변경되지 않습니다. 따라서 사용자는 페이지를 새로 고침하여 맵을 다시 로드해야 합니다.
- 고대비가 감지되면 맵 스타일이 자동으로 고대비로 전환되고 모든 기본 제공 컨트롤은 고대비 스타일을 사용합니다. 예를 들어 ZoomControl, 피치 컨트롤, 나침반 제어, 스타일 컨트롤 및 기타 기본 제공 컨트롤은 고대비 스타일을 사용합니다.
- 높은 콘트라스트의 두 가지 유형이 있습니다, 빛과 어두운. 맵 컨트롤에서 고대비 유형을 감지할 수 있는 경우 맵의 동작이 그에 따라 조정됩니다. 라이트가 있으면 grayscale_light 맵 스타일이 로드됩니다. 형식을 검색할 수 없거나 어두운 경우 high_contrast_dark 스타일이 로드됩니다.
- 사용자 지정 컨트롤을 만드는 경우 기본 제공 컨트롤이 고대비 스타일을 사용하고 있는지 아는 것이 유용합니다. 개발자는 맵 컨테이너 div에 css 클래스를 추가하여 확인할 수 있습니다. 추가될 CSS 클래스는 `high-contrast-dark` 및 `high-contrast-light`. 자바 스크립트를 사용 하 여 확인 하려면, 사용 하 여:

```javascript
map.getMapContainer().classList.contains("high-contrast-dark")
```

또는 다음을 사용하십시오.

```javascript
map.getMapContainer().classList.contains("high-contrast-light")
```

## <a name="keyboard-shortcuts"></a>바로 가기 키

맵에는 맵을 더 쉽게 사용할 수 있도록 기본 제공된 여러 가지 키보드 단축키가 있습니다. 이러한 키보드 단축키는 맵에 포커스가 있을 때 작동합니다.

| Key      | 작업                            |
|----------|-----------------------------------|
| `Tab` | 맵의 컨트롤과 팝업을 탐색합니다. |
| `ESC` | 맵의 모든 요소에서 최상위 맵 요소로 포커스를 이동합니다. |
| `Ctrl` + `Shift` + `D` | 화면 판독기 세부 수준 토글.  |
| 왼쪽 화살표 키 | 맵 왼쪽 100픽셀 이동 |
| 오른쪽 화살표 키 | 맵 오른쪽 100픽셀 이동 |
| 아래쪽 화살표 키 | 맵을 100픽셀 아래로 이동 |
| 위쪽 화살표 키 | 맵을 100픽셀 위로 이동 |
| `Shift`+ 최대 화살표 | 지도 피치를 10도 증가시음 |
| `Shift`+ 아래쪽 화살표 | 지도 피치 를 10도 감소 |
| `Shift`+ 오른쪽 화살표 | 지도를 시계 방향으로 15도 회전 |
| `Shift`+ 왼쪽 화살표 | 지도를 시계 반대 방향으로 15도 회전 |
| 플러스 기호`+`( <sup>*</sup>) 또는`=`같음 기호 () | 확대 |
| 마이너스 기호, 하이픈`-`() <sup>*</sup>또는`_`밑줄 () | 축소 | 
| `Shift`+ 지역을 그릴지도에 마우스 드래그 | 영역으로 확대 |

<sup>*</sup>이러한 키 단축키는 일반적으로 키보드에서 동일한 키를 공유합니다. 이러한 바로 가기는 사용자 환경을 개선하기 위해 추가되었습니다. 또한 사용자가 이러한 바로 가기에 대해 시프트 키를 사용하는지 여부는 중요하지 않습니다.

## <a name="screen-reader-support"></a>화면 판독기 지원

사용자는 키보드를 사용하여 맵을 탐색할 수 있습니다. 화면 판독기가 실행 중이면 맵 상태 변경 내용 알림이 사용자에게 표시됩니다. 예를 들어 맵을 이동하거나 확대/축소하면 맵 변경 내용 알림이 사용자에게 표시됩니다. 기본적으로 맵은 맵 중앙의 확대/축소 수준과 좌표를 제외한 간단한 설명을 제공합니다. `Ctrl`  +  `Shift`  +  `D`사용자는 바로 가기 키보드를 사용하여 이러한 설명의 세부 수준을 전환할 수 있습니다.

기본 지도에 추가되는 모든 정보에는 화면 판독기 사용자를 위한 텍스트 정보가 있어야 합니다. 적절한 경우 [액세스 가능한 리치 인터넷 응용 프로그램(ARIA)](https://www.w3.org/WAI/standards-guidelines/aria/)및 alt 및 title 특성을 추가해야 합니다. 

## <a name="make-popups-keyboard-accessible"></a>팝업 키보드에 액세스할 수 있도록 설정

마커 또는 기호는 맵의 위치를 나타내는 데 자주 사용됩니다. 위치에 대한 추가 정보는 일반적으로 사용자가 마커와 상호 작용할 때 팝업에 표시됩니다. 대부분의 응용 프로그램에서는 사용자가 마커를 클릭하거나 탭할 때 팝업이 나타납니다. 그러나 클릭하고 탭하려면 사용자가 각각 마우스와 터치 스크린을 사용해야 합니다. 좋은 방법은 키보드를 사용할 때 팝업에 액세스할 수 있도록 하는 것입니다. 이 기능은 각 데이터 포인트에 대한 팝업을 만들고 맵에 추가하여 수행할 수 있습니다. 

다음 예제는 기호 레이어를 사용하여 맵의 관심 지점을 로드하고 각 관심 지점에 대한 팝업을 맵에 추가합니다. 각 팝업에 대한 참조는 각 데이터 포인트의 속성에 저장됩니다. 마커를 클릭할 때와 같이 마커에 대해 검색할 수도 있습니다. 맵에 초점을 맞출 때 탭 키를 누르면 사용자가 맵의 각 팝업을 단계별로 진행할 수 있습니다.

## <a name="additional-accessibility-tips"></a>추가 내게 필요한 옵션 팁

다음은 웹 매핑 응용 프로그램에 더 쉽게 액세스할 수 있도록 하는 몇 가지 추가 팁입니다.

- 맵에 많은 대화형 포인트 데이터를 표시하는 경우 혼란을 줄이고 클러스터링을 사용하는 것이 좋습니다. 
- 텍스트/기호와 배경색 간의 색상 대비 비율이 4.5:1 이상인지 확인합니다.
- 화면 판독기(ARIA, alt 및 제목 속성) 메시지를 짧고 설명적이며 의미 있는 메시지로 유지합니다. 불필요한 전문 용어와 약어는 피하십시오.
- 사용자가 쉽게 소화할 수 있는 짧은 의미 있는 정보를 제공하기 위해 화면 판독기로 전송된 메시지를 최적화해 보십시오. 예를 들어 맵이 이동하는 경우와 같이 빈도가 높은 화면 판독기를 업데이트하려면 다음 사항을 수행하는 것이 좋습니다.
    - 화면 판독기를 업데이트하기 위해 맵이동이 완료될 때까지 기다립니다.
    - 업데이트를 몇 초마다 한 번씩 제한합니다. 
    - 논리적인 방식으로 메시지를 결합합니다. 
- 정보를 전달하는 유일한 방법으로 색을 사용하지 않는 것이 좋습니다. 텍스트, 아이콘 또는 패턴을 사용하여 색상을 보완하거나 바꿉을 사용합니다. 몇 가지 고려할 사항:
    - 거품 레이어를 사용하여 데이터 점 간의 상대 값을 표시하는 경우 각 거품의 반지름 을 조정하거나 거품을 색칠하거나 둘 다에 대해 고려하십시오. 
    - 삼각형, 별, 사각형 등 다양한 메트릭 범주에 대해 서로 다른 아이콘이 있는 기호 레이어를 사용하는 것이 좋습니다. 심볼 레이어는 아이콘의 크기 조정도 지원합니다. 텍스트 레이블을 표시할 수도 있습니다.
    - 선 데이터를 표시하는 경우 너비를 사용하여 무게 또는 크기를 나타낼 수 있습니다. 대시 배열 패턴을 사용하여 다양한 범주의 선을 나타낼 수 있습니다. 기호 레이어는 선을 따라 아이콘을 오버레이하는 선과 함께 사용할 수 있습니다. 화살표 아이콘을 사용하면 선의 흐름이나 방향을 표시하는 데 유용합니다.
    - 다각형 데이터를 표시하는 경우 줄무늬와 같은 패턴을 색상 대신 사용할 수 있습니다. 
- 히트맵, 타일 레이어 및 이미지 레이어와 같은 일부 시각화는 시각 장애가 있는 사용자가 액세스할 수 없습니다. 몇 가지 고려할 사항:
    - 맵에 추가할 때 레이어가 표시되는 내용을 화면 판독기로 설명합니다. 예를 들어 기상 레이더 타일 레이어가 표시되는 경우 화면 판독기에서 "날씨 레이더 데이터가 맵에 오버레이됨"이라고 말하면 됩니다.
- 마우스 가리키기 필요한 기능의 양을 제한합니다. 이러한 기능은 키보드 또는 터치 장치를 사용하여 응용 프로그램과 상호 작용하는 사용자가 액세스할 수 없습니다. 클릭 가능한 아이콘, 링크 및 단추와 같은 대화형 콘텐츠에 대해 호버 스타일을 두는 것이 좋습니다.
- 키보드를 사용하여 응용 프로그램을 탐색해 보십시오. 탭 순서가 논리적인지 확인합니다.
- 바로 가기 키를 만드는 경우 두 키 이하로 제한하십시오.
