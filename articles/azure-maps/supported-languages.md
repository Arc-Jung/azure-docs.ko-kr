# <a name="localization-support-in-azure-maps"></a>Azure Maps에서 지역화 지원

## <a name="azure-maps-supported-views"></a>Azure Maps 지원 보기

> [!Note]
> Azure Maps 다음 국가/지역에서 릴리스할 2019 년 8 월 1 일에서:
>  * 아르헨티나
>  * 인도
>  * 모로코
>  * 파키스탄
>
> 2019 년 8 월 1 일 이후에 **보기** 매개 변수 설정은 위에 나열 된 새 지역/국가 대 한 반환 된 맵 콘텐츠를 정의 합니다. REST Api 및 Sdk 서비스를 사용 하는 필요에 따라 View 매개 변수를 설정 했는지 확인 하는 것이 좋습니다.
>  
>
>  **Rest Api:**
>  
>  필요에 따라 View 매개 변수를 설정 했는지 확인 합니다. View 매개 변수 지정 학적 분쟁 콘텐츠의 집합 Azure Maps 서비스를 통해 반환 됩니다. 
>
>  영향을 받는 Azure Maps REST 서비스:
>    
>    * 지도 타일 가져오기
>    * 지도 이미지 가져오기 
>    * 유사 항목 검색을 이용
>    * 지점 검색을 이용
>    * 검색 POI 범주 가져오기
>    * 근접 검색을 이용
>    * 검색 주소 가져오기
>    * 구조화 된 검색 주소 가져오기
>    * 역방향 주소 검색을 가져오기
>    * 검색 주소 역방향 간 거리 가져오기
>    * 기 하 도형 내 게시물 검색
>    * 게시물 검색 주소 Batch 미리 보기
>    * 검색 주소 역방향 Batch 미리 보기 게시
>    * 경로 따라 게시물 검색
>    * 게시 검색에 대 한 유사 항목 일괄 처리 미리 보기
>
>    
>  **SDKs:**
>
>  필요에 따라 View 매개 변수를 설정한 경우 웹 SDK 및 Android SDK의 최신 버전을 확인 합니다. Sdk을 영향을 받습니다.
>
>    * Azure Maps 웹 SDK
>    * Azure Maps Android SDK


Azure Maps **보기** 매개 변수 ("사용자 지역 매개 변수" 라고도 함)은 해당 국가/지역을 지정 학적 집합이 있는 콘텐츠 분쟁에 대 한 올바른 맵에 표시 되는 ISO 3166 국가 코드는 두 개의 문자 테두리 및 맵에 표시 된 레이블을 포함 하 여 Azure Maps 서비스를 통해 반환 합니다. 

기본적으로 View 매개 변수 설정 **통합**요청에서 정의 하지 않은 경우에 합니다. 것이 사용자의 위치를 확인 하 고 다음 해당 위치에 대해 올바르게 View 매개 변수를 설정 해야 합니다. 설정 하는 옵션이 또는 ' 보기 = Auto', 요청 IP 주소를 기반으로 지도 데이터를 반환 하는 합니다.  Azure Maps에서 View 매개 변수를 사용 하 여 관련 법률을 준수 해야 합니다를 포함 하 여 맵, 이미지 및 기타 데이터와 타사 콘텐츠는는 Azure Maps를 통해 액세스 하도록 승인 하는 있는 국가 관한 매핑을 제공 됩니다.
