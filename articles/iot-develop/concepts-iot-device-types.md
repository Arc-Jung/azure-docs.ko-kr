# <a name="overview-of-azure-iot-device-types"></a>Azure IoT 장치 유형 개요
IoT 장치는 광범위 한 하드웨어 플랫폼에서 선택 됩니다. 데스크톱 컴퓨터에서 볼 수 있는 것 처럼, 가장 작은 8 비트 Mcu 최신 x86 Cpu까지 모두 있습니다. 많은 변수는 IoT 장치에 대해 선택 하는 하드웨어에 대 한 결정에 해당 하며,이 문서에서는 몇 가지 주요 차이점에 대해 설명 했습니다.

## <a name="key-hardware-differentiators"></a>주요 하드웨어 차이점
하드웨어 선택 시의 몇 가지 중요 한 요소는 비용, 전력 소비, 네트워킹, 사용 가능한 입력 및 출력입니다.

* **비용:** 최종 제품을 대량으로 생성 하는 경우 일반적으로 더 저렴 한 장치를 사용 합니다. 그러나이를 절충 하는 것은 매우 제한 된 장치에서 장치 개발 비용이 더 많이 들 수 있다는 것입니다. 개발 비용은 생성 된 모든 장치에 걸쳐 분산 될 수 있으므로 단위당 개발 비용이 낮아집니다.

* **전원:** 장치가 배터리를 활용 하 고 power grid에 연결 되지 않은 경우 장치가 소비 하는 전력의 양에 중요 합니다. Mcu은 낮은 전력 시나리오를 위해 설계 된 경우가 많으며 배터리 수명을 확장 하는 데 더 적합할 수 있습니다.

* **네트워크 액세스:** 장치를 클라우드 서비스에 연결 하는 방법에는 여러 가지가 있습니다. 이더넷, Wi-fi 및 셀룰러 및 사용 가능한 옵션 중 일부입니다. 선택한 연결 유형은 장치가 배포 되는 위치 및 사용 방법에 따라 달라 집니다. 예를 들어, 셀룰러는 높은 검사를 제공 하는 매력적인 옵션 이지만 트래픽이 많은 장치의 경우 비용이 많이 들 수 있습니다. 하드웨어 장비로 연결 이더넷은 저렴 한 데이터 비용을 제공 하지만 이식할 가능성이 낮은 단점이 있습니다.

* **입력 및 출력:** 장치에서 사용 가능한 입력 및 출력은 장치 운영 기능에 직접적인 영향을 줍니다. 일반적으로 마이크로 컨트롤러는 칩에 직접 빌드되는 많은 i/o 함수를 포함 하 고 직접 연결할 수 있는 다양 한 센서를 제공 합니다.

## <a name="microcontrollers-vs-microprocessors"></a>마이크로컨트롤러 vs 마이크로프로세서
IoT 장치는 두 가지 광범위 한 범주 (Mcu)와 마이크로프로세서 (MPUs)로 구분할 수 있습니다.

**Mcu** 는 MPUs 보다 저렴 하 고 작동 하는 것이 더 간단 합니다. MCU는 칩 자체 내에서 메모리, 인터페이스, i/o 등의 많은 함수를 포함 합니다. MPU은 지 원하는 칩의 구성 요소에서이 기능을 그립니다. MCU는 종종 RTOS (실시간 OS)를 사용 하거나 운영 체제 미 설치 (OS 없음)를 실행 하 고 외부 이벤트에 대 한 실시간 응답과 매우 결정적 반응을 제공 합니다.

일반적으로 **MPUs** 는 Windows, Linux 또는 MacOSX와 같은 범용 OS를 실행 하 여 비결 정적 실시간 응답을 제공 합니다. 일반적으로 작업이 완료 되는 것은 보장 되지 않습니다.
