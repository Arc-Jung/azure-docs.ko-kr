# <a name="azure-security-center-for-iot-frequently-asked-questions"></a>IoT용 Azure Security Center 질문과 대답

이 문서에서는 IoT용 Azure 보안 센터에 대해 자주 묻는 질문과 답변 목록을 제공합니다.

## <a name="does-azure-provide-support-for-iot-security"></a>Azure에서 IoT 보안에 대한 지원을 제공합니까?

Azure는 Azure 보안 센터를 통해 전반적인 보안 솔루션의 일부로 IoT 보안을 모니터링하고 관리하기 위한 통합 보기를 제공합니다. 응용 프로그램 개발자인 경우 IoT Hub 보기를 사용하여 IoT 응용 프로그램 보안을 관리할 수 있습니다.

## <a name="what-is-azures-unique-value-proposition-for-iot-security"></a>IoT 보안에 대한 Azure의 고유한 가치 제안은 무엇입니까?

IoT용 Azure 보안 센터를 사용하면 기업이 기존 사이버 보안 보기를 전체 IoT 솔루션으로 확장할 수 있습니다. Azure는 비즈니스 솔루션에 대한 종단 간 보기를 제공하여 엔터프라이즈 보안 태세 및 수집된 데이터를 기반으로 비즈니스 관련 작업 및 의사 결정을 내릴 수 있도록 합니다. Azure IoT, Azure IoT Edge 및 Azure 보안 센터를 사용한 통합 된 보안을 사용하면 필요한 보안으로 원하는 솔루션을 만들 수 있습니다.

## <a name="who-is-azure-security-center-for-iot-made-for"></a>IoT용 Azure 보안 센터는 누구를 위해 만들어졌습니까?

IoT용 Azure 보안 센터는 Azure IoT Hub Security 내에 통합되어 있으며 일상적인 비즈니스 솔루션 보안 작업에 대한 관리를 제공합니다. 또한 IoT용 Azure 보안 센터는 Azure 보안 센터 기능에 통합되어 있으며 전반적인 보안 솔루션의 일부로 IoT 보안을 모니터링하고 관리하기 위한 통합 보기를 제공합니다.

## <a name="how-does-azure-security-center-for-iot-compare-to-the-competition"></a>IoT용 Azure 보안 센터는 경쟁사와 어떻게 비교됩니까?

다른 솔루션은 고객이 자체 솔루션을 만들 수 있는 기능 집합을 제공하지만 IoT용 Azure 보안 센터는 모든 관련 Azure 리소스의 보안전반에 걸쳐 넓은 보기를 제공하는 고유한 종단 간 IoT 보안 솔루션을 제공합니다. Azure를 사용하면 IoT Hub 모듈 트윈을 빠르게 배포하고 전체 통합할 수 있으므로 기존 장치 관리 도구와 쉽게 통합할 수 있습니다.

## <a name="do-i-have-to-be-an-azure-security-center-customer-to-use-this-service"></a>이 서비스를 사용하려면 Azure 보안 센터 고객이어야 합니까?

아니요, 하지만 추천입니다. Azure 보안 센터가 없는 IoT용 Azure 보안 센터는 제한된 연결된 리소스 데이터를 수신하고 잠재적인 공격 표면, 위협 및 잠재적 공격에 대한 제한된 분석을 제공합니다.

## <a name="do-i-have-to-be-an-azure-iot-customer"></a>Azure IoT 고객이어야 합니까?

예. IoT용 Azure 보안 센터는 Azure IoT 연결 및 인프라에 의존합니다.

## <a name="do-i-have-to-install-an-agent"></a>에이전트를 설치해야 합니까?

IoT용 Microsoft Azure 보안 센터를 사용하도록 설정하려면 IoT 장치에 에이전트를 설치해야 하는 것은 아닙니다. 다음 세 가지 옵션 중에서 선택할 수 있으며, 선택한 옵션에 따라 다양한 수준의 보안 모니터링 및 관리 기능을 얻을 수 있습니다.

1. 수정 유무에 관계없이 IoT 보안 에이전트용 Azure 보안 센터를 설치합니다. 이 옵션은 장치 동작 및 액세스에 대한 최고 수준의 향상된 보안 통찰력을 제공합니다.

1. 사용자 고유의 에이전트를 만들고 IoT 보안 메시지 스키마에 대한 Microsoft Azure 보안 센터를 구현합니다. 이 옵션을 사용하면 장치 보안 에이전트 위에 IoT 분석 도구에 대한 Microsoft Azure 보안 센터를 사용할 수 있습니다.

1. IoT 장치에 보안 에이전트를 설치하지 않습니다. 이 옵션을 사용하면 보안 모니터링 및 관리 기능이 감소하여 IoT Hub 통신 모니터링이 가능합니다.

## <a name="what-does-the-azure-security-center-for-iot-agent-do"></a>IoT 에이전트용 Azure 보안 센터는 무엇을 합니까?

IoT 에이전트용 Azure 보안 센터는 장치 구성, 동작 및 액세스(구성 검색), 프로세스 & 연결에 대한 장치 수준 위협 범위를 제공합니다. IoT 보안 에이전트용 Azure 보안 센터는 비즈니스 관련 데이터 또는 활동을 검색하지 않습니다.

## <a name="where-can-i-get-the-azure-security-center-for-iot-security-agent"></a>IoT 보안 에이전트용 Azure 보안 센터는 어디에서 구할 수 있습니까?

IoT 보안 에이전트용 Azure 보안 센터는 오픈 소스이며 GitHub에서 32비트 및 64비트 https://github.com/Azure/Azure-Security-IoTWindows 및 Linux 버전에서 사용할 수 있습니다.

## <a name="where-does-the-azure-security-center-for-iot-agent-get-installed"></a>IoT 에이전트용 Azure 보안 센터는 어디에서 설치됩니까?

자세한 설치 및 에이전트 배포 정보는 GitHub에서 찾을 수 https://github.com/Azure/Azure-IoT-Security있습니다.

## <a name="what-are-the-dependencies-and-prerequisites-of-the-agent"></a>에이전트의 종속성 및 전제 조건은 무엇입니까?

IoT용 Azure 보안 센터는 다양한 플랫폼을 지원합니다. [지원되는 장치 플랫폼을](how-to-deploy-agent.md) 참조하여 특정 장치에 대한 지원을 확인합니다.

## <a name="which-data-is-collected-by-the-agent"></a>에이전트가 수집하는 데이터는 무엇입니까?

연결, 액세스, 방화벽 구성, 프로세스 목록 & OS 기준은 에이전트에 의해 수집됩니다.

## <a name="how-much-data-will-the-agent-generate"></a>에이전트가 생성할 데이터의 양

에이전트 데이터 생성은 장치, 응용 프로그램, 연결 유형 및 고객 에이전트 구성에 의해 구동됩니다. 장치와 IoT 솔루션 간의 가변성이 높기 때문에 먼저 랩 또는 테스트 설정에 에이전트를 배포하여 생성된 데이터의 양을 측정하는 동안 필요에 맞는 특정 구성을 관찰, 학습 및 설정하는 것이 좋습니다. 서비스를 시작한 후 IoT용 Azure 보안 센터 에이전트는 구성 및 사용자 지정 프로세스를 처리하는 데 도움이 되는 에이전트 처리량을 최적화하기 위한 운영 권장 사항을 제공합니다.

## <a name="how-can-i-control-my-billing"></a>청구를 제어하는 방법은 무엇입니까?

IoT용 Azure 보안 센터는 구성 가능한 에이전트 검사, 데이터 버퍼 및 에이전트에서 생성된 데이터의 양을 늘리거나 줄이는 사용자 지정 경고를 만드는 기능을 제공합니다.

## <a name="do-agent-messages-use-up-quota-from-iot-hub"></a>에이전트 메시지는 IoT Hub에서 할당량을 사용합니까?

예. 에이전트 전송 데이터는 IoT Hub 할당량에 계산됩니다.

## <a name="what-next-ive-installed-an-agent-and-dont-see-any-activities-or-logs"></a>다음 단계 에이전트를 설치했는데 활동이나 로그가 표시되지 않습니다...

1. 에이전트 [유형이 장치의 지정된 OS 플랫폼에 맞는지](how-to-deploy-agent.md) 확인

1. [에이전트가 장치에서 실행 중이지 확인합니다.](how-to-agent-configuration.md)

1. IoT Hub에서 **보안에** [서비스가 성공적으로 활성화되어](quickstart-onboard-iot-hub.md) 있는지 확인합니다.

1. IoT 모듈용 Azure 보안 센터를 통해 장치가 [IoT Hub에서 구성되어](quickstart-create-security-twin.md)있는지 확인합니다.

활동 또는 로그를 계속 사용할 수 없는 경우 Azure 보안 IoT 파트너에게 문의하여 추가 도움을 요청하십시오.

## <a name="what-happens-when-the-internet-connection-stops-working"></a>인터넷 연결이 작동을 중지하면 어떻게 되나요?

에이전트는 장치가 실행되는 동안 데이터를 계속 실행하고 저장합니다. 데이터는 크기 구성에 따라 보안 메시지 캐시에 저장됩니다. 장치가 연결을 다시 하면 보안 메시지 전송이 다시 시작됩니다.

## <a name="if-the-device-is-restarted-will-the-security-agent-self-recover"></a>장치가 다시 시작되면 보안 에이전트가 자체 복구합니까?

보안 에이전트는 각 장치를 다시 시작할 때마다 자동으로 다시 실행되도록 설계되었습니다.

## <a name="can-the-agent-affect-the-performance-of-the-device-or-other-installed-software"></a>에이전트가 장치 또는 설치된 다른 소프트웨어의 성능에 영향을 줄 수 있습니까?

에이전트는 다른 응용 프로그램/프로세스로 컴퓨터 리소스를 사용하며 정상적인 장치 작업을 중단해서는 안 됩니다. 에이전트가 실행되는 장치의 리소스 소비는 설정 및 구성과 결합됩니다. 프로덕션 환경에서 배포하기 전에 다른 IoT 응용 프로그램 및 기능과의 상호 운용성과 함께 포함된 환경에서 에이전트 구성을 테스트하는 것이 좋습니다.

## <a name="im-making-some-maintenance-on-the-device-can-i-turn-off-the-agent"></a>장치에 약간의 유지 보수를 하고 있습니다. 에이전트를 끌 수 있습니까?

에이전트를 끌 수 없습니다.

## <a name="is-there-a-way-to-test-if-the-agent-is-working-correctly"></a>에이전트가 올바르게 작동하는지 테스트하는 방법이 있습니까?

에이전트가 통신을 중지하거나 보안 메시지를 보내지 못하면 **장치가 자동 경고가** 생성됩니다.

## <a name="can-i-create-my-own-alerts"></a>내 자체 경고를 만들 수 있습니까?

예. IP 주소 및 열린 포트와 같은 미리 결정된 동작 집합에 사용자 지정된 경고를 설정할 수 있습니다. [사용자 지정 경고 만들기를](quickstart-create-custom-alerts.md) 참조하여 사용자 지정 경고 및 알림을 만드는 방법에 대해 자세히 알아봅니다.

## <a name="where-can-i-see-logs-can-i-customize-logs"></a>로그는 어디에서 볼 수 있나요? 로그를 사용자 지정할 수 있나요?

- 연결된 Log Analytics 작업 영역을 사용하여 경고 및 권장 사항을 볼 수 있습니다. 작업 영역에서 저장소 크기와 기간을 구성합니다.

- 보안 에이전트의 원시 데이터는 Log Analytics 계정에도 저장할 수 있습니다. 이 옵션의 구성을 변경하기 전에 크기, 기간, 저장소 요구 사항 및 관련 비용을 고려합니다.

## <a name="why-should-i-add-azure-security-center-for-iot-to-the-module-identity-what-is-it-used-for"></a>IoT용 Azure 보안 센터를 모듈 ID에 추가해야 하는 이유는 무엇입니까? 용도는?

IoT용 Azure 보안 센터 모듈은 에이전트 구성 및 관리에 사용됩니다.
