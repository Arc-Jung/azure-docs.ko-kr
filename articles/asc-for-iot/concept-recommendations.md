# <a name="security-recommendations"></a>보안 권장 사항

IoT용 Azure 보안 센터는 Azure 리소스 및 IoT 장치를 검사하고 공격 표면을 줄이기 위한 보안 권장 사항을 제공합니다.
보안 권장 사항은 실행 가능하며 고객이 보안 모범 사례를 준수할 수 있도록 지원하는 것을 목표로 합니다.

이 문서에서는 IoT Hub 및/또는 IoT 장치에서 트리거할 수 있는 권장 사항 목록을 찾을 수 있습니다.

## <a name="recommendations-for-iot-devices"></a>IoT 장치에 대한 권장 사항

장치 권장 사항은 장치 보안 상태를 개선하기 위한 통찰력과 제안을 제공합니다.

| 심각도 | 속성                                                      | 데이터 원본 | 설명                                                                                                                                                                                           |
|----------|-----------------------------------------------------------|-------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 중간   | 장치에서 포트 열기                                      | 에이전트       | 장치에서 수신 대기 끝점이 발견되었습니다.                                                                                                                                                        |
| 중간   | 체인 중 하나에서 발견되는 허용 방화벽 정책입니다. | 에이전트       | 허용된 방화벽 정책(입력/출력)을 찾습니다. 방화벽 정책은 기본적으로 모든 트래픽을 거부하고 장치에서 필요한 통신을 허용하는 규칙을 정의해야 합니다.                               |
| 중간   | 입력 체인에서 허용 방화벽 규칙이 발견되었습니다.     | 에이전트       | 방화벽의 규칙이 광범위한 IP 주소 또는 포트에 허용되는 패턴을 포함하는 것으로 확인되었습니다.                                                                                    |
| 중간   | 출력 체인에서 허용 방화벽 규칙이 발견되었습니다.    | 에이전트       | 방화벽의 규칙이 광범위한 IP 주소 또는 포트에 허용되는 패턴을 포함하는 것으로 확인되었습니다.                                                                                   |
| 중간   | 운영 시스템 기준 유효성 검사가 실패했습니다.           | 에이전트       | 장치는 [CIS Linux 벤치마크를](https://www.cisecurity.org/cis-benchmarks/)준수하지 않습니다.                                                                                                        |

### <a name="operational-recommendations-for-iot-devices"></a>IoT 장치에 대한 운영 권장 사항

운영 권장 사항은 보안 에이전트 구성을 개선하기 위한 통찰력과 제안을 제공합니다.

| 심각도 | 속성                                    | 데이터 원본 | 설명                                                                       |
|----------|-----------------------------------------|-------------|-----------------------------------------------------------------------------------|
| 낮음      | 에이전트가 사용하지 않은 메시지를 보냅니다.          | 에이전트       | 지난 24시간 동안 보안 메시지의 10% 이상이 4KB보다 작습니다.  |
| 낮음      | 보안 트윈 구성이 최적이 아님 | 에이전트       | 보안 쌍 구성이 최적이 아닙니다.                                        |
| 낮음      | 보안 쌍 구성 충돌    | 에이전트       | 보안 쌍 구성에서 충돌이 확인되었습니다. |                          |
|

## <a name="recommendations-for-iot-hub"></a>IoT 허브에 대한 권장 사항

권장 사항 경고는 환경의 보안 상태를 개선하기 위한 작업에 대한 통찰력과 제안을 제공합니다.

| 심각도 | 속성                                                     | 데이터 원본 | 설명                                                                                                                                                                                                             |
|----------|----------------------------------------------------------|-------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 높음     | 여러 장치에서 사용되는 동일한 인증 자격 증명 | IoT Hub     | IoT Hub 인증 자격 증명은 여러 장치에서 사용됩니다. 합법적인 장치를 가장하는 불법 장치를 나타낼 수 있습니다. 중복 자격 증명 사용은 악의적인 행위자에 의한 장치 가장의 위험을 증가시킵니다. |
| 중간   | 기본 IP 필터 정책은 거부되어야 합니다.                  | IoT Hub     | IP 필터 구성에는 허용된 트래픽에 대해 정의된 규칙이 있어야 하며 기본적으로 다른 모든 트래픽을 거부해야 합니다.                                                                                                     |
| 중간   | IP 필터 규칙에는 큰 IP 범위가 포함됩니다.                   | IoT Hub     | 허용 IP 필터 규칙 소스 IP 범위가 너무 큽니다. 지나치게 허용되는 규칙은 IoT 허브를 악의적인 행위자에게 노출시킬 수 있습니다.                                                                                       |
| 낮음      | IoT Hub에서 진단 로그 사용                       | IoT Hub     | 로그를 사용하도록 설정하고 최대 1년간 보존합니다. 로그를 유지하면 보안 사고가 발생하거나 네트워크가 손상된 경우 조사를 위해 활동 추적을 다시 만들 수 있습니다.                                       |
|
