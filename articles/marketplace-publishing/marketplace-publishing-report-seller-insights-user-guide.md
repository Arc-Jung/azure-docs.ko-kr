# <a name="how-to-use-the-microsoft-azure-marketplace-seller-insights-portal"></a>Microsoft Azure Marketplace Seller Insights 포털을 사용하는 방법

##<a name="signing-in"></a>로그인

관리자는 계정이 만들어진 메일을 통해 판매자에게 조언하고 Seller Insights 포털의 URL을 판매자와 공유해야 합니다(https://reports.azure.com).

판매자는 관리자가 프로필을 설정할 때 어떤 계정을 사용했는지에 따라 개인 Microsoft 계정 또는 회사 계정을 사용하여 Seller Insights 포털에 로그인할 수 있습니다.  관리자는 어떤 계정을 사용할지 판매자에게 조언해야 합니다.

   ![계정 형식][1]



***Seller Insights 포털을 시작합니다.***

1. 로그인하면 ‘요약’ 페이지로 이동합니다.
2. 잠시 시간을 내셔서 개인 정보 취급 방침을 검토하세요.
3. 언제든지 Seller Insights 포털에 관련된 의문 사항이 포함된 메일을 통해 '문의'할 수 있습니다.
4. 페이지 맨 위에 있는 탭을 사용하여 Seller Insights 포털을 탐색할 수 있습니다.  Seller Insights 포털에서 현재 위치를 확인하려면 이 가이드의 페이지 맨 위에 있는 탭을 찾아 보세요.
5. 언제든지 스마일을 클릭하여 Seller Insights 포털에 관련된 피드백을 보냅니다.
6. 오른쪽 위 모퉁이에 있는 자신의 이름을 클릭하여 언제든지 Seller Insights 포털에서 로그아웃할 수 있습니다.

   ![기본 로그인 페이지][2]



##<a name="resetting-a-password"></a>암호 재설정

***계정 및 암호 유효성 검사***

1. ***사용 중인 계정이 등록되어 있고 암호가 작동하지 않는지 확인합니다.*** Azure Active Directory에 등록된 계정을 입력하면 회사 또는 학교(조직 계정) 계정이나 개인 LiveID 계정으로 로그인하는 방법을 선택하는 프롬프트가 자동으로 표시됩니다.
2. https://reports.azure.com으로 이동하고 이전에 로그인한 계정에서 로그아웃하거나, 비공개 창에서 페이지를 열어 로그인된 다른 계정이 없는지 확인합니다.
3. 암호를 입력하지 않고 게시자 메일 주소를 입력합니다. Azure Active Directory에 등록된 경우(관리자 계정은 관리자 계정이 설정한 조직 계정이 등록되는 것처럼 등록됨) ***4단계***로 리디렉션됩니다.
4. "회사 또는 학교 계정”을 선택합니다. 회사 또는 학교 계정(‘조직 계정’)은 Azure에 등록된 회사에서 설정합니다. 개인 계정(‘LiveID’)은 Microsoft에 등록된 개인 계정입니다.
 * 사용자가 이전에 회사 메일 주소를 사용하여 LiveID 계정을 설정하고, 회사가 나중에 Azure로 온보딩되고, 같은 메일 주소가 조직 계정으로 등록되었을 수 있습니다. 이는 이 사용자가 LiveID를 만드는 데 사용한 암호가 포함된 LiveID 계정과 회사에서 사용하는 별도 암호가 포함된 조직 계정을 보유할 수 있음을 의미합니다.
5. Seller Insights 포털의 경우 "회사 또는 학교 계정"을 선택할 때 LiveID/암호 조합이 아닌 조직 계정/암호 조합을 사용 중인지 확인해야 합니다.
6. 조직 계정의 경우 게시자 로그인 페이지로 리디렉션됩니다.
7. 여기에서 조직 계정의 암호가 작동하는지 여부를 확인합니다. 작동하지 않을 경우 ***암호 재설정***으로 이동합니다.

***암호 재설정***

아래 링크에 따라 [셀프 서비스 암호 재설정] 마법사를 사용하여 암호를 다시 설정해 보세요.
 * 참고: 관리자 계정이 설정한 조직 계정에 대해 관리자가 사용자가 직접 암호를 재설정할 수 있는 기능을 켠 경우에는 사용자가 암호를 재설정할 수 있습니다. 해당 기능을 켜지 않은 경우에는 관리자에게 문의할 수 있는 암호 재설정 양식이 제공됩니다.
