# Azure SQL 데이터베이스 V12로 업그레이드 계획 및 준비

이 항목에서는 Azure SQL 데이터베이스를 버전 V11에서 V12로 업그레이드하기 위해 수행해야 하는 계획 및 준비 작업에 대해 설명합니다.


새 [Azure 포털](http://portal.azure.com/)을 사용하여 V12로 업그레이드를 지원할 수 있습니다.


> [AZURE.NOTE]
> 테스트 데이터베이스, 데이터베이스 복사본 또는 새 데이터베이스를 미리 보기로 업그레이드하는 것이 좋습니다. 업무상 사용해야 하는 프로덕션 데이터베이스의 경우에는 미리 보기 기간이 끝날 때까지 기다려야 합니다.<br/><br/>

## A. 미리 계획


다음 하위 섹션에서는 Azure SQL 데이터베이스를 V12로 업그레이드하기 위해 작업을 수행하기 전에 처리해야 하는 학습 및 의사 결정에 대해 설명합니다.

### A.1 버전 확인


이 문서에서는 Microsoft Azure SQL 데이터베이스를 버전 V11에서 V12로 업그레이드하는 경우를 다룹니다. 보다 형식적으로, 버전 번호는 Transact-SQL 문 **SELECT @@version;**에서 보고된 다음 두 값에 가깝습니다.


- 11.0.9228.18*(V11)*
- 12.0.2000.8*(또는 좀 더 높은 버전, V12)*


### A.2 서비스 계층 계획


V12부터 Azure SQL 데이터베이스는 기본, 표준 및 프리미엄이라는 서비스 계층만 지원합니다. Web 및 Business 서비스 계층은 V12에서 지원되지 않으며 따라서 현재 Web 또는 Business 서비스 계층에 있는 Azure SQL 데이터베이스를 업그레이드할 계획인 경우 새 서비스 계층을 결정해야 합니다.


기본, 표준 및 프리미엄 서비스 계층에 대한 자세한 내용은 다음을 참조하세요.


- [SQL 데이터베이스 Web/Business 데이터베이스를 새 서비스 계층으로 업그레이드](http://azure.microsoft.com/documentation/articles/sql-database-upgrade-new-service-tiers/)
- [Azure SQL 데이터베이스 가격](http://azure.microsoft.com/pricing/details/sql-database/)


### A.3 지역에서 복제 구성 검토


Azure SQL 데이터베이스가 지역에서 복제에 대해 구성된 경우 업그레이드 준비 작업을 시작하기 전에 현재 구성을 문서화하고 지역에서 복제를 중지해야 합니다. 업그레이드가 완료된 후 지역에서 복제에 대해 데이터베이스를 다시 구성해야 합니다.


전략은 원본을 그대로 두고 데이터베이스 복사본을 테스트하는 것입니다.


### A.4 바로 업그레이드 및 데이터베이스를 새 서버로 복사


Azure SQL 데이터베이스를 V11에서 V12로 업그레이드하기 위해 선택할 수 있는 두 가지 기술이 있습니다.


- 바로 업그레이드: 이 프로세스가 완료되면 V11 데이터베이스가 더 이상 존재하지 않습니다. SQL 데이터베이스 서버가 V12입니다.
- 데이터베이스를 V12 서버로 복사: 이 프로세스는 V11 데이터베이스를 그대로 두고 V12 Azure SQL 데이터베이스 서버에 V11 데이터베이스의 업그레이드된 복사본을 만듭니다. <br/> 클라이언트 코드에서 데이터베이스에 액세스하는 데 사용되는 DNS 정보가 더 이상 V11에 액세스하지 않으며 V11 데이터베이스에 액세스하려면 DNS 정보를 편집해야 합니다.


## B. 준비 작업


계획을 완료한 후에는 다음 하위 섹션에 설명된 작업 단계를 수행하여 최종 업그레이드 단계를 준비할 수 있습니다.


최종 업그레이드 단계에 대한 자세한 설명은 이 도움말 항목의 맨 위에 링크된 도움말 항목에서 확인할 수 있습니다.


### B.1 서비스 계층 작업


Web 및 Business 서비스 가격 계층은 V12에서 지원되지 않습니다.


V11 Azure SQL 데이터베이스가 Web 또는 Business 데이터베이스인 경우 업그레이드 프로세스에서 데이터베이스를 지원되는 계층으로 전환할 수 있게 해줍니다. 업그레이드 중 데이터베이스의 작업 기록에 맞는 계층이 권장됩니다. 그러나 지원되는 모든 계층을 선택할 수 있습니다.


업그레이드를 시작하기 전에 Web 및 Business 계층에서 V11 데이터베이스를 전환하여 업그레이드 중 필요한 단계를 줄일 수 있습니다. 이 작업은 Azure 포털에서 수행할 수 있습니다.


전환할 서비스 계층이 확실하지 않은 경우 표준 계층의 S0 수준이 적절한 초기 선택일 수 있습니다.


### B.2 업그레이드 중 지역에서 복제 일시 중단


데이터베이스에 대해 지역에서 복제가 활성화된 경우 V12로 업그레이드를 실행할 수 없습니다. 먼저 지역에서 복제 사용을 중지하도록 데이터베이스를 다시 구성해야 합니다.


업그레이드가 완료되면 지역에서 복제를 다시 사용하도록 데이터베이스를 구성할 수 있습니다.


## <a id="limitations"></a>C. V12로 업그레이드하는 중과 업그레이드한 후의 제한 사항


이 섹션에서는 Azure SQL 데이터베이스 V12로 업그레이드하는 작업과 관련된 제한 사항에 대해 설명합니다.


### C.1 V12용 미리 보기 포털


다음 두 Azure 관리 포털 중 첫 번째만 V12 데이터베이스를 지원합니다.


- [http://portal.azure.com/](http://portal.azure.com/)
 - 이 새 포털은 미리 보기 상태이며, 아직 GA(일반 공급)되지 않았습니다.<br/><br/>
- [http://manage.windowsazure.com/](http://manage.windowsazure.com/)
 - 이 이전 포털은 V12를 지원하도록 업데이트되지 않습니다.


고객은 VS2013(Visual Studio 2013)을 사용하여 Azure SQL 데이터베이스에 연결하는 것이 좋습니다. VS2013은 다음과 같은 작업에 사용될 수 있습니다.


- T-SQL 문 실행
- 스키마 디자인
- 온라인 또는 오프라인으로 데이터베이스 개발


대신 완전한 기능을 갖춘 무료 버전의 VS2013인 [Visual Studio Community 2013](https://www.visualstudio.com/ko-kr/news/vs2013-community-vs.aspx/)에 연결할 수 있습니다.


이전 Azure 관리 포털의 데이터베이스 페이지에서 **Visual Studio에서 열기**를 클릭하여 Azure SQL 데이터베이스에 연결할 컴퓨터에서 VS2013을 시작할 수 있습니다.


또는 SSMS(SQL Server Management Studio) 2014 [CU6](http://support.microsoft.com/kb/3031047/)을 사용하여 Azure SQL 데이터베이스에 연결할 수 있습니다. 자세한 내용은 블로그 게시물<br/>[Azure SQL 데이터베이스용 클라이언트 도구 업데이트](http://azure.microsoft.com/blog/2014/12/22/client-tooling-updates-for-azure-sql-database/)를 참조하세요.


### C.2 V12로 업그레이드 *during* 제한 사항

다음 표에서는 업그레이드 프로세스 중에 적용되는 제한 사항에 대해 설명합니다.


| 제한 사항 | 설명 |
| :--- | :--- |
| 업그레이드 기간 | 크기가 50GB를 초과하는 대형 데이터베이스의 경우 업그레이드 프로세스에 최대 24시간이 걸릴 수 있습니다. |
| DNS 항목 업데이트 지연 | 업그레이드가 완료된 후 시스템에서 클라이언트 응용 프로그램의 연결을 위해 V12 데이터베이스에 대한 DNS 항목을 업데이트하는 데 몇 분 정도 걸립니다. |
| 데이터베이스 생성 안 함 | 업그레이드가 진행되는 동안 대상 V12 Azure SQL 데이터베이스 서버에서 데이터베이스를 만들기 위한 다음 작업을 사용할 수 없습니다. <p></p> * 새 데이터베이스 만들기 <br/> * 서버에 데이터베이스 복사 <br/> * 삭제된 데이터베이스 복원 <br/> * 지정 시간으로 데이터베이스 복원 <br/><br/> 그러나 업그레이드 중 지정 시간 복원 지원은 V12 미리 보기 기간이 끝나기 전에는 지원될 수 있는 기능입니다. |
| 지역에서 복제 안 함 | 현재 V11에서 업그레이드와 관련된 V12 서버에서는 지역에서 복제가 지원되지 않습니다. |
| 경고 규칙 | V12 데이터베이스에는 경고 규칙을 추가할 수 없습니다. |


### C.3 V12로 업그레이드 *after* 제한 사항

다음 표에서는 업그레이드 프로세스 후에 적용되는 제한 사항에 대해 설명합니다.


| 제한 사항 | 설명 |
| :--- | :--- |
| V11 데이터베이스에 대한 DNS | V11 데이터베이스를 새 V12 서버로 복사하는 기술로 업그레이드를 완료하면 클라이언트 코드에서 연결하는 데 사용하는 DNS 정보가 V12 데이터베이스를 참조합니다. <p> </p> 클라이언트 코드에서 다시 V11 데이터베이스를 참조하려면 편집된 DNS 정보를 사용해야 합니다. |
| V11로 되돌릴 수 없음 | 바로 업그레이드 후에는 결과를 되돌리거나 실행 취소할 수 없습니다. |
| Web 또는 Business 계층 | 업그레이드가 시작되고 나면 새 V12 데이터베이스에 대한 서버에서 Web 또는 Business 서비스 계층을 더 이상 인식하거나 허용할 수 없습니다. |
| Azure 포털의 가격 계층 카드에 50% 할인이 반영되지 않음 | 미리 보기 기간 중에는 최신 Azure SQL 데이터베이스 미리 보기 업데이트(V12)에 등록된 데이터베이스가 50% 미리 보기 할인* 됩니다. 미리 보기 포털의 서비스 가격 계층에는 할인이 표시되지 않지만 할인이 적용됩니다.<br/><br/> 50% 할인 혜택은 모든 지역에 대해 **2015년 3월 31일**에 만료될 때까지 모든 지역에서 적용됩니다. 데이터베이스가 GA(일반 공급) 상태로 발표된 지역에서도 할인이 적용됩니다.<br/><br/> * 최신 Azure SQL 데이터베이스 업데이트 V12 기능 사용 시에는 사용권 계약(예: 기업 계약, Microsoft Azure 계약 또는 Microsoft 온라인 구독 계약)의 미리 보기 약관과 해당하는 [Microsoft Azure 미리 보기 추가 사용 특약](http://azure.microsoft.com/support/legal/preview-supplemental-terms/)이 적용됩니다.  미리 보기 기간 동안 Microsoft 또는 재판매인(해당하는 경우)은 이 미리 보기에 등록된 모든 데이터베이스에 대해 50% 미리 보기 할인을 적용하여 GA(일반 공급) 가격의 절반에 해당하는 요금을 청구합니다. Microsoft는 미리 보기 기간 및 미리 보기 할인가 적용 만료 30일 전에 전자 메일을 통해 알림을 제공합니다. |


### C.4 V12로 업그레이드한 *after*의 내보내기 및 가져오기


[Azure 웹 포털](http://portal.azure.com/)을 사용하여 V12 데이터베이스를 내보내거나 가져올 수 있습니다. 또한 다음 중에서 원하는 도구를 사용할 수도 있습니다.


- SSMS(SQL Server Management Studio)
- Visual Studio 2013
- 데이터 계층 응용 프로그램 프레임워크(DacFX)


그러나 이러한 도구를 사용하려면 먼저 해당 도구가 새 V12 기능을 지원하도록 최신 업데이트를 설치해야 합니다.


- [SQL Server Management Studio 2014 누적 업데이트 6](http://support2.microsoft.com/kb/3031047)
- [Visual Studio 2013의 SQL Server 데이터베이스 2015년 2월 업데이트](https://msdn.microsoft.com/data/hh297027)
- [Azure SQL 데이터베이스 V12의 2015년 2월 데이터 계층 응용 프로그램 프레임워크(DacFx)](http://www.microsoft.com/download/details.aspx?id=45886)


> [AZURE.NOTE] 위 도구 링크는 2015년 3월 2일부터 업데이트되었습니다. 이러한 도구의 이 새 업데이트를 사용하는 것이 좋습니다.


#### C.4.1 bacpac 파일 가져오기 관련 일시적 문제


 (2015년 2월 18일 수요일 현재)


V12로 업그레이드된 Azure SQL 데이터베이스 서버에서 bacpac 파일을 내보낼 때 알려진 문제가 발생합니다. 즉, 내보낸 bacpac 파일에 잘못된 script_deployment_databases 개체가 포함됩니다. 이 잘못된 개체를 포함하는 bacpac 파일은 SSMS(SQL Server Management Studio), SqlPackage.exe 또는 데이터 계층 응용 프로그램 프레임워크(DacFX) API와 같은 도구를 사용하여 가져올 수 없습니다.


그러나 Azure 웹 포털을 사용하여 해당 bacpac 파일을 Azure SQL 데이터베이스로 가져올 수는 있습니다. 2015년 2월 27일(금요일) 쯤 해당하는 도구의 업데이트를 비롯하여 이 문제에 대한 영구적인 수정 사항이 출시될 예정입니다. 출시 전에 파일 이름 확장명에 .bacpac가 들어 있는 이미 내보낸 파일을 복구하기 위해 추가 지원이 필요한 경우 Microsoft 지원에 문의하세요.


이러한 내보내기 문제와 함께, 최근 버전 V12로 업그레이드한 일부 서버와 고객의 경우 bacpac 파일을 가져오려고 하면 다른 오류가 발생할 수도 있습니다. 이 사용 권한 관련 오류는 일시적인 현상이며 해당하는 서버에서 보통 1일 이내에 해결됩니다. 이 문제도 2015년 2월 23일(월요일)이 시작되는 주 중에 영구적으로 해결될 것으로 예상됩니다. 현재는 가져오기 작업을 다시 시도하면 성공할 수도 있습니다. bacpac 파일을 Azure SQL 데이터베이스로 가져오는 데 추가 지원이 필요하면 Microsoft 지원에 문의하세요.


필요한 경우 다음 단계를 수행하여 Microsoft 지원에 문의할 수 있습니다.


1. Azure 포털로 이동합니다.
2. 오른쪽 위에 표시되어 있는 계정 이름을 마우스 오른쪽 단추로 클릭합니다.
3. 상황에 맞는 메뉴가 표시되면 지원 항목을 클릭합니다.
 - 해당 항목의 레이블은 **Microsoft 지원에 문의** 또는 **도움말 + 지원**입니다.


> [AZURE.NOTE] (2015년 3월 2일 월요일)
> 
> 이 C.4.1 섹션에 설명된 문제는 해결되었습니다. 고객은 특별한 지식이나 작업 없이 계속 사용할 수 있습니다.
> 
> 자동화된 가져오기/내보내기가 bacpac 파일의 기간에 상관없이 정상적으로 작동합니다.
> 
> DacFx를 사용하여 bacpac 파일을 처리하는 고객은 업데이트된 도구를 다운로드해야 합니다. 다운로드 링크는 C.4 섹션에서 확인할 수 있습니다.


### C.5 삭제한 V11 데이터베이스를 V12로 복원

다음 시나리오에서는 삭제된 V11 Azure SQL 데이터베이스를 V12 Azure SQL 데이터베이스 서버로 복원할 수 있음을 설명합니다.

1. V11 Azure SQL 데이터베이스 서버가 있다고 가정합니다. <br/> 서버에서 사용되지 않는 Web 및 Business 서비스 계층에 데이터베이스가 있습니다.
2. 데이터베이스를 삭제합니다.
3. 서버를 V12로 업그레이드합니다.
4. 그런 다음 데이터베이스를 서버로 복원합니다. <br/> 데이터베이스가 표준 서비스 계층의 S0 수준에서 V12로 업그레이드됩니다.
5. S0을 사용하지 않으려는 경우 지원되는 임의 서비스 계층으로 데이터베이스를 전환할 수 있습니다.



## D. 실패 해결

이상한 이유로 업그레이드가 실패할 경우 V11 데이터베이스가 활성 상태로 유지되고 정상적으로 사용할 수 있습니다.
