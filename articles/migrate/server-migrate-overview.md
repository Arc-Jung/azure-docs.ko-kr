# <a name="select-a-vmware-migration-option"></a>VMware 마이그레이션 옵션 선택

Azure Migrate 서버 마이그레이션 도구를 사용하여 VMware VM을 Azure로 마이그레이션할 수 있습니다. 이 도구는 VMware VM 마이그레이션을 위해 다음 두 가지 옵션을 제공합니다.

- 에이전트 없는 복제를 통한 마이그레이션. VM에 아무 것도 설치하지 않고 해당 VM을 마이그레이션합니다.
- 복제용 에이전트를 통한 마이그레이션. VM에 복제용 에이전트를 설치합니다.




## <a name="compare-migration-methods"></a>마이그레이션 방법 비교

이러한 선택된 비교를 사용하여 사용할 방법을 결정할 수 있습니다. [에이전트 없는](migrate-support-matrix-vmware-migration.md#agentless-vmware-servers) 및 [에이전트 기반](migrate-support-matrix-vmware-migration.md#agent-based-vmware-servers) 마이그레이션에 대한 전체 지원 요구 사항을 검토할 수도 있습니다.

**설정** | **에이전트 없음** | **에이전트 기반**
--- | --- | ---
**Azure 권한** | Azure 마이그레이션 프로젝트를 만들고 Azure 마이그레이션 어플라이언스를 배포할 때 생성된 Azure AD 앱을 등록하려면 권한이 필요합니다. | Azure 구독에 대한 참여자 권한이 필요합니다. 
**동시 복제** | vCenter 서버에서 최대 100개의 VM을 동시에 복제할 수 있습니다.<br/> 마이그레이션을 위해 VM이 50개 이상인 경우 VM의 여러 일괄 처리를 만듭니다.<br/> 한 번에 더 복제하면 성능에 영향을 미칩니다. | 해당 없음
**어플라이언스 배포** | [Azure 마이그레이션 어플라이언스는](migrate-appliance.md) 온-프레미스에 배포됩니다. | [Azure 마이그레이션 복제 어플라이언스가](migrate-replication-appliance.md) 온-프레미스에 배포됩니다.
**사이트 복구 호환성** | 호환. | 사이트 복구를 사용하여 컴퓨터에 대한 복제를 설정한 경우 Azure 마이그레이션 서버 마이그레이션을 사용하여 복제할 수 없습니다.
**대상 디스크** | 관리 디스크 | 관리 디스크
**디스크 제한** | OS 디스크: 2TB<br/><br/> 데이터 디스크: 4TB<br/><br/> 최대 디스크: 60 | OS 디스크: 2TB<br/><br/> 데이터 디스크: 8TB<br/><br/> 최대 디스크: 63
**통과 디스크** | 지원되지 않음 | 지원됨
**UEFI 부츠** | 지원되지 않음 | Azure에서 마이그레이션된 VM은 BIOS 부팅 VM으로 자동으로 변환됩니다.<br/><br/> OS 디스크에는 최대 4개의 파티션이 있어야 하며 볼륨은 NTFS로 포맷되어야 합니다.


## <a name="deployment-steps-comparison"></a>배포 단계 비교

제한 을 검토한 후 각 솔루션을 배포하는 단계를 이해하면 선택할 옵션을 결정하는 데 도움이 될 수 있습니다.

**작업** | **세부 정보** |**에이전트 없음** | **에이전트 기반**
--- | --- | --- | ---
**평가** | 마이그레이션 하기 전에 서버를 평가 합니다.  평가는 선택 사항입니다. 컴퓨터를 마이그레이션하기 전에 평가하는 것이 좋지만 그렇게 할 필요는 없습니다. <br/><br/> 평가를 위해 Azure Migrate는 VM을 검색하고 평가하기 위해 경량 어플라이언스를 설정합니다. | 평가 후 에이전트 없는 마이그레이션을 실행 하는 경우 평가에 대 한 설정 된 동일한 Azure 마이그레이션 어플라이언스는 에이전트 없는 마이그레이션에 사용 됩니다.  |  평가 후 에이전트 기반 마이그레이션을 실행하는 경우 평가를 위해 설정된 어플라이언스는 에이전트 없는 마이그레이션 중에 사용되지 않습니다. 추가 검색 및 평가를 수행하지 않으려면 어플라이언스를 그대로 두거나 제거할 수 있습니다.
**마이그레이션을 위해 VMware 서버 및 VM 준비** | VMware 서버 및 VM에서 여러 설정을 구성합니다. | 필수 | 필수
**서버 마이그레이션 도구 추가** | Azure 마이그레이션 프로젝트에서 Azure 마이그레이션 서버 마이그레이션 도구를 추가합니다. | 필수 | 필수
**Azure 마이그레이션 어플라이언스 배포** | VM 검색 및 평가를 위해 VMware VM에 경량 어플라이언스를 설정합니다. | 필수 | 필요하지 않습니다.
**VM에 모빌리티 서비스 설치** | 복제하려는 각 VM에 모빌리티 서비스 설치 | 필요하지 않음 | 필수
**Azure 마이그레이션 서버 마이그레이션 복제 어플라이언스 배포** | VM을 검색하기 위해 VM웨어 VM에 어플라이언스를 설정하고 VM에서 실행되는 모빌리티 서비스와 Azure 마이그레이션 서버 마이그레이션 간에 브리지 | 필요하지 않음 | 필수
**VM을 복제합니다.** VM 복제를 사용하도록 설정. | 복제 설정을 구성하고 복제할 VM을 선택합니다. | 필수 | 필수
**테스트 마이그레이션 실행** | 테스트 마이그레이션을 실행하여 모든 것이 예상대로 작동하는지 확인합니다. | 필수 | 필수
**전체 마이그레이션 실행** | VM을 마이그레이션합니다. | 필수 | 필수
