# <a name="assessments-in-azure-migrate-server-assessment"></a>Azure 마이그레이션의 평가: 서버 평가

이 문서에서는 [Azure 마이그레이션: 서버 평가](migrate-services-overview.md#azure-migrate-server-assessment-tool) 도구에서 평가에 대한 개요를 제공합니다. 이 도구는 온-프레미스 VMware 가상 컴퓨터, Hyper-V VM 및 Azure로 마이그레이션할 물리적 서버를 평가할 수 있습니다.

## <a name="whats-an-assessment"></a>평가란 무엇입니까?

서버 평가 도구를 사용하여 평가는 준비 상태를 측정하고 온-프레미스 서버를 Azure로 마이그레이션하는 효과를 예측합니다.

> [!NOTE]
> Azure 정부에서 [지원되는 대상](migrate-support-matrix.md#supported-geographies-azure-government) 평가 위치를 검토합니다. 평가의 VM 크기 권장 사항은 정부 클라우드 리전을 위해 특별히 VM 시리즈를 사용합니다. VM 유형에 대해 [자세히 알아보세요.](https://azure.microsoft.com/global-infrastructure/services/?regions=usgov-non-regional,us-dod-central,us-dod-east,usgov-arizona,usgov-iowa,usgov-texas,usgov-virginia&products=virtual-machines)

## <a name="types-of-assessments"></a>평가 유형

서버 평가를 사용하여 만든 평가는 데이터의 시점 스냅샷입니다. 서버 평가는 두 가지 유형의 평가를 제공합니다.

**평가 유형** | **세부 정보** | **Data**
--- | --- | ---
**성능 기반** | 수집된 성과 데이터를 기반으로 권장 사항을 만드는 평가 | VM 크기 권장 사항은 CPU 및 RAM 사용률 데이터를 기반으로 합니다.<br/><br/> 디스크 유형 권장 사항은 IOPS(초당 입력/출력 작업)와 온-프레미스 디스크의 처리량을 기반으로 합니다. 디스크 유형은 Azure 표준 HDD, Azure 표준 SSD 및 Azure 프리미엄 디스크입니다.
**온-프레미스** | 성과 데이터를 사용하여 권장 사항을 만들지 않는 평가 | VM 크기 권장 사항은 온-프레미스 VM 크기를 기반으로 합니다.<br/><br> 권장되는 디스크 유형은 평가에 대해 선택한 저장소 유형을 기반으로 합니다.

## <a name="how-do-i-run-an-assessment"></a>평가를 실행하려면 어떻게 해야 합니까?

평가를 실행하는 방법에는 두 가지가 있습니다.

- 경량 Azure 마이그레이션 어플라이언스에서 수집한 서버 메타데이터를 사용하여 컴퓨터를 평가합니다. 어플라이언스는 온-프레미스 컴퓨터를 검색합니다. 그런 다음 컴퓨터 메타데이터 및 성능 데이터를 Azure 마이그레이션으로 보냅니다.
- CSV(쉼표 구분 값) 형식으로 가져온 서버 메타데이터를 사용하여 컴퓨터를 평가합니다.

## <a name="how-do-i-assess-with-the-appliance"></a>어플라이언스로 평가하려면 어떻게 해야 합니까?

온-프레미스 서버를 검색하기 위해 Azure 마이그레이션 어플라이언스를 배포하는 경우 다음 단계를 수행합니다.

1. 서버 평가와 함께 작동하도록 Azure 및 온-프레미스 환경을 설정합니다.
1. 첫 번째 평가를 위해 Azure 프로젝트를 만들고 서버 평가 도구를 추가합니다.
1. 경량 Azure 마이그레이션 어플라이언스를 배포합니다. 어플라이언스는 온-프레미스 컴퓨터를 지속적으로 검색하고 컴퓨터 메타데이터 및 성능 데이터를 Azure Migrate로 보냅니다. 어플라이언스를 VM 또는 물리적 컴퓨터로 배포합니다. 평가하려는 컴퓨터에 는 아무것도 설치할 필요가 없습니다.

어플라이언스가 시스템 검색을 시작한 후 평가할 컴퓨터를 그룹으로 수집하고 그룹에 대한 평가를 실행할 수 있습니다.

[VMware,](tutorial-prepare-vmware.md) [Hyper-V](tutorial-prepare-hyper-v.md)또는 물리적 서버에 대한 [자습서를](tutorial-prepare-physical.md) 수행하여 다음 단계를 시도해 보십시오.

## <a name="how-do-i-assess-with-imported-data"></a>가져온 데이터로 평가하려면 어떻게 해야 합니까?

CSV 파일을 사용하여 서버를 평가하는 경우 어플라이언스가 필요하지 않습니다. 대신 다음 단계를 수행합니다.

1. 서버 평가와 함께 작동하도록 Azure를 설정합니다.
1. 첫 번째 평가를 위해 Azure 프로젝트를 만들고 서버 평가 도구를 추가합니다.
1. CSV 템플릿을 다운로드하고 서버 데이터를 추가합니다.
1. 템플릿을 서버 평가로 가져옵니다.
1. 가져오기와 함께 추가된 서버를 검색하고 그룹으로 수집하고 그룹에 대한 평가를 실행합니다.

## <a name="what-data-does-the-appliance-collect"></a>어플라이언스는 어떤 데이터를 수집합니까?

평가를 위해 Azure 마이그레이션 어플라이언스를 사용하는 경우 [VMware](migrate-appliance.md#collected-data---vmware) 및 [Hyper-V](migrate-appliance.md#collected-data---hyper-v)에 대해 수집된 메타데이터 및 성능 데이터에 대해 알아봅니다.

## <a name="how-does-the-appliance-calculate-performance-data"></a>어플라이언스는 성능 데이터를 어떻게 계산합니까?

어플라이언스를 검색에 사용하는 경우 다음 단계를 사용하여 계산 설정에 대한 성능 데이터를 수집합니다.

1. 어플라이언스는 실시간 샘플 포인트를 수집합니다.

    - **VMware VM**: 샘플 포인트는 20초마다 수집됩니다.
    - **하이퍼-VVM:** 샘플 포인트는 30초마다 수집됩니다.
    - **물리적 서버**: 샘플 포인트가 5분마다 수집됩니다.

1. 어플라이언스는 샘플 포인트를 결합하여 10분마다 단일 데이터 포인트를 만듭니다. 데이터 점을 만들기 위해 어플라이언스는 모든 샘플에서 피크 값을 선택합니다. 그런 다음 데이터 포인트를 Azure로 보냅니다.
1. 서버 평가는 지난 달의 모든 10분 데이터 포인트를 저장합니다.
1. 평가를 만들 때 서버 평가는 권한 부여에 사용할 적절한 데이터 포인트를 식별합니다. 식별은 *성능 기록* 및 *백분위수 사용률에 대한 백분위수*값을 기반으로 합니다.

    - 예를 들어 성능 기록이 1주일이고 백분위수 사용률이 95번째 백분위수인 경우 서버 평가는 지난 주에 대한 10분 샘플 포인트를 정렬합니다. 그것은 오름차순으로 그들을 정렬하고 권리화에 대한 95 번째 백분위수 값을 선택합니다.
    - 95번째 백분위수 값은 이상값을 무시하면 되며, 이는 99번째 백분위수(percentile)를 선택한 경우에 포함될 수 있습니다.
    - 해당 기간의 최대 사용량을 선택하고 이상값을 놓치지 않으려면 백분위수 사용률에 대해 99번째 백분위수를 선택합니다.

1. 이 값에 컴포트 팩터를 곱하여 어플라이언스가 수집하는 이러한 메트릭에 대한 효과적인 성능 활용 데이터를 가져옵니다.

    - CPU 사용률
    - RAM 사용률
    - 디스크 IOPS(읽기 및 쓰기)
    - 디스크 처리량(읽기 및 쓰기)
    - 네트워크 처리량(인/아웃)

## <a name="how-are-assessments-calculated"></a>평가는 어떻게 계산되나요?

서버 평가는 온-프레미스 컴퓨터의 메타데이터 및 성능 데이터를 사용하여 평가를 계산합니다. Azure 마이그레이션 어플라이언스를 배포하는 경우 평가는 어플라이언스가 수집하는 데이터를 사용합니다. 그러나 CSV 파일을 사용하여 가져온 평가를 실행하는 경우 계산에 대한 메타데이터를 제공합니다.

계산은 다음 세 단계로 수행됩니다.

1. **Azure 준비 성 계산**: 컴퓨터가 Azure로 마이그레이션하는 데 적합한지 여부를 평가합니다.
1. **크기 조정 권장 사항 계산:** 계산, 스토리지 및 네트워크 크기 조정을 예측합니다.
1. **월별 비용 계산**: 마이그레이션 후 Azure에서 컴퓨터를 실행하기 위한 예상 월별 계산 및 저장소 비용을 계산합니다.

계산은 이전 순서로 되어 있습니다. 컴퓨터 서버는 이전 단계를 통과하는 경우에만 이후 단계로 이동합니다. 예를 들어 서버가 Azure 준비 단계에 실패하면 Azure에 적합하지 않은 것으로 표시됩니다. 크기 조정 및 비용 계산은 해당 서버에 대해 수행되지 않습니다.

## <a name="whats-in-an-assessment"></a>평가의 내용

다음은 서버 평가평가에 포함된 내용입니다.

속성 | 세부 정보
--- | ---
**대상 위치** | 마이그레이션할 위치입니다. 서버 평가는 현재 이러한 대상 Azure 지역을 지원합니다.<br/><br/> 오스트레일리아 동부, 오스트레일리아 남동부, 브라질 남부, 캐나다 중부, 캐나다 동부, 중앙 인도, 미국 중부, 중국 동부, 중국 북부, 동아시아, 미국 동부, 미국 동부 2, 독일 중부, 독일 북동부, 일본 동부, 일본 서부, 한국 중부, 한국, 북중부, 북유럽, 미국 중남부, 동남아시아, 남인도, 영국 남부, 영국 서부, 미국 고브 애리조나, 미국 고브 애리조나, 미국 고브, 미국 고브 , 미국 중서부, 유럽 서부, 인도 서부, 미국 서부 및 미국 서부 2.
**대상 스토리지 디스크(크기 조정)** | Azure의 저장소에 사용할 디스크 유형입니다. <br/><br/> 대상 스토리지 디스크를 프리미엄 관리, 표준 SSD 관리 또는 표준 HDD 관리 디스크로 지정합니다.
**대상 스토리지 디스크(성능 기반 크기 조정)** | 대상 저장소 디스크 유형을 자동, 프리미엄 관리, 표준 HDD 관리 또는 표준 SSD 관리로 지정합니다.<br/><br/> **자동**: 디스크 권장 사항은 IOPS 및 처리량을 의미하는 디스크의 성능 데이터를 기반으로 합니다.<br/><br/>**프리미엄 또는 표준**: 평가는 선택한 저장소 유형 내에서 디스크 SKU를 권장합니다.<br/><br/> 99.9%의 단일 인스턴스 VM 서비스 수준 계약(SLA)을 원하는 경우 프리미엄 관리 디스크를 사용하는 것이 좋습니다. 이렇게 하면 평가의 모든 디스크를 프리미엄 관리 디스크로 권장합니다.<br/><br/> Azure Migrate는 마이그레이션 평가를 위해 관리되는 디스크만 지원합니다.
**Azure 예약 가상 컴퓨터 인스턴스** | 평가의 비용 추정을 고려하여 [예약된 인스턴스를](https://azure.microsoft.com/pricing/reserved-vm-instances/) 지정합니다.<br/><br/> Azure Migrate는 현재 종량제 오퍼에 대해서만 Azure 예약 VM 인스턴스를 지원합니다.
**크기 조정 기준** | Azure VM의 크기를 조정하는 데 사용됩니다.<br/><br/> 크기 조정 또는 성능 기반 크기 조정을 사용합니다.
**성능 기록** | 성능 기반 크기 조정과 함께 사용됩니다. 성능 기록은 성능 데이터를 평가할 때 사용되는 기간을 지정합니다.
**백분위 수 사용률** | 성능 기반 크기 조정과 함께 사용됩니다. 백분위수 사용률은 권한 지정에 사용되는 성능 샘플의 백분위수 값을 지정합니다.
**VM 시리즈** | 권한 지정을 고려할 Azure VM 시리즈입니다. 예를 들어 Azure에 A 시리즈 VM이 필요한 프로덕션 환경이 없는 경우 A 시리즈를 계열 목록에서 제외할 수 있습니다.
**쾌적 인자** | 평가 중에 사용되는 버퍼입니다. VM의 CPU, RAM, 디스크 및 네트워크 사용률 데이터에 적용됩니다. 계절적 사용량, 짧은 성능 기록 및 향후 사용량 증가와 같은 문제를 설명합니다.<br/><br/> 예를 들어 사용률이 20%인 10코어 VM은 일반적으로 2코어 VM을 생성합니다. 컴포트 팩터가 2.0이면 대신 4코어 VM이 생성됩니다.
**제안** | 등록된 [Azure 오퍼입니다.](https://azure.microsoft.com/support/legal/offer-details/) 서버 평가는 해당 오퍼의 비용을 추정합니다.
**통화** | 계정의 청구 통화입니다.
**할인(%)** | Azure 혜택 위에 제공되는 구독별 할인입니다. 기본 설정은 0%입니다.
**VM 작동 시간** | 지속적으로 실행되지 않는 Azure VM의 월별 기간 및 일별 시간입니다. 예상 비용은 해당 기간을 기준으로 합니다.<br/><br/> 기본값은 월 31일, 하루 24시간입니다.
**Azure 하이브리드 혜택** | 소프트웨어 보증이 있고 [Azure 하이브리드 혜택을](https://azure.microsoft.com/pricing/hybrid-use-benefit/)받을 수 있는지 여부를 지정합니다. 설정에 기본값 "예"가 있는 경우 Windows 이외의 운영 체제에 대한 Azure 가격은 Windows VM에 대해 고려됩니다.

서버 평가를 통해 평가를 만드는 [모범 사례를 검토합니다.](best-practices-assessment.md)

## <a name="calculate-readiness"></a>준비 성 계산

모든 컴퓨터가 Azure에서 실행하기에 적합한 것은 아닙니다. 서버 평가는 모든 온-프레미스 컴퓨터를 평가하고 준비 범주를 할당합니다.

- **Azure에 대한 준비**: 변경 사항 없이 컴퓨터를 있는 것으로 Azure로 마이그레이션할 수 있습니다. 전체 Azure 지원으로 Azure에서 시작됩니다.
- **Azure에 대한 조건부 준비**: 컴퓨터가 Azure에서 시작되지만 전체 Azure 지원이 없을 수 있습니다. 예를 들어 Azure는 이전 버전의 Windows Server를 실행하는 컴퓨터를 지원하지 않습니다. 이러한 컴퓨터를 Azure로 마이그레이션하기 전에 주의해야 합니다. 준비 상태를 해결하려면 평가에서 제안하는 수정 지침을 따르십시오.
- **Azure에 대한 준비가 되지 않음**: 컴퓨터가 Azure에서 시작되지 않습니다. 예를 들어 온-프레미스 컴퓨터의 디스크에 64TB 이상을 저장하는 경우 Azure는 컴퓨터를 호스트할 수 없습니다. 마이그레이션 전에 문제를 해결하려면 수정 가이드를 따르십시오.
- **준비 없음**: Azure Migrate는 메타데이터가 부족하여 컴퓨터의 준비 상태를 확인할 수 없습니다.

준비 상태를 계산하기 위해 서버 평가는 다음 표에 요약된 컴퓨터 속성 및 운영 체제 설정을 검토합니다.

## <a name="calculating-sizing"></a>크기 조정 계산

컴퓨터가 Azure에 대한 준비로 표시되면 서버 평가에서 크기 조정 권장 사항을 만듭니다. 이러한 권장 사항은 Azure VM 및 디스크 SKU를 식별합니다. 크기 조정 계산은 온-프레미스 크기 조정 또는 성능 기반 크기 조정을 사용 하는지 여부에 따라 달라 집니다.

### <a name="calculate-sizing-as-is-on-premises"></a>크기 조정 계산(온-프레미스로)

 온-프레미스 크기 조정을 사용하는 경우 서버 평가에서는 VM 및 디스크의 성능 기록을 고려하지 않습니다.

- **크기 조정**: 서버 평가는 온-프레미스에 할당된 크기에 따라 Azure VM SKU를 할당합니다.
- **저장소 및 디스크 크기:** 서버 평가는 평가 속성에 지정된 저장소 유형을 살펴보고 적절한 디스크 유형을 권장합니다. 가능한 스토리지 유형은 표준 HDD, 표준 SSD 및 프리미엄입니다. 기본 저장소 유형은 프리미엄입니다.
- **네트워크 크기 조정**: 서버 평가는 온-프레미스 컴퓨터에서 네트워크 어댑터를 고려합니다.

### <a name="calculate-sizing-performance-based"></a>크기 조정 계산(성능 기반)

성능 기반 크기 조정을 사용하는 경우 서버 평가는 다음과 같이 크기 조정 권장 사항을 지정합니다.

- 서버 평가는 Azure에서 VM 크기 및 디스크 유형을 식별하기 위해 컴퓨터의 성능 기록을 고려합니다.
- CSV 파일을 사용하여 서버를 가져오는 경우 지정한 값이 사용됩니다. 이 방법은 온-프레미스 컴퓨터를 과도하게 할당하고 사용률이 낮으며 비용을 절감하기 위해 Azure VM을 올바르게 크기를 조정하려는 경우에 특히 유용합니다.
- 성능 데이터를 사용하지 않으려면 이전 섹션에서 설명한 대로 크기 조정 기준을 온-프레미스로 재설정합니다.

#### <a name="calculate-storage-sizing"></a>스토리지 크기 계산

저장소 크기 조정을 위해 Azure Migrate는 컴퓨터에 연결된 각 디스크를 Azure 디스크에 매핑하려고 시도합니다. 크기 조정은 다음과 같이 작동합니다.

1. 서버 평가는 필요한 총 IOPS를 얻기 위해 디스크의 읽기 및 쓰기 IOPS를 추가합니다. 마찬가지로 읽기 및 쓰기 처리량 값을 추가하여 각 디스크의 총 처리량을 가져옵니다.
1. 저장소 유형을 자동으로 지정한 경우 선택한 형식은 유효 IOPS 및 처리량 값을 기반으로 합니다. 서버 평가는 Azure의 표준 HDD, 표준 SSD 또는 프리미엄 디스크에 디스크를 매핑할지 여부를 결정합니다. 저장소 유형이 이러한 디스크 유형 중 하나로 설정된 경우 서버 평가는 선택한 저장소 유형 내에서 디스크 SKU를 찾으려고 시도합니다.
1. 디스크는 다음과 같이 선택됩니다.
    - 서버 평가에서 필요한 IOPS 및 처리량이 있는 디스크를 찾을 수 없는 경우 컴퓨터가 Azure에 적합하지 않은 것으로 표시됩니다.
    - 서버 평가에서 적합한 디스크 집합을 찾으면 평가 설정에 지정된 위치를 지원하는 디스크를 선택합니다.
    - 적격 디스크가 여러 개 있는 경우 서버 평가는 가장 낮은 비용으로 디스크를 선택합니다.
    - 디스크에 대한 성능 데이터를 사용할 수 없는 경우 구성 디스크 크기는 Azure에서 표준 SSD 디스크를 찾는 데 사용됩니다.

#### <a name="calculate-network-sizing"></a>네트워크 크기 조정 계산

서버 평가는 온-프레미스 컴퓨터에 연결된 네트워크 어댑터의 수와 필요한 성능을 지원하는 Azure VM을 찾으려고 합니다.

- 온-프레미스 VM의 효과적인 네트워크 성능을 얻기 위해 서버 평가는 모든 네트워크 어댑터에서 컴퓨터 외의 데이터 전송 속도를 집계합니다(네트워크 아웃). 그런 다음 편안함 계수를 적용합니다. 결과 값을 사용하여 필요한 네트워크 성능을 지원할 수 있는 Azure VM을 찾습니다.
- 서버 평가는 네트워크 성능과 함께 Azure VM이 필요한 수의 네트워크 어댑터를 지원할 수 있는지 여부도 고려합니다.
- 네트워크 성능 데이터를 사용할 수 없는 경우 서버 평가에서는 VM 크기 조정에 대한 네트워크 어댑터 수만 고려합니다.

#### <a name="calculate-compute-sizing"></a>계산 크기 계산

서버 평가는 저장소 및 네트워크 요구 사항을 계산한 후 CPU 및 RAM 요구 사항을 고려하여 Azure에서 적합한 VM 크기를 찾습니다.

- Azure 마이그레이션은 효과적인 사용 코어와 RAM을 살펴보고 적합한 Azure VM 크기를 찾습니다.
- 적합한 크기가 없으면 컴퓨터가 Azure에 적합하지 않은 것으로 표시됩니다.
- 적합한 크기가 있으면 Azure Migrate는 스토리지 및 네트워킹 계산을 적용합니다. 그런 다음 최종 VM 크기 권장 사항에 대한 위치 및 가격 계층 설정을 적용합니다.
- 적합한 Azure VM 크기가 여러 개인 경우 비용이 가장 낮은 크기가 권장됩니다.

## <a name="confidence-ratings-performance-based"></a>신뢰도 등급(성과 기반)

Azure 마이그레이션의 각 성능 기반 평가는 신뢰도 등급과 연결됩니다. 등급범위는 별 1개(최저)에서 5개(최고) 등급입니다. 신뢰도 등급은 Azure Migrate가 제공하는 크기 권장 사항의 안정성을 예측하는 데 도움이 됩니다.

- 신뢰도 등급은 평가에 할당됩니다. 등급은 평가를 계산하는 데 필요한 데이터 포인트의 가용성을 기반으로 합니다.
- 성능 기반 크기 조정을 위해서는 서버 평가가 필요합니다.
    - CPU 및 VM RAM에 대한 사용률 데이터입니다.
    - VM에 연결된 모든 디스크에 대한 디스크 IOPS 및 처리량 데이터입니다.
    - VM에 연결된 각 네트워크 어댑터에 대한 성능 기반 크기를 처리하는 네트워크 I/O입니다.

이러한 사용률 번호를 사용할 수 없는 경우 크기 권장 사항이 신뢰할 수 없는 것일 수 있습니다.

> [!NOTE]
> 가져온 CSV 파일을 사용하여 평가된 서버에 대해 신뢰도 등급이 할당되지 않습니다. 등급은 온-프레미스 평가에도 적용되지 않습니다.

### <a name="ratings"></a>등급

이 표에서는 사용 가능한 데이터 포인트의 백분율에 따라 평가 신뢰도 등급을 보여 주며, 이는 다음과 같은 영향을 미칩니다.

   **데이터 요소 가용성** | **신뢰 등급**
   --- | ---
   0-20% | 별 1개
   21-40% | 별 2개
   41-60% | 별 3개
   61-80% | 별 4개
   81-100% | 별 5개

### <a name="low-confidence-ratings"></a>낮은 신뢰도 등급

평가가 낮은 신뢰도 등급을 받을 수 있는 몇 가지 이유는 다음과 같습니다.

- 평가를 만드는 기간 동안 환경을 프로파일하지 않았습니다. 예를 들어 성능 기간을 하루로 설정한 평가를 만드는 경우 모든 데이터 요소가 수집될 때까지 검색을 시작한 후 최소 하루 이상 기다려야 합니다.
- 평가가 계산되는 동안 일부 VM이 종료되었습니다. 일부 기간 동안 VM이 꺼져 있으면 서버 평가에서 해당 기간 동안 성능 데이터를 수집할 수 없습니다.
- 일부 VM은 평가가 계산되는 동안 만들어졌습니다. 예를 들어 지난 달의 성능 기록에 대한 평가를 만들었지만 일부 VM이 1주일 전에 만들어졌다고 가정합니다. 새 VM의 성능 기록은 전체 기간 동안 존재하지 않습니다.

> [!NOTE]
> 평가의 신뢰도 등급이 별 5개 미만인 경우 어플라이언스가 환경을 프로파일로 지정한 다음 평가를 다시 계산할 때까지 하루 이상 기다리는 것이 좋습니다. 그렇지 않으면 성능 기반 크기 조정이 불안정할 수 있습니다. 이 경우 평가를 온-프레미스 크기 조정으로 전환하는 것이 좋습니다.

## <a name="calculate-monthly-costs"></a>월별 비용 계산

크기 조정 권장 사항이 완료되면 Azure Migrate는 마이그레이션 후의 계산 및 저장소 비용을 계산합니다.

- **계산 비용**: Azure Migrate는 권장Azure VM 크기와 Azure 청구 API를 사용하여 VM의 월별 비용을 계산합니다.

    계산은 다음을 고려합니다.
    - 운영 체제
    - 소프트웨어 보증
    - 예약 인스턴스
    - VM 작동 시간
    - 위치
    - 통화 설정

    서버 평가는 모든 컴퓨터에서 비용을 집계하여 총 월별 계산 비용을 계산합니다.

- **저장소 비용**: 컴퓨터에 연결된 모든 디스크의 월별 비용을 집계하여 컴퓨터의 월별 저장소 비용을 계산합니다.

    서버 평가는 모든 컴퓨터의 저장소 비용을 집계하여 총 월별 저장소 비용을 계산합니다. 현재 계산에서는 평가 설정에 지정된 오퍼를 고려하지 않습니다.

비용은 평가 설정에 지정된 통화로 표시됩니다.
