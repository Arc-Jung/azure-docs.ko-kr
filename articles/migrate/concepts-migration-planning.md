# <a name="build-migration-plan-with-azure-migrate"></a>Azure Migrate를 사용하여 마이그레이션 계획 빌드

이 문서에서는 [Azure Migrate](migrate-services-overview.md)를 사용 하 여 Azure에 마이그레이션 계획을 구축 하는 데 도움이 되는 간략 한 가이드를 제공 합니다.다른 질문이 있는 경우 다음 리소스를 확인 하세요.

- Azure Migrate에 대 한 [일반적인 질문](resources-faq.md)
- [Azure Migrate 어플라이언스](common-questions-appliance.md) 에 대 한 질문
- [서버 마이그레이션](common-questions-server-migration.md) 에 대 한 질문
- [Azure Migrate 포럼](https://aka.ms/AzureMigrateForum) 에서 질문에 대 한 답변 받기

## <a name="define-the-goals-of-cloud-migration"></a>클라우드 마이그레이션 목표 정의

마이그레이션 계획을 구축 하기 전에 클라우드로 이동 하는 데 도움이 되는 [동기](https://docs.microsoft.com/azure/cloud-adoption-framework/strategy/motivations) 를 이해 하 고 평가 하 여 더 성공적인 비즈니스 결과를 생성 하는 것이 중요 합니다. [Azure에 대 한 클라우드 채택 프레임 워크](https://docs.microsoft.com/azure/cloud-adoption-framework) 에서 설명 하는 것 처럼 비즈니스에 적합 한 다양 한 트리거와 마이그레이션 방법이 있을 수 있습니다.  

**중요한 비즈니스 이벤트** | **마이그레이션 결과**
--- | ---
데이터 센터 종료 | 비용 절감
병합기, 취득 또는 divestiture | 공급 업체 또는 기술 복잡성 감소
자본 지출 절감 | 내부 작업의 최적화
업무상 중요한 기술 지원 종료 | 비즈니스 민첩성 향상
규정 준수 변경에 대 한 응답 | 새로운 기술 역량 준비
새 데이터 주권 요구 사항 | 시장 수요를 충족 하도록 크기 조정
장애 완화 및 IT 안정성 개선 | 지리적 요구에 맞게 크기 조정

마이그레이션 동기는 Azure로 마이그레이션하여 달성 하고자 하는 전략적 목표와 결과를 반영 하는 데도 도움이 될 수 있습니다. 다음 단계는 워크 로드에 맞게 조정 된 Azure에 대 한 마이그레이션 경로를 식별 하 고 계획 하는 것입니다. Azure Migrate: 서버 평가 도구를 사용 하 여 온-프레미스 워크 로드를 평가 하 고 마이그레이션에 도움이 되는 지침 및 도구를 제공 합니다.

## <a name="understand-your-digital-estate"></a>디지털 공간 이해

온-프레미스 인프라, 응용 프로그램 및 종속성을 이해 하 여 Azure로 마이그레이션하려는 워크 로드를 식별 하 고 최적화 된 비용 예측을 얻을 수 있습니다. 서버 평가 도구를 통해 다음과 같은 질문에 대답할 수 있습니다.

### <a name="what-workloads-are-in-use"></a>사용 중인 작업은 무엇 인가요?

경량 Azure Migrate 어플라이언스를 사용 하 여 온-프레미스 VMware Vm, Hyper-v Vm 및 물리적 서버에 대 한 에이전트 없는 검색을 수행할 수 있습니다. 연속 검색은 컴퓨터 구성 및 성능 메타 데이터를 수집 하며,이를 사용 하 여 설치 된 응용 프로그램의 인벤토리 및 온-프레미스 컴퓨터에서 실행 되는 역할/기능을 가져올 수도 있습니다. Azure Migrate 어플라이언스는 다음을 수집 합니다.

- 컴퓨터, 디스크 및 Nic의 메타 데이터 세부 정보

- 응용 프로그램 및 역할/기능을 비롯 한 설치 된 응용 프로그램  

- CPU 및 메모리 사용률, 디스크 IOPS 및 처리량을 포함 하는 성능 데이터

그런 다음 응용 프로그램 인벤토리 목록을 내보내 작업에서 실행 되는 모든 SQL Server 인스턴스를 확인 하 고 Azure Migrate: 데이터베이스 평가 도구를 사용 하 여 준비 상태를 파악 합니다.

 ![포털의 응용 프로그램 인벤토리](./media/concepts-migration-planning/application-inventory-portal.png)

 ![응용 프로그램 인벤토리 내보내기](./media/concepts-migration-planning/application-inventory-export.png)

서버 평가 도구의 검색 데이터와 함께 기존 CMDB 데이터를 사용 하 여 서버 및 데이터베이스 공간 보기를 빌드하고 비즈니스 단위, 응용 프로그램 소유자, 지역 등에 걸친 서버 배포를 이해할 수 있습니다 .이를 통해 마이그레이션할 워크 로드의 우선 순위를 지정할 수 있습니다.

### <a name="what-dependencies-exist-between-workloads"></a>워크 로드 간에 존재 하는 종속성은 무엇 인가요?

서버를 검색 한 후에는 에이전트 없는 종속성 매핑을 사용 하 여 상호 의존적인 서버를 Azure로 이동 하기 위한 서버 간 종속성 및 최적화 전략을 시각화 하 고 식별 합니다. 시각화를 사용 하면 특정 컴퓨터가 사용 중인지 또는 마이그레이션 대신 서비스 해제할 수 있는지를 파악할 수 있습니다.  종속성을 분석 하 여 아무 것도 남아 있지 않도록 하 고 마이그레이션 중에 예기치 않은 중단이 발생 하지 않도록 해야 합니다. 응용 프로그램 인벤토리 및 종속성 매핑을 수행한 후에는 신뢰도 높은 그룹을 만들고 서버 평가를 시작할 수 있습니다.

 ![종속성 매핑](./media/concepts-migration-planning/expand-client-group.png)

### <a name="are-they-optimized-and-sized-properly"></a>최적화 되어 있고 적절히 크기를 조정 하나요?

Azure는 시간에 따라 클라우드 용량 크기를 조정할 수 있는 유연성을 제공 하므로 마이그레이션 기능을 통해 서버에 할당 된 CPU 및 메모리 리소스를 최적화할 수 있습니다. 이전에 식별 된 그룹에 대 한 평가를 만들어 작업의 성능 기록을 이해 하기 위해 Azure의 rightsized VM Sku 및 디스크 권장 사항에서 매우 중요 합니다.

## <a name="assess-your-readiness-for-migration"></a>마이그레이션의 준비 상태 평가

### <a name="readiness-and-suitability-analysis-for-azure"></a>Azure에 대 한 준비 및 적합성 분석
Azure VM 평가 보고서를 내보내고 다음 준비 범주를 기준으로 필터링 하 여 Azure의 VM 준비 상태를 이해 합니다.

- **Azure 준비 완료**: 이러한 컴퓨터를 변경 없이 azure로 마이그레이션할 수 있습니다.  

- **조건적으로 azure에 대해 준비**됨: 이러한 컴퓨터를 azure로 마이그레이션할 수 있지만 평가에 제공 된 수정 지침에 따라 이러한 서버에서 약간만 변경 해야 합니다.

- **Azure에 대해 준비 되지 않음**: 이러한 컴퓨터는 그대로 azure로 마이그레이션할 수 없으며 마이그레이션 전에 수정 지침에 따라 문제를 해결 해야 합니다.

- **준비 상태 알 수 없음**: Azure Migrate에서 메타 데이터가 부족 하 여 컴퓨터의 준비 상태를 확인할 수 없습니다.

데이터베이스 평가를 사용 하 여 SQL Server 데이터 공간을 Azure SQL Database 또는 Azure SQL 관리 되는 인스턴스로 마이그레이션하기 위한 준비 상태를 평가할 수 있습니다. 각 SQL server 인스턴스에 대 한 마이그레이션 준비 상태 비율을 확인할 수 있습니다. 또한 각 인스턴스에 대해 Azure에서 권장 되는 대상, 잠재적인 마이그레이션 차단기, 주요 변경 수, Azure SQL DB/Azure SQL VM의 준비 및 호환성 수준을 확인할 수 있습니다. 마이그레이션 블 로커의 영향을 파악 하 고이를 해결 하기 위한 권장 사항에 대해 자세히 알아볼 수 있습니다.

 ![데이터베이스 평가](./media/concepts-migration-planning/database-assessment-portal.png)

### <a name="sizing-recommendations"></a>크기 조정 권장 사항

컴퓨터가 Azure 준비 완료로 표시 되 면 서버 평가는 크기 조정 권장 구성을 사용 하 여 Vm에 대 한 Azure VM 및 디스크 SKU를 식별 합니다. 성능 기록을 고려 하지 않고 해당 온-프레미스 구성에 따라 성능 기록을 기반으로 하 여 크기 조정 권장 사항을 표시 하도록 선택할 수 있습니다. 데이터베이스의 경우 데이터베이스 평가의 데이터베이스 SKU, 가격 책정 계층 및 계산 수준에 대 한 권장 사항을 볼 수 있습니다.  

### <a name="compute-assessments-to-get-estimated-costs-for-running-the-workloads-in-azure"></a>계산 평가-Azure에서 작업을 실행 하는 데 드는 예상 비용을 얻을 수 있습니다.

평가의 *성능 기반* 올바른 크기 조정 옵션을 사용 하면 Azure에 대 한 워크 로드를 최적화할 수 있습니다. 규모 조정 외에도 비용을 절감 하는 데 도움이 되는 몇 가지 다른 방법이 있습니다.

- **예약 된 인스턴스**: 예약 된 인스턴스를 사용 하면 Windows 및 Linux (vm)에서 1 년 또는 3 년 용어로 종 량 제 가격 책정에 비해 비용을 크게 줄일 수 있습니다.

- **Azure 하이브리드 혜택**: 소프트웨어 보증을 포함 하는 Windows Server 온-프레미스 라이선스를 azure로 가져와서 예약 인스턴스 옵션과 결합할 수 있습니다.

- **EA (기업계약 제품)**: 기업계약는 구독에 적용 가능한 기본 제공 절감 효과를 제공 합니다.

- **제품**: 개발 또는 테스트 vm에 대 한 낮은 요금을 제공 하는 종량제 개발/테스트 및 Enterprise 개발/테스트와 같은 여러 Azure 제품이 있습니다.

- **VM 작동 시간**: 비용을 줄이기 위해 Azure vm이 실행 되는 기간 (일) 및 일일 시간 (RI에는 적용 되지 않음)을 언급할 수 있습니다.

- **대상 지역**: geography의 특정 지역으로 마이그레이션하는 것이 더 비용 효율적일 수 있는 경우 서로 다른 지역에서 여러 평가를 만들어 비교할 수 있습니다.

- **성능 기반 권장 사항**: 클라우드 비용을 절감 하는 데 도움이 되는 Rightsized의 Azure VM 권장 사항을 사용 하는 것이 가장 좋습니다.

### <a name="visualize-data"></a>데이터 시각화

포털에서 Azure 준비 및 월별 비용 배포를 사용 하 여 서버 평가 보고서를 확인 하 고, 검색 및 평가 데이터에 대 한 추가 시각화를 적용 하 여 마이그레이션 계획을 더욱 풍부 하 게 만들 수 있습니다. 다양 한 속성 조합에 대 한 평가를 여러 개 만들고 비즈니스에 가장 적합 한 속성 집합을 선택할 수 있습니다.  

 ![평가 개요](./media/concepts-migration-planning/assessment-summary.png)

### <a name="evaluate-gaps-and-potential-blockers"></a>간격 및 잠재적 차단 평가

마이그레이션할 응용 프로그램 및 기본 인프라를 결정 하는 동안 이러한 응용 프로그램에 대 한 가동 중지 시간 제약 조건을 확인 하 고 응용 프로그램과 기본 인프라 간의 작동 종속성을 찾습니다. 이 분석을 통해 RTO (복구 시간 목표)에 맞는 마이그레이션을 계획 하 고 데이터 손실을 최소화 하는 데 도움이 될 수 있습니다. 마이그레이션하기 전에 서버 평가 보고서 및 Azure Migrate: 데이터베이스 평가 권장 사항에 따라 서버와 SQL 데이터베이스의 마이그레이션을 차단할 수 있는 호환성 문제 또는 지원 되지 않는 기능을 검토 하 고 완화 하는 것이 좋습니다.

### <a name="first-workloads-to-target-and-approach"></a>대상 및 접근 방식에 대 한 첫 번째 워크 로드

마이그레이션을 결정 하는 데 중요 한 정보를 모두 만들었으므로 먼저 마이그레이션해야 하는 응용 프로그램 및 워크 로드를 우선 순위를 지정 해야 합니다. 모든 기능을 완벽 하 게 형성 하기 전에이 전략의 결함을 낼 수 있도록 원하는 응용 프로그램을 체계적이 고 제어 가능한 방식으로 마이그레이션하기 위한 "적용 및 학습" 접근 방법을 개발 합니다. 또한 복잡성, 마이그레이션 시간, 비즈니스 긴급도, 프로덕션/비 프로덕션 환경, 규정 준수 및 보안 요구 사항, 응용 프로그램 지식 등의 전략적 요소를 사용 하 여 마이그레이션할 응용 프로그램 그룹의 우선 순위를 지정할 수 있습니다.

권장 되는 마이그레이션 전략은 다음과 같습니다.

- **빠른 Wins 우선 순위**지정: 평가 보고서를 사용 하 여 완전히 준비 되었으며 Azure로 마이그레이션하기 위한 최소한의 노력을 필요로 하는 서버 및 데이터베이스를 포함 하 여 낮은 수준의 과일을 식별할 수 있습니다.
    - Azure 준비: 평가 보고서를 내보내고 "Azure에 대해 준비 됨" 인 모든 컴퓨터를 필터링 합니다. 이는 Azure Migrate: 서버 마이그레이션 도구를 사용 하 여 리프트 하 고 이동할 수 있는 첫 번째 컴퓨터 그룹 일 수 있습니다.
    - OS 지원 종료: 평가 보고서를 내보내고 Windows Server 2008 및 Windows Server 2008 R2 운영 체제를 실행 하는 모든 컴퓨터를 필터링 합니다. 이러한 Sku는 지원 종료 되며 Azure만 azure로 마이그레이션할 때 무료 3 년간의 보안 업데이트를 제공 합니다. 예약 된 인스턴스를 결합, Azure 하이브리드 혜택 및 사용 하는 경우 저장이 훨씬 더 높을 수 있습니다.
    - SQL Server 마이그레이션: 데이터베이스 평가 권장 사항을 사용 하 여 Azure SQL database에 사용할 수 있는 데이터베이스를 마이그레이션할 수 있습니다. 데이터베이스 마이그레이션 및 Azure Migrate: 서버 마이그레이션을 사용 하 여 Azure SQL VM에 준비 된 데이터베이스를 Azure Migrate.
    - 소프트웨어 지원 종료: 응용 프로그램 인벤토리를 내보내고 지원 종료에 도달할 수 있는 모든 소프트웨어/확장을 필터링 합니다. 이러한 응용 프로그램의 우선 순위를 지정 해야 합니다.
    - 과도 하 게 프로 비전 된 Vm: 평가 보고서를 내보내고 CPU 사용률이 낮은 컴퓨터 (%)를 필터링 합니다. 및 메모리 사용률 (%).  이 기회를 사용 하 여 Azure에서 rightsized의 VM으로 마이그레이션하고 미달 사용 리소스에 대해 지불 된 항목을 저장할 수 있습니다.
    - 용량 제약 조건: 평가 보고서를 내보내고 CPU 사용률이 높은 컴퓨터 (%)를 필터링 합니다. 및 메모리 사용률 (%).  과도 하 게 변형 된 Vm이 중단 되는 것을 방지할 수 있으며, Azure로 마이그레이션하여 자동 크기 조정 기능을 활용 하 여 수요를 충족 시킬 수 있습니다. 또한 디스크 IOPS 및 처리량을 분석 하 고 요구 사항에 가장 적합 한 권장 디스크 유형을 검색 하 여 평가 보고서에서 저장소 제약 조건을 파악할 수 있습니다.

- **작은**작업을 시작 하 여 큰 위험으로 최소화 되 고 복잡 한 응용 프로그램 및 워크 로드를 이동 하 여 마이그레이션 전략에 대 한 신뢰도를 구축할 수 있습니다. 또한 Azure Migrate 평가 권장 사항을 조직의 CMDB 리포지토리와 교차 하 여 파일럿 마이그레이션의 개발/테스트 환경 워크 로드를 찾고 마이그레이션할 수 있습니다. 프로덕션 워크 로드를 마이그레이션할 때 이러한 파일럿에서 학습을 사용할 수 있습니다.  

- **규제/업계 요구 사항 준수**: Azure는 제품의 폭과 깊이를 기준으로 업계에서 가장 큰 규정 준수 포트폴리오를 유지 관리 합니다. 이를 통해 Azure로 마이그레이션의 우선 순위를 지정 하 고 국가, 지역 및 산업별 표준 및 법률을 준수할 수 있습니다. 이는 비즈니스에 중요 한 정보를 포함 하거나 중요 한 정보를 보유 하는 조직에서 특히 중요 합니다. 표준 및 규정 abound 특정 한 경우에는 자주 변경 하 여를 유지 하기 어려울 수 있습니다.  

## <a name="finalize-the-migration-planandprepare-formigration"></a>마이그레이션 계획을 완료 하 고 마이그레이션 준비

마이그레이션 계획을 완료 하기 전에 이러한 주요 마이그레이션 고려 사항이 마이그레이션 계획에 hindrance 하지 않도록 합니다.

- 네트워크 대역폭 및 대기 시간에 대 한 제약 조건을 평가 하 여 예기치 않은 지연을 유발 하 고 마이그레이션 복제 속도를 방해할 수 있습니다.

- 마이그레이션된 응용 프로그램에 대 한 성능 및 사용자 수용 테스트를 수행 하거나, 데이터베이스 연결 문자열 및 웹 서버 구성 업데이트, 가공선 실행 및 정리 등의 마이그레이션 후 앱 다시 정의를 실행 하는 시간을 버퍼링 합니다.

- 권장 되는 Azure 권한 및 마이그레이션에 필요한 서버/데이터베이스 액세스 역할 및 사용 권한 모델을 검토 합니다.

- 조직을 준비 하 고 디지털 변환에 대 한 인력 맞춤을 보장 합니다. 성공적인 조직 변경에는 견고한 학습 기초가 중요 합니다. Azure 기본 사항, 솔루션 아키텍처 및 보안에 대 한 과정을 비롯 하 여 [Microsoft Learn](https://docs.microsoft.com/learn/azure/?ocid=CM_Discovery_Checklist_PDF)에서 제공 되는 무료 교육을 확인 하세요. 팀에서 [Azure 인증](https://www.microsoft.com/learning/certification-overview.aspx?ocid=CM_Discovery_Checklist_PDF)도 탐색 하도록 권장   합니다.  

- 필요한 경우 구현에 대 한 지원을 받으세요. 대부분의 조직에서는 클라우드 마이그레이션을 지원 하기 위해 외부 지원을 옵트인 합니다. 개인화 된 지원을 통해 신속 하 고 안전 하 게 Azure로 이동 하려면 [Azure 전문가 관리 서비스 공급자](https://www.microsoft.com/solution-providers/search?cacheId=9c2fed4f-f9e2-42fb-8966-4c565f08f11e&ocid=CM_Discovery_Checklist_PDF)   또는 [FastTrack for Azure](https://azure.microsoft.com/programs/azure-fasttrack/?ocid=CM_Discovery_Checklist_PDF)을 고려 하세요.  

마이그레이션하려는 응용 프로그램의 목록/그룹, 앱 및 데이터베이스 가용성과 가동 중지 시간 제약 조건, 원하는 마이그레이션 마일스 톤에 대 한 자세한 정보를 포함 하는 효과적인 클라우드 마이그레이션 계획을 만듭니다. 마이그레이션 계획에는 데이터 복사에 소요 되는 기간을 고려해 야 하며, 마이그레이션 후 테스트 및 간에 작업에 대 한 버퍼를 포함 해야 합니다. 마이그레이션 후 테스트에는 마이그레이션된 응용 프로그램이 예상 대로 작동 하 고 모든 데이터베이스 개체와 데이터 관계가 성공적으로 클라우드로 전송 되도록 하는 기능, 통합, 보안 및 성능 테스트 사용 사례가 포함 되어야 합니다. 마이그레이션 후 테스트에는 마이그레이션된 응용 프로그램이 예상 대로 작동 하 고 모든 데이터베이스 개체와 데이터 관계가 성공적으로 클라우드로 전송 되도록 하는 기능, 통합, 보안 및 성능 테스트 사용 사례가 포함 되어야 합니다.  

이 분석을 사용 하 여 마이그레이션 로드맵을 작성 하 고 유지 관리 기간을 선언 하 여 가동 중지 시간을 최소화 하면서 응용 프로그램 및 데이터베이스를 마이그레이션하고 마이그레이션 중에 잠재적 운영/비즈니스 영향을 제한할 수 있습니다.  

Azure에 대 한 전체 규모의 마이그레이션을 시작 하기 전에 항상 Azure Migrate의 *테스트 마이그레이션* 기능을 사용 하 여 테스트 하 고 진행 하는 것이 좋습니다. 이 실제 데이터는 관련 된 실제 시간을 예상 하 고 마이그레이션 계획에 필요한 조정 작업을 수행 하는 데 도움이 됩니다. 또한 테스트 마이그레이션은 마이그레이션 계획을 사용 하 여 잠재적인 문제를 검색 하 고 실제 마이그레이션이 수행 되기 전에 해결할 수 있는 기회를 제공 합니다.  

마이그레이션할 준비가 되 면 Azure Migrate의 *서버 마이그레이션 도구* 및 Azure Migrate의 *데이터 마이그레이션 서비스* 를 사용 하 여 종단 간 추적을 통해 원활한 통합 마이그레이션 환경을 제공 합니다. 서버 마이그레이션 도구는 AWS, GCP 등의 가동 중지 시간을 포함 하 여 고객 데이터 센터 또는 다른 사설 또는 공용 클라우드의 온-프레미스에서 호스트 되는 Vm 및 서버 마이그레이션을 지원 합니다. Azure Database Migration Service는 가동 중지 시간을 최소화하면서 여러 데이터베이스 소스에서 Azure 데이터 플랫폼으로 원활하게 마이그레이션할 수 있도록 설계된 완벽하게 관리되는 서비스입니다.  

> [!NOTE]
> VMware Vm의 경우 서버 평가는 vCenter Server VM에 대해 지정 된 운영 체제를 사용 하 여 게스트 OS 분석을 처리 합니다. VMware에서 실행 되는 Linux Vm의 경우 현재 게스트 OS의 정확한 커널 버전을 식별 하지 않습니다.
