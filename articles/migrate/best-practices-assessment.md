# <a name="best-practices-for-creating-assessments"></a>평가 만들기에 대 한 모범 사례

[Azure Migrate](migrate-overview.md) 는 허브를 검색, 평가 및 Microsoft Azure에 앱, 인프라 및 워크 로드를 마이그레이션할 수 있도록 도구를 제공 합니다. 허브는 Azure Migrate 도구 및 타사 독립 소프트웨어 공급 업체 (ISV) 제품에 포함 되어 있습니다. 

이 문서에서는 Azure Migrate Server 평가 도구를 사용 하 여 평가 만들 때 모범 사례를 요약 합니다. 

## <a name="about-assessments"></a>평가 대 한

Azure Migrate Server 평가 사용 하 여 만든 평가 데이터의 지정 시간 스냅숏을 합니다. Azure Migrate에서 평가 다음과 같은 두 종류가 있습니다.

**평가 유형** | **세부 정보** | **데이터**
--- | --- | ---
**성능 기반** | 수집 된 성능 데이터를 기반으로 권장 사항을 제공 하는 평가 | 권장 VM 크기는 CPU 및 메모리 사용률 데이터를 기반으로 합니다.<br/><br/> 디스크 형식 권장 사항 (표준 또는 프리미엄 관리 디스크)는 온-프레미스 디스크의 처리량 및 IOPS를 기반으로 합니다.
**으로-온-프레미스** | 권장 구성을 생성 성능 데이터를 사용 하지 않는 평가 합니다. | 온-프레미스 VM 크기를 기반으로 하는 VM 크기 권장 사항<br/><br> 권장 되는 디스크 종류는 평가 대 한 설정 저장소 형식에서 선택한 항목에 기반 합니다.

### <a name="example"></a>예제
예를 들어, 20% 사용률 및 8GB의 메모리 사용률이 10%를 사용 하 여 4 개의 코어를 사용 하 여 온-프레미스 VM이 있는 경우 평가 다음과 같을 수: 있습니다

- **성능 기반 평가**:
    - 코어 및 메모리 (0.8 개 코어), core 및 0.8 4GB 메모리 사용률 기반으로 하는 것이 좋습니다.
    - 평가 30%의 기본 쾌적 인자를 적용합니다.
    - VM 권장 사항: ~1.4 코어 (0.8 x1.3) 및 ~1.4 GB 메모리.
- **으로-(온-프레미스) 평가**:
    -  4 개의 코어를 사용 하 여 VM을 권장합니다. 8GB의 메모리입니다.

## <a name="best-practices-for-creating-assessments"></a>평가 만들기에 대 한 모범 사례

Azure Migrate 어플라이언스는 지속적으로 온-프레미스 환경의 프로필 하 고 Azure에 메타 데이터 및 성능 데이터를 보냅니다. 평가 만들기에 대 한 이러한 모범 사례를 따릅니다.

- **만들기-평가**: 으로 만들 수 있습니다-검색 후 즉시 평가 됩니다.
- **성능 기반 평가 만들기**: 검색을 설정한 후 하루에 성능 기반 평가 실행 하기 전에 기다리는 최소 하는 것이 좋습니다.
    - 성능 데이터를 수집 하는 데 시간이 걸립니다. 적어도 하루를 기다린 평가 실행 하기 전에 충분 한 성능 데이터 요소는 확인 합니다.
    - 성능 데이터에 대 한 어플라이언스 각 성능 메트릭에 대 한 20 초 마다 실시간 데이터 요소를 수집 하 고 단일 5 분 데이터 시점까지 롤백합니다. 어플라이언스 평가 계산에 대 한 매시간 Azure에는 5 분 데이터 요소를 보냅니다.  
- **최신 데이터를 가져오려면**: 평가 최신 데이터로 자동으로 업데이트 되지 않습니다. 최신 데이터를 사용 하 여 평가 업데이트 하려면 다시 실행 해야 합니다. 
- **기간 일치 하는지 확인**: 성능 기반 평가 실행 하는 경우 있는지 프로필 평가 기간에 대 한 환경입니다. 예를 들어, 1 주일으로 설정 하는 성능 기간을 사용 하 여 평가 만들 경우 수집 되도록 모든 데이터 요소에 대 한 검색을 시작한 후에 최소한 주에 대 한 대기 해야 합니다. 그렇지 않으면 평가 별 5 개의 등급을 얻이 없습니다. 
- **데이터 요소를 누락 되지 않도록**: 누락 된 성능 기반 평가에서 데이터 요소에 다음과 같은 문제가 발생할 수 있습니다.
    - Vm을 평가 하는 동안 전원 꺼짐 및 성능 데이터를 수집 하지 않습니다. 
    - 성능 기록을 기반 월 동안 Vm을 만들면. 해당 Vm에 대 한 데이터를 1 개월 미만 됩니다. 
    - 검색 된 이후 즉시 평가 만들어지거나 평가 시간 성능 데이터 컬렉션에 일치 하지 않습니다.

## <a name="best-practices-for-confidence-ratings"></a>신뢰 등급에 대 한 모범 사례

성능 기반 평가 실행 하면 (가장 낮음) 별 5 (가장 높음)에서 1 별 등급을 신뢰도 평가를 획득할 수는 있습니다. 신뢰 등급 데 효과적으로:
- Azure Migrate Server 평가 P s/처리량 데이터 디스크 및 VM CPU/메모리 사용률 데이터가 필요합니다.
- VM에 연결 된 각 네트워크 어댑터에 대해 Azure Migrate 네트워크 입/출력 데이터에 필요 합니다.
- 사용률 데이터를 vCenter Server에서에서 사용할 수 없는 경우 Azure Migrate가 권장 크기를 신뢰할 수 있는 수 없습니다. 

사용 가능한 데이터 요소의 백분율에 따라 평가의 신뢰 등급은 다음 표에 요약 되어 있습니다.

   **데이터 지점 가용성** | **신뢰 등급**
   --- | ---
   0%-20% | 별 1개
   21%-40% | 별 2개
   41%-60% | 별 3개
   61%-80% | 별 4개
   81%-100% | 별 5개

- 5 개 보다 적은 평가 대 한 신뢰 등급을 받은 경우 적어도 하루를 기다린 다음 평가 다시 계산 합니다.
- 낮은 등급을는 크기 조정 권장 사항 불안정할 수를 의미 합니다. 이 경우 것이 좋습니다으로 사용할 평가 속성을 수정 하는-온-프레미스 평가 됩니다.

## <a name="common-assessment-issues"></a>일반적인 평가 문제

평가 영향을 주는 몇 가지 일반적인 환경 문제를 해결 하는 방법은 다음과 같습니다.

###  <a name="out-of-sync-assessments"></a>비동기 아웃 평가

만든 평가 추가 하거나 평가 만든 후 그룹에서 컴퓨터를 제거 하는 경우 표시 됩니다 **동기화 아웃**합니다. 평가 다시 실행 (**다시 계산**) 그룹 변경 내용을 반영 하도록 합니다.

### <a name="outdated-assessments"></a>오래 된 평가

평가 된 그룹에 있는 Vm에 온-프레미스 변경 내용이 평가 표시 됩니다 **오래 된**합니다. 변화를 반영 하도록 평가 다시 실행 합니다.

### <a name="missing-data-points"></a>누락 데이터 요소가

평가 다양 한 이유로 모든 데이터 요소에 없을 수 있습니다.

- Vm을 평가 하는 동안 전원이 수 및 성능 데이터를 수집 하지 않습니다. 
- 성능 기록을 기준으로 해당 월 동안 Vm을 만들 수 있습니다, 그리고 따라서 해당 성능 데이터는 1 개월 미만입니다. 
- 검색 후 즉시 평가 만들었습니다. 지정된 된 기간에 대 한 성능 데이터를 수집 하기 위해 평가 실행 하기 전에 지정 된 기간을 대기 해야 합니다. 예를 들어, 한 주에 대 한 성능 데이터를 평가 하려는 경우 검색 후 일주일 대기 해야 합니다. 평가 하지 않으면 5 개의 별 등급을 받지 합니다.
