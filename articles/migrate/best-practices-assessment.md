# <a name="best-practices-for-creating-assessments"></a>평가 를 만들기 위한 모범 사례

[Azure Migrate](migrate-overview.md)는 앱, 인프라 및 워크로드를 검색, 평가 및 Microsoft Azure로 마이그레이션하는 데 도움이 되는 도구의 허브를 제공합니다. 허브에는 Azure Migrate 도구와 타사 ISV(독립 소프트웨어 공급업체) 제품이 포함되어 있습니다.

이 문서에서는 Azure 마이그레이션 서버 평가 도구를 사용하여 평가를 만들 때의 모범 사례를 요약합니다.

## <a name="about-assessments"></a>평가에 관하여

Azure 마이그레이션 서버 평가를 사용하여 만든 평가는 데이터의 시점 스냅숏입니다. Azure 마이그레이션에는 두 가지 유형의 평가가 있습니다.

**평가 유형** | **세부 정보** | **데이터**
--- | --- | ---
**성능 기반** | 수집된 성과 데이터를 기반으로 권장 사항을 만드는 평가 | VM 크기 권장 사항은 CPU 및 메모리 사용률 데이터를 기반으로 합니다.<br/><br/> 디스크 유형 권장 사항(표준 HDD/SSD 또는 프리미엄 관리 디스크)은 온-프레미스 디스크의 IOPS 및 처리량을 기반으로 합니다.
**온-프레미스** | 성능 데이터를 사용하지 않는 평가로 권장 사항을 만듭니다. | VM 크기 권장 사항은 온-프레미스 VM 크기를 기반으로 합니다.<br/><br> 권장되는 디스크 유형은 평가에 대한 저장소 유형 설정에서 선택한 내용을 기반으로 합니다.

### <a name="example"></a>예제
예를 들어, 4개의 코어가 20%의 사용률을 가진 온-프레미스 VM과 10% 사용률의 8GB 메모리가 있는 경우 평가는 다음과 같습니다.

- **성능 기반 평가**:
    - 코어(4 x 0.20 = 0.8) 및 메모리(8GB x 0.10 = 0.8) 사용률을 기반으로 유효 코어 및 메모리를 식별합니다.
    - 크기 조정에 사용할 값을 얻기 위해 평가 속성에 지정된 컴포트 계수를 적용합니다(1.3x라고 가정해 보겠습니다). 
    - ~1.04 코어(0.8 x 1.3) 및 ~1.04GB(0.8 x 1.3) 메모리를 지원할 수 있는 Azure에서 가장 가까운 VM 크기를 권장합니다.

- **있는 것(온-프레미스) 평가:**
    -  4개의 코어가 있는 VM을 권장합니다. 8GB메모리.

## <a name="best-practices-for-creating-assessments"></a>평가 를 만들기 위한 모범 사례

Azure 마이그레이션 어플라이언스는 온-프레미스 환경을 지속적으로 프로파일러내고 메타데이터 및 성능 데이터를 Azure로 보냅니다. 어플라이언스를 사용하여 검색된 서버의 평가에 대한 다음 모범 사례를 따르십시오.

- **있는 것으로 만들기 평가:** 컴퓨터가 Azure 마이그레이션 포털에 표시되면 즉시 있는 것으로 평가를 만들 수 있습니다.
- **성능 기반 평가 만들기**: 검색을 설정한 후 성능 기반 평가를 실행하기 전에 최소 하루 이상 기다리는 것이 좋습니다.
    - 성능 데이터를 수집하는 데는 시간이 걸립니다. 평가를 실행하기 전에 적어도 하루를 기다리면 성능 데이터 요소가 충분합니다.
    - 성능 기반 평가를 실행하는 경우 평가 기간 동안 환경을 프로파일업해야 합니다. 예를 들어 성능 기간이 1주로 설정된 평가를 만드는 경우 검색을 시작한 후 모든 데이터 요소가 수집될 때까지 최소 1주일 동안 기다려야 합니다. 그렇지 않으면 평가에서 5성급 등급을 받지 못합니다.
- **평가 다시 계산**: 평가는 시적 스냅샷이므로 최신 데이터로 자동으로 업데이트되지 않습니다. 최신 데이터로 평가를 업데이트하려면 평가를 다시 계산해야 합니다.

다음 을 통해 Azure 마이그레이션으로 가져온 서버의 평가에 대한 다음 모범 사례를 따릅니다. CSV 파일:

- **있는 것으로 만들기 평가:** 컴퓨터가 Azure 마이그레이션 포털에 표시되면 즉시 있는 것으로 평가를 만들 수 있습니다.
- **성능 기반 평가 만들기**: 특히 온-프레미스에서 서버 용량을 과도하게 프로비저닝한 경우 비용 대비 비용을 더 잘 예측할 수 있습니다. 그러나 성능 기반 평가의 정확성은 서버에 대해 지정한 성능 데이터에 따라 달라집니다. 
- **평가 다시 계산**: 평가는 시적 스냅샷이므로 최신 데이터로 자동으로 업데이트되지 않습니다. 평가를 가져온 최신 데이터로 업데이트하려면 평가를 다시 계산해야 합니다.

## <a name="best-practices-for-confidence-ratings"></a>신뢰도 등급에 대한 모범 사례

성과 기반 평가를 실행하면 평가에 1성급(최저)에서 5성급(최고)까지의 신뢰도 등급이 부여됩니다. 신뢰도 등급을 효과적으로 사용하려면 다음을 수행하십시오.
- Azure 마이그레이션 서버 평가에는 VM CPU/메모리에 대한 사용률 데이터가 필요합니다.
- 온-프레미스 VM에 연결된 각 디스크에 대해 읽기/쓰기 IOPS/처리량 데이터가 필요합니다.
- VM에 연결된 각 네트워크 어댑터에 대해 네트워크 인/아웃 데이터가 필요합니다.

선택한 기간에 사용할 수 있는 데이터 포인트의 백분율에 따라 평가에 대한 신뢰도 등급은 다음 표에 요약되어 있습니다.

   **데이터 요소 가용성** | **신뢰 등급**
   --- | ---
   0%-20% | 별 1개
   21%-40% | 별 2개
   41%-60% | 별 3개
   61%-80% | 별 4개
   81%-100% | 별 5개


## <a name="common-assessment-issues"></a>일반적인 평가 문제

평가에 영향을 주는 몇 가지 일반적인 환경 문제를 해결하는 방법은 다음과 같습니다.

###  <a name="out-of-sync-assessments"></a>동기화 되지 않습니다 평가

평가를 만든 후 그룹에서 컴퓨터를 추가하거나 제거하면 만든 평가가 **동기화되지**않은 것으로 표시됩니다. 그룹 변경 사항을 반영하기 위해 평가를 다시**실행(다시**계산)합니다.

### <a name="outdated-assessments"></a>오래된 평가

평가된 그룹에 있는 VM에 대한 온-프레미스 변경 사항이 있는 경우 평가가 **오래된**것으로 표시됩니다. 변경 내용을 반영하려면 평가를 다시 실행합니다.

### <a name="low-confidence-rating"></a>낮은 신뢰도 등급

평가에는 다음과 같은 여러 가지 이유로 모든 데이터 요소가 없을 수 있습니다.

- 평가를 작성하는 기간 동안 환경을 프로파일링하지 않았습니다. 예를 들어 성능 기간을 1주로 설정한 *성능 기반 평가를* 만드는 경우 모든 데이터 요소가 수집될 때까지 검색을 시작한 후 최소 1주일 동안 기다려야 합니다. 언제든지 **다시 계산을** 클릭하여 최신 적용 신뢰도 등급을 확인할 수 있습니다. 신뢰도 등급은 *성능 기반* 평가를 만들 때만 적용됩니다.

- 평가 계산 기간에 일부 VM이 종료되었습니다. 평가 기간 중 일부 VM이 꺼지면 Server Assessment에서 해당 기간의 성능 데이터를 수집할 수 없습니다.

- Server Assessment에서 검색이 시작된 후 VM 몇 개가 생성되었습니다. 예를 들어 마지막 1달의 성능 기록에 대한 평가를 만들려고 하는데, 일부 VM이 불과 일주일 전에 환경에서 생성되었습니다. 이 경우 새 VM의 성능 데이터를 전체 기간에 사용할 수 없으며 신뢰 등급이 낮아집니다.
