# <a name="create-and-run-a-job-in-your-azure-iot-central-application"></a>Azure IoT Central 애플리케이션에서 작업 만들기 및 실행

Microsoft Azure IoT Central에서는 작업을 사용하여 연결된 디바이스를 대규모로 관리할 수 있습니다. 작업을 사용 하면 대량 장치 속성, 설정 및 명령에 대 한 업데이트 합니다. 이 문서에서는 사용자 고유의 응용 프로그램에서 작업을 사용 하 여 시작 하는 방법을 단계별로 안내 합니다.

## <a name="create-and-run-a-job"></a>작업 만들기 및 실행

이 섹션에서는 작업을 만들고 실행하는 방법을 보여줍니다. 여러 refrigerated 별도 컴퓨터에 대 한 팬 속도를 높이 방법을 보여줍니다.

1. 탐색 창에서 작업으로 이동합니다.

1. 선택 **+ 새로 만들기** 새 작업을 만듭니다.

    ![새 작업 만들기](./media/howto-run-a-job/createnewjob.png)

1. 이름 및 만들 작업을 식별 하는 설명을 입력 합니다.

1. 에 적용할 작업을 원하는 장치 집합을 선택 합니다. 장치 설정 후 장치 집합에 연결 된 장치를 사용 하 여 채우기 오른쪽을 표시 됩니다. 손상 된 장치 집합을 선택 하면 장치가 표시 하 고 장치 집합에는 손상 된 메시지를 참조 하세요.

1. 다음으로, (설정, 속성 또는 명령)을 정의 하는 작업의 유형을 선택 합니다. 선택 **+** 옆에 있는 작업의 유형을 선택 하 고 작업을 추가 합니다.

    ![작업 구성](./media/howto-run-a-job/configurejob.png)

1. 오른쪽에서 작업을 실행 하려는 장치를 선택 합니다. 맨 위의 확인란을 선택 하면 모든 장치는 전체 장치 집합에서 선택 됩니다. 옆의 확인란을 선택 하 여 **이름을**, 현재 페이지의 모든 장치를 선택 합니다.

1. 장치를 선택한 후 선택 **실행할** 하거나 **저장**합니다. 작업에서 기본 나타납니다 **작업** 페이지입니다. 이 보기에서 현재 실행 중인 작업의 기록과 모든 이전에 작업 실행을 볼 수 있습니다. 실행 중인 작업 목록 맨 위에 있는 항상 표시 됩니다. 저장 된 작업은 언제 든 지 실행 하거나 편집을 계속 다시 열 수 있습니다.

    ![작업 보기](./media/howto-run-a-job/viewjob.png)

    > [!NOTE]
    > 최대 30 일 이전에 실행된 작업의 기록을 볼 수 있습니다.

1. 작업의 개요를 얻으려면 목록에서 보려는 작업을 선택 합니다. 이 개요 작업 세부 정보, 장치 및 장치 상태 값을 포함합니다. 이 개요에서 선택할 수도 있습니다 **작업 세부 정보 다운로드** 의 장치 및 해당 상태 값을 포함 하 여 작업 세부 정보를.csv 파일을 다운로드 합니다. 이 정보는 문제 해결에 유용할 수 있습니다.

    ![디바이스 상태 보기](./media/howto-run-a-job/downloaddetails.png)

### <a name="stop-a-running-job"></a>실행 중인 작업 중지

실행 중인 작업을 중지 하려면 선택 하 고 **중지** 패널에 있습니다. 작업 상태 변경 작업이 중지 됩니다.

   ![작업 중지](./media/howto-run-a-job/stopjob.png)

### <a name="run-a-stopped-job"></a>중지된 작업 실행

현재 중지 되어 있는 작업을 실행 하려면 중지 작업을 선택 합니다. 선택할 **실행** 패널에 있습니다. 이제 작업을 반영 하도록 작업 상태 변경 내용을 다시 실행 됩니다.

   ![다시 시작된 작업](./media/howto-run-a-job/resumejob.png)

## <a name="copy-a-job"></a>작업 복사

사용자가 만든 기존 작업을 복사 하려면 주 작업 페이지에서 선택 하 고 선택 **복사**합니다. 편집할 작업 구성의 새 복사본을 엽니다. 저장 하거나 새 작업을 실행할 수 있습니다. 선택한 장치 집합에 모든 변경 사항이 생겼는지, 편집할이 복사 작업에 반영 하는 있습니다.

   ![복사 작업](./media/howto-run-a-job/copyjob.png)

## <a name="view-the-job-status"></a>작업 상태 확인

작업을 만든 후 합니다 **상태** 작업의 최신 상태 메시지를 사용 하 여 열 업데이트 합니다. 다음 표에서 가능한 상태 값을 나열합니다.

| 상태 메시지       | 상태 의미                                          |
| -------------------- | ------------------------------------------------------- |
| Completed            | 모든 디바이스에서 이 작업이 실행되었습니다.              |
| 실패               | 이 작업은 실패했으며 디바이스에서 완전히 실행되지 않았습니다.  |
| Pending              | 이 작업은 장치에서 실행 아직 되지 않았습니다.         |
| 실행 중              | 이 작업이 현재 디바이스에서 실행되고 있습니다.             |
| 중지됨              | 이 작업을 사용자가 수동으로 중지했습니다.           |

상태 메시지 작업에 연결 된 장치에 간략하게 나옵니다. 다음 표에서 가능한 장치 상태 값을 나열합니다.

| 상태 메시지       | 상태 의미                                                     |
| -------------------- | ------------------------------------------------------------------ |
| Succeeded            | 장치에서 작업이 성공적으로 실행 하는 횟수입니다.       |
| 실패               | 장치에서 실행 하는 작업이 실패 한 횟수입니다.       |

### <a name="view-the-device-status"></a>디바이스 상태 확인

영향을 받는 모든 장치 작업의 상태를 확인 하려면 작업을 선택 합니다. 장치 목록과 해당 상태 값을 포함 하 여 작업 세부 정보를 포함 하는.csv 파일을 다운로드 하려면 **작업 세부 정보 다운로드**합니다. 각 장치 이름 옆의 표시 상태 메시지를 다음 중 하나:

| 상태 메시지       | 상태 의미                                                                |
| -------------------- | ----------------------------------------------------------------------------- |
| Completed            | 이 디바이스에서 작업이 실행되었습니다.                                     |
| 실패               | 이 디바이스에서 작업 실행이 실패했습니다. 오류 메시지에 자세한 정보가 표시 됩니다.  |
| Pending              | 아직 작업 되지 않은이 장치에서 실행 됩니다.                                   |

> [!NOTE]
> 장치를 삭제 하는 경우 장치를 선택할 수 없습니다 및 장치 ID를 사용 하 여 삭제 된 것으로 표시
