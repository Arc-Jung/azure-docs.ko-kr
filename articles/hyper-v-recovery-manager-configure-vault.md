Azure Hyper-V 복구 관리자 구성
==============================

Azure Hyper-V 복구 관리자는 System Center 2012 서비스 팩 1(SP1) 또는 System Center 2012 R2에서 VMM(Virtual Machine Manager) 서버의 사설 클라우드에 있는 Hyper-V 가상 컴퓨터의 보호를 조정하고 관리합니다. Hyper-V 복구 관리자는 하나의 온-프레미스 Hyper-V 호스트 서버에서 다른 온-프레미스 Hyper-V 호스트 서버로의 가상 컴퓨터 장애 조치(Failover)를 오케스트레이션합니다. 보호 구성을 저장하는 데 Azure의 Hyper-V 복구 관리자 자격 증명 모음이 사용됩니다.

이 자습서 정보
--------------

이 자습서는 Hyper-V 복구 관리자 배포에 관해 빠른 연습을 제공합니다. 자세한 내용은 다음을 읽어 보시기 바랍니다.

-   [Hyper-V 복구 관리자용 계획 가이드](http://go.microsoft.com/fwlink/?LinkId=321294)?전체 배포를 시작하기 전에 완료해야 하는 계획 단계에 관한 자세한 정보를 제공합니다.
-   [Hyper-V 복구 관리자용 배포 가이드](http://go.microsoft.com/fwlink/?LinkId=321295)?전체 배포를 위한 단계별 지침을 제공합니다.
-   이 자습서를 수행하는 동안 문제가 발생하면 [Hyper-V 복구 관리자: 일반 오류 시나리오 및 해결 방법](http://go.microsoft.com/fwlink/?LinkId=389879)을 살펴보거나 [Azure 복구 서비스 포럼](http://go.microsoft.com/fwlink/?LinkId=313628)(영문)에 문의 사항을 게시하십시오.

시작하기 전에
-------------

이 자습서를 시작하기 전에 필수 구성 요소를 확인하십시오.

### Hyper-V 복구 관리자 필수 구성 요소

-   \*\*Azure 계정\*\*?Azure 복구 서비스 기능을 사용할 수 있는 Azure 계정이 필요합니다. 계정이 없거나 기능이 등록되어 있지 않으면 [Azure 무료 평가판](http://aka.ms/try-azure) 및 [Hyper-V 복구 관리자 가격 정보](http://go.microsoft.com/fwlink/?LinkId=378268)를 참조하십시오.
-   \*\*인증서(.cer)\*\*?Hyper-V 복구 관리자 자격 증명 모음에 VMM 서버를 등록하려면 자격 증명 모음에 대한 공개 키가 포함된 관리 인증서(.cer)를 업로드해야 합니다. 다음 사항에 유의하십시오.
    -   학습용의 경우 Makecert.exe 도구를 사용하여 만드는 자체 서명된 인증서를 사용할 수 있습니다. 전체 배포의 경우 계획 가이드의 [필수 구성 요소 및 지원](%20http://go.microsoft.com/fwlink/?LinkId=386485)에 설명되어 있는 인증서 필수 구성 요소에 부합하는 유효한 SSL 인증서를 사용할 수 있습니다.
    -   각 자격 증명 모음에는 언제라도 하나의 .cer 인증서만 연결됩니다. 필요에 따라 새 인증서를 업로드하여 기존 인증서를 덮어쓸 수 있습니다.
-   \*\*인증서(.pfx 파일)\*\*?.cer 인증서는 .pfx 파일(개인 키 포함)로 내보내야 합니다. 보호하려는 가상 컴퓨터가 포함된 각 VMM 서버로 이 파일을 가져옵니다. 그런 다음 배포하는 동안 각 VMM 서버에 Hyper-V 복구 관리자 공급자 에이전트를 설치할 때 .pfx 파일을 선택하여 VMM 서버를 자격 증명 모음에 등록합니다.

### VMM 필수 구성 요소

-   System Center 2012 SP1 또는 System Center 2012 R2에서 실행 중인 최소한 하나의 VMM 서버.
-   VMM 사설 클라우드. 하나의 VMM 서버만 실행 중인 경우 구성된 두 개의 클라우드가 있어야 합니다. 두 개 이상의 VMM 서버를 실행하는 경우 보호하려는 원본 VMM 서버와 복구에 사용할 대상 VMM 서버에 각각 최소한 하나의 클라우드가 있어야 합니다. 보호하려는 기본 클라우드는 다음을 포함해야 합니다.
    -   하나 이상의 VMM 호스트 그룹
    -   각 호스트 그룹에 있는 하나 이상의 Hyper-V 호스트 서버
-   장애 조치(Failover) 이후 가상 컴퓨터를 VM 네트워크와 연결하려면 Hyper-V 복구 관리자에서 네트워크 매핑을 구성합니다. 배포를 시작하기 전에 다음 사항을 확인하십시오.
    -   원본 VMM 서버의 클라우드에 있는 가상 컴퓨터가 VM 네트워크에 연결되었는지 확인합니다. 해당 VM 네트워크는 클라우드와 연결된 논리 네트워크에 연결되어야 합니다.
    -   대상 VMM 서버의 대상 클라우드에 해당 VM 네트워크가 있는지 확인합니다. 해당 VM 네트워크는 대상 클라우드와 연결된 해당 논리 네트워크에 연결되어야 합니다.

인증서 얻기 및 구성
-------------------

아래와 같이 인증서를 얻고 구성합니다.

1.  [연습용 인증서 얻기](#obtaincert)?MakeCert 도구를 사용하여 인증서를 얻거나 [인증서 요구 사항](http://go.microsoft.com/fwlink/?LinkId=321294)을 준수하는 기존 인증서를 사용합니다.
2.  [.pfx 형식으로 인증서 내보내기](#exportcert)?인증서가 있거나 인증서를 만들었던 서버에서 .cer 파일을 .pfx 파일로 내보냅니다(개인 키 포함).
3.  [VMM 서버로 .pfx 인증서 가져오기](#importcert)?.pfx 파일을 내보낸 후 내보낸 파일을 자격 증명 모음에 등록하려는 각 VMM 서버의 개인 인증서 저장소로 가져옵니다.

### 자체 서명된 인증서(.cer) 얻기

MakeCert 도구를 사용하여 모든 인증서 요구 사항을 준수하는 .cer x.509 인증서를 아래와 같이 만듭니다.

1.  MakeCert를 실행하려는 컴퓨터에 최신 버전의 [Windows SDK](http://go.microsoft.com/fwlink/?LinkId=378269)를 다운로드합니다. makecert.exe 명령은 핵심 Windows SDK(소프트웨어 개발 키트)의 일부이므로 전체 SDK를 다운로드하고 설치할 필요가 없습니다.
2.  위치 지정 페이지에서 \*\*Install the Windows Software Development Kit for Windows 8.1 to this computer\*\*를 선택합니다.
3.  CEIP(사용자 환경 개선 프로그램)에 참여 페이지에서 원하는 설정을 선택합니다.
4.  사용권 계약 페이지에서 계약 내용에 동의하려면 \*\*동의함\*\*을 클릭합니다.
5.  설치하려는 기능을 선택하십시오. 페이지에서 \*\*Windows SDK(소프트웨어 개발 키트)\*\*를 제외한 다른 모든 옵션의 선택을 취소합니다.
6.  설치를 완료한 후 makecert.exe가 C:\\ProgramFiles (x86)\\Windows Kits\\*WindowsVersion*\\bin\\x64 폴더에 나타나는지 확인합니다.
7.  관리자 권한으로 명령 프롬프트(cmd.exe)를 열고 makecert.exe 폴더로 이동합니다.
8.  다음 명령을 실행하여 자체 서명된 인증서를 만듭니다. CertificateName을 인증서에 사용할 이름으로 바꾸고, -e 다음에 인증서의 실제 만료 날짜를 지정합니다.

인증서가 만들어지고 같은 폴더에 저장됩니다. 다음 단계에서 이 인증서를 내보내기 위해 인증서를 좀더 액세스하기 쉬운 위치로 이동할 수도 있습니다.

### .pfx 형식으로 인증서 내보내기

이 절차의 단계를 완료하여 .cer 파일을 .pfx 형식으로 내보냅니다.

1.  시작 화면에서 \*\*mmc.exe\*\*를 입력하여 MMC(Microsoft Management Console)를 시작합니다.
2.  \*\*파일\*\* 메뉴에서 \*\*스냅인 추가/제거\*\*를 클릭합니다. \*\*스냅인 추가/제거\*\* 대화 상자가 나타납니다.
3.  \*\*사용 가능한 스냅인\*\*에서 \*\*인증서\*\*를 클릭한 다음 \*\*추가\*\*를 클릭합니다.
4.  \*\*컴퓨터 계정\*\*을 선택한 후 \*\*다음\*\*을 클릭합니다.
5.  \*\*로컬 컴퓨터\*\*를 선택한 후 \*\*마침\*\*을 클릭합니다.
6.  MMC의 콘솔 트리에서 \*\*인증서\*\*를 확장한 후 \*\*개인\*\*을 확장합니다.
7.  세부 정보 창에서 관리할 인증서를 클릭합니다.
8.  \*\*동작\*\* 메뉴에서 \*\*모든 작업\*\*을 가리킨 다음 \*\*내보내기\*\*를 클릭합니다. 인증서 내보내기 마법사가 나타납니다. \*\*다음\*\*을 클릭합니다.
9.  \*\*개인 키 내보내기\*\* 페이지에서 \*\*예\*\*를 클릭하여 개인 키를 내보냅니다. \*\*다음\*\*을 클릭합니다. 이 작업은 설치 후 개인 키를 다른 서버로 내보낼 경우에만 필요합니다.
10. 파일 내보내기 형식 페이지에서 \*\*개인 정보 교환 ? PKCS \#12(.PFX)\*\*를 선택합니다. \*\*다음\*\*을 클릭합니다.
11. \*\*암호\*\* 페이지에서 개인 키를 암호화하는 데 사용되는 암호를 입력하고 확인합니다. \*\*다음\*\*을 클릭합니다.
12. 마법사 페이지에 따라 인증서를 .pfx 형식으로 내보냅니다.

### VMM 서버로 .pfx 인증서 가져오기

서버를 내보낸 후 자격 증명 모음에 등록하려는 각 VMM 서버에 복사한 다음 아래와 같이 가져옵니다. VMM 서버에서 MakeCert.exe를 실행한 경우에는 해당 서버에 인증서를 가져올 필요가 없습니다.

1.  개인 키(.pfx) 인증서 파일을 로컬 서버의 위치에 복사합니다.
2.  인증서 MMC 스냅인에서 \*\*컴퓨터 계정\*\*을 선택한 후 다음을 클릭합니다.
3.  로컬 컴퓨터를 선택하고 \*\*마침\*\*을 클릭합니다. \*\*스냅인 추가/제거\*\* 대화 상자에서 \*\*확인\*\*을 클릭합니다.
4.  MMC에서 \*\*인증서\*\*를 확장하고, \*\*개인\*\*을 마우스 오른쪽 단추로 클릭하고, 모든 작업을 가리킨 후 \*\*가져오기\*\*를 클릭하여 인증서 가져오기 마법사를 시작합니다.
5.  인증서 가져오기 마법사 시작 페이지에서 \*\*다음\*\*을 클릭합니다.
6.  가져올 파일 페이지에서 \*\*찾아보기\*\*를 클릭하고 가져올 인증서가 포함된 .pfx 인증서 파일이 있는 폴더를 찾습니다. 적합한 파일을 선택한 다음 \*\*열기\*\*를 클릭합니다.
7.  암호 페이지의 \*\*암호\*\* 상자에 이전 절차에서 지정한 개인 키 파일에 대한 암호를 입력한 후 \*\*다음\*\*을 클릭합니다.
8.  인증서 저장소 페이지에서 \*\*모든 인증서를 다음 저장소에 저장\*\*을 선택하고, \*\*찾아보기\*\*를 클릭하고, \*\*개인\*\* 저장소를 선택하고, \*\*확인\*\*을 클릭한 후 다음을 클릭합니다.
9.  인증서 가져오기 마법사 완료 페이지에서 \*\*마침\*\*을 클릭합니다.

이 단계를 완료하면 자격 증명 모음에 업로드하기 위해 .cer 인증서를 선택할 수 있고, 공급자 설치 중 VMM 서버를 등록할 때 .pfx 인증서를 선택할 수 있습니다.

1단계: 자격 증명 모음 만들기

1.  [관리 포털](https://manage.windowsazure.com)에 로그인합니다.

2.  **데이터 서비스**를 클릭한 다음 **복구 서비스**를 클릭합니다.

    ![미리 보기 프로그램](./media/hyper-v-recovery-manager-configure-vault/RS_PreviewProgram.png)

3.  **복구 서비스**를 클릭하고 **새로 만들기**를 클릭하고 **Hyper-V 복구 관리자 자격 증명 모음**을 가리킨 후 **빠른 생성**을 클릭합니다.

    ![새 자격 증명 모음](./media/hyper-v-recovery-manager-configure-vault/RS_hvvault.png)

4.  **이름**에 자격 증명 모음을 식별하기 위한 이름을 입력합니다.

5.  **지역**에서 자격 증명 모음에 대한 지리적 지역을 선택합니다. 사용 가능한 지리적 지역에는 아시아 태평양 남동부, 유럽 북부 및 미국 동부가 포함됩니다.*\*\**

6.  **구독**에 구독 세부 정보를 입력합니다.

7.  **자격 증명 모음 만들기**를 클릭합니다.

포털의 아래쪽에 있는 알림에서 자격 증명 모음 상태를 확인합니다. 자격 증명 모음이 성공적으로 만들어졌다는 메시지가 표시되고 복구 서비스 페이지에서 \*\*온라인\*\*으로 표시됩니다.

2단계: 인증서(.cer) 업로드

1.  **복구 서비스** 페이지에서 필요한 자격 증명 모음을 엽니다.
2.  빠른 시작 아이콘을 클릭하여 빠른 시작 페이지를 엽니다.

    ![빠른 시작 아이콘](./media/hyper-v-recovery-manager-configure-vault/RS_QuickStartIcon.png)

3.  **인증서 관리**를 클릭합니다.

    ![빠른 시작](./media/hyper-v-recovery-manager-configure-vault/RS_QuickStart.png)

4.  **인증서 관리** 대화 상자에서 **파일 찾아보기**를 클릭하여 자격 증명 모음에 업로드할 .cer 파일을 찾습니다.

    ![인증서 관리](./media/hyper-v-recovery-manager-configure-vault/RS_ManageCert.png)

3단계: 공급자 다운로드 및 설치 자격 증명 모음에 등록하려는 각 VMM 서버에 Hyper-V 복구 관리자 공급자를 설치합니다. 최신 버전의 공급자 설치 파일은 Azure 다운로드 센터에 저장되어 있습니다. VMM 서버에서 설치 파일을 실행하면 공급자가 설치되고 VMM 서버는 자격 증명 모음에 등록됩니다.

1.  **빠른 시작** 페이지에서 **공급자 다운로드**를 클릭하여 공급자 설치 .exe 파일을 받습니다. 이 파일을 VMM 서버에서 실행하여 공급자 설치를 시작합니다.

    ![다운로드 에이전트](./media/hyper-v-recovery-manager-configure-vault/RS_installwiz.png)

2.  단계별 지침에 따라 공급자 설치를 완료합니다.

    ![설치 완료](./media/hyper-v-recovery-manager-configure-vault/RS_SetupComplete.png)

3.  공급자 설치를 완료한 후 마법사 단계에 따라 VMM 서버를 자격 증명 모음에 등록합니다.
4.  VMM 서버에서 실행 중인 공급자를 인터넷에 연결하는 방법을 인터넷 연결 페이지에서 지정합니다. 공급자는 서버에서 기본 인터넷 연결 설정을 사용할 수 있거나 사용자 지정 설정을 사용하도록 **인터넷 요청에 프록시 서버 사용**을 클릭할 수 있습니다.

    ![인터넷 설정](./media/hyper-v-recovery-manager-configure-vault/RS_ProviderProxy.png)

    이 연습 중에 사용자 지정 설정을 사용하려면 배포 가이드의 [2단계: 공급자 설치 및 VMM 서버 등록](http://go.microsoft.com/fwlink/?LinkId=378266)의 정보를 읽어 보십시오.

5.  인증서 등록 페이지에서 자격 증명 모음에 업로드한 .cer에 해당하는 .pfx 파일을 선택합니다.

    ![인증서 등록](./media/hyper-v-recovery-manager-configure-vault/RS_CertReg1.png)

    ![인증서 자격 증명 모음](./media/hyper-v-recovery-manager-configure-vault/RS_CertReg2.png)

6.  VMM 서버 페이지에서 VMM 서버에 대한 이름을 지정합니다. 이 이름은 Hyper-V 복구 관리자 콘솔에서 서버를 식별하는 데 사용됩니다.
7.  **클라우드 데이터를 자격 증명 모음과 동기화**를 선택하여 VMM 서버에 있는 모든 사설 클라우드의 데이터를 Hyper-V 복구 관리자 자격 증명 모음과 동기화합니다. 이 작업은 각 서버에서 한 번만 수행해야 합니다. 모든 클라우드를 동기화하지 않는 경우 클라우드 보호 설정을 구성하기 전에 각 클라우드를 개별적으로 게시하여 동기화할 수 있습니다.
8.  **등록**을 클릭하여 프로세스를 완료합니다.

    ![인터넷 설정](./media/hyper-v-recovery-manager-configure-vault/RS_PublishCloudSetup.png)

이 단계에서 Hyper-V 복구 관리자가 VMM 서버의 메타데이터를 검색하여 장애 조치(Failover) 및 복구를 오케스트레이션합니다. 서버가 성공적으로 등록되면 자격 증명 모음에서 서버 페이지의 \*\*리소스\*\* 탭에 서버 이름이 표시됩니다.

4단계: 클라우드 보호 설정 구성
------------------------------

VMM 서버가 등록되면 클라우드 보호 설정을 구성할 수 있습니다. 공급자를 VMM 서버에 설치했을 때 **클라우드 데이터를 자격 증명 모음과 동기화** 옵션을 사용할 수 없으면 VMM 콘솔의 Hyper-V 복구 관리자에 클라우드를 게시해야 합니다. Hyper-V 복구 관리자가 클라우드를 감지하면 보호 설정을 구성할 수 있습니다.

### 클라우드 게시

1.  VMM 콘솔에서 **VM 및 서비스** 작업 영역을 엽니다.
2.  **VM 및 서비스** 창에서 게시할 클라우드를 엽니다.
3.  클라우드 속성의 **일반** 페이지에서 **Azure Hyper-V 복구 관리자로 이 클라우드에 대한 구성 데이터 보내기**를 선택하여 클라우드를 게시합니다. 클라우드가 게시되면 자격 증명 모음에 표시됩니다.

    ![클라우드](./media/hyper-v-recovery-manager-configure-vault/RS_PublishCloud.png)

    ![게시된 클라우드](./media/hyper-v-recovery-manager-configure-vault/RS_Clouds.png)

### 클라우드 구성

보호하도록 클라우드를 구성하려면 다음을 수행합니다.

1.  빠른 시작 페이지에서 **보호 설정 구성**을 클릭합니다.
2.  **보호되는 항목** 탭에서 구성할 클라우드를 선택하고 **구성** 탭으로 이동합니다.

    ![클라우드 구성](./media/hyper-v-recovery-manager-configure-vault/RS_CloudConfig.png)

3.  **대상 위치**에서 복구에 사용할 클라우드를 관리하는 VMM 서버를 지정합니다.
4.  **대상 클라우드**에서 원본 클라우드의 가상 컴퓨터 장애 조치(Failover)에 사용할 대상 클라우드를 지정합니다.
5.  **복사 빈도**에서 기본 설정을 그대로 둡니다. 이 값은 원본 위치와 대상 위치 사이에 데이터를 동기화해야 하는 빈도를 지정합니다. 이 설정은 Hyper-V 호스트가 Windows Server 2012 R2를 실행 중일 경우에만 관련이 있습니다. 다른 서버의 경우에는 기본 설정인 5분이 사용됩니다.
6.  **추가 복구 지점**에서 기본 설정을 그대로 둡니다. 이 값은 추가 복구 지점을 만들지 여부를 지정합니다. 기본값인 0인 경우에는 주 가상 컴퓨터의 가장 최근 복구 지점만 복제본 호스트 서버에 저장됩니다.
7.  **응용 프로그램에 일관된 스냅숏의 빈도**에서 기본 설정을 그대로 둡니다. 이 값은 스냅숏을 만드는 빈도를 지정합니다. 스냅숏은 VSS(볼륨 섀도 복사본 서비스)를 사용하여 스냅숏이 만들어질 때 응용 프로그램이 일관된 상태가 되도록 합니다. 자습서 연습에서는 이 값을 설정하지 않으려면 구성하는 추가 복구 지점의 수보다 적도록 설정되어야 합니다.
8.  **데이터 전송 압축**에서 전송되는 복제된 데이터를 압축할지 여부를 지정합니다.
9.  **인증**에서 기본 및 복구 Hyper-V 호스트 서버 간에 트래픽을 인증하는 방법을 지정합니다. 작업 중인 Kerberos 환경이 구성되어 있지 않으면 이 연습용으로 HTTPS를 선택하는 것이 좋습니다. HTTPS를 선택한 호스트 서버의 경우 호스트 서버에서 서버 인증서를 사용하여 서로 인증하고 트래픽이 암호화됩니다. Hyper-V 복구 관리자는 HTTPS 인증을 사용하도록 자동으로 인증서를 구성합니다. 수동으로 구성할 필요가 없습니다. 이 설정은 Windows Server 2012 R2에서 실행 중인 Hyper-V 호스트 서버와만 관련이 있습니다.
10. **포트**에서 기본 설정을 그대로 둡니다. 이 값은 원본 및 대상 Hyper-V 호스트 컴퓨터가 복제 트래픽을 수신하는 포트 번호를 설정합니다.
11. **초기 복제 설정**에서 정기 복제가 시작되기 전에 원본 위치에서 대상 위치로 데이터의 초기 복제를 처리할 방법을 지정합니다.
    -   **네트워크 이용**?네트워크를 이용하여 데이터를 복사하면 시간이 많이 걸리고 리소스가 많이 필요할 수 있습니다. 클라우드에 상대적으로 하드 디스크가 작은 가상 컴퓨터가 포함된 경우와 주 VMM 서버가 넓은 대역폭을 사용하여 보조 VMM 서버에 연결된 경우 이 옵션을 사용하는 것이 좋습니다. 복사를 즉시 시작하도록 지정하거나 시간을 선택할 수 있습니다. 네트워크 복제를 사용하는 경우 사용률이 낮은 시간에 예약하는 것이 좋습니다.
    -   **오프라인**?이 방법은 외부 미디어를 사용하여 초기 복제를 수행하도록 지정합니다. 이 방법은 네트워크 성능 저하를 방지하려는 경우나 지리적으로 멀리 떨어진 위치에 유용합니다. 이 방법을 사용하려면 원본 클라우드에서 내보내기 위치를 지정하고 대상 클라우드에서 가져오기 위치를 지정합니다. 가상 컴퓨터에 대한 보호를 설정할 때 가상 하드 디스크가 지정된 내보내기 위치에 복사됩니다. 이 하드 디스크를 대상 사이트에 보내고 가져오기 위치에 복사합니다. 시스템에서는 가져온 정보를 복제 가상 컴퓨터에 복사합니다. 오프라인 복제 필수 구성 요소의 전체 목록을 보려면 배포 가이드에서 [3단계: VMM 클라우드의 보호 설정 구성](http://go.microsoft.com/fwlink/?LinkId=323469)을 참조하십시오.

#### 보호 구성 후 설정

클라우드를 구성한 후 원본 및 대상 클라우드에 구성된 모든 클러스터 및 호스트 서버가 복제에 대해 구성됩니다. 특히, 다음 항목이 구성됩니다.

-   복제 트래픽용 포트가 열리도록 Hyper-V 복제본에 사용되는 방화벽 규칙
-   복제에 필요한 인증서가 설치됩니다.
-   Hyper-V 복제본 설정이 구성됩니다.

클라우드 설정은 **구성** 탭에서 수정할 수 있습니다. 다음 사항에 유의하십시오.

-   보호할 가상 컴퓨터의 복구 요구 사항을 충족하는 대상 클라우드를 선택하는 것이 좋습니다.
-   클라우드는 단일 클라우드 쌍에 기본 클라우드 또는 대상 클라우드로만 속할 수 있습니다.
-   클라우드 구성을 저장하고 나면 작업이 생성되고 **작업** 탭에서 모니터링할 수 있습니다. 클라우드 구성을 저장한 후 대상 위치 또는 대상 클라우드를 수정하려면 클라우드 구성을 제거한 후 클라우드를 다시 구성해야 합니다.

5단계: 네트워크 매핑 구성
-------------------------

**네트워크** 탭에서 원본 VMM 서버의 VM 네트워크와 대상 VMM 서버의 VM 네트워크 간에 매핑을 구성합니다. 네트워크 매핑은 장애 조치(Failover) 이후 복제 가상 컴퓨터가 해당 네트워크에 연결되게 하며, 복제 가상 컴퓨터가 VM 네트워크에 액세스할 수 있는 호스트에 배치되도록 합니다. 계획 가이드의 [필수 구성 요소 및 지원](%20http://go.microsoft.com/fwlink/?LinkId=386485)에서 VMM 네트워크 요구 사항을 확인하십시오. 다음과 같이 네트워크를 매핑하는 것이 좋습니다.

-   보호에 대해 구성된 각 클라우드에서 사용하는 네트워크를 매핑합니다.
-   복제 가상 컴퓨터를 배치하는 동안 매핑이 사용되므로 가상 컴퓨터의 보호를 활성화하기 전에 네트워크 매핑을 수행합니다. 보호를 활성화한 후에 매핑을 구성하면 제대로 작동하지만 일부 복제 가상 컴퓨터를 마이그레이션해야 할 수 있습니다.

네트워크를 매핑하려면 다음을 수행합니다.

1.  빠른 시작 페이지에서 **네트워크 매핑 구성**을 클릭합니다.
2.  **원본 위치**에서 원본 VMM 서버를 선택합니다.
3.  **대상 위치**에서 대상 VMM 서버를 선택합니다. 단일 VMM 서버로 Hyper-V 복구 관리자를 배포하는 경우 원본 서버와 대상 서버가 동일합니다. 원본 VM 네트워크 및 연결된 대상 VM 네트워크의 목록이 표시됩니다. 매핑되지 않은 네트워크에 대해서는 빈 값이 표시됩니다.
4.  원본 및 대상 목록에서 매핑되지 않은 항목을 선택하고 **매핑**을 클릭합니다. 서비스는 대상 서버에서 VM 네트워크를 감지하고 표시합니다.

    ![인증서 관리](./media/hyper-v-recovery-manager-configure-vault/RS_networks.png)

5.  대상 네트워크 선택 페이지에서 대상 VMM 서버에서 사용할 대상 VM 네트워크를 선택합니다. ![대상 네트워크](./media/hyper-v-recovery-manager-configure-vault/RS_TargetNetwork.png)

6.  원본 및 대상 네트워크 이름 옆의 정보 아이콘을 클릭하여 각 네트워크에 대한 서브넷 및 종류를 봅니다.

    대상 네트워크를 선택하는 경우 원본 네트워크를 사용하는 보호된 클라우드가 표시되며, 이 클라우드와 연결된 대상 VM 네트워크의 가용성도 표시됩니다. 보호에 사용된 모든 클라우드에 사용할 수 있는 대상 네트워크를 선택하는 것이 좋습니다.

7.  확인 표시를 클릭하여 매핑 프로세스를 완료합니다. 매핑 프로세스를 추적하는 작업이 시작됩니다. **작업** 탭에서 볼 수 있습니다.

    매핑은 원본 VM 네트워크에 연결된 가상 컴퓨터에 해당하는 기존의 복제 가상 컴퓨터를 대상 VM 네트워크에 연결합니다. 또한 원본 VM 네트워크에 연결된 가상 컴퓨터에서 복제를 활성화한 후에 생성된 새 복제 가상 컴퓨터도 대상 VM 네트워크에 연결합니다.

### 네트워크 매핑 수정

**네트워크** 탭에서 네트워크 매핑을 수정하거나 제거할 수 있습니다. 다음 사항에 유의하십시오.

-   선택한 매핑을 해제하면 해당 매핑이 제거되고 매핑 당시 연결되었던 네트워크에서 복제 가상 컴퓨터와의 연결이 끊어집니다.
-   선택한 매핑을 수정하면 변경 사항이 업데이트되고 제공된 새 네트워크 설정에 복제 가상 컴퓨터가 연결됩니다.

6단계: 가상 컴퓨터의 보호 활성화
--------------------------------

서버, 클라우드 및 네트워크가 제대로 구성되었으면 클라우드에서 가상 컴퓨터에 대한 보호를 설정할 수 있습니다. 보호할 각 가상 컴퓨터를 마우스 오른쪽 단추로 클릭하고 \*\*복구 사용\*\*을 선택하여 VMM 콘솔에서 보호를 설정합니다.

보호를 사용하도록 설정했으면 클라우드의 가상 컴퓨터 목록에서 해당 가상 컴퓨터를 볼 수 있습니다. \*\*작업\*\* 탭에서 보호 사용 동작의 진행 상황을 볼 수 있습니다.

![가상 컴퓨터](./media/hyper-v-recovery-manager-configure-vault/RS_Clouds.png)

가상 컴퓨터의 보호를 설정하면 세 개의 작업이 생성됩니다. 먼저 보호 사용 작업이 실행됩니다. 그런 다음 초기 복제가 완료된 후 두 개의 보호 완료 작업이 더 실행됩니다. 이러한 세 작업이 성공적으로 완료되어야만 가상 컴퓨터를 장애 조치(Failover)할 준비가 됩니다.

7단계: 복구 계획 구성 및 실행
-----------------------------

복구 계획은 단일 단위로 처리될 수 있도록 가상 컴퓨터를 서로 그룹화하고 그룹이 장애 조치(Failover)되는 순서를 지정합니다. 가상 컴퓨터의 그룹 정의뿐만 아니라 자동 스크립트를 실행하거나 수동 동작의 확인을 기다리도록 복구 계획을 사용자 지정할 수 있습니다. 복구 계획을 만들려면 다음을 수행합니다.

1.  빠른 시작 페이지에서 **복구 계획 만들기**를 클릭합니다.
2.  복구 계획 이름 및 대상 지정 페이지에서 이름, 원본 및 대상 VMM 서버를 입력합니다. 단일 VMM 서버에서 Hyper-V 복구 관리자를 배포하는 경우 원본 및 대상 서버가 동일합니다. 원본 VMM 서버는 보호가 설정된 가상 컴퓨터를 포함해야 합니다.

    ![복구 계획 만들기](./media/hyper-v-recovery-manager-configure-vault/RS_RecoveryPlan1.png)

3.  가상 컴퓨터 선택 페이지에서 복구 계획에 추가할 가상 컴퓨터를 선택합니다. 이러한 가상 컴퓨터는 복구 계획 기본 그룹(그룹 1)에 추가됩니다.
4.  확인 표시를 클릭하여 복구 계획을 만듭니다. 만드는 복구 계획은 **복구 계획** 탭에서 삭제할 수 있습니다.

    ![복구 계획 VM](./media/hyper-v-recovery-manager-configure-vault/RS_RecoveryPlan2.png)

복구 계획을 만들고 나면 다음 작업을 수행할 수 있습니다.

-   계획을 사용자 지정합니다. 새 그룹을 복구 계획에 추가하거나 가상 컴퓨터를 그룹에 추가할 수 있습니다. 지정된 그룹 전후로 완료된 스크립트 및 수동 작업 형식으로 사용자 지정 작업을 정의할 수 있습니다. 그룹 이름이 점점 늘어납니다. 기본 복구 계획 그룹인 그룹 1 이후에 그룹 2를 만들고 그런 다음 그룹 3 등을 만듭니다. 가상 컴퓨터가 그룹 순서에 따라 장애 조치(Failover)됩니다. 복구 계획 사용자 지정에 대한 자세한 내용은 배포 가이드에서 [6단계: 복구 계획 만들기 및 사용자 지정](http://go.microsoft.com/fwlink/?LinkId=378271)을 참조하십시오.
-   테스트 장애 조치(Failover)를 실행합니다.

### 복구 계획 실행

복구 계획은 사전 진단 테스트 또는 계획된 장애 조치(Failover)의 일부로써 실행되거나 계획되지 않은 장애 조치(Failover) 시 실행될 수 있습니다. 이 연습에서는 테스트 장애 조치(Failover)를 실행하는 방법에 대해 설명합니다. 다른 유형의 장애 조치(Failover)에 대한 내용은 [작업 및 모니터링 가이드](hhttp://go.microsoft.com/fwlink/?LinkId=378272)를 읽어 보십시오.

### 계획 실행

테스트 장애 조치(Failover)를 실행하여 테스트 환경에서 복구 계획을 확인합니다. 복구 계획 및 가상 컴퓨터 장애 조치(Failover) 전략이 예상한 대로 작동 중인지 확인하는 데 테스트 장애 조치(Failover)가 유용합니다. 테스트 장애 조치(Failover)에서는 격리된 네트워크에서 장애 조치(Failover) 및 복구 메커니즘을 시뮬레이션합니다.

##### 시작하기 전

테스트 장애 조치(Failover)가 트리거되면 장애 조치(Failover) 이후에 테스트 가상 컴퓨터를 네트워크에 연결하는 방법을 지정하도록 요청됩니다.

-   기존 네트워크를 사용하려면 프로덕션에는 사용되지 않는 별도의 논리 네트워크를 이 용도로 만드는 것이 좋습니다.
-   테스트 VM 네트워크를 자동으로 만드는 옵션을 선택하는 경우 테스트 장애 조치(Failover)를 완료한 이후에 임시 네트워크 및 테스트 가상 컴퓨터를 자동으로 정리합니다.
-   VLAN(가상 LAN) 기반 논리 네트워크를 사용 중인 경우 논리 네트워크에 추가한 네트워크 사이트가 격리되어 있는지 확인합니다.
-   Windows 네트워크 가상화 기반 논리 네트워크를 사용 중인 경우 Hyper-V 복구 관리자가 격리된 VM 네트워크를 자동으로 만듭니다.

##### 테스트 장애 조치(Failover) 실행

다음과 같이 복구 계획용 테스트 장애 조치(Failover)를 실행합니다.

1.  \*\*복구 계획\*\* 탭에서 필요한 복구 계획을 선택합니다.
2.  장애 조치(Failover)를 시작하려면 \*\*테스트 장애 조치(Failover)\*\* 단추를 클릭합니다.
3.  테스트 장애 조치(Failover) 확인 페이지에서 다음과 같이 테스트 장애 조치(Failover) 이후에 가상 컴퓨터를 네트워크에 연결하는 방법을 지정합니다.

-   \*\*없음\*\*?VM 네트워크가 테스트 장애 조치(Failover)에 사용되지 않도록 지정하려면 이 설정을 선택합니다. 네트워크 구성 대신 개별 가상 컴퓨터를 테스트하려면 이 옵션을 사용합니다. 또한 테스트 장애 조치(Failover) 기능이 어떻게 작동하는지를 빠르게 알 수 있습니다. 장애 조치(Failover) 이후 테스트 가상 컴퓨터가 네트워크에 연결되지 않습니다.
-   \*\*기존 네트워크 사용\*\*?테스트 장애 조치(Failover)에 사용할 VM 네트워크를 이미 만들어 격리한 경우 이 옵션을 사용합니다. 장애 조치(Failover) 이후에 테스트 장애 조치(Failover)에 사용된 모든 테스트 가상 컴퓨터가 \*\*VM 네트워크\*\*에 지정된 네트워크에 연결됩니다.
-   \*\*자동으로 만들기\*\*?논리 네트워크에서 지정하는 설정에 따라 Hyper-V 복구 관리자가 자동으로 VM 네트워크 및 관련 네트워크 사이트를 만들도록 지정하려면 이 설정을 선택합니다. 복구 계획에서 VM 네트워크를 두 개 이상 사용할 경우 이 옵션을 사용합니다. Windows 네트워크 가상화 네트워크의 경우 이 옵션을 사용하여 복제 가상 컴퓨터의 네트워크에서 동일한 설정(서브넷 및 IP 주소 풀)으로 VM 네트워크를 자동으로 만들 수 있습니다. 이러한 VM 네트워크는 테스트 장애 조치(Failover)가 완료된 후 자동으로 정리됩니다.

테스트 장애 조치(Failover)를 완료한 후 다음을 수행합니다.

1.  가상 컴퓨터가 성공적으로 시작되는지 확인합니다.
2.  가상 컴퓨터가 성공적으로 시작되는지 확인한 후 격리된 환경을 정리하기 위해 테스트 장애 조치(Failover)를 완료합니다. 자동으로 VM 네트워크를 만들도록 선택한 경우 정리 작업에서 모든 테스트 가상 컴퓨터 및 테스트 네트워크를 삭제합니다.
3.  \*\*참고\*\*를 클릭하여 테스트 장애 조치(Failover)와 연관된 항목을 기록한 후 저장합니다.
4.  복구 계획에 대해 테스트 장애 조치(Failover)가 시작되면 복구 계획의 세부 정보 페이지에 장애 조치(Failover) 프로세스가 표시됩니다. 개별 테스트 장애 조치(Failover) 단계 및 상태를 보고 해당 테스트 장애 조치(Failover)와 관련된 참고를 보거나 만들 수 있습니다. 또한 \*\*작업\*\* 탭에서 장애 조치(Failover) 작업을 확인하여 그 세부 정보를 볼 수 있습니다. 세부 정보에는 장애 조치(Failover)와 관련된 각 작업, 상태, 시작 시간 및 기간이 표시됩니다. 장애 조치(Failover) 목록의 작업을 Excel 스프레드시트로 내보낼 수 있습니다.

다음 사항에 유의하십시오.

-   테스트 장애 조치(Failover) 확인 페이지에 테스트 가상 컴퓨터가 만들어질 VMM 서버에 대한 세부 정보가 표시됩니다.
-   **작업** 탭에서 테스트 장애 조치(Failover) 작업의 진행 상황을 살펴볼 수 있습니다.

8단계: 모니터링
---------------

### 작업 탭 및 대시보드 사용

#### 작업

**작업** 탭에는 Hyper-V 복구 관리자 자격 증명 모음에서 수행한 주 작업이 표시됩니다. 이 작업에는 클라우드에 대한 보호 구성, 가상 컴퓨터의 보호 설정 및 해제, 장애 조치(Failover) 실행(계획됨, 계획되지 않음 또는 테스트), 계획되지 않은 장애 조치(Failover) 커밋 및 역방향 복제가 포함됩니다.

**작업** 탭에서 다음 작업을 수행할 수 있습니다.

-   **작업 쿼리 실행**? 지정한 조건과 일치하는 작업을 검색하는 쿼리를 실행할 수 있습니다. 다음 매개 변수를 사용하여 작업을 필터링할 수 있습니다.

-   특정 VMM 서버에서 실행한 작업을 찾습니다.
-   클라우드, 가상 컴퓨터, 네트워크, 클라우드의 복구 계획 또는 이 모두에서 수행된 작업을 찾습니다.
-   특정 시간 범위 동안 발생한 작업을 식별합니다.

쿼리는 최대 200개의 작업을 반환할 수 있습니다. 따라서 반환 횟수가 최대값보다 적도록 쿼리 매개 변수의 범위를 좁히는 것이 좋습니다.

-   **작업 세부 정보 가져오기**?**작업** 목록에서 작업을 클릭하여 세부 정보를 가져올 수 있습니다. 세부 정보에는 작업의 이름, 상태, 시작 시간 및 지속 시간이 포함되어 있습니다. 작업 세부 정보 탭에서 작업 세부 정보를 Excel 스프레드시트로 내보내거나, 실패하거나 건너뛰거나 취소된 작업을 다시 시작해볼 수 있습니다.
-   **작업 내보내기**?작업 쿼리 결과를 Excel 스프레드시트로 내보낼 수 있습니다.
-   **다시 시작**?실패한 작업을 다시 시작하여 다시 실행할 수 있습니다.
-   **오류 세부 정보**?실패한 작업에 대해 **오류 세부 정보**를 클릭하여 작업의 오류 목록을 가져올 수 있습니다. 각 오류에 대해 가능한 원인 및 권장 해결 방법이 표시됩니다. 문제 해결을 위해 오류 설명이 클립보드에 복사될 수 있습니다.

> **작업** 탭은 다음 정보를 제공합니다.

>-   \*\*이름\*\*?작업 이름
>-   \*\*항목\*\*?클라우드 이름, 복구 계획, VMM 서버 또는 작업이 실행된 가상 컴퓨터
>-   \*\*상태\*\*?작업 상태 - 완료, 취소, 실패 등
>-   \*\*유형\*\*?작업 유형
>-   \*\*시작 시간\*\*?작업이 시작된 시간
>-   \*\*작업 시간\*\*?작업 기간

![작업 탭](./media/hyper-v-recovery-manager-configure-vault/RS_Jobs.png)

#### 작업 쿼리 실행

다음과 같은 지정한 조건과 일치하는 작업을 검색하는 쿼리를 실행할 수 있습니다.

-   특정 VMM 서버에서 실행된 작업
-   특정 클라우드, 가상 컴퓨터, 네트워크, 복구 계획 또는 이 모두에서 수행된 작업
-   특정 시간 범위 동안 발생한 작업

작업 쿼리를 실행하려면 다음을 수행합니다.

1.  \*\*작업\*\* 탭을 엽니다.
2.  \*\*서버\*\*에서 작업이 실행된 VMM 서버를 지정합니다.
3.  \*\*유형\*\*에서 클라우드, 가상 컴퓨터, 네트워크 또는 클라우드의 복구 계획에서 작업이 수행되었는지 여부를 지정합니다.
4.  \*\*상태\*\*에서 작업 상태를 지정합니다.
5.  \*\*기간\*\*에서 작업이 발생한 시간 범위를 지정합니다.

#### 대시보드

\*\*대시보드\*\* 탭은 조직의 Hyper-V 복구 관리자 사용에 대한 간략한 개요를 제공합니다. Hyper-V 복구 관리자 자격 증명 모음에 의해 보호되는 가상 컴퓨터를 한 곳에서 볼 수 있는 수단입니다. 대시보드에서는 다음 기능을 제공합니다.

**대시보드** 탭에서 다음 작업을 수행할 수 있습니다.

-   **공급자 다운로드**?VMM 서버에 설치할 Hyper-V 복구 관리자 공급자를 다운로드합니다.
-   **인증서 관리**?자격 증명 모음과 연결된 인증서의 설정을 수정합니다.
-   **삭제?**자격 증명 모음을 삭제합니다.
-   **가상 컴퓨터 다시 동기화**?Hyper-V 복구 관리자가 예상대로 동기화되지 않은 기본 및 복구 가상 컴퓨터가 있음을 감지한 경우, 관련 가상 컴퓨터의 목록을 보고 대시보드에서 다시 동기화할 가상 컴퓨터를 선택할 수 있습니다.

**대시보드** 탭은 다음 정보를 제공합니다.

-   \*\*간략 상태\*\*?복구 서비스 및 Hyper-V 복구 관리자 자격 증명 모음에 대한 중요한 구성 정보를 표시합니다. 즉, 자격 증명 모음이 온라인 상태인지 여부, 자격 증명 모음에 할당된 인증서, 인증서의 만료 날짜 및 서비스에 대한 구독 정보를 볼 수 있습니다.
-   \*\*최근 작업\*\*?최근 24시간 동안 성공이나 실패한 작업 또는 진행 중이거나 동작을 대기 중인 작업을 보여 줍니다.
-   \*\*문제\*\*?대시보드는 VMM 서버 연결 문제 및 클라우드 구성 또는 가상 컴퓨터 복제 동기화 문제에 대한 정보를 보여 줍니다. 문제에 대한 세부 정보를 얻거나 문제와 관련된 작업을 보거나 가상 컴퓨터를 다시 동기화해볼 수 있습니다.
