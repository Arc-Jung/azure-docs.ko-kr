# <a name="how-to-configure-a-dataset-mapping-for-a-received-share-in-azure-data-share-preview"></a>Azure 데이터 공유 미리 보기에서 받은 공유 데이터 집합 매핑을 구성 하는 방법

이 문서에서는 Azure 데이터 공유 미리 보기를 사용 하 여 받은 공유 데이터 집합 매핑을 구성 하는 방법에 설명 합니다. 데이터 공유 초대를 수락 하도록 선택한 경우이 작업을 수행 하려는 "Accept 나중에 구성 하 고". 이 고, 그렇지 받은 데이터에 대 한 대상 저장소 계정을 변경 하려는 하기만 하면 됩니다. 

## <a name="navigate-to-a-received-data-share"></a>수신한 데이터를 공유로 이동

Azure 데이터 공유 서비스에서 받은 공유로 이동 하 고 선택 합니다 **세부 정보** 탭 합니다. 

![데이터 집합 매핑](./media/dataset-mapping.png "데이터 집합 매핑") 

대상, 할당 하 고 클릭 하려는 데이터 집합 옆의 확인란 **+ 대상에 매핑할**합니다. 대상 저장소 계정을 이미 구성한 다른 저장소 계정에 매핑을 변경 하려는 경우 첫 번째의 매핑을 해제 해야 합니다. 

![대상에 매핑할](./media/dataset-map-target.png "맵 대상") 

## <a name="select-a-new-storage-account"></a>새 저장소 계정을 선택 합니다. 

데이터를에서 이동 하려는 저장소 계정을 선택 합니다. 에 이미 있는 모든 데이터를 저장소 계정에 이전에 매핑되어 있는지 참고 새 저장소 계정에 자동으로 이동 되지 않습니다.

![대상 저장소 계정](./media/map-target.png "대상 저장소")
