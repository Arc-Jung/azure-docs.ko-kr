# <a name="how-to-configure-a-dataset-mapping-for-a-received-share-in-azure-data-share-preview"></a>Azure 데이터 공유 미리 보기에서 수신 된 공유에 대 한 데이터 집합 매핑을 구성 하는 방법

이 문서에서는 Azure 데이터 공유 미리 보기를 사용 하 여 수신 된 공유에 대 한 데이터 집합 매핑을 구성 하는 방법을 설명 합니다. 데이터 공유 초대를 수락 했지만 "나중에 수락 및 구성" 하도록 선택 하는 경우이 작업을 수행할 수 있습니다. 그렇지 않은 경우에는 수신 된 데이터에 대 한 대상 저장소 계정을 변경 하는 것이 좋습니다. 

## <a name="navigate-to-a-received-data-share"></a>받은 데이터 공유로 이동

Azure 데이터 공유 서비스에서 받은 공유로 이동 하 여 **세부 정보** 탭을 선택 합니다. 

![데이터 집합 매핑](./media/dataset-mapping.png "데이터 집합 매핑") 

대상을 할당 하려는 데이터 집합 옆의 상자를 선택 하 고 **+ 지도를**클릭 하 여 대상으로 지정 합니다. 대상 저장소 계정을 이미 구성 하 고 다른 저장소 계정에 대 한 매핑을 변경 하려는 경우 먼저 매핑을 해제 해야 할 수 있습니다. 

대상(./media/dataset-map-target.png "맵에 대상 맵에") ![매핑] 

## <a name="select-a-new-storage-account"></a>새 저장소 계정 선택 

데이터를 저장할 저장소 계정을 선택 합니다. 이전에 매핑한 저장소 계정에 이미 있는 모든 데이터는 자동으로 새 저장소 계정으로 이동 되지 않습니다.

![대상 저장소 계정](./media/map-target.png "대상 저장소")
