# <a name="azure-data-share-preview-concepts"></a>Azure 데이터 공유 미리 보기 개념 

Azure 데이터 공유 미리 보기 데이터 공유와 관련 된 몇 가지 새로운 용어를 소개 합니다. 이 문서는 자주 사용 되는 일부 서비스 전체에서 사용 되는 볼 수 있는 용어를 설명 합니다. 

## <a name="data-provider"></a>데이터 공급자

데이터 공급자는 소비자를 사용 하 여 데이터를 공유 하는 조직. 일반적으로 데이터 공급자는 소유자 또는 데이터의 큐레이터 수 있습니다. 데이터 공급자는 다양 한 종류의 데이터를 공유 하려고 합니다. 데이터 공급자를 공유 하려고 할 수 있는 데이터의 예로 point of sales 또는 시계열 데이터와 같은 원시 데이터를 포함 합니다. 데이터 공급자를 이미 분석 및 정보를 포함 하는 사전 처리, 큐 레이트 데이터를 공유할 수도 있습니다. 

## <a name="data-consumer"></a>데이터 소비자 

데이터 소비자는 데이터 공급자에서 데이터를 수신 하는 조직입니다. 데이터 소비자는 통찰력을 끌어내기 자신의 데이터를 사용 하 여 공유 데이터를 조인 하려는 수 있습니다. 경우에 따라 데이터 소비자는 이미 처리 된 데이터 수신할 수 있습니다. 

## <a name="data-share"></a>데이터 공유

데이터 공유는 단일 엔터티로 공유 되는 데이터 집합의 그룹입니다. Azure 데이터 공유에서 지원 되는 Azure 데이터 원본에서 데이터 집합 수 있습니다. 현재 Azure 데이터 공유는 Azure Blob Storage 및 Azure Data Lake Store를 지원합니다. 

## <a name="share-subscription"></a>구독 공유 

데이터 소비자는 데이터 공급자에서 데이터 공유 초대를 수락 하면 공유 구독이 만들어집니다. 데이터 공급자를 이동 하 여 현재 공유 구독을 볼 수 있습니다 **전송 공유** Azure 데이터에서 계정을 공유 하 고 선택 하 **구독 공유**합니다.

데이터 소비자는 경우 해당 구독이 활성 공유로 이동 하 여 확인할 수 있습니다 **받은 공유** 받은 해당 공유의 상태를 검토 합니다. 

## <a name="snapshot"></a>스냅숏

데이터 공유 초대를 수락 하면 스냅숏은 데이터 소비자가 만들 수 있습니다. 사용자가 초대에 동의 하는 경우 공유 데이터의 전체 스냅숏을 트리거할 수 있습니다. 스냅샷은 데이터 소비자에 스냅숏이 생성 되는 특정 시점에서 데이터의 복사본입니다. 

스냅숏의-전체 및 증분는 다음과 같은 두 종류가 있습니다. 전체 스냅숏 데이터 공유에 있는 모든 데이터를 포함 합니다. 증분 스냅숏을 업데이트/추가 된 마지막 스냅숏이 트리거된 하므로 모든 데이터를 포함 합니다. 

## <a name="snapshot-settings-in-azure-data-share"></a>Azure 데이터 공유에서 스냅숏 설정
 
데이터 공급자는 데이터 공유에 대 한 스냅숏 설정을 설정할 수 있습니다. 이 설정은 데이터 소비자가 발생 하는 증분 업데이트를 받을 수 있습니다. 데이터 공급자가 공유 된 데이터에 대 한 업데이트를 받으려면 데이터 소비자를 원하는 경우이 설정을 사용 해야 합니다. 

데이터 공급자는이 설정을 사용 하도록 설정 하면, 되풀이 간격을 선택할 수 있습니다. 되풀이 간격을 시간별 또는 일별 수 있습니다. 

데이터 소비자가 새 스냅숏을 먼저 생성은 이후에 변경 된 모든 데이터를 포함 하는 증분 업데이트를 받으려면이 스냅숏 일정에 참여할 수 있습니다. 

## <a name="invitation"></a>초대

데이터 공급자에는 여러 수신자가 해당 데이터 공유를 초대할 수 있습니다. 받는 사람에 게 데이터 공유를 추가 하 여 변경할 수 있습니다. 초대는 데이터 공유를 만든 후에 추가할 수 있습니다. 

데이터 공급자는 전송 된 후 초대를 삭제할 수 있습니다. 데이터 공급자를 삭제 하면 초대를 승인 된 후 데이터 소비자는 활성 공유 구독이 아직 수 참고 합니다. 초대를 삭제 하는 데이터 공급자를 아직 수락 하지 않았습니다 데이터 소비자 수락할 수 없습니다. 

## <a name="recipient"></a>받는 사람

수신자는 데이터 공유 초대를 받는 사람이 합니다. 일반적으로 데이터 공급자는 자신이 만든 데이터 공유를 받는 사람을 추가 합니다. 초대 받는 사람이 초대를 수락 되 면 되는 데이터 소비자 됩니다.
