# <a name="azure-data-share-concepts"></a>Azure 데이터 공유 개념 

Azure 데이터 공유는 데이터 공유와 관련된 몇 가지 새로운 용어를 소개합니다. 이 문서에서는 서비스 전체에서 자주 사용되는 용어를 설명합니다. 

## <a name="data-provider"></a>데이터 공급자

데이터 공급자는 소비자와 데이터를 공유하는 조직입니다. 일반적으로 데이터 공급자는 데이터의 소유자 또는 큐레이터일 수 있습니다. 데이터 공급자는 다양한 유형의 데이터를 공유하려고 합니다. 데이터 공급자가 공유할 수 있는 데이터의 예로는 판매 시점 또는 열계 데이터와 같은 원시 데이터가 포함됩니다. 데이터 공급자는 이미 분석 및 인사이트를 포함하는 미리 처리되고 선별된 데이터를 공유할 수도 있습니다. 

## <a name="data-consumer"></a>데이터 소비자 

데이터 소비자는 데이터 공급자로부터 데이터를 수신하는 조직입니다. 데이터 소비자는 통찰력을 도출하기 위해 공유 데이터를 자신의 데이터와 결합하려고 할 수 있습니다. 경우에 따라 데이터 소비자가 이미 처리된 데이터를 수신할 수 있습니다. 

## <a name="data-share"></a>Data Share

데이터 공유는 단일 엔터티로 공유되는 데이터 집합 그룹입니다. 데이터 집합은 Azure 데이터 공유에서 지원되는 여러 Azure 데이터 원본에서 온 것일 수 있습니다. 현재 Azure 데이터 공유는 Azure Blob 저장소 및 Azure 데이터 레이크 저장소를 지원합니다. 

## <a name="share-subscription"></a>구독 공유 

공유 구독은 데이터 소비자가 데이터 공급자의 데이터 공유 초대를 수락할 때 만들어집니다. 데이터 공급자는 Azure Data Share 계정에서 **보낸 공유로** 이동하여 구독 을 공유 구독을 선택하여 활성 공유 구독을 볼 수 **있습니다.**

데이터 소비자는 수신된 공유로 이동하여 **수신된** 공유의 상태를 확인하여 활성 공유 구독이 있는지 확인할 수 있습니다. 

## <a name="snapshot"></a>스냅샷

데이터 공유 초대를 수락할 때 데이터 소비자가 스냅샷을 만들 수 있습니다. 초대를 수락하면 초대를 수락하면 공유된 데이터의 전체 스냅숏을 트리거할 수 있습니다. 스냅숏은 데이터 소비자가 스냅숏을 생성한 시점의 데이터 복사본입니다. 

스냅숏에는 전체 및 증분의 두 가지 유형이 있습니다. 전체 스냅숏에는 데이터 공유 내의 모든 데이터가 포함됩니다. 증분 스냅숏에는 마지막 스냅숏이 트리거된 이후 업데이트/추가된 모든 데이터가 포함됩니다. 

## <a name="snapshot-settings-in-azure-data-share"></a>Azure 데이터 공유의 스냅샷 설정
 
데이터 공급자는 데이터 공유에 대한 스냅샷 설정을 사용하도록 설정할 수 있습니다. 이 설정을 사용하면 데이터 소비자가 발생하는 증분 업데이트를 받을 수 있습니다. 데이터 공급자가 데이터 소비자가 공유된 데이터에 대한 업데이트를 수신하도록 하려는 경우 이 설정을 사용하도록 설정해야 합니다. 

데이터 공급자가 이 설정을 활성화하는 경우 되풀이 간격을 선택할 수 있습니다. 되풀이 간격은 매시간 또는 매일 될 수 있습니다. 

데이터 소비자는 이 스냅숏 일정에 옵트인하여 증분 업데이트를 받을 수 있으며, 여기에는 새 스냅숏을 처음 생성한 이후 변경된 데이터가 포함됩니다. 

## <a name="invitation"></a>초대

데이터 공급자는 여러 받는 사람을 데이터 공유에 초대할 수 있습니다. 데이터 공유에 받는 사람을 추가하여 이 작업을 수행할 수 있습니다. 데이터 공유를 만든 후에도 초대장을 추가할 수 있습니다. 

데이터 공급자는 초대가 수락되지 않은 경우 초대를 보낸 후 삭제할 수 있습니다. 데이터 공급자가 초대를 삭제했지만 아직 수락되지 않은 경우 데이터 소비자는 초대를 수락할 수 없습니다. 

초대장은 하루에 5회까지 재전송할 수 있습니다. 

## <a name="recipient"></a>받는 사람

받는 사람은 데이터 공유에 대한 초대를 받는 사람입니다. 일반적으로 데이터 공급자는 자신이 만든 데이터 공유에 받는 사람을 추가합니다. 초대받는 사람이 초대를 수락하면 데이터 소비자가 됩니다.
