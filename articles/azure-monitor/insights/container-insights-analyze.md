# <a name="monitor-your-kubernetes-cluster-performance-with-azure-monitor-for-containers"></a>컨테이너에 대한 Azure 모니터를 통해 Kubernetes 클러스터 성능 모니터링

컨테이너용 Azure 모니터를 사용하면 성능 차트 및 상태 상태를 사용하여 두 가지 관점에서 Azure Kubernetes 서비스(AKS), Azure 스택 또는 기타 환경에서 호스팅되는 Kubernetes 클러스터의 워크로드를 모니터링할 수 있습니다. 클러스터에서 직접 모니터링하거나 Azure Monitor의 구독에서 모든 클러스터를 볼 수 있습니다. 특정 AKS 클러스터를 모니터링할 때도 Azure 컨테이너 인스턴스를 볼 수 있습니다.

이 문서에서는 두 가지 관점과 Azure Monitor가 검색된 문제를 신속하게 평가, 조사 및 해결하는 데 도움이 되는 방법을 이해하는 데 도움이 됩니다.

컨테이너에 대해 Azure 모니터를 사용하도록 설정하는 방법에 대한 자세한 내용은 [컨테이너에 대한 온보드 Azure 모니터를](container-insights-onboard.md)참조하십시오.

Azure Monitor는 구독의 리소스 그룹에 배포된 Linux 및 Windows Server 2019를 실행하는 모니터링되는 모든 Kubernetes 클러스터의 상태 상태를 보여 주는 다중 클러스터 보기를 제공합니다. 솔루션에서 모니터링하지 않는 모든 환경에서 검색된 클러스터를 보여 줍니다. 클러스터 상태를 즉시 이해할 수 있으며 여기에서 노드 및 컨트롤러 성능 페이지로 드릴다운하거나 탐색하여 클러스터의 성능 차트를 볼 수 있습니다. 검색되고 모니터링되지 않은 것으로 식별된 AKS 클러스터의 경우 언제든지 모니터링을 활성화할 수 있습니다. 

Linux 클러스터와 비교하여 컨테이너에 대한 Azure Monitor를 사용하여 Windows [here](container-insights-overview.md#what-does-azure-monitor-for-containers-provide) Server 클러스터를 모니터링하는 주요 차이점은 개요 문서에서 설명합니다.

## <a name="multi-cluster-view-from-azure-monitor"></a>Azure Monitor에서 다중 클러스터 보기

배포된 모든 Kubernetes 클러스터의 상태 상태를 보려면 Azure 포털의 왼쪽 창에서 **모니터를** 선택합니다. **인사이트** 섹션에서 **컨테이너**를 선택합니다. 

![Azure Monitor 다중 클러스터 대시보드 예제](./media/container-insights-analyze/azmon-containers-multiview.png)

그리드에 표시되는 결과의 범위를 파악하여 다음과 같은 클러스터를 표시할 수 있습니다.

* **Azure** - Azure Kubernetes 서비스에서 호스팅되는 AKS 및 AKS 엔진 클러스터
* **Azure 스택(미리 보기)** - Azure 스택에서 호스팅되는 AKS-엔진 클러스터
* **비Azure(미리 보기)** - 온-프레미스에서 호스팅되는 Kubernetes 클러스터
* **모두** - Azure, Azure 스택 및 컨테이너에 대한 Azure 모니터에 온보온화되는 온-프레미스 환경에서 호스팅되는 모든 Kubernetes 클러스터보기

특정 환경에서 클러스터를 보려면 페이지의 왼쪽 상단 모서리에 있는 **환경** 알 약에서 클러스터를 선택합니다.

![환경 알약 선택기 예제](./media/container-insights-analyze/clusters-multiview-environment-pill.png)

**모니터링되는 클러스터** 탭에서 다음을 배웁니다.

- 중요 또는 비정상 상태에 있는 클러스터 수와 정상 상태또는 보고되지 않은 클러스터 수(알 수 없는 상태라고 함)입니다.
- 모든 Azure [Kubernetes 엔진(AKS 엔진)](https://github.com/Azure/aks-engine) 배포가 정상인지 여부입니다.
- 클러스터당 배포되는 노드 및 사용자 및 시스템 포드 수입니다.
- 사용 가능한 디스크 공간 및 용량 문제가 있는 경우

다음과 같은 상태가 있습니다. 

* **정상:** VM에 대한 문제가 발견되지 않으며 필요에 따라 작동합니다. 
* **중요**: 정상 운영 상태를 예상대로 복원하기 위해 해결해야 하는 하나 이상의 중요한 문제가 감지됩니다.
* **경고**: 해결해야 하는 문제가 하나 이상 발견되거나 상태 상태가 심각해질 수 있습니다.
* **알 수 없음:** 서비스가 노드 또는 창과 연결할 수 없는 경우 상태가 알 수 없음 상태로 변경됩니다.
* **찾을 수 없습니다**: 이 솔루션의 작업 영역이 포함된 작업 영역, 리소스 그룹 또는 구독이 삭제되었습니다.
* **무단**: 사용자가 작업 영역에서 데이터를 읽을 수 있는 권한이 없습니다.
* **오류**: 작업 영역에서 데이터를 읽으려고 할 때 오류가 발생했습니다.
* **잘못 구성된**: 컨테이너에 대한 Azure 모니터가 지정된 작업 영역에서 올바르게 구성되지 않았습니다.
* **데이터 없음**: 데이터가 지난 30분 동안 작업 영역에 보고되지 않았습니다.

상태 상태는 한 가지 예외를 제외하고 세 상태 *중 최악으로* 전체 클러스터 상태를 계산합니다. 세 상태 중 어느 한 개라도 알 수 없는 경우 전체 클러스터 상태는 **알 수 없음을**표시합니다.

## <a name="view-performance-directly-from-a-cluster"></a>클러스터에서 직접 성능 보기

컨테이너에 대한 Azure Monitor에 대한 액세스는 왼쪽 창에서 **인사이트** > **클러스터를** 선택하거나 다중 클러스터 보기에서 클러스터를 선택한 경우 AKS 클러스터에서 직접 사용할 수 있습니다. 클러스터에 대한 정보는 다음 네 가지 관점으로 구성됩니다.

- 클러스터
- 노드 
- Controllers 
- 컨테이너

>[!NOTE]
>이 문서의 나머지 부분에서 설명된 환경은 다중 클러스터 보기에서 선택할 때 Azure 스택 또는 기타 환경에서 호스팅되는 Kubernetes 클러스터의 성능 및 상태 상태를 볼 때도 적용됩니다. 

기본 페이지가 열리고 클러스터의 주요 실적 메트릭을 표시하는 네 개의 라인 성능 차트를 표시합니다. 

![클러스터 탭의 예제 성능 차트](./media/container-insights-analyze/containers-cluster-perfview.png)

성능 차트에는 다음 네 가지 성능 메트릭이 표시됩니다.

- **노드 CPU&nbsp;사용률**: 전체 클러스터에 대한 CPU 사용률의 집계된 관점입니다. 시간 범위에 대한 결과를 필터링하려면 차트 위의 백분위수 선택기에서 **평균,** **최소**, **50번째,** **90번째,** **95번째**또는 **최대를** 선택합니다. 필터는 개별적으로 또는 결합하여 사용할 수 있습니다. 
- **노드 메모리&nbsp;사용률**: 전체 클러스터에 대한 메모리 사용률의 집계된 관점입니다. 시간 범위에 대한 결과를 필터링하려면 차트 위의 백분위수 선택기에서 **평균,** **최소**, **50번째,** **90번째,** **95번째**또는 **최대를** 선택합니다. 필터는 개별적으로 또는 결합하여 사용할 수 있습니다. 
- **노드 수**: Kubernetes의 노드 수 및 상태입니다. 표시되는 클러스터 노드의 상태는 합계, 준비 됨 및 준비 되지 않음입니다. 개별적으로 필터링하거나 차트 위의 선택기에서 결합할 수 있습니다. 
- **활성 포드 수**: Kubernetes의 포드 수 및 상태입니다. 표시되는 창의 상태는 합계, 보류 중, 실행 중, 알 수 없음, 성공 또는 실패입니다. 개별적으로 필터링하거나 차트 위의 선택기에서 결합할 수 있습니다. 

왼쪽 및 오른쪽 화살표 키를 사용하여 차트의 각 데이터 포인트를 순환합니다. 위쪽 및 아래쪽 화살표 키를 사용하여 백분위수 선을 순환합니다. 차트 중 하나의 오른쪽 상단에 있는 핀 아이콘을 선택하여 선택한 차트를 마지막으로 본 Azure 대시보드에 고정합니다. 대시보드에서 차트의 크기를 조정하고 위치를 조정할 수 있습니다. 대시보드에서 차트를 선택하면 컨테이너에 대한 Azure Monitor로 리디렉션되고 올바른 범위와 보기가 로드됩니다.

컨테이너용 Azure Monitor는 자체 플롯 차트를 만들고, 추세를 상호 연관 및 조사하고, 대시보드에 고정할 수 있는 Azure Monitor [메트릭 탐색기도](../platform/metrics-getting-started.md)지원합니다. 메트릭 탐색기에서 메트릭 [기반 경고 규칙의](../platform/alerts-metric.md)기초로 메트릭을 시각화하기 위해 설정한 조건을 사용할 수도 있습니다.

## <a name="analyze-nodes-controllers-and-container-health"></a>노드, 컨트롤러 및 컨테이너 상태 분석

**노드,** **컨트롤러**및 **컨테이너** 탭으로 전환하면 속성 창이 페이지 오른쪽에 자동으로 표시됩니다. Kubernetes 객체를 구성하기 위해 정의한 레이블을 포함하여 선택한 항목의 속성을 표시합니다. Linux 노드를 선택하면 **로컬 디스크 용량** 섹션에는 노드에 표시되는 각 디스크에 대해 사용 가능한 디스크 공간과 사용되는 백분율도 표시됩니다. 창에서 **>>** 링크를 선택하여 창을 보거나 숨깁니다.

계층 구조에서 개체를 펼치면 선택한 개체에 따라 속성 창이 업데이트됩니다. 창에서 창 상단의 **라이브 데이터 보기(미리 보기)** 링크를 선택하여 Kubernetes 컨테이너 로그(stdout/stderror), 이벤트 및 포드 메트릭을 볼 수도 있습니다. 이 데이터를 보기 위해 액세스 권한을 부여하고 제어하는 데 필요한 구성에 대한 자세한 내용은 [라이브 데이터 설정(미리 보기)을](container-insights-livedata-setup.md)참조하십시오. 클러스터 리소스를 검토하는 동안 컨테이너에서 이 데이터를 실시간으로 볼 수 있습니다. 이 기능에 대한 자세한 내용은 [Kubernetes 로그, 이벤트 및 포드 메트릭을 실시간으로 보는 방법을 참조하세요.](container-insights-livedata-overview.md) 미리 정의된 로그 검색을 기반으로 작업 영역에 저장된 Kubernetes 로그 데이터를 보려면 분석 드롭다운 목록의 **보기에서** **컨테이너 로그 보기를** 선택합니다. 이 항목에 대한 자세한 내용은 [데이터를 분석하기 위한 검색 로그를](container-insights-log-search.md#search-logs-to-analyze-data)참조하십시오.

페이지 상단의 **+ 필터 추가** 옵션을 사용하여 **서비스,** **노드,** **네임스페이스**또는 **노드 풀별로**보기에 대한 결과를 필터링합니다. 필터 범위를 선택한 후 **선택 값** 필드에 표시된 값 중 하나를 선택합니다. 필터가 구성되면 AKS 클러스터의 모든 큐브 뷰를 보면서 전역적으로 적용됩니다. 수식은 등호만 지원됩니다. 첫 번째 필터에 필터를 추가하여 결과를 더 좁힐 수 있습니다. 예를 들어 **노드별로**필터를 지정하는 경우 두 번째 필터에 대한 **서비스** 또는 **네임스페이스만** 선택할 수 있습니다.

한 탭에서 필터를 지정하는 것은 다른 탭을 선택할 때 계속 적용됩니다. 지정된 필터 옆에 있는 **x** 기호를 선택하면 삭제됩니다. 

**노드** 탭으로 전환하면 행 계층 구조는 클러스터의 노드로 시작하는 Kubernetes 개체 모델을 따릅니다. 노드를 확장하여 노드에서 실행되는 하나 이상의 포드를 봅니다. 두 개 이상의 컨테이너가 포드에 그룹화되면 계층구조의 마지막 행으로 표시됩니다. 또한 호스트에 프로세서 또는 메모리 압력이 있는 경우 호스트에서 실행 중인 비Pod 관련 워크로드 수를 볼 수도 있습니다.

![성능 뷰에서 Kubernetes 노드 계층 구조의 예](./media/container-insights-analyze/containers-nodes-view.png)

Windows Server 2019 OS를 실행하는 Windows 서버 컨테이너는 목록에 있는 모든 Linux 기반 노드 후에 표시됩니다. Windows Server 노드를 확장하면 노드에서 실행되는 하나 이상의 포드 및 컨테이너를 볼 수 있습니다. 노드를 선택하면 속성 창에 버전 정보가 표시됩니다. Windows Server 노드에 에이전트가 설치되어 있지 않기 때문에 에이전트 정보는 제외됩니다. 

![나열된 Windows 서버 노드가 있는 노드 계층 구조 예제](./media/container-insights-analyze/nodes-view-windows.png) 

Azure 컨테이너 인스턴스 Linux OS를 실행하는 가상 노드는 목록의 마지막 AKS 클러스터 노드 다음으로 표시됩니다. 컨테이너 인스턴스 가상 노드를 확장하면 노드에서 실행되는 하나 이상의 컨테이너 인스턴스 포드 및 컨테이너를 볼 수 있습니다. 메트릭은 노드에 대해 수집및 보고되지 않으며 포드에 대해서만 보고됩니다.

![Container Instances가 나열된 노드 계층 구조 예제](./media/container-insights-analyze/nodes-view-aci.png)

확장된 노드에서 노드에서 실행되는 포드 또는 컨테이너에서 컨트롤러로 드릴다운하여 해당 컨트롤러에 대해 필터링된 성능 데이터를 볼 수 있습니다. 특정 노드에 대한 **컨트롤러** 열 아래의 값을 선택합니다.
 
![성능 보기에서 보여 주는 노드에서 컨트롤러로의 드릴다운 예제](./media/container-insights-analyze/drill-down-node-controller.png)

페이지 상단의 컨트롤러 또는 컨테이너를 선택하여 해당 개체의 상태 및 리소스 사용률을 검토합니다. 메모리 사용률을 검토하려면 **메트릭** 드롭다운 목록에서 **메모리 RSS** 또는 **메모리 작업 집합을**선택합니다. **메모리 RSS**는 Kubernetes 버전 1.8 이상에서만 지원됩니다. 그렇지 않으면 **Min에&nbsp; ** 대한 값을 *&nbsp;NaN으로*볼 수 있으며, 이는 정의되지 않거나 표현할 수 없는 값을 나타내는 숫자 데이터 형식 값입니다.

![컨테이너 노드 성능 보기](./media/container-insights-analyze/containers-node-metric-dropdown.png)

**메모리 작업 집합은** 상주 메모리와 가상 메모리(캐시)를 모두 포함하며 응용 프로그램에서 사용하는 내용의 총값입니다. **메모리 RSS는** 주 메모리만 표시합니다(즉, 상주 메모리에 불과합니다). 이 메트릭은 사용 가능한 메모리의 실제 용량을 보여 주며, 상주 메모리와 가상 메모리의 차이점은 무엇입니까?

- 상주 메모리 또는 주 메모리는 클러스터의 노드에서 사용할 수 있는 실제 컴퓨터 메모리 양입니다.

- 가상 메모리는 운영 체제에서 메모리 압력을 받을 때 메모리에서 디스크로 데이터를 교환한 다음 필요할 때 메모리로 다시 가져오는 데 사용되는 하드 디스크 공간(캐시)을 예약합니다.

기본적으로 성능 데이터는 지난 6시간을 기반으로 하지만 왼쪽 상단에 있는 **TimeRange** 옵션을 사용하여 창을 변경할 수 있습니다. 또한 백분위수 선택기에서 **최소,** **평균,** **50,** **90,** **95**및 **최대를** 선택하여 시간 범위 내에서 결과를 필터링할 수 있습니다. 

![데이터 필터링에 대한 백분위 선택 영역](./media/container-insights-analyze/containers-metric-percentile-filter.png)

**추세** 열 아래의 막대 그래프 위로 마우스를 가져가면 각 막대는 15분의 샘플 기간 내에 선택한 메트릭에 따라 CPU 또는 메모리 사용량을 표시합니다. 키보드를 통해 추세 차트를 선택한 후 Alt+Page 위쪽 키 또는 Alt+Page 아래로 키를 사용하여 각 막대를 개별적으로 순환합니다. 막대 위로 마우스를 가져가면 와 같은 세부 정보를 얻을 수 있습니다.

![트렌드 가로수 차트 호버오버 예제](./media/container-insights-analyze/containers-metric-trend-bar-01.png) 

다음 예제에서 목록의 첫 번째 노드에 대 한 *aks-nodepool1-* **컨테이너에** 대 한 값은 9입니다. 이 값은 배포된 총 컨테이너 수의 롤업입니다.

* 자체 관리 또는 관리되는 Kubernetes 비컨테이너 프로세스

* kubelet  

* 노드에서 실행되는 시스템 프로세스

* 노드 하드웨어 또는 VM에서 실행되는 다른 비 Kubernetes 워크로드

## <a name="workbooks"></a>통합 문서

통합 문서는 텍스트, [로그 쿼리,](../log-query/query-language.md)메트릭 및 매개 [변수를](../platform/data-platform-metrics.md)풍부한 대화형 보고서로 결합합니다. 통합 문서는 동일한 Azure 리소스에 대한 액세스 권한이 있는 다른 팀 멤버에 의해 편집될 수 있습니다.

컨테이너용 Azure 모니터에는 시작하는 데 필요한 네 가지 통합 문서가 포함되어 있습니다.

- **디스크 용량**: 다음 관점에서 컨테이너 내의 노드에 표시되는 각 디스크에 대한 대화형 디스크 사용 차트를 제공합니다.

    - 모든 디스크에 대한 디스크 백분율 사용량입니다.
    - 모든 디스크에 대한 디스크 공간이 확보됩니다.
    - 각 노드의 디스크, 사용된 공간의 백분율, 사용된 공간의 백분율 추세, 사용 디스크 공간(GiB) 및 사용 사용 디스크 공간(GiB) 추세를 표시하는 그리드입니다. 테이블에서 행을 선택하면 사용된 공간과 사용 가능한 디스크 공간(GiB)의 백분율이 행 아래에 표시됩니다. 

- **디스크 IO**: 다음 관점에서 컨테이너 내의 노드에 표시되는 각 디스크에 대한 대화형 디스크 사용률 차트를 제공합니다.

    - 디스크 I/O는 모든 디스크에 대해 읽기 바이트/초, 바이트/초 쓰기, 바이트/초 읽기 및 쓰기 추세로 요약됩니다.
    - 8개의 성능 차트는 디스크 I/O 병목 현상을 측정하고 식별하는 데 도움이 되는 주요 성과 지표를 보여 줍니다.

- **Kubelet**: 키 노드 운영 통계를 표시하는 두 개의 그리드를 포함합니다.

    - 노드 그리드별 개요는 각 노드에 대한 총 작업, 총 오류 및 성공적인 작업을 백분율로 요약합니다.
    - 작업 유형별 개요는 각 작업에 대해 총 작업, 총 오류 및 성공한 작업을 백분율 및 추세로 요약합니다.

- **네트워크**: 각 노드의 네트워크 어댑터에 대한 대화형 네트워크 사용률 차트를 표시하고 그리드는 네트워크 어댑터의 성능을 측정하는 데 도움이 되는 핵심 성능 지표를 제공합니다.

통합 문서 보기 드롭다운 목록에서 각 통합 문서를 선택하여 이러한 **통합 문서에** 액세스합니다.

![통합 문서 드롭다운 목록 보기](./media/container-insights-analyze/view-workbooks-dropdown-list.png)
