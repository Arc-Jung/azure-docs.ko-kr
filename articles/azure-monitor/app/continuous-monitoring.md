# <a name="add-continuous-monitoring-to-your-release-pipeline"></a>릴리스 파이프라인에 연속 모니터링 추가

Azure 파이프라인은 Azure 응용 프로그램 인사이트와 통합되어 소프트웨어 개발 수명 주기 전반에 걸쳐 DevOps 릴리스 파이프라인을 지속적으로 모니터링할 수 있습니다. 

지속적인 모니터링을 통해 릴리스 파이프라인은 응용 프로그램 인사이트 및 기타 Azure 리소스의 모니터링 데이터를 통합할 수 있습니다. 릴리스 파이프라인이 응용 프로그램 인사이트 경고를 감지하면 파이프라인은 경고가 해결될 때까지 배포를 게이트하거나 롤백할 수 있습니다. 모든 검사가 통과하면 수동 개입 없이 테스트에서 프로덕션까지 자동으로 배포를 진행할 수 있습니다. 

## <a name="configure-continuous-monitoring"></a>연속 모니터링 구성

1. [Azure DevOps에서](https://dev.azure.com)조직 및 프로젝트를 선택합니다.
   
1. 프로젝트 페이지의 왼쪽 메뉴에서 **파이프라인 릴리스를 선택합니다.** > **Releases** 
   
1. **새** 화살표 옆에 있는 화살표를 드롭다운하고 **새 릴리스 파이프라인을**선택합니다. 또는 아직 파이프라인이 없는 경우 나타나는 페이지에서 **새 파이프라인을** 선택합니다.
   
1. 템플릿 창 **선택에서** **지속적인 모니터링을 사용하여 Azure 앱 서비스 배포를**검색하고 선택한 다음 **적용을**선택합니다. 

   ![새 Azure 파이프라인 릴리스 파이프라인](media/continuous-monitoring/001.png)

1. **1단계** 상자에서 스테이지 작업 보기에 대한 하이퍼링크를 **선택합니다.**

   ![스테이지 작업 보기](media/continuous-monitoring/002.png)

1. **1단계** 구성 창에서 다음 필드를 완료합니다. 

    | 매개 변수        | 값 |
   | ------------- |:-----|
   | **스테이지 이름**      | 스테이지 이름을 제공하거나 스테이지 **1에**그대로 둡니다. |
   | **Azure 구독** | 드롭다운하고 사용하려는 연결된 Azure 구독을 선택합니다.|
   | **앱 유형** | 드롭다운하고 앱 유형을 선택합니다. |
   | **App Service 이름** | Azure 앱 서비스의 이름을 입력합니다. |
   | **응용 프로그램 인사이트를 위한 리소스 그룹 이름**    | 드롭다운하고 사용할 리소스 그룹을 선택합니다. |
   | **Application Insights 리소스 이름** | 드롭다운하고 선택한 리소스 그룹에 대한 응용 프로그램 인사이트 리소스를 선택합니다.

1. 기본 경고 규칙 설정으로 파이프라인을 저장하려면 Azure DevOps 창에서 오른쪽 상단에 **저장을** 선택합니다. 설명 주석을 입력한 다음 **확인을**선택합니다.

## <a name="modify-alert-rules"></a>경고 규칙 수정

기본 제공, 지속적인 모니터링 템플릿을 **사용 하 고 Azure 앱 서비스 배포에는** **가용성,** **실패한 요청,** **서버 응답 시간**및 서버 예외의 네 가지 경고 규칙이 **있습니다.** 규칙을 더 추가하거나 서비스 수준 요구 사항에 맞게 규칙 설정을 변경할 수 있습니다. 

경고 규칙 설정을 수정하려면 다음을 수행하십시오.

1. 릴리스 파이프라인 페이지의 왼쪽 창에서 **응용 프로그램 인사이트 경고 구성을 선택합니다.**

1. Azure **모니터 경고** 창에서 **경고 규칙** **옆의** 타원을 선택합니다.
   
1. 경고 **규칙** 대화 상자에서 **가용성**과 같은 경고 규칙 옆의 드롭다운 기호를 선택합니다. 
   
1. 요구 사항에 맞게 **임계값** 및 기타 설정을 수정합니다.
   
   ![경고 수정](media/continuous-monitoring/003.png)
   
1. **확인을**선택한 다음 Azure DevOps 창에서 오른쪽 상단에 **저장을** 선택합니다. 설명 주석을 입력한 다음 **확인을**선택합니다.

## <a name="add-deployment-conditions"></a>배포 조건 추가

릴리스 파이프라인에 배포 게이트를 추가하면 설정한 임계값을 초과하는 경고가 원치 않는 릴리스 승격을 방지합니다. 경고를 해결하면 배포가 자동으로 진행될 수 있습니다.

배포 게이트를 추가하려면 다음을 수행하십시오.

1. 기본 파이프라인 페이지에서 **단계**에서 연속 모니터링 게이트가 필요한 단계에 따라 **배포 전 조건** 또는 배포 후 **조건** 기호를 선택합니다.
   
   ![배포 전 조건](media/continuous-monitoring/004.png)
   
1. 배포 **전 조건** 구성 창에서 **게이트를** **사용 설정으로**설정합니다.
   
1. 배포 **게이트**옆에 **는 추가를**선택합니다.
   
1. 드롭다운 메뉴에서 **Azure 모니터 쿼리 경고를 선택합니다.** 이 옵션을 사용하면 Azure 모니터 및 응용 프로그램 인사이트 경고에 모두 액세스할 수 있습니다.
   
   ![Azure 모니터 경고 쿼리](media/continuous-monitoring/005.png)
   
1. **평가 옵션에서** **게이트 재평가 시간 및 게이트가** 실패한 시간 시간 사이의 시간 과 같은 설정에 대해 원하는 값을 **입력합니다.** 

## <a name="view-release-logs"></a>릴리스 로그 보기

릴리스 로그에서 배포 게이트 동작 및 기타 릴리스 단계를 확인할 수 있습니다. 로그를 열려면 다음을 수행합니다.

1. 파이프라인 페이지의 왼쪽 메뉴에서 **릴리스를** 선택합니다. 
   
1. 모든 릴리스를 선택합니다. 
   
1. **단계에서**릴리스 요약을 보려면 모든 단계를 선택합니다. 
   
1. 로그를 보려면 릴리스 요약에서 **로그 보기를** 선택하거나, 모든 단계에서 **성공** 또는 **실패** 한 하이퍼링크를 선택하거나, 모든 단계에서 마우스를 가져 가서 **로그를 선택합니다.** 
   
   ![릴리스 로그 보기](media/continuous-monitoring/006.png)
