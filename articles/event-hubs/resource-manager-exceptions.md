# <a name="azure-event-hubs---resource-manager-exceptions"></a>Azure 이벤트 허브 - 리소스 관리자 예외
이 문서에서는 템플릿 또는 직접 호출을 통해 Azure 리소스 관리자를 사용하여 Azure 이벤트 허브와 상호 작용할 때 생성된 예외를 나열합니다.

> [!IMPORTANT]
> 이 문서는 자주 업데이트됩니다. 업데이트를 다시 확인하시기 바랍니다.

다음 섹션에서는 Azure 리소스 관리자를 통해 표시되는 다양한 예외/오류를 제공합니다.

## <a name="error-code-conflict"></a>오류 코드: 충돌

| 오류 코드 | 오류 하위 코드 | 오류 메시지 | 설명 | 권장 |
| ---------- | ------------- | ------------- | ----------- | -------------- |
| 충돌 | 40300 | EventHub 형식의 최대 리소스 수에 도달했거나 초과했습니다. 실제: #, 최대 허용: # | 네임스페이스가 포함할 수 있는 이벤트 허브 수에 대한 [할당량에](event-hubs-quotas.md) 도달했습니다. | 네임스페이스에서 사용되지 않거나 불필요한 이벤트 허브를 삭제하거나 [전용 클러스터로](event-hubs-dedicated-overview.md)업그레이드하는 것이 좋습니다. |
| 충돌 | none | DR(재해 복구) 구성은 복제가 진행 중이므로 삭제할 수 없습니다. DR 구성을 삭제하기 전에 Failover 또는 break 페어링을 시도합니다. | [GeoDR 복제가](event-hubs-geo-dr.md) 진행 중이므로 현재 구성을 삭제할 수 없습니다. | 구성 의 삭제를 차단 해제하려면 복제가 완료될 때까지 기다리거나 장애 조치(failover)를 트리거하거나 GeoDR 페어링을 중단합니다. |
| 충돌 | none | 네임스페이스 업데이트가 백 엔드에서 충돌로 실패했습니다. | 이 네임스페이스에서 다른 작업이 현재 수행중입니다. | 현재 작업이 완료될 때까지 기다린 다음 다시 시도합니다. |

## <a name="error-code-429"></a>오류 코드: 429

| 오류 코드 | 오류 하위 코드 | 오류 메시지 | 설명 | 권장 |
| ---------- | ------------- | ------------- | ----------- | -------------- |
| 429 | none | 전환시 네임스페이스 프로비저닝 | 이 네임스페이스에서 다른 작업이 현재 수행중입니다. | 현재 작업이 완료될 때까지 기다린 다음 다시 시도합니다. |
| 429 | none | 재해 복구 작업이 진행 중입니다. | [GeoDR](event-hubs-geo-dr.md) 작업은 현재 이 네임스페이스 또는 페어링에서 수행중입니다. | 현재 GeoDR 작업이 완료될 때까지 기다린 다음 다시 시도합니다. |

## <a name="error-code-badrequest"></a>오류 코드: BadRequest

| 오류 코드 | 오류 하위 코드 | 오류 메시지 | 설명 | 권장 |
| ---------- | ------------- | ------------- | ----------- | -------------- |
| BadRequest | 40,000 | 이벤트 허브에 대해 파티션카운트를 변경할 수 없습니다. | Azure Event Hubs의 기본 또는 표준 계층은 파티션 변경을 지원하지 않습니다. | 기본 또는 표준 계층 네임스페이스에 원하는 파티션 수를 사용하여 새 이벤트 허브를 만듭니다. 전용 클러스터에 대해 파티션 확장이 [지원됩니다.](event-hubs-dedicated-overview.md) |
| BadRequest | 40,000 | 메시지 보존인Days의 값 '#'은 기본 계층에 대해 유효하지 않습니다. 값은 '1'일(들)을 초과할 수 없습니다. | 기본 계층 이벤트 허브 네임스페이스는 최대 1일의 메시지 보존만 지원합니다. | 하루 이상의 메시지 보존이 원하는 경우 [표준 이벤트 허브 네임스페이스를 만듭니다.](event-hubs-create.md) | 
| BadRequest | none | 지정된 이름을 사용할 수 없습니다. | 네임스페이스 이름은 고유해야 하며 지정된 이름이 이미 지정되어 있어야 합니다. | 지정된 이름을 가진 기존 네임스페이스의 소유자인 경우 삭제하면 데이터 손실이 발생할 수 있습니다. 그런 다음 동일한 이름으로 다시 시도하십시오. 네임스페이스를 삭제하는 것이 안전하지 않거나 소유자가 아닌 경우 다른 네임스페이스 이름을 선택합니다. |
| BadRequest | none | 지정된 구독이 네임스페이스 할당량에 도달했습니다. | 구독이 보유할 수 있는 네임스페이스 수에 대한 [할당량에](event-hubs-quotas.md) 도달했습니다. | 이 구독에서 사용되지 않는 네임스페이스를 삭제하거나, 다른 구독을 만들거나, [전용 클러스터로](event-hubs-dedicated-overview.md)업그레이드하는 것이 좋습니다. |
| BadRequest | none | 보조 네임스페이스를 업데이트할 수 없습니다. | [GeoDR 페어링의](event-hubs-geo-dr.md)보조 네임스페이스이기 때문에 네임스페이스를 업데이트할 수 없습니다. | 적절한 경우 이 페어링에서 기본 네임스페이스를 대신 변경합니다. 그렇지 않으면 GeoDR 페어링을 끊어 변경합니다. |
| BadRequest | none | 기본 SKU에서 자동 팽창을 설정할 수 없습니다. | 기본 계층 이벤트 허브 네임스페이스에서는 자동 팽창을 사용할 수 없습니다. | 네임스페이스에서 [자동 팽창을 사용하려면](event-hubs-auto-inflate.md) 표준 계층인지 확인합니다. |
| BadRequest | none | 네임스페이스를 만들 수 있는 용량이 부족합니다. 이벤트 허브 관리자에게 문의하십시오. | 선택한 영역은 용량에 있으며 더 많은 네임스페이스를 만들 수 없습니다. | 네임스페이스를 수용할 다른 지역을 선택합니다. |
| BadRequest | none | 네임스페이스 '네임스페이스 네임네임'이 '기본' 계층을 사용하기 때문에 엔터티 유형 'ConsumerGroup'에서 작업을 수행할 수 없습니다.  | 기본 계층 이벤트 허브 네임스페이스에는 한 소비자 그룹(기본값)의 [할당량-할당량.md#이벤트 허브-기본 및 표준 및 표준 ---할당량 및 제한)이 있습니다. 더 많은 소비자 그룹을 만드는 것은 지원되지 않습니다. | 기본 소비자 그룹($Default)을 계속 사용하거나 필요한 경우 대신 표준 계층 이벤트 허브 네임스페이스를 사용하는 것이 좋습니다. | 
| BadRequest | none | 네임스페이스 '네임스페이스 이름'이 없습니다. | 제공된 네임스페이스를 찾을 수 없습니다. | 네임스페이스 이름이 올바른지 다시 한 번 확인하고 구독에서 찾을 수 있습니다. 그렇지 않은 경우 [이벤트 허브 네임스페이스를 만듭니다.](event-hubs-create.md) | 
| BadRequest | none | 리소스의 위치 속성이 포함된 네임스페이스와 일치하지 않습니다. | 네임스페이스의 지역과 일치하지 않아 특정 지역에서 이벤트 허브를 만드는 데 실패했습니다. | 네임스페이스와 동일한 지역에서 이벤트 허브를 만들어 보십시오. | 

## <a name="error-code-internal-server-error"></a>오류 코드: 내부 서버 오류

| 오류 코드 | 오류 하위 코드 | 오류 메시지 | 설명 | 권장 |
| ---------- | ------------- | ------------- | ----------- | -------------- |
| 내부 서버 오류 | none | 내부 서버 오류. | 이벤트 허브 서비스에 내부 오류가 발생했습니다. | 실패한 작업을 다시 시도합니다. 작업이 계속 실패하면 지원팀에 문의하십시오. |
