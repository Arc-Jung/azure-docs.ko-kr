# <a name="azure-government-compliance"></a>Azure Government 규정 준수
## <a name="blueprint-customer-responsibilities-matrix-streamlining-ato-processes"></a>Blueprint 고객 책임 매트릭스: ATO 프로세스 간소화
Azure Blueprint 프로그램은 고객이 문서에 액세스하여 에이전시 또는 시스템별 ATO(authorization to operate) 프로세스를 지원할 수 있습니다. 이 문서에서는 Azure Blueprint CRM(고객 책임 매트릭스) 및 보안 담당자가 이 문서를 활용하는 방법을 자세히 살펴봅니다.

Azure Blueprint CRM은 다용성을 염두에 두고 설계되었습니다. Azure에서 고객 구축이 보안 제어를 구현해야 하는 제어 요구 사항만 정의하는 스프레드시트 형식입니다.

> [!NOTE]
> Azure Blueprint에서 "고객"이란 Azure IaaS/PaaS 환경 내에 직접 구축하는 조직을 말합니다. Azure 고객으로는 정부 또는 정부 기관을 대신하여 Azure에 직접 구축하는 타사 ISV가 포함될 수 있습니다.
> 
> 

CRM은 고객 책임이 포함된 명시적 <a href="http://nvlpubs.nist.gov/nistpubs/SpecialPublications/NIST.SP.800-53r4.pdf"> NIST SP 800-53 수정 버전 4 </a> 컨트롤을 제공합니다. 여기에는 고객과 Azure가 책임을 공유하는 제어뿐 아니라 온전히 고객의 책임인 제어까지 포함됩니다. 제어는 최대한 명시적이어야 하는 제어 하위 요구 사항으로 분류됩니다.

예를 들어 AC-1은 ATO를 검색하는 시스템에 대한 액세스 제어 정책 및 절차를 문서화해야 합니다. 이 제어의 경우 Azure에는 Azure 인프라 및 플랫폼을 관리하는 데 사용되는 액세스 제어 메커니즘과 관련된 내부 정책과 절차가 있습니다. 또한 고객은 Azure에 구축한 고객 시스템 내부에서 사용할 고유의 액세스 제어 정책과 절차를 만들어야 합니다. CRM 문서는 특정 콘텐츠를 포함하는 정책과 절차가 필요한 AC-1a 파트뿐 아니라 고객이 적어도 연 1회 이상 이러한 문서를 검토하고 업데이트해야 하는 AC-1b까지 제어합니다. 

CRM 및 Azure Blueprint 시스템 보안 계획 템플릿에 액세스하려면 AzureBlueprint@microsoft.com으로 전자 메일을 보내 주세요.

CRM에 대해 많은 의견을 보내 주시기 바랍니다. 고객 ATO 프로세스를 최대한 효과적으로 간소화하려면 문서 반복 작업의 중심이 되는 고객의 의견이 매우 중요 합니다.
