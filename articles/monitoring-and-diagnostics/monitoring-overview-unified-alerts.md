# <a name="unified-alerts-in-azure-monitor"></a>Azure Monitor의 통합 경고

## <a name="overview"></a>개요

> [!NOTE]
>  여러 구독의 경고를 관리하고 경고 상태 및 스마트 그룹을 도입한 새로운 통합 경고 환경은 현재 공개 미리 보기로 제공되고 있습니다. 이 향상된 환경과 이러한 환경을 활성화하는 프로세스에 대한 설명은 이 문서의 마지막 섹션을 참조하세요.


이 문서에서는 Azure Monitor의 통합 경고 환경을 설명합니다. [이전 경고 환경](monitoring-overview-alerts.md)은 Azure Monitor 메뉴의 **경고(클래식)** 옵션에서 보기 가능합니다. 

## <a name="features-of-the-unified-alert-experience"></a>통합 경고 환경의 기능

통합 경고 환경은 클래식 환경에 비해 다음과 같은 장점이 있습니다.

-   **향상된 알림 시스템**: [작업 그룹](https://docs.microsoft.com/azure/monitoring-and-diagnostics/monitoring-action-groups)은 여러 경고에서 재사용할 수 있는 알림 및 작업의 명명된 그룹입니다. 
- **통합 작성 환경**: Azure Monitor, Log Analytics, Application Insights 전체의 메트릭, 로그 및 활동 로그에 대한 경고와 경고 규칙을 한곳에서 관리할 수 있습니다. 
- **Azure Portal에서 발생한 Log Analytics 경고 보기**: 이제 Azure Portal에서 다른 소스의 경고와 함께 Log Analytics 경고를 볼 수 있습니다. 이전에는 다른 소스의 경보가 별도의 포털에 있었습니다.
- **발생한 경고 및 경고 규칙의 분리**: 이제 경고 규칙이 경고와 구분됩니다. 경고 규칙은 경고를 트리거하는 조건의 정의입니다. 경고는 경고 규칙 실행의 인스턴스입니다.
- **향상된 워크플로**: 통합 경고 작성 환경이 경고 규칙을 구성하는 프로세스를 단계별로 안내합니다.
 
메트릭 경고는 클래식 메트릭 경고에 비해 다음과 같이 개선되었습니다.

-   **개선된 대기 시간**: 메트릭 경고를 1분마다 실행할 수 있습니다. 클래식 메트릭 경고는 항상 5분 주기로 실행됩니다. 로그를 수집하는 데 걸리는 시간으로 인해 여전히 로그 경고는 1분 이상 지연됩니다. 
-   **다차원 메트릭 지원**: 차원 메트릭에 대해 경고할 수 있으므로, 특정 메트릭 인스턴스를 모니터링할 수 있습니다.
-   **메트릭 조건을 보다 세밀하게 제어**: 메트릭의 최대, 최소, 평균 및 총 값을 모니터링하는 보다 풍부한 경고 규칙을 정의할 수 있습니다.
-   **여러 메트릭의 결합 모니터링**: 단일 규칙을 사용하여 최대 2개의 메트릭을 모니터링할 수 있습니다. 두 메트릭이 지정된 기간 동안 해당 임계값을 위반하면 경고가 트리거됩니다.
-   **로그의 메트릭**(제한된 공개 미리 보기): 이제 Log Analytics로 이동하는 일부 로그 데이터를 추출하고 Azure Monitor 메트릭으로 변환한 다음, 다른 메트릭과 마찬가지로 경고할 수 있습니다. 


## <a name="alert-rules"></a>경고 규칙
통합 경고 환경은 여러 경고 형식의 작성 환경을 통합하는 동시에 다음 개념을 사용하여 경고 규칙을 경고와 구별합니다.

| 항목 | 정의 |
|:---|:---|
| 경고 규칙 | 경고를 만드는 조건 정의. 경고 규칙은 _대상 리소스_, _신호_, _조건_ 및 _논리_로 구성됩니다. 경고 규칙은 _사용_ 상태일 때만 활성화됩니다.
| 대상 리소스 | 경고에 사용할 수 있는 특정 리소스 및 신호를 정의합니다. 대상은 Azure 리소스일 수 있습니다.<br><br>예: 가상 머신, 저장소 계정, 가상 머신 확장 집합, Log Analytics 작업 영역, Application Insights 리소스 |
| 신호 | 대상 리소스에서 내보낸 데이터의 원본입니다. 지원되는 신호 형식은 *메트릭*, *활동 로그*, *Application Insights* 및 *로그*입니다. |
| 조건 | 대상 리소스에 적용되는 _신호_와 _논리_의 조합입니다.<br><br>예: CPU의 백분율 > 70%, 서버 응답 시간 > 4ms, 로그 쿼리의 결과 개수 > 100 등 |
| 논리 | 신호가 예상 범위/값 내에 있는지 확인하는 사용자 정의 논리입니다. |
| 조치 | 경고가 발생할 때 수행할 작업입니다. 경고가 발생할 때 여러 작업을 수행할 수 있습니다. 이러한 경고는 작업 그룹을 지원합니다.<br><br>예: 메일 주소로 메일 보내기, 웹후크 URL 호출. |
| 조건 모니터링 | 메트릭 경고를 생성한 조건이 해결되었는지 여부를 나타냅니다. 메트릭 경고 규칙은 주기적으로 특정 메트릭을 샘플링합니다. 경고 규칙의 조건이 충족되면 “발생” 조건이 포함된 새 경고가 만들어집니다.  메트릭을 다시 샘플링할 때 여전히 조건이 충족되면 아무 작업도 수행되지 않습니다.  조건이 충족되지 않으면 경고 조건이 “해결됨”으로 변경됩니다. 다음에 조건이 충족되면 “발생” 조건이 포함된 다른 경고가 만들어집니다. |


## <a name="alert-pages"></a>경고 페이지
통합 경고는 한 위치에서 모든 Azure 경고를 보고 관리할 수 있습니다. 다음 섹션에서는 통합 환경의 각 페이지가 수행하는 기능을 설명합니다.

### <a name="alerts-overview-page"></a>경고 개요 페이지
**경고** 개요 페이지에는 발생한 모든 경고의 집계 요약 정보 및 사용하도록 설정된 총 경고 규칙 수가 표시됩니다. 구독 또는 필터 매개 변수를 변경하면 집계 및 발생한 경고 목록을 업데이트합니다.

 ![alerts-overview](./media/monitoring-overview-unified-alerts/alerts-preview-overview2.png) 

### <a name="alert-rules-management"></a>경고 규칙 관리
**규칙**은 전체 Azure 구독의 모든 경고 규칙을 관리하는 단일 페이지입니다. 모든 경고 규칙을 나열하며 대상 리소스, 리소스 그룹, 규칙 이름 또는 상태에 따라 정렬할 수 있습니다. 또한 이 페이지에서 경고 규칙을 편집하거나 사용/사용 안 함으로 설정할 수 있습니다.

 ![alerts-rules](./media/monitoring-overview-unified-alerts/alerts-preview-rules.png)


## <a name="create-an-alert-rule"></a>경고 규칙 만들기
모니터링 서비스 또는 신호 형식에 관계없이 일관된 방식으로 경고를 작성할 수 있습니다. 발생한 모든 경고 및 관련 세부 정보는 단일 페이지에 제공됩니다.
 
다음과 같은 세 단계를 통해 새 경고 규칙을 만들 수 있습니다.
1. 경고의 _대상_을 선택합니다.
1. 대상에 사용 가능한 신호 중에서 _신호_를 선택합니다.
1. 신호의 데이터에 적용할 _논리_를 지정합니다.
 
이 간소화된 작성 프로세스에서는 Azure 리소스를 선택하기 전에 지원되는 모니터링 원본 또는 신호를 더 이상 알 필요가 없습니다. 사용 가능한 신호 목록은 선택한 대상 리소스를 기준으로 자동 필터링되며, 경고 규칙의 논리를 정의하는 과정을 안내합니다.

경고 규칙을 만드는 방법에 대한 자세한 내용은 [Azure Monitor를 사용하여 경고 만들기, 보기 및 관리](monitor-alerts-unified-usage.md)를 참조하세요.

경고는 여러 Azure 모니터링 서비스 전체에서 제공됩니다. 각 서비스를 사용하는 방법 및 시기에 대한 자세한 내용은 [Azure 응용 프로그램 및 리소스 모니터링](./monitoring-overview.md)을 참조하세요. 다음 표에서는 Azure 전체에서 사용할 수 있는 경고 규칙 유형 목록을 제공합니다. 또한 통합 경고 환경에서 현재 지원되는 항목을 나열합니다.

| **모니터 원본** | **신호 유형**  | **설명** | 
|-------------|----------------|-------------|
| Azure Monitor | 메트릭  | [근 실시간 메트릭 경고](monitoring-near-real-time-metric-alerts.md)라고도 하는 메트릭 경고는 1분 주기의 메트릭 조건 평가를 지원하고 다중 메트릭 및 다차원 메트릭 규칙을 허용합니다. 지원되는 리소스 종류 목록은 [Azure Portal의 Azure 서비스에 대한 최신 메트릭 경고](monitoring-near-real-time-metric-alerts.md#metrics-and-dimensions-supported)에 제공됩니다.<br>[클래식 메트릭 경고](monitoring-overview-alerts.md)는 새로운 경고 환경에서 지원되지 않습니다. Azure Portal의 경고(클래식) 아래에서 찾을 수 있습니다. 클래식 경고는 최신 경고로 아직 이동되지 않은 일부 메트릭 유형도 지원합니다. 전체 목록은 [지원되는 메트릭](https://docs.microsoft.com/azure/monitoring-and-diagnostics/monitoring-supported-metrics)을 참조하세요. |
| Log Analytics | 로그  | 로그 검색 쿼리가 특정 조건을 충족하면 알림을 받거나 자동화된 작업을 실행합니다. Log Analytics의 경고가 [새 환경으로 복사](monitoring-alerts-extend.md)됩니다. *메트릭으로 Log Analytics 로그*의 [미리 보기](monitoring-alerts-extend-tool.md)가 제공됩니다. 미리 보기를 통해 특정 유형의 로그를 사용하고 메트릭으로 변환할 수 있으며, 새로운 경고 환경을 사용하여 메트릭에 대해 경고할 수 있습니다. 미리 보기는 기본 Azure Monitor 메트릭과 함께 가져오려는 비 Azure 로그가 있는 경우에 유용합니다. |
| 활동 로그 | 활동 로그 | 선택한 대상에서 만든 모든 만들기, 업데이트 및 삭제 작업 레코드를 포함합니다. |
| 서비스 상태 | 활동 로그  | 통합 경고에서 지원되지 않습니다. [서비스 알림에 대한 활동 로그 경고 만들기](monitoring-activity-log-alerts-on-service-notifications.md)를 참조하세요.  |
| Application Insights | 로그  | 응용 프로그램의 성능 세부 정보가 있는 로그를 포함합니다. 분석 쿼리를 사용하면 응용 프로그램 데이터에 따라 수행할 작업의 조건을 정의할 수 있습니다. |
| Application Insights | 메트릭 | 통합 경고에서 지원되지 않습니다. [메트릭 경고](../application-insights/app-insights-alerts.md)를 참조하세요. |
| Application Insights | 웹 가용성 테스트 | 통합 경고에서 지원되지 않습니다.  [웹 테스트 경고](../application-insights/app-insights-monitor-web-app-availability.md)를 참조하세요. Application Insights에 데이터를 보내도록 계측되는 모든 웹 사이트에서 사용할 수 있습니다. 웹 사이트의 가용성이나 응답성이 기대 이하이면 알림을 받습니다. |

## <a name="enhanced-unified-alerts-public-preview"></a>향상된 통합 경고(공개 미리 보기)

향상된 통합 경고 환경은 2018년 6월 1일에 Azure Monitor의 공개 미리 보기로 릴리스되었습니다. 이 환경은 2018년 3월에 릴리스되었으며, 개별 경고를 관리 및 집계하고 경고 상태를 수정하는 기능을 제공하는 [통합 경고](#overview)의 이점을 활용합니다. 이 섹션에서는 새 기능 및 Azure Portal에서 새 경고 페이지를 탐색하는 방법을 설명합니다.

### <a name="enhanced-unified-alerts"></a>향상된 통합 경고

새 환경은 클래식 통합 환경에는 없는 다음과 같은 기능을 제공합니다.

- **구독의 알림 보기**: 이제 단일 보기에서 여러 구독의 개별 경고 인스턴스를 보고 관리할 수 있습니다.
- **경고 상태 관리**: 이제 경고에 완료 확인 여부를 나타내는 상태가 있습니다.
- **스마트 그룹으로 경고 구성**: 스마트 그룹이 자동으로 관련 경고를 그룹화하므로 개별 경고가 아닌 경고 집합으로 관리할 수 있습니다.

### <a name="enable-enhanced-unified-alerts"></a>향상된 통합 경고 사용
경고 페이지 맨 위에 있는 배너를 선택하면 새로운 통합 경고 환경을 사용할 수 있습니다. 이 프로세스는 지난 30일 동안 지원되는 서비스에서 발생한 경고를 포함하는 경고 저장소를 만듭니다. 새 환경을 사용하도록 설정한 후 이 배너를 선택하면 새 환경과 기존 환경 간에 전환할 수 있습니다.

> [!NOTE]
>  처음에 새 환경을 사용할 수 있을 때까지 몇 분 정도 걸릴 수 있습니다.

![배너](media/monitoring-overview-unified-alerts/opt-in-banner.png)

새 환경을 사용하도록 설정하면 액세스 가능한 모든 구독이 등록됩니다. 전체 구독을 사용할 수 있지만, 새 환경을 선택한 사용자만 새 환경을 볼 수 있습니다. 구독에 액세스할 수 있는 다른 사용자는 별도로 환경을 활성화해야 합니다.

새 경고 환경을 사용하도록 설정해도 경고 규칙의 작업 그룹 또는 알림 구성에 영향을 주지 않습니다. 발생한 경고 인스턴스를 Azure Portal에서 살펴보고 관리하는 방법만 바뀝니다.

### <a name="smart-groups"></a>스마트 그룹
스마트 그룹을 사용하면 관련 경고를 개별 경고로 관리하는 대신 단일 단위로 관리하여 노이즈를 줄일 수 있습니다. 스마트 그룹의 세부 정보를 살펴보고 경고와 유사하게 상태를 설정할 수 있습니다. 각 경고는 단일 스마트 그룹의 구성원입니다.

기계 학습을 사용하여 단일 문제를 나타내는 관련 경고를 결합하면 자동으로 스마트 그룹이 만들어집니다. 경고가 만들어지면 과거 패턴, 유사한 속성, 유사한 구조 등의 정보에 따라 알고리즘이 해당 경고를 새 스마트 그룹 또는 기존 스마트 그룹에 추가합니다. 

현재 알고리즘은 구독 내 동일한 모니터 서비스의 경고만 고려합니다. 스마트 그룹은 이 통합을 통해 경고 노이즈를 99%까지 줄일 수 있습니다. 스마트 그룹 세부 정보 페이지에서 경고가 그룹에 포함된 이유를 볼 수 있습니다.

스마트 그룹의 이름은 첫 번째 경고의 이름입니다. 스마트 그룹을 만들거나 이름을 바꿀 수 없습니다.


### <a name="alert-states"></a>경고 상태
향상된 통합 경고는 경고 상태 개념이 도입되었습니다. 경고가 해결 과정에 있는지 지정하는 경고 상태를 설정할 수 있습니다. 경고가 만들어지면 *신규* 상태가 됩니다. 경고를 확인할 때 및 경고를 닫을 때 상태를 변경할 수 있습니다. 모든 상태 변경은 경고 기록에 저장됩니다.

다음은 지원되는 경고 상태입니다.

| 시스템 상태 | 설명 |
|:---|:---|
| 새로 만들기 | 문제가 방금 검색되었으며 아직 검토되지 않았습니다. |
| 확인됨 | 관리자가 경고를 검토하고 작업을 시작했습니다. |
| 닫힘 | 문제가 해결되었습니다. 경고가 닫힌 후 다른 상태로 변경하면 경고를 다시 열 수 있습니다. |

경고 상태는 모니터 조건과 다릅니다. 오류 조건이 더 이상 충족되지 않으면 메트릭 경고 규칙이 경고를 _해결됨_ 조건으로 설정할 수 있습니다. 경고 상태는 사용자가 설정하며 모니터 조건과 독립적 관계입니다. 시스템에서 모니터 조건을 “해결됨”으로 설정할 수 있지만, 경고 상태는 사용자가 변경할 때까지 변경되지 않습니다.

#### <a name="change-the-state-of-an-alert-or-smart-group"></a>경고 또는 스마트 그룹의 상태 변경
스마트 그룹의 상태를 설정하여 개별 경고의 상태를 변경하거나 여러 경고를 함께 관리할 수 있습니다.

경고 세부 정보 보기에서 **경고 상태 변경**을 선택하여 경고 상태를 변경합니다. 또는 세부 정보 보기에서 **스마트 그룹 상태 변경**을 선택하여 스마트 그룹 상태를 변경합니다. 먼저 목록 보기에서 여러 항목을 선택하고 페이지 맨 위에서 **상태 변경**을 클릭하여 여러 항목의 상태를 동시에 변경합니다. 

어떤 방법을 사용하든 드롭다운 메뉴에서 새 상태를 선택합니다. 그런 다음, 필요에 따라 주석을 입력합니다. 단일 항목을 변경하는 경우, 스마트 그룹의 모든 경고에 동일한 변경 내용을 적용하는 옵션을 사용할 수도 있습니다.

![상태 변경](media/monitoring-overview-unified-alerts/change-tate.png)

### <a name="alerts-page"></a>경고 페이지
기본 경고 페이지는 특정 시간 내에 생성된 경고의 요약 정보를 제공합니다. 심각도 상태별 총 경고 수를 나타내는 열을 통해 심각도별 총 경고 수가 표시됩니다. 아무 심각도나 선택하면 해당 심각도를 기준으로 필터링된 [모든 경고](#all-alerts-page) 페이지가 열립니다.

![경고 페이지](media/monitoring-overview-unified-alerts/alerts-page.png)

페이지 맨 위에 있는 드롭다운 메뉴에서 값을 선택하여 이 보기를 필터링할 수 있습니다.

| 열 | 설명 |
|:---|:---|
| 구독 | Azure 구독을 최대 5개까지 선택합니다. 선택한 구독의 경고만 보기에 포함됩니다. |
| 리소스 그룹 | 단일 리소스 그룹을 선택합니다. 선택한 리소스 그룹의 대상이 있는 경고만 보기에 포함됩니다. |
| 시간 범위 | 선택한 기간 내에 발생한 경고만 보기에 포함됩니다. 지원되는 값은 지난 1시간, 지난 24시간, 지난 7일 및 지난 30일입니다. |

[경고] 페이지 맨 위에서 다음 값을 선택하면 다른 페이지가 열립니다.

| 값 | 설명 |
|:---|:---|
| 총 경고 수 | 선택한 조건과 일치하는 총 경고 수입니다. 이 값을 선택하면 필터 없이 [모든 경고] 보기가 열립니다. |
| 스마트 그룹 | 선택한 조건과 일치하는 경고에서 만들어진 스마트 그룹의 총수입니다. 이 값을 선택하면 [모든 경고] 보기에 스마트 그룹 목록이 열립니다.
| 총 경고 규칙 수 | 선택한 구독 및 리소스 그룹에 있는 총 경고 규칙 수입니다. 이 값을 선택하면 선택한 구독 및 리소스 그룹을 기준으로 필터링된 규칙 보기가 열립니다.


### <a name="all-alerts-page"></a>[모든 경고] 페이지 
[모든 경고] 페이지를 사용하면 선택한 기간 내에 생성된 경고 목록을 볼 수 있습니다. 개별 경고 목록 또는 경고가 포함된 스마트 그룹 목록을 볼 수 있습니다. 페이지 맨 위에 있는 배너를 선택하면 보기가 전환됩니다.

![[모든 경고] 페이지](media/monitoring-overview-unified-alerts/all-alerts-page.png)

페이지 맨 위에 있는 드롭다운 메뉴에서 다음 값을 선택하여 보기를 필터링할 수 있습니다.

| 열 | 설명 |
|:---|:---|
| 구독 | Azure 구독을 최대 5개까지 선택합니다. 선택한 구독의 경고만 보기에 포함됩니다. |
| 리소스 그룹 | 단일 리소스 그룹을 선택합니다. 선택한 리소스 그룹의 대상이 있는 경고만 보기에 포함됩니다. |
| 리소스 종류 | 리소스 종류를 하나 이상 선택합니다. 선택한 형식의 대상이 있는 경고만 보기에 포함됩니다. 이 열은 리소스 그룹을 지정한 후에만 사용할 수 있습니다. |
| 리소스 | 리소스를 선택합니다. 해당 리소스가 대상으로 지정된 경고만 보기에 포함됩니다. 이 열은 리소스 종류를 지정한 후에만 사용할 수 있습니다. |
| 심각도 | 경고 심각도를 선택하거나, 심각도에 상관없이 모든 경고를 포함하려면 *모두*를 선택합니다. |
| 조건 모니터링 | 모니터 조건을 선택하거나, 조건에 상관없이 모든 경고를 포함하려면 *모두*를 선택합니다. |
| 경고 상태 | 경고 상태를 선택하거나, 상태에 상관없이 모든 경고를 포함하려면 *모두*를 선택합니다. |
| 서비스 모니터링 | 서비스를 선택하거나, 모든 서비스를 포함하려면 *모두*를 선택합니다. 서비스를 대상으로 사용하는 규칙에서 생성된 경고만 포함됩니다. |
| 시간 범위 | 선택한 기간 내에 발생한 경고만 보기에 포함됩니다. 지원되는 값은 지난 1시간, 지난 24시간, 지난 7일 및 지난 30일입니다. |

페이지 맨 위에서 **열**을 선택하여 표시할 열을 선택합니다. 

### <a name="alert-detail-page"></a>경고 세부 정보 페이지
경고를 선택하면 경고 세부 정보 페이지가 표시됩니다. 이 페이지에서 경고 세부 정보를 확인하고, 경고 상태를 변경할 수 있습니다.

![경고 세부 정보](media/monitoring-overview-unified-alerts/alert-detail.png)

경고 세부 정보 페이지에는 다음과 같은 섹션이 있습니다.

| 섹션 | 설명 |
|:---|:---|
| 기본 정보 | 경고에 대한 속성과 기타 중요한 정보를 표시합니다. |
| 기록 | 경고에서 수행한 각 작업과 경고의 변경 내용을 나열합니다. 현재는 상태 변경으로 제한되어 있습니다. |
| 스마트 그룹 | 경고가 포함된 스마트 그룹에 대한 정보입니다. *경고 수*는 스마트 그룹에 포함된 경고 수를 나타냅니다. 여기에는 동일한 스마트 그룹에서 지난 30일간 생성된 다른 경고가 포함됩니다.  이것은 경고 목록 페이지의 시간 필터와 아무 관계도 없습니다. 경고를 선택하면 세부 정보를 볼 수 있습니다. |
| 자세한 내용 | 일반적으로 경고를 만든 원본 형식과 관련된 추가 컨텍스트 정보를 표시합니다. |


### <a name="smart-group-detail-page"></a>스마트 그룹 세부 정보 페이지
스마트 그룹을 선택하면 스마트 그룹 세부 정보 페이지가 표시됩니다. 이 페이지에서 그룹을 만드는 데 사용된 추론을 포함하여 스마트 그룹 세부 정보를 확인하고, 스마트 그룹 상태를 변경할 수 있습니다.
 
![스마트 그룹 세부 정보](media/monitoring-overview-unified-alerts/smart-group-detail.png)


스마트 그룹 세부 정보 페이지에는 다음과 같은 섹션이 있습니다.

| 섹션 | 설명 |
|:---|:---|
| 경고 | 스마트 그룹에 포함된 개별 경고를 나열합니다. 경고를 선택하면 경고 세부 정보 페이지가 열립니다. |
| 기록 | 스마트 그룹에서 수행한 각 작업과 스마트 그룹의 변경 내용을 나열합니다. 현재는 상태 변경 및 경고 멤버 자격 변경으로 제한되어 있습니다. |
