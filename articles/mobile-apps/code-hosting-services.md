# <a name="cloud-hosted-source-code-management-services"></a>클라우드 호스 티 드 소스 코드 관리 서비스
여러 팀 멤버가 동일한 코드 베이스에서 작업 하는 개발 팀이 있는 경우 코드를 호스트 하는 데 적합 한 장소를 찾아야 합니다. 데이터를 클라우드에 저장 하 고 중앙 리포지토리를 사용 하면 모든 사용자가 코드 파일을 업로드 하 고, 편집 하 고, 관리 하 고, 프로젝트의 다른 개발자와 상호 작용 하 고, 사용자가 어디에 있든 상관 없이 코드에 쉽게 액세스할 수 있는 기능을 제공 합니다. 인터넷에 연결 되어 있어야 합니다.

클라우드 호스트는 하드웨어 구성이 거의 필요 하지 않으므로 온-프레미스 옵션 보다 훨씬 더 간단 하며, 조직에서 보다 민첩 한 방식으로 구현 프로세스를 완료할 수 있습니다.

## <a name="benefits-of-hosting-source-code-in-the-cloud"></a>클라우드에서 소스 코드를 호스트 하는 이점
- 어디에서 나 데이터를 보고 관리할 수 있는 **중앙 집중식 클라우드 저장소** 입니다.
- 팀 내에서 **더 나은 공동 작업 및 클리너 코드** 를 사용 하 여 코드를 추적 하 고 프로젝트를 관리 하 여 우수한 소프트웨어의 지속적인 업데이트를 보장 합니다.
- **참여** 하 고 프로젝트에 **쉽게 기여할 수** 있습니다.
- 팀이 더 빠르고 간편 하 게 프로젝트에 **참여할** 수 있으므로 **릴리스 주기가 더 빨라졌습니다** .
- 하드웨어, 서버, VPN 등을 유지 관리 하는 데 걱정 하지 않고도 **비용을 절감** 하세요.

다음 서비스를 사용 하 여 클라우드에서 응용 프로그램 데이터를 호스팅합니다.

## <a name="github"></a>GitHub
[GitHub](https://github.com/) 는 다양 한 프로그래밍 언어에서 소스 코드 프로젝트를 호스트 하 고 모든 반복에 대 한 다양 한 변경 내용을 추적 하는 오픈 소스 리포지토리 호스팅 서비스입니다.

**주요 기능**
- 모든 코드를 한 곳에 포함 하 여 **코드를 호스팅합니다** . 모든 리포지토리에는 전용, 공개 또는 오픈 소스가 포함 되어 있으며 코드를 호스트 하 고, 버전을 만들고, 릴리스 하는 데 도움이 되는 도구가 포함 되어 있습니다.
- 원활한 **코드 검토** 및 기본 제공 검토 도구를 사용 하면 팀 프로세스의 핵심 부분을 코드에서 검토할 수 있습니다.
    - 분기를 보호 하 고 변경 내용을 제안 하 고 검토를 요청 합니다.
    - 컨텍스트에서 차이점 및 설명을 참조 하 고 명확한 피드백을 받습니다.
- 초기에 맞춰 조정 하 고 GitHub의 **프로젝트 관리 도구**를 사용 하 여 더 많은 작업을 수행 하세요.
    - 프로젝트의 큰 그림을 참조 하세요.
    - GitHub 내의 코드 바로 옆에 있는 **작업 보드** .
    - **카드를 끌어서** 놓아 팀 구성원에 게 문제 또는 끌어오기 요청을 할당할 수 있습니다.
    - 진행률을 구성 하 고 추적 하는 **마일스 톤** 입니다.
    - 유용 하 게 사용할 수 있지만 특정 문제나 끌어오기 요청에 속하지 않는 아이디어를 캡처하기 위한 **정보** 입니다.
- **[GitHub Marketplace](https://github.com/marketplace)** 에서 응용 프로그램을 구입 하 여 원활 하 게 통신 하 고 작업을 자동화 하기 위한 적절 한 도구를 쉽게 찾아서 선택 합니다.
- 팀 **관리** -팀 및 액세스 권한을 구성 하는 데 도움이 되는 **사용자 역할** , 팀 및 액세스 권한을 구성 하는 데 도움이 되는 토론 스레드 도구, 토픽 및 팀의 대화를 유지 하는 **토론 스레드** 도구 및 빠르게 설정할 **커뮤니티 지침** 계정을 사용 하 여 새 팀 구성원
- 이러한 작업을 수행 **하려면 인기 있는** 인기 있는 프로젝트를 **찾아보세요** .
- **네트워크를** 통해 업계의 다른 사용자에 게 알아보세요.

## <a name="azure-devops"></a>Azure DevOps
[Azure DevOps](https://azure.microsoft.com/services/devops/) 는 원본 제어 옵션으로 TFVC ( [Azure Repos](https://azure.microsoft.com/services/devops/repos/) 및 [Team Foundation 버전 제어)](https://docs.microsoft.com/azure/devops/repos/tfvc/index?view=azure-devops) 를 지원 합니다. 높은 품질의 코드를 보장 하기 위해 공동 작업 코드 검토, 고급 파일 관리, 코드 검색 및 분기 정책으로 무제한 무료 개인 리포지토리를 제공 합니다. Azure Repos은 기본 AAD 지원 및 고급 정책이 필요한 대규모 조직 뿐만 아니라 소규모 프로젝트에도 적합 합니다.
    
**주요 기능**
- 공용 및 개인 리포지토리에 대 한 **무제한 클라우드 호스트 Git 소스 코드 리포지토리**
    - 모든 Git 클라이언트에 대 한 지원.
    - 웹 후크 및 API 통합.
- 리포지토리 끌어오기 요청에서 **다음 빌드** 를 시작 합니다.
    - 각 변경에 대해 스레드된 토론 및 연속 통합을 사용 하 여 더 나은 코드를 작성 하기 위해 공동 작업 합니다.
    - Azure Pipelines 또는 개인 도구로 CI/CD(지속적인 통합/지속적인 업데이트)를 설정하여 끌어오기 요청이 완료될 때마다 자동으로 빌드, 테스트 및 배포를 트리거하세요.
    - 분기 정책을 사용 하 여 코드 품질을 보호 합니다.
- 코드 검토를 포함 하 여 Team Foundation 버전 제어를 사용 하는 **중앙 집중식 버전 제어**.
- Xcode, Eclipse, IntelliJ, Android Studio, Visual Studio, Visual Studio Code 등을 사용 하 여 **코드에 연결** 합니다.
- **강력한 의미 체계 코드 검색**
- **Enterprise** 는 AD (Azure Active Directory)와의 기본 통합을 포함 하므로 코드 리포지토리에 대 한 액세스를 관리 하는 프로세스를 간소화 합니다.
- 코드 검토의 최소 수와 같은 분기 정책으로 **코드 품질을 보장** 하 고 성공적인 빌드를 요구 하며 Git 병합 전략을 적용 합니다.
- **선호 하는 개발 환경을 연결** 하 여 리포지토리에 액세스 하 고 작업을 관리 합니다.
