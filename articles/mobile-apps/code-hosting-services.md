# <a name="cloud-hosted-source-code-management-services"></a>클라우드 호스팅 소스 코드 관리 서비스
동일한 코드 베이스에서 작업하는 여러 팀 멤버가 있는 개발 팀이 있는 경우 코드를 호스팅할 적절한 장소를 찾아야 합니다. 클라우드에 데이터를 저장하고 중앙 집중식 리포지토리를 사용하면 모든 사용자가 코드 파일을 업로드, 편집 및 관리할 수 있습니다. 또한 프로젝트에서 다른 개발자와 상호 작용할 수도 있습니다. 인터넷에 연결되어 있는 한 어디에 있든 코드에 쉽게 액세스할 수 있습니다.

클라우드 호스팅은 온-프레미스 옵션보다 훨씬 쉽습니다. 하드웨어 구성이 적고 조직에서 보다 민첩한 방식으로 구현 프로세스를 완료할 수 있습니다.

## <a name="benefits-of-hosting-source-code-in-the-cloud"></a>클라우드에서 소스 코드를 호스팅하는 이점
- **중앙 집중식 클라우드 스토리지:** 어디서나 데이터를 보고 관리할 수 있습니다.
- **더 나은 협업 및 깔끔한 코드:** 팀 내에서 코드를 추적하고 프로젝트를 관리하여 우수한 소프트웨어가 지속적으로 제공되도록 합니다.
- **더 쉽게 참여하세요:** 프로젝트에 쉽게 기여할 수 있습니다.
- **더 빠른 릴리스 주기:** 팀에서 더 빠르게 작업하고 프로젝트에 쉽게 기여할 수 있습니다.
- **비용 절감:** 자체 하드웨어, 서버, VPN 등을 유지 관리하는 것에 대해 걱정하지 마십시오.

다음 서비스를 사용하여 클라우드에서 응용 프로그램 데이터를 호스팅합니다.

## <a name="github"></a>GitHub
[GitHub는](https://github.com/) 다양한 프로그래밍 언어로 소스 코드 프로젝트를 호스팅하는 오픈 소스 리포지토리 호스팅 서비스입니다. GitHub는 모든 반복에 대한 다양한 변경 사항을 추적합니다.

**주요 기능**
- 코드 호스팅을 사용하여 모든 코드를 한 곳에 보관합니다. 개인 저장소, 공용 및 오픈 소스 리포지토리에는 모두 코드를 호스팅, 버전 및 릴리스하는 데 도움이 되는 도구가 갖춰져 있습니다.
- 코드를 검토하고 기본 제공 검토 도구를 사용하여 코드 검토를 모든 팀 프로세스의 필수적인 부분으로 만듭니다.
    - 지점을 보호하고, 변경 사항을 제안하고, 검토를 요청합니다.
    - 차이를 발견하고, 맥락에서 댓글을 달고, 명확한 피드백을 얻을 수 있습니다.
- GitHub의 프로젝트 관리 도구를 사용하여 조기에 조정하고, 정렬상태를 유지하고, 더 많은 작업을 수행할 수 있습니다.
    - 프로젝트의 큰 그림을 참조하십시오.
    - GitHub 내부의 코드 바로 옆에 있는 작업 보드를 사용합니다.
    - 카드를 드래그하여 문제를 할당하거나 팀 멤버에게 요청을 가져옵니다.
    - 이정표를 설정하여 진행 상황을 구성하고 추적합니다.
    - 유용할 수 있지만 특정 문제나 끌어오기 요청에 속하지 않는 아이디어를 캡처하기 위해 메모를 작성합니다.
- 쉽게 발견하고 [GitHub 마켓 플레이스에서](https://github.com/marketplace)응용 프로그램을 구입하여 작업의 더 나은 의사 소통과 자동화를위한 올바른 도구를 선택합니다.
- 다음을 사용하여 팀을 관리하고 성장시다. 
    - 팀을 구성하고 권한에 액세스하는 데 도움이 되는 사용자 역할입니다.
    - 주제와 팀에 초점을 맞춘 대화를 유지하는 토론 스레드 도구입니다.
    - 계정을 사용하여 새 팀 멤버를 신속하게 설정할 수 있는 커뮤니티 지침입니다.
- 검색하고 그들을 따라 인기있는 프로젝트를 스타.
- 업계의 다른 사람들로부터 네트워크를 가지고 배우십시오.

**참조**
- [GitHub](https://github.com/)
- [GitHub 가이드](https://guides.github.com/)
- [GitHub 커뮤니티 포럼](https://github.community/)
- [GitHub 마켓플레이스](https://github.com/marketplace)

## <a name="azure-devops"></a>Azure DevOps
[Azure DevOps는](https://azure.microsoft.com/services/devops/) [Azure 리포지토리](https://azure.microsoft.com/services/devops/repos/) 및 [팀 기초 버전 제어(TFVC)를](https://docs.microsoft.com/azure/devops/repos/tfvc/index?view=azure-devops) 소스 제어 옵션으로 지원합니다. 공동 작업 코드 검토, 고급 파일 관리, 코드 검색 및 분기 정책을 갖춘 무제한 무료 개인 리포지토리가 있어 고품질 코드를 보장합니다. Azure Repos는 네이티브 Azure Active Directory 지원 및 고급 정책이 필요한 소규모 프로젝트 및 대규모 조직에 적합합니다.
    
**주요 기능**
- 공용 및 개인 리포지토리에 무제한 클라우드 호스팅 Git 소스 코드 리포지토리를 사용합니다.
    - 모든 Git 클라이언트에 대한 지원을 받으십시오.
    - 웹 후크 및 API 통합을 사용합니다.
- 리포지토리 풀 요청에서 다음 빌드시작:
    - 각 변경에 대해 스레드 토론 및 지속적인 통합을 사용하여 더 나은 코드를 빌드하기 위해 공동 작업합니다.
    - 완료된 모든 끌어오기 요청으로 빌드, 테스트 및 배포를 자동으로 트리거하도록 연속 통합/연속 전달(CI/CD)을 설정합니다. Azure 파이프라인 또는 도구를 사용할 수 있습니다.
    - 분기 정책으로 코드 품질을 보호합니다.
- 코드 검토를 포함하는 팀 파운데이션 버전 제어를 통해 중앙 집중식 버전 제어를 유지관리합니다.
- Xcode, 이클립스, IntelliJ, 안드로이드 스튜디오, 비주얼 스튜디오, 비주얼 스튜디오 코드 등을 사용하여 코드에 연결합니다.
- 강력한 의미 체계 코드 검색을 사용합니다.
- 엔터프라이즈 지원 기능의 이점을 누리실 수 있습니다. Azure DevOps에는 Azure Active Directory와 네이티브 통합이 있어 코드 리포지토리에 대한 액세스 관리 프로세스를 간소화합니다.
- 최소 코드 검토 수, 성공적인 빌드에 대한 요구 사항 및 Git 병합 전략 의 적용과 같은 분기 정책으로 코드 품질을 보장합니다.
- 즐겨 찾는 개발 환경을 연결하여 리포지토리에 액세스하고 작업을 관리합니다.
