# <a name="cloud-hosted-source-code-management-services"></a>클라우드에서 호스트 되는 소스 코드 관리 서비스
동일한 코드 베이스를 사용 하는 여러 팀 멤버가 있는 개발 팀이 있는 경우 코드를 호스트 하는 데 적합 한 장소를 찾아야 합니다. 클라우드에 데이터를 저장 하 고 중앙 리포지토리를 사용 하면 모든 사용자가 코드 파일을 업로드, 편집 및 관리할 수 있습니다. 또한 프로젝트의 다른 개발자와 상호 작용할 수 있습니다. 인터넷에 연결 되어 있는 한, 어디에 있든 관계 없이 코드에 쉽게 액세스할 수 있습니다.

클라우드 호스팅은 온-프레미스 옵션 보다 훨씬 쉽습니다. 이 작업을 수행 하려면 더 짧은 하드웨어 구성이 필요 하며, 조직에서 보다 민첩 한 방식으로 구현 프로세스를 완료할 수 있습니다.

## <a name="benefits-of-hosting-source-code-in-the-cloud"></a>클라우드에서 소스 코드를 호스트 하는 이점
- **중앙 집중식 클라우드 저장소:** 어디에서 나 데이터를 보고 관리 합니다.
- **더 나은 공동 작업 및 클리너 코드:** 뛰어난 소프트웨어를 지속적으로 제공할 수 있도록 팀 내에서 코드를 추적 하 고 프로젝트를 관리 하세요.
- **더 쉽게 관련:** 프로젝트에 쉽게 기여할 수 있습니다.
- **더 빠른 릴리스 주기:** 팀에서 더 빠르게 작업 하 고 프로젝트에 쉽게 기여할 수 있습니다.
- **비용 절감:** 하드웨어, 서버, VPN 등을 유지 관리할 필요가 없습니다.

다음 서비스를 사용 하 여 클라우드에서 응용 프로그램 데이터를 호스팅합니다.

## <a name="github"></a>GitHub
[GitHub](https://github.com/) 는 다양 한 프로그래밍 언어로 소스 코드 프로젝트를 호스팅하는 오픈 소스 리포지토리 호스팅 서비스입니다. GitHub는 모든 반복에 대 한 다양 한 변경 내용을 추적 합니다.

**주요 기능**
- 코드 호스팅을 사용 하 여 모든 코드를 한 곳에 보관 합니다. 개인, 공용 및 오픈 소스 리포지토리에는 모두 호스트, 버전 및 릴리스 코드를 위한 도구가 포함 되어 있습니다.
- 코드를 검토 하 고 기본 제공 검토 도구를 사용 하 여 코드 검토를 팀 프로세스의 필수적인 부분으로 만듭니다.
    - 분기를 보호 하 고, 변경 내용을 제안 하 고, 검토를 요청 합니다.
    - 차이점을 파악 하 고, 컨텍스트에 주석을 추가 하 고, 명확한 피드백을 받습니다.
- 초기에 맞춰 조정 하 고 GitHub의 프로젝트 관리 도구를 사용 하 여 더 많은 작업을 수행 하세요.
    - 프로젝트의 큰 그림을 참조 하세요.
    - GitHub 내에서 코드 바로 옆에 있는 작업 보드를 사용 합니다.
    - 카드를 끌어서 문제를 할당 하거나 팀 멤버에 게 요청을 끌어옵니다.
    - 마일스 톤을 구성 하 고 진행률을 추적 합니다.
    - 유용 하지만 특정 문제나 끌어오기 요청에 속하지 않는 아이디어를 캡처하기 위해 메모를 작성 합니다.
- [GitHub Marketplace](https://github.com/marketplace)에서 응용 프로그램을 구입 하 여 원활 하 게 통신 하 고 작업을 자동화 하기 위한 적절 한 도구를 쉽게 찾아서 선택 합니다.
- 다음을 사용 하 여 팀을 관리 하 고 확장 합니다. 
    - 팀 및 액세스 권한을 구성 하는 데 도움이 되는 사용자 역할
    - 토픽 및 팀에서 대화를 유지 하는 데 도움이 되는 토론 스레드 도구입니다.
    - 계정을 사용 하 여 새 팀 구성원을 신속 하 게 설정 하는 커뮤니티 지침
- 이러한 작업을 수행 하려면 인기 있는 인기 있는 프로젝트를 찾아보세요.
- 네트워크를 통해 업계의 다른 사용자에 게 알아보세요.

**참조**

- [GitHub 가이드](https://guides.github.com/)

## <a name="azure-devops"></a>Azure DevOps
[Azure DevOps](https://azure.microsoft.com/services/devops/) 는 원본 제어 옵션으로 TFVC ( [Azure Repos](https://azure.microsoft.com/services/devops/repos/) 및 [Team Foundation 버전 제어)](https://docs.microsoft.com/azure/devops/repos/tfvc/index?view=azure-devops) 를 지원 합니다. 높은 품질의 코드를 보장 하기 위해 공동 작업 코드 검토, 고급 파일 관리, 코드 검색 및 분기 정책으로 무제한 무료 개인 리포지토리를 제공 합니다. Azure Repos은 네이티브 Azure Active Directory 지원 및 고급 정책을 필요로 하는 소규모 프로젝트와 대규모 조직에 적합 합니다.
    
**주요 기능**
- 공용 및 개인 리포지토리에 대해 무제한 클라우드 호스트 Git 소스 코드 리포지토리를 사용 합니다.
    - 모든 Git 클라이언트에 대 한 지원을 받습니다.
    - 웹 후크 및 API 통합을 사용 합니다.
- 리포지토리 끌어오기 요청에서 다음 빌드를 시작 합니다.
    - 각 변경에 대해 스레드된 토론 및 연속 통합을 사용 하 여 더 나은 코드를 작성 하기 위해 공동 작업 합니다.
    - 완료 된 모든 끌어오기 요청으로 빌드, 테스트 및 배포를 자동으로 트리거하기 위해 CI/CD (지속적인 통합/지속적인 업데이트)를 설정 합니다. Azure Pipelines 또는 도구를 사용할 수 있습니다.
    - 분기 정책을 사용 하 여 코드 품질을 보호 합니다.
- 코드 검토를 포함 하는 Team Foundation 버전 제어를 사용 하 여 중앙 집중식 버전 제어를 유지 합니다.
- Xcode, Eclipse, IntelliJ, Android Studio, Visual Studio, Visual Studio Code 등을 사용 하 여 코드에 연결 합니다.
- 강력한 의미 체계 코드 검색을 사용 합니다.
- 엔터프라이즈급 기능을 활용 하세요. Azure DevOps는 코드 리포지토리에 대 한 액세스를 관리 하는 프로세스를 간소화 하는 Azure Active Directory와의 기본 통합을 포함 합니다.
- 코드 검토의 최소 수, 성공적인 빌드에 대 한 요구 사항 및 Git 병합 전략 적용과 같은 분기 정책으로 코드 품질을 보장 합니다.
- 선호 하는 개발 환경을 연결 하 여 리포지토리에 액세스 하 고 작업을 관리 합니다.
