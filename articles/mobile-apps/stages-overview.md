# <a name="different-stages-in-mobile-application-development"></a>모바일 애플리케이션 개발의 다양한 단계
모바일 응용 프로그램을 빌드하려면 여러 단계가 포함됩니다. Microsoft 서비스, 도구 및 기술을 쉽게 활용하여 고급 모바일 응용 프로그램을 빌드할 수 있습니다.

## <a name="app-development"></a>앱 개발
Objective-C 및 Java와 같은 언어를 사용하성으로 네이티브 단일 플랫폼 클라이언트 응용 프로그램을 빌드할 수 있습니다. Xamarin, React 네이티브 및 Unity를 사용하여 플랫폼 간 앱을 빌드할 수 있습니다. 또한 코르도바를 사용하여 하이브리드 앱을 빌드할 수 있습니다. 클라이언트 응용 프로그램을 구성하려면 Visual Studio, Mac용 Visual Studio 또는 PC 및 Mac의 Visual Studio 코드와 같은 IID 및 코드 편집기를 사용합니다.

## <a name="plan-and-design-with-back-end-services"></a>백 엔드 서비스로 계획 및 설계
 Microsoft는 모바일 응용 프로그램에 대한 안전하고 확장 가능하며 신뢰할 수 있는 백 엔드를 설정하기 위한 다양한 서비스를 제공합니다. 자체 서버를 유지 관리하지 않고도 서버없는 API 및 트리거를 만들 수 있습니다. 푸시 알림, 오프라인 데이터 동기화, 인증 및 데이터 서비스를 사용하여 다양한 앱 구성 시나리오를 지원하는 포괄적이고 광범위한 모바일 백 엔드를 형성할 수 있습니다. 
 
 백 엔드 서비스를 빌드하는 데는 많은 요구 사항이 있습니다. 그러려면 다음 작업을 수행해야 합니다.
   - 장치의 데이터 저장을 제한합니다.
   - 여러 장치에서 데이터를 동기화합니다.
   - 네트워크 연결 없이 응용 프로그램이 실행되는지 확인합니다.
   - 사용자에게 알림을 보냅니다.
   - 배터리 소모를 줄입니다.
   - 확장 가능하고 안정적이며 안전한 백 엔드를 구축합니다.
   - 서버 유지 관리를 자동화합니다.

## <a name="devops-and-continuous-monitoring"></a>데브옵스 및 지속적인 모니터링
모바일 및 클라우드 중심 애플리케이션을 만들려면 CI(연속 통합), 연속 전송(CD) 및 지속적인 모니터링 서비스를 활용하여 모든 플랫폼에 대한 네이티브 빌드를 생성합니다. 수천 개의 물리적 장치에서 UI 자동화 테스트를 실행하거나 베타 테스트 채널을 통해 릴리스를 자동화하거나 앱 스토어에 직접 배포할 수 있습니다. 또한 해당 응용 프로그램에 대한 사용 분석, 실패 및 예외 정보를 수집할 수도 있습니다.

## <a name="additional-services"></a>서비스
빌드되는 응용 프로그램의 유형 및 특성에 따라 많은 Azure 서비스를 사용할 수 있습니다.
  - Azure Cognitive Search를 통해 빠른 쿼리를 통해 모든 유형의 내부 데이터를 검색합니다.
  - 응용 프로그램에서 인공 지능을 사용하는 Azure 인지 서비스입니다.
  - 응용 프로그램에서 실시간 통신을 활성화하는 Azure SignalR입니다.
