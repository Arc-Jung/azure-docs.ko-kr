# <a name="perform-automated-testing-on-thousands-of-mobile-devices-simultaneously"></a>수천 개의 모바일 장치에서 동시에 자동화 된 테스트 수행
장치 조각화, 응용 프로그램 복잡성, 빠른 릴리스 주기, 짧은 세션 및 높은 사용자 기대치로 인해 테스트는 어려울 수 있으며 점점 더 복잡할 수 있습니다. 뛰어난 고객 환경을 제공 하는 철저 하 게 테스트 된 응용 프로그램을 정기적으로 제공 해야 합니다. 테스트는 쉬운 작업이 아니라 개발자가 각 릴리스를 수동으로 테스트 하는 데 적합 하지 않습니다.

테스트 과제는 효율적으로 세울 수 있습니다. 자동화 된 테스트를 사용 하 여 다양 한 장치에서 모바일 응용 프로그램을 테스트할 수 있습니다. 클라우드 서비스를 사용 하 여 모바일 앱의 릴리스 속도를 높이고 응용 프로그램의 개발 및 배포에 병목 현상을 추가 하는 테스트 문제를 극복할 수 있습니다.

## <a name="importance-of-automated-testing"></a>자동화 된 테스트의 중요성
- 다양 한 폼 팩터, 연결 유형 및 OS 버전을 대상으로 하 여 테스트 검사를 늘립니다.
- 다양 한 장치 및 OS 검사를 테스트 하 여 시간, 비용 및 리소스를 절약 합니다.
- 반복적인 수동 테스트를 방지 하 고 새로운 자동화 된 소프트웨어 테스트를 만들고 복잡 한 기능을 처리 하는 데 더 많은 시간을 확보 하 여 정확성을 향상 시킵니다.
- 지속적으로 테스트 하 고, 응용 프로그램 성능을 분석 하 고, 사용자가 응용 프로그램을 실행 하기 전에 문제를 찾아 해결 하 고 빠르게 릴리스 하세요.
- 광범위 한 장치에서 고객에 게 최상의 환경을 제공 하 여 고품질 응용 프로그램을 제공 하세요.

## <a name="visual-studio-app-center"></a>Visual Studio App Center
[App Center 테스트](/appcenter/test-cloud/) 는 네이티브 및 하이브리드 모바일 앱을 위해 Android 및 iOS를 통해 클라우드에서 수천 개의 실제 장치에 대해 자동화 된 UI 테스트를 실행 하는 데 사용할 수 있는 클라우드 기반 서비스입니다. 2000 개 이상의 장치를 사용 하 여 테스트를 용이 하 게 하기 위해 자동으로 실행 될 수 있는 테스트 스크립트를 작성할 수 있습니다.

**주요 기능**
   - 사용자 환경에 가장 근접 한 부분을 제공 하기 위해 루 팅 되거나 탈 옥이 아닌 실제 장치에서 실행 되는 테스트입니다.
   - 스크린샷에 자동으로 캡처되고 각 테스트 단계에 연결 됩니다. 테스트 실행에서 대상으로 지정 된 모든 장치에 대해 나란히 볼 수 있습니다.
   - 테스트 실행의 모든 대상 장치에 대 한 장치 로그 및 실패 한 테스트 단계에 대 한 스택 추적.
   - 동시 테스트는 여러 장치에서 동시에 실행 되어 결과에 대 한 시간을 단축 합니다.
   - Android 및 iOS 용 Appium (JUnit Java), android 용 Espresso (Java), android 및 iOS 용 Calabash, android 및 iOS 용 UITest 및 ios 용 XCUITest와 같은 인기 있는 테스트 프레임 워크 지원.
   - IOS, Android Xamarin 및 기본에 대 한 플랫폼 지원.
