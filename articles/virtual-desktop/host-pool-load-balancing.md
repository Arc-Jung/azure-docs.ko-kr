# <a name="host-pool-load-balancing-methods"></a>호스트 풀 부하 분산 메서드

Windows 가상 데스크톱은 두 가지 로드 밸런싱 방법을 지원합니다. 각 메서드는 호스트 풀의 리소스에 연결할 때 사용자의 세션을 호스트할 세션 호스트를 결정합니다.

다음 로드 밸런싱 방법은 Windows 가상 데스크톱에서 사용할 수 있습니다.

- 광범위한 우선 로드 밸런싱을 사용하면 호스트 풀의 세션 호스트 간에 사용자 세션을 균등하게 배포할 수 있습니다.
- 깊이 우선 로드 밸런싱을 사용하면 호스트 풀의 사용자 세션으로 세션 호스트를 포화할 수 있습니다. 첫 번째 세션이 세션 제한 임계값에 도달하면 로드 밸런서가 한도에 도달할 때까지 호스트 풀의 다음 세션 호스트로 새 사용자 연결을 지시합니다.

각 호스트 풀은 특정한 로드 밸런싱의 한 가지 유형만 구성할 수 있습니다. 그러나 두 로드 밸런싱 방법은 현재 있는 호스트 풀에 관계없이 다음 동작을 공유합니다.

- 사용자가 이미 호스트 풀에 세션이 있고 해당 세션에 다시 연결하는 경우 로드 밸러블러는 기존 세션을 사용하여 세션 호스트로 성공적으로 리디렉션됩니다. 이 동작은 해당 세션 호스트의 AllowNewConnections 속성이 False로 설정된 경우에도 적용됩니다.
- 사용자가 호스트 풀에 세션이 아직 없는 경우 로드 밸런서에서는 AllowNewConnections 속성이 로드 균형 조정 중에 False로 설정된 세션 호스트를 고려하지 않습니다.

## <a name="breadth-first-load-balancing-method"></a>폭 우선 로드 밸런싱 방법

광범위한 우선 로드 밸런싱 방법을 사용하면 사용자 연결을 배포하여 이 시나리오에 최적화할 수 있습니다. 이 방법은 풀이 된 가상 데스크톱 환경에 연결하는 사용자에게 최상의 환경을 제공하려는 조직에 이상적입니다.

너비 우선 메서드는 새 연결을 허용하는 세션 호스트를 먼저 쿼리합니다. 그런 다음 메서드는 세션 수가 가장 적은 세션 호스트를 선택합니다. 동점이 있는 경우 메서드는 쿼리에서 첫 번째 세션 호스트를 선택합니다.

## <a name="depth-first-load-balancing-method"></a>깊이 우선 로드 밸런싱 방법

깊이 우선 로드 밸런싱 방법을 사용하면 한 번에 하나의 세션 호스트를 포화하여 이 시나리오에 최적화할 수 있습니다. 이 방법은 호스트 풀에 할당된 가상 시스템의 수를 보다 세밀한 제어를 원하는 비용 에 민감한 조직에 이상적입니다.

깊이 우선 메서드는 새 연결을 허용하고 최대 세션 제한을 초과하지 않은 세션 호스트를 쿼리합니다. 그런 다음 메서드는 세션 수가 가장 많은 세션 호스트를 선택합니다. 동점이 있는 경우 메서드는 쿼리에서 첫 번째 세션 호스트를 선택합니다.
