# 구성 표를 사용하여 크로스-프레미스 가상 네트워크 만들기

이 항목에서는 다음 구성 표 집합에서 이전에 지정한 설정을 사용하여 크로스-프레미스 가상 네트워크를 만드는 과정을 안내합니다.

- 표 V: 크로스-프레미스 가상 네트워크 구성
- 표 S: 가상 네트워크의 서브넷
- 표 D: 온-프레미스 DNS 서버
- 표 L: 로컬 네트워크의 주소 접두사

일반적으로 Azure의 IT 작업 구성에 대해 설명하며 크로스-프레미스 가상 네트워크를 포함하는 항목에서 이러한 표에 정보를 입력합니다. 예제는 [1단계: Azure 구성](virtual-machines-workload-intranet-sharepoint-phase1.md)을 참조하세요.

다음 절차에서는 이러한 표의 정보를 참조하여 가상 네트워크 구성 프로세스를 안내합니다. 다른 항목에서 이러한 표의 설정을 아직 지정하지 않은 상태로 크로스-프레미스 가상 네트워크를 구성하려는 경우 [Azure 가상 네트워크에 대한 프레미스 사이 사이트 간 연결 구성](https://msdn.microsoft.com/library/dn133795.aspx)을 참조하세요.

> [AZURE.NOTE]이 절차에서는 사이트 간 VPN 연결을 사용하는 가상 네트워크를 만드는 과정을 안내합니다. 사이트 간 연결에 대해 Express 경로를 사용하는 방법에 대한 자세한 내용은 [Express 경로 기술 개요](https://msdn.microsoft.com/library/dn606309.aspx)를 참조하세요.
 
## 구성 표 설정을 사용하여 새 크로스-프레미스 Azure 가상 네트워크 만들기

1. [Azure 관리 포털](https://manage.windowsazure.com/)에 로그인합니다.
2. 작업 표시줄에서 **새로 만들기 > 네트워크 서비스 > 가상 네트워크 > 사용자 지정 만들기**를 클릭합니다.
3. 가상 네트워크 정보 페이지에서 다음을 수행합니다.
- **이름**에 표 V의 항목 1에 해당하는 이름을 입력합니다.
- **위치**에서 표 V의 항목 2에 해당하는 지역을 선택합니다. 
4. 다음 화살표를 클릭하여 계속합니다.
5. DNS 서버 및 VPN 연결 페이지에서 다음을 수행합니다.
- **DNS 서버**에서 표 D의 각 항목에 대해 온-프레미스 DNS 서버의 이름과 IP 주소를 구성합니다.
- **사이트 간 연결**에서 **사이트 간 VPN 구성**을 선택합니다.
- 이미 구성한 로컬 네트워크를 사용한 경우 **로컬 네트워크**에서 해당 이름을 선택합니다. 새 로컬 네트워크를 만들려면 **로컬 네트워크**에서 **새 로컬 네트워크 지정**을 선택합니다.
- 구독의 로컬 네트워크를 아직 구성하지 않은 경우에는 **로컬 네트워크** 필드가 표시되지 않습니다. 
6. 다음 화살표를 클릭하여 계속합니다.
7. 5단계에서 **새 로컬 네트워크 지정**을 선택한 경우 사이트 간 연결 페이지에서 다음을 수행합니다.
- **이름**에 표 V의 항목 3에 해당하는 이름(로컬 네트워크 이름)을 입력합니다.
- **VPN 장치 IP 주소**에 표 V의 항목 4에 해당하는 주소를 입력합니다.
- **주소 공간**에서 표 L의 각 항목에 대해 조직 네트워크의 IP 주소 공간 접두사를 **시작 IP**에, 접두사 길이를 **CIDR(주소 개수)**에 입력합니다.
8. 다음 화살표를 클릭하여 계속합니다.
9. 가상 네트워크 주소 공간 페이지에서 다음을 수행합니다.
- **주소 공간**에서 표 V의 항목 5에 해당하는 가상 네트워크의 개인 IP 주소 공간 접두사를 **시작 IP**에, 접두사 길이를 **CIDR(주소 개수)**에 입력합니다.
- **서브넷**에서 표 S의 각 항목에 대해 다음을 수행합니다.
	- **서브넷** 열에 서브넷의 이름을 입력하고 필요한 경우 기본 이름을 덮어씁니다.
	- 서브넷의 개인 IP 주소 공간 접두사를 **시작 IP**에, 접두사 길이를 **CIDR(주소 개수)**에 입력합니다.
- **게이트웨이 서브넷 추가**를 클릭합니다.
10. 확인 표시를 클릭하여 구성을 완료합니다.
