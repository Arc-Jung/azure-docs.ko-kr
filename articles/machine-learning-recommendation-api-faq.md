## 권장 사항이란 무엇입니까?
고객에 대한 교차 판매 및 상향 판매의 권장 사항에 의존하는 조직과 비즈니스를 위한 Microsoft Azure 기계 학습 권장 사항은 Azure를 기반으로 한 셀프 서비스 권장 사항 엔진입니다. 이는 행렬 인수분해를 핵심 알고리즘으로 사용한 공동 작업 필터링 구현입니다. RESTful HTTP 인터페이스를 통해 API로 응용 프로그램 개발자에게 액세스할 수 있습니다. 권장 사항은 현재 공개 미리 보기 상태이며 일반 공급 서비스로 제공될 예정입니다.

## 권장 사항으로 무엇을 수행할 수 있습니까?
권장 사항은 항목 또는 항목 집합을 입력으로 이용하고 관련 권장 사항 목록을 반환합니다. 예를 들면 다음과 같습니다. 온라인 소매점의 고객이 제품을 클릭합니다. 온라인 소매점은 해당 제품을 권장 사항에 대한 입력으로 전송하고, 그 결과로 제품 목록을 가져오고, 이러한 제품 중 고객에게 표시할 제품을 결정합니다. 권장 사항을 사용하여 온라인 스토어를 최적화하거나 내부 판매 부서나 콜 센터에 정보를 제공할 수 있습니다.


## 출시 기간 동안 제한 사항이 있습니까?

* 구독당 최대 모델 수: 10
* 카탈로그에 포함할 수 있는 최대 항목 수: 100,000
* POST로 전송할 수 있는 최대 데이터 크기(예: 카탈로그 데이터 가져오기, 사용 데이터 가져오기)는 200MB입니다.
* 활성화되지 않은 권장 사항 모델에 대한 초당 트랜잭션 수는 최대 2TPS이고, 활성화된 권장 사항 모델 빌드에만 최대 20TPS를 포함할 수 있습니다.

## 구매 및 요금 청구 
구독에 대한 자세한 내용은 여기에서 찾을 수 있습니다.

###출시 기간 동안 권장 사항 비용은 얼마입니까?
권장 사항은 구독 기반 서비스입니다. 요금 청구는 월별 트랜잭션 볼륨에 따라 수행됩니다. 출시 기간 동안 서비스는 무료이고 월별 100, 000개 트랜잭션으로 제한됩니다.

###권장 사항 추적 및/또는 저장 사용자 활동을 포함하는 것과 관련된 비용이 있습니까?
현재는 없습니다.

###권장 사항에 무료 평가판이 있습니까?
출시 기간 동안 서비스는 무료이고 월별 100, 000개 트랜잭션으로 제한됩니다.

###출시 기간은 언제 종료됩니까?
출시 기간 및 해당 가격은 몇 개월 동안 지속될 예정입니다. 예정된 상태 및 가격 변경에 대해서는 구독 사용자에게 알려드립니다.

###일반 공급(GA) 상태가 되면 비용은 얼마가 듭니까?
일반 공급될 때의 권장 사항 가격은 아직 결정되지 않았습니다. 가격이 변경될 경우 구독자에게 알려드립니다.

###권장 사항에 대한 요금은 언제 청구됩니까?
유료 구독은 월별 요금이 있는 구독입니다. 유료 구독을 구매하면 첫째 달 사용에 대한 요금이 즉시 청구됩니다. 구독 페이지(및 해당 세금)에 대한 제안 옆에 표시된 금액으로 요금이 청구됩니다. 구독을 취소할 때까지 최초 구매와 같은 달력 날짜에 매월 이 월별 요금이 청구됩니다. 구독에 대한 자세한 내용을 보려면 여기를 클릭하세요.

###더 높은 계층 서비스로 업그레이드하려면 어떻게 해야 합니까?
출시 기간 동안 서비스는 월별 100,000개 트랜잭션으로 제한되고 더 높은 계층으로 업그레이드할 수 없습니다.

권장 사항이 일반 공급될 경우 - 구독이 충분한 트랜잭션을 제공하지 않는다고 판단되면 더 높은 트랜잭션 제한을 포함한 구독으로 업그레이드할 수 있습니다.

구독을 업그레이드할 경우:

* 이전 구독에 남아 있는 트랜잭션은 새 구독에 추가되지 않습니다. 
* 이전 구독에 사용되지 않은 트랜잭션이 있더라도 새 구독에 대한 전체 가격을 지급해야 합니다.

구독 업그레이드 프로세스

* 마켓플레이스에 로그인합니다.
* 내 데이터 탭을 클릭합니다.
* 업그레이드하려는 구독 뒤에서 사용을 클릭합니다. 
* 오른쪽 창에 사용할 수 있는 모든 구독이 회색으로 표시된 구독과 함께 나열됩니다. 업그레이드하려는 구독에 대한 라디오 단추를 클릭합니다.
* 업그레이드하려면 대화 상자에서 확인을 클릭합니다. 업그레이드하지 않으려면 취소를 클릭합니다.

중요: 요금 청구 및 사용에 따라 영향이 있으므로 업그레이드하기 전에 대화 상자를 주의깊게 읽어야 합니다.

구독에 대한 자세한 내용을 보려면 [여기](http://msdn.microsoft.com/library/gg312164.aspx)를 클릭하세요.

###권장 사항에 대한 내 구독은 언제 종료됩니까?
구독은 고객이 구독을 취소할 때 종료됩니다. 구독을 취소하려면 아래 지침을 참조하세요.

###권장 사항 구독을 취소하려면 어떻게 해야 합니까?
구독을 취소하려면 다음 단계를 수행합니다. 현재 구독이 유료 구독이면 구독은 현재 청구 기간이 끝날 때까지 계속 유효합니다. 취소를 즉시 적용하려면 [여기](https://support.microsoft.com/oas/default.aspx?gprid=17024&st=1&wfxredirect=1&sd=gn)에 문의하세요.

청구 기간이 끝나기 전에 취소하는 경우나 청구 기간에 사용되지 않은 트랜잭션에 대해서는 환불이 제공되지 않습니다.

* 마켓플레이스에 로그인합니다.
* 내 데이터 탭을 클릭합니다.
* 취소하려는 구독을 찾습니다.
* 데이터 집합 이름 및 상태의 오른쪽에서 취소를 클릭합니다. 현재 청구 기간이 끝나거나 트랜잭션 제한에 도달할 때 중 먼저 발생하는 시기까지 이 구독을 사용할 수 있습니다.

새 구독을 구입할 수 있도록 즉시 구독을 취소하려면 [여기](https://support.microsoft.com/oas/default.aspx?gprid=17024&st=1&wfxredirect=1&sd=gn)로 이동하여 고객 지원 센터에서 티켓을 발행하세요.


## 권장 사항 시작하기

###권장 사항이 고객에게 적합합니까? 
Microsoft Azure 기계 학습 권장 사항은 고객에 대한 교차 판매 및 상향 판매의 권장 사항에 의존하는 조직과 비즈니스에 적합합니다. 웹 사이트, 영업 인력, 내부 판매 인력 또는 콜 센터를 대하는 고객이 있는 경우 및 수십 개의 제품이나 서비스에 대한 카탈로그를 제공할 경우 - 최종적으로 권장 사항 소비를 활용할 수 있습니다. 권장 사항을 사용한 실험은 매우 간단하게 디자인되었습니다. 현재 API 기반 버전은 필수 기본 프로그래밍 기술을 수행하지 않습니다. 지원이 필요한 경우 웹사이트를 개발한 공급 업체에 문의하세요. 내부 IT 부서 또는 사내 개발자가 있는 경우 - 권장 사항을 작업에 사용할 수 있어야 합니다. 

###권장 사항을 설정하기 위한 필수 구성 요소는 무엇입니까?
권장 사항을 사용하려면 카탈로그와 관련된 사용자 선택 로그가 있어야 합니다. 해당 로그가 없고 웹 사이트를 대하는 고객이 있을 경우 권장 사항에서 대신 사용자 활동을 수집할 수 있습니다. 권장 사항에는 제품 또는 서비스에 대한 카탈로그도 필요합니다. 카탈로그가 없을 경우 권장 사항에서는 실제 고객 사용 데이터를 사용하고 카탈로그를 추출할 수 있습니다. "묵시적" 카탈로그에는 사용자 트랜잭션의 일부로 "보고"되지 않은 항목이 포함되지 않습니다.

###처음에 권장 사항을 설정하려면 어떻게 해야 합니까?
권장 사항을 구독하고 나서   [여기](https://onedrive.live.com/view.aspx?cid=8536718B52F71725&resid=8536718B52F71725!118&app=Word&authkey=!AOLb7aR6D0cVqMQ)에서 API 설명서를 사용하여 서비스를 설정해야 합니다.

###API 설명서는 어디서 찾을 수 있습니까? 
API 설명서는 [여기](https://onedrive.live.com/view.aspx?cid=8536718B52F71725&resid=8536718B52F71725!118&app=Word&authkey=!AOLb7aR6D0cVqMQ)에 있습니다.

###카탈로그 및 사용 데이터를 권장 사항에 업로드하는 데 사용할 수 있는 옵션은 무엇입니까?
카탈로그 및 사용 데이터를 업로드하기 위한 두 가지 옵션은 다음과 같습니다. CRM 시스템 또는 기타 로그에서 이러한 데이터를 내보내고 권장 사항에 업로드하거나, 사용자 활동을 추적할 웹 사이트에 태그를 추가할 수 있습니다. 후자의 경우 데이터는 Azure에 저장됩니다.

##유지 관리 및 지원

###내 데이터 집합은 얼마나 확장할 수 있습니까?
각 데이터 집합은 카탈로그 항목 최대 100,000 개와 사용 데이터 최대 2048MB를 포함할 수 있습니다.
또한 구독은 데이터 집합(모델) 최대 10 개를 포함할 수 있습니다.

기술 지원은 [여기](https://social.msdn.microsoft.com/forums/azure/en-US/home?forum=MachineLearning)에서 사용할 수 있습니다.
