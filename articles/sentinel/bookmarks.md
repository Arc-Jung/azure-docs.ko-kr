# <a name="keep-track-of-data-during-hunting-with-azure-sentinel"></a>Azure Sentinel을 사용하여 검색하는 동안 데이터 추적

위협 검색은 일반적으로 악의적인 동작의 증거를 찾는 로그 데이터의 산맥을 검토해야 합니다. 이 과정에서 조사관은 잠재적인 가설을 검증하고 타협의 전체 이야기를 이해하는 일환으로 기억하고, 다시 방문하고, 분석하고자 하는 이벤트를 찾습니다.

Azure Sentinel에서 책갈피를 검색하면 **Azure Sentinel - Log에서**실행한 쿼리와 관련이 있다고 판단되는 쿼리 결과를 보존하여 이 작업을 수행할 수 있습니다. 참고 문맥 관찰을 기록하고 메모와 태그를 추가하여 결과를 참조할 수도 있습니다. 북마크된 데이터는 쉽게 공동 작업을 위해 귀하와 팀원에게 표시됩니다.

**사냥** 창의 **북마크** 탭에서 언제든지 북마크데이터를 다시 방문할 수 있습니다. 필터링 및 검색 옵션을 사용하여 현재 조사에 대한 특정 데이터를 빠르게 찾을 수 있습니다. 또는 Log Analytics 작업 영역의 **HuntingBookmark** 테이블에서 북마크된 데이터를 직접 볼 수 있습니다. 예를 들어:

> [!div class="mx-imgBorder"]
> ![헌팅북마크 테이블 보기](./media/bookmarks/bookmark-table.png)

표에서 책갈피를 확인하면 북마크가 지정된 데이터를 다른 데이터 원본과 필터링, 요약 및 조인할 수 있으므로 확증적인 증거를 쉽게 찾을 수 있습니다.

현재 미리 보기에서 몇 번의 클릭만으로 로그를 검색하는 동안 긴급하게 해결해야 하는 문제를 발견하면 몇 번의 클릭만으로 책갈피를 만들고 인시던트에 승격하거나 기존 인시던트에 책갈피를 추가할 수 있습니다. 인시던트에 대한 자세한 내용은 [자습서: Azure Sentinel을 통해 인시던트 조사](tutorial-investigate-cases.md)를 참조하십시오. 

또한 미리 보기에서 책갈피 세부 정보에서 **조사를** 클릭하여 북마크된 데이터를 시각화할 수 있습니다. 그러면 대화형 엔터티 그래프 다이어그램 및 타임라인을 사용하여 결과를 보고 조사하고 시각적으로 전달할 수 있는 조사 환경이 시작됩니다.

## <a name="add-a-bookmark"></a>책갈피 추가

1. Azure 포털에서 **Sentinel** > **위협 관리** > **사냥으로** 이동하여 의심스럽고 비정상적인 동작에 대한 쿼리를 실행합니다.

2. 사냥 쿼리 중 하나를 선택하고 오른쪽에 있는 검색 쿼리 세부 정보에서 **쿼리 실행을**선택합니다. 

3. **쿼리 결과 보기를 선택합니다.** 예를 들어:
    
    > [!div class="mx-imgBorder"]
    > ![Azure Sentinel 사냥의 쿼리 결과 보기](./media/bookmarks/new-processes-observed-example.png)
    
    이 작업을 수행하면 **로그** 창에서 쿼리 결과가 열립니다.

4. 로그 쿼리 결과 목록에서 확인란을 사용하여 흥미로운 정보를 포함하는 하나 이상의 행을 선택합니다.

5. **책갈피 추가**선택:
    
    > [!div class="mx-imgBorder"]
    > ![쿼리에 사냥 책갈피 추가](./media/bookmarks/add-hunting-bookmark.png)

6. 오른쪽에 있는 **책갈피 추가** 창에서 선택적으로 책갈피 이름을 업데이트하고 태그를 추가하며 항목에 대해 흥미로운 항목을 식별하는 데 도움이 되는 메모를 추가합니다.

7. 쿼리 **정보** 섹션에서 드롭다운 상자를 사용하여 **계정,** **호스트**및 **IP 주소** 엔터티 유형에 대한 쿼리 결과에서 정보를 추출합니다. 이 작업은 선택한 엔터티 유형을 쿼리 결과의 특정 열에 매핑합니다. 예를 들어:
    
    > [!div class="mx-imgBorder"]
    > ![책갈피를 사냥하기 위한 엔터티 유형 매핑](./media/bookmarks/map-entity-types-bookmark.png)
    
    조사 그래프(현재 미리 보기)에서 책갈피를 보려면 **계정,** **호스트**또는 **IP 주소인**하나 이상의 엔터티 유형을 매핑해야 합니다. 

5. **저장을** 클릭하여 변경 내용을 커밋하고 책갈피를 추가합니다. 북마크된 모든 데이터는 다른 조사자와 공유되며 공동 조사 환경을 위한 첫 번째 단계입니다.

 
> [!NOTE]
> 로그 쿼리 결과는 Azure Sentinel에서 이 창을 열 때마다 책갈피를 지원합니다. 예를 들어 탐색 모음에서 **일반** > **로그를** 선택하거나 조사 그래프에서 이벤트 링크를 선택하거나 인시던트에 대한 전체 세부 정보(현재 미리 보기)에서 경고 ID를 선택합니다. Azure Monitor에서 직접 같은 다른 위치에서 **로그** 창을 열때 책갈피를 만들 수 없습니다.

## <a name="view-and-update-bookmarks"></a>책갈피 보기 및 업데이트 

1. Azure 포털에서 **Sentinel** > **위협 관리** > **사냥으로**이동합니다. 

2. **북마크** 탭을 선택하여 북마크 목록을 봅니다.

3. 특정 책갈피를 찾으려면 검색 상자 또는 필터 옵션을 사용합니다.

4. 개별 책갈피를 선택하고 오른쪽 세부 정보 창에서 책갈피 세부 정보를 봅니다.

5. 필요에 따라 변경하면 자동으로 저장됩니다.

## <a name="exploring-bookmarks-in-the-investigation-graph"></a>조사 그래프에서 책갈피 탐색

> [!IMPORTANT]
> 조사 그래프와 조사 그래프에서 책갈피를 탐색하는 것은 현재 공개 미리 보기입니다.
> 이러한 기능은 서비스 수준 계약 없이 제공되며 프로덕션 워크로드에는 권장되지 않습니다.
> 자세한 내용은 [Microsoft Azure Preview에 대한 추가 사용 약관](https://azure.microsoft.com/support/legal/preview-supplemental-terms/)을 참조하세요.

1. Azure 포털에서 **Sentinel** > **위협 관리** > **찾기** > **책갈피** 탭으로 이동하여 조사할 책갈피 또는 책갈피를 선택합니다.

2. 책갈피 세부 정보에서 하나 이상의 엔터티가 매핑되었는지 확인합니다. 예를 들어 **ENTITIES의**경우 **IP,** **컴퓨터**또는 **계정**에 대한 항목이 표시됩니다.

3. 조사 그래프에서 책갈피를 보려면 **조사를** 클릭합니다.

조사 그래프를 사용하는 지침은 [조사 그래프를 사용하여 자세히 알아봅을 참조하세요.](tutorial-investigate-cases.md#use-the-investigation-graph-to-deep-dive)

## <a name="add-bookmarks-to-a-new-or-existing-incident"></a>새 인시던트 또는 기존 인시던트에 북마크 추가

> [!IMPORTANT]
> 새 인시던트 또는 기존 인시던트에 책갈피를 추가하는 것은 현재 공개 미리 보기입니다.
> 이 기능은 서비스 수준 계약 없이 제공되며 프로덕션 워크로드에는 권장되지 않습니다.
> 자세한 내용은 [Microsoft Azure Preview에 대한 추가 사용 약관](https://azure.microsoft.com/support/legal/preview-supplemental-terms/)을 참조하세요.

1. Azure 포털에서 **Sentinel** > **위협 관리** > **찾기** > **책갈피** 탭으로 이동하여 인시던트에 추가할 책갈피 또는 책갈피를 선택합니다.

2. 명령 모음에서 **인시던트 작업(미리 보기)을** 선택합니다.
    
    > [!div class="mx-imgBorder"]
    > ![인시던트에 북마크 추가](./media/bookmarks/incident-actions.png)

3. 필요에 따라 **새 인시던트 만들기** 또는 **기존 인시던트에 추가**중 하나를 선택합니다. 그렇다면
    
    - 새 인시던트: 인시던트에 대한 세부 정보를 선택적으로 업데이트한 다음 **만들기를**선택합니다.
    - 기존 인시던트에 책갈피를 추가하려면 인시던트 하나를 선택한 다음 **추가를**선택합니다. 

인시던트 내의 책갈피를 보려면: **센티넬** > **위협 관리** > **인시던트로** 이동하여 책갈피를 사용하여 인시던트를 선택합니다. **전체 세부 정보 보기를**선택한 다음 **북마크** 탭을 선택합니다.

> [!TIP]
> 명령 모음의 **인시던트 작업(미리 보기)** 옵션대신 하나 이상의 북마크에 대한 컨텍스트**메뉴(...**)를 사용하여 **새 인시던트 만들기,** **기존 인시던트 에 추가**및 **인시던트에서 제거**옵션을 선택할 수 있습니다. 

## <a name="view-bookmarked-data-in-logs"></a>로그에서 북마크된 데이터 보기

북마크된 쿼리, 결과 또는 기록을 보려면 **찾기** > **책갈피** 탭에서 책갈피를 선택하고 세부 정보 창에 제공된 링크를 사용합니다. 

- **원본 쿼리를 보고** Logs 창에서 원본 **쿼리를 봅니다.**

- **북마크 로그를 보고** 업데이트한 사람, 업데이트된 값 및 업데이트가 발생한 시간을 포함하는 모든 책갈피 메타데이터를 확인합니다.

또한 **사냥** > 북마크 탭의 명령 모음에서 **북마크 로그를** 선택하여 모든**북마크에** 대한 원시 북마크 데이터를 볼 수도 있습니다.

> [!div class="mx-imgBorder"]
> ![북마크 로그](./media/bookmarks/bookmark-logs.png)

이 보기에는 연결된 메타데이터가 있는 모든 책갈피를 표시합니다. KQL(키워드 [쿼리 언어)](https://docs.microsoft.com/sharepoint/dev/general-development/keyword-query-language-kql-syntax-reference) 쿼리 쿼리를 사용하여 찾고 있는 특정 책갈피의 최신 버전으로 필터링할 수 있습니다.

> [!NOTE]
> 책갈피를 만드는 시간과 **책갈피** 탭에 표시되는 시간 사이에 상당한 지연(분 단위로 측정)이 있을 수 있습니다.

## <a name="delete-a-bookmark"></a>책갈피 삭제
 
1.  Azure 포털에서 **Sentinel** > **위협 관리** > **찾기** > **책갈피** 탭으로 이동하여 삭제할 책갈피 또는 책갈피를 선택합니다. 

2. 선택 항목을 마우스 오른쪽 단추로 클릭하고 책갈피 또는 책갈피를 삭제하는 옵션을 선택합니다. 예를 들어 책갈피를 하나만 선택한 경우 **책갈피를 삭제하고** 두 개의 책갈피를 선택한 경우 **책갈피를 2개 삭제합니다.**
    
책갈피를 삭제시키면 북마크 탭의 목록에서 **책갈피를** 제거합니다. Log Analytics 작업 영역에 대한 **HuntingBookmark** 테이블에는 이전 북마크 항목이 계속 포함되지만 최신 항목은 **SoftDelete** 값을 true로 변경하여 이전 책갈피를 쉽게 필터링할 수 있습니다. 책갈피를 삭제해도 다른 책갈피 또는 경고와 연결된 조사 경험에서 엔터티가 제거되지는 않습니다.
