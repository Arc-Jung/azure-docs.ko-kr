# <a name="keep-track-of-data-during-hunting-with-azure-sentinel"></a>Azure 센티널을 사용 하 여 데이터 추적 유지

위협 검색은 일반적으로 악의적인 동작의 증거를 찾는 로그 데이터의 산을 검토 해야 합니다. 이 과정에서 잠재적 가설의 유효성을 검사 하 고 손상에 대 한 전체 스토리를 이해 하는 과정에서 기억할 수 있는 이벤트를 사관, 다시 방문 하 고, 분석할 이벤트를 찾을 수 있습니다.

Azure 센티널의 구하기 책갈피를 사용 하면 **Azure 센티널**에서 실행 한 쿼리를 유지 하 여 관련 된 쿼리 결과와 함께이 작업을 수행할 수 있습니다. 메모 및 태그를 추가 하 여 상황에 맞는 관찰을 기록 하 고 검색 결과를 참조할 수도 있습니다. 책갈피가 설정 된 데이터는 쉽게 공동 작업을 위해 사용자와 동료가 볼 수 있습니다.

언제 든 지 **조합 창의** **책갈피** 탭에서 책갈피 데이터를 다시 선택할 수 있습니다. 필터링 및 검색 옵션을 사용 하 여 현재 조사에 대 한 특정 데이터를 신속 하 게 찾을 수 있습니다. 또는 Log Analytics 작업 영역의 **HuntingBookmark** 테이블에서 직접 책갈피가 설정 된 데이터를 볼 수 있습니다. 예를 들어:

> [!div class="mx-imgBorder"]
> ![HuntingBookmark 테이블 보기](./media/bookmarks/bookmark-table.png)

테이블에서 책갈피를 보면 책갈피를 사용 하는 데이터를 다른 데이터 원본과 필터링, 요약 및 조인 하 여 corroborating 증명 정보를 쉽게 찾을 수 있습니다.

현재 미리 보기 상태에서 로그를 검색 하는 동안 급하게에 해결 해야 하는 항목이 있는 경우 두 번의 클릭으로 책갈피를 만들어 인시던트에 승격 하거나 기존 인시던트에 책갈피를 추가할 수 있습니다. 인시던트에 대 한 자세한 내용은 [자습서: Azure 센티널로 인시던트 조사](tutorial-investigate-cases.md)를 참조 하세요. 

또한 미리 보기에서 책갈피 세부 정보에서 **조사** 를 클릭 하 여 책갈피가 설정 된 데이터를 시각화할 수 있습니다. 그러면 대화형 엔터티-그래프 다이어그램과 타임 라인을 사용 하 여 결과를 확인, 조사 및 시각적으로 전달할 수 있는 조사 환경이 시작 됩니다.

## <a name="add-a-bookmark"></a>책갈피 추가

1. Azure Portal에서 **센티널**  >  **Threat management**  >  **구하기** 로 이동 하 여 의심 스러운 비정상 동작에 대 한 쿼리를 실행 합니다.

2. 구하기 쿼리 중 하나를 선택 하 고 오른쪽의 구하기 쿼리 세부 정보에서 **쿼리 실행**을 선택 합니다. 

3. **쿼리 결과 보기**를 선택 합니다. 예를 들어:
    
    > [!div class="mx-imgBorder"]
    > ![Azure 센티널 구하기에서 쿼리 결과 보기](./media/bookmarks/new-processes-observed-example.png)
    
    이 작업을 수행 하면 **로그** 창에 쿼리 결과가 열립니다.

4. 쿼리 결과 로그 목록에서 확인란을 사용 하 여 관심 있는 정보를 포함 하는 행을 하나 이상 선택 합니다.

5. **책갈피 추가**를 선택 합니다.
    
    > [!div class="mx-imgBorder"]
    > ![쿼리에 사냥 책갈피 추가](./media/bookmarks/add-hunting-bookmark.png)

6. 오른쪽의 **책갈피 추가** 창에서 필요에 따라 책갈피 이름, 태그 추가 및 메모를 업데이트 하 여 항목에 대해 흥미로운 사항을 식별할 수 있습니다.

7. **쿼리 정보** 섹션에서 드롭다운 상자를 사용 하 여 **계정**, **호스트**및 **IP 주소** 엔터티 형식에 대 한 쿼리 결과에서 정보를 추출 합니다. 이 작업은 선택한 엔터티 형식을 쿼리 결과의 특정 열에 매핑합니다. 예를 들어:
    
    > [!div class="mx-imgBorder"]
    > ![구하기 책갈피에 대 한 엔터티 형식 매핑](./media/bookmarks/map-entity-types-bookmark.png)
    
    조사 그래프 (현재 미리 보기 상태)에서 책갈피를 보려면 **계정**, **호스트**또는 **IP 주소**중 하나 이상의 엔터티 형식을 매핑해야 합니다. 

5. **저장** 을 클릭 하 여 변경 내용을 커밋하고 책갈피를 추가 합니다. 모든 책갈피가 설정 된 데이터는 다른 사관 공유 되며 공동 작업 조사 환경을 위한 첫 번째 단계입니다.

 
> [!NOTE]
> 로그 쿼리 결과는이 창이 Azure 센티널에서 열릴 때마다 책갈피를 지원 합니다. 예를 들어 **General**  >  탐색 모음에서 일반**로그** 를 선택 하거나, 조사 그래프에서 이벤트 링크를 선택 하거나, 인시던트의 전체 세부 정보 (현재 미리 보기 상태)에서 경고 ID를 선택 합니다. 다른 위치에서 **로그** 창을 열 때 (예: Azure Monitor에서 직접) 책갈피를 만들 수 없습니다.

## <a name="view-and-update-bookmarks"></a>책갈피 보기 및 업데이트 

1. Azure Portal에서 **센티널**  >  **Threat management**  >  **구하기**로 이동 합니다. 

2. 책갈피 목록을 보려면 **책갈피** 탭을 선택 합니다.

3. 특정 책갈피를 찾으려면 검색 상자 또는 필터 옵션을 사용 합니다.

4. 개별 책갈피를 선택 하 고 오른쪽 세부 정보 창에서 책갈피 정보를 확인 합니다.

5. 필요에 따라 변경 하 여 자동으로 저장 합니다.

## <a name="exploring-bookmarks-in-the-investigation-graph"></a>조사 그래프에서 책갈피 탐색

> [!IMPORTANT]
> 조사 그래프에서 책갈피를 탐색 하는 것은 현재 공개 미리 보기로 제공 됩니다.
> 이러한 기능은 서비스 수준 계약 없이 제공 되며 프로덕션 워크 로드에는 권장 되지 않습니다.
> 자세한 내용은 [Microsoft Azure Preview에 대한 추가 사용 약관](https://azure.microsoft.com/support/legal/preview-supplemental-terms/)을 참조하세요.

1. Azure Portal에서 **센티널**  >  **Threat management**  >  **구하기**  >  **책갈피** 탭으로 이동 하 고 조사 하려는 책갈피 또는 책갈피를 선택 합니다.

2. 책갈피 세부 정보에서 하나 이상의 엔터티가 매핑 되었는지 확인 합니다. 예를 들어 **엔터티의**경우 **IP**, **컴퓨터**또는 **계정**에 대 한 항목을 볼 수 있습니다.

3. 조사 그래프에서 책갈피를 보려면 **조사** 를 클릭 합니다.

조사 그래프를 사용 하는 방법에 대 한 지침은 [조사 그래프를 사용 하 여 자세히](tutorial-investigate-cases.md#use-the-investigation-graph-to-deep-dive)알아보기를 참조 하세요.

## <a name="add-bookmarks-to-a-new-or-existing-incident"></a>새 인시던트 또는 기존 인시던트에 책갈피 추가

> [!IMPORTANT]
> 새 인시던트 또는 기존 인시던트에 책갈피를 추가 하는 것은 현재 공개 미리 보기 상태입니다.
> 이 기능은 서비스 수준 계약 없이 제공 되며 프로덕션 워크 로드에는 권장 되지 않습니다.
> 자세한 내용은 [Microsoft Azure Preview에 대한 추가 사용 약관](https://azure.microsoft.com/support/legal/preview-supplemental-terms/)을 참조하세요.

1. Azure Portal에서 **센티널**  >  **Threat management**  >  **구하기**  >  **책갈피** 탭으로 이동 하 고 인시던트에 추가 하려는 책갈피 또는 책갈피를 선택 합니다.

2. 명령 모음에서 **인시던트 작업 (미리 보기)** 을 선택 합니다.
    
    > [!div class="mx-imgBorder"]
    > ![인시던트에 책갈피 추가](./media/bookmarks/incident-actions.png)

3. 필요에 따라 **새 인시던트를 만들거나** **기존 인시던트에 추가**중 하나를 선택 합니다. 그런 다음:
    
    - 새 인시던트: 필요에 따라 인시던트의 세부 정보를 업데이트 한 후 **만들기**를 선택 합니다.
    - 기존 인시던트에 책갈피를 추가 하려면 인시던트 하나를 선택 하 고 **추가**를 선택 합니다. 

인시던트 내에서 책갈피를 보려면 **센티널**  >  **위협 관리**  >  **인시던트** 로 이동 하 고 책갈피로 해당 인시던트를 선택 합니다. **전체 세부 정보 보기**를 선택한 다음 **책갈피** 탭을 선택 합니다.

> [!TIP]
> 명령 모음의 **인시던트 작업 (미리 보기)** 옵션에 대 한 대 안으로, 하나 이상의 책갈피에 대 한 상황에 맞는 메뉴 (**...**)를 사용 하 여 **새 인시던트를 만들고**, **기존 인시던트에 추가**하 고, **인시던트에 제거할**수 있는 옵션을 선택할 수 있습니다. 

## <a name="view-bookmarked-data-in-logs"></a>로그에서 책갈피 데이터 보기

책갈피를 사용 하는 쿼리, 결과 또는 기록을 보려면 **사냥**책갈피 탭에서 책갈피를 선택 하  >  **Bookmarks** 고 세부 정보 창에 제공 된 링크를 사용 합니다. 

- **원본 쿼리** 를 보고 **로그** 창에서 원본 쿼리를 볼 수 있습니다.

- **책갈피 로그를 확인** 하 여 모든 책갈피 메타 데이터를 표시 합니다. 여기에는 업데이트 한 사람, 업데이트 된 값 및 업데이트 발생 시간이 포함 됩니다.

또한 사용자는 [ **사냥**책갈피] 탭의 명령 모음에서 **책갈피 로그** 를 선택 하 여 모든 책갈피에 대 한 원시 책갈피 데이터를 볼 수 있습니다  >  **Bookmarks** .

> [!div class="mx-imgBorder"]
> ![책갈피 로그](./media/bookmarks/bookmark-logs.png)

이 보기에는 연결 된 메타 데이터가 있는 모든 책갈피가 표시 됩니다. KQL ( [키워드 쿼리 언어](https://docs.microsoft.com/sharepoint/dev/general-development/keyword-query-language-kql-syntax-reference) ) 쿼리를 사용 하 여 찾으려는 특정 책갈피의 최신 버전으로 필터링 할 수 있습니다.

> [!NOTE]
> 책갈피를 만드는 시간과 **책갈피 탭에** 표시 되는 시간 사이에 상당한 지연 (분 단위)이 있을 수 있습니다.

## <a name="delete-a-bookmark"></a>책갈피 삭제
 
1.  Azure Portal에서 **센티널**  >  **Threat management**  >  **구하기**  >  **책갈피** 탭으로 이동 하 고 삭제할 책갈피 또는 책갈피를 선택 합니다. 

2. 선택 항목을 마우스 오른쪽 단추로 클릭 하 고 책갈피 또는 책갈피를 삭제 하는 옵션을 선택 합니다. 예를 들어 책갈피를 하나만 선택한 경우 **책갈피를 삭제** 하 고 두 책갈피를 선택한 경우 책갈피 **2** 개를 삭제 합니다.
    
책갈피를 삭제 하면 **책갈피 탭의** 목록에서 책갈피가 제거 됩니다. Log Analytics 작업 영역에 대 한 **HuntingBookmark** 테이블에는 이전 책갈피 항목이 계속 포함 되지만 최신 항목은 **소프트 삭제** 값을 true로 변경 하 여 이전 책갈피를 쉽게 필터링 할 수 있도록 합니다. 책갈피를 삭제 해도 다른 책갈피 또는 경고와 관련 된 조사 환경에서 엔터티가 제거 되지 않습니다.
