# <a name="advanced-multistage-attack-detection-in-azure-sentinel"></a>Azure 센티넬의 고급 다단계 공격 탐지


> [!IMPORTANT]
> Azure Sentinel의 일부 Fusion 기능은 현재 공개 미리 보기 상태입니다.
> 이러한 기능은 서비스 수준 계약 없이 제공되며 프로덕션 워크로드에는 권장되지 않습니다. 특정 기능이 지원되지 않거나 기능이 제한될 수 있습니다. 자세한 내용은 [Microsoft Azure Preview에 대한 추가 사용 약관](https://azure.microsoft.com/support/legal/preview-supplemental-terms/)을 참조하세요.



Azure Sentinel은 기계 학습을 기반으로 하는 Fusion 기술을 사용하여 킬 체인의 여러 단계에서 관찰되는 비정상적인 동작과 의심스러운 활동을 결합하여 다단계 공격을 자동으로 탐지할 수 있습니다. 그런 다음 Azure Sentinel은 잡기가 매우 어려운 인시던트를 생성합니다. 이러한 인시던트는 두 개 이상의 경고 또는 활동을 가두는 것입니다. 의도적으로 이러한 인시던트는 볼륨이 낮고 충실도가 높으며 심각도가 높습니다.

사용자 환경에 맞게 사용자 지정된 이 검색은 거짓 긍정 률을 줄일 뿐만 아니라 제한적이거나 누락된 정보로 공격을 감지할 수도 있습니다.

## <a name="configuration-for-advanced-multistage-attack-detection"></a>고급 다단계 공격 검색 구성

이 검색은 Azure Sentinel에서 기본적으로 활성화됩니다. 상태를 확인하거나 다른 솔루션을 사용하여 여러 경고를 기반으로 인시던트를 만들수 있으므로 비활성화하려면 다음 지침을 사용합니다.

1. 아직 로그인하지 않은 경우 [Azure 포털](https://portal.azure.com)에 로그인합니다.

2. Azure **센티넬** > **구성** > **분석으로** 이동

3. **활성 규칙을** 선택하고 **이름** 열에서 **고급 다단계 공격 탐지를** 찾습니다. **STATUS** 열을 확인하여 이 검색이 활성화되어 있는지 또는 비활성화되어 있는지 확인합니다.

4. 상태를 변경하려면 이 항목을 선택하고 **고급 다단계 공격 탐지** 블레이드에서 **편집을**선택합니다.

5. 규칙 **만들기 마법사** 블레이드에서 상태 변경이 자동으로 선택되므로 **다음: 검토**를 선택한 다음 저장 을 **선택합니다.** 

규칙 템플릿은 고급 다단계 공격 탐지에는 적용되지 않습니다.

> [!NOTE]
> Azure Sentinel은 현재 30일간의 기록 데이터를 사용하여 기계 학습 시스템을 학습합니다. 이 데이터는 기계 학습 파이프라인을 통과할 때 Microsoft의 키를 사용하여 항상 암호화됩니다. 그러나 Azure Sentinel 작업 영역에서 CMK를 사용하도록 설정한 경우 [CMK(고객 관리 키)를](customer-managed-keys.md) 사용하여 학습 데이터가 암호화되지 않습니다. Fusion을 옵트아웃하려면 **Azure Sentinel** \> **구성** \> **분석 \> \> 활성 규칙 고급 다단계 공격 감지** 및 **상태** 열에서 **비활성화를 선택합니다.**

## <a name="fusion-using-palo-alto-networks-and-microsoft-defender-atp"></a>팔로 알토 네트워크와 마이크로 소프트 수비수 ATP를 사용하여 융합
