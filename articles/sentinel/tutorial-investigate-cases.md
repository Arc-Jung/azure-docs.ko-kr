# <a name="tutorial-investigate-incidents-with-azure-sentinel-preview"></a>자습서: Azure 센티널 Preview를 사용 하 여 인시던트 조사

> [!IMPORTANT]
> Azure Sentinel은 현재 공개 미리 보기로 제공됩니다.
> 이 미리 보기 버전은 서비스 수준 계약 없이 제공되며 프로덕션 워크로드에는 사용하지 않는 것이 좋습니다. 특정 기능이 지원되지 않거나 기능이 제한될 수 있습니다. 자세한 내용은 [Microsoft Azure Preview에 대한 추가 사용 약관](https://azure.microsoft.com/support/legal/preview-supplemental-terms/)을 참조하세요.

이 자습서는 Azure 센티널로 위협을 검색 하는 데 도움이 됩니다.

[데이터 원본을](quickstart-onboard.md) Azure 센티널에 연결 하 고 나면 의심 스러운 상황이 발생 하면 알림이 표시 됩니다. 이 작업을 수행할 수 있도록 Azure 센티널을 사용 하면 사용자 환경에서 변칙 및 위협을 깊게 조사 하기 위해 할당 하 고 사용할 수 있는 인시던트를 생성 하는 고급 경고 규칙을 만들 수 있습니다. 

> [!div class="checklist"]
> * 인시던트 만들기
> * 인시던트 조사
> * 위협에 대응

## <a name="investigate-incidents"></a>인시던트 조사

인시던트는 여러 경고를 포함할 수 있습니다. 특정 조사에 대 한 모든 관련 증거의 집계입니다. **분석** 페이지에서 정의한 경고에 따라 인시던트가 생성 됩니다. 심각도 및 상태와 같은 경고와 관련 된 속성은 인시던트 수준에서 설정 됩니다. Azure 센티널에서 찾고 있는 위협 종류와이를 찾는 방법을 알 수 있게 되 면 인시던트를 조사 하 여 검색 된 위협을 모니터링할 수 있습니다. 

1. **인시던트**를 선택 합니다. **인시던트** 페이지를 사용 하면 보유 한 인시던트 수, 열려 있는 수, **진행**중인 것으로 설정 된 수 및 닫힌 횟수를 알 수 있습니다. 각 인시던트에 대해 발생 한 시간 및 인시던트의 상태를 확인할 수 있습니다. 심각도를 확인 하 여 먼저 처리할 항목을 결정 합니다. 인시던트 페이지 에서 **경고** 탭을 클릭 하 여 인시던트에 관련 된 모든 경고를 표시 합니다. 이전에 인시던트의 일부로 매핑한 엔터티는 **엔터티** 탭에서 볼 수 있습니다.  상태 또는 심각도와 같이 필요에 따라 인시던트를 필터링 할 수 있습니다. **인시던트** 탭을 살펴보면 **분석**에 정의 된 검색 규칙에 의해 트리거되는 경고를 포함 하는 미해결 인시던트가 표시 됩니다. 위쪽에서 활성 인시던트, 새 인시던트 및 진행 중인 인시던트를 확인할 수 있습니다. 심각도 별로 모든 인시던트에 대 한 개요를 볼 수도 있습니다.

   ![경고 대시보드](./media/tutorial-investigate-cases/cases.png)

2. 조사를 시작 하려면 특정 인시던트를 클릭 합니다. 오른쪽의 심각도, 관련 엔터티 수 (매핑 기반)의 요약을 포함 하 여 인시던트에 대 한 자세한 정보를 볼 수 있습니다. 각 인시던트에는 고유한 ID가 있습니다. 인시던트의 심각도는 인시던트에 포함 된 가장 심각한 경고에 따라 결정 됩니다.  

1. 인시던트의 경고 및 엔터티에 대 한 자세한 내용을 보려면 인시던트 페이지에서 **전체 세부 정보 보기** 를 클릭 하 고 인시던트 정보를 요약 하는 관련 탭을 검토 합니다.  전체 인시던트 보기는 경고의 모든 증거, 연결 된 경고 및 엔터티를 통합 합니다.

1. 경고 탭 에서 경고 자체 (트리거된 경우) 및 설정 된 임계값을 초과 하는 정도를 검토 합니다. 경고에 대 한 모든 관련 정보 (경고를 트리거한 쿼리, 쿼리당 반환 된 결과 수 및 경고에 대 한 플레이 북을 실행 하는 기능)를 볼 수 있습니다. 인시던트를 더 자세히 드릴 다운 하려면 적중 횟수를 클릭 합니다. 그러면 결과를 생성 한 쿼리와 Log Analytics 경고를 트리거한 결과가 열립니다.

3. **엔터티** 탭에서 경고 규칙 정의의 일부로 매핑한 모든 엔터티를 볼 수 있습니다. 

4. 인시던트를 적극적으로 조사 하는 경우에는 종료할 때까지 인시던트 상태를 **진행** 중으로 설정 하는 것이 좋습니다. 인시던트를 닫을 수도 있습니다. 여기에서 **닫힘 해결** 됨은 인시던트가 처리 되었음을 나타내는 인시던트의 상태이 고, **닫힌** 해제는 처리가 필요 하지 않은 인시던트의 상태입니다. 인시던트를 종결 하는 이유를 설명 하는 설명이 필요 합니다.

5. 인시던트를 특정 사용자에 게 할당할 수 있습니다. 각 인시던트에 대해 인시던트 **소유자** 필드를 설정 하 여 소유자를 할당할 수 있습니다. 모든 인시던트는 할당 되지 않음으로 시작 합니다. 인시던트를 확인 하 고 이름을 기준으로 필터링 하 여 소유 하 고 있는 모든 인시던트를 확인할 수 있습니다. 

5. 조사 맵을 확인 하려면 **조사** 를 클릭 하 고 수정 단계를 통해 위반 범위를 확인 합니다. 



## <a name="respond-to-threats"></a>위협에 대응

Azure 센티널은 플레이 북을 사용 하 여 위협에 대응 하기 위한 두 가지 기본 옵션을 제공 합니다. 경고가 트리거될 때 플레이 북가 자동으로 실행 되도록 설정 하거나 경고에 대 한 응답으로 플레이 북를 수동으로 실행할 수 있습니다.

- 플레이 북를 구성할 때 경고가 트리거될 때 플레이 북가 자동으로 실행 되도록 설정할 수 있습니다. 

- 플레이 **북 보기** 를 클릭 한 다음 실행할 플레이 북를 선택 하 여 경고 내에서 플레이 북를 수동으로 실행할 수 있습니다.
