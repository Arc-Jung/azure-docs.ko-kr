# <a name="connect-windows-security-events"></a>Windows 보안 이벤트 연결 

보안 이벤트 커넥터를 사용하면 Windows 시스템(서버 및 워크스테이션, 물리적 및 가상)의 모든 보안 이벤트를 Azure Sentinel 작업 영역으로 스트리밍할 수 있습니다. 이를 통해 대시보드에서 Windows 보안 이벤트를 보고, 사용자 지정 경고를 만드는 데 사용하고, 이를 사용하여 조사를 개선하여 조직의 네트워크에 대한 더 많은 통찰력을 제공하고 보안 작업을 확장할 수 있습니다. 기능. 다음 집합 중에서 스트리밍할 이벤트를 선택할 수 있습니다.<a name="event-sets"></a>

- **모든 이벤트** - 모든 Windows 보안 및 AppLocker 이벤트.
- **공통** - 감사를 위한 표준 이벤트 집합입니다. 전체 사용자 감사 추적이 이 집합에 포함됩니다. 예를 들어 사용자 로그인 및 사용자 로그아웃 이벤트(이벤트 번호 4624, 4634)를 모두 포함합니다. 또한 보안 그룹 변경, 키 도메인 컨트롤러 Kerberos 작업 및 허용된 모범 사례에 따라 다른 유형의 이벤트와 같은 감사 작업도 있습니다.

    **Common** 이벤트 집합에는 그다지 일반적이지 않은 일부 유형의 이벤트가 포함될 수 있습니다.  **이는 Common** 집합의 주요 포인트는 전체 감사 추적 기능을 유지하면서 이벤트 볼륨을 보다 관리하기 쉬운 수준으로 줄이는 것입니다.

- **최소** - 잠재적인 위협을 나타낼 수 있는 작은 이벤트 집합입니다. 이 집합에는 전체 감사 추적이 포함되지 않습니다. 성공적인 위반을 나타낼 수 있는 이벤트와 발생률이 매우 낮은 기타 중요한 이벤트만 다룹니다. 예를 들어 성공 및 실패한 사용자 로그온(이벤트 번호 4624, 4625)을 포함하지만 감사에 중요하지만 위반 감지에 는 의미가 없으며 상대적으로 많은 볼륨을 가지는 로그아웃 정보(4634)를 포함하지 않습니다. 이 집합의 데이터 볼륨대부분은 로그인 이벤트 및 프로세스 생성 이벤트(이벤트 ID 4688)로 구성됩니다.

- **없음** - 보안 또는 AppLocker 이벤트가 없습니다. 이 설정은 커넥터를 사용하지 않도록 설정하는 데 사용됩니다.

> - Azure 보안 센터에 보안 이벤트 컬렉션을 그대로 둡니다. Azure Sentinel 및 Azure 보안 센터에서 이러한 이벤트를 쿼리하고 분석할 수 있습니다. 그러나 Azure Sentinel에서 커넥터의 연결 상태를 모니터링하거나 구성을 변경할 수는 없습니다. 이것이 중요한 경우 두 번째 옵션을 고려하십시오.
>
> - Azure 보안 센터에서 [보안 이벤트 컬렉션을 사용하지 않도록 설정한](../security-center/security-center-enable-data-collection.md) 다음 Azure Sentinel에서 보안 이벤트 커넥터만 추가합니다. 첫 번째 옵션과 마찬가지로 Azure Sentinel 및 Azure 보안 센터에서 이벤트를 쿼리하고 분석할 수 있지만 이제 커넥터의 연결 상태를 모니터링하거나 Azure Sentinel에서만 해당 구성을 변경할 수 있습니다.

## <a name="set-up-the-windows-security-events-connector"></a>Windows 보안 이벤트 커넥터 설정

Azure Sentinel에서 Windows 보안 이벤트를 수집하려면 다음을 수행하십시오.

1. Azure Sentinel 탐색 메뉴에서 **데이터 커넥터를 선택합니다.** 커넥터 목록에서 **보안 이벤트를**클릭한 다음 오른쪽 하단의 커넥터 **열기 페이지** 단추를 클릭합니다. 그런 다음 이 섹션의 나머지 부분에서 설명한 대로 **지침** 탭 아래의 화면 지침을 따릅니다.

1. **필수 구성 조건**아래에 설명된 대로 적절한 사용 권한이 있는지 확인합니다.

1. Azure Sentinel으로 보안 이벤트를 스트리밍하려는 컴퓨터에 [로그 분석 에이전트(Microsoft](../azure-monitor/platform/log-analytics-agent.md) 모니터링 에이전트 또는 MMA라고도 함)를 다운로드하여 설치합니다.

    Azure 가상 시스템의 경우:
    
    1. Azure **Windows 가상 컴퓨터에서 에이전트 설치를**클릭한 다음 아래 링크에서 에이전트 설치를 클릭합니다.
    1. 연결하려는 각 가상 컴퓨터에 대해 오른쪽에 나타나는 목록에서 해당 이름을 클릭한 다음 **연결을**클릭합니다.

    Azure Windows가 아닌 컴퓨터(물리적, 가상 온프레마 또는 다른 클라우드의 가상)의 경우:

    1. Azure **가 아닌 Windows 컴퓨터에서 에이전트 설치를**클릭한 다음 아래에 표시되는 링크를 클릭합니다.
    1. 오른쪽에 나타나는 적절한 다운로드 링크를 **클릭합니다.**
    1. 다운로드한 실행 파일을 사용하여 선택한 Windows 시스템에 에이전트를 설치하고 위에서 언급한 다운로드 링크 아래에 나타나는 **작업 영역 ID 및 키를** 사용하여 에이전트를 구성합니다.

    > [!NOTE]
    >
    > 인터넷 연결이 없는 Windows 시스템이 Azure Sentinel으로 이벤트를 계속 스트리밍할 수 있도록 하려면 오른쪽 하단의 링크를 사용하여 **OMS 게이트웨이를** 별도의 컴퓨터에 다운로드하여 설치하여 프록시 역할을 합니다.  수집하려는 이벤트가 있는 각 Windows 시스템에 Log Analytics 에이전트를 설치해야 합니다.
    >
    > 이 시나리오에 대한 자세한 내용은 [ **Log Analytics 게이트웨이** 설명서를](../azure-monitor/platform/gateway.md)참조하십시오.

    추가 설치 옵션 및 자세한 내용은 [ **Log Analytics 에이전트** 설명서를](../azure-monitor/platform/agent-windows.md)참조하십시오.

1. 스트리밍할 이벤트[집합(모두, 공통 또는 최소)을](#event-sets)선택합니다.

1. **업데이트**를 클릭합니다.

1. Windows 보안 이벤트에 대한 로그 분석에서 관련 `SecurityEvent` 스키마를 사용하려면 쿼리 창을 입력합니다.
