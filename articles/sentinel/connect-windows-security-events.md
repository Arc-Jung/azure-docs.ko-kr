# <a name="connect-windows-security-events"></a>Windows 보안 이벤트 연결 



Azure 센티널 작업 영역에 연결 된 Windows 서버에서 모든 보안 이벤트를 스트리밍할 수 있습니다. 이 연결을 통해 대시보드를 보고, 사용자 지정 경고를 만들고, 조사를 개선할 수 있습니다. 이렇게 하면 조직의 네트워크에 대 한 자세한 정보를 제공 하 고 보안 작업 기능을 향상 시킬 수 있습니다.  스트리밍할 이벤트를 선택할 수 있습니다.

- **모든 이벤트** -모든 Windows 보안 및 AppLocker 이벤트입니다.
- **Common** -감사 목적으로 표준 이벤트 집합입니다. 전체 사용자 감사 내역이이 집합에 포함 됩니다. 예를 들어이 집합에는 사용자 로그인 및 사용자 로그 아웃 이벤트 (이벤트 ID 4634)가 모두 포함 됩니다. Microsoft는 보안 그룹 변경, 주요 도메인 컨트롤러 Kerberos 작업, 그리고 업계의 조직에서 권장하는 기타 이벤트와 같은 감사 작업을 이 집합에 포함했습니다.

양이 매우 적은 이벤트는 일반 집합에 포함되었습니다. 모든 이벤트를 선택하는 대신 이러한 이벤트를 선택하는 주된 이유는 특정 이벤트를 필터링하여 제외하기 위해서가 아니라 이벤트의 양을 줄이기 위해서이기 때문입니다.
- **최소** -잠재적 위협을 나타낼 수 있는 작은 이벤트 집합입니다. 이 옵션을 사용 하도록 설정 하면 전체 감사 내역을 사용할 수 없습니다.  이 집합은 성공적인 위반 및 매우 낮은 양의 중요 한 이벤트를 나타낼 수 있는 이벤트만 포함 합니다. 예를 들어이 집합에는 사용자 성공 및 실패 한 로그인 (이벤트 Id 4624, 4625)이 포함 되지만 감사에는 중요 하지만 검색에는 의미가 없으며 볼륨이 매우 많은 로그 아웃 정보가 포함 되지 않습니다. 이 집합의 데이터 볼륨 대부분은 로그인 이벤트 및 프로세스 생성 이벤트 (이벤트 ID 4688)입니다.
- **없음** -보안 또는 AppLocker 이벤트가 없습니다.

> [!NOTE]
> 
> - 데이터는 Azure 센티널을 실행 하는 작업 영역의 지리적 위치에 저장 됩니다.
> - Azure Security Center와 Azure 센티널이 동일한 작업 영역에서 실행 되는 경우 보안 이벤트 커넥터는 Azure Security Center 또는 Azure 센티널 에서만 연결할 수 있습니다. Azure 센티널에서 이러한 이벤트를 관리 하려면 Azure Security Center에서 연결을 끊고 Azure 센티널에만 연결 하는 것이 좋습니다.

## <a name="set-up-the-windows-security-events-connector"></a>Windows 보안 이벤트 커넥터 설정

Windows 보안 이벤트를 Azure 센티널과 완전히 통합 하려면 다음을 수행 합니다.

1. Azure 센티널 포털에서 **데이터 커넥터** 를 선택한 다음 **Windows 보안 이벤트** 타일을 클릭 합니다. 
1. 스트리밍할 데이터 형식을 선택 합니다.
1. **업데이트**를 클릭합니다.
6. Windows 보안 이벤트에 대 한 Log Analytics에서 관련 스키마를 사용 하려면 **Securityevent**를 검색 합니다.
