# <a name="azure-blockchain-tokens-composability"></a>Azure 블록 체인 토큰 구성 가능성

[!INCLUDE [Preview note](./includes/preview.md)]

토큰 구성가능성은 고급 시나리오에 대한 토큰을 만들 수 있는 유연성을 제공합니다. [미리 빌드된 네 개의 토큰 템플릿을](templates.md#base-token-types)사용하여 구현할 수 없는 복잡한 시나리오가 있을 수 있습니다. 토큰 구성 기능을 사용하면 정의된 동작을 추가하거나 제거하여 고유한 토큰 템플릿을 디자인하여 고유한 토큰 템플릿을 만들 수 있습니다. 새 토큰 템플릿을 만들 때 Azure 블록 체인 토큰은 모든 토큰 문법 규칙을 확인합니다. 구성된 템플릿은 연결된 블록 체인 네트워크에서 발급하기 위한 Azure 블록 체인 토큰 서비스에 저장됩니다.

다음 섹션의 [토큰 동작을](templates.md#token-behaviors) 사용하여 토큰 템플릿을 디자인할 수 있습니다.

## <a name="burnable-b"></a>연소 가능 (b)

공급에서 토큰을 제거하는 기능.

예를 들어 온라인 신용 카드 포인트를 기프트 카드로 사용하면 신용 카드 포인트가 소진됩니다.

## <a name="delegable-g"></a>위임식 (g)

소유한 토큰에서 수행된 작업을 위임하는 기능입니다.

대리인은 토큰의 소유자로서 작업을 수행할 수 있습니다. 예를 들어 위임 가능한 토큰을 사용하여 투표를 구현할 수 있습니다. 위임 가능한 토큰을 사용하면 투표 토큰 소유자가 다른 사람이 대신 투표하도록 할 수 있습니다.

## <a name="logable-l"></a>로그 가능(l)

로그 할 수있는 기능.

예를 들어 특정 영화를 상영하는 각 극장에 영화 배포를 위해 로그 가능한 토큰을 발행할 수 있습니다. 영화를 재생하려면 영화의 출시 기간 동안 로열티 지급액이 상영될 수 있으므로 각 상영에 대한 트랜잭션을 기록해야 합니다. 액터 빌드는 동영상 토큰을 사용하여 배포판에서 극장당 상영되는 영화당 지불금의 유효성을 검사할 수 있습니다.

## <a name="mint-able-m"></a>민트 가능 (m)

토큰 클래스에 대한 추가 토큰을 박하하는 능력. minter 역할에는 mintable 동작이 포함됩니다.

예를 들어 로열티 프로그램을 구현하려는 소매 업체는 로열티 프로그램에 mintable 토큰을 사용할 수 있습니다. 고객 기반이 증가함에 따라 고객을 위한 추가 로열티 포인트를 받을 수 있습니다.  

## <a name="non-subdividable-or-whole-d"></a>비종속 또는 전체 (~d)

토큰이 더 작은 부분으로 분할되는 것을 방지하기 위한 제한입니다.

예를 들어, 하나의 아트 페인팅은 여러 개의 작은 부분으로 세분화할 수 없습니다. 

## <a name="non-transferable-t"></a>양도 불가(~t)

초기 토큰 소유자의 소유권 변경을 방지하기 위한 제한입니다.

예를 들어, 대학 졸업장은 양도할 수 없는 토큰입니다. 졸업장이 졸업한 후에는 졸업에서 다른 사람으로 편입할 수 없습니다.

## <a name="roles-r"></a>역할 (r)

특정 동작에 대한 토큰 템플릿 클래스 내에서 역할을 정의하는 기능입니다.

토큰 생성 시 토큰이 지원하는 역할 이름 목록을 제공할 수 있습니다. 역할을 지정하면 사용자는 이러한 동작에 역할을 할당할 수 있습니다. 현재는 minter 역할만 지원됩니다.

## <a name="singleton-s"></a>싱글톤 (들)

하나의 토큰을 공급할 수 있도록 제한합니다.

예를 들어 박물관 아티팩트는 단일 토큰입니다. 박물관 유물은 독특합니다. 아티팩트를 나타내는 토큰에는 공급장치에 단일 항목만 있습니다.

## <a name="subdividable-d"></a>종속 가능 (d)

토큰을 더 작은 부분으로 나눌 수 있는 기능.

예를 들어, 달러는 센트로 나눌 수 있습니다.

## <a name="transferable-t"></a>양도 가능 (t)

토큰의 소유권을 이전하는 기능입니다.

예를 들어 속성 제목은 양도 가능한 토큰으로, 속성이 판매될 때 한 사람에서 다른 사람으로 이전할 수 있습니다.
