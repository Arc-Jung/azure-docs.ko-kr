# <a name="azure-blockchain-tokens-templates"></a>Azure 블록 체인 토큰 템플릿

[!INCLUDE [Preview note](./includes/preview.md)]

Azure 블록 체인 토큰 템플릿은 원장 기반 토큰의 생성 및 배포를 단순화하는 표준화되고 재사용 가능한 템플릿입니다. 템플릿은 [토큰 분류 프레임워크(TTF)](overview.md#token-taxonomy-framework) 문법을 기반으로 하는 수식으로 구성됩니다. 문법은 기본 토큰 유형과 토큰에 대한 동작 집합을 포함합니다.  

예를 **들어, θ{d, m, b, r}** 토큰 템플릿은 하위 분리 가능, minable, 연소 가능 및 역할 지원이 있는 fungible 기본 토큰을 설명합니다.
  
## <a name="base-token-types"></a>기본 토큰 유형

특정 자산에 대한 원장 기반 토큰을 정의하고 만들 때 사용할 기본 토큰을 고려하는 것이 중요합니다.

### <a name="fungible"></a>펑기블 (것)과 함께

Fungible 토큰(θF)은 동일한 클래스 또는 계열에 있는 한 서로 교환 가능한 값을 갖습니다. 한 토큰은 다른 토큰과 동일한 값을 가지거나 지정된 토큰 수량은 다른 토큰수량과 동일한 값을 가합니다. 예를 들어, 달러는 펑니블 토큰입니다. 두 사람이 각각 달러 지폐를 들고 있다면, 그들은 결과없이 이러한 달러 지폐를 교환 할 수 있습니다. 달러 지폐는 동일한 가치를 가합니다. 

### <a name="non-fungible"></a>비펑성

비fungible 토큰 (θN)은 일반적으로 다른 값을 가지고 같은 유형의 다른 토큰과 교환 할 수 없습니다. 예를 들어 속성 제목은 fungible 토큰입니다. 아파트 단지의 두 개의 다른 아파트에 대한 속성 제목은 반드시 단위의 위치 또는 단위가있는 바닥중 하나에, 동일한 값의 되지 않습니다. 두 속성 제목 토큰의 인식된 값은 같지 않습니다.

### <a name="hybrid"></a>하이브리드

하이브리드 토큰은 펑키 토큰과 비펑블 토큰의 구성 요소가있는 토큰입니다. 하이브리드 토큰은 다른 토큰 형식의 클래스를 소유하는 기본 토큰 유형입니다.

#### <a name="hybrid-non-fungible-base-with-fungible-segments"></a>펑블 세그먼트가 있는 하이브리드 비펑성 베이스

fungible 세그먼트 토큰이있는 하이브리드 비 융균 기반은 펑키 할 수있는 토큰 세그먼트가있는 비 융성 기반을 가지고 있습니다.
예를 들어, 콘서트 티켓은 콘서트 날짜와 시간이 비fungible 기본 토큰인 하이브리드 토큰입니다. 주어진 콘서트의 다양한 좌석 섹션의 티켓은 펑키 토큰이있는 세그먼트입니다. 티켓은 개별 좌석 섹션에서 교환할 수 있지만 섹션에는 적용되지 않습니다.

#### <a name="hybrid-fungible-base-with-non-fungible-segments"></a>비 펑키 세그먼트와 하이브리드 펑니블 베이스

비 fungible 세그먼트 토큰 하이브리드 fungible 베이스는 비 fungible 토큰 세그먼트와 펑키 할 수있는 기반을 가지고 있습니다. 예를 들어, 모기지 담보 보안은 여러 소유자가 많은 소유자에 걸쳐 분할 되는 fungible 기반 하이브리드 토큰입니다. 보안은 서로 교환할 수 있습니다. 개별 모기지는 특정 모기지 담보 대출을 나타내는 비 fungible 세그먼트입니다.

## <a name="token-behaviors"></a>토큰 동작

토큰 동작은 토큰의 기능 또는 제한을 정의합니다. 이 동작에는 토큰 정의의 일부인 지원 속성이 포함됩니다. 동작은 모든 토큰 유형 또는 하나만 적용할 수 있습니다. 동작은 동작에 미치는 영향에 따라 내부 또는 외부일 수 있습니다. 내부 동작은 토큰 자체의 속성을 활성화하거나 제한합니다. 외부 동작은 외부 행위자로부터 동작을 호출하거나 제한합니다.

지원되는 Azure 블록 체인 토큰에 대한 자세한 내용은 토큰 [구성](composability.md)프레임워크(TTF) 토큰 동작을 참조하세요.

## <a name="pre-built-token-templates"></a>미리 빌드된 토큰 템플릿

Azure 블록 체인 토큰은 수정없이 사용할 수있는 네 개의 미리 빌드 된 토큰 템플릿을 제공합니다. 대부분의 사용 사례에 대해 이러한 미리 빌드된 템플릿을 호출하여 토큰을 빠르게 생성, 배포 및 관리하기 시작할 수 있습니다.

### <a name="commodity-tokens"></a>상품 토큰

상품 토큰은 일관된 가치를 가지며 양도할 수 있습니다. 예를 들어, 오일 배럴 또는 에너지 단위.

**θF{~d, t, m, b, r}** - 펑블, 전체, 양도 가능, 채굴 가능, 화상 가능, 역할 지원

많은 블록 체인 시나리오에는 공급망 또는 여러 조직 전반에 걸쳐 투명성과 가시성이 필요합니다. 상품 토큰은 이러한 일반적인 사용 사례를 기반으로 합니다. 토큰은 상호 교환 가능하고 일관적입니다. 상품 토큰 템플릿은 메타데이터로 유연하고 사용자 정의 할 수 있습니다.

### <a name="qualified-tokens"></a>자격을 갖춘 토큰

정규화된 토큰은 획득한 것을 나타내며 일반적으로 하나의 엔터티와 연결되며 전송할 수 없습니다. 예를 들어, 졸업장 또는 주차 위반.

**θN{s,~t}** - 비펑성, 싱글톤 및 양도불가

다양한 감사 및 증명 시나리오에서는 토큰의 소유권을 변경할 수 없습니다. 연결이 좋든 나쁘든 자격을 갖춘 토큰을 제공해야 하는 사용 사례 집합이 있습니다.

### <a name="asset-tokens"></a>자산 토큰

자산 토큰은 항목에 따라 고유한 가치를 가지며 상품화되지 않습니다. 예를 들어 박물관 아티팩트 또는 속성 제목입니다.

**θN{s,t}** - 비펑성, 싱글톤 및 양도 가능

자산 토큰은 상품 토큰과 혼동될 수 있습니다. 두 토큰의 주요 차이점은 자산 토큰이 본질적으로 고유하며 가치는 토큰 유형과 독립적이라는 것입니다. 예를 들어, 기성 작가의 유화와 같은 예술 작품은 자산 토큰입니다. 그러나 모나리자의 미술 판화는 상품 토큰으로 간주됩니다. 마찬가지로 속성 제목은 속성의 주관적인 특성에 값이 있기 때문에 자산 토큰입니다.

### <a name="ticket-tokens"></a>티켓 토큰

티켓 토큰은 일관된 값을 가지지만 일반적으로 만료됩니다. 예를 들어, 비행기 티켓입니다.

**θN{m, b, r}** - 비펑성, 채굴 가능, 연소 가능, 역할 지원이 있습니다.

티켓 토큰은 일반적으로 일반 상품 토큰과 다른 만료 날짜가 있습니다. 예를 들어 비행기 티켓, 콘서트 티켓 또는 스포츠 티켓은 모두 특정 사용 날짜가 지정된 좌석이 할당된 좌석을 선택할 수 있습니다. 날짜나 좌석 공간 간에는 티켓을 쉽게 교환할 수 없습니다.
