# <a name="understand-business-glossary-features-in-azure-purview"></a>Azure 부서의 범위의 비즈니스 용어집 기능 이해

이 문서에서는 Azure 부서의 범위의 비즈니스 용어집 기능에 대 한 개요를 제공 합니다. 

## <a name="business-glossary"></a>비즈니스 용어집

용어집은 비즈니스 사용자를 위한 어휘를 제공 합니다.  서로 관련 될 수 있고 서로 다른 컨텍스트에서 이해할 수 있도록 분류할 수 있는 비즈니스 용어로 구성 됩니다. 그런 다음 이러한 용어를 데이터베이스, 테이블, 열 등의 자산에 매핑할 수 있습니다. 이를 통해 데이터 리포지토리와 관련 된 기술 전문 용어를 추상화 하 여 비즈니스 사용자가 어휘에서 보다 친숙 한 데이터를 검색 하 고 사용할 수 있습니다.


비즈니스 용어집은 용어 모음입니다. 각 용어는 조직의 개체를 나타내며 동일한 개체를 나타내는 여러 용어가 있을 가능성이 높습니다. 고객은 클라이언트, 구매 또는 구매자 라고도 합니다. 이러한 여러 용어는 서로 관계가 있습니다. 이러한 용어 간의 관계는 다음 중 하나일 수 있습니다.

- 동의어-동일한 정의를 사용 하는 여러 용어
- 관련-유사한 정의가 있는 다른 이름

동일한 용어는 여러 비즈니스 개체를 암시 하기도 합니다. 조직 내에서 각 용어를 잘 정의 하 고 명확 하 게 이해 하는 것이 중요 합니다.

## <a name="custom-attributes"></a>사용자 지정 특성

Azure 부서의 범위는 비즈니스 용어집 용어에 대 한 8 가지 기본 특성을 지원 합니다.
- 이름
- 정의
- 데이터 관리자
- 데이터 전문가
- 머리글자어
- 동의어
- 관련 용어
- 리소스

이러한 특성은 편집 하거나 삭제할 수 없습니다. 그러나 이러한 특성은 조직에서 용어를 완전히 정의 하는 데 충분 하지 않습니다. 이 문제를 해결 하기 위해 부서의 범위는 용어집에 대 한 사용자 지정 특성을 정의할 수 있는 기능을 제공 합니다.

## <a name="term-templates"></a>용어 템플릿

용어 템플릿은 카탈로그에서 논리적으로 그룹화 할 용어집 사용자 지정 특성을 제공 합니다. 기능을 사용 하면 모든 관련 사용자 지정 특성을 템플릿에서 그룹화 한 후 용어집 용어를 만드는 동안 템플릿을 적용할 수 있습니다. 예를 들어 비용 센터, 수익 센터, 회계 코드와 같은 모든 재무 관련 사용자 지정 특성은 용어 템플릿 재무 템플릿에서 그룹화 할 수 있으며 재무 템플릿은 재무 용어집 용어를 만드는 데 사용할 수 있습니다.

모든 표준 특성은 시스템 기본 템플릿에서 그룹화 됩니다. 사용자가 만드는 모든 용어 템플릿에는 템플릿 생성 프로세스의 일부로 생성 된 추가 사용자 지정 특성과 함께 이러한 특성이 포함 됩니다.

## <a name="glossary-vs-classification-vs-sensitivity-labels"></a>용어집 vs 분류 vs 민감도 레이블

용어, 분류 및 레이블은 데이터 자산에 대 한 주석이 며, 각 항목은 카탈로그의 컨텍스트에서 서로 다른 의미를 갖습니다. 

### <a name="glossary"></a>용어

위에서 설명한 것 처럼 비즈니스 용어집 용어는 조직의 비즈니스 어휘를 정의 하 고 회사의 여러 부서 간 격차를 브리징 하는 데 도움이 됩니다.

### <a name="classifications"></a>분류

분류는 엔터티에 할당 될 수 있는 주석입니다. 분류의 유연성을 통해 다음과 같은 여러 시나리오에 사용할 수 있습니다.

- 데이터 자산에 저장 된 데이터의 특성 이해
- 액세스 제어 정책 정의

부서의 범위에는 현재 100 개 이상의 시스템 분류자가 있으며 카탈로그에서 고유한 분류자를 정의할 수 있습니다. 검색 프로세스의 일부로 자동으로 이러한 분류를 검색 하 고 데이터 자산 및 스키마에 적용 합니다. 그러나 언제 든 지 재정의할 수 있습니다. 사용자 재정의는 자동 검색으로 대체 되지 않습니다.

### <a name="sensitivity-labels"></a>민감도 레이블

민감도 레이블은 hindering 생산성 및 공동 작업 없이 조직의 데이터를 분류 하 고 보호할 수 있도록 하는 주석 유형입니다. 민감도 레이블은 조직 데이터 내에서 분류 유형의 범주를 식별 하 고 각 범주에 적용할 정책을 그룹화 하는 데 사용 됩니다. 부서의 범위는 Microsoft 365와 동일한 중요 한 정보 유형을 사용 하 여 전체 콘텐츠와 데이터 공간에 대 한 기존 보안 정책 및 보호를 확장할 수 있게 합니다. 동일한 레이블은 부서의 범위의 Microsoft Office 제품 및 데이터 자산에서 공유할 수 있습니다.
