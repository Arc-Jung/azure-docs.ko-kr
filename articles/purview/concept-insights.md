# <a name="understand-insights-in-azure-purview"></a>Azure 부서의 범위의 통찰력 이해

이 문서에서는 Azure 부서의 범위의 정보 기능에 대 한 개요를 제공 합니다.

통찰력은 부서의 범위의 주요 핵심 요소 중 하나입니다. 이 기능을 통해 고객은 카탈로그에 투명 한 단일 창을 제공 하 고 데이터 원본 관리자, 비즈니스 사용자, 데이터 관리자, 데이터 담당자 및 보안 관리자에 게 특정 정보를 제공할 수 있습니다. 현재 부서의 범위에는 공개 미리 보기에서 고객에 게 제공 되는 다음과 같은 정보 보고서가 있습니다.

## <a name="asset-insights"></a>Asset Insights

이 보고서는 데이터 공간에 대 한 bird 's 조감도와 원본 유형별 배포를 분류 및 파일 크기 (일부)로 제공 합니다. 이 보고서는 부서의 범위 계정을 관리 하 고 조직의 데이터 공간 내에 특정 분류를 사용 하 여 존재 하는 자산 수를 확인 하는 데 관심을 맞춘 수 있는 여러 유형의 사용자에 게 유용 합니다. 

이 보고서는 그래프 및 Kpi를 통해 광범위 한 통찰력을 제공 하 고 나중에 잘못 된 파일 등의 특정 변칙을 자세히 설명 합니다. 또한이 보고서는 고객이 특정 분류를 사용 하 여 자산 수를 확인 하 고, 원본 유형과 최상위 폴더로 정보를 분석 하 고, 추가 조사를 위해 자산 목록을 볼 수 있는 종단 간 고객 환경을 지원 합니다.

## <a name="scan-insights"></a>검색 정보

이 보고서를 사용 하면 관리자가 검색의 전반적인 상태 (성공한 횟수, 실패 한 횟수, 취소 된 횟수)를 파악할 수 있습니다. 이 보고서는 지난 7 일 또는 최근 30 일 동안 부서의 범위 계정에서 실행 된 검색에 대 한 상태 업데이트를 제공 합니다.

또한 관리자는이 보고서를 사용 하 여 실패 한 검색 및 특정 원본 유형에 대해 자세히 알아볼 수 있습니다. 사용자가 더 자세히 조사할 수 있도록 보고서는 "원본" 환경 내에서 검색 기록 페이지로 이동 하는 데 도움이 됩니다.

## <a name="glossary-insights"></a>용어집 정보

이 보고서는 비즈니스 사용자 및 데이터 관리자 용어집에 대 한 상태 보고서를 제공 합니다. 사용자는이 보고서를 통해 상태별로 용어 배포를 이해 하 고, 자산에 첨부 된 용어 수와 자산에 아직 연결 되지 않은 용어를 확인할 수 있습니다. 비즈니스 사용자는 용어집 사용 약관의 완전성에 대해 알아볼 수도 있습니다. 

이 보고서는 비즈니스 사용자 또는 데이터 관리자에서 집중적으로 사용 해야 하는 상위 항목을 요약 하 여 조직에 대 한 완전 하 고 사용 가능한 용어집을 만듭니다. 사용자는 "용어집 정보" 환경에서 "용어집" 환경으로 이동 하 여 특정 용어집 용어를 변경할 수도 있습니다.

## <a name="classification-insights"></a>분류 정보

이 보고서는 분류 된 데이터가 있는 위치, 검색 중에 발견 된 분류, 분류 된 파일 자체로 드릴 다운에 대 한 세부 정보를 제공 합니다. 이를 통해 보안 관리자는 조직의 데이터 공간에서 발견 된 정보 유형을 이해할 수 있습니다. 

Azure 부서의 범위에서 분류는 subject 태그와 비슷하며 데이터 공간에서 특정 유형의 콘텐츠를 표시 하 고 식별 하는 데 사용 됩니다.

분류 정보 보고서를 사용 하 여 특정 분류가 포함 된 콘텐츠를 식별 하 고 리포지토리에 보안을 추가 하거나 더 안전한 위치로 콘텐츠를 이동 하는 등의 필수 작업을 이해 합니다.

자세한 내용은 [Azure 부서의 범위에서 데이터에 대 한 분류](classification-insights.md)정보를 참조 하세요.

## <a name="sensitivity-labeling-insights"></a>민감도 레이블 지정 정보

이 보고서는 검색 중에 발견 된 민감도 레이블 및 레이블이 지정 된 파일 자체에 대 한 드릴 다운에 대 한 세부 정보를 제공 합니다. 이를 통해 보안 관리자는 조직의 데이터 공간에서 발견 된 정보의 보안을 유지할 수 있습니다. 

Azure 부서의 범위에서 민감도 레이블은 분류 유형 범주 및 각 범주에 적용 하려는 그룹 보안 정책을 식별 하는 데 사용 됩니다.

레이블 지정 정보 보고서를 사용 하 여 콘텐츠에서 찾은 민감도 레이블을 식별 하 고 특정 리포지토리 또는 파일에 대 한 액세스를 관리 하는 등의 필수 작업을 이해 합니다.

자세한 내용은 [Azure 부서의 범위에서 데이터에 대 한 민감도 레이블 정보](sensitivity-insights.md)를 참조 하세요.

## <a name="file-extension-insights"></a>파일 확장명 정보

이 보고서는 파일 자체에 대 한 드릴 다운 뿐만 아니라 검색 중에 발견 되는 파일 확장명 또는 파일 형식에 대 한 세부 정보를 제공 합니다. 

파일 확장명 정보 보고서를 사용 하 여 각 시간에 포함 된 파일의 수, 해당 파일의 위치 및 중요 한 데이터에 대 한 병렬로 검색 가능 여부를 파악할 수 있습니다.

자세한 내용은 [Azure 부서의 범위에서 데이터에 대 한 파일 확장명](file-extension-insights.md)정보를 참조 하세요.
