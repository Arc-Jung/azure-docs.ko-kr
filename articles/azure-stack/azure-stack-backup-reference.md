# <a name="infrastructure-backup-service-reference"></a>인프라 백업 서비스 참조

## <a name="azure-backup-infrastructure"></a>Azure 백업 인프라

*적용 대상: Azure Stack 통합 시스템 및 Azure Stack 개발 키트*

인프라 관리 환경 및 azure Stack 포털, Azure Resource Manager를 구성 하는 여러 서비스 구성 됩니다. Azure Stack의 어플라이언스 비슷한 관리 환경을 솔루션의 운영자에 게 노출 복잡도 줄여에 중점을 둡니다.

인프라 백업은 백업 작업의 복잡성을 내 면 화하고 하도록 설계 되었으며 및 인프라 서비스에 대 한 데이터를 복원할 솔루션을 관리 하 고 사용자에 게 SLA를 유지 관리에 집중할 수 연산자를 확인 합니다.

동일한 시스템에 백업을 저장 하지 않으려면 외부 공유에 백업 데이터를 내보낼 필요 합니다. 외부 공유를 필요한 유연성 관리자가 기존 회사 BC/DR 정책을 기반으로 데이터를 저장할 위치를 확인할 수 있습니다. 

### <a name="infrastructure-backup-components"></a>인프라 Backup 구성 요소

다음 구성 요소를 포함 하는 인프라 백업:

 - **인프라 백업 컨트롤러**  
 인프라 백업 컨트롤러로 인스턴스화되는 하 고 모든 Azure Stack 클라우드에 상주 합니다.
 - **백업 리소스 공급자**  
 사용자 인터페이스 및 응용 프로그램 프로그램 인터페이스 (API) s의 Azure Stack 인프라에 대 한 기본 백업 기능을 노출 백업 리소스 공급자 (백업 RP)으로 구성 됩니다.

#### <a name="infrastructure-backup-controller"></a>인프라 백업 컨트롤러

인프라 백업 컨트롤러는 Service Fabric 서비스는 Azure Stack 클라우드에 대 한 인스턴스화됩니다. 백업 리소스를 지역 수준 및 캡처 지역별 서비스 데이터 AD, CA, Azure Resource Manager에서 만들어진 CRP, SRP, NRP, KeyVault, RBAC 합니다. 

### <a name="backup-resource-provider"></a>백업 리소스 공급자

백업 리소스 공급자는 기본 구성 및 백업 리소스의 목록에 대 한 Azure Stack 포털에 사용자 인터페이스를 제공 합니다. 연산자는 사용자 인터페이스에서 다음 작업을 수행할 수 있습니다.

 - 백업을 외부 저장소 위치, 자격 증명 및 암호화 키를 제공 하 여 처음으로 사용 하도록 설정
 - 완료 하는 뷰가 생성 백업 리소스 상태에서 및 리소스 만들기
 - 컨트롤러 백업에서 백업 데이터를 삽입 하는 위치는 저장소 위치를 수정 합니다.
 - 외부 저장소 위치에 액세스 하려면 백업 컨트롤러를 사용 하는 자격 증명 수정
 - 백업 컨트롤러 백업을 암호화 하는 데 사용 하는 암호화 키를 수정 합니다. 


## <a name="backup-controller-requirements"></a>백업 컨트롤러 요구 사항

이 섹션에서는 인프라 백업에 대 한 중요 한 요구 사항을 설명 합니다. 정보를 검토 하 여 신중 하 게 한 다음 다시 참조 되도록 필요에 따라 배포 및 후속 작업 중 및 Azure Stack 인스턴스에 대 한 백업을 사용 하도록 설정 하기 전에 유지 하는 것이 좋습니다.

요구 사항은 다음과 같습니다.

  - **네트워크 요구 사항** – 다른 저장소 위치에 대 한 네트워크 요구 사항에 설명 합니다.

### <a name="encryption-requirements"></a>암호화 요구 사항

1901 년부터 인프라 백업 서비스는 사용 하 여 인증서 공개 키를 사용 하 여 (합니다. 개인 키를 사용 하 여 백업 데이터와 인증서를 암호화 하려면 CER) (합니다. PFX) 클라우드 복구 하는 동안 백업 데이터를 해독 합니다.   
 - 인증서 키 전송에 사용 되 고 인증 된 보안 통신을 설정 하는 데 사용 되지 않습니다. 따라서 인증서는 자체 서명 된 인증서를 수 있습니다. Azure Stack 외부 인터넷 액세스가 필요 하지 않도록 루트 또는이 인증서에 대 한 트러스트를 확인 하지 않아도 됩니다.
 
자체 서명 된 인증서는 개인 키를 사용 하 여 여러 개 있는 공개 키를 사용 하 여 한 두 부분으로 제공 됩니다.
 - 백업 데이터를 암호화 합니다. (내보냅니다 공개 키를 사용 하 여 인증서. CER 파일)는 백업 데이터를 암호화 하는 데 사용 됩니다.
 - 백업 데이터를 암호 해독: (내보냅니다 개인 키를 사용 하 여 인증서. PFX 파일)는 백업 데이터를 해독 하는 데 사용 됩니다.

공개 키를 사용 하 여 인증서 (합니다. CER) 내부 비밀 회전 하 여 관리 되지 않습니다. 새 자체 서명 된 인증서를 만들고 새 파일을 사용 하 여 백업 설정을 업데이트 해야 인증서를 회전 하려면 (합니다. CER)입니다.  
 - 모든 기존 백업은 이전 공개 키를 사용 하 여 암호화 상태를 유지 합니다. 새 백업을 새 공개 키를 사용 합니다. 
 
개인 키를 사용 하 여 클라우드 복구 중 사용 된 인증서 (합니다. 보안상의 이유로 Azure Stack에서 PFX) 유지 되지 않습니다. 이 파일을 클라우드 복구 중에 명시적으로 제공 해야 합니다.  

**이전 버전과 호환성 모드** 1901 년부터 암호화 키 지원 사용 되지 않으며 이후 릴리스에서 제거 됩니다. 백업이 암호화 키를 사용 하 여 이미 활성화 된 1811에서 업데이트 하는 경우 암호화 키를 사용 하도록 Azure Stack 계속 됩니다. 이전 버전과 호환성 모드를 최소한 3 릴리스에 대 한 지원 됩니다. 이 시간 후 인증서를 해야 합니다. 
 * 인증서에 암호화 키를 업데이트 하는 것은 단방향 작업입니다.  
 * 모든 기존 백업 암호화 키를 사용 하 여 암호화 된 상태로 유지 됩니다. 새 백업 인증서를 사용 합니다. 

## <a name="infrastructure-backup-limits"></a>인프라 백업 제한

계획, 배포 및 Microsoft Azure Stack 인스턴스를 작동 하는 대로 이러한 제한을 고려 합니다. 다음 표에서 이러한 제한을 설명합니다.

### <a name="infrastructure-backup-limits"></a>인프라 백업 제한
| 제한 식별자                                                 | 제한        | 설명                                                                                                                                    |
|------------------------------------------------------------------|--------------|---------------------------------------------------------------------------------------------------------------------------------------------|
| 백업 유형                                                      | 전체만    | 인프라 백업 컨트롤러는 전체 백업만 지원합니다. 증분 백업은 지원 되지 않습니다.                                          |
| 예약 된 백업                                                | 예약 된 백업과 수동  | 백업 컨트롤러는 예약 된 백업과 주문형 백업을 지원합니다.                                                                                 |
| 최대 동시 백업 작업                                   | 1            | 백업 컨트롤러의 인스턴스당 하나의 활성 백업 작업이 지원 됩니다.                                                                  |
| 네트워크 스위치 구성                                     | 이 문서에서 다루지 않는 내용 | 관리자는 OEM 도구를 사용 하 여 네트워크 스위치 구성을 백업 해야 합니다. 각 OEM 공급 업체에서 제공 하는 Azure Stack에 대 한 설명서를 참조 하십시오. |
| 하드웨어 수명 주기 호스트                                          | 이 문서에서 다루지 않는 내용 | 관리자는 OEM 도구를 사용 하 여 하드웨어 수명 주기 호스트 백업 해야 합니다. 각 OEM 공급 업체에서 제공 하는 Azure Stack에 대 한 설명서를 참조 하십시오.      |
| 파일 공유의 최대 수                                    | 1            | 하나의 파일 공유를 사용 하 여 백업 데이터를 저장할 수 있습니다.                                                                                        |
| App Services, 함수, SQL, mysql 리소스 공급자 데이터를 백업 합니다. | 이 문서에서 다루지 않는 내용 | 배포 및 관리 부가 가치가 있는 Microsoft에서 만든 RPs에 대해 게시 하는 지침을 참조 하세요.                                                  |
| 백업에서 타사 리소스 공급자                              | 이 문서에서 다루지 않는 내용 | 배포 및 관리 부가 가치가 있는 타사 공급 업체에서 만든 RPs에 대해 게시 하는 지침을 참조 하세요.                                          |
