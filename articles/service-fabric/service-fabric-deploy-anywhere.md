# 서비스 패브릭을 사용하여 Windows Server 또는 Linux에서 원하는 위치에 배포
서비스 패브릭을 사용하면 Windows Server 또는 Linux를 실행 중인 컴퓨터 또는 VM에서 서비스 패브릭 클러스터를 만들 수 있습니다. 이렇게 하면 서비스 패브릭 응용 프로그램을 모든 환경에 배포하고 실행할 수 있으며, 서로 연결된 Windows Server 또는 Linux 컴퓨터 집합이 있는 클라우드 공급자 또는 온-프레미스로 사용할 수 있습니다. 서비스 패브릭은 서비스 패브릭 클러스터를 만들 수 있는 설치 패키지를 제공합니다. 이 기능의 주요 장점은 응용 프로그램이 실행되는 위치를 선택하기 때문에 서비스 패브릭을 사용하여 응용 프로그램을 빌드할 때 공급업체 교착 상태가 없다는 것입니다. 또한 고객마다 응용 프로그램을 실행하고자 하는 환경에 대한 요구 사항이 다를 수 있기 때문에 더 폭넓은 고객층을 확보할 수 있습니다. 예를 들어 의료 및 금융 업계 고객의 요구 사항은 자동차 또는 여행 산업 고객의 요구 사항과 다를 수 있습니다. 이 기능에 대한 Technical Preview는 2016년 1분기에 출시될 예정입니다.

## 지원되는 운영 체제
* Windows Server 2012 R2 * Windows Server 2016 * Linux 운영 체제를 실행하는 컴퓨터 또는 VM에서 클러스터를 만들 수 있습니다.

## 지원되는 프로그래밍 언어
* C# * Java 프로그래밍 언어를 사용하여 서비스 패브릭 응용 프로그램을 작성할 수 있습니다.

## 클러스터 만들기 및 구성
서비스 패브릭은 [Microsoft 다운로드 센터](https://www.microsoft.com/ko-KR/download)에서 다운로드되는 설치 패키지를 제공합니다. 이 패키지를 다운로드한 후에는 구성 파일을 변경하여 클러스터에 대한 설정을 지정해야 합니다. 클러스터 설정을 편집한 다음 설치 스크립트를 실행하여 클러스터 설정에서 지정한 컴퓨터에 클러스터 스패닝을 만듭니다. 설치 프로세스에 대한 정확한 정보는 2016년 1분기에 출시되는 이 기능의 Technical Preview를 참조하세요.

## 클러스터 관리 
클러스터를 업데이트하려면 클러스터 설정 파일을 수정하고 업데이트 스크립트를 실행하여 변경 내용을 적용해야 합니다.

## 클라우드 배포 및 온-프레미스 배포
서비스 패브릭 클러스터 온-프레미스를 만드는 프로세스는 VM 집합이 있는 선택한 클라우드에서 클러스터를 만드는 프로세스와 비슷합니다. VM 프로비전의 첫 번째 단계는 사용 중인 온-프레미스 환경 또는 클라우드에서 관리하지만, VM 집합 간에 네트워크 연결을 설정한 후 서비스 패브릭 패키지를 설치하고, 클러스터 설정을 편집하고, 클러스터 생성 및 관리 스크립트를 실행하는 다음 단계는 비슷합니다. 이렇게 하면 새 호스팅 환경을 대상으로 선택할 때 서비스 패브릭 클러스터 운영 및 관리에 대한 지식과 경험이 전달됩니다.

## 원하는 위치에 배포 기능의 이점
* 공급업체 종속성이 없습니다.
* 서비스 패브릭 응용 프로그램을 한 번 작성한 다음 변경 없이 또는 최소한의 변경으로 여러 호스팅 환경에서 실행할 수 있습니다.
* 호스팅 플랫폼 간에 서비스 패브릭 응용 프로그램 빌드에 대한 지식이 전달됩니다.
* 배포 환경 간에 서비스 패브릭 클러스터 실행에 대한 운영 경험이 전달됩니다.
* .호스팅 환경 제약을 받지 않아 고객 도달 범위가 넓습니다.
* 데이터 센터 또는 클라우드에 블랙아웃이 발생한 경우 서비스를 다른 배포 환경으로 이동함으로써 안정성이 강화되고 대규모 가동 중단이 방지됩니다.

## 원하는 위치에 배포와 서비스 패브릭 클러스터에 호스트된 Azure의 차이점
서비스에 호스트된 Azure를 사용하여 서비스 패브릭 클러스터를 실행할 경우 원하는 위치에 배포 옵션을 사용할 때보다 몇 가지 이점이 있습니다. 따라서 클러스터 실행 위치가 크게 중요하지 않은 경우 클러스터에 호스트된 Azure로 실행하는 것이 좋습니다. Azure에는 다른 Azure 기능과 서비스가 통합되어 클러스터 운영 및 관리를 더 쉽게 수행할 수 있습니다. * Azure 포털: Azure 포털을 사용하여 클러스터를 쉽게 만들고 관리할 수 있습니다. * Azure 리소스 관리자: ARM을 사용하면 클러스터에서 사용되는 모든 리소스를 하나의 단위로 사용함으로써 관리를 용이하게 하고 비용 추적 및 청구를 간소화할 수 있습니다. * 진단: Azure에는 진단을 위해 WAD(Windows Azure 진단)와 Operational Insights가 통합되어 있습니다. * 자동 크기 조정: 클러스터에 호스트된 Azure의 경우 자동 크기 조정 기능을 기본으로 제공합니다. 원하는 위치에 배포 기능을 사용하는 다른 환경에서는 사용자 고유 자동 크기 조정 기능을 빌드하거나 서비스 패브릭에서 클러스터 크기 조정을 위해 제공하는 API를 사용하여 수동으로 조정해야 합니다.
