# 서비스 패브릭의 기술 개요

서비스 패브릭은 확장 가능하고 안정적이며 대기 시간이 짧고 관리하기 쉬운 클라우드용 응용 프로그램을 간편하게 빌드할 수 있는 분산 시스템 플랫폼입니다. 다시 말해서 서비스 패브릭에서 항상 응용 프로그램의 가용성을 유지하고 규모를 조정하므로 사용자는 비즈니스 요구 사항에 집중할 수 있습니다.

## 주요 개념

**클러스터** - 응용 프로그램 인스턴스가 배포되는 네트워크로 연결된 가상 또는 실제 컴퓨터 집합입니다. 클러스터의 규모를 컴퓨터 수천 대로 확장할 수 있습니다.

**노드** - 클러스터에 있는 주소 지정이 가능한 단위입니다. 노드는 배치 속성 및 고유 ID와 같은 특징이 있습니다. 노드는 클러스터를 조인하고 Fabric.exe를 실행 중인 운영 체제와의 상관 관계를 지정할 수 있습니다.

**응용 프로그램/응용 프로그램 유형** - (마이크로) 서비스의 모음입니다. 응용 프로그램 유형은 하나 이상의 서비스 유형에 대한 컨테이너라고 생각하면 됩니다. 클러스터(그 자체가 여러 노드로 구성)가 어떻게 여러 ApplicationTypes으로 구성되는지 살펴보려면 [응용 프로그램 모델](service-fabric-application-model.md) 문서를 참조하세요.

**서비스/서비스 유형** - 독립 실행형 기능(독립적으로 시작 및 실행 가능)을 수행하는 코드 및 구성입니다(예: 큐 서비스 또는 데이터베이스 서비스). ApplicationType은 하나 이상의 ServiceTypes으로 구성됩니다. 서비스 유형은 두 가지가 있습니다.

- 상태 비저장 서비스: 상태가 있으며, 상태가 Azure 데이터베이스 또는 Azure 테이블 저장소와 같은 외부 저장소에 보관되는 서비스입니다. 이 서비스의 인스턴스가 활성 상태인 노드가 다운되면 다른 노드에서 다른 인스턴스가 자동으로 시작됩니다.

- 상태 저장 서비스: 상태가 있으며, 클러스터의 다른 노드에서 복제본 간의 복제를 통해 안정성을 실현하는 서비스입니다. 상태 저장 서비스에는 주 복제본 하나와 보조 복제본 여러 개가 있습니다. 이 서비스의 복제본이 활성 상태인 노드가 다운되면 다른 노드에서 새 복제본이 시작됩니다. 다운된 노드의 복제본이 주 복제본이면 보조 복제본이 자동으로 새로운 주 복제본으로 승격됩니다.

**응용 프로그램 인스턴스** - 클러스터에서 특정 응용 프로그램 유형의 응용 프로그램 인스턴스를 여러 개 만들 수 있으며 각 인스턴스는 구체적인 이름을 갖습니다. 같은 유형 또는 다른 유형의 다른 응용 프로그램 인스턴스에서 각 응용 프로그램 인스턴스를 독립적으로 관리하고 버전을 지정할 수 있습니다. 또한 응용 프로그램 인스턴스는 리소스 및 보안 격리를 정의합니다.

**서비스 인스턴스** - 서비스 유형에 대해 인스턴스화된 코드입니다. 각 서비스 인스턴스는 `fabric:/`로 시작하는 고유한 이름을 가지며 명명된 특정 응용 프로그램 인스턴스와 연결됩니다.

**응용 프로그램 패키지** - 특정 응용 프로그램에 대 해 결합된 서비스 코드 패키지 및 구성 파일의 컬렉션입니다. 배포되는 물리적 파일이자 파일 및 폴더 형식 레이아웃입니다. 예를 들어 전자 메일 응용 프로그램에 대한 응용 프로그램 패키지는 큐 서비스 패키지, 프런트엔드 서비스 패키지 및 데이터베이스 서비스 패키지를 포함할 수 있습니다.

**프로그래밍 모델** - 서비스 패브릭에서 응용 프로그램 빌드에 사용할 수 있는 두 가지 프로그래밍 모델이 있습니다.

- 신뢰할 수 있는 서비스 - `StatelessService` 및 `StatefulService` .NET 클래스를 기반으로 상태 비저장 및 상태 저장 서비스를 구축하고, .NET 신뢰할 수 있는 컬렉션(사전 및 큐)에 상태를 저장하고, 웹 API 및 WCF 같은 다양한 통신 스택을 플러그인하는 기능을 제공하는 API입니다. 이 프로그래밍 모델은 여러 상태 단위에 걸쳐 계산을 수행해야 하는 응용 프로그램에 적합합니다.

- 신뢰할 수 있는 행위자 - 가상 행위자 프로그래밍 모델을 통해 상태 비저장 및 상태 저장 개체를 구축하는 API로, 독립적인 상태 및 계산 단위가 여러 개 있는 응용 프로그램에 적합합니다.
