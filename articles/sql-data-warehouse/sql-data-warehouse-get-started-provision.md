# 시작: SQL 데이터 웨어하우스에 프로비전 #

이 문서는 Azure에서 SQL 데이터 웨어하우스에 프로비전할 수 있도록 지원하는 가이드입니다. 이 가이드에 따라 다음 작업을 수행합니다.

1. 새 SQL 데이터 웨어하우스 데이터베이스 만들기.
2. 새로운 논리 서버 구성.
3. 외부 클라이언트 액세스를 사용하도록 Azure 방화벽 규칙 설정.

## Azure 무료 평가판 ##
아래 작업을 완료하려면 Azure 구독이 필요합니다. Azure 구독에 대한 액세스가 아직 없는 경우, 이를 해결하는 것이 실제 첫 번째 단계입니다.

SQL 데이터 웨어하우스를 포함한 Azure의 서비스 중 하나를 시도할 수 있는 [무료 평가판][]을 얻을 수 있습니다.


## Azure 포털에 로그인 ##

구독이 있으면 [Azure 포털][]에 로그인할 수 있습니다. 계속해서 지금 로그인합니다.

이 다음 일련의 단계에서는 신속하게 새로운 논리 서버를 스핀업하고 새 SQL 데이터 웨어하우스 데이터베이스를 만듭니다.

## SQL 데이터 웨어하우스 서비스 검색

Azure 포털에서 SQL 데이터 웨어하우스 서비스를 찾는 것이 첫 번째 할 일입니다.

Azure 포털의 왼쪽 아래 모서리에는 새로 만들기 단추가 있습니다. 새로 만들기 단추는 Azure 내에서 모든 새 서비스를 만들기 위한 시작 지점입니다.

- 지금 새로 만들기 단추를 클릭합니다.

### 데이터 + 저장소

새로 만들기 단추를 클릭하면 Azure 내의 모든 서비스 범주를 엽니다. SQL 데이터 웨어하우스는 "데이터 + 저장소" 범주에 있습니다.

- **데이터 + 저장소**를 클릭하여 드릴인하고 이 범주에 대해 Azure에서 제공한 서비스를 봅니다.

### SQL 데이터 웨어하우스

볼 수 있듯이 Azure는 다양한 데이터 및 저장소 엔진을 제공합니다. 그러나 이 시작 가이드는 SQL 데이터 웨어하우스용입니다.

- 계속해서 **SQL 데이터 웨어하우스**를 선택합니다.

## SQL 데이터 웨어하우스 구성

프로비전 프로세스를 완료하려면 SQL 데이터 웨어하우스를 구성하기만 하면 됩니다.


### 데이터베이스 이름

첫 번째 구성은 데이터베이스의 이름을 지정하는 것입니다.



- 이 빠른 시작의 경우 데이터베이스의 이름을 "MySQLDW"로 지정합니다.


> [AZURE.NOTE]고유한 데이터베이스를 만들 때 물론 원하는 대로 이름을 지정할 수 있습니다. 그러나 Azure의 기본 명명 요구 사항을 따라야 합니다.

### 성능

성능 옵션은 *중요*합니다. SQL 데이터 웨어하우스는 이 슬라이더를 통해 확장성이 뛰어난 기능을 제공합니다. 데이터 웨어하우스를 구성할 때만 아니라 언제든지 성능을 늘리거나 줄일 수 있습니다. 슬라이드를 오른쪽으로 멀리 밀수록 원하는 사용할 수 있는 리소스가 늘어납니다. 이 리소스가 더 이상 필요하지 않은 경우 즉시 슬라이더를 다시 이동하여 비용을 절약할 수 있습니다. SQL 데이터 웨어하우스를 사용하면 데이터 웨어하우스를 다시 만들거나 데이터를 이동하지 않고도 필요에 따라 성능 프로필을 변경할 수 있습니다.

- 이제 슬라이더를 사용하여 오른쪽으로 슬라이더를 밀면 DWU(데이터 웨어하우스 단위)가 늘어나고 다시 왼쪽으로 밀면 줄어드는 것을 알 수 있습니다.

- 이 단계를 떠나기 전에 왼쪽으로 슬라이더를 반환해야 합니다. 새 데이터 웨어하우스가 작으므로 리소스는 그다지 많이 필요하지 않습니다. 평가판의 나머지 부분을 위해 리소스를 아낍니다.

### 원본 선택

이 옵션은 빈 데이터베이스로 시작하는 선택 사항을 제공합니다. 시작 지점으로 새 데이터베이스를 선택합니다.

> [AZURE.NOTE]두 번째 옵션을 사용할 수도 있습니다. 기존 복원 지점인 복원 옵션에서 데이터베이스를 만들 수 있습니다.

### 논리 서버

새 SQL 데이터 웨어하우스 데이터베이스 논리 서버에 상주합니다. 논리 서버는 많은 데이터베이스에 대한 구성 일관성을 제공하고 Azure 데이터 센터에 대한 서비스를 찾습니다.

설정해야 하는 옵션은 1입니다. 서버 이름 2입니다. 서버 관리자 이름 3입니다. 암호 4입니다. 데이터 센터 위치 5입니다. Azure 서비스가 서버에 액세스하도록 허용

자유롭게 필요에 따라 이 값을 설정할 수 있습니다. 서버 이름은 고유해야 합니다. 네트워크 대기 시간을 줄일 수 있도록 가까이 있는 데이터 센터를 선택하는 것이 좋습니다. SQL 데이터 웨어하우스는 또한 Azure의 다른 서비스를 활용하는 강력한 기능을 포함합니다. 따라서 Azure 서비스 액세스에 대한 확인란이 설정된 채로 두는 것이 좋습니다.

> [AZURE.NOTE]SQL 데이터 웨어하우스는 V12 서버를 사용해야 합니다. 이 옵션이 YES로 설정되어있는지 확인합니다. Azure SQL 데이터베이스 및 SQL 데이터 웨어하우스 데이터베이스에서 논리 서버를 공유할 수도 있습니다. 그러나 V12 서버여야 합니다.

> [AZURE.NOTE]서버 이름, 서버 관리자 이름 및 암호를 어딘가에 기록하여 안전하게 둡니다. SQL 데이터 웨어하우스 데이터베이스에 연결하는 데 이 정보가 필요합니다.

### 리소스 그룹
리소스 그룹은 컨테이너로, Azure 리소스의 컬렉션을 관리할 수 있도록 디자인되었습니다.

이 빠른 시작에서는 기본값에서 구성된 리소스 그룹을 유지해도 괜찮습니다.

[리소스 그룹](../azure-portal/resource-group-portal.md)에 대해 알아봅니다.

### 구독
단일 사용자는 Azure를 하나 이상 구독할 수 있습니다. 사용자 로그인과 연관된 둘 이상의 구독이 있는 경우 사용할 구독을 선택할 수 있습니다.

그러나 이 가이드의 목적을 위해서는 기본값을 사용해야 합니다.

자 이제 SQL 데이터 웨어하우스를 만들어 봅니다!

## 데이터 웨어하우스 만들기 ##
만들기 단추를 클릭하면 데이터 웨어하우스 만들기가 완료됩니다.

축하합니다. 첫 번째 SQL 데이터 웨어하우스 데이터베이스를 만들었습니다.

이제 [Azure 포털][]로 돌아가야 합니다. SQL 데이터 웨어하우스 데이터베이스가 페이지에 추가된 것을 알 수 있습니다.


이 시점에서는 SQL 데이터 웨어하우스 데이터베이스에 액세스할 수 있는 사람은 없습니다. 모두 안전하게 유지하기 위해 기본적으로 데이터베이스는 아직 클라이언트가 액세스하도록 구성되지 않습니다.

따라서 프로비전 프로세스의 마지막 단계는 외부 액세스에 대한 서비스를 구성하는 것입니다.

## Azure 방화벽 구성 ##

처음으로 Azure 방화벽을 구성하려면:

1. 왼쪽 탐색 블레이드에서 **찾아보기**를 클릭합니다.

2. **SQL Server**를 선택합니다.

3. 논리적 SQL Server를 선택합니다.

4. 설정을 선택합니다.

5. **방화벽**을 클릭합니다.

6. 방화벽 규칙을 설정합니다.

    여기서 수행할 몇 가지 작업이 있습니다. 작업은 다음과 같습니다. - 방화벽 규칙에 이름을 지정합니다. - 고정 IP 주소가 없는 경우 IP 범위를 제공합니다.

    > [AZURE.NOTE]포함해야 하는 클라이언트 IP 주소 범위는 외부 또는 공용 IP 주소입니다. 사용자의 외부 IP 주소를 찾으려면 <a href="http://www.whatismyip.com" target="\_blank">www.whatismyip.com</a>과 같은 여러 웹 사이트를 사용할 수 있습니다.

7. 방화벽 규칙을 저장합니다.


이제 방화벽을 구성했으므로 바탕 화면에서 방금 만든 Azure SQL 데이터 웨어하우스에 연결할 수 있습니다.
