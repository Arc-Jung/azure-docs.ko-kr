# Azure SQL 데이터 웨어하우스란?

Azure SQL 데이터 웨어하우스는 페타바이트 볼륨의 관계형 및 비관계형 데이터를 처리할 수 있는 엔터프라이즈급 분산 데이터베이스입니다. 입증된 SQL 기능을 사용하여 몇 초 내에 확장, 축소 및 일시 중지가 가능한 업계 최초의 클라우드 데이터 웨어하우스입니다.

SQL 데이터 웨어하우스는 SQL Server의 엔터프라이즈급 관계형 데이터베이스 최고 품질 및 안정성을 클라우드 컴퓨팅과 혼합합니다. SQL 데이터 웨어하우스의 핵심 디자인 구성 요소에 대해 더 알아보세요.

## 계산 리소스를 확장, 축소 또는 일시 중지
SQL 데이터 웨어하우스는 각각 독립적으로 크기를 조정할 수 있는 분리된 저장소 및 계산을 도입합니다. SQL 데이터 웨어하우스를 통해 클라우드 규모 저장소 및 Azure 계산을 사용할 경우 필요할 때 필요한 만큼만 쿼리 성능에 대한 비용을 지불할 수 있습니다. 이제 몇 초 내에 계산을 동적으로 배포, 증가, 축소 및 일시 중지하여 비즈니스에 필요한 정보를 구입할 수 있습니다. 슬라이더를 왼쪽이나 오른쪽으로 이동하기만 하면 간단하게 계산량을 변경할 수 있습니다.

저장소와 독립적으로 계산량을 완벽하게 제어하는 기능을 사용하여 SQL 데이터 웨어하우스에서 계산을 일시 중지할 수 있습니다. 현재 위치를 유지하면서 계산이 중앙 풀로 다시 푸시되므로 비용이 절감됩니다. 필요할 때 계산을 다시 시작하고 데이터와 계산을 워크로드에 사용할 수 있게 하면 됩니다.

> [AZURE.NOTE]데이터 웨어하우스 단위(DWU)는 SQL 데이터 웨어하우스 내 계산 리소스에 대한 규모 단위입니다.

- 더 빠른 결과가 필요하다면 DWU를 늘리고 더 큰 숫자의 DWU에 대해 지불하고, 그 정도로 강력한 능력이 필요 없다면 DWU를 줄이고 되돌아가서 더 적은 양의 DWU에 대해 지불하세요. 계산 능력은 DWU의 수에 비례하여 증가합니다. DWU가 두 배가 되면 계산 리소스도 두 배가 됩니다. 
- 저녁이나 주말에 쿼리를 실행하지 않아도 된다면 계산 리소스를 일시 중지하여 실행 중인 모든 쿼리를 취소하고 데이터 웨어하우스에 할당된 모든 DWU를 제거합니다. 데이터 저장소는 그대로 유지되지만 계산 리소스는 무료입니다. 월요일 아침에 쿼리 실행을 시작해야 하는 경우 계산 리소스를 다시 시작해도 됩니다. 

## 획기적인 성능을 위한 대규모 병렬 처리 및 Columnstore 인덱스
SQL 데이터 웨어하우스는 Microsoft의 대규모 병렬 처리(MPP) 아키텍처 및 SQL Server의 Columnstore 인덱스 기술을 사용하여 획기적인 성능을 제공합니다.

> [AZURE.NOTE]MPP는 병렬 컴퓨팅을 사용하여 대규모 데이터 문제를 해결할 수 있는 분할되고 정리된 방식입니다. 데이터가 많은 컴퓨팅 리소스에 걸쳐 분할 및 분산되며 각 컴퓨팅 리소스가 병렬로 데이터의 해당 부분에서 작동합니다.

- 그 이면에 있는 비법은 Microsoft의 분산 쿼리 기술입니다. SQL 데이터 웨어하우스는 쿼리 작업의 비용 평가를 기반으로 분산된 쿼리를 최적화하는 방법을 파악하는 고급 쿼리 최적화 프로그램을 사용합니다. 또한, 컴퓨팅 리소스 사이에서 쿼리 수행에 필요한 대로 데이터를 효율적으로 이동해주는 고급 알고리즘 및 기술도 제공합니다.
- Columnstore 인덱스는 데이터 웨어하우스 쿼리에서 빠른 쿼리 성능을 달성하는 데 있어 핵심입니다. Columnstore 인덱스는 열 기반 저장소를 사용하여 기존의 행 기반 저장소와 비교하여 최대 5배 높은 압축 향상, 최대 10배 높은 쿼리 성능 향상을 달성할 수 있습니다. 데이터 웨어하우스 쿼리는 일반적으로 전체 테이블 또는 테이블의 전체 파티션을 검색하기 때문에 Columnstore 인덱스에서 잘 작동합니다. 반면에 OLTP 쿼리는 테이블에서 특정 행을 검색하기 때문에 이진 트리 인덱스에서 잘 작동합니다.


## 엔터프라이즈급 SQL Server 경험을 제공하는 하이브리드 클라우드
SQL 데이터 웨어하우스는 SQL Server의 입증된 관계형 데이터베이스 엔진을 기반으로 하며, 저장된 프로시저, 사용자 정의 함수, 테이블 분할, 인덱스 및 데이터 정렬 등 엔터프라이즈 데이터 웨어하우스에서 기대하는 기능들을 포함하고 있습니다.

> [AZURE.NOTE]이미 TRANSACT-SQL에 대해 알고 있다면 해당 지식을 쉽게 SQL 데이터 웨어하우스에 전송할 수 있습니다. 고급 사용자이든 초보 사용자이든 상관 없이 설명서의 예제들이 시작에 도움을 줄 것입니다.

- SQL 데이터 웨어하우스는 SQL Server의 Transact-SQL, Columnstore 인덱스 및 PolyBase 기술을 분석 플랫폼 시스템의 대용량 병렬 처리(MPP) 아키텍처와 함께 사용하여 이 고유하고 통합된 PaaS(Platform-as-a-Service) 데이터 웨어하우스 경험을 창출합니다.  

- SQL Server, SQL 데이터 웨어하우스, SQL 데이터베이스 및 분석 플랫폼 시스템 간의 TRANSACT-SQL 및 기능 공통성을 통해 데이터 요구 사항에 맞는 솔루션을 개발할 수 있습니다. 성능, 보안 및 확장 요구 사항에 따라 데이터를 보관할 위치를 결정한 다음, 필요에 따라 데이터를 온-프레미스와 클라우드 사이에 전송할 수 있습니다.


## 관계형 및 비관계형 모두에 대한 쿼리
Polybase를 통해 일반 테이블처럼 Azure blob 저장소 또는 Hadoop 파일 시스템(HDFS)에 보관 중인 비관계형 데이터를 쿼리할 수 있습니다. Polybase를 사용하여 비관계형 데이터를 쿼리하거나 비관계형 데이터를 SQL 데이터 웨어하우스로 가져옵니다.

> [AZURE.NOTE]Polybase는 사용이 쉬우며 사용자에게 이미 익숙한 동일한 T-SQL 명령을 사용하여 다양한 원본에서 데이터를 활용할 수 있게 해줍니다. Hadoop을 활용하기 위해 HiveQL 또는 기타 언어를 배울 필요가 없습니다.

- Polybase는 통합 시 중립적입니다. Hadoop의 모든 배포에 지원되는 모든 기능과 특징을 노출합니다. Polybase에서 읽는 데이터는 구분된 파일 또는 ORC 파일 등 다양한 형식일 수 있습니다.
- PolyBase는 외부 테이블을 사용하여 비관계형 데이터에 액세스합니다. 테이블 정의는 SQL 데이터 웨어하우스에 저장되며 데이터는 Hadoop 또는 Azure blob 저장소에 외부적으로 저장됩니다.


## 유지 관리 비용 없이 인프라 배포 보호
SQL 데이터 웨어하우스는 몇 초 만에 쉽게 배포됩니다. 이 서비스는 번거롭게 소프트웨어를 패치 및 유지 관리해야 할 필요성을 제거하는 완전히 관리되는 제품입니다. SQL 데이터 웨어하우스에는 백업 기능이 내장되어 있어서 셀프 서비스 복원을 지원하며, 서비스가 데이터 복원 지점의 스냅숏을 만들면서 자동으로 데이터를 Azure 저장소에 백업합니다.
