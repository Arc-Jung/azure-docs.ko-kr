# SQL 데이터 웨어하우스 미리 보기 기대치

이 문서는 SQL 데이터 웨어하우스 미리 보기 기능 및 GA(일반 공급)의 서비스에 대한 목표에 대해 설명합니다. 공개 미리 보기 기능을 강화함에 따라 이 정보는 지속적으로 업데이트됩니다.

SQL 데이터 웨어하우스의 목표:

- 예측 가능한 성능 및 페타바이트 크기 데이터까지의 선형 확장성.
- SLA(서비스 수준 계약)에 따른 모든 데이터 웨어하우스 작업에 대한 높은 안정성.

SQL 데이터 웨어하우스를 일반 공급으로 올리기에 앞서 이러한 목표를 지향하여 끊임 없이 노력할 것입니다.

## 예측 가능하고 확장 가능한 성능

Azure SQL 데이터 웨어하우스는 데이터 웨어하우스에 사용할 수 있는 컴퓨팅 리소스(CPU, 메모리, 저장소 I/O)를 측정하는 방법으로 DWU(데이터 웨어하우스 단위)를 소개합니다. DWU 수가 증가하면 리소스가 증가합니다. DWU 수가 증가하면 SQL 데이터 웨어하우스가 더 많이 배포된 리소스를 통해 작업을 병렬로 실행합니다(예: 데이터 로드 또는 쿼리). 대기 시간이 줄어들고 성능이 향상됩니다.

데이터 웨어하우스에는 다음과 같이 2가지 기본 성능 메트릭이 있습니다.

- 로드 비율입니다. 초당 데이터 웨어하우스로 로드될 수 있는 레코드 수입니다. 특히 PolyBase를 통해 Azure Blob 저장소에서 클러스터형 열-저장소 인덱스가 있는 테이블로 가져올 수 있는 레코드 수를 측정합니다. 
- 검색 비율입니다. 초당 데이터 웨어하우스에서 순차적으로 검색될 수 있는 레코드 수입니다. 특히 클러스터형 열-저장소 인덱스에 있는 쿼리에 의해 반환된 레코드의 수를 측정합니다.


일부 중요한 성능 개선 사항을 측정하고 있으며 곧 예상 비율을 공유할 예정입니다. 미리 보기 중에 이러한 비율을 높이고 예측 가능하게 확장할 수 있도록 지속적인 개선에 힘쓰겠습니다(예: 압축 및 캐싱 증가).


## SLA를 통한 높은 안정성

### 데이터 보호 

SQL 데이터 웨어하우스는 지리적으로 중복된 Blob을 사용하여 Azure 저장소에 모든 데이터를 저장합니다. 데이터의 동기 복사본 세 개는 지역화된 오류(예: 저장소 드라이브 오류) 발생 시 투명 한 데이터 보호를 보장하기 위해 로컬 Azure 지역에 유지됩니다. 또한 자세한 비동기 복사본 세 개 (재해 복구) 국가별 오류가 발생할 경우 데이터 보호를 보장하기 위해 원격 Azure 지역에 유지됩니다. 허용 가능한 동기화 대기 시간(예: 미국 동부 및 미국 서부)을 유지하기 위해 로컬 및 원격 지역이 쌍을 이루고 있습니다.


### 백업

Azure SQL 데이터 웨어하우스는 Azure 저장소 스냅숏을 사용하여 적어도 8시간마다 모든 데이터를 백업합니다. 이러한 스냅숏은 7일 동안 유지됩니다. 이를 통해 지난 7일 내에 적어도 21개의 특정 시점까지, 마지막 스냅숏 작업 시까지 데이터를 복원할 수 있습니다. PowerShell 또는 REST API를 사용하여 스냅숏으로부터 데이터를 복원할 수 있습니다.

스냅숏은 지역별 오류(재해 복구) 시 추가된 복구 기능을 위해 원격 Azure 지역에 비동기적으로 복사됩니다.


### 쿼리 완료 

SQL 데이터 웨어하우스는 해당 데이터에 사용자 데이터 및 제어 쿼리 실행의 일부를 각각 저장하는 하나 이상의 계산 노드 간에 데이터를 저장합니다. MPP(대규모 병렬 처리) 아키텍처의 일환으로 쿼리는 계산 노드에 걸쳐 병렬로 실행합니다. SQL 데이터 웨어하우스는 계산 노드 오류를 자동으로 감지 및 완화합니다. 하지만 미리 보기 중에 작업(예: 데이터 로드 또는 쿼리)은 개별 노드 오류로 인해 실패할 수 있습니다. 미리 보기 중에 노드 오류에도 불구하고 작업을 성공적으로 완료하기 위해 지속적으로 개선하고 있습니다.

원격 분석 데이터에 기반하여 Azure SQL 데이터 웨어하우스의 현재 안정성을 98%로 추정합니다. 즉, 평균적으로 100개의 쿼리 중 2개가 시스템 오류로 인해 실패할 수 있다는 뜻입니다. SLA는 아닙니다. 쿼리 실패의 확률은 실행 시간에 따라 증가합니다. 예를 들어 2시간 이상 실행하는 쿼리는 10분 미만으로 실행하는 쿼리보다 실패 확률이 훨씬 높습니다. 미리 보기 중에 실행 시간에 상관 없이 동일한 수준의 안정성을 보장하도록 안정성을 개선하고 있습니다. 이러한 개선 사항이 출시되면 기대한 안정성이 업데이트될 예정입니다. 일반 공급의 경우 SLA를 통해 안정성이 지원됩니다.

### 서비스 가동 시간

Azure SQL 데이터 웨어하우스는 중요한 수정 사항을 설치하기 위해 매월 최대 4개의 유지보수 이벤트를 보유할 수 있습니다. 각 이벤트는 최대 2시간 동안 쿼리 오류를 유발할 수 있습니다. 시간은 서비스에 할당된 DWU 수에 따라 달라집니다. 이러한 이벤트를 48시간 전에 알리기 위해 최선을 다 할 것입니다.
