# <a name="manage-cluster-horizontal-scaling-to-accommodate-changing-demand"></a>변화 하는 요구를 수용 하기 위해 수평적 확장 클러스터 관리

클러스터 크기를 적절하게 조정하는 것은 Azure 데이터 탐색기의 성능에 중요합니다. 하지만 절대 정확도 사용 하 여 클러스터에 대 한 수요를 예측할 수 없습니다. 정적 클러스터 크기는 모두 적합 미달 사용 또는 초과 사용, 발생할 수 있습니다.

가장 좋은 방법은 변화하는 요구에 따라 용량을 추가하고 제거하여 클러스터의 *크기를 조정*하는 것입니다. 크기 조정에 대 한 두 가지 워크플로 가지가 있습니다. 
* 확장 및 축소 라고도 수평적 크기 조정 합니다.
* 확장 및 축소는 라고도 수직적 크기 조정 합니다.

이 문서는 가로 크기 조정 워크플로 설명합니다.

수평적 크기 조정 기능을 사용 하면 미리 정의 된 규칙 및 일정에 따라 자동으로 인스턴스 개수를 조정 합니다. 이 문서에 설명 된 대로 Azure portal에서 클러스터에 대 한 자동 크기 조정 설정을 지정 합니다.

## <a name="steps-to-configure-horizontal-scaling"></a>수평적 크기 조정 구성 하는 단계

Azure portal에서 데이터 탐색기 클러스터 리소스를 이동 합니다. 아래는 **설정을** 을 선택 **규모**합니다. 

원하는 자동 크기 조정 메서드를 선택 합니다. **수동 크기 조정**, **액세스에 최적화 된 자동 크기 조정** 하거나 **사용자 지정 자동 크기 조정**합니다.

### <a name="manual-scale"></a>수동 크기 조정

수동 크기 조정에는 클러스터 생성을 사용 하 여 기본 설정입니다. 즉, 클러스터 자동으로 변경 되지 않는 정적 클러스터 용량이 있어야 합니다. 정적 용량 막대를 사용 하 여 선택한 다음 클러스터의 확장 설정 변경 될 때까지 변경 되지 않습니다.

   ![수동 크기 조정 메서드](media/manage-cluster-horizontal-scaling/manual-scale-method.png)

### <a name="optimized-autoscale"></a>최적화 된 자동 크기 조정

최적화 된 자동 크기 조정 권장 되는 자동 크기 조정 방법입니다. 액세스에 최적화 된 자동 크기 조정을 구성 하는 단계:

1. 액세스에 최적화 된 자동 크기 조정 옵션을 선택 하 고 하한값 및 상한값 클러스터 인스턴스의 크기를 선택 합니다. 자동 크기 조정 이러한 제한을 간에 수행 됩니다.
2. 저장을 클릭합니다.

   ![최적화 된 자동 크기 조정 메서드](media/manage-cluster-horizontal-scaling/optimized-autoscale-method.png)

작동 하기 시작 액세스에 최적화 된 자동 크기 조정 메커니즘 저장을 클릭 하 고이 작업은 클러스터의 활동 로그에 표시 됩니다. 이 자동 크기 조정 메서드 클러스터 성능 및 비용을 최적화 하는: 클러스터 성능와 낮은 비용을 유지 하는 것은 수 확장에 미달 사용 상태를 가져오기 시작 하 고 클러스터의 상태가 시작 하는 경우 초과 사용을는 것이 제대로 작동 되도록 규모가 확장

### <a name="custom-autoscale"></a>사용자 지정 자동 크기 조정

사용자 지정 자동 크기 조정 방법을 사용 하면 수 동적으로 메트릭을 기반으로 지정 하는 클러스터 크기를 조정 합니다. 다음 그림에는 흐름 및 사용자 지정 자동 크기 조정을 구성 하는 단계를 보여 줍니다. 자세한 내용은 아래 그래픽

1. 에 **자동 크기 조정 설정 이름** 상자에 같은 이름을 제공 *스케일 아웃: 캐시 사용률*합니다. 

   ![크기 조정 규칙](media/manage-cluster-horizontal-scaling/custom-autoscale-method.png)

2. 에 대 한 **크기 조정 모드**를 선택 **메트릭에 따라 크기 조정**합니다. 이 모드는 동적 확장을 제공 합니다. 선택할 수도 있습니다 **특정 인스턴스 수로 크기 조정**합니다.

3. 선택 **규칙 추가 +** 합니다.

4. 오른쪽의 **크기 조정 규칙** 섹션에서 각 설정의 값을 지정합니다.

    **조건**

    | 설정 | 설명 및 값 |
    | --- | --- |
    | **시간 집계** | **평균** 같은 집계 기준을 선택합니다. |
    | **메트릭 이름** | **캐시 사용률**과 같이 크기 조정 작업의 기준이 될 메트릭을 선택합니다. |
    | **시간 조직 통계** | **평균**, **최솟값**, **최댓값**, **합계** 중에서 선택합니다. |
    | **Operator** | **보다 크거나 같음**과 같은 적절한 옵션을 선택합니다. |
    | **임계값** | 적절한 값을 선택합니다. 예를 들어, 캐시 사용률에 대 한 80%는 좋은 출발점입니다. |
    | **기간(분)** | 시스템이 메트릭을 계산하는 경우에 다시 확인하는 적절한 기간을 선택합니다. 처음에는 기본값 10분으로 시작합니다. |
    |  |  |

    **동작**

    | 설정 | 설명 및 값 |
    | --- | --- |
    | **연산** | 규모를 감축하거나 규모를 확장하는 적절한 옵션을 선택합니다. |
    | **인스턴스 수** | 메트릭 조건을 충족할 경우 추가하거나 제거할 노드 또는 인스턴스 수를 선택합니다. |
    | **정지(분)** | 크기 조정 작업 간의 적절한 시간 간격을 선택합니다. 기본값 5분으로 시작합니다. |
    |  |  |

5. **추가**를 선택합니다.

6. 왼쪽의 **인스턴스 제한** 섹션에서 각 설정의 값을 지정합니다.

    | 설정 | 설명 및 값 |
    | --- | --- |
    | **최소** | 사용률에 관계없이 클러스터가 인스턴스 수를 이보다 낮게 줄이지 않습니다. |
    | **최대값** | 사용률에 관계없이 클러스터가 인스턴스 수를 이보다 높게 늘리지 않습니다. |
    | **Default** | 인스턴스의 기본 수입니다. 이 설정은 사용 하 여 리소스 메트릭을 읽는 데 문제가 있으면 됩니다. |
    |  |  |

7.           **저장**을 선택합니다.

이제 Azure 데이터 탐색기 클러스터에 규모 확장 작업을 구성했습니다. 규모 감축 작업에 대한 다른 규칙을 추가합니다. 클러스터 크기 조정 문제를 사용 하 여 도움이 필요 하면 [지원 요청을 여세요](https://portal.azure.com/#blade/Microsoft_Azure_Support/HelpAndSupportBlade/overview) Azure portal에서 합니다.
