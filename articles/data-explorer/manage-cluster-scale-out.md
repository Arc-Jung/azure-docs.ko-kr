# <a name="manage-cluster-scale-out-to-accommodate-changing-demand"></a>변화하는 요구를 수용하도록 클러스터 확장 관리

클러스터 크기를 적절하게 조정하는 것은 Azure 데이터 탐색기의 성능에 중요합니다. 하지만 100% 정확도로 클러스터에 대한 요구를 예측할 수 없습니다. 정적 클러스터 크기는 사용량 미달 또는 사용량 초과로 이어질 수 있으며, 둘 다 이상적인 경우는 아닙니다. 가장 좋은 방법은 변화하는 요구에 따라 용량을 추가하고 제거하여 클러스터의 *크기를 조정*하는 것입니다. 이 문서에서는 클러스터 확장을 관리하는 방법을 보여 줍니다.

클러스터로 이동하고 **설정** 아래에서 **규모 확장**을 선택합니다. **구성** 아래에서 **자동 크기 조정 사용**을 선택합니다.

![자동 크기 조정 사용](media/manage-cluster-scaling/enable-autoscale.png)

다음은 다음 몇 단계의 흐름을 보여 주는 그래픽입니다. 자세한 내용은 아래 그래픽 아래에 나와 있습니다.

![크기 조정 규칙](media/manage-cluster-scaling/scale-rule.png)

1. **자동 크기 조정 설정 이름**에 *Scale-out: cache utilization*과 같은 이름을 지정합니다.

1. **크기 조정 모드** 아래에서 **메트릭 기준 크기 조정**을 선택합니다. 이 모드에서는 동적 크기 조정을 지원합니다. **특정 인스턴스 수로 크기 조정**을 선택할 수도 있습니다.

1. **규칙 추가**를 선택합니다.

1. 오른쪽의 **크기 조정 규칙** 섹션에서 각 설정의 값을 지정합니다.

    | 설정 | 설명 및 값 |
    | --- | --- | --- |
    | **시간 집계** | **평균** 같은 집계 기준을 선택합니다. |
    | **메트릭 이름** | **캐시 사용률**과 같이 크기 조정 작업의 기준이 될 메트릭을 선택합니다. |
    | **시간 조직 통계** | **평균**, **최솟값**, **최댓값**, **합계** 중에서 선택합니다. |
    | **연산자** | **보다 크거나 같음**과 같은 적절한 옵션을 선택합니다. |
    | **임계값** | 적절한 값을 선택합니다. 예를 들어, 캐시 사용률의 경우 80%가 좋은 시작점입니다. |
    | **Duration** | 시스템이 메트릭을 계산하는 경우에 다시 확인하는 적절한 기간을 선택합니다. 기본값 10분으로 시작합니다. |
    | **작업** | 규모를 감축하거나 규모를 확장하는 적절한 옵션을 선택합니다. |
    | **인스턴스 수** | 메트릭 조건을 충족할 경우 추가하거나 제거할 노드 또는 인스턴스 수를 선택합니다. |
    | **정지(분)** | 크기 조정 작업 간의 적절한 시간 간격을 선택합니다. 기본값 5분으로 시작합니다. |
    |  |  |

1. **추가**를 선택합니다.

1. 왼쪽의 **인스턴스 제한** 섹션에서 각 설정의 값을 지정합니다.

    | 설정 | 설명 및 값 |
    | --- | --- | --- |
    | *최소* | 사용률에 관계없이 클러스터는 이 인스턴스 수 밑으로 더 이상 축소되지 않습니다. |
    | *최댓값* | 사용률에 관계없이 클러스터는 이 인스턴스 수 위로 더 이상 확장되지 않습니다. |
    | *기본값* | 리소스 메트릭을 읽는 데 문제가 있을 경우에 사용되는 인스턴스의 기본 수입니다. |
    |  |  |

1. **저장**을 선택합니다.

이제 Azure 데이터 탐색기 클러스터에 규모 확장 작업을 구성했습니다. 규모 감축 작업에 대한 다른 규칙을 추가합니다. 이렇게 하면 자신이 지정하는 메트릭에 따라 클러스터 크기를 동적으로 조정할 수 있습니다.

클러스터 크기 조정 문제와 관련하여 지원이 필요한 경우에는 [Azure Portal](https://portal.azure.com/#blade/Microsoft_Azure_Support/HelpAndSupportBlade/overview)에서 지원 요청을 개설하세요.
