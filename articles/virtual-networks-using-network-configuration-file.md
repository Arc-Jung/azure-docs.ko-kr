#네트워크 구성 파일을 사용하여 가상 네트워크 구성

## 개요

가상 네트워크를 구성하려는 경우 관리 포털 마법사를 사용하거나, 네트워크 구성 파일을 만들고 가져올 수 있습니다. Azure에서는 네트워크 구성 파일을 사용하여 가상 네트워크 설정을 정의합니다. 

초기에는 관리 포털 마법사를 사용하여 가상 네트워크 구성을 만든 다음 이후에 구성 파일을 직접 편집하여 변경하는 것을 선호할 수 있습니다. 예를 들어 별도의 구독에 대해 여러 가상 네트워크를 구성하는 경우 처음에 관리 포털 마법사를 사용하여 네트워크 구성 파일을 만들 수 있습니다. 그런 다음 템플릿으로 사용할 파일을 내보내고 이를 편집하여 다른 설정을 지정한 다음 파일을 관리 포털로 다시 가져올 수 있습니다. 이는 둘 이상의 구독이 있을 때 여러 가상 네트워크를 만드는 효율적인 방법일 수 있습니다. 

또는 클라우드 서비스나 가상 컴퓨터를 네트워크에 배포하기 전에 네트워크 구성 설정을 변경하려는 경우 파일을 내보내 편집한 다음 Azure로 다시 가져올 수 있습니다. 또한 가상 네트워크를 다시 만들려는 경우 네트워크 구성 파일을 사용하여 네트워크 구성 설정을 백업할 수 있습니다.

##네트워크 구성 파일 만들기 및 수정 
네트워크 구성 파일을 작성하는 가장 간단한 방법은 기존 가상 네트워크 구성에서 네트워크 설정을 내보낸 다음 가상 네트워크에 대해 구성할 설정을 포함하도록 파일을 수정하는 것입니다. 샘플 파일을 가져와 이를 수정할 수도 있습니다.

네트워크 구성 파일을 편집하려면 파일을 열고 적절히 변경한 다음 저장하기만 하면 됩니다. 모든 *xml* 편집기를 사용하여 네트워크 구성 파일을 변경할 수 있습니다. 

네트워크 구성 파일 스키마 설정에 대한 지침을 엄격히 준수해야 합니다. 네트워크 구성 파일을 작성하면 파일 내의 설정이 현재 Azure 구독에 대한 설정을 덮어씁니다. 파일의 값을 설정 지침과 다르게 변경하면 가상 네트워크가 의도한 방식으로 구성되지 않을 수 있으며, 경우에 따라 Azure에서 파일을 가져올 수 없게 됩니다. 네트워크 구성 파일에 포함된 특정 설정에 대한 자세한 내용은 [Azure 가상 네트워크 구성 스키마](https://msdn.microsoft.com/library/azure/jj157100.aspx)를 참조하세요. 

Azure에서는 배포된 항목이 있는 서브넷을 "사용 중"인 것으로 간주합니다. 사용 중인 서브넷은 수정할 수 없습니다. 수정하기 전에 서브넷에 배포한 항목을 수정 중이지 않은 다른 서브넷으로 이동하세요.   [다른 서브넷으로 VM 또는 역할 인스턴스 이동](https://msdn.microsoft.com/library/azure/dn643636.aspx)을 참조하세요.



## 관리 포털을 사용하여 가상 네트워크 내보내기 및 가져오기  
PowerShell 또는 관리 포털을 사용하여 네트워크 구성 파일에 포함된 내보내기 네트워크 구성 설정을 가져올 수 있습니다. 다음은 관리 포털을 사용하여 내보내고 가져오는 데 유용한 지침입니다. 

### 네트워크 설정을 내보내려면
내보내면 구독의 가상 네트워크에 대한 모든 설정이 .xml 파일에 기록됩니다. 

1. **관리 포털**에 로그인합니다.
2. 관리 포털의 **네트워크** 페이지 맨 아래에서 **내보내기**를 클릭합니다. 
3. **네트워크 구성 내보내기** 창에서 네트워크 설정을 내보낼 구독을 선택했는지 확인합니다. 그런 다음 오른쪽 아래에 있는 확인 표시를 클릭합니다. 
4. 메시지가 나타나면 *NetworkConfig.xml* 파일을 선택한 위치에 저장합니다.
### 네트워크 설정을 가져오려면


1. **관리 포털**의 왼쪽 아래에 있는 탐색 창에서 **새로 만들기**를 클릭합니다.
2. **네트워크 서비스** -> **가상 네트워크** -> **구성 가져오기**를 클릭합니다.
3. **네트워크 구성 파일 가져오기** 페이지에서 네트워크 구성 파일을 찾은 후 **다음** 화살표를 클릭합니다.
4. **네트워크 빌드** 페이지의 화면에 변경하거나 만들 네트워크 구성 섹션을 보여 주는 정보가 표시됩니다. 변경 사항이 올바른 경우 확인 표시를 클릭하여 가상 네트워크를 계속 업데이트하거나 만듭니다.
