# <a name="troubleshoot-azure-netapp-files-resource-provider-errors"></a>Azure NetApp Files 리소스 공급자 오류 문제 해결 

이 문서에서는 일반적인 Azure NetApp 파일 리소스 공급자 오류, 원인, 솔루션 및 해결 방법(사용 가능한 경우)에 대해 설명합니다.

## <a name="common-azure-netapp-files-resource-provider-errors"></a>일반적인 Azure NetApp 파일 리소스 공급자 오류

***베어메탈테넌티드는 변경할 수 없습니다.***  

이 오류는 볼륨을 업데이트하거나 패치하려고 `BaremetalTenantId` 할 때 발생하며 속성에 변경된 값이 있습니다.

* 원인:   
볼륨을 업데이트하려고 하고 속성의 `BaremetalTenantId` 값이 Azure에 저장된 값과 다릅니다.
* 해결 방법:   
패치 및 `BaremetalTenantId` 업데이트(put) 요청에 포함되지 마십시오. 또는 요청에서 `BaremetalTenantId` 동일한지 확인합니다.

***서비스 수준은 변경할 수 없습니다.***  

이 오류는 용량 풀에 이미 볼륨이 있는 경우 다른 서비스 수준으로 용량 풀을 업데이트하거나 패치하려고 할 때 발생합니다.

* 원인:   
풀에 볼륨이 포함된 경우 용량 풀 서비스 수준을 업데이트하려고 합니다.
* 해결 방법:   
용량 풀에서 모든 볼륨을 삭제한 다음 서비스 수준을 변경합니다.
* 해결 방법:   
다른 용량 풀을 만든 다음 새 용량 풀에서 볼륨을 다시 만듭니다.

***풀Id를 변경할 수 없습니다.***  

이 오류는 변경된 `PoolId` 속성으로 용량 풀을 업데이트하거나 패치하려고 할 때 발생합니다.

* 원인:   
용량 풀 `PoolId` 속성을 업데이트하려고 합니다. 이 `PoolId` 속성은 읽기 전용 속성이며 변경할 수 없습니다.
* 해결 방법:   
패치 및 `PoolId` 업데이트(put) 요청에 포함되지 마십시오.  또는 요청에서 `PoolId` 동일한지 확인합니다.

***생성 토큰은 변경할 수 없습니다.***

이 오류는 볼륨을 만든 후`CreationToken`파일 경로 () 변경 하려고 할 때 발생 합니다. 파일 경로`CreationToken`() 볼륨을 만들 때 설정 해야 하 고 나중에 변경할 수 없습니다.

* 원인:   
볼륨을 만든 후 지원되는`CreationToken`작업이 아닌 파일 경로() 변경하려고 합니다. 
* 해결 방법:   
파일 경로를 변경할 필요가 없는 경우 요청에서 매개 변수를 제거하여 오류 메시지를 해제하는 것이 좋습니다.
* 해결 방법:   
파일 경로()를`CreationToken`변경해야 하는 경우 새 파일 경로를 사용하여 새 볼륨을 만든 다음 데이터를 새 볼륨으로 마이그레이션할 수 있습니다.

***CreationToken은 16자 이상이어야 합니다.***

이 오류는 파일 경로`CreationToken`() 길이 요구 사항을 충족 하지 않는 경우에 발생 합니다. 파일 경로의 길이는 길이가 하나 이상이어야 합니다.

* 원인:   
파일 경로가 비어 있습니다.  API를 사용하여 볼륨을 만들 때 생성 토큰이 필요합니다. Azure 포털을 사용하는 경우 파일 경로가 자동으로 생성됩니다.
* 해결 방법:   
하나 이상의 문자를 파일 경로()로`CreationToken`입력합니다.

***도메인 이름은 변경할 수 없습니다.***

이 오류는 Active Directory에서 도메인 이름을 변경하려고 할 때 발생합니다.

* 원인:   
도메인 이름 속성을 업데이트하려고 합니다.
* 해결 방법:    
없음 도메인 이름은 변경할 수 없습니다.
* 해결 방법:   
Active Directory 구성을 사용하여 모든 볼륨을 삭제합니다. 그런 다음 Active Directory 구성을 삭제하고 볼륨을 다시 만듭니다.

***개체 ExportPolicy.Rules[RuleIndex]에 대한 중복 값 오류입니다.***

이 오류는 내보내기 정책이 고유 인덱스로 정의되지 않은 경우에 발생합니다. 내보내기 정책을 정의할 때 모든 내보내기 정책 규칙에는 1에서 5 사이의 고유한 인덱스가 있어야 합니다.

* 원인:   
정의된 내보내기 정책이 내보내기 정책 규칙에 대한 요구 사항을 충족하지 않습니다. 최소 한 개의 내보내기 정책 규칙과 최대 5개의 내보내기 정책 규칙이 있어야 합니다.
* 해결 방법:   
인덱스가 아직 사용되지 않았는지, 인덱스가 1에서 5까지의 범위에 있는지 확인합니다.
* 해결 방법:   
설정하려는 규칙에 대해 다른 인덱스를 사용합니다.

***오류 {작업} {리소스TypeName}***

이 오류는 리소스에서 작업을 수행하는 동안 다른 오류 처리가 오류를 처리하지 못했을 때 표시됩니다.   여기에는 텍스트 '오류'가 포함됩니다. 는 `{action}` `getting`(의) `creating`및 `updating` `deleting`수 있습니다.  의 `{resourceTypeName}` 는 `resourceTypeName` (예 `netAppAccount`: `capacityPool` `volume`, , 및 등등)입니다.

* 원인:   
이 오류는 원인을 알 수 없는 처리되지 않은 예외입니다.
* 해결 방법:   
Azure 지원 센터에 문의하여 로그에 대한 자세한 이유를 보고합니다.
* 해결 방법:   
없음

***파일 경로 이름에는 문자, 숫자 및 하이픈(""-"")만 포함될 수 있습니다.***

이 오류는 파일 경로에 지원되지 않는 문자(예: 마침표(""""),쉼표("""""),밑줄("_")" 또는 달러 기호("$")가 포함되어 있는 경우에 발생합니다.

* 원인:   
파일 경로에는 지원되지 않는 문자(예: 마침표(""""), 쉼표(""""),밑줄("_") 또는 달러 기호("$")가 포함되어 있습니다.
* 해결 방법:   
입력한 파일 경로에서 알파벳 문자, 숫자 또는 하이픈("-")이 아닌 문자를 제거합니다.
* 해결 방법:   
밑줄을 하이픈으로 바꾸거나 공백 대신 대문자로 사용하여 새 단어의 시작을 나타낼 수 있습니다.  예를 들어 "새 볼륨" 대신 "NewVolume"을 사용합니다.

***FileSystemId를 변경할 수 없습니다.***

이 오류는 을 변경하려고 `FileSystemId`할 때 발생합니다.  변경은 `FileSystemdId` 지원되는 작업이 아닙니다. 

* 원인:   
볼륨을 만들 때 파일 시스템의 ID가 설정됩니다. `FileSystemId`이후에는 변경할 수 없습니다.
* 해결 방법:   
패치 및 `FileSystemId` 업데이트(put) 요청에 포함되지 마십시오.  또는 요청에서 `FileSystemId` 동일한지 확인합니다.

***ID가 있는 ActiveDirectory: '{문자열}'이 존재하지 않습니다.***

`{string}` 부분은 Active Directory 연결에 `ActiveDirectoryId` 대해 속성에 입력한 값입니다.

* 원인:   
Active Directory 구성으로 계정을 만들 때 비어 있어야 `ActiveDirectoryId` 하는 값을 입력했습니다.
* 해결 방법:   
만들기(put) `ActiveDirectoryId` 요청에 포함되지 마십시오.

***잘못된 api 버전입니다.***

API 버전이 제출되지 않았거나 잘못된 값을 포함합니다.

* 원인:   
쿼리 매개 변수의 `api-version` 값에는 잘못된 값이 포함되어 있습니다.
* 해결 방법:   
올바른 API 버전 값을 사용합니다.  리소스 공급자는 많은 API 버전을 지원합니다. 값은 yyy-mm-dd 의 형식입니다.

***에 대해 {1}잘못된 값 '{value}'를 받았습니다.***

`RuleIndex`이 메시지는 에 대한 `AllowedClients` `UnixReadOnly` `UnixReadWrite` `Nfsv3` `Nfsv4`필드의 오류를 나타냅니다.

* 원인:   
`RuleIndex`입력 유효성 검사 요청이 다음 필드 `AllowedClients` `UnixReadOnly` `UnixReadWrite` `Nfsv` `Nfsv4`중 하나 이상에 대해 실패했습니다.
* 해결 방법:   
명령줄에 필요한 매개 변수와 충돌하지 않는 모든 매개 변수를 설정해야 합니다. 예를 `UnixReadOnly` 들어, 매개 변수와 `UnixReadWrite` 매개 변수를 동시에 설정할 수 없습니다.
* 해결 방법:   
위의 솔루션을 참조하십시오.

***{0} vlan에 {1} {2} 대한 IP 범위가 이미 사용 중입니다.***

이 오류는 사용된 IP 범위의 내부 레코드가 새로 할당된 IP 주소와 충돌하기 때문에 발생합니다.

* 원인:   
볼륨 생성에 할당된 IP 주소가 이미 등록되어 있습니다.
그 이유는 이전에 실패한 볼륨 생성일 수 있습니다.
* 해결 방법:   
Azure 지원 센터에 문의하십시오.

***'{속성}'에 대한 값이 없습니다.***

이 오류는 요청에서 필수 속성이 누락되었음을 나타냅니다. 문자열 {property} 누락 된 속성의 이름을 포함 합니다.

* 원인:   
하나 이상의 속성에 대해 입력 유효성 검사 요청이 실패했습니다.
* 해결 방법:   
요청의 모든 필수 및 충돌없는 속성, 특히 오류 메시지의 속성을 설정해야 합니다.

***마운트 타겟은 변경할 수 없습니다.***

이 오류는 사용자가 볼륨 MountTargets 속성을 업데이트하거나 패치하려고 할 때 발생합니다.

* 원인:   
볼륨 `MountTargets` 속성을 업데이트하려고 합니다. 이 속성을 변경하는 것은 지원되지 않습니다.
* 해결 방법:   
패치 및 `MountTargets` 업데이트(put) 요청에 포함되지 마십시오.  또는 요청에서 동일한지 확인합니다. `MountTargets`

***이미 사용 중인 이름입니다.***

이 오류는 리소스의 이름이 이미 사용 중임을 나타냅니다.

* 원인:   
기존 리소스에 사용되는 이름으로 리소스를 만들려고 합니다.
* 해결 방법:   
리소스를 만들 때 고유한 이름을 사용합니다.

***이미 사용 중인 파일 경로입니다.***

이 오류는 볼륨의 파일 경로가 이미 사용 중임을 나타냅니다.

* 원인:   
기존 볼륨과 동일한 파일 경로가 있는 볼륨을 만들려고 합니다.
* 해결 방법:   
볼륨을 만들 때 고유한 파일 경로를 사용합니다.

***이름이 너무 깁니다.***

이 오류는 리소스 이름이 최대 길이 요구 사항을 충족하지 않음을 나타냅니다.

* 원인:   
리소스 이름이 너무 깁니다.
* 해결 방법:   
리소스에 대해 더 짧은 이름을 사용합니다.

***파일 경로가 너무 깁니다.***

이 오류는 볼륨의 파일 경로가 최대 길이 요구 사항을 충족하지 않음을 나타냅니다.

* 원인:   
볼륨 파일 경로가 너무 깁니다.
* 해결 방법:   
더 짧은 파일 경로를 사용합니다.

***이름이 너무 짧습니다.***

이 오류는 리소스 이름이 최소 길이 요구 사항을 충족하지 않음을 나타냅니다.

* 원인:   
리소스 이름이 너무 짧습니다.
* 해결 방법:   
리소스에 더 긴 이름을 사용합니다.

***파일 경로가 너무 짧습니다.***

이 오류는 볼륨 파일 경로가 최소 길이 요구 사항을 충족하지 않음을 나타냅니다.

* 원인:   
볼륨 파일 경로가 너무 짧습니다.
* 해결 방법:   
볼륨 파일 경로의 길이를 늘립니다.

***Azure NetApp 파일 API에 연결할 수 없습니다.***

Azure API는 Azure NetApp 파일 API를 사용하여 볼륨을 관리합니다. 이 오류는 API 연결에 문제가 있음을 나타냅니다.

* 원인:   
기본 API가 응답하지 않아 내부 오류가 발생합니다. 이 오류는 일시적일 수 있습니다.
* 해결 방법:   
이 문제는 일시적일 수 있습니다. 요청은 잠시 후 성공해야 합니다.
* 해결 방법:   
없음 기본 API는 볼륨 관리에 필수적입니다.

***'에{0}대한 작업 결과 ID를 찾을 수 없습니다.***

이 오류는 내부 오류로 인해 작업이 완료되지 못하고 있음을 나타냅니다.

* 원인:   
내부 오류가 발생하여 작업이 완료되지 못했습니다.
* 해결 방법:   
이 오류는 일시적일 수 있습니다. 잠시 기다린 다음 다시 시도하십시오. 문제가 지속되면 기술 지원 팀에서 문제를 조사할 수 있는 티켓을 만듭니다.
* 해결 방법:   
몇 분 간 기다렸다가 문제가 지속되는지 확인합니다.

***프로토콜 형식 CIFS 및 NFS를 혼합할 수 없습니다.***

이 오류는 볼륨을 만들려고 할 때 발생하며 볼륨 속성에 CIFS(SMB) 및 NFS 프로토콜 유형이 모두 있습니다.

* 원인:   
CIFS(SMB) 및 NFS 프로토콜 유형은 모두 볼륨 속성에 사용됩니다.
* 해결 방법:   
프로토콜 유형 중 하나를 제거합니다.
* 해결 방법:   
프로토콜 형식 속성을 비워 두거나 null로 둡니다.

***항목 수: 개체에 대한 {value} : ExportPolicy.Rules[RuleIndex]는 최소 최대 범위를 벗어났습니다.***

이 오류는 내보내기 정책 규칙이 최소 또는 최대 범위 요구 사항을 충족하지 않는 경우에 발생합니다. 내보내기 정책을 정의하는 경우 최소 한 개의 내보내기 정책 규칙과 최대 5개의 내보내기 정책 규칙이 있어야 합니다.

* 원인:   
정의한 내보내기 정책이 필요한 범위를 충족하지 않습니다.
* 해결 방법:   
인덱스가 아직 사용되지 않았는지, 인덱스가 1에서 5까지의 범위에 있는지 확인합니다.
* 해결 방법:

위의 솔루션을 참조하십시오.

***볼륨이 삭제되고 현재 삭제할 수 없습니다.***

이 오류는 볼륨이 이미 삭제될 때 볼륨을 삭제하려고 할 때 발생합니다.

* 원인:   
볼륨을 삭제하려고 할 때 볼륨이 이미 삭제되고 있습니다.
* 해결 방법:   
현재 삭제 작업이 완료될 때까지 기다립니다.
* 해결 방법:   
위의 솔루션을 참조하십시오.

***볼륨이 업데이트되고 있으며 현재 삭제할 수 없습니다.***

이 오류는 업데이트 중인 볼륨을 삭제하려고 할 때 발생합니다.

* 원인:   
볼륨을 삭제하려고 할 때 볼륨이 업데이트됩니다.
* 해결 방법:   
업데이트 작업이 완료될 때까지 기다린 다음 삭제를 다시 시도합니다.
* 해결 방법:   
위의 솔루션을 참조하십시오.

***볼륨을 찾지 못했거나 성공적으로 생성되지 않았습니다.***

이 오류는 볼륨 생성에 실패하고 볼륨을 변경하거나 볼륨에 대한 스냅숏을 만들려고 할 때 발생합니다.

* 원인:   
볼륨이 없거나 생성에 실패했습니다.
* 해결 방법:   
올바른 볼륨을 변경하고 볼륨 생성에 성공적이 있는지 확인합니다. 또는 스냅숏을 만드는 볼륨이 존재하는지 확인합니다.
* 해결 방법:   
없음  위의 솔루션을 참조하십시오. 

***지정된 생성 토큰이 이미 있습니다.***

이 오류는 볼륨을 만들려고 시도하고 볼륨이 이미 있는 만들기 토큰(내보내기 경로)을 지정할 때 발생합니다.

* 원인:   
볼륨 생성 중에 지정한 생성 토큰(내보내기 경로)이 이미 다른 볼륨과 연결되어 있습니다. 
* 해결 방법:   
다른 생성 토큰을 선택합니다.  또는 다른 볼륨을 삭제합니다.

***지정된 생성 토큰이 예약됨***

이 오류는 볼륨을 만들려고 할 때 발생하며 "기본" 또는 "없음"을 파일 경로(생성 토큰)로 지정합니다.

* 원인:    
볼륨을 만들려고 하고 "기본값" 또는 "없음"을 파일 경로(생성 토큰)로 지정합니다.
* 해결 방법:   
다른 파일 경로(생성 토큰)를 선택합니다.
 
***활성 디렉터리 자격 증명이 사용 중입니다.***

이 오류는 하나 이상의 SMB 볼륨이 여전히 존재하는 계정에서 Active Directory 구성을 삭제하려고 할 때 발생합니다.  SMB 볼륨은 삭제하려는 Active Directory 구성을 사용하여 만들어졌습니다.

* 원인:   
계정에서 Active Directory 구성을 삭제하려고 하지만 Active Directory 구성을 사용하여 처음에 만든 SMB 볼륨이 하나 이상 남아 있습니다. 
* 해결 방법:   
먼저 Active Directory 구성을 사용하여 만든 모든 SMB 볼륨을 삭제합니다.  그런 다음 구성 삭제를 다시 시도합니다.

***자격 증명이 사용 중인 경우 조직 단위 할당을 수정할 수 없습니다.***

이 오류는 Active Directory 구성의 조직 단위를 변경하려고 할 때 발생하지만 하나 이상의 SMB 볼륨이 여전히 존재합니다.  SMB 볼륨은 삭제하려는 Active Directory 구성을 사용하여 만들어졌습니다.

* 원인:   
Active Directory 구성의 조직 단위를 변경하려고 합니다.  그러나 Active Directory 구성을 사용하여 처음에 만든 SMB 볼륨이 하나 이상 남아 있습니다.
* 해결 방법:   
 먼저 Active Directory 구성을 사용하여 만든 모든 SMB 볼륨을 삭제합니다.  그런 다음 구성 삭제를 다시 시도합니다. 

***이미 진행 중인 활성 디렉터리 업데이트***

이 오류는 편집 작업이 이미 진행 중인 Active Directory 구성을 편집하려고 할 때 발생합니다.

* 원인:   
Active Directory 구성을 편집하려고 하지만 다른 편집 작업이 이미 진행 중입니다.
* 해결 방법:   
현재 실행 중인 편집 작업이 완료될 때까지 기다립니다.

***선택한 자격 증명을 사용하여 모든 볼륨을 먼저 삭제합니다.***

이 오류는 Active Directory 구성을 삭제하려고 할 때 발생하지만 하나 이상의 SMB 볼륨이 여전히 존재합니다.  SMB 볼륨은 삭제하려는 Active Directory 구성을 사용하여 만들어졌습니다.

* 원인:   
Active Directory 구성을 삭제하려고 하지만 Active Directory 구성을 사용하여 처음에 만든 SMB 볼륨이 하나 이상 남아 있습니다.
* 해결 방법:   
먼저 Active Directory 구성을 사용하여 만든 모든 SMB 볼륨을 삭제합니다.  그런 다음 구성 삭제를 다시 시도합니다. 

***리전에서 활성 디렉터리 자격 증명이 없습니다.***

이 오류는 SMB 볼륨을 만들려고 하지만 해당 지역의 계정에 Active Directory 구성이 추가되지 않은 경우 발생합니다.

* 원인:   
SMB 볼륨을 만들려고 하지만 활성 디렉터리 구성이 계정에 추가되지 않았습니다. 
* 해결 방법:   
SMB 볼륨을 만들기 전에 계정에 Active Directory 구성을 추가합니다.

***DNS 서버를 쿼리할 수 없습니다. 네트워크 구성이 올바른지, DNS 서버를 사용할 수 있는지 확인합니다.***

이 오류는 SMB 볼륨을 만들려고 하지만 Active Directory 구성에 지정된 DNS 서버에 연결할 수 없는 경우에 발생합니다. 

* 원인:   
SMB 볼륨을 만들려고 하지만 Active Directory 구성에 지정된 DNS 서버에 연결할 수 없습니다.
* 해결 방법:   
Active Directory 구성을 검토하고 DNS 서버 IP 주소가 정확하고 연결할 수 있는지 확인합니다.
DNS 서버 IP 주소에 문제가 없는 경우 방화벽이 액세스를 차단하지 않는지 확인합니다.

***동시 작업이 너무 많습니다.***

이 오류는 구독에 대해 세 개의 다른 스냅숏 만들기 작업이 이미 진행 중인 경우 스냅숏을 만들려고 할 때 발생합니다.

* 원인:   
구독에 대해 다른 세 개의 스냅숏 만들기 작업이 이미 진행 중인 경우 스냅숏을 만들려고 합니다. 
* 해결 방법:   
스냅샷 만들기 작업은 완료하는 데 몇 초 정도 걸릴 수 있습니다.  몇 초 동안 기다렸다가 스냅숏 만들기 작업을 다시 시도합니다.

***추가 작업을 생성할 수 없습니다. 진행 중인 작업이 완료될 때까지 기다렸다가 다시 시도하십시오.***

이 오류는 특정 상황에서 볼륨을 만들거나 삭제하려고 할 때 발생할 수 있습니다.

* 원인:   
특정 상황에서 볼륨을 만들거나 삭제하려고 합니다.
* 해결 방법:   
잠시 기다렸다가 작업을 다시 시도합니다.

***볼륨이 이미 상태 간에 전환중입니다.***

이 오류는 현재 전환 상태(즉, 현재 생성, 업데이트 또는 삭제 상태)에 있는 볼륨을 삭제하려고 할 때 발생할 수 있습니다.

* 원인:   
현재 전환 상태에 있는 볼륨을 삭제하려고 합니다.
* 해결 방법:   
현재 실행 중인 상태 전환( 상태 전환) 작업이 완료될 때까지 기다린 다음 작업을 다시 시도합니다.

***원본 볼륨 스냅숏에서 새 볼륨을 분할하지 못했습니다.***

 이 오류는 스냅샷에서 볼륨을 만들려고 할 때 발생할 수 있습니다.  

* 원인:   
스냅샷에서 볼륨을 만들려고 하면 볼륨이 오류 상태로 끝납니다.
* 해결 방법:   
볼륨을 삭제한 다음 스냅샷에서 볼륨 생성 작업을 다시 시도합니다.
