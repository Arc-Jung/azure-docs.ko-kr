# Azure에 로그인하려고 할 때 내 신용 카드가 진행되지 않습니다.
이 문서에서는 신용 카드 또는 직불 카드를 사용하여 등록 문제의 공통적인 원인을 설명합니다.

## 신용 카드 또는 직불 카드 외에 다른 지불 옵션이 있습니까?
Azure 지불 옵션은 다음을 포함합니다.

- 신용 카드 또는 직불 카드(가상 및 선불 카드 제외)
- 청구서. 청구서 옵션을 선택하면 같은 Azure 구독에 대한 신용 카드 옵션으로 이동할 수 없습니다. 회사로서 청구된 대체 지불 방법을 알아보는 데 관심이 있다면 [Azure 청구 - 청구 방법](https://azure.microsoft.com/pricing/invoicing/)을 읽으십시오. 청구 지불 방법은 수표, 심야 수표 및 전신 송금을 포함합니다.

##  Azure 활성화 또는 지불에 대해 지원되는 신용 카드를 사용하고 있습니까?
- 현재 가상 카드가 아니고 선불 카드가 아닌 신용/직불 카드만 허용됩니다.
- 허용되는 신용/직불 카드 공급자는 국가에 따라 다릅니다.

제안: - 가상 또는 선불 직불/신용 카드를 사용하지 않습니다. - 지불 또는 활성화에 실패하면 유효하고 지원되는 대체 신용 카드를 사용해 보십시오. - 지불 방법이 활성화되고 은행이 보안 정책상 온라인 거래를 차단하지 않는지 확인하십시오.

## 신용 카드 정보가 정확하고 완전합니까?
- 신용 카드 정보가 정확하고 신용 카드 은행 청구 정보와 일치해야 합니다.
- 입력한 이름은 기호/문자 없이 신용 카드에 인자된 이름과 정확히 일치해야 합니다.
- 입력한 주소는 청구서가 전송된 주소이며 도로 주소, 시/군/구, 시/도 및 우편 번호를 포함하여 신용 카드 명세서에 나타난 대로 정확히 입력됩니다.
- CVV 코드는 신용 카드 뒷면 또는 전면에 나타난 대로 삽입됩니다.
- 은행과 협력하여 카드상의 정보에서 맞춤법을 고치거나 다른 문제를 해결하십시오.

제안: - 프로필 정보는 청구 명세서를 생성하는 데에도 사용되므로 등록 시 정확한 정보를 제공하십시오.

## 계정 프로필 정보가 정확하고 완전합니까?
- 계정 소유자는 등록 중에 요청된 모든 관련 정보를 삽입해야 계정을 소유한 개인이 식별되고 필요 시 정확한 청구 명세서를 생성할 수 있습니다.
- 틀리거나 불완전한 필드의 예: 이름 또는 성 누락, 잘못된 전화 번호, 비표준(특수) 문자.

제안: - Azure 계정을 이미 생성한 경우 등록 시 또는 [프로필 설정](https://account.windowsazure.com/Profile/)에서 프로필의 각 필드를 꼼꼼히 점검하십시오. - 프로필 정보는 청구 명세서를 생성하는 데에도 사용되므로 정확한 정보를 제공하십시오.

## MSDN, BizSpark 또는 MPN 같은 한 Azure 혜택 계획을 활성화하려고 합니까?  
본인의 혜택 프로그램 채널을 통해 선택한 계획에 대한 자격이 있는지 여부 확인: - [MSDN 계정 페이지](https://msdn.microsoft.com/subscriptions/manage/default.aspx)에서 자격 상태를 확인합니다. - 상태를 확인할 수 없는 경우 [MSDN 구독 고객 서비스 센터](https://msdn.microsoft.com/subscriptions/contactus.aspx)에 문의하십시오. - MPN - [MPN 포털](https://mspartner.microsoft.com/en/us/Pages/Locale.aspx)에 로그인하여 자격 상태를 확인하십시오. 적절한 [클라우드 플랫폼 역량](https://mspartner.microsoft.com/en/us/pages/membership/cloud-platform-competency.aspx)을 가지고 있는 경우 추가 혜택의 자격이 있을 수 있습니다. - 상태를 확인할 수 없는 경우 [MPN 지원](https://mspartner.microsoft.com/en/us/Pages/Support/Premium/contact-support.aspx)에 문의하십시오. - Bizpark: - [BizSpark 포털](https://www.microsoft.com/bizspark/default.aspx#start-two)에 로그인하고 BizSpark 및 BizSpark Plus에 대한 본인의 자격 상태를 확인합니다. 본인의 상태를 확인할 수 없는 경우 [BizSpark 팀에 문의](mailto:bizspark@microsoft.com?subject=BizSpark%20Support&body=Thank%20you%20for%20contacting%20BizSpark.%20Please%20provide%20as%20much%20of%20the%20following%20information%20as%20possible,%20as%20it%20will%20help%20expedite%20our%20response%20to%20you.%0aContact%20name:%0aStartup%20name:%0aMicrosoft%20Account/Live%20ID:%0aSpecific%20description%20of%20issue%20experienced%20or%20question:%0a%0aThank%20you,%0a%0aThe%20BizSpark%20Team)로 이메일을 보내서 BizSpark 지원에 문의하십시오.

제안:

새 혜택 구독에 등록하는 데 오류가 발생하면 [Azure 구독 페이지](http://account.windowsazure.com/Subscriptions)로 이동하여 구독이 생성되었는지 확인할 수 있습니다. 일부 유형의 구독은 보류 중으로 나타나고 몇 분 후 활성화될 수 있습니다. 몇 분 후 상태가 변경되지 않으면 [Azure 지원](http://go.microsoft.com/fwlink/?linkid=544831&clcid=0x409)에 문의하여 무료 지원을 받을 수 있습니다.

## 과거에 Azure 구독을 이미 사용한 적이 있고 지금 Azure 무료 평가판을 활성화하려 합니까?
Azure 무료 평가판 제품은 무료 평가판 활성화 한 번으로 제한됩니다.

제안:

과거에 Azure 구독을 이미 활성화했기 때문에 무료 평가판 활성화에 실패한 경우 Azure 무료 평가판 계획 대신에 [Microsoft Azure 제품 정보](http://azure.microsoft.com/support/legal/offer-details/)에 나열된 사용 가능한 또 다른 Azure 계획을 사용하십시오.

## Azure 지원 문의
이 문제 해결 단계를 수행한 후 등록 문제가 계속 발생하면 [Azure 지원](http://go.microsoft.com/fwlink/?linkid=544831&clcid=0x409)에 문의하십시오.
