# <a name="choose-the-right-iot-solution"></a>올바른 IoT 솔루션 선택

비즈니스를 위한 IoT 솔루션을 빌드하기 위해서는 일반적으로 *플랫폼 서비스* 또는 *관리형 앱 플랫폼* 방법 중 하나를 선택하여 사용합니다.

플랫폼 서비스는 사용자 지정된 유연한 IoT 애플리케이션을 위한 구성 요소를 제공합니다. 디바이스를 연결하고 데이터를 수집, 저장 및 분석할 때 추가 옵션과 코드를 선택할 수 있습니다. Azure IoT 플랫폼 서비스에는 제품 Azure IoT Hub 및 Azure Digital Twins가 포함됩니다.

관리형 앱 플랫폼을 사용하면 결과를 얻는 데 필요한 의사 결정을 줄임으로써 플랫폼 서비스보다 더 빠르게 앱을 빌드할 수 있습니다. 관리형 앱 플랫폼은 솔루션의 대부분 요소를 처리하므로, 산업 기술 자료를 추가하고 디바이스를 확장 및 연결하는 데 집중할 수 있습니다. Azure IoT Central은 관리형 앱 플랫폼입니다.

이러한 두 방법 중 하나를 선택하려면 다음을 고려해야 합니다.

- 솔루션을 관리하는 방법
- 솔루션에 대해 원하는 사용자 지정 및 제어 수준
- 원하는 가격 책정 구조

## <a name="management"></a>관리

시스템 관리 시간과 리소스를 어디에 사용하시겠습니까? 

- 솔루션의 기본 서비스를 완전히 제어하려면 플랫폼 서비스 방식을 선택합니다. 예를 들어 다음을 원할 때입니다.

    - 요구 사항에 맞게 크기 조정 및 보안 서비스를 관리합니다.
    - 사내 또는 파트너 전문 지식을 활용하여 디바이스를 등록하고 서비스를 프로비저닝합니다.

- IoT 애플리케이션 및 디바이스의 크기 조정, 보안 및 관리를 처리하는 플랫폼을 활용하려면 관리형 앱 플랫폼 방식을 선택합니다.

## <a name="control"></a>제어

사용자 지정하려는 솔루션의 요소가 무엇인가요?

- 솔루션 아키텍처에 대한 전체 사용자 지정 및 제어를 위해서는 플랫폼 서비스 방식을 선택합니다.

- 브랜딩, 대시보드, 사용자 역할, 디바이스 및 원격 분석을 사용자 지정하려면 관리형 앱 플랫폼 방식을 선택합니다. 단, 기본 IoT 시스템 관리 오버헤드를 처리하고 싶지 않은 경우입니다.

## <a name="pricing"></a>가격

요구 사항에 가장 적합한 가격 책정 구조는 무엇인가요?

- 서비스를 세부적으로 조정하고 전체 비용을 제어하려면 플랫폼 서비스 방식을 선택합니다.

- 간단하고 예측 가능한 가격 책정 구조에는 관리형 앱 플랫폼 방식을 선택합니다.

## <a name="summary"></a>요약

플랫폼 서비스 방식은 다음을 원하는 클라우드 솔루션 및 디바이스 전문 지식을 갖춘 비즈니스에 적합합니다.

- 솔루션의 서비스를 미세 조정합니다.
- 솔루션의 서비스에 대한 제어 수준이 높습니다.
- 솔루션의 전체 사용자 지정

관리형 앱 플랫폼 방식은 다음에 해당하는 비즈니스에 적합합니다.

- 시스템 디자인, 개발 및 관리에 광범위한 리소스를 전용으로 사용할 수 없습니다.
- 예측 가능한 가격 책정 구조가 필요합니다.
- 일부 사용자 지정 기능이 필요합니다.
