# 역할 기반 액세스 제어 문제 해결

## 소개

[역할 기반 액세스 제어](../role-based-access-control-configure.md)는 Azure의 리소스에 대해 세분화된 액세스 권한을 위임하는 데 사용할 수 있는 유용한 기능입니다. 이 기능을 통해 특정 사용자에게 필요한 리소스에 대한 권한을 적절하게 부여할 수 있습니다. 그러나 Azure 리소스의 리소스 모델이 복잡하여 권한 부여 대상을 정확하게 파악하기가 어려운 경우도 있습니다.

이 문서에서는 Azure 클래식 포털의 몇 가지 역할을 사용하는 경우의 결과를 소개합니다. 모든 리소스 유형을 다루기 위해 포함된 3가지 일반적인 역할인 * 소유자 * 참여자 * 읽기 권한자가 있습니다.

소유자와 참여자는 관리 환경에 대한 모든 권한을 가집니다. 이 두 역할의 차이점은, 참여자의 경우 다른 사용자나 그룹에 액세스 권한을 제공할 수 없다는 것입니다. 읽기 권한자 역할은 좀 더 복잡하므로 자세히 파악해야 합니다. 액세스 권한을 부여하는 정확한 방법에 대한 자세한 내용은 [이 문서를 참조](../role-based-access-control-configure.md)하세요.

## 앱 서비스 작업

### 읽기 전용 권한

사용자나 자신에게 단일 웹앱 읽기 권한만 부여하는 경우 예기치 않게 일부 기능을 사용하지 못하게 될 수 있습니다. 다음 관리 기능을 사용하려면 참여자나 소유자에게 제공되는 웹앱에 대한 **쓰기** 권한이 필요하며, 읽기 전용 시나리오에서는 이러한 기능을 사용할 수 없습니다.

1. 시작, 중지 등의 명령
2. 일반 구성, 크기 조정 설정, 백업 설정, 모니터링 설정 등의 설정 변경
3. 게시 자격 증명과 앱 설정, 연결 문자열 등의 기타 암호 액세스
4. 스트리밍 로그
5. 진단 로그 구성
6. 콘솔(명령 프롬프트)
7. 활성 및 최근 배포(로컬 Git 연속 배포의 경우)
8. 예상 소요 시간
9. 웹 테스트
10. 가상 네트워크(쓰기 권한이 있는 사용자가 이전에 가상 네트워크를 구성한 경우에만 읽기 권한자에게 표시됨)

이러한 타일에 액세스할 수 없는 경우 웹앱에 대한 참여자 권한을 할당받아야 합니다.

### 관련 리소스 사용

웹앱에서 몇 가지 리소스가 상호 연동되는 경우 웹앱의 구조가 복잡해집니다. 아래에는 웹 사이트 몇 개가 포함된 일반적인 리소스 그룹이 나와 있습니다.

![웹앱 리소스 그룹](./media/role-based-access-control-troubleshooting/website-resource-model.png)

따라서 사용자에게 웹 사이트에 대한 권한만 부여하면 웹 사이트 블레이드의 기능을 대부분 완전히 사용할 수 없게 됩니다.

1. 다음 항목을 사용하려면 웹 사이트에 해당하는 **앱 서비스 계획**에 대한 권한이 필요합니다.  
    * 웹앱의 가격 책정 계층 보기(예: 무료 또는 표준)
    * 크기 조정 구성(예: 인스턴스 수, 가상 컴퓨터 크기, 자동 크기 조정 설정)
    * 할당량(예: 저장소, 대역폭, CPU)
2. 다음 항목을 사용하려면 웹 사이트를 포함하는 전체 **리소스 그룹**에 대한 권한이 필요합니다.  
    * SSL 인증서 및 바인딩. SSL 인증서는 같은 리소스 그룹과 지리적 위치의 사이트 간에 공유될 수 있기 때문입니다.
    * 경고 규칙
    * 자동 크기 조정 설정
    * Application Insights 구성 요소
    * 웹 테스트

## 가상 컴퓨터 작업

웹앱과 마찬가지로 가상 컴퓨터 블레이드의 일부 기능 역시 가상 컴퓨터 또는 리소스 그룹의 기타 리소스에 대한 쓰기 권한이 있어야 사용할 수 있습니다.

가상 컴퓨터 관련 리소스는 다음과 같습니다. * 도메인 이름 * 가상 네트워크 * 저장소 계정 * 경고 규칙

1. 다음 항목을 사용하려면 가상 컴퓨터에 대한 **쓰기** 권한이 필요합니다.  
    * 끝점
    * IP 주소
    * 디스크
    * 확장
2. 다음 항목을 사용하려면 가상 컴퓨터와 해당 가상 컴퓨터가 속한 **리소스 그룹**(도메인 이름 포함) 둘 다에 대한 쓰기 권한이 필요합니다.  
    * 가용성 집합
    * 부하 분산된 집합
    * 경고 규칙

이러한 타일에 액세스할 수 없는 경우 관리자에게 리소스 그룹에 대한 참여자 권한을 요청해야 합니다.
