# <a name="request-process-and-email-notifications-in-azure-ad-entitlement-management"></a>Azure AD 권한 관리에서 프로세스 및 전자 메일 알림 요청

사용자가 액세스 패키지에 요청을 제출하면 프로세스가 해당 액세스 요청을 제공하기 시작합니다. Azure AD 권한 관리는 프로세스 중에 주요 이벤트가 발생할 때 승인자 및 요청자에 전자 메일 알림을 보냅니다. 이 문서에서는 요청 프로세스 및 전송되는 전자 메일 알림을 설명합니다.

## <a name="request-process"></a>요청 프로세스

액세스 패키지에 액세스해야 하는 사용자는 액세스 요청을 제출할 수 있습니다. 정책 구성에 따라 요청에 승인이 필요할 수 있습니다. 요청이 승인되면 프로세스는 액세스 패키지의 각 리소스에 대한 사용자 액세스 권한을 할당하기 시작합니다. 다음 다이어그램은 프로세스와 다른 상태에 대한 개요를 보여 주며 다음과 같은 각 상태를 보여 주며 있습니다.

![승인 프로세스 다이어그램](./media/entitlement-management-process/request-process.png)

| 시스템 상태 | 설명 |
| --- | --- |
| 제출됨 | 사용자가 요청을 제출합니다. |
| 승인 보류 중 | 액세스 패키지에 대한 정책이 승인이 필요한 경우 요청은 보류 중인 승인으로 이동합니다. |
| 만료됨 | 승인 요청 시간 시간 시간 내에 승인을 승인하지 않으면 요청이 만료됩니다. 다시 시도하려면 사용자가 요청을 다시 제출해야 합니다. |
| 거부됨 | 승인자가 요청을 거부합니다. |
| 승인됨 | 승인자가 요청을 승인합니다. |
| 배달 | 액세스 패키지의 모든 리소스에 대한 액세스 권한이 사용자에게 **할당되지 않았습니다.** 외부 사용자인 경우 사용자가 리소스 디렉터리에 아직 액세스하지 못했을 수 있습니다. 또한 동의 프롬프트를 수락하지 않았을 수도 있습니다. |
| 배달됨 | 액세스 패키지의 모든 리소스에 대한 액세스 권한이 사용자에게 할당되었습니다. |
| 액세스 확장 | 정책에서 확장이 허용되는 경우 사용자는 할당을 확장했습니다. |
| 액세스가 만료되었습니다. | 액세스 패키지에 대한 사용자의 액세스가 만료되었습니다. 다시 액세스하려면 사용자가 요청을 제출해야 합니다. |

## <a name="email-notifications"></a>이메일 알림

승인자인 경우 액세스 요청을 승인해야 할 때 이메일 알림이 전송됩니다. 액세스 요청이 완료되면 알림을 받을 수도 있습니다. 또한 요청자인 경우 요청 상태를 나타내는 이메일 알림도 전송됩니다.

다음 다이어그램은 이러한 전자 메일 알림이 승인자 또는 요청자에게 전송되는 시기를 보여 준다. 전자 [메일 알림 테이블을](entitlement-management-process.md#email-notifications-table) 참조하여 다이어그램에 표시된 전자 메일 알림에 해당하는 번호를 찾습니다.

### <a name="access-request-emails"></a>액세스 요청 이메일

요청자가 승인을 요구하도록 구성된 액세스 패키지에 대한 액세스 요청을 제출하면 정책에 추가된 모든 승인자는 요청 세부 정보가 있는 전자 메일 알림을 받게 됩니다. 이메일의 세부 사항은 다음과 같습니다 : 요청자의 이름 조직 및 비즈니스 정당화; 요청된 액세스 시작 및 종료 날짜(제공된 경우). 세부 정보에는 요청이 제출된 시기와 요청이 만료되는 시기도 포함됩니다.

이메일에는 승인자가 클릭하여 내 액세스로 이동하여 액세스 요청을 승인하거나 거부할 수 있는 링크가 포함되어 있습니다. 다음은 액세스 요청을 완료하기 위해 첫 번째 승인자 또는 두 번째 승인자(2단계 승인이 활성화된 경우)에게 전송되는 샘플 전자 메일 알림입니다.

![패키지 이메일 액세스 요청 승인](./media/entitlement-management-shared/approver-request-email.png)

승인자는 미리 알림 이메일을 받을 수도 있습니다. 이메일은 승인자가 요청에 대한 결정을 내리도록 요청합니다. 다음은 승인자가 조치를 취하도록 상기시키기 위해 받는 샘플 전자 메일 알림입니다.

![미리 알림 액세스 요청 이메일](./media/entitlement-management-process/approver-access-request-reminder-email.png)

### <a name="alternate-approvers-request-emails"></a>대체 승인자가 이메일을 요청합니다.

대체 승인자 설정이 활성화되어 있고 요청이 아직 보류 중이면 전달됩니다. 대체 승인자는 요청을 승인하거나 거부하는 이메일을 받게 됩니다. 1단계 및 2단계에서 대체 승인자를 활성화할 수 있습니다. 다음은 대체 승인자가 받는 알림의 샘플 전자 메일입니다.

![대체 승인자가 이메일을 요청합니다.](./media/entitlement-management-process/alternate-approver-email-fwd-request.png)

승인자와 대체 승인자 모두 요청을 승인하거나 거부할 수 있습니다.

### <a name="approved-or-denied-emails"></a>승인됨 또는 거부된 이메일

 승인자가 요청자가 제출한 액세스 요청을 받으면 액세스 요청을 승인하거나 거부할 수 있습니다. 승인자가 의사 결정에 대한 비즈니스 근거를 추가해야 합니다. 다음은 요청이 승인된 후 승인자 및 대체 승인자에게 보낸 샘플 전자 메일입니다.

![패키지 이메일 액세스 승인 요청](./media/entitlement-management-process/approver-request-email-approved.png)

액세스 요청이 승인되고 해당 액세스가 프로비전되면 요청자가 액세스 패키지에 액세스할 수 있다는 전자 메일 알림이 요청자에게 전송됩니다. 다음은 액세스 패키지에 대한 액세스 권한이 부여될 때 요청자에게 전송되는 샘플 전자 메일 알림입니다.

![승인된 요청자 액세스 요청 이메일](./media/entitlement-management-process/requestor-email-approved.png)

액세스 요청이 거부되면 요청자가 전자 메일 알림을 보송합니다. 다음은 액세스 요청이 거부될 때 요청자에게 전송되는 샘플 전자 메일 알림입니다.

![요청자 요청 거부 된 이메일](./media/entitlement-management-process/requestor-email-denied.png)

### <a name="2-stage-approval-access-request-emails"></a>2단계 승인 액세스 요청 이메일

2단계 승인을 사용하도록 설정한 경우 요청자가 액세스 권한을 받기 전에 각 단계에서 하나씩 요청을 승인해야 합니다.

1단계 동안 첫 번째 승인자가 액세스 요청 이메일을 수신하여 결정을 내보입니다. 요청을 승인하면 단계 1의 모든 첫 번째 승인자 및 대체 승인자가 단계 1이 완료되었다는 알림을 받게 됩니다. 다음은 1단계가 완료되면 전송되는 알림의 샘플 전자 메일입니다.

![2단계 액세스 요청 이메일](./media/entitlement-management-process/approver-request-email-2stage.png)

첫 번째 또는 대체 승인자가 1단계에서 요청을 승인하면 2단계가 시작됩니다. 2단계 동안 두 번째 승인자가 액세스 요청 알림 이메일을 받게 됩니다. 2단계의 두 번째 승인자 또는 대체 승인자가 요청을 승인 또는 거부하기로 결정한 후 알림 전자 메일은 첫 번째 및 두 번째 승인자 및 1단계 및 2단계의 모든 대체 승인자와 요청자로 전송됩니다.

### <a name="expired-access-request-emails"></a>만료된 액세스 요청 이메일

승인자가 요청을 승인하거나 거부하지 않은 경우 액세스 요청이 만료될 수 있습니다. 

요청이 구성된 만료 날짜에 도달하고 만료되면 승인자가 더 이상 승인하거나 거부할 수 없습니다. 다음은 첫 번째, 두 번째(2단계 승인이 활성화된 경우) 및 대체 승인자모두에게 전송된 알림의 샘플 전자 메일입니다.

![승인자가 액세스 요청 이메일이 만료됨](./media/entitlement-management-process/approver-request-email-expired.png)

또한 요청자가 액세스 요청이 만료되었음을 알리고 액세스 요청을 다시 제출해야 한다는 전자 메일 알림도 전송됩니다. 다음 다이어그램은 요청자의 경험과 액세스 확장 을 요청할 때 받는 전자 메일 알림을 보여 주어집니다.

![요청자 액세스 프로세스 흐름 확장](./media/entitlement-management-process/requestor-expiration-request-flow.png) 

다음은 액세스 요청이 만료되면 요청자에게 전송되는 샘플 전자 메일 알림입니다.

![요청자가 액세스 요청 이메일이 만료되었습니다.](./media/entitlement-management-process/requestor-email-request-expired.png)
