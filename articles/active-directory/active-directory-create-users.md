# Azure AD에서 사용자 만들기 또는 편집

Microsoft 클라우드 서비스에 액세스하는 모든 사용자에 대한 계정을 만들어야 합니다. 사용자 계정을 변경할 수도 있으며 더 이상 필요하지 않은 계정은 삭제할 수 있습니다. 기본적으로 사용자에게는 관리자 권한이 없지만 선택적으로 할당할 수 있습니다.

## 사용자 만들기

1. **Active Directory**를 클릭한 다음 조직의 디렉터리 이름을 선택합니다.
2. **사용자** 페이지에서 **사용자 추가**를 클릭합니다.
3. **이 사용자에 대해 알리기** 페이지에 대한 **사용자 형식**으로 다음 중 하나를 선택합니다.
	
	- **조직의 새 사용자** – 디렉터리에 새 사용자 계정 만들기
	- **기존 Microsoft 계정을 사용하는 사용자** – 디렉터리에 기존 사용자 Microsoft 계정(예: outlook 계정) 추가
	- **다른 Azure AD 디렉터리의 사용자** – 다른 Azure AD 디렉터리가 원본인 사용자 계정을 디렉터리에 추가(참고: 다른 디렉터리의 사용자를 선택하려면 해당 디렉터리의 멤버여야 함)
	- **파트너 회사의 사용자** - 파트너 회사 사용자를 디렉터리에 초대하고 권한 부여([Azure Active Directory B2B 참조](active-directory-b2b-what-is-azure-ad-b2b.md))
	

4. 선택한 옵션에 따라 사용자 이름과 전자 메일 주소를 입력하거나 파트너 사용자에 대한 CSV 파일을 업로드합니다.
5. 사용자 **프로필** 페이지에 사용자의 이름과 성, 별명 및 역할 드롭다운 메뉴의 사용자 역할을 입력합니다. 사용자 및 관리자 역할에 대한 자세한 내용은 [Azure AD에서 관리자 역할 할당](active-directory-assign-admin-roles.md)을 참조하세요. **다단계 인증 사용** 여부를 지정합니다.
6. **임시 암호 가져오기** 페이지에서 **만들기**를 클릭합니다.

조직에서 둘 이상의 도메인을 사용하는 경우 사용자 계정을 만들 때 다음과 같은 문제를 알고 있어야 합니다.

- 사용자 계정을 처음 만드는 경우 도메인 간에 동일한 UPN(사용자 계정 이름)을 갖는 여러 개의 사용자 계정을 만들 수 있습니다(예: geoffgrisso@contoso.onmicrosoft.com 다음에 geoffgrisso@contoso.com).
- geoffgrisso@contoso.com 다음에 geoffgrisso@contoso.onmicrosoft.com을 만들 수는 없습니다.

## 사용자 편집

Azure 클래식 포털에서 사용자를 편집하려면

1. **Active Directory**를 클릭한 다음 조직의 디렉터리 이름을 클릭합니다.
2. **사용자** 페이지에서 편집하려는 사용자의 표시 이름을 클릭합니다.
3. 변경을 완료하고 **저장**을 클릭합니다.

편집하려는 사용자가 온-프레미스 Active Directory 서비스와 동기화된 경우 오류 메시지가 나타나며, 이 절차를 사용하여 사용자를 편집할 수 없게 됩니다. 사용자를 편집하려면 로컬 Active Directory 관리 도구를 사용합니다.

## 사용자의 암호 다시 설정

1. **Active Directory**를 클릭한 다음 조직의 디렉터리 이름을 클릭합니다.
2. **사용자** 페이지에서 편집하려는 사용자의 표시 이름을 클릭합니다.
3. 포털의 아래쪽에서 **암호 재설정**을 클릭합니다.
4. 암호 재설정 대화 상자에서 **재설정**을 클릭합니다.
5. 암호가 재설정되었는지 확인하려면 확인 표시를 클릭합니다.

## 외부 사용자 만들기

Azure AD에서, 사용자가 소속된 다른 Azure AD 디렉터리에서 Microsoft 계정을 사용하여 또는 CSV 파일을 사용하여 파트너 회사에서 Azure AD 디렉터리에 사용자를 추가할 수도 있습니다. 외부 사용자를 만들려면 포털에서 사용자를 만들고 **사용자 형식**으로 **다른 Azure AD 디렉터리의 사용자** 또는 **파트너 회사의 사용자**를 선택합니다.

두 가지 방법으로 만든 사용자 모두 다른 디렉터리가 원본이며 **외부 사용자**로 생성됩니다. 외부 사용자는 계정과 자격 증명을 새로 만들 필요 없이 계정 하나로 이미 디렉터리에 포함된 사용자와 공동으로 작업할 수 있습니다. 외부 사용자는 로그인 시 홈 디렉터리에 의해 인증되며 이러한 인증은 사용자가 포함된 다른 모든 디렉터리에서 유효합니다.

## 외부 사용자 관리 및 제한 사항

다른 디렉터리의 사용자를 내 디렉터리에 추가하면 해당 사용자는 내 디렉터리에서 외부 사용자가 됩니다. 처음에 표시 이름 및 사용자 이름이 사용자의 "홈 디렉터리"에서 복사된 후 내 디렉터리에서 해당 외부 사용자에 스탬프로 지정됩니다. 이때부터 해당 이름 및 외부 사용자 개체의 다른 속성은 완전히 별개로 인식됩니다. 홈 디렉터리의 사용자가 변경되면(사용자의 이름 변경, 직위 추가 등) 변경 내용이 내 디렉터리의 해당 외부 사용자 계정으로 전파되지 않습니다.

두 개체 사이의 유일한 연계는 사용자가 항상 홈 디렉터리에 대해 인증을 받거나 자신의 Microsoft 계정으로 인증을 받는다는 것입니다. 외부 사용자에 대해 암호를 재설정하거나 Multi-Factor Authentication을 사용하도록 설정하는 옵션이 표시되지 않는 이유가 바로 여기에 있습니다. 현재는 홈 디렉터리 또는 Microsoft 계정의 인증 정책만 사용자가 로그인할 때 평가됩니다.

> [AZURE.NOTE]
디렉터리의 외부 사용자는 여전히 비활성화할 수 있으며 이 경우 해당 외부 사용자는 사용자 디렉터리에 액세스할 수 없게 됩니다.

홈 디렉터리에서 사용자가 삭제되거나 사용자가 자신의 Microsoft 계정을 취소하더라도 해당 외부 사용자는 여전히 내 디렉터리에 존재합니다. 그러나 해당 사용자는 더 이상 홈 디렉터리 또는 Microsoft 계정에서 인증을 받을 수 없으므로 내 디렉터리의 리소스에 액세스할 수 없게 됩니다.

현재 Azure AD 외부 사용자의 액세스를 지원하는 서비스는 다음과 같습니다.

- Azure 클래식 포털: 여러 디렉터리의 관리자인 외부 사용자가 각 디렉터리를 관리할 수 있습니다.
- SharePoint Online: 외부 공유가 활성화될 경우 SharePoint Online에서 권한을 부여한 리소스에 외부 사용자가 액세스할 수 있습니다.
- Dynamics CRM: PowerShell을 통해 라이선스를 부여할 경우 사용이 허가된 리소스에 외부 사용자가 액세스할 수 있습니다.

다음은 Azure AD 외부 사용자의 알려진 제한 사항입니다.

- 관리자인 외부 사용자는 홈 디렉터리 외부에 있는 디렉터리(B2B)에 파트너 회사의 사용자를 추가할 수 없습니다.
- 외부 사용자가 홈 디렉터리 외부에 있는 디렉터리의 다중 테넌트 응용 프로그램에 동의할 수 없습니다.
- Visual Studio Online은 현재 외부 사용자의 액세스를 지원하지 않습니다.
- PowerBI는 현재 외부 사용자의 액세스를 지원하지 않습니다. 
- Office 포털은 현재 외부 사용자에 대한 라이선싱을 지원하지 않습니다.

## 게스트

**게스트**는 디렉터리의 사용자 중에서 UserType 특성이 "게스트"로 설정된 사용자입니다. 일반 사용자는 디렉터리의 멤버임을 나타내기 위해 UserType이 "멤버"로 지정됩니다. 게스트는 SharePoint 문서, 응용 프로그램, Azure 리소스 등의 특정 리소스에 액세스할 수 있도록 내 디렉터리에 초대된 다른 디렉터리의 사용자를 말합니다.

게스트는 디렉터리에서 제한된 권한 집합을 갖습니다. 이러한 권한은 작업 중인 리소스와 연결된 사용자 및 그룹과 상호 작용할 수 있도록 하면서 디렉터리의 다른 사용자에 대한 정보를 검색하는 게스트의 능력을 제한합니다. 게스트는 다음과 같은 기능을 사용할 수 있습니다.

- 자신이 할당된 Azure 구독과 연결된 다른 사용자 및 그룹 보기
- 자신이 속한 그룹의 그룹 멤버 보기
- 사용자의 전체 전자 메일 주소를 알고 있는 경우 디렉터리의 다른 사용자 조회
- 조회하는 사용자의 제한된 특성 보기 - 표시 이름, 전자 메일 주소, UPN(사용자 계정 이름) 및 축소한 사진으로 제한됨 
- 테넌트의 확인된 도메인 목록 가져오기
- 응용 프로그램에 동의하여 내 디렉터리에 있을 때와 동일한 액세스 권한 부여

## 사용자 액세스 정책 구성

디렉터리의 **구성** 탭에는 외부 사용자의 액세스를 제어하는 옵션이 포함되어 있습니다. 이러한 옵션은 디렉터리 전역 관리자가 Azure 클래식 포털의 UI(Windows PowerShell 또는 API 메서드가 없음)에서만 변경할 수 있습니다. Azure 클래식 포털에서 **구성** 탭을 열려면 **Active Directory**를 클릭한 다음 디렉터리의 이름을 클릭합니다.
