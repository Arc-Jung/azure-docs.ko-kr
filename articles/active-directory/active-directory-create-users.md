# Azure Active Directory에서 사용자 만들기 또는 편집

Microsoft 클라우드 서비스에 액세스하는 모든 사용자에 대해 Azure AD(Azure Active Directory)에 계정을 만들어야 합니다. 사용자 계정을 변경할 수도 있으며 더 이상 필요하지 않은 계정은 삭제할 수 있습니다. 기본적으로 사용자에게는 관리자 권한이 없지만 선택적으로 할당할 수 있습니다.

## 사용자 만들기

1. **Active Directory**를 클릭한 다음 조직의 디렉터리 이름을 클릭합니다.
2. **사용자** 페이지에서 **사용자 추가**를 클릭합니다.
3. **이 사용자에 대해 알리기** 페이지에 대한 **사용자 형식**으로 다음 중 하나를 선택합니다.
	1. **조직 내 새 사용자** – 디렉터리 내에서 새 사용자 계정을 만들고 관리할 것임을 나타냅니다.
	2. **기존 Microsoft 계정 사용자** – Microsoft 계정으로 Azure에 액세스하는 공동 관리자와 함께 Azure 리소스 작업을 수행하기 위해 기존 Microsoft 계정을 디렉터리에 추가할 것임을 나타냅니다.
	3. **다른 Azure AD 디렉터리의 사용자** – 다른 Azure AD 디렉터리가 원본인 사용자 계정을 사용자 디렉터리에 추가할 것임을 나타냅니다. 다른 디렉터리에서 사용자를 선택하려면 해당 디렉터리의 구성원이어야 합니다.
4. 선택한 옵션에 따라 사용자 이름 또는 이 사용자가 로그인하는 데 사용하는 Microsoft 계정 이름을 입력합니다.
5. 사용자 **프로필** 페이지에 사용자의 이름과 성, 별명 및 역할 드롭다운 메뉴의 사용자 역할을 입력합니다. 사용자 및 관리자 역할에 대한 자세한 내용은 [Azure AD에서 관리자 역할 할당](active-directory-assign-admin-roles.md)을 참조하세요. **다단계 인증 사용** 여부를 지정합니다.
6. **임시 암호 가져오기** 페이지에서 **만들기**를 클릭합니다.

조직에서 둘 이상의 도메인을 사용하는 경우 사용자 계정을 만들 때 다음과 같은 문제를 알고 있어야 합니다.

- 사용자 계정을 처음 만드는 경우 도메인 간에 동일한 UPN(사용자 계정 이름)을 갖는 여러 개의 사용자 계정을 만들 수 있습니다(예: geoffgrisso@contoso.onmicrosoft.com 다음에 geoffgrisso@contoso.com).
- geoffgrisso@contoso.com 다음에 geoffgrisso@contoso.onmicrosoft.com을 만들 수는 없습니다.

## 사용자 편집

편집하려는 사용자가 온-프레미스 Active Directory 서비스와 동기화된 경우 오류 메시지가 나타나며, 이 절차를 사용하여 사용자를 편집할 수 없게 됩니다. 사용자를 편집하려면 로컬 Active Directory 관리 도구를 사용합니다.

Azure 클래식 포털에서 사용자를 편집하려면

1. **Active Directory**를 클릭한 다음 조직의 디렉터리 이름을 클릭합니다.
2. **사용자** 페이지에서 편집하려는 사용자의 표시 이름을 클릭합니다.
3. 변경을 완료하고 **저장**을 클릭합니다.

## 사용자의 암호 다시 설정

1. **Active Directory**를 클릭한 다음 조직의 디렉터리 이름을 클릭합니다.
2. **사용자** 페이지에서 편집하려는 사용자의 표시 이름을 클릭합니다.
3. 포털의 아래쪽에서 **암호 재설정**을 클릭합니다.
4. 암호 재설정 대화 상자에서 **재설정**을 클릭합니다.
5. 암호가 재설정되었는지 확인하려면 확인 표시를 클릭합니다.

## 외부 사용자 만들기

Azure AD에서 다른 Azure AD 디렉터리의 사용자 또는 Microsoft 계정을 가진 사용자를 Azure AD 디렉터리에 추가할 수도 있습니다. 사용자는 최대 20개의 서로 다른 디렉터리의 멤버일 수 있습니다.

다른 디렉터리에서 추가된 사용자는 외부 사용자입니다. 외부 사용자는 새 계정 및 자격 증명을 사용하여 로그인하지 않아도 디렉터리에 이미 포함되어 있는 사용자와 공동으로 작업할 수 있습니다(예: 테스트 환경). 외부 사용자는 로그인 시 홈 디렉터리에 의해 인증되며 이러한 인증은 사용자가 멤버로 포함되는 다른 모든 디렉터리에서 유효합니다.

외부 사용자를 만들려면 Azure 클래식 포털에서 사용자를 만들고 **사용자 형식**으로 **다른 Azure AD 디렉터리의 사용자**를 선택합니다.

## 외부 사용자 관리 및 제한 사항

하나의 디렉터리에 포함된 사용자를 새 디렉터리에 추가하면 해당 사용자는 새 디렉터리에서 외부 사용자가 됩니다. 처음에 표시 이름 및 사용자 이름이 사용자의 "홈 디렉터리"에서 복사된 후 다른 디렉터리에서 해당 외부 사용자에 스탬프로 지정됩니다. 이때부터 해당 이름 및 외부 사용자 개체의 다른 속성은 완전히 별개로 인식됩니다. 홈 디렉터리에서 사용자를 변경하는 경우(사용자의 이름 변경, 직책 추가 등) 변경 내용이 다른 디렉터리의 해당 외부 사용자 계정으로 전파되지 않습니다.

두 개체 사이의 유일한 연계는 사용자가 항상 홈 디렉터리에 대해 인증을 받거나 자신의 Microsoft 계정으로 인증을 받는다는 것입니다. 외부 사용자 계정에 대해 암호를 재설정하거나 다단계 인증을 사용하도록 설정하는 옵션이 표시되지 않는 이유가 바로 이 때문입니다. 현재, 홈 디렉터리 또는 Microsoft 계정의 인증 정책만 사용자가 로그인할 때 평가됩니다.

> [AZURE.NOTE]디렉터리의 외부 사용자는 여전히 비활성화할 수 있으며 이 경우 해당 외부 사용자는 사용자 디렉터리에 액세스할 수 없게 됩니다.

홈 디렉터리에서 사용자가 삭제되거나 사용자가 자신의 Microsoft 계정을 취소하더라도 해당 외부 사용자는 여전히 디렉터리에 존재합니다. 그러나 해당 사용자는 더 이상 홈 디렉터리 또는 Microsoft 계정에서 인증을 받을 수 없으므로 디렉터리의 리소스에 액세스할 수 없게 됩니다.

여러 디렉터리의 관리자는 Azure 클래식 포털에서 해당 디렉터리를 관리할 수 있습니다. 그러나 Office 365 등의 다른 응용 프로그램에서는 현재 다른 디렉터리의 외부 사용자로 서비스를 할당 및 액세스할 수 있는 환경을 제공하고 있지 않습니다. 향후, 개발자들에게 여러 디렉터리의 구성원인 사용자가 앱을 작동할 수 있도록 지원하는 방법을 제공할 예정입니다.

현재 관리자는 홈 디렉터리의 다중 테넌트 응용 프로그램에만 동의할 수 있으며 홈 디렉터리의 액세스 패널을 통해서만 SaaS 앱 및 SSO에 대해 프로비전될 수 있도록 제한됩니다. Microsoft 계정 사용자는 다중 테넌트 응용 프로그램에 동의하거나 액세스 패널을 사용할 수 없다는 점에서 동일한 제한이 적용됩니다.

## 게스트

**게스트**는 사용자 형식이 "게스트"로 설정된 디렉터리의 사용자입니다. 일반 사용자는 디렉터리의 멤버임을 나타내기 위해 사용자 형식이 "멤버"로 지정됩니다. 게스트는 Azure 구독에 Microsoft 계정을 추가하거나 SharePoint의 문서를 외부 사용자와 공유하는 경우처럼 디렉터리 외부의 사용자와 리소스를 공유할 때 만들어집니다.

게스트는 디렉터리에서 제한된 권한 집합을 갖습니다. 이러한 권한은 작업 중인 리소스와 연결된 사용자 및 그룹과 상호 작용할 수 있도록 하면서 디렉터리의 다른 사용자에 대한 정보를 검색하는 게스트의 능력을 제한합니다. 예를 들어 Azure 구독에 할당된 게스트는 Azure 구독과 연결된 다른 사용자 및 그룹을 볼 수 있습니다. 또한 해당 사용자의 전체 메일 주소를 알 경우, 디렉터리에서 구독에 대해 액세스 권한을 부여해야 하는 다른 사용자를 찾을 수도 있습니다. 게스트는 다른 사용자의 제한된 속성 집합만 볼 수 있습니다. 이러한 속성은 표시 이름, 메일 주소, UPN(사용자 계정 이름) 및 축소판 사진으로 제한됩니다.

## 사용자 액세스 정책 구성

디렉터리의 **구성** 탭에는 외부 사용자의 액세스를 제어하는 옵션이 포함되어 있습니다. 이러한 옵션은 디렉터리 전역 관리자가 전체 Azure 클래식 포털의 UI(Windows PowerShell 또는 API 메서드가 없음)에서만 변경할 수 있습니다. Azure 클래식 포털에서 **구성** 탭을 열려면 **Active Directory**를 클릭한 다음 디렉터리의 이름을 클릭합니다.

![][1]

그러면 외부 사용자의 액세스를 제어하는 옵션을 편집할 수 있습니다.

![][2]

기본적으로 게스트는 디렉터리의 내용을 열거할 수 없으므로 **구성원 목록**에서 사용자나 그룹을 확인할 수 없으며, 사용자의 전체 메일 주소를 입력해 사용자를 검색한 다음 액세스 권한을 부여할 수는 있습니다. 게스트에 기본적으로 적용되는 제한 집합은 다음과 같습니다.

- 디렉터리의 사용자와 그룹을 열거할 수 없습니다.
- 메일 주소를 아는 사용자의 제한적 세부 정보를 확인할 수 있습니다.
- 이름을 아는 그룹의 제한적 세부 정보를 확인할 수 있습니다.

게스트는 사용자나 그룹의 제한적 세부 정보를 확인하는 기능을 통해 다른 사용자를 초대하고 공동 작업 중인 사용자의 일부 세부 정보를 확인할 수 있습니다.
