# Azure AD를 사용하여 학습할 수 있는 다음 단계는 무엇입니까?


### 이 항목에 대한 다른 문서
[Azure AD란?](active-directory-whatis.md)<br> [작동 원리](active-directory-works.md)<br> [시작](active-directory-get-started.md)<br> [다음 단계](active-directory-next-steps.md)<br> [자세한 정보](active-directory-learn-map.md)

## Azure AD에서 사용자 관리하기
사용자 계정 관리는 사용자를 만들거나 삭제하고 사용 권한을 할당하는 작업뿐만 아니라 액세스 및 사용에 대한 보고서를 생성하고, 암호 정책을 설정하고, 암호 셀프 서비스 재설정을 사용하도록 설정하고, 다단계 인증을 설정하는 등의 작업도 포함됩니다. [사용자 관리에 대해 자세히 알아보세요](active-directory-create-users.md).

## 암호 동기화 및 관리
사용자가 온-프레미스 네트워크에 로그인하는 데 사용하는 것과 동일한 암호를 사용하여 Microsoft 클라우드 서비스(예: Office 365, Dynamics CRM, Microsoft Intune)에 로그인할 수 있도록 사용자 암호를 동기화합니다. [암호 관리에 대해 자세히 알아보세요](active-directory-manage-passwords.md).

## Azure AD에서 그룹 관리
각 계정별로 사용 권한을 할당하지 않고 그룹을 사용하여 한 번에 많은 계정에 일련의 사용 권한을 할당합니다. 또한 로컬 온-프레미스 Windows Server Active Directory에서 그룹을 동기화할 수도 있습니다. Azure Active Directory의 그룹 관리에 대해 자세히 알아보세요. [그룹 관리에 대해 자세히 알아보세요](active-directory-manage-groups.md).

## 등록된 모바일 장치에 조건부 액세스 정책 적용
Azure AD 장치 등록은 사용자가 로그인할 때 장치를 인증하는 데 사용되는 ID로 장치를 프로비전합니다. 그런 다음 인증된 장치 및 그 장치의 특성을 사용하여 클라우드 및 온-프레미스에 호스트되는 응용 프로그램에 조건부 액세스 정책을 적용할 수 있습니다. [Azure AD 장치 등록에 대한 자세히 알아보세요](active-directory-conditional-access.md).
