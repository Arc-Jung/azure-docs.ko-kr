# <a name="migrate-groups-from-one-forest-to-another-for-azure-ad-connect"></a>Azure AD Connect에 대해 한 포리스트에서 다른 포리스트로 그룹 마이그레이션

이 문서에서는 마이그레이션된 그룹 개체가 클라우드의 기존 개체와 일치하도록 그룹을 한 포리스트에서 다른 포리스트로 성공적으로 마이그레이션하는 데 필요한 단계를 설명합니다.

## <a name="prerequisites"></a>사전 요구 사항

- Azure AD Connect 버전 1.5.18.0 이상
- 소스 앵커 특성은`mS-DS-ConsistencyGuid`

버전 1.5.18.0부터 Azure AD Connect는 그룹에 `mS-DS-ConsistencyGuid` 대한 사용을 지원하기 시작했습니다. 원본 `mS-DS-ConsistencyGuid` 앵커 특성으로 선택되고 값이 AD에 채워지면 Azure AD Connect는 `mS-DS-ConsistencyGuid` 불변 Id의 값을 사용합니다. 그렇지 않으면 을 사용하는 `objectGUID`것으로 돌아갑니다. 그러나 Azure AD **Connect는** AD의 `mS-DS-ConsistencyGuid` 특성에 값을 다시 쓰지 않습니다.

그룹 개체가 한 포리스트(예: F1)에서 다른 포리스트(예: F2)로 이동하는 포리스트 간 `mS-DS-ConsistencyGuid` 이동 시나리오에서는 `objectGUID` 포리스트 F1의 `mS-DS-ConsistencyGuid` 개체에서 F2의 개체 속성으로 값(IF PRESENT) 또는 값을 복사해야 합니다. 

포리스트 F1에서 포리스트 F2로 단일 그룹을 마이그레이션하는 방법을 보려면 다음 스크립트를 지침으로 사용하십시오. 여러 그룹에 대한 마이그레이션을 수행하는 지침으로 이 것을 자유롭게 사용하십시오.

먼저 포리스트 `objectGUID` F1에서 그룹 개체와 `mS-DS-ConsistencyGuid` 그룹 개체를 가져옵니다. 이러한 특성은 CSV 파일로 내보내됩니다.
```
<#
DESCRIPTION
============
This script will take DN of a group as input.
It then copies the objectGUID and mS-DS-ConsistencyGuid values along with other attributes of the given group to a CSV file.

This CSV file can then be used as input to Export-Group script
#>
Param(
       [ValidateNotNullOrEmpty()]
       [string]
       $dn,

       [ValidateNotNullOrEmpty()]
       [string]
       $outputCsv
)

$defaultProperties = @('samAccountName', 'distinguishedName', 'objectGUID', 'mS-DS-ConsistencyGuid')
$group  = Get-ADGroup -Filter "DistinguishedName -eq '$dn'" -Properties $defaultProperties -ErrorAction Stop
$results = @()
if ($group -eq $null)
{
       Write-Error "Group not found"
}
else
{
       $objectGUIDValue = [GUID]$group.'objectGUID'
       $mSDSConsistencyGuidValue = "N/A"
       if ($group.'mS-DS-ConsistencyGuid' -ne $null)
       {
              $mSDSConsistencyGuidValue = [GUID]$group.'mS-DS-ConsistencyGuid'
       }
       $adgroup = New-Object -TypeName PSObject
       $adgroup | Add-Member -MemberType NoteProperty -Name samAccountName -Value $($group.'samAccountName')
       $adgroup | Add-Member -MemberType NoteProperty -Name distinguishedName -Value $($group.'distinguishedName')
       $adgroup | Add-Member -MemberType NoteProperty -Name objectGUID -Value $($objectGUIDValue)
       $adgroup | Add-Member -MemberType NoteProperty -Name mS-DS-ConsistencyGuid -Value $($mSDSConsistencyGuidValue)
       $results += $adgroup
}

Write-Host "Exporting group to output file"
$results | Export-Csv "$outputCsv" -NoTypeInformation

```

다음으로 생성된 출력 CSV 파일을 사용하여 `mS-DS-ConsistencyGuid` 포리스트 F2의 대상 개체에 특성을 스탬프합니다.


```
<#
DESCRIPTION
============
This script will take DN of a group as input and the CSV file that was generated by Import-Group script
It copies either the objectGUID or mS-DS-ConsistencyGuid value from CSV file to the given object.

#>
Param(
       [ValidateNotNullOrEmpty()]
       [string]
       $dn,

       [ValidateNotNullOrEmpty()]
       [string]
       $inputCsv
)

$group  = Get-ADGroup -Filter "DistinguishedName -eq '$dn'" -ErrorAction Stop
if ($group -eq $null)
{
       Write-Error "Group not found"
}

$csvFile = Import-Csv -Path $inputCsv -ErrorAction Stop
$msDSConsistencyGuid = [GUID] $csvFile.'mS-DS-ConsistencyGuid'
$objectGuid = [GUID] $csvFile.'objectGUID'
$targetGuid = $msDSConsistencyGuid

if ($msDSConsistencyGuid -eq "N/A")
{
       $targetGuid = $objectGuid
}

Set-ADGroup -Identity $dn -Replace @{'mS-DS-ConsistencyGuid'=$targetGuid} -ErrorAction Stop

```
