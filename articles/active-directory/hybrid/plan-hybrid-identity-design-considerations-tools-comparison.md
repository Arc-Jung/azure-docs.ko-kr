# <a name="hybrid-identity-directory-integration-tools-comparison"></a>하이브리드 ID 디렉터리 통합 도구 비교
디렉터리 통합 도구는 수년에 걸쳐 성장하고 발전했습니다.  


- [FIM](https://docs.microsoft.com/previous-versions/windows/desktop/forefront-2010/ff182370%28v%3dvs.100%29) 및 [MIM은](https://docs.microsoft.com/microsoft-identity-manager/microsoft-identity-manager-2016) 여전히 지원되며 주로 온-프레미스 시스템 간의 동기화를 가능하게 합니다.   [FIM Windows Azure AD 커넥터는](https://docs.microsoft.com/previous-versions/mim/dn511001(v=ws.10)?redirectedfrom=MSDN) FIM 및 MIM 모두에서 지원되지만 새 배포에는 권장되지 않습니다 .
- [Azure AD Connect 동기화는](how-to-connect-sync-whatis.md) AD DS 포리스트와 Azure AD 간의 동기화를 위해 DirSync 및 Azure AD Sync에서 이전에 릴리스된 구성 요소 및 기능을 통합합니다.  
- [Azure AD Connect 클라우드 프로비저닝은](../cloud-provisioning/what-is-cloud-provisioning.md) AD DS에서 Azure AD로 동기화하기 위한 새로운 Microsoft 에이전트로, 인수한 회사의 AD 포리스트가 모회사의 AD 포리스트에서 격리된 합병 및 인수와 같은 시나리오에 유용합니다.

Azure AD Connect 동기화와 Azure AD Connect 클라우드 프로비저닝의 차이점에 대해 자세히 알아보려면 [Azure AD Connect 클라우드 프로비저닝이란 무엇입니까?](../cloud-provisioning/what-is-cloud-provisioning.md)
