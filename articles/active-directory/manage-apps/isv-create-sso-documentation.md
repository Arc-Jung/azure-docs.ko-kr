# <a name="create-and-publish-single-sign-on-documentation-for-your-application"></a>만들기 및 응용 프로그램에 대 한 single sign-on 설명서를 게시 합니다.   

## <a name="documentation-on-your-site"></a>사이트에 대 한 설명서

채택 용이성이 중요 한 요인으로 기업 소프트웨어 의사 결정 합니다. 선택을 취소 하려면 쉬운 설명서 도입에 고객에 게를 지원 하며 지원 비용을 낮춥니다. 수천 개의 소프트웨어 공급 업체에서 작업할 Microsoft 무슨 일 표시 했습니다.

최소한 사이트에서 해당 설명서에 다음 항목을 포함 하는 것이 좋습니다.

* SSO 기능 소개

  * 지원 되는 프로토콜

  * 버전 및 SKU

  * 설명서 링크를 사용 하 여 지원 되는 Id 공급자 목록

* 응용 프로그램에 대 한 라이선스 정보

* SSO를 구성 하는 것에 대 한 역할 기반 액세스 제어

* SSO 구성 단계

  * SAML 공급자의 예상 값을 사용 하 여 UI 구성 요소

  * Id 공급자에 전달할 서비스 공급자 정보

* OIDC/OAuth 경우

  * 비즈니스 근거를 사용 하 여 동의 하는 데 필요한 사용 권한 목록

* 파일럿 사용자에 대 한 테스트 단계

* 오류 코드 및 메시지를 포함 한 문제 해결 정보

* 고객을 위한 메커니즘을 지원 합니다.

## <a name="documentation-on-the-microsoft-site"></a>Microsoft 사이트에 대 한 설명서

또한 Azure Marketplace에서 응용 프로그램을 게시, 하는 Azure Active Directory 응용 프로그램 갤러리를 사용 하 여 응용 프로그램을 나열 하는 경우 Microsoft는 단계별 프로세스를 설명 하는 상호 고객에 대 한 설명서를 생성 합니다. 예를 확인할 수 있습니다 [여기](https://aka.ms/appstutorial)합니다. 이 문서는 갤러리에 제출에 따라 만들어지고 GitHub 계정을 사용 하 여 응용 프로그램을 변경 하는 경우 쉽게 업데이트할 수 있습니다.
