# 그룹에 대한 동적 멤버 자격 문제 해결

| 증상 | 작업 |
|--------------------------------------------------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 그룹에 대한 규칙을 구성했으나 그룹에서 멤버 자격이 업데이트되지 않음 | 디렉터리 구성에 탭에서 위임된 그룹 관리 사용 설정이 예로 설정되어 있는지 확인합니다. Azure Active Directory Premium 라이선스가 할당된 계정으로 로그인한 경우에만 이 설정이 표시됩니다. 규칙의 사용자 특성 값을 확인합니다. 규칙을 충족하는 사용자가 있나요? |
| 규칙을 구성했으나 규칙의 기존 멤버가 제거됨 | 예상된 결과입니다. 규칙을 사용하도록 설정하거나 변경하면 그룹의 기존 멤버가 제거됩니다. 규칙 평가에서 사용자의 결과 집합이 그룹 멤버로 추가됩니다. |
| 규칙을 추가하거나 변경하는 즉시 멤버 자격 변경 내용이 표시되지 않는 이유는? | 전용 멤버 자격 평가가 비동기 백그라운드 프로세스에서 주기적으로 수행됩니다. 테넌트의 사용자 수와 규칙 결과로 만들어진 그룹의 크기가 소요 시간을 결정하는 요인입니다. 일반적으로 사용자 수가 적은 테넌트에는 2-3분 내에 그룹 멤버 자격 변경 내용이 표시됩니다. 사용자 수가 많은 테넌트는 채우는 데 30분 이상이 걸릴 수 있습니다. |
