# <a name="how-to-give-risk-feedback-in-azure-ad-identity-protection"></a>방법: Azure AD ID 보호에서 위험 피드백 제공

Azure AD ID 보호를 사용하면 위험 평가에 대한 피드백을 제공할 수 있습니다. 다음 문서에는 Azure AD ID Protection의 위험 평가에 대한 피드백을 제공하고자 하는 시나리오와 이를 통합하는 방법에 대해 설명합니다.

## <a name="what-is-a-detection"></a>탐지란 무엇입니까?

ID 보호 검색은 ID 위험 관점에서 의심스러운 활동을 나타내는 지표입니다. 이러한 의심스러운 활동을 위험 탐지라고 합니다. 이러한 ID 기반 탐지는 추론, 기계 학습을 기반으로 하거나 파트너 제품에서 올 수 있습니다. 이러한 검색은 로그인 위험 및 사용자 위험을 결정하는 데 사용됩니다.

* 사용자 위험은 ID가 손상될 가능성을 나타냅니다.
* 로그인 위험은 로그인이 손상될 가능성을 나타냅니다(예: 로그인은 ID 소유자가 승인하지 않음).

## <a name="why-should-i-give-risk-feedback-to-azure-ads-risk-assessments"></a>Azure AD의 위험 평가에 위험 피드백을 제공해야 하는 이유는 무엇입니까? 

Azure AD 위험 피드백을 제공해야 하는 몇 가지 이유가 있습니다.

- **Azure AD의 사용자 또는 로그인 위험 평가가 올바르지 않습니다.** 예를 들어 '위험한 로그인' 보고서에 표시된 로그인은 양호하고 로그인의 모든 검색은 거짓 긍정이었습니다.
- **Azure AD의 사용자 또는 로그인 위험 평가가 올바른지 확인했습니다.** 예를 들어 '위험한 로그인' 보고서에 표시된 로그인은 실제로 악의적이며 Azure AD가 해당 로그인의 모든 검색이 참 긍정임을 알기를 원합니다.
- **Azure AD ID 보호 외부에서 해당 사용자의 위험을 수정했으며** 사용자의 위험 수준을 업데이트하려고 합니다.

## <a name="how-does-azure-ad-use-my-risk-feedback"></a>Azure AD는 내 위험 피드백을 어떻게 사용합니까?

Azure AD는 피드백을 사용하여 기본 사용자 및/또는 로그인의 위험과 이러한 이벤트의 정확성을 업데이트합니다. 이 피드백은 최종 사용자를 보호하는 데 도움이 됩니다. 예를 들어 로그인이 손상되어 있는지 확인하면 Azure AD는 사용자의 위험과 로그인의 집계 위험(실시간 위험이 아님)을 높음으로 즉시 증가시킵니다. 이 사용자가 사용자 위험 정책에 포함되어 있어 위험도가 높은 사용자가 암호를 안전하게 재설정하도록 하면 다음에 로그인할 때 자동으로 수정됩니다.

## <a name="how-should-i-give-risk-feedback-and-what-happens-under-the-hood"></a>위험 피드백을 제공어떻게 해야 하며, 후드 아래에서 어떤 일이 발생합니까?

Azure AD에 위험 피드백을 제공하는 시나리오 및 메커니즘은 다음과 같습니다.

| 시나리오 | 피드백을 제공하는 방법? | 후드 아래 에서 무슨 일이 일어나는가? | 메모 |

| **로그인이 손상되지 않음(거짓 긍정)** <br> '위험한 로그인' 보고서에는 위험에 처한 로그인 [위험 상태 = 위험 상태]가 표시되지만 로그인이 손상되지 않았습니다. | 로그인을 선택하고 '로그인 안전 확인'을 클릭합니다. | Azure AD는 로그인의 집계 위험을 없음으로 이동합니다[위험 상태 = 안전 확인됨; 위험 수준(집계) = -] 사용자 위험에 미치는 영향을 반전시바립니다. | 현재 '로그인 안전 확인' 옵션은 '위험한 로그인' 보고서에서만 사용할 수 있습니다. |
| **로그인 손상(참 긍정)** <br> '위험한 로그인' 보고서는 위험이 낮은 [위험 수준(집계) = 낮음]이 있는 위험에 처한 로그인 [위험 상태 = 위험 상태]를 표시하고 로그인이 실제로 손상되었습니다. | 로그인을 선택하고 '로그인 이 손상된 확인'을 클릭합니다. | Azure AD는 로그인의 집계 위험과 사용자 위험을 높음 [위험 상태 = 손상된 것으로 확인된 상태로 이동합니다.] 위험 수준 = 높음]. | 현재 '로그인 손상 확인' 옵션은 '위험한 로그인' 보고서에서만 사용할 수 있습니다. |
| **사용자가 손상된 경우(참 긍정)** <br> '위험한 사용자' 보고서는 위험이 낮은 [위험 수준 = 낮음]을 가진 위험에 처한 사용자 [위험 상태 = 위험 상태]를 보여 주며 해당 사용자가 실제로 손상되었습니다. | 사용자를 선택하고 '사용자 손상 확인'을 클릭합니다. | Azure AD는 사용자 위험을 높음 [위험 상태 = 손상된 것으로 확인된 상태로 이동합니다. 위험 수준 = 높음] 및 새 탐지 '관리자확인 된 사용자 손상'을 추가합니다. | 현재 '사용자 손상 확인' 옵션은 '위험한 사용자' 보고서에서만 사용할 수 있습니다. <br> '관리자가 사용자 손상 확인'이라는 검색은 '위험한 사용자' 보고서의 '로그인에 연결되지 않은 위험 감지' 탭에 표시됩니다. |

| **사용자가 손상되지 않음(거짓 긍정)** <br> '위험한 사용자' 보고서는 위험에 처한 사용자에게 표시되지만 사용자가 손상되지는 않습니다. | 사용자를 선택하고 '사용자 위험 해제'를 클릭합니다. 이 프로세스는 Azure AD에 사용자가 손상되지 않음을 확인합니다. | Azure AD는 사용자 위험을 없음으로 이동 [위험 상태 = 해제됨; 위험 수준 = -]. | '사용자 위험 해제'를 클릭하면 사용자 및 과거 로그인에 대한 모든 위험이 닫힙집니다. 이 작업은 실행 취소할 수 없습니다. |
| 나는 사용자 위험을 닫고 싶지만 사용자가 손상되었는지 여부 / 안전여부. | 사용자를 선택하고 '사용자 위험 해제'를 클릭합니다. 이 프로세스는 Azure AD에 사용자가 더 이상 손상되지 않는지 확인합니다. | Azure AD는 사용자 위험을 없음으로 이동 [위험 상태 = 해제됨; 위험 수준 = -]. | '사용자 위험 해제'를 클릭하면 사용자 및 과거 로그인에 대한 모든 위험이 닫힙집니다. 이 작업은 실행 취소할 수 없습니다. '암호 재설정'을 클릭하여 사용자를 수정하거나 사용자에게 자격 증명을 안전하게 재설정/변경하도록 요청하는 것이 좋습니다. |

ID 보호의 사용자 위험 검색에 대한 피드백은 오프라인으로 처리되며 업데이트하는 데 다소 시간이 걸릴 수 있습니다. 위험 처리 상태 열은 피드백 처리의 현재 상태를 제공합니다.
