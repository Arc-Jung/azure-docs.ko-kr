# <a name="how-to-give-risk-feedback-in-azure-ad-identity-protection"></a>방법: Azure AD Id 보호에서 위험 피드백을 제공 합니다.

Azure AD Id 보호를 사용 하면 해당 위험 평가에 피드백을 제공할 수 있습니다. 다음 문서는 Azure AD Id 보호 위험 평가 및에서는 통합 하는 방법에 대 한 피드백을 제공 하려는 시나리오를 나열 합니다.

## <a name="what-is-a-detection"></a>검색 이란?

Id 보호 검색은 identity 위험 관점에서 의심 스러운 활동의 표시기입니다. 이러한 의심 스러운 활동 위험 이벤트 라고 합니다. 이러한 id를 기반으로 검색 추론에 따라, 기계 학습 또는 파트너 제품에서 가져올 수 있습니다. 이러한 검색은 로그인 위험 및 사용자 위험을 확인 하는 데 사용 됩니다.

* 사용자 위험 id가 손상 될 확률을 나타냅니다.
* 로그인 위험 로그인에 손상 된 확률을 나타냅니다 (예를 들어 로그인 권한을 부여 하지 않았습니다 identity 소유자).

## <a name="why-should-i-give-risk-feedback-to-azure-ads-risk-assessments"></a>Azure AD의 위험 평가에 위험 피드백을 제공 해야는 이유 

Azure AD 위험 피드백을 제공 해야 하는 이유는 여러 가지가 있습니다.

1. **잘못 된 Azure AD의 사용자 또는 로그인 위험 평가 찾을**합니다. 예를 들어, ' 위험한 로그인 ' 보고서에 표시 된 로그인 무해 한 되었으며 해당 로그인으로 모든 검색 된 거짓 긍정입니다.
1. **Azure AD는 사용자의 유효성을 검사 하거나 로그인 위험 평가 올바르지**합니다. 예를 들어, ' 위험한 로그인 ' 보고서에 표시 된 로그인 실제로 악의적인 되어 Azure ad는 로그인 시 모든 검색 참 긍정 되었는지 알아야 합니다.
1. **Azure AD Id 보호 외부에서 해당 사용자에 대 한 위험을 재구성** 및 업데이트할 사용자의 위험 수준.

## <a name="how-does-azure-ad-use-my-risk-feedback"></a>Azure AD 내 위험 피드백을 사용 하는 어떻게 하나요?

Azure AD 기본 사용자 및/또는 로그인 위험을 업데이트 하려면 여러분의 의견을 사용 합니다. 이 피드백을 통해 최종 사용자를 보호 합니다. 예를 들어 손상 된 로그인을 확인 하 고 나면 즉시 Azure AD 사용자의 위험 및 로그인의 집계 (없습니다 실시간 위험)에 위험이 높은 증가 합니다. 이 사용자는 강제로 위험 수준이 높은 사용자가 안전 하 게 암호를 재설정 하도록 사용자 위험 정책에 포함 하는 경우 사용자에서 자동으로 해결 자체 다음에 로그인 합니다.
