# <a name="how-to-investigate-risk"></a>방법: 위험 조사

위험한 로그인, 위험한 사용자 및 위험 검색 보고서를 사용 하 여 사용자 환경에서 위험을 조사 하 고 통찰력을 얻을 수 있습니다. 위험한 로그인, 사용자 및 검색을 필터링 하 고 정렬 하는 기능을 사용 하 여 조직의 잠재적 침입을 더 잘 이해할 수 있습니다. 

## <a name="risky-users-report"></a>위험한 사용자 보고서

위험한 사용자 보고서에서 제공되는 정보를 사용하면 다음과 같은 질문에 대한 대답을 찾을 수 있습니다.

- 어떤 사용자의 위험 수준이 높은가요?
- 어떤 사용자의 위험 상태가 수정됨인가요?

이 보고서의 첫 번째 진입점은 보안 페이지의 **조사** 섹션입니다.

![위험한 사용자 보고서](./media/howto-investigate-risky-users-signins/01.png)

위험한 사용자 보고서에는 다음을 표시하는 기본 보기가 있습니다.

- 이름
- 위험 상태
- 위험 수준
- 위험 세부 정보
- 위험을 마지막으로 업데이트한 날짜
- 형식
- Status

![위험한 사용자 보고서](./media/howto-investigate-risky-users-signins/03.png)

도구 모음에서 **열**을 클릭하여 목록 보기를 사용자 지정할 수 있습니다.

![위험한 사용자 보고서](./media/howto-investigate-risky-users-signins/04.png)

열 대화 상태를 사용하여 추가 필드를 표시하거나 이미 표시된 필드를 제거할 수 있습니다.

목록 보기에서 항목을 클릭하면 가로 보기로 해당 항목에 대해 사용할 수 있는 세부 정보를 모두 얻을 수 있습니다.

![위험한 사용자 보고서](./media/howto-investigate-risky-users-signins/05.png)

세부 정보 보기는 다음을 보여 줍니다.

- 기본 정보
- 최근 위험한 로그인
- 로그인에 연결 되지 않은 위험 감지
- 위험 기록

또한 다음을 수행할 수 있습니다.

![위험한 사용자 보고서](./media/howto-investigate-risky-users-signins/08.png)

- 해당 사용자에 대한 로그인 보고서를 볼 수 있는 모든 로그인 바로 가기를 확인할 수 있습니다.
- 위험한 사용자로 플래그가 지정된 사용자의 모든 로그인을 볼 수 있는 모든 위험한 로그인을 확인할 수 있습니다.
- 사용자 ID가 손상되었다고 생각되는 경우 사용자의 암호를 재설정할 수 있습니다.
- 사용자의 활성 위험 검색이 가양성 이라고 생각 하는 경우 사용자 위험을 해제 합니다. 자세한 내용은 [Azure AD ID 보호에서 위험 검색에 대 한 피드백 제공](howto-provide-risk-event-feedback.md)문서를 참조 하세요.

### <a name="filter-risky-users"></a>위험한 사용자 필터링

보고되는 데이터를 자신에게 적합한 수준으로 좁히려면 다음 기본 필드를 사용하여 위험한 사용자 데이터를 필터링하면 됩니다.

- 이름
- Username
- 위험 상태
- 위험 수준
- 형식
- Status

![위험한 사용자 보고서](./media/howto-investigate-risky-users-signins/06.png)

**이름** 필터를 사용하면 관심 있는 사용자의 이름이나 UPN(사용자 계정 이름)을 지정할 수 있습니다.

**위험 상태** 필터를 사용하면 다음을 선택할 수 있습니다.

- 위험 노출
- 수정됨
- 해제됨

**위험 수준** 필터를 사용하면 다음을 선택할 수 있습니다.

- 높음
- 중간
- 낮음

**유형** 필터를 사용하면 다음을 선택할 수 있습니다.

- Guest
- 멤버

**상태** 필터를 사용하면 다음을 선택할 수 있습니다.

- 삭제됨
- 활성

### <a name="download-risky-users-data"></a>위험한 사용자 데이터 다운로드

Azure Portal 외부에서 작업 하려는 경우 위험한 사용자 데이터를 다운로드할 수 있습니다. 다운로드를 클릭 하면 가장 최근 2500 레코드의 CSV 파일이 생성 됩니다. 

![위험한 사용자 보고서](./media/howto-investigate-risky-users-signins/07.png)

도구 모음에서 [열]을 클릭하여 목록 보기를 사용자 지정할 수 있습니다.
 
그러면 추가 필드를 표시하거나 이미 표시된 필드를 제거할 수 있습니다.
 
위험한 사용자에 대해 자세히 알아보려면 세부 정보 서랍을 클릭하여 확장

## <a name="risky-sign-ins-report"></a>위험한 로그인 보고서

위험한 로그인 보고서에서 제공되는 정보를 사용하면 다음과 같은 질문에 대한 대답을 찾을 수 있습니다.

- 지난 주에 익명 IP 주소 위험이 검색 된 성공한 로그인 수는 몇 개입니까?
- 어떤 사용자가 지난달에 손상된 것으로 확인되었나요?
- 어떤 사용자가 Office 365 포털에 대한 위험한 로그인을 가지나요?

이 보고서의 첫 번째 진입점은 보안 페이지의 **조사** 섹션입니다.

![위험한 로그인 보고서](./media/howto-investigate-risky-users-signins/02.png)

위험한 로그인 보고서에는 다음을 보여 주는 기본 보기가 있습니다.

- 사용자
- 애플리케이션
- 로그인 상태
- 위험 상태
- 위험 수준(집계)
- 위험 수준(실시간)
- 조건부 액세스
- 필요한 MFA  

![위험한 로그인 보고서](./media/howto-investigate-risky-users-signins/09.png)

도구 모음에서 **열**을 클릭하여 목록 보기를 사용자 지정할 수 있습니다.

![위험한 사용자 보고서](./media/howto-investigate-risky-users-signins/11.png)

열 대화 상태를 사용하여 추가 필드를 표시하거나 이미 표시된 필드를 제거할 수 있습니다.

목록 보기에서 항목을 클릭하면 가로 보기로 해당 항목에 대해 사용할 수 있는 세부 정보를 모두 얻을 수 있습니다.

![위험한 사용자 보고서](./media/howto-investigate-risky-users-signins/12.png)

- MFA 정보
- 조건부 액세스

![위험한 사용자 보고서](./media/howto-investigate-risky-users-signins/13.png)

- 손상됨 확인

자세한 내용은 [Azure AD ID 보호에서 위험 검색에 대 한 피드백 제공](howto-provide-risk-event-feedback.md)문서를 참조 하세요.

### <a name="filter-risky-sign-ins"></a>위험한 로그인 필터링

보고되는 데이터를 자신에게 적합한 수준으로 좁히려면 다음 기본 필드를 사용하여 위험한 사용자 데이터를 필터링하면 됩니다.

- 사용자
- 애플리케이션
- 로그인 상태
- 위험 상태
- 위험 수준(집계)
- 위험 수준(실시간)
- 조건부 액세스
- 날짜
- 위험 수준 유형

![위험한 로그인 보고서](./media/howto-investigate-risky-users-signins/14.png)

**이름** 필터를 사용하면 관심 있는 사용자의 이름이나 UPN(사용자 계정 이름)을 지정할 수 있습니다.

**애플리케이션** 필터를 사용하면 사용자가 액세스하려고 시도한 클라우드 앱을 지정할 수 있습니다.

**로그인 상태** 필터를 사용하면 다음을 선택할 수 있습니다.

- 모두
- Success
- 실패

**위험 상태** 필터를 사용하면 다음을 선택할 수 있습니다.

- 위험 노출
- 손상 확인됨
- 안전 확인됨
- 해제됨
- 수정됨

**위험 수준(집계)** 필터를 사용하면 다음을 선택할 수 있습니다.

- 높음
- 중간
- 낮음

**위험 수준(실시간)** 필터를 사용하면 다음을 선택할 수 있습니다.

- 높음
- 중간
- 낮음

**조건부 액세스** 필터를 사용 하면 다음을 선택할 수 있습니다.

- 모두
- 적용되지 않음
- Success
- 실패

**날짜** 필터를 사용하면 반환되는 데이터의 시간 범위를 정의할 수 있습니다.
가능한 값은

- 지난 1개월
- 최근 7일
- 지난 24시간
- 사용자 지정 시간 간격

Azure Portal 외부에서 작업 하려는 경우 위험한 로그인 데이터를 다운로드할 수 있습니다. 다운로드를 클릭 하면 가장 최근 2500 레코드의 CSV 파일이 생성 됩니다.

## <a name="risk-detections-report"></a>위험 검색 보고서

위험 검색 보고서는 테 넌 트의 모든 Id 보호 위험 검색에 대 한 통찰력을 제공 합니다.


위험 검색 보고서에는 다음을 보여 주는 기본 보기가 있습니다.

- 날짜

- 사용자

- IP 주소

- 위치

- 검색 유형

- 위험 상태

- 위험 수준

- 요청 ID
 

도구 모음에서 **열**을 클릭하여 목록 보기를 사용자 지정할 수 있습니다. 열 대화 상태를 사용하여 추가 필드를 표시하거나 이미 표시된 필드를 제거할 수 있습니다.

목록 보기에서 항목을 클릭하면 가로 보기로 해당 항목에 대해 사용할 수 있는 세부 정보를 모두 얻을 수 있습니다.


세부 정보 보기에는 위험 검색에 대 한 추가 정보가 표시 됩니다.

자세히 보기에는 다음에 대 한 링크도 포함 되어 있습니다.

- 사용자의 위험 보고서 보기
- 사용자 로그인 보기
- 사용자의 위험한 로그인 보기
- 연결 된 위험한 로그인 보기
- 사용자의 위험 검색 보기


### <a name="filter-risk-detections"></a>위험 검색 필터링

보고 된 데이터를 자신에 게 적합 한 수준으로 좁히려면 다음 기본 필드를 사용 하 여 위험 검색 데이터를 필터링 할 수 있습니다.

- 검색 시간
- 검색 유형
- 위험 상태
- 위험 수준
- 사용자
- Username
- IP 주소
- 위치
- 검색 타이밍
- 활동 
- Source
- 토큰 발급자 유형
- 요청 ID
- 상관 관계 ID


**검색 시간** 필터를 사용 하면 반환 된 데이터에 대 한 기간을 정의할 수 있습니다. 이 필터를 사용 하면 다음을 선택할 수 있습니다.
- 최근 90일
- 지난 30일
- 최근 7일
- 지난 24시간
- 사용자 지정 시간 간격

**검색 유형** 필터를 사용 하 여 위험 검색 유형 (예: 익숙하지 않은 로그인 속성)을 지정할 수 있습니다.

**위험 상태** 필터를 사용하면 다음을 선택할 수 있습니다.

- 위험 노출
- 손상 확인됨
- 안전 확인됨
- 해제됨
- 수정됨

**위험 수준** 필터를 사용하면 다음을 선택할 수 있습니다.

- 높음
- 중간
- 낮음

**사용자** 필터를 사용 하면 관심 있는 사용자의 이름 또는 개체 ID를 지정할 수 있습니다.

사용자 **이름** 필터를 사용 하면 관심 있는 사용자의 UPN (사용자 계정 이름)을 지정할 수 있습니다.

**검색 타이밍** 필터를 사용 하 여 검색의 실시간 또는 오프 라인 여부를 지정할 수 있습니다. 참고: 로그인 위험 정책을 사용 하 여 실시간 감지로 로그인 할 수 있습니다. 

**활동** 필터를 사용 하면 검색을 로그인 (예: 익명 IP 주소) 또는 사용자 (예: 유출 자격 증명)에 연결할지 여부를 지정할 수 있습니다.

**원본** 필터를 사용 하 여 위험 검색의 원본 (예: Azure AD 또는 Microsoft Cloud App Security)을 지정할 수 있습니다. 

**토큰 발급자 유형** 필터를 사용 하 여 토큰이 발급 된 위치 (예: Azure AD 또는 Ad Federation Services)를 지정할 수 있습니다.


### <a name="download-risk-detections-data"></a>위험 검색 데이터 다운로드

Azure Portal 외부에서 작업 하려는 경우 위험 검색 데이터를 다운로드할 수 있습니다. 다운로드를 클릭 하면 가장 최근 5000 레코드의 CSV 파일이 생성 됩니다.
