# <a name="what-is-risk"></a>위험이란 무엇입니까?

Azure AD ID 보호의 위험 검색에는 디렉터리의 사용자 계정과 관련된 확인된 의심스러운 작업이 포함됩니다.

ID 보호는 조직이 이러한 의심스러운 작업을 확인하고 신속하게 대응할 수 있는 강력한 리소스에 대한 액세스를 제공합니다. 

![위험한 사용자 및 로그인을 보여주는 보안 개요](./media/concept-identity-protection-risks/identity-protection-security-overview.png)

## <a name="risk-types-and-detection"></a>위험 유형 및 탐지

위험 **사용자** 및 **로그인에는** 두 가지 유형의 감지 또는 계산 **실시간** 및 **오프라인이**있습니다.

### <a name="user-risk"></a>사용자 위험

사용자 위험은 지정된 ID 또는 계정이 손상될 가능성을 나타냅니다. 

이러한 위험은 보안 연구원, 법 집행 전문가, Microsoft의 보안 팀 및 기타 신뢰할 수 있는 소스를 비롯한 Microsoft의 내부 및 외부 위협 인텔리전스 소스를 사용하여 오프라인으로 계산됩니다.

| 위험 감지 | 설명 |
| --- | --- |
| 유출된 자격 증명 | 이 위험 검색 유형은 사용자의 유효한 자격 증명이 유출되었음을 나타냅니다. 사이버 범죄자가 합법적인 사용자의 유효한 암호를 손상시키는 경우 범죄자는 종종 이러한 자격 증명을 공유합니다. 이 공유는 일반적으로 어두운 웹에 공개적으로 게시 하 여 이루어집니다., 붙여 넣기 사이트, 또는 거래 하 고 암시장에 자격 증명을 판매 하 여. Microsoft 유출 된 자격 증명 서비스는 어두운 웹, 붙여 넣기 사이트 또는 기타 소스에서 사용자 자격 증명을 획득하면 Azure AD 사용자의 현재 유효한 자격 증명에 대해 확인되어 유효한 일치 항목을 찾습니다. |
| Azure AD 위협 인텔리전스 | 이 위험 검색 유형은 지정된 사용자에게 는 비정상적이거나 Microsoft의 내부 및 외부 위협 인텔리전스 소스를 기반으로 알려진 공격 패턴과 일치하는 사용자 활동을 나타냅니다. |

### <a name="sign-in-risk"></a>로그인 위험

로그인 위험은 지정된 인증 요청이 ID 소유자가 승인하지 않을 가능성을 나타냅니다. 

이러한 위험은 보안 연구원, 법 집행 전문가, Microsoft의 보안 팀 및 기타 신뢰할 수 있는 소스를 비롯한 Microsoft의 내부 및 외부 위협 인텔리전스 소스를 사용하여 실시간으로 계산하거나 오프라인으로 계산할 수 있습니다.

| 위험 감지 | 검색 유형 | 설명 |
| --- | --- | --- |
| 익명 IP 주소 | 실시간 | 이 위험 검색 유형은 익명 IP 주소(예: Tor 브라우저 또는 익명 VPN)의 로그인을 나타냅니다. 악의적일 수 있는 의도로 IP 주소, 위치, 디바이스 등의 로그인 원격 분석 정보를 숨기려는 사용자가 대개 이러한 IP 주소를 사용합니다. |
| 비정상적 이동 | 오프라인 | 이 위험 검색 유형은 지리적으로 먼 위치에서 발생하는 두 개의 로그인을 식별하며, 여기서 위치 중 하나 이상이 사용자에게 비정형적일 수 있습니다( 과거 동작). 다른 요인 중에서 이 Machine Learning 알고리즘은 두 번의 로그인 간 시간과 사용자가 첫 번째 위치에서 두 번째 위치로 이동하는 데 걸리는 시간을 고려하여 서로 다른 사용자가 동일한 자격 증명을 사용하고 있음을 나타냅니다. <br><br> 이 알고리즘은 조직의 다른 사용자가 정기적으로 사용하는 VPN 및 위치와 같은 불가능한 이동 조건에 영향을 주는 확실한 "가양성"을 무시합니다. 이 시스템은 새 사용자의 로그인 동작을 학습하는 동안 14일 또는 10개의 로그인 의 초기 학습 기간을 가수합니다. |
| 맬웨어 연결 IP 주소 | 오프라인 | 이 위험 검색 유형은 봇 서버와 적극적으로 통신하는 것으로 알려진 맬웨어에 감염된 IP 주소의 로그인을 나타냅니다. 이 검색은 봇 서버가 활성화되어 있는 동안 봇 서버와 접촉한 IP 주소와 사용자 장치의 IP 주소를 상호 연관시켜 결정합니다. |
| 일반적이지 않은 로그인 속성 | 실시간 | 이 위험 검색 유형은 과거 로그인 기록(IP, 위도/경도 및 ASN)을 고려하여 비정상적인 로그인을 찾습니다. 시스템은 사용자가 사용한 이전 위치에 대한 정보를 저장하고 이러한 "익숙한" 위치를 고려합니다. 위험 검색은 익숙한 위치 목록에 없는 위치에서 로그인이 발생하면 트리거됩니다. 새로 생성된 사용자는 알고리즘이 사용자의 동작을 학습하는 동안 익숙하지 않은 로그인 속성 위험 감지가 꺼지는 기간 동안 "학습 모드"에 있습니다. 학습 모드 기간은 동적이며 알고리즘이 사용자의 로그인 패턴에 대한 충분한 정보를 수집하는 데 걸리는 시간에 따라 달라집니다. 최소 기간은 5일입니다. 사용자는 오랜 기간 동안 사용하지 않으면 학습 모드로 돌아갈 수 있습니다. 또한 시스템은 익숙한 디바이스 및 익숙한 위치에 지리적으로 가까운 위치에서 시도한 로그인을 무시합니다. <br><br> 기본 인증이나 레거시 프로토콜의 경우에도 이 검색이 실행됩니다. 이러한 프로토콜에는 클라이언트 ID와 같은 최신 속성이 없으므로 가양성을 줄일 수 있는 원격 분석 정보가 제한적으로만 제공됩니다. 따라서 고객은 최신 인증 방식으로 이전하는 것이 좋습니다. |
| 관리자가 사용자 손상확인 | 오프라인 | 이 검색은 관리자가 위험한 사용자 UI에서 '사용자 손상 확인'을 선택했거나 riskyUsers API를 사용했음을 나타냅니다. 이 사용자가 손상된 것을 확인한 관리자를 확인하려면 UI 또는 API를 통해 사용자의 위험 기록을 확인합니다. |
| 악성 IP 주소 | 오프라인 | 이 검색은 악의적인 IP 주소의 로그인을 나타냅니다. IP 주소는 IP 주소 또는 기타 IP 평판 원본에서 받은 잘못된 자격 증명으로 인해 높은 실패율을 기반으로 악의적인 것으로 간주됩니다. |
| 의심스러운 받은 편지함 조작 규칙 | 오프라인 | 이 검색은 [Microsoft 클라우드 앱 보안(MCAS)에서](/cloud-app-security/anomaly-detection-policy#suspicious-inbox-manipulation-rules)발견됩니다. 이 검색 기능은 사용자 환경을 프로파일링하여 메시지나 폴더를 삭제하거나 이동하는 의심스러운 규칙이 사용자의 받은 편지함에 설정된 경우 경고를 트리거합니다. 이는 사용자의 계정이 손상되었으며, 메시지가 의도적으로 숨겨지고 있으며, 사서함이 조직에서 스팸 또는 맬웨어를 배포하는 데 사용되고 있음을 나타낼 수 있습니다. |
| 이동 불가능 | 오프라인 | 이 검색은 [Microsoft 클라우드 앱 보안(MCAS)에서](/cloud-app-security/anomaly-detection-policy#impossible-travel)발견됩니다. 이 검색은 사용자가 첫 번째 위치에서 두 번째 위치로 이동하는 데 걸리는 시간보다 짧은 기간 내에 지리적으로 먼 위치에서 시작되는 두 개의 사용자 활동(단일 또는 복수 세션)을 식별합니다. 이는 다른 사용자가 동일한 자격 증명을 사용하고 있음을 나타냅니다. |

### <a name="other-risk-detections"></a>기타 위험 탐지

| 위험 감지 | 검색 유형 | 설명 |
| --- | --- | --- |
| 추가 위험 감지 | 실시간 또는 오프라인 | 이 검출은 위의 프리미엄 검출 중 하나가 검출되었다는 것을 나타냅니다. 프리미엄 검색은 Azure AD Premium P2 고객에게만 표시되므로 Azure AD Premium P2 라이선스가 없는 고객에 대해 "추가 위험 감지"라는 제목이 지정됩니다. |
