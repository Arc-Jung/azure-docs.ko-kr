# <a name="monitor-azure-fxt-edge-filer-hardware-status"></a>Azure FXT Edge 필터가 하드웨어 상태 모니터링

Azure FXT Edge 필터가 하이브리드 저장소 캐시 시스템은 관리자가 하드웨어는 작동 방식을 이해 하는 데 섀시에 기본 제공 되는 여러 상태 표시등입니다.

## <a name="system-health-status"></a>시스템 상태

높은 수준에서 캐시 작업을 모니터링 하려면 소프트웨어 제어판의를 사용 하 여 **대시보드** 에 설명 된 대로 페이지는 [컨트롤 패널 대시보드 가이드](https://azure.github.io/Avere/legacy/dashboard/4_7/html/ops_dashboard_index.html)

## <a name="hardware-status-leds"></a>하드웨어 상태 Led

이 섹션에서는 Azure FXT Edge 필터가 하드웨어에 기본 제공 되는 다양 한 상태 표시등을 설명 합니다.

### <a name="hard-drive-status-leds"></a>하드 드라이브 상태 Led

![가로 설명선 레이블 2 사용 하 여 하드 드라이브 앞의 그림 (왼쪽된 상단 모서리)을 1 (왼쪽된 아래 모서리) 및 3 (오른쪽)](media/fxt-monitor/fxt-drive-callouts.png)

각 드라이브 캐리어에 두 개의 상태 led가: 활동 표시기 (1) 및 상태 표시기 (2). 

* 드라이브를 사용 중인 경우 활동 LED (1) 표시등입니다.  
* 상태 LED (2)는 아래 표에 코드를 사용 하 여 드라이브의 상태를 나타냅니다.

| 드라이브 상태 LED 상태              | 의미  |
|-------------------------------------|----------------------------------------------------------|
| 두 번째 두 번씩 녹색 깜빡임      | 드라이브를 식별 *또는* <br> 제거를 위한 드라이브 준비  |
| Off (켜지)                         | 시작 완료 되지 않았습니다 *또는* <br>드라이브를 제거할 준비가 되었습니다. |
| 깜빡임은 녹색에서 주황색으로 변함 및 해제       | 드라이브 오류 예측   |
| 초당 4 번 깜빡입니다 주황색 | 실패 한 드라이브   |
| 단색 녹색                         | 드라이브가 온라인 |

(3) 드라이브의 오른쪽에 있는 드라이브의 용량 및 기타 정보를 사용 하 여 레이블이 지정 됩니다.

드라이브 숫자 드라이브 사이의 공간에 인쇄 됩니다. Azure FXT Edge 필터가에 드라이브 0 드라이브 왼쪽 위에 이며 1 드라이브 바로 아래입니다. 번호 매기기가 해당 패턴에서 계속 합니다. 

![숫자 및 용량 레이블이 드라이브를 보여 주는 FXT 섀시에 베이 이상의 하드 드라이브의 사진](media/fxt-drives-photo.png)

## <a name="left-control-panel"></a>왼쪽된 제어판

왼쪽된 전면 제어판에는 다양 한 상태 LED 표시기 (1) 및 대규모 빛나는 시스템 상태 표시기 (2). 

![왼쪽 및 오른쪽에 옅은 대규모 시스템 상태 표시기에 레이블을 지정 하는 2에 대 한 상태 표시기에 레이블을 지정 하는 1 사용 하 여 왼쪽된 상태 패널](media/fxt-monitor/fxt-control-panel-left.jpg)

이러한 솔루션을 시도한 후 오류를 여전히 있는 경우 [지원에 문의](fxt-support-ticket.md) 도움말에 대 한 합니다.

|![온도 아이콘](media/fxt-monitor/fxt-temp-icon.jpg) | 온도 상태 | 열 오류-예를 들어 팬 고장 또는 주변 온도 범위를 벗어났습니다. | 다음 주소 지정이 가능한 조건을 확인 합니다. <br>냉각 팬 누락 되었거나 실패 했습니다. <br>시스템의 표지에 무선 흐름판 메모리 모듈 비워 두거나 백 필러 대괄호 제거 <br>주변 온도 너무 높습니다. <br>외부 기류가 려 지 |
|![전기 아이콘](media/fxt-monitor/fxt-electric-icon.jpg) | 전기 상태 | 전기 오류-예를 들어 전압 범위 벗어남 PSU, 또는 실패 한 전압 regulator 실패 |  특정 문제에 대 한 시스템 메시지를 시스템 이벤트 로그를 확인 합니다. PSU 문제가 있으면 LED PSU 상태를 확인 하 고 필요한 경우에 PSU를 다시 연결 합니다. |

### <a name="system-health-status-indicator"></a>시스템 상태 상태 표시기

컨트롤 왼쪽된 패널의 오른쪽에 있는 큰 lit 단추 전체 시스템 상태를 나타내며 시스템 ID 모드로 단위 로케이터 조명으로도 사용 됩니다.

시스템 상태 및 ID 시스템 ID 모드 및 시스템 상태 모드를 전환 하는 단추를 누릅니다.

|상태 시스템 상태 | 조건 |
|-------------------------------------------|-----------------------------------------------|
| 진한 파랑 | 일반 작업: 시스템이 켜지 면 일반적으로 운영 및 시스템 ID 모드 활성 상태가 아닙니다. <br/>시스템 ID 모드로 전환 하려는 경우 시스템 상태 및 ID 단추를 누릅니다. |
| 깜박이 파란색 | 시스템 ID 모드가 활성화 됩니다. 시스템 상태 모드로 전환 하려는 경우 시스템 상태 및 시스템 ID 단추를 누릅니다. |
| 주황색 | 유사 시 대기 모드 시스템이입니다. 문제가 지속 되 면 [Microsoft 고객 서비스 및 지원에 문의](fxt-support-ticket.md)합니다. |
| 깜박거림 | 시스템 오류입니다. 특정 오류 메시지에 대 한 시스템 이벤트 로그를 확인 합니다. 시스템 펌웨어 및 시스템 구성 요소를 모니터링 하는 에이전트에서 생성 된 이벤트 및 오류 메시지에 대 한 내용은 qrl.dell.com의 오류 코드 조회 페이지를 참조 하세요. |
