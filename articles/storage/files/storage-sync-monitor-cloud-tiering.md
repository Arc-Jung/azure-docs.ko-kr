# <a name="monitor-cloud-tiering"></a>클라우드 계층화 모니터링
클라우드 계층화 정책을 모니터링 하는 방법에는 두 가지가 있습니다. 서버 끝점 속성 블레이드 및 Azure Monitor입니다.

## <a name="monitoring-via-server-endpoint"></a>서버 끝점을 통해 모니터링

[Azure Portal](https://portal.azure.com/)에 로그인 한 다음 모니터링할 서버 끝점에 대 한 **서버 끝점 속성** 블레이드로 이동 하 여 클라우드 계층화 섹션으로 스크롤합니다. 

![서버 끝점 속성의 클라우드 계층화 섹션 스크린샷](media/storage-sync-monitoring-cloud-tiering/cloud-tiering-monitoring-5.png)

**공간 절약** 은 클라우드 계층화를 사용 하도록 설정 하 여 저장 되는 공간 크기입니다. 로컬 볼륨 크기가 충분히 크기 때문에 모든 데이터를 저장할 수 있는 경우에는 0%의 공간을 절약할 수 있습니다. 0% 또는 공간 절약 비율이 낮으면 클라우드 계층화가 현재 로컬 캐시 크기와 관련 되지 않았음을 나타낼 수 있습니다. 

**Cache hit rate** 는 로컬 캐시에서 여는 파일의 백분율입니다. Cache hit rate는 캐시에서 직접 연 파일/총 파일 수로 계산 됩니다. 100%의 캐시 적중률은 지난 1 시간 동안 열었던 모든 파일이 로컬 캐시에 이미 있는 것을 의미 합니다. 수신 하는 것이 목표 인 경우 현재 정책이 제대로 작동 하 고 있음을 알 수 있습니다.

> [!NOTE]
> 임의 액세스 패턴을 포함 하는 워크 로드에는 일반적으로 캐시 적중률이 더 낮습니다. 

**총 크기 (클라우드)** 는 클라우드와 동기화 된 파일의 크기입니다. 

**캐시 된 크기 (서버)** 는 서버에 있는 파일의 총 크기 (다운로드 및 계층화)입니다. 경우에 따라 캐시 된 크기가 클라우드에서 파일의 전체 크기 보다 클 수 있습니다. 서버 볼륨의 클러스터 크기와 같은 변수를 통해이를 유발할 수 있습니다. 캐시 된 크기가 원하는 크기 보다 큰 경우 사용 가능한 볼륨 공간 정책을 늘려 보세요. 

**유효 볼륨 정책은** Azure 파일 동기화에서 서버 끝점이 있는 볼륨에 남겨 둘 여유 공간을 결정 하는 데 사용 됩니다. 동일한 볼륨에 여러 서버 끝점이 있으면 서버 끝점에서 가장 높은 사용 가능한 볼륨 공간 정책이 해당 볼륨의 모든 서버 끝점에 대 한 유효 볼륨 정책이 됩니다. 예를 들어 동일한 볼륨에 두 개의 서버 끝점이 있는 경우 하나는 사용 가능한 볼륨 공간이 30%이 고 다른 하나는 사용 가능한 볼륨 공간이 50% 이면 두 서버 끝점 모두에 대 한 유효 볼륨 사용 가능한 공간 정책은 50%가 됩니다.

**현재 사용 가능한 볼륨 공간** 은 온-프레미스 서버에서 현재 사용할 수 있는 볼륨의 사용 가능한 공간입니다. 사용 가능한 볼륨 공간 정책이 실행 되기 전에 더 많은 사용 가능한 볼륨의 사용 가능한 공간이 있는 경우에는 날짜 정책을 사용 하지 않도록 설정 하는 것이 좋습니다. 다른 문제는 현재 계층화 된 파일에서 가장 최근에 액세스 한 파일이 정책이 시작 되기 전에 남은 사용 가능한 볼륨 공간 보다 큰 것일 수 있습니다. 이 경우 로컬 볼륨 크기를 늘려야 합니다. 

## <a name="monitoring-via-azure-monitor"></a>Azure Monitor를 통한 모니터링

**저장소 동기화 서비스로** 이동 하 여 측면 탐색에서 **메트릭** 을 선택 합니다. 

클라우드 계층화에서 특정 송신을 모니터링 하는 데 사용할 수 있는 세 가지 메트릭이 있습니다.

- 클라우드 계층화 회수 크기
    - 회수 된 데이터의 총 크기 (바이트) 이며 회수 되는 데이터의 양을 식별 하는 데 사용할 수 있습니다.
- 응용 프로그램별 클라우드 계층화 회수 크기
    - 응용 프로그램에서 회수 한 데이터의 총 크기 (바이트) 이며 데이터의 회수를 식별 하는 데 사용할 수 있습니다.
- 클라우드 계층화 회수 처리량
    - 이는 데이터를 바이트 단위로 회수 하는 속도를 측정 하 고 성능에 대해 우려 하는 경우 사용 해야 합니다. 

그래프에 표시 하려는 항목에 대 한 자세한 내용은 **필터 추가** 및 **분할 적용** 을 참조 하세요.
 
Azure 파일 동기화에 대 한 다양 한 유형의 메트릭과 사용 방법에 대 한 자세한 내용은 [Monitor Azure 파일 동기화](storage-sync-files-monitoring.md)를 참조 하세요.

메트릭을 사용 하는 방법에 대 한 자세한 내용은 [Azure 메트릭 탐색기 시작](../../azure-monitor/essentials/metrics-getting-started.md)하기를 참조 하세요.

클라우드 계층화 정책을 변경 하려는 경우 [클라우드 계층화 정책 선택](storage-sync-choose-cloud-tiering-policies.md)을 참조 하세요.
