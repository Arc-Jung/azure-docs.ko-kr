# <a name="optimize-content-delivery-for-your-scenario"></a>시나리오에 대한 콘텐츠 배달 최적화

전 세계의 수많은 대상 고객에게 콘텐츠를 배달하는 경우 콘텐츠의 최적화된 배달을 보장해야 합니다. Azure CDN은 다운로드할 콘텐츠 유형이 웹 사이트인지, 라이브 스트림인지, 비디오인지, 대용량 파일인지에 따라 배달 환경을 최적화하여 눈에 띄는 성능 향상을 제공할 수 있습니다. CDN 끝점을 만들 때 "optimized for"(최적화됨) 메뉴 옵션에서 시나리오를 지정합니다. 선택하는 옵션에 따라 CDN 끝점에서 배달되는 콘텐츠에 적용될 최적화가 결정됩니다.

최적화 선택 항목은 '모범 사례' 동작을 사용하여 콘텐츠 배달 성능을 향상시키고 원본 오프로드를 개선하도록 설계되었습니다. 시나리오 선택은 부분 캐싱, 개체 청크, 원본 오류 다시 시도 정책 등에 대한 구성을 수정하여 성능에 영향을 줍니다. 

이 문서에서는 다양한 최적화 기능과 해당 기능을 사용해야 하는 경우에 대해 대략적인 개요를 제공합니다. 기능 및 제한 사항에 대한 자세한 내용은 각 개별 최적화 형식에 대한 해당 문서를 참조하세요.

> [!NOTE]
> "optimized for"(최적화됨) 메뉴 옵션은 선택한 공급자에 따라 달라질 수 있습니다. CDN 공급자는 시나리오에 따라 다양한 방법으로 향상된 기능을 적용합니다. 

> * Akamai의 Azure CDN은 일반 웹 배달, 일반 미디어 스트리밍, 주문형 비디오 미디어 스트리밍 및 대용량 파일 다운로드를 지원합니다. 

> * Verizon의 Azure CDN은 일반 웹 배달만 지원하며, 최적화 형식을 선택하지 않고도 주문형 비디오 및 대용량 파일 다운로드에서 사용할 수 있습니다.

> * 다양한 공급자 간의 성능 차이를 테스트하여 배달에 최적인 공급자를 선택하는 것이 가장 좋습니다.

## <a name="optimization-for-specific-scenarios"></a>특정 시나리오에 대한 최적화

다음 시나리오 중 하나에 대해 CDN 끝점을 최적화할 수 있습니다.  

1.  일반 웹 배달
2.  일반 미디어 스트리밍
3.  주문형 비디오 미디어 스트리밍
4.  대용량 파일 다운로드

4가지 시나리오 최적화 구성 형식은 아래에서 설명합니다. 

### <a name="general-web-delivery-optimization"></a>일반 웹 배달 최적화

일반 웹 배달은 가장 일반적인 옵션입니다. 웹 페이지 및 웹 응용 프로그램과 같은 일반 웹 콘텐츠 최적화를 위해 설계되었습니다. 이 최적화는 파일 및 비디오 다운로드에도 사용할 수 있습니다.

일반적인 웹 사이트에는 일반적으로 정적 및 동적 콘텐츠가 포함됩니다. 정적 콘텐츠로는 캐시하고 다른 사용자에게 전달할 수 있는 이미지, JavaScript 라이브러리 및 스타일시트가 있습니다. 동적 콘텐츠는 사용자 프로필에 맞게 조정된 뉴스 항목과 같이 개별 사용자에 맞게 개인 설정됩니다. 동적 콘텐츠는 쇼핑 카트 콘텐츠와 같이 각 사용자에게 고유하기 때문에 캐시할 필요가 없습니다. 일반 웹 배달은 전체 웹 사이트를 최적화할 수 있습니다.  

> [!NOTE]
> 이 최적화를 사용하는 경우

> Akamai의 Azure CDN을 사용하는 경우 평균 파일 크기가 10MB보다 작으면 이 최적화를 사용할 수 있습니다. 평균 파일 크기가 10MB보다 큰 경우 "optimized for: large file download"(최적화됨: 대용량 파일 다운로드)를 사용하세요.

### <a name="general-media-streaming-optimization"></a>일반 미디어 스트리밍 최적화

일반 미디어 스트리밍은 라이브 스트리밍과 주문형 비디오 스트리밍 둘 다에 끝점을 사용해야 하는 경우 권장됩니다.

예를 들어 비디오 콘텐츠의 잦은 버퍼링과 같이 클라이언트에 늦게 도착하는 패킷으로 인해 시청 환경이 저하될 수 있으므로 미디어 스트리밍은 시간이 중요합니다. 미디어 스트리밍 최적화를 사용하면 미디어 콘텐츠 배달의 대기 시간을 줄이고 최종 사용자에게 부드러운 스트리밍 환경을 제공할 수 있습니다. 

이 시나리오는 Azure Media Service 고객에게 일반적입니다. Azure Media Services를 사용하면 하나의 스트리밍 끝점을 라이브와 주문형 스트리밍 둘 다에 사용할 수 있습니다. 이 시나리오에서 고객은 라이브에서 주문형 스트리밍으로 변경할 때 다른 끝점으로 전환할 필요가 없습니다. 일반 미디어 스트리밍 최적화는 이 유형의 시나리오를 지원합니다.

Verizon의 Azure CDN에서는 일반 웹 배달 최적화 형식을 사용하여 스트리밍 미디어 콘텐츠를 배달할 수 있습니다.

[여기](cdn-media-streaming-optimization.md)에서 미디어 스트리밍 최적화에 대해 자세히 알아보세요.

### <a name="video-on-demand-streaming-optimization"></a>주문형 비디오 스트리밍 최적화

VOD(주문형 비디오) 미디어 스트리밍 최적화를 사용하면 VOD 스트리밍 콘텐츠가 향상됩니다. VOD 스트리밍에 끝점을 사용하는 경우 이 옵션을 사용할 수 있습니다.

Verizon의 Azure CDN에서는 일반 웹 배달 최적화 형식을 사용하여 스트리밍 미디어 콘텐츠를 배달할 수 있습니다.

[여기](cdn-media-streaming-optimization.md)에서 미디어 스트리밍 최적화에 대해 자세히 알아보세요.

> [!NOTE]
> 이 최적화를 사용하는 경우

> 끝점이 주로 VOD 콘텐츠를 제공하는 경우에 사용합니다. 이 최적화와 일반 미디어 스트리밍 최적화의 주요 차이점은 라이브 스트리밍 시나리오를 사용할 때보다 연결 다시 시도 시간 제한이 훨씬 더 짧아진다는 점입니다.

### <a name="large-file-download-optimization"></a>대용량 파일 다운로드 최적화

Akamai의 Azure CDN에서 1.8GB보다 큰 파일을 배달하려면 이 최적화를 사용해야 합니다.  Verizon의 Azure CDN에서는 일반 웹 배달 최적화의 파일 다운로드 크기에 대한 제한이 없습니다.

Akamai의 Azure CDN을 사용할 경우 대용량 파일 다운로드는 10MB보다 큰 콘텐츠에 대해 최적화됩니다. 평균 파일 크기가 10MB보다 작은 경우 일반 웹 배달을 사용할 수 있습니다. 평균 파일 크기가 일관되게 10MB보다 큰 경우(예: 펌웨어 또는 소프트웨어 업데이트) 큰 파일에 대해 별도의 끝점을 만드는 것이 더 효과적일 수 있습니다. 

Verizon의 Azure CDN에서는 일반 웹 배달 최적화 형식을 사용하여 스트리밍 미디어 콘텐츠를 배달할 수 있습니다.

[여기](cdn-large-file-optimization.md)에서 대용량 파일 최적화에 대해 자세히 알아보세요.
