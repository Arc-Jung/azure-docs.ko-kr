# <a name="azure-cdn-from-microsoft-standard-rules-engine-actions"></a>Microsoft 표준 규칙 엔진 작업에서 Azure CDN

이 문서에는 Microsoft [표준 규칙 엔진](cdn-standard-rules-engine.md)에서 제공 하는 Azure CONTENT DELIVERY NETWORK (CDN) 작업에 대 한 자세한 설명이 나와 있습니다.

규칙의 두 번째 부분은 동작입니다. 작업은 일치 조건 집합으로 식별 되는 요청 유형에 적용 되는 동작을 정의 합니다.

## <a name="actions"></a>작업

다음 작업을 사용할 수 있습니다. 

## <a name="cache-expiration"></a>캐시 만료

이 작업을 통해 규칙 일치 조건에 지정 된 요청에 대해 끝점의 TTL을 덮어쓸 수 있습니다.

**필수 필드**

캐시 동작 |                
---------------|----------------
바이패스 캐시 | 이 옵션을 선택 하 고 규칙을 일치 시킬 경우 콘텐츠가 캐시 되지 않습니다.
덮어쓸 | 이 옵션을 선택 하 고 규칙을 일치 시킬 경우 원본에서 반환 된 TTL 값을 작업에 지정 된 값으로 덮어씁니다.
누락 된 경우 설정 | 이 옵션을 선택 하 고 규칙을 일치 시킬 때 원본에서 반환 된 TTL 값이 없는 경우 규칙은 작업에 지정 된 값으로 TTL을 설정 합니다.

**추가 필드**

일 | 시간 | 분 | 초
-----|-------|---------|--------
int | int | int | int 

## <a name="cache-key-query-string"></a>캐시 키 쿼리 문자열

이 동작을 사용 하면 쿼리 문자열에 따라 캐시 키를 수정할 수 있습니다.

**필수 필드**

동작 | 설명
---------|------------
포함 | 이 옵션을 선택 하 고 규칙이 일치 하면 캐시 키를 생성할 때 매개 변수에 지정 된 쿼리 문자열이 포함 됩니다. 
모든 고유한 URL 캐시 | 이 옵션을 선택 하 고 규칙을 일치 시킬 경우 각 고유 URL에는 자체 캐시 키가 있습니다. 
제외 | 이 옵션을 선택 하 고 규칙이 일치 하면 캐시 키를 생성할 때 매개 변수에 지정 된 쿼리 문자열이 제외 됩니다.
쿼리 문자열 무시 | 이 옵션을 선택 하 고 규칙을 일치 시킬 경우 캐시 키를 생성할 때 쿼리 문자열이 고려 되지 않습니다. 

## <a name="modify-request-header"></a>요청 헤더 수정

이 작업을 통해 원본으로 전송 되는 요청에 있는 헤더를 수정할 수 있습니다.

**필수 필드**

액션(Action) | HTTP 헤더 이름 | 값
-------|------------------|------
추가 | 이 옵션을 선택 하 고 규칙을 일치 시킬 경우 헤더 이름에 지정 된 헤더가 지정 된 값을 가진 요청에 추가 됩니다. 헤더가 이미 있으면 값이 기존 값에 추가 됩니다. | 문자열
덮어쓰기 | 이 옵션을 선택 하 고 규칙을 일치 시킬 경우 헤더 이름에 지정 된 헤더가 지정 된 값을 가진 요청에 추가 됩니다. 헤더가 이미 있으면 값이 기존 값을 덮어씁니다. | 문자열
삭제 | 이 옵션을 선택 하 고 규칙을 일치 시키고 규칙에 지정 된 헤더가 있으면 요청에서 삭제 됩니다. | 문자열

## <a name="modify-response-header"></a>응답 헤더 수정

이 작업을 통해 최종 클라이언트에 반환 되는 응답에 있는 헤더를 수정할 수 있습니다.

**필수 필드**

액션(Action) | HTTP 헤더 이름 | 값
-------|------------------|------
추가 | 이 옵션을 선택 하 고 규칙을 일치 시킬 경우 헤더 이름에 지정 된 헤더가 지정 된 값을 가진 응답에 추가 됩니다. 헤더가 이미 있으면 값이 기존 값에 추가 됩니다. | 문자열
덮어쓰기 | 이 옵션을 선택 하 고 규칙을 일치 시킬 경우 헤더 이름에 지정 된 헤더가 지정 된 값을 가진 응답에 추가 됩니다. 헤더가 이미 있으면 값이 기존 값을 덮어씁니다. | 문자열
삭제 | 이 옵션을 선택 하 고 규칙을 일치 시키고 규칙에 지정 된 헤더가 있으면 응답에서 삭제 됩니다. | 문자열

## <a name="url-redirect"></a>URL 리디렉션

이 작업을 통해 최종 클라이언트를 새 URL로 리디렉션할 수 있습니다. 

**필수 필드**

필드 | 설명 
------|------------
형식 | 요청자에 게 반환할 응답 형식을 선택 합니다. 옵션은-302 Found, 301 이동함, 307 임시 리디렉션 및 308 영구 리디렉션을 제공 합니다.
프로토콜 | 요청, HTTP 또는 HTTPS 일치
호스트 이름 | 요청을 리디렉션할 호스트 이름을 선택 합니다. 들어오는 호스트를 유지 하려면 비워 두세요.
path | 리디렉션에 사용할 경로를 정의 합니다. 들어오는 경로를 유지 하려면 비워 두세요.  
쿼리 문자열 | 리디렉션에 사용 되는 쿼리 문자열을 정의 합니다. 들어오는 쿼리 문자열을 유지 하려면 비워 두세요. 
조각 | 리디렉션에 사용할 조각을 정의 합니다. 들어오는 조각을 유지 하려면 비워 두세요. 

절대 URL을 사용하는 것이 좋습니다. 상대 URL을 사용하면 CDN URL을 잘못된 경로로 리디렉션할 수 있습니다. 

## <a name="url-rewrite"></a>URL 다시 쓰기

이 작업을 사용 하면 원본으로 라우팅하는 요청의 경로를 다시 작성할 수 있습니다.

**필수 필드**

필드 | 설명 
------|------------
원본 패턴 | 대체할 URL 경로에서 원본 패턴을 정의 합니다. 현재 소스 패턴은 접두사 기반 일치를 사용 합니다. 모든 URL 경로를 일치 시키려면 소스 패턴 값으로 "/"를 사용 합니다.
대상 | 재작성에 사용할 대상 경로를 정의 합니다. 원본 패턴을 덮어씁니다.
일치 하지 않는 경로 유지 | 그렇다면 원본 패턴 뒤의 나머지 경로는 새 대상 경로에 추가 됩니다.
