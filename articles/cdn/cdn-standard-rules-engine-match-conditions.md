# <a name="azure-cdn-from-microsoft-standard-rules-engine-match-conditions"></a>Microsoft 표준 규칙 엔진 일치 조건의 Azure CDN

이 문서에서는 Microsoft [표준 규칙 엔진](cdn-standard-rules-engine.md)의 CDN (Azure Content Delivery Network)에 대해 사용 가능한 일치 조건에 대 한 자세한 설명을 제공 합니다.

규칙의 첫 번째 부분은 일치 조건 집합입니다. 각 규칙에는 일치 조건이 최대 4 개 있을 수 있습니다. 일치 조건은 규칙에 정의 된 작업이 수행 되는 특정 유형의 요청을 식별 합니다. 여러 일치 조건을 사용 하는 경우 및 논리를 사용 하 여 함께 그룹화 됩니다.

예를 들어 다음을 수행하는 데 일치 조건을 사용할 수 있습니다.

- 특정 IP 주소 또는 국가/지역에서 생성 된 필터 요청
- 헤더 정보에 의한 필터 요청
- 모바일 또는 데스크톱 장치에서 요청을 필터링 합니다.

## <a name="match-conditions"></a>일치 조건

다음 일치 조건을 사용할 수 있습니다. 

### <a name="device-type"></a>디바이스 유형 

장치 유형 일치 조건은 해당 속성에 따라 모바일 또는 데스크톱 장치에서 수행 된 요청을 식별 합니다.  

**필수 필드**

연산자 | 지원 되는 값
---------|----------------
Equals, 같지 않음 | 모바일, 데스크톱


### <a name="http-version"></a>HTTP 버전

HTTP 버전 일치 조건은 요청이 도착 하는 HTTP 버전에 따라 요청을 식별 합니다.

**필수 필드**

연산자 | 지원 되는 값
---------|----------------
Equals, 같지 않음 | 2.0, 1.1, 1.0, 0.9, 모두


### <a name="request-cookies"></a>요청 쿠키

요청 쿠키 일치 조건은 들어오는 요청의 쿠키 정보를 기반으로 하는 요청을 식별 합니다.

**필수 필드**

쿠키 이름 | 연산자 | 쿠키 값 | 사례 변환
------------|----------|--------------|---------------
문자열 | [표준 연산자 목록](#standard-operator-list) | 문자열, Int | 변환 없음, 대문자, 소문자

주요 정보
- 쿠키 이름을 지정할 때 별표 (*)를 포함 한 와일드 카드 값은 지원 되지 않습니다. 정확히 일치 하는 쿠키 이름만 비교할 수 있습니다.
- 이 일치 조건의 인스턴스당 단일 쿠키 이름만 지정할 수 있습니다.
- 쿠키 이름 비교는 대/소문자를 구분하지 않습니다.
- 각각을 하나의 공백으로 구분하여 여러 쿠키 값을 지정합니다. 
- 쿠키 값은 와일드 카드 값을 사용할 수 있습니다.
- 와일드 카드 값을 지정 하지 않은 경우 정확히 일치 하는 항목만이 일치 조건을 만족 합니다. 예를 들어 "Value"를 지정 하면 "Value"와 일치 하지만 "Value1"과는 일치 하지 않습니다. 

### <a name="post-argument"></a>Post 인수

**필수 필드**

인수 이름 | 연산자 | 인수 값 | 사례 변환
--------------|----------|----------------|---------------
문자열 | [표준 연산자 목록](#standard-operator-list) | 문자열, Int | 변환 없음, 대문자, 소문자

### <a name="query-string"></a>쿼리 문자열

쿼리 문자열 일치 조건은 지정 된 쿼리 문자열 매개 변수를 포함 하는 요청을 식별 합니다. 이 매개 변수는 지정된 패턴과 일치하는 값으로 설정됩니다. 요청 URL의 쿼리 문자열 매개 변수(예: parameter=value)가 이 조건이 충족되는지 여부를 결정합니다. 이 일치 조건은 쿼리 문자열 매개 변수를 이름으로 식별하고 매개 변수 값에 하나 이상의 값을 허용합니다.

**필수 필드**

연산자 | 쿼리 문자열 | 사례 변환
---------|--------------|---------------
[표준 연산자 목록](#standard-operator-list) | 문자열, Int | 변환 없음, 대문자, 소문자

### <a name="remote-address"></a>원격 주소

원격 주소 일치 조건은 요청자의 위치 또는 IP 주소에 따라 요청을 식별 합니다.

**필수 필드**

연산자 | 지원 되는 값
---------|-----------------
모두 | 해당 없음
지리적 일치 | 국가 코드
IP 일치 | IP 주소 (공백으로 구분)
모두 아님 | 해당 없음
지리적으로 일치 하지 않음 | 국가 코드
IP 일치 하지 않음 | IP 주소 (공백으로 구분)

주요 정보:

- CIDR 표기법을 사용합니다.
- 각각을 단일 공백으로 구분하여 여러 IP 주소 및/또는 IP 주소 블록을 지정합니다. 예:
  - **IPv4 예제**: 1.2.3.4 10.20.30.40은 1.2.3.4 또는 10.20.30.40 주소에서 도착하는 요청과 일치합니다.
  - **IPv6 예제**: 1:2:3:4:5:6:7:8 10:20:30:40:50:60:70:80은 1:2:3:4:5:6:7:8 또는 10:20:30:40:50:60:70:80 주소에서 도착하는 요청과 일치합니다.
- IP 주소 블록에 대한 구문은 뒤에 슬래시와 접두사 크기가 오는 기본 IP 주소입니다. 예:
  - **IPv4 예제**: 5.5.5.64/26은 5.5.5.64에서 5.5.5.127까지의 주소에서 도착하는 요청과 일치합니다.
  - **IPv6 예제**: 1:2:3:/48은 1:2:3:0:0:0:0:0에서 1:2:3:ffff:ffff:ffff:ffff:ffff까지의 주소에서 도착한 요청과 일치합니다.

### <a name="request-body"></a>요청 본문

**필수 필드**

연산자 | 요청 본문 | 사례 변환
---------|--------------|---------------
[표준 연산자 목록](#standard-operator-list) | 문자열, Int | 변환 없음, 대문자, 소문자

### <a name="request-header"></a>요청 헤더

**필수 필드**
헤더 이름 | 연산자 | 헤더 값 | 사례 변환
------------|----------|--------------|---------------
문자열 | [표준 연산자 목록](#standard-operator-list) | 문자열, Int | 변환 없음, 대문자, 소문자

### <a name="request-method"></a>요청 메서드

**필수 필드**

연산자 | 지원 되는 값
---------|----------------
Equals, 같지 않음 | GET, POST, PUT, DELETE, HEAD, OPTIONS, TRACE

주요 정보:

- GET 요청 메서드만 CDN에서 캐시 된 콘텐츠를 생성할 수 있습니다. 다른 모든 요청 메서드는 네트워크를 통해 프록시 설정됩니다. 

### <a name="request-protocol"></a>요청 프로토콜

**필수 필드**

연산자 | 지원 되는 값
---------|----------------
Equals, 같지 않음 | HTTP, HTTPS

### <a name="request-url"></a>요청 URL

**필수 필드**

연산자 | 요청 URL | 사례 변환
---------|-------------|---------------
[표준 연산자 목록](#standard-operator-list) | 문자열, Int | 변환 없음, 대문자, 소문자

주요 정보:

- 요청 URL을 입력할 때 프로토콜 정보를 포함 하는지 확인 합니다. 예: "https://www. [yourdomain] .com "

### <a name="url-file-extension"></a>URL 파일 확장명

**필수 필드**

연산자 | 내선 번호 | 사례 변환
---------|-----------|---------------
[표준 연산자 목록](#standard-operator-list) | 문자열, Int | 변환 없음, 대문자, 소문자

주요 정보:

- 확장의 경우 앞에 마침표를 포함 하지 마십시오. 예를 들어 .html 대신 html을 사용 합니다.

### <a name="url-file-name"></a>URL 파일 이름

**필수 필드**

연산자 | 파일 이름 | 사례 변환
---------|-----------|---------------
[표준 연산자 목록](#standard-operator-list) | 문자열, Int | 변환 없음, 대문자, 소문자

주요 정보:

- 여러 파일 이름을 지정 하려면 각 파일 이름을 하나의 공백으로 구분 합니다. 

### <a name="url-path"></a>URL 경로

**필수 필드**

연산자 | 값 | 사례 변환
---------|-------|---------------
[표준 연산자 목록](#standard-operator-list) | 문자열, Int | 변환 없음, 대문자, 소문자

주요 정보:

- 파일 이름 값은 와일드 카드 값을 사용할 수 있습니다. 예를 들어 각각의 파일 이름 패턴은 하나 이상의 별표(*)로 구성될 수 있으며 각 별표는 하나 이상의 문자 시퀀스와 일치합니다.

## <a name="reference-for-rules-engine-match-conditions"></a>규칙 엔진 일치 조건에 대한 참조

### <a name="standard-operator-list"></a>표준 연산자 목록

표준 연산자 목록을 포함 하는 규칙의 경우 다음 연산자를 사용할 수 있습니다.

- 모두
- equals 
- contains 
- 시작 문자 
- 다음으로 끝 
- 다음보다 적음
- 작거나 같음
- 다음보다 큼
- 크거나 같음
- 모두 아님
- 포함 안 함
- 다음으로 시작 안 함 
- 다음으로 끝나지 않음 
- 보다 작지 않음
- 작거나 같음
- 보다 크지 않음
- 작거나 같음

"보다 작음" 또는 "보다 크거나 같음"과 같은 숫자 연산자의 경우 사용 되는 비교는 길이를 기준으로 합니다. 이 경우 일치 조건의 값은 비교 하려는 길이와 같은 정수 여야 합니다.
