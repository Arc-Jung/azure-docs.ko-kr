# Microsoft Azure CDN의 실시간 통계

## 개요

이 문서에서는 Microsoft Azure CDN의 실시간 통계에 대해 설명합니다. 이 기능은 클라이언트에 콘텐츠를 제공할 때 CDN의 성능에 대한 실시간 데이터를 제공합니다.

> [AZURE.NOTE] 실시간 통계는 프리미엄 CDN 계층의 기능입니다. 표준 및 프리미엄 CDN 기능을 비교하려면 [Azure CDN 개요](cdn-overview.md)를 참조하세요.

HTTP 기반 플랫폼에 대한 실시간 통계를 볼 때 다음과 같은 그래프가 제공됩니다.

* [대역폭](#bandwidth)
* [상태 코드](#status-codes)
* [캐시 상태](#cache-statuses)
* [연결](#connections)

> [AZURE.NOTE] 위의 각 그래프는 지정된 기간에 대한 실시간 통계를 표시합니다. 지정된 시간이 경과되면 데이터의 슬라이딩 윈도우가 표시됩니다. 즉, 새 데이터에 대한 공간을 만들기 위해 이전 데이터가 그래프에서 제거됩니다. 그래프의 기간 옵션으로 이 슬라이딩 윈도우에 대한 기간을 설정할 수 있습니다.

## 실시간 통계에 액세스

1. CDN 프로필 블레이드에서 **관리** 단추를 클릭합니다.

	![CDN 프로필 블레이드 관리 단추](./media/cdn-real-time-stats/cdn-manage-btn.png)

	CDN 관리 포털이 열립니다.

2. **분석** 탭을 마우스로 가리킨 후 **실시간 통계** 플라이아웃을 마우스로 가리킵니다. **HTTP 큰 플랫폼**을 클릭합니다.

	보고서 옵션이 표시됩니다.

## 대역폭

대역폭 그래프는 지정된 기간 동안 현재 플랫폼에 사용된 대역폭의 양을 표시합니다. 그래프의 음영 처리된 부분은 대역폭 사용량을 나타냅니다. 현재 사용 중인 대역폭의 정확한 양이 꺾은선형 그래프 바로 아래 표시됩니다.

> [AZURE.NOTE] 보고서 대역폭 사용량에 사용된 단위는 b/s(비트/초), Kb/s(킬로비트/초), Mb/s(메가비트/초) 또는 Gb/s(기가비트/초)입니다.

## 상태 코드

상태 코드 그래프는 지정된 기간 동안 HTTP 응답 코드가 발생한 빈도를 나타내는 색이 지정된 선으로 구성됩니다. 그래프 왼쪽(Y축)은 요청에 대해 상태 코드가 반환되는 빈도를 나타내고 그래프 아래쪽(X축)은 시간 진행을 나타냅니다.

상태 코드의 목록은 그래프 바로 위에 표시됩니다. 이 목록은 꺾은선형 그래프에 포함할 수 있는 각 상태 코드와 해당 상태 코드에 대한 현재 초당 발생 수를 나타냅니다. 기본적으로 선은 그래프에서 이 상태 코드 각각에 대해 표시됩니다. 그러나 CDN 구성에 대해 특별한 의미가 있는 상태 코드만 모니터하도록 선택할 수 있습니다. 원하는 상태 코드 옵션만 표시하고 다른 모든 옵션은 지워서 이 작업을 수행할 수 있습니다. 그래프에 표시될 상태 코드에 만족하면 그래프 새로 고침을 클릭해야 합니다. 이렇게 하면 지워진 상태 코드가 그래프에 포함되지 않도록 할 수 있습니다.

> [AZURE.NOTE] **그래프 새로 고침** 옵션을 사용하면 그래프가 지워집니다. 그런 다음 선택한 상태 코드만 표시됩니다.

각 상태 코드 옵션은 아래에서 설명합니다.

이름 | 설명
-----|------------
초당 총 적중 수 | 현재 플랫폼에 대한 초당 총 요청 수를 그래프에 표시할지 여부를 결정합니다. 이 옵션을 기본 표시기로 사용하면 특정 상태 코드에 의해 구성되는 총 적중 수의 백분율을 확인할 수 있습니다.
초당 2xx | 현재 플랫폼에서 초당 발생하는 2xx 상태 코드(예: 200, 201, 202 등)의 총 수를 그래프에 표시할지 여부를 결정합니다. 이러한 유형의 상태 코드는 클라이언트에 요청이 성공적으로 전달되었음을 나타냅니다.
초당 304 | 현재 플랫폼에서 초당 발생하는 304 상태 코드의 총 수를 그래프에 표시할지 여부를 결정합니다. 이 상태 코드는 요청된 자산이 HTTP 클라이언트에서 마지막으로 검색된 이후 수정되지 않았음을 나타냅니다.
초당 3xx | 현재 플랫폼에서 초당 발생하는 3xx 상태 코드(예: 300, 301, 302 등)의 총 수를 그래프에 표시할지 여부를 결정합니다. 이 옵션은 304 상태 코드 항목을 제외합니다. 이러한 유형의 상태 코드는 리디렉션이 발생하는 요청을 나타냅니다.
초당 403 | 현재 플랫폼에서 초당 발생하는 403 상태 코드의 총 수를 그래프에 표시할지 여부를 결정합니다. 이 상태 코드는 요청이 승인되지 않은 것으로 간주되었음을 나타냅니다. 이 상태 코드에 대해 가능한 원인 중 하나는 무단 사용자가 토큰 기반 인증으로 보호되는 자산을 요청하는 경우입니다.
초당 404 | 현재 플랫폼에서 초당 발생하는 404 상태 코드의 총 수를 그래프에 표시할지 여부를 결정합니다. 이 상태 코드는 요청된 자산을 찾을 수 없음을 나타냅니다.
초당 4xx | 현재 플랫폼에서 초당 발생하는 4xx 상태 코드(예: 400, 401, 402, 405 등)의 총 수를 그래프에 표시할지 여부를 결정합니다. 이 옵션은 403 및 404 상태 코드 항목을 제외합니다. 이 상태 코드는 요청된 자산이 클라이언트에 전달되지 않았음을 나타냅니다.
초당 5xx | 현재 플랫폼에서 초당 발생하는 5xx 상태 코드(예: 500, 501, 502 등)의 총 수를 그래프에 표시할지 여부를 결정합니다.
초당 기타 | 기타 모든 상태 코드에 대한 총 발생 수를 그래프에 보고할지 여부를 결정합니다.

또한 특정 상태 코드에 대해 기록된 데이터를 임시로 숨기도록 선택할 수도 있습니다. 원하는 상태 코드 옵션을 지워서 그래프 바로 아래 영역에서 이 작업을 수행할 수 있습니다. 선택한 상태 코드는 그래프에서 즉시 숨겨집니다. 해당 상태 코드 옵션을 표시하면 해당 옵션이 다시 표시됩니다.

> [AZURE.NOTE] 그래프 바로 아래 색이 지정된 옵션은 그래프에 표시된 항목에만 영향을 줍니다. 그래프가 해당 상태 코드를 계속 추적할지 여부에는 영향을 주지 않습니다.

## 캐시 상태

캐시 상태 그래프는 색이 지정된 선으로 구성되며 여기서 선은 특정 캐시 상태 유형이 지정된 기간에 발생하는 빈도를 나타냅니다. 그래프 왼쪽(Y축)은 요청에 대해 캐시 상태가 반환되는 빈도를 나타내고 그래프 아래쪽(X축)은 시간 진행을 나타냅니다.

캐시 상태 목록이 그래프 바로 위에 표시됩니다. 이 목록은 꺾은선형 그래프에 포함할 수 있는 각 캐시 상태와 해당 캐시 상태에 대한 현재 초당 발생 수를 나타냅니다. 기본적으로 선은 그래프에서 이 캐시 상태 각각에 대해 표시됩니다. 그러나 CDN 구성에 대해 특별한 의미가 있는 캐시 상태만 모니터하도록 선택할 수 있습니다. 원하는 캐시 상태 옵션만 표시하고 다른 모든 옵션은 지워서 이 작업을 수행할 수 있습니다. 그래프에 표시될 캐시 상태에 만족하면 그래프 새로 고침을 클릭해야 합니다. 이렇게 하면 지워진 상태 코드가 그래프에 포함되지 않도록 할 수 있습니다.

> [AZURE.NOTE] **그래프 새로 고침** 옵션을 사용하면 그래프가 지워집니다. 그런 다음 선택한 캐시 상태만 표시됩니다.

또한 특정 응답 코드에 대해 기록된 데이터를 임시로 숨기도록 선택할 수도 있습니다. 그래프 바로 아래 영역에서 원하는 응답 코드 옵션을 지워서 이 작업을 수행할 수 있습니다. 선택한 응답 코드는 그래프에서 즉시 숨겨집니다. 해당 응답 코드 옵션을 표시하면 해당 옵션이 다시 표시됩니다.

> [AZURE.NOTE] 그래프 바로 아래 색이 지정된 옵션은 그래프에 표시된 항목에만 영향을 줍니다. 그래프가 해당 상태 코드를 계속 추적할지 여부에는 영향을 주지 않습니다.

## 연결

분당 평균 연결 수에 대한 그래픽 표현을 통해 서버에 얼마나 많은 연결이 설정되었는지 확인할 수 있습니다. 연결은 CDN을 통과하는 자산에 대한 각 요청으로 구성됩니다.
