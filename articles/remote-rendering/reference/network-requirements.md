# <a name="network-requirements"></a>네트워크 요구 사항

Azure 원격 렌더링에서 좋은 사용자 환경을 위해 Azure 데이터 센터에 대한 안정적이고 대기 시간이 짧은 네트워크 연결은 매우 중요합니다. 네트워크 조건이 불량하면 서버 측 장면 그래프를 업데이트할 때 연결이 끊어지거나 불안정하거나 과민하거나 '점프' 홀로그램이 발생할 수 있으며 지연이 발생할 수 있습니다.

## <a name="guidelines-for-network-connectivity"></a>네트워크 연결 에 대한 지침

정확한 네트워크 요구 사항은 원격 장면 그래프의 수정 횟수 및 빈도, 렌더링된 뷰의 복잡성 등 특정 사용 사례에 따라 다르지만, 가능한 한 좋은 환경을 보장하기 위한 여러 가지 지침이 있습니다.

* 네트워크에 경쟁 트래픽이 없다고 가정할 때 인터넷 연결은 Azure 원격 렌더링의 단일 사용자 세션에 대해 최소 **50Mbps 다운스트림** 및 **10Mbps 업스트림을** 일관되게 지원해야 합니다. 더 나은 경험을 위해 더 높은 요금을 권장합니다. 동일한 네트워크에 있는 사용자가 많을수록 이러한 요구 사항은 그에 따라 확장됩니다.
* 5 **GHz Wi-Fi 대역을** 사용하면 일반적으로 2.4 GHz Wi-Fi 대역보다 더 나은 결과를 얻을 수 있지만 둘 다 작동해야합니다.
* 근처에 다른 Wi-Fi 네트워크가 있는 경우 이러한 다른 네트워크에서 사용할 Wi-Fi 채널을 사용하지 마십시오. [WifiInfoView와](https://www.nirsoft.net/utils/wifi_information_view.html) 같은 네트워크 검색 도구를 사용하여 Wi-Fi 네트워크가 사용하는 채널이 경쟁 트래픽이 없는지 확인할 수 있습니다.
* **Wi-Fi 리피터** 또는 LAN 오버 파워라인 포워딩을 사용하지 마십시오.
* 동일한 Wi-Fi 네트워크에서 비디오 또는 게임 스트리밍과 같이 **대역폭이 치열한 트래픽이 경쟁하지 않도록** 합니다.
* **좋은 Wi-Fi 신호 강도를** 갖는 것은 필수적이다. 가능하면 Wi-Fi 액세스 포인트 가까이에 머무르고 클라이언트 장치와 액세스 포인트 사이의 장애물을 피하십시오.
* [해당 지역의](regions.md)가장 가까운 **Azure 데이터 센터에** 항상 연결해야 합니다. 데이터 센터가 가까울수록 네트워크 대기 시간이 낮아지므로 홀로그램 안정성에 큰 영향을 미칩니다.

## <a name="network-performance-tests"></a>네트워크 성능 테스트

네트워크 연결 의 품질이 Azure 원격 렌더링을 실행하기에 충분한지 여부를 초기에 이해하려는 경우 사용할 수 있는 기존 온라인 도구가 있습니다. Azure 원격 렌더링 클라이언트 응용 프로그램을 실행하려는 장치와 동일한 Wi-Fi에 연결된 합리적으로 강력한 랩톱에서 이러한 온라인 도구를 실행하는 것이 좋습니다. 휴대폰 또는 HoloLens2에서 테스트를 실행하여 얻은 결과는 일반적으로 저전력 엔드포인트 장치에서 상당한 변동을 보이는 것으로 입증되어 유용하지 않습니다. 랩톱을 배치하는 위치는 Azure 원격 렌더링 클라이언트 응용 프로그램을 실행하는 장치를 사용할 것으로 예상되는 위치와 대략 같아야 합니다.

다음은 네트워크 연결을 빠르게 테스트하기 위한 몇 가지 간단한 단계입니다.

1. **www.speedtest.net 같은 네트워크 테스트 도구를 실행하여 네트워크 연결의 전체 대기 시간 및 업스트림/다운스트림 대역폭에 대한 데이터를 가져옵니다.**
가장 가까운 서버를 선택하고 테스트를 실행합니다. 서버가 Azure 원격 렌더링에 연결되는 Azure 데이터 센터는 아니지만 결과 데이터는 인터넷 연결 및 Wi-Fi의 성능을 이해하는 데 여전히 유용합니다.
   * Azure 원격 렌더링에 대한 **최소 요구 사항:** 약 40Mbps 다운스트림 및 5Mbps 업스트림.
   * Azure 원격 렌더링에 **권장:** 약 100Mbps 다운스트림 및 10Mbps 업스트림.
테스트를 여러 번 실행하고 최악의 결과를 취하는 것이 좋습니다.
1. **Azure 데이터 센터에 대한 대기 시간을 측정하는 www.azurespeed.com 같은 도구를 사용합니다.** 가장 가까운 Azure 원격 렌더링에서 지원하는 Azure 데이터 [supported regions](regions.md)센터를 선택하고 대기 **시간 테스트를**실행합니다. 표시되는 숫자에 차이가 있는 경우 결과에 안정화할 시간을 줍니다.
   * Azure 원격 렌더링에 대한 **최소 요구 사항:** 대기 시간은 일관되게 100ms 미만이어야 합니다.
   * Azure 원격 렌더링에 **권장:** 대기 시간은 일관되게 70ms 미만이어야 합니다.

대기 시간이 짧은 것이 Azure 원격 렌더링이 네트워크에서 잘 작동한다는 보장은 아니지만 일반적으로 이러한 테스트가 성공적으로 통과된 상황에서는 성능이 우수합니다.
Azure 원격 렌더링을 실행할 때 불안정하거나 과민하거나 홀로그램 점프와 같은 아티팩트가 발생하는 경우 [문제 해결 가이드를](../resources/troubleshoot.md)참조하십시오.
