# <a name="update-a-power-bi-app-offer"></a>Power BI 앱 제품 업데이트

이 문서에서는 [클라우드 파트너 포털에서](https://cloudpartner.azure.com/) Microsoft Power BI 앱 오퍼를 업데이트한 다음 오퍼를 다시 게시하는 방법을 보여 줍니다. 다음은 오퍼를 업데이트하는 몇 가지 일반적인 이유입니다.

- Power BI에서 앱의 콘텐츠를 업데이트하고 새로 패키지된 앱에서 설치 URL을 받으려면
- 오퍼의 Azure 마켓플레이스 메타데이터(영업, 마케팅 또는 지원 정보 및 자산)를 업데이트하려면
 
포털의 **비교** 및 **기록** 기능을 사용하여 이러한 변경 내용을 추적할 수 있습니다.

## <a name="common-update-operations"></a>일반적인 업데이트 작업

Power BI 앱 제품의 일부 속성은 AppSource에 게시된 후에는 변경할 수 없습니다. 허용되지 않는 변경 사항에는 **오퍼 ID** 및 **게시자 ID가**포함됩니다. 그러나 당신은 여전히 특성의 넓은 범위를 변경할 수 있습니다. 다음은 몇 가지 일반적인 변경 사항입니다.

### <a name="update-app-content-in-power-bi"></a>Power BI에서 앱 콘텐츠 업데이트

Power BI 앱은 새 콘텐츠, 보안 패치, 기능 등을 통해 주기적으로 업데이트되는 경우가 많습니다. 앱이 변경되면 다음 단계에 따라 설치 URL을 업데이트합니다.

1.  클라우드 파트너 [포털에](https://cloudpartner.azure.com/)로그인합니다.
2.  **모든 제품** 아래에서 업데이트하려는 제품을 찾습니다.
3.  기술 **정보** 탭에서 새 설치 URL을 입력합니다.
4.  게시를 **Publish** 선택하여 워크플로를 시작하여 새 앱 버전을 AppSource에 게시합니다.


### <a name="update-the-offers-marketplace-metadata"></a>오퍼의 마켓플레이스 메타데이터 업데이트

오퍼의 마켓플레이스 메타데이터에는 회사 이름, 로고 및 기타 정보가 포함됩니다. 메타데이터를 업데이트하려면 다음 단계를 따르십시오.

1.  클라우드 파트너 [포털에](https://cloudpartner.azure.com/)로그인합니다.
2.  **모든 제품** 아래에서 업데이트하려는 제품을 찾습니다.
3.  **스토어프론트 세부 정보** 탭에서 [문서 Power BI 앱 스토어프론트 세부 정보 탭의](./cpp-storefront-details-tab.md)지침에 따라 메타데이터를 변경합니다.
4.  **게시**를 선택하여 변경 내용을 게시하는 워크플로를 시작합니다.


## <a name="the-compare-feature"></a>비교 기능

게시된 쿠폰을 업데이트할 때 변경한 내용을 확인할 수 있습니다. 비교 기능을 사용하려면 다음을 **수행하십시오.**

1.  편집 프로세스의 어느 시점에서든 오퍼의 **비교** 버튼을 선택합니다.

    ![비교 버튼](./media/compare-feature-button.png)

2.  마케팅 자산과 메타데이터의 버전이 나란히 표시됩니다.


## <a name="history-of-publishing-actions"></a>게시 작업의 기록

클라우드 파트너 포털 왼쪽에서 쿠폰의 게시 내역을 보려면 **기록** 탭을 엽니다. AppSource 오퍼에서 타임스탬프가 찍힌 작업의 기록을 볼 수 있습니다.
