# <a name="update-a-power-bi-app-offer"></a>Power BI 앱 제안 업데이트

이 문서에서는 [Cloud 파트너 포털](https://cloudpartner.azure.com/) 에서 Microsoft Power BI App 제품을 업데이트 한 다음 제품을 다시 게시 하는 방법을 보여 줍니다. 제품을 업데이트 하는 몇 가지 일반적인 이유는 다음과 같습니다.

- Power BI에서 앱 콘텐츠를 업데이트 하 고 새로 패키지 된 앱에서 설치 URL을 가져오려면
- 제품의 Azure Marketplace 메타 데이터 (판매, 마케팅 또는 지원 정보 및 자산)를 업데이트 하려면
 
포털의 **비교** 및 **기록** 기능을 사용 하 여 이와 같은 변경 내용을 추적 합니다.

## <a name="common-update-operations"></a>일반적인 업데이트 작업

제품이 AppSource에 게시 된 후에는 Power BI 앱 제품의 일부 특성을 변경할 수 없습니다. 허용 되지 않는 변경 내용에는 **제품 id** 및 **게시자 id가**있습니다. 하지만 계속 해 서 광범위 한 특성을 변경할 수 있습니다. 다음은 몇 가지 일반적인 변경 내용입니다.

### <a name="update-app-content-in-power-bi"></a>Power BI에서 앱 콘텐츠 업데이트

Power BI 앱은 종종 새로운 콘텐츠, 보안 패치, 기능 등을 사용 하 여 정기적으로 업데이트 됩니다. 앱이 변경 되 면 다음 단계에 따라 설치 URL을 업데이트 합니다.

1.  [클라우드 파트너 포털](https://cloudpartner.azure.com/)에 로그인합니다.
2.  **모든 제품** 아래에서 업데이트하려는 제품을 찾습니다.
3.  **기술 정보** 탭에서 새 설치 URL을 입력 합니다.
4.  **게시** 를 선택 하 여 새 앱 버전을 appsource에 게시 하는 워크플로를 시작 합니다.


### <a name="update-the-offers-marketplace-metadata"></a>제품의 Marketplace 메타 데이터를 업데이트 합니다.

제품의 Marketplace 메타 데이터에는 회사 이름, 로고 및 기타 정보가 포함 됩니다. 메타 데이터를 업데이트 하려면 다음 단계를 수행 합니다.

1.  [클라우드 파트너 포털](https://cloudpartner.azure.com/)에 로그인합니다.
2.  **모든 제품** 아래에서 업데이트하려는 제품을 찾습니다.
3.  **Storefront details** (세부 정보) 탭에서 [Power BI app Storefront details](./cpp-storefront-details-tab.md)(문서 정보) 탭의 지침에 따라 메타 데이터를 변경 합니다.
4.  **게시**를 선택하여 변경 내용을 게시하는 워크플로를 시작합니다.


## <a name="the-compare-feature"></a>비교 기능

게시 된 제안을 업데이트할 때 변경한 내용을 확인할 수 있습니다. **비교** 기능을 사용 하려면 다음을 수행 합니다.

1.  편집 프로세스의 어느 시점에서 든 제품의 **비교** 단추를 선택 합니다.

    ![비교 단추](./media/compare-feature-button.png)

2.  마케팅 자산과 메타데이터의 버전이 나란히 표시됩니다.


## <a name="history-of-publishing-actions"></a>게시 작업의 기록

제품의 게시 기록을 보려면 Cloud 파트너 포털 왼쪽에서 **기록** 탭을 엽니다. 여기에서 AppSource 제품에 대 한 타임 스탬프 작업의 기록이 표시 됩니다.
