# <a name="power-bi-apps-storefront-details-tab"></a>Power BI 앱 상점 세부 정보 탭

**새 제안** 페이지의 **상점 세부 정보** 탭을 사용하여 잠재 고객에게 마케팅, 영업 및 법적 정보를 제공합니다. 또한 이 탭에서는 마켓플레이스에서 창출된 잠재 고객을 관리하는 방법도 지정합니다. 이 긴 양식은 **제안 세부 정보**, **목록 세부 정보**, **마케팅 아티팩트**, **법적 정보**, **고객 지원**, **잠재 고객 관리**의 6가지 섹션으로 구분됩니다.  필드 레이블에 별표(*)가 추가되면 필수 항목임을 나타냅니다.


## <a name="offer-details-section"></a>제안 세부 정보 섹션

이 섹션에서는 AppSource 제안에 대한 일반적인 정보를 입력합니다.

![상점 세부 정보 탭의 제안 세부 정보 섹션](./media/offer-details-section.png)

다음 표에서는 이러한 필드의 이름과 용도를 설명합니다.

|   필드               |   설명                                                                           |
|-----------------------|-----------------------------------------------------------------------------------------|
| **제품 요약**     | 앱에 대한 간단한 용도입니다. 최대 길이는 100자입니다.                             |
| **제품 설명** | 앱에 대한 설명입니다. 최대 길이는 3,000자이며, 단순 HTML 형식을 지원합니다. |
|                       |                                                                                         |


## <a name="listing-details-section"></a>목록 세부 정보 섹션

이 두 번째 섹션에서는 일반적으로 사용되는 산업, 가장 적합하게 적용되는 범주, 호환되는 제품 및 관련 검색 용어와 같이 앱에 대한 추가 컨텍스트를 제공합니다.

![상점 세부 정보 탭의 목록 세부 정보 섹션](./media/listing-details-section.png)

 다음 표에서는 이러한 필드의 이름과 용도를 설명합니다.
 
|   필드                                  |   설명                                                        |
| --------------                           | ---------------------                                                |
| **산업**                           | 앱에 가장 부합되는 산업을 선택합니다. 앱이 여러 업계와 관련되는 경우 이 필드는 비워 둘 수 있습니다.      |
| **범주**                           | 앱과 관련이 있는 범주를 선택합니다. 최대 3개를 선택합니다.     |
| **앱 도움말 링크**               | 앱에 대한 온라인 도움말이 있는 페이지의 URL입니다.           |
| **앱이 작동하는 제품(최대 3개)** | 앱이 작동하는 특정 제품을 나열합니다. 제품은 3개까지 입력할 수 있습니다. 제품을 나열하려면 [새로 만들기] 옆에 있는 더하기 기호를 클릭합니다. 그러면 열린 새 텍스트 필드가 만들어져 앱이 작동하는 제품의 이름을 입력할 수 있습니다.      |
| **검색 키워드(최대 3개)**              | AppSource를 사용하면 고객이 키워드를 기반으로 검색을 수행할 수 있습니다. 고객에게 애플리케이션을 표시할 키워드 집합을 입력할 수 있습니다. 예를 들어 애플리케이션이 "내 이메일 보내기 앱"인 경우 키워드 중 일부가 이메일, 메일 보내기, 메일 앱일 수 있습니다. 사용자가 AppSource 검색 상자에서 앱을 검색하는 데 사용할 수 있는 단어를 선택합니다. |
|  |  |


## <a name="marketing-artifacts-section"></a>마케팅 아티팩트 섹션

세 번째 섹션에서는 브랜딩 및 마케팅 자료를 업로드할 수 있게 합니다.  이 섹션은 **로고**, **비디오**, **문서** 및 **스크린샷**의 네 가지 섹션으로 구분됩니다. 로고 및 스크린샷은 필수 마케팅 아티팩트이지만, 고객에게 가장 적절하게 호소하기 위해 모든 아티팩트를 사용하는 것이 좋습니다.

![상점 세부 정보 탭의 마케팅 아티팩트 섹션](./media/marketing-artifacts-section.png)

 
|    필드                             |    설명                                                    |
|   -----------                        |    -------------                                                  |
| *로고*                              |                                                                   |
| **제안 로고(png 형식, 48x48)**   | 검색이 완료되면 AppSource의 앱 개요 또는 앱 결과에 표시됩니다. 48px\*48px 해상도의 png 형식만 지원됩니다.  |
| **제안 로고(png 형식, 216x216)** | AppSource의 앱 세부 정보 페이지에 표시됩니다.  216px\*216px 해상도의 png 형식만 지원됩니다.  |
| *비디오*                             |                                                                   |
| **Name**                             | 앱의 이름 또는 제목입니다.                                          |
| **URL**                              | YouTube 또는 Vimeo에서 호스팅되는 비디오 URL입니다.                              |
| **썸네일**                        | 앱의 썸네일 이미지입니다.  1280px\*720px 해상도의 png 형식만 지원됩니다.   |
| *문서*                          | 선택 사항이며, 최대 3개의 문서를 업로드할 수 있습니다. 여기에 업로드한 문서는 AppSource의 "자세한 정보" 아래에 표시됩니다.  |
| **Name**                             | 지원하는 문서의 이름 또는 제목입니다.                              |
| **파일**                             | 업로드 문서는 pdf 형식이어야 합니다.                             |
| *스크린샷*                        | 선택 사항이며, 최대 5개의 스크린샷을 제공할 수 있습니다.                        |
| **Name**                             | 스크린샷의 이름 또는 제목입니다.                                       |
| **이미지**                            | 화면 캡처 이미지를 업로드할 수 있으며, 1280px\*720px 해상도의 png 형식이어야 합니다.  | 
|   |   |


### <a name="logo-guidelines"></a>로고 지침

[Cloud 파트너 포털](https://cloudpartner.azure.com)에 업로드되는 모든 로고는 다음 지침을 따라야 합니다.

- 로고의 배경에 그라데이션 효과를 사용하지 않습니다.
- 회사 또는 브랜드 이름이 포함된 텍스트를 로고에 배치하지 않습니다. 로고의 모양과 느낌은 "평면적"이어야 하며 그라데이션은 사용하지 않습니다.
- 로고를 늘리지 않습니다.


## <a name="legal-section"></a>법적 정보 섹션

이 네 번째 섹션에서는 각 제안에 필요한 두 가지 법률 문서, 즉 개인정보처리방침 및 사용 약관을 제공할 수 있습니다.

![상점 세부 정보 탭의 법적 정보 섹션](./media/legal-section.png)

|   필드                |   설명                           |
|------------------------|--------------------------------------   |
| **개인정보처리방침 URL** | 게시된 개인정보처리방침에 대한 URL       |
| **사용 약관**       | 일반 텍스트 또는 단순 HTML 형식으로 작성된 정책입니다.     |
|  |  |


## <a name="customer-support-section"></a>고객 지원 섹션

온라인 고객 지원 페이지에 대한 **지원 URL**을 제공합니다.  이 온라인 지원 페이지에서 고객에게 전화, 이메일 및 라이브 채팅과 같은 여러 연락처 옵션을 제공하는 것이 좋습니다. 


## <a name="lead-management-section"></a>잠재 고객 관리 섹션

마지막 섹션에서는 AppSource 제안에서 창출된 잠재 고객을 수집할 수 있습니다. 이 잠재 고객 정보에 대해 드롭다운 목록에서 제공하는 스토리지 옵션은 다음과 같습니다.

|    필드               |   잠재 대상 고객                               |
|------------------------|--------------------------------------            |
|  **없음**              | 잠재 고객이 수집되지 않습니다(기본값).  |
| **Azure Blob(사용되지 않음)** | 컨테이너 이름과 연결 문자열로 지정된 [Azure Blob](https://docs.microsoft.com/azure/storage/blobs/storage-blobs-overview)입니다.  이 선택 항목은 더 이상 사용되지 않습니다. 대신 **Azure Table**을 사용합니다.  |
| **Azure Table**        | 연결 문자열로 지정된 [Azure 테이블](https://docs.microsoft.com/azure/cosmos-db/table-storage-overview)입니다.  |
| **Dynamics CRM Online** | URL 및 인증 자격 증명으로 지정된 [Microsoft Dynamics 365 Online](https://dynamics.microsoft.com/) 인스턴스입니다. |
| **HTTPS 엔드포인트**     | JSON 페이로드로 지정된 HTTPS 엔드포인트입니다.   |
| **Marketo**            | 서버 ID, munchkin ID 및 양식 ID로 지정된 [Marketo](https://www.marketo.com/) 인스턴스입니다.   |
| **Salesforce**         | 개체 식별자로 지정된 [Salesforce](https://www.salesforce.com/) 데이터베이스입니다. |
|  |  |

제안이 게시되면 잠재 고객 연결의 유효성을 검사하고 테스트 잠재 고객을 지정된 대상으로 자동으로 보냅니다. 잠재 고객 정보는 지속적으로 관리해야 하며, 이러한 설정은 고객 관리 아키텍처를 반영하도록 즉시 업데이트해야 합니다.
