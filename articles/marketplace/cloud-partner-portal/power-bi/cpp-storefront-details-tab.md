# <a name="power-bi-app-storefront-details-tab"></a>파워 BI 앱 스토어프론트 세부 정보 탭

새 **쿠폰** 페이지에서 **스토어프론트 세부정보** 탭을 사용하여 잠재 고객에게 마케팅, 판매 및 법적 정보를 제공합니다. 이 탭에서 Azure Marketplace에서 생성하는 잠재 고객에 대한 관리를 설정할 수도 있습니다. 이 긴 양식은 세부 **정보**제공, **리스팅 세부 정보,** **마케팅 아티팩트,** **법률,** **고객 지원**및 **리드 관리의**6 개 섹션으로 나뉩니다.  필드 레이블 끝에 있는 별표(*)는 필드가 필요하다는 것을 의미합니다.


## <a name="offer-details-section"></a>제품 세부 정보 섹션

쿠폰 **세부 정보** 섹션에서 AppSource 오퍼에 대한 일반 정보를 입력합니다.

![상점 세부 정보 탭의 제품 세부 정보 섹션](./media/offer-details-section.png)

다음 표에서는 이러한 필드에 대한 자세한 정보를 제공합니다. 필수 필드는 별표(*)로 표시됩니다.  

|   필드               |   설명                                                                           |
|-----------------------|-----------------------------------------------------------------------------------------|
| **오퍼 요약\***     | 최대 100자까지 사용하는 앱의 간략한 목적입니다.                             |
| **제안 설명\*** | 최대 3,000자까지 사용하는 앱에 대한 설명입니다. 이 필드는 간단한 HTML 서식을 지원합니다. |
|   |    |


## <a name="listing-details-section"></a>목록 세부 정보 섹션

**세부정보 목록** 섹션에서는 앱이 일반적으로 사용되는 산업, 앱 카테고리, 호환 되는 제품 및 관련 검색어와 같은 앱의 컨텍스트에 대한 정보를 수집합니다.

![상점 세부 정보 탭의 목록 세부 정보 섹션](./media/listing-details-section.png)

이 섹션의 필드를 완료하려면 다음 표를 사용합니다.  필수 필드는 별표(*)로 표시됩니다.
 
|   필드                                  |   설명                                                        |
| --------------                           | ---------------------                                                |
| **산업**                           | 앱이 가장 적합한 산업을 선택합니다. 앱이 여러 산업과 관련된 경우 이 필드를 비워 둡니다.      |
| **카테고리\***                           | 앱과 관련된 카테고리를 최대 3개까지 선택합니다.     |
| **앱 도움말 링크\***               | 앱에 대한 온라인 도움말을 제공하는 페이지에 URL을 제공합니다.           |
| **앱에서 작동하는 제품(최대 3)** | **새** 플러스 기호를 선택하여 텍스트 필드를 만듭니다. 필드에 앱에서 작동하는 제품의 이름을 입력합니다. 최대 3개의 제품을 나열합니다.       |
| **검색 키워드(최대 3)**              | 사용자가 AppSource에서 응용 프로그램을 검색하는 데 사용할 키워드를 최대 3개까지 입력합니다. 예를 들어 응용 프로그램을 "내 이메일 앱"이라고 하는 경우 키워드는 **전자 메일,** **메일 링**및 **메일 앱일**수 있습니다. |
|  |  |


## <a name="marketing-artifacts-section"></a>마케팅 아티팩트 섹션

마케팅 **아티팩트** 섹션에서 AppSource에 표시할 브랜딩 및 마케팅 자료를 업로드합니다.  이 섹션은 **로고,** **비디오,** **문서**및 스크린샷의 네 가지 하위 **섹션으로 나뉩니다.** 로고와 스크린샷은 유일한 필수 마케팅 아티팩트입니다. 그러나 최고의 고객 호소를 위해 동영상과 문서도 추가하는 것이 좋습니다.

![상점 세부 정보 탭의 마케팅 아티팩트 섹션](./media/marketing-artifacts-section.png)

이 섹션의 필드를 완료하려면 다음 표를 사용합니다. 필수 필드는 별표(*)로 표시됩니다.
 
|    필드                             |    설명                                                    |
|   -----------                        |    -------------                                                  |
| *로고*                              |                                                                   |
| **오퍼 로고(png 형식, 48x48)\***   | 로고를 업로드하여 앱 개요 또는 앱 검색 결과에 표시합니다. AppSource는 48 x 48 px의 해상도로 PNG 형식만 지원합니다.  |
| **제품 로고(.png 형식, 216x216)\*** | 로고를 업로드하여 앱의 세부정보 페이지에 표시합니다.  AppSource는 216 x 216 px의 해상도로 PNG 형식만 지원합니다.  |
| *비디오*                             |                                                                   |
| **이름**                             | 앱 이름 또는 제목을 입력합니다.                                          |
| **Url**                              | YouTube 또는 Vimeo에서 호스팅되는 동영상의 URL을 입력합니다.                              |
| **미리 보기**                        | 앱의 축소판 이미지를 추가합니다.  AppSource는 1280 x 720 px의 해상도로 PNG 형식만 지원합니다.   |
| *문서*                          | 앱소스에 표시할 문서를 최대 3개까지 추가하여 자세히 알아보기 제목 **아래에 표시합니다.**  |
| **이름**                             | 지원 문서의 이름이나 제목을 입력합니다.                              |
| **파일**                             | PDF 파일을 업로드합니다.                             |
| *스크린샷\**                      | 최대 5개의 스크린샷을 추가할 수 있습니다.                        |
| **이름**                             | 스크린샷의 이름이나 제목을 입력합니다.                                       |
| **이미지**                            | PNG 스크린샷을 업로드합니다. 해상도는 1280 x 720 px여야 합니다.  | 
|   |   |

[클라우드 파트너 포털에](https://cloudpartner.azure.com) 업로드한 로고는 다음을 *수행해서는 안 됩니다.*

- 그라데이션을 사용합니다. 로고의 모양과 느낌은 평평해야 합니다.
- 회사 이름, 브랜드 이름 또는 기타 텍스트를 포함합니다. 
- 늘어난 것 봐.

## <a name="legal-section"></a>법적 정보 섹션

**법률** 섹션에서각 제안에 필요한 두 가지 법적 문서( 개인 정보 보호 정책 및 이용 약관)를 제공합니다.

![상점 세부 정보 탭의 법적 정보 섹션](./media/legal-section.png)

이 섹션의 필드를 완료하려면 다음 표를 사용합니다.

|   필드                |   설명                           |
|------------------------|--------------------------------------   |
| **개인정보 처리방침 URL\*** | 게시된 개인 정보 보호 정책에 대한 URL       |
| **이용 약관\***       | 일반 텍스트 또는 간단한 HTML로 서식이 지정된 사용 정책     |
|  |  |


## <a name="customer-support-section"></a>고객 지원 섹션

고객 **지원** 섹션에서 온라인 고객 지원 페이지에 대한 **지원 URL을** 제공합니다.  이 페이지에서는 전화, 이메일 및 실시간 채팅과 같은 여러 연락처 옵션을 제공해야 합니다. 


## <a name="lead-management-section"></a>잠재 고객 관리 섹션

잠재 **고객 관리** 섹션에서 AppSource에서 제공하는 고객 잠재 고객을 수집하도록 시스템을 설정합니다. 잠재 고객에 대한 저장소 옵션을 선택하려면 다음 표를 사용하십시오.

|    필드               |   잠재 대상 고객                               |
|------------------------|--------------------------------------            |
|  **없음**              | 잠재 고객을 수집하지 않습니다. 기본 옵션입니다.  |
| **Azure Blob(사용되지 않음)** | 컨테이너 이름과 연결 문자열을 사용하여 [Azure Blob 저장소를](https://docs.microsoft.com/azure/storage/blobs/storage-blobs-overview) 지정합니다.  이 선택은 더 이상 사용되지 않습니다. 대신 **Azure 테이블을** 사용합니다.  |
| **Azure Table**        | 연결 문자열을 사용하여 [Azure 테이블 저장소를](https://docs.microsoft.com/azure/cosmos-db/table-storage-overview) 지정합니다.  |
| **다이내믹스 CRM 온라인** | URL 및 인증 자격 증명을 사용하여 [Dynamics 365를](https://dynamics.microsoft.com/) 지정합니다. |
| **HTTPS 끝점**     | JSON 페이로드를 사용하여 HTTPS 끝점을 지정합니다.   |
| **Marketo**            | 서버 ID, 먼치킨 ID 및 양식 ID를 사용하여 [Marketo](https://www.marketo.com/) 인스턴스를 지정합니다.   |
| **Salesforce**         | 개체 식별자를 사용하여 [Salesforce를](https://www.salesforce.com/) 지정합니다. |
|  |  |

제품이 게시되면 잠재 고객 연결의 유효성을 검사하고 테스트 잠재 고객을 지정된 대상으로 자동으로 보냅니다. 잠재 고객 정보를 지속적으로 관리하고 현재 고객 관리 아키텍처를 반영하도록 설정을 신속하게 업데이트합니다.
