# <a name="power-bi-app-offer"></a>Power BI 앱 제품

>[!Important]
>2020년 4월 13일부터 Power BI 앱 제공의 관리를 파트너 센터로 이전합니다. 마이그레이션 후 파트너 센터에서 오퍼를 만들고 관리합니다. Power BI [앱 만들기 개요의](https://aka.ms/AzurePBIAppCreationOverview) 지침을 따라 마이그레이션된 오퍼를 관리합니다.

|              |                                |
|--------------|--------------------------------|
| 이 문서에서는 Microsoft [AppSource 마켓플레이스에](https://appsource.microsoft.com/)Power BI 앱을 게시하는 방법을 설명합니다.  Power BI 앱은 데이터 집합, 보고서 및 대시보드를 포함하여 사용자 지정 가능한 Power BI 콘텐츠를 패키지화합니다. 그런 다음 AppSource를 통해 다른 Power BI 테넌트에 앱을 배포하고 개발자가 허용하는 조정 및 사용자 지정을 수행하고 자신의 데이터에 연결할 수 있습니다. | ![Power BI 아이콘](./media/powerbi-icon.png) |


이 문서는 세 가지 주요 부분으로 나뉩니다.

-   [필수 구성 조건](./cpp-prerequisites.md). Power BI 앱 오퍼를 만들고 게시하기 위한 기술 및 비즈니스 요구 사항입니다.
-   [Power BI 앱 오퍼를 만듭니다.](./cpp-create-offer.md) [클라우드 파트너 포털을](https://cloudpartner.azure.com)사용하여 Power BI 앱 오퍼 항목을 만드는 방법.
-   [Power BI 앱 오퍼를 게시합니다.](./cpp-publish-offer.md) 게시를 위해 AppSource에 새 오퍼를 제출하는 방법 및 기존 오퍼를 업데이트하는 방법


## <a name="publishing-steps"></a>게시 단계

Power BI 앱 오퍼를 게시하기 위한 상위 단계는 다음과 같습니다.

![Power BI 앱은 게시 단계를 제공합니다.](media/publishing-steps.png)

Power BI 앱제공 게시 프로세스는 다음과 같습니다.

1. Power BI에서 템플릿 응용 프로그램을 만듭니다. 이 작업은 오퍼의 기본 기술 자산을 나타내는 패키지 설치 URL을 생성합니다. 또한 이 때 테스트 패키지를 사전 프로덕션으로 승격합니다. 자세한 내용은 [Power BI 템플릿 앱이란 무엇입니까?](https://docs.microsoft.com/power-bi/service-template-apps-overview) 
2. 공식 이름, 설명, 로고 등을 포함하여 오퍼의 마케팅 자료를 수집하거나 생성합니다. 
3. 오퍼의 법률 및 지원 *문서(이용 약관,* *개인 정보 보호 정책,* *지원 정책,* 사용자 도움말 등)를 수집하거나 생성합니다.
4. 오퍼 만들기: 클라우드 파트너 포털을 사용하여 제안 설명, 마케팅 자료, 법률 정보, 지원 정보 및 자산 사양을 포함하여 제품의 세부 정보를 구성합니다.  오퍼가 완전히 지정되면 게시를 위해 제출합니다.
5. 클라우드 파트너 포털에서 게시 프로세스를 모니터링합니다.  이 단계에서 AppSource 온보딩 팀 테스트, 유효성 검사 및 응용 프로그램을 인증합니다. 
6. 앱이 성공적으로 인증되면 테스트 환경에서 검토하여 릴리스합니다. 
7. 전원 BI 앱은 AppSource에 나열됩니다 (그것은 "라이브 간다").
8. Power BI에서 프로덕션 전 프로덕션 패키지를 프로덕션으로 승격합니다. 자세한 내용은 [템플릿 앱 릴리스 관리를](https://docs.microsoft.com/power-bi/service-template-apps-create#manage-the-template-app-release)참조하십시오.
