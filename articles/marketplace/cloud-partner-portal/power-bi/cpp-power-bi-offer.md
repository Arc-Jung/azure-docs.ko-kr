# <a name="power-bi-application-offer"></a>Power BI 애플리케이션 제품

|                  |                                |
|------------------|--------------------------------|
| 이 섹션에서는 Microsoft [AppSource Marketplace](https://appsource.microsoft.com/)에 Power BI 애플리케이션을 게시하는 방법을 설명합니다. Power BI 애플리케이션은 Power BI 콘텐츠를 패키지하고, 데이터 흐름을 사용하여 보고서 및 대시보드를 공통 데이터 스토리지의 데이터에 연결합니다. 그런 다음, AppSource 통해 다른 테넌트에 앱을 배포할 수 있습니다. | ![Power BI 아이콘](./media/powerbi-icon.png) |


이 섹션은 다음과 같은 주요 부분으로 구분됩니다.

-   [필수 조건](./cpp-prerequisites.md) - Power BI 앱 제품을 만들거나 게시하기 위한 필요한 기술 및 비즈니스 요구 사항입니다.
-   [Power BI 제품 만들기](./cpp-create-offer.md) - [Cloud 파트너 포털](https://cloudpartner.azure.com)을 사용하여 새 Power BI 앱 제품 항목을 만드는 데 필요한 단계입니다.
-   [Power BI 앱 제품 게시](./cpp-publish-offer.md) - AppSource에 게시할 수 있도록 새 제품을 제출하는 방법 및 기존 제품을 업데이트하는 방법입니다.


## <a name="publishing-steps"></a>게시 단계

다음 다이어그램에서는 Power BI 앱 제품을 게시하는 고급 단계를 설명합니다.

![Power BI 앱 게시 단계](media/publishing-steps.png)

Power BI 앱 게시 프로세스

1. Power BI에서 서비스 애플리케이션을 만듭니다(패키지 설치 URL 만들기 결과).  이 URL은 제품에 대한 기술 자산을 나타냅니다.  자세한 내용은 [Power BI란?](https://go.microsoft.com/fwlink/?linkid=2028636)을 참조하세요.
2. 다음을 비롯한 비즈니스 자산을 수집하거나 만듭니다.
    - 마케팅 자료
    - 법률 및 지원 문서
3.  제품 만들기 - Cloud 파트너 포털을 사용하여 제품에 대한 세부 사항 및 정보를 구성합니다. 제품 설명, 마케팅 자료, 법적 정보, 지원 정보 및 자산 사양이 포함됩니다.
4.  제품 인증 및 게시 - 이 제출은 게시 프로세스를 시작합니다.
    - AppSource 온보딩 팀에서 사용자의 애플리케이션을 테스트, 유효성 검사 및 인증합니다. 
    - 인증된 후에는 해당 테스트 환경에서 검토하고 릴리스해야 합니다.

이러한 모든 단계를 수행한 후에 AppSource에서 앱이 “실시간으로 실행”됩니다.
