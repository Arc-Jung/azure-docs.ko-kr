# <a name="dynamics-365-for-customer-engagement-storefront-details-tab"></a>Dynamics 365 for Customer Engagement 상점 세부 정보 탭

**상점 세부 정보** 탭을 사용하여 영업 및 마케팅에 대한 정보와 아티팩트를 제공합니다.  이 탭에는 **제안 세부 정보**, **목록 세부 정보**, **마케팅 아티팩트**, **법적 정보**, **고객 지원**, **잠재 고객 관리**의 6가지 섹션이 있습니다.  별표(*)가 추가된 레이블은 필수 항목임을 나타냅니다.


## <a name="offer-details-section"></a>제안 세부 정보 섹션

이 섹션에서는 제안의 요약과 설명을 제공합니다.  이 정보는 마켓플레이스에서 눈에 띄게 표시됩니다.
 
![상점 세부 정보 탭의 제안 세부 정보 섹션](./media/storefront-tab-offer-details.png)

다음 표에서는 이 탭의 필드에 대해 설명합니다. 

|    **필드**          |  **설명**               |
|  ---------------      |  ---------------               |
|  제품 요약        | 제품 가치 제안의 요약입니다. 제품 검색 페이지에 표시됩니다. 형식은 최대 100자의 일반 텍스트입니다. |
|  제품 설명    | 앱 세부 정보 페이지에 표시되는 설명입니다. 형식은 최대 1,300자의 간단한 HTML입니다(p, em, ul, li, ol 및 header 태그 포함).  |
|  |  | 


## <a name="listing-details-section"></a>목록 세부 정보 섹션

이 섹션에서는 마켓플레이스 목록에 대한 세부 정보를 제공합니다.

![상점 세부 정보 탭의 목록 세부 정보 섹션](./media/storefront-tab-listing-details.png)

다음 표에서는 이 탭의 필드에 대해 설명합니다. 

|    **필드**          |  **설명**               |
|  ---------------      |  ---------------               |
|  산업           | 앱에 가장 적합한 산업을 선택합니다(최대 2개). 아무 것도 적용되지 않는 경우 선택하지 않은 채 비워 둘 수 있습니다.  |
|  범주           | 앱과 관련된 범주를 선택합니다. 1~3개의 옵션을 선택합니다.  |
|  앱 유형             | AppSource에서 사용할 앱에 대한 평가판의 유형을 선택합니다. **체험**은 애플리케이션을 추가 비용 없이 사용할 수 있음을 나타내고, **평가판**은 고객이 짧은 기간 동안 추가 비용 없이 앱을 사용해 볼 수 있음을 나타내며, **나에게 연락**은 고객이 AppSource에서 앱 평가판을 요청할 수 있음을 나타냅니다.  |
| 앱 도움말 링크 | 앱에 대한 온라인 설명서 또는 다른 도움말 리소스의 URL입니다.  |
| 지원되는 국가/지역 | 팝업 대화 상자를 통해 앱이 제공되는 지리적 지역을 선택할 수 있습니다.  기본값은 전 세계 배포입니다. |
| 지원되는 언어   | 앱에서 지원하는 언어를 선택합니다. 현재는 몇 가지 언어 옵션만 지원됩니다.  (앱이이 목록에 없는 추가 언어를 지 원하는 경우 계속 제품을 게시 하 고이 정보를 전자 메일: appsource\@microsoft.com.)  |
| 앱 버전           | 앱의 버전 번호    |
| 앱 릴리스 날짜      | 앱의 릴리스/게시 날짜 |
| 앱과 연동되는 제품 | 앱과 상호 작용하는 제품 또는 서비스입니다. 최대 3개의 제품을 나열할 수 있습니다. 제품을 나열하려면 **+ 새로 만들기**를 클릭하고 표시된 텍스트 상자에서 제품 이름을 입력합니다.  |
| 검색 키워드       | 앱과 연결된 키워드 세트입니다(최대 3개).  AppSource를 사용하면 고객이 키워드를 기준으로 검색할 수 있습니다.  사용자가 앱을 검색할 때 사용할 일반적인 단어를 선택합니다.  |
|  숨기기 키             | 라이브 상태가 되기 전에 제안을 미리 볼 수 있도록 제안 URL과 결합된 비밀 키입니다.  이 키는 암호가 *아닙니다*.  영숫자만 포함할 수 있습니다.  |
|  |  |


## <a name="marketing-artifacts-section"></a>마케팅 아티팩트 섹션

이 섹션에서는 로고 이미지, 판촉 또는 교육용 비디오, 제품 설명서 및 사용자 인터페이스 스크린샷과 같은 마케팅 참고 자료를 제공합니다.

![상점 세부 정보 탭의 마케팅 아티팩트 섹션](./media/storefront-tab-market-artifacts.png)

다음 표에서는 이 탭의 필드에 대해 설명합니다. 

|      필드            |    설명                 |
|  ---------------      |  ---------------               |
| 제품 로고(.png 형식, 48x48) | 앱의 검색 페이지에 표시할 이미지를 업로드합니다. 48x48 픽셀 크기의 PNG 형식이어야 합니다. |
| 제품 로고(.png 형식, 216x216) | 앱의 세부 정보 페이지에 표시할 이미지를 업로드합니다. 216x216 픽셀 크기의 PNG 형식이어야 합니다. |
| 동영상                | 앱에 대한 판촉 또는 교육용 비디오를 업로드합니다(최대 4개).  각 비디오에 대한 비디오 이름, URL(YouTube 또는 Vimeo만 해당) 및 관련 썸네일을1280x720 픽셀 크기의 PNG 형식으로 입력합니다.  | 
| 문서             | 판촉 또는 교육용 제품 문서를 PDF 형식으로 업로드합니다(최대 3개).  각 문서에 대한 문서 이름을 입력합니다.  |
| 스크린샷           | 앱 UI 이미지를 1280x720 픽셀 크기의 PNG 형식으로 업로드합니다(최대 5개).  이러한 스크린샷은 앱에 대한 AppSource 세부 정보 페이지에 표시됩니다. |
|  |  |


## <a name="legal-section"></a>법적 정보 섹션

이 섹션에서는 앱에 대한 *개인정보처리방침* 및 *사용 약관*을 제공합니다.

![상점 세부 정보 탭의 법적 정보 섹션](./media/storefront-tab-legal.png)

다음 표에서는 이 탭의 필드에 대해 설명합니다. 

|      필드            |    설명                 |
|  ---------------      |  ---------------               |
| 개인정보처리방침 URL    | 앱에 대한 온라인 개인정보처리방침의 URL입니다.  |
| 사용 약관          | 앱에 대한 일반 텍스트 형식의 사용 약관입니다.  AppSource 고객은 이러한 약관에 동의해야 앱 평가판을 사용할 수 있습니다. |
|  |  |


## <a name="customer-support-section"></a>고객 지원 섹션

이 섹션에서는 앱에 대한 **지원 URL**을 제공합니다.


## <a name="lead-management-section"></a>잠재 고객 관리 섹션

이 섹션에서는 제안 목록에서 생성된 잠재 고객을 처리하는 메커니즘을 제공합니다.  일반적으로 잠재 고객은 CRM(고객 관계 관리) 시스템에 저장됩니다.  지원되는 대상은 **없음**, **Azure Table**, **Dynamics CRM Online**, **HTTPS 엔드포인트**, **Marketo** 및 **Salesforce**입니다.  자세한 내용은 [고객 잠재 고객 가져오기](../../cloud-partner-portal-orig/cloud-partner-portal-get-customer-leads.md)를 참조하세요.
