# <a name="offer-listing-best-practices"></a>목록에 제품 추가 모범 사례

|:--- |:--- |

### <a name="category"></a>범주

### <a name="industries"></a>산업

업계 선택은 Azure Marketplace에 게시 된 AppSource 및 컨설팅 서비스에 게시 된 제품에만 적용 됩니다. 제품이 산업별 요구 사항을 해결 하 고 제품 설명에서 산업별 기능을 호출 하는 경우 산업 및/또는 감축할를 선택 합니다. 최대 2 개의 산업을 선택 하 고, 선택한 산업 당 2 개 (2) 감축할를 선택할 수 있습니다.

제품이 AppSource에서 선택한 제품 아래에 표시 되도록 앱이 작동 하는 제품을 선택 합니다.

### <a name="search-keywords"></a>검색 키워드

키워드는 고객이 검색할 때 제품을 찾는 데 도움이 될 수 있습니다. 제품에 대 한 최상위 검색 키워드를 식별 하 고 제품 요약 및 설명 뿐만 아니라 제품 목록 세부 정보 섹션의 키워드 섹션에 포함 합니다.

## <a name="link-to-your-offer-page-from-your-website"></a>웹 사이트에서 제품 페이지에 연결

사이트의 AppSource 또는 Azure Marketplace 배지에서 마켓플레이스의 목록에 연결하는 경우 URL 끝에 다음 쿼리 매개 변수를 포함하여 강력한 분석 및 보고를 지원할 수 있습니다.
* **src**: 트래픽이 AppSource(예: 웹 사이트, LinkedIn 또는 Facebook)로 라우팅되는 원본을 포함합니다.
* **mktcmpid**: 문자, 숫자, 밑줄 및 하이픈(예: *blogpost_12*)의 조합으로 최대 16자까지 포함할 수 있는 마케팅 캠페인 ID입니다.

다음 예제 URL에는 앞의 두 쿼리 매개 변수가 모두 포함되어 있습니다. `https://appsource.microsoft.com/product/dynamics-365/mscrm.04931187-431c-415d-8777-f7f482ba8095?src=website&mktcmpid=blogpost_12`

이러한 매개 변수를 AppSource URL에 추가하면 [Cloud 파트너 포털](https://cloudpartner.azure.com/#insights)의 분석 대시보드에서 캠페인 효과를 검토할 수 있습니다.
