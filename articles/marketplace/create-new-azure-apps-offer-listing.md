# <a name="how-to-configure-your-azure-application-offer-listing-details"></a>Azure 애플리케이션 제품 목록 정보를 구성 하는 방법

Azure 애플리케이션 제품에 대 한 **제품 목록** 페이지에 제공 하는 정보는 Microsoft 상용 marketplace online 스토어에 표시 됩니다. 여기에는 제품, 스크린샷 및 마케팅 자산에 대 한 설명이 포함 됩니다. 이 모양을 확인 하려면 [제안 목록 세부 정보](plan-azure-application-offer.md#offer-listing-details)를 참조 하세요.

> [!NOTE]
> 제안 설명이 "이 응용 프로그램은 [영어가 아닌 언어] 에서만 사용할 수 있는 경우 설명, 문서, 스크린샷, 사용 약관 등의 제품 목록 내용이 영어로 표시 되지 않아도 됩니다. 또한 제품 목록 콘텐츠에 사용 되는 것 이외의 언어로 콘텐츠를 제공 하는 _데 유용한 링크_ URL을 제공 하는 것도 허용 됩니다.

## <a name="marketplace-details"></a>마켓플레이스 세부 정보

**제품 목록** 페이지의 **Marketplace 세부 정보** 에서 다음 단계를 완료 합니다. 이러한 필수 정보에 대 한 자세한 내용은 [제품 목록 세부 정보](plan-azure-application-offer.md#offer-listing-details)를 참조 하세요.

1. **이름** 상자는 이전에 **새 제품** 대화 상자에서 입력 한 이름으로 미리 채워집니다. 언제든지 이름을 변경할 수 있습니다. 여기에 입력하는 이름은 고객에게 제품 목록의 제목으로 표시됩니다.
1. **검색 결과 요약** 상자에서 최대 100 문자 텍스트를 입력 합니다. 이 요약은 marketplace에서 검색 결과를 나열 하는 데 사용 됩니다.
1. **간단한 설명** 상자에 최대 256 자까지 일반 텍스트를 입력 합니다. 이 요약은 제품의 세부 정보 페이지에 표시 됩니다.
1. **설명** 상자에 제품에 대 한 설명을 입력 합니다. 이 텍스트 상자에는 설명이 더 유용 하 게 만드는 데 사용할 수 있는 다양 한 텍스트 편집기 컨트롤이 있습니다. HTML 태그를 사용 하 여 설명 서식을 지정할 수도 있습니다. 이 상자에는 HTML 태그와 공백을 포함 하 여 최대 3000 자의 텍스트를 입력할 수 있습니다. HTML 서식 지정에 대 한 자세한 내용은 [상용 marketplace 제품 설명에서 지원 되는 html 태그](supported-html-tags.md)를 참조 하세요.
1. 필드 **검색 키워드** 상자에서 고객이 상용 marketplace에서 제품을 찾는 데 사용할 수 있는 최대 3 개의 검색 키워드를 입력 합니다. 검색에 해당 텍스트가 자동으로 포함 되므로 제품 **이름** 및 **설명을** 포함할 필요가 없습니다.
1. **개인 정보 취급 방침 링크** 상자에 조직의 개인 정보 취급 방침에 대 한 링크 (https로 시작)를 입력 합니다. 앱이 개인 정보 법률 및 규정을 준수 하 고 유효한 개인 정보 취급 방침을 제공 하는지 확인 해야 합니다.

## <a name="add-supplemental-links-optional"></a>보조 링크 추가 (선택 사항)

추가 온라인 설명서에 대 한 링크를 추가 하려면 다음 단계를 완료 합니다.

1. 앱 또는 관련 서비스에 대 한 선택적 보조 온라인 문서를 추가 하려면 **유용한 링크** 에서 **링크 추가** 를 선택 합니다.
1. 표시 되는 필드에 제목 (최대 255 자)을 입력 하 고 `https://` 온라인 문서에 대 한 링크 (부터)를 입력 합니다.
1. 다른 링크를 입력 하려면 1 ~ 2 단계를 반복 합니다.

## <a name="enter-your-contact-information"></a>연락처 정보를 입력합니다.

**연락처 정보** 에서 다음 연락처에 대 한 정보를 제공 합니다.

- **지원 연락처** (필수) – 일반적인 지원 질문을 위한 연락처입니다.
- **엔지니어링 연락처** (필수) - 기술 관련 질문을 위한 연락처입니다. 이 정보를 사용 하 여 인증 문제를 포함 하 여 제품에 문제가 있는 경우 사용자에 게 연락 합니다.
- **CSP 프로그램 연락처** (선택 사항) – 지원 및 비즈니스 문제 이 정보는 CSP 파트너에만 표시 됩니다.

각 연락처에 대해 이름, 전화 번호 및 전자 메일 주소를 제공 합니다 (공개적으로 표시 되지 않음). 지원 담당자에 게 지원 URL이 필요 합니다 (공개적으로 표시 됨).

1. **지원 연락처** 상자에서 이름, 전자 메일 주소, 전화 번호 및 지원 페이지 URL을 입력 합니다.
1. **엔지니어링 연락처** 상자에서 이름, 전자 메일 주소 및 전화 번호를 입력 합니다.
1. 필드 **CSP 프로그램 연락처** 상자에서 이름, 전자 메일 주소 및 전화 번호를 입력 합니다.
1. Csp [(클라우드 솔루션 공급자) 프로그램](cloud-solution-providers.md)에 대 한 제품을 확장 하려면 **Csp 프로그램 마케팅 자료** 상자에 마케팅 자료에 대 한 링크를 제공 합니다.

    > [!NOTE]
    > CSP 프로그램은 CSP 파트너가 제품을 번들, 시장 및 재판매 수 있도록 하 여 더 광범위 한 범위의 고객에 게 제품을 확장 합니다. 이러한 재판매인은 제품 마케팅 자료에 액세스해야 합니다. 자세한 내용은 [Microsoft를 사용 하 여 출시 하기](https://partner.microsoft.com/reach-customers/gtm)를 참조 하세요.

## <a name="add-marketplace-media"></a>Marketplace 미디어 추가

### <a name="store-logos"></a>스토어 로고

**로고** 에서 216 x 216 및 350 x 350 픽셀 사이의 PNG 형식으로 **많은** 로고를 업로드 합니다. 파트너 센터는 **작은** (48 x 48) 및 **보통** (90 x 90) 로고를 자동으로 만듭니다 .이 로고는 나중에 원하는 경우 바꿀 수 있습니다.

3 가지 로고 크기는 모두 온라인 매장의 다른 위치에서 사용 됩니다.

- Azure Marketplace의 제품 목록 페이지에 많은 로고가 표시 됩니다.
- Microsoft Azure에서 새 리소스를 만들 때 medium 로고가 표시 됩니다.
- Azure Marketplace 검색 결과에 작은 로고가 표시 됩니다.

### <a name="add-screenshots-optional"></a>스크린샷 추가 (선택 사항)

제품을 보여 주는 최대 5 개의 스크린샷을 추가 합니다. 모든 이미지는 1280 x 720 픽셀 크기 및의 크기 여야 합니다. PNG 형식입니다.

1. **스크린샷** 에서을 끌어 놓습니다. **스크린샷** 상자에 PNG 파일을 놓습니다.
1. **이미지 캡션 추가** 옆의 편집 아이콘을 클릭 합니다.
1. 표시 되는 대화 상자에서 캡션을 입력 하 고 **확인** 을 클릭 합니다.
1. 1 ~ 3 단계를 반복 하 여 추가 스크린샷을 추가 합니다.

### <a name="add-videos-optional"></a>비디오 추가 (선택 사항)

제품을 설명 하는 YouTube 또는 Vimeo 비디오에 대 한 링크를 추가할 수 있습니다. 이러한 비디오는 고객과 함께 고객에 게 표시 됩니다. 비디오의 축소판 이미지를 입력 해야 합니다. 크기는 1280 x 720 픽셀 크기와입니다. PNG 형식입니다. 제품 당 최대 4 개의 비디오를 추가할 수 있습니다.

1. 비디오 **에서** **비디오 추가** 링크를 선택 합니다.
1. 표시 되는 상자에 비디오의 이름과 링크를 입력 합니다.
1. 을 끌어서 놓습니다. 회색 **축소판** 상자에 PNG 파일 (1280 x 720 픽셀)을 표시 합니다.
1. 다른 비디오를 추가 하려면 1 ~ 3 단계를 반복 합니다.

> [!TIP]
> 파일을 업로드하는 동안 문제가 발생한 경우 로컬 네트워크가 파트너 센터에서 사용하는 https://upload.xboxlive.com 서비스를 차단하지 않는지 확인하세요.
