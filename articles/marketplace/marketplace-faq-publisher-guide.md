# <a name="common-questions-about-the-microsoft-commercial-marketplace"></a>Microsoft 상업적 marketplace에 대 한 일반적인 질문

이 문서에서는 Microsoft 상업적 marketplace에 대 한 자주 묻는 질문에 답변 합니다.

## <a name="general"></a>일반

### <a name="what-is-microsoft-commercial-marketplace"></a>Microsoft 상업적 marketplace 란 무엇 인가요?

상업적 marketplace는 파트너에 의해 구축 된 솔루션과 비즈니스 및 정부 기관을 연결 합니다. 파트너 센터에서 제공 하는 파트너는 파트너 센터에서 제품을 만들고 관리 하며, 고객은 Microsoft AppSource, Azure Marketplace, 클라우드 솔루션 공급자 프로그램의 대리점, 공동 판매 프로그램을 통한 영업 팀 등의 제품 내 환경을 통해 솔루션을 검색 하 고 구매할 수 있습니다.

자세히 알아보려면 [Microsoft 상업적 marketplace 허브](https://partner.microsoft.com/asset/collection/commercial-marketplace#/)로 이동 하세요.

### <a name="whats-the-difference-between-microsoft-appsource-and-azure-marketplace"></a>Microsoft AppSource와 Azure Marketplace의 차이점은 무엇 인가요?

AppSource에는 Microsoft 365, Dynamics 365 및 Power Platform 서비스에 통합 되는 앱과 같은 비즈니스 및 업계 솔루션이 포함 됩니다.

Azure Marketplace은 Azure를 기반으로 하거나 Azure 구독 자체 내에서 사용 되는 IT 및 개발자 솔루션을 포함 합니다.

자세히 알아보려면 [구매자 가이드](https://aka.ms/MarketplaceBuyerGuide)로 이동 하세요.

### <a name="why-should-i-make-my-software-or-services-available-in-the-microsoft-commercial-marketplace"></a>Microsoft 상업적 marketplace에서 소프트웨어 또는 서비스를 사용 하도록 설정 해야 하는 이유는 무엇 인가요?

상업적 marketplace는 시장 시장 공동 작업 및 판매 채널 및 마켓플레이스 보상을 위한 출발점입니다. Websites 및 제품 내 환경은 사용자의 제품을 고객에 게 제공 하는 수백만 명의 고객에 게 표시 합니다. 상거래 기능을 빌드하거나 구매할 필요도 없습니다. 

상업적 marketplace는 계획 별로 141 지역에서 사용할 수 있습니다. 

많은 국가에서 세금 지불을 관리 하 고 대리점의 글로벌 채널 네트워크에 대 한 액세스를 제공 합니다. 고객의 Microsoft 고객 계약 또는 기업계약를 사용 하 여 새 구매 관계를 만들지 않고도 조직에 판매할 수 있습니다.

자세히 알아보려면 [marketplace를 통해 판매](https://partner.microsoft.com/asset/detail/why-sell-through-marketplace)하는 이유에서 비디오를 참조 하세요.

### <a name="is-there-a-listing-fee-for-the-microsoft-commercial-marketplace"></a>Microsoft 상업적 marketplace에 대 한 목록 요금이 있나요?

아니요, 상용 marketplace에서 제품을 게시할 비용은 없습니다. 적은 비율의 판매를 트랜잭션 요금으로 유지 하 고 Marketplace 보상 프로그램, 플랫폼 혁신 및 상업적 마켓플레이스 작업에 투자 합니다.

### <a name="how-do-i-sign-up-to-be-a-publisher-in-the-microsoft-commercial-marketplace"></a>Microsoft 상업적 marketplace에서 게시자로 등록 어떻게 할까요?

상업적 marketplace에서 제품을 만들려면 Microsoft 파트너 계약에 동의 하 고 게시자 계약에 동의 하 여 조직에서 Microsoft 파트너 여야 합니다.

상업적 marketplace 게시자로 등록 하려면 [파트너 센터](https://partner.microsoft.com/dashboard/account/v3/enrollment/introduction/azureisv)로 이동 합니다.

### <a name="how-can-customers-engage-with-my-offers-in-the-commercial-marketplace"></a>고객은 상업적 marketplace에서 내 제품에 어떻게 참여할 수 있나요?

고객이 상용 marketplace 제품을 사용할 수 있는 방법은 제품을 만드는 방법에 따라 달라 집니다.

- 자동으로 제공 하는 솔루션의 경우, BYOL (사용자 라이선스 필요) 또는 무료 인지 *여부에 관계*없이 고객은 결과 수 있습니다.
- 고객에 게 *드라이브 테스트*를 요청 하는 데모 (테스트 드라이브)를 지 원하는 제품입니다. 
- SaaS (Software as a service)는 상용 marketplace 외부의 평가판을 제공 하는 *무료 평가판*을 제공 합니다.
- 간단한 목록을 통해 고객은 *연락처*를 요청할 수 있습니다.

### <a name="how-do-i-create-offers-for-the-commercial-marketplace"></a>상업적 marketplace에 대 한 제품을 만들 어떻게 할까요? 있나요?

등록 하 고 게시자 계약에 동의한 후에는 파트너 센터의 상업 marketplace 페이지에 액세스할 수 있습니다. 여기에서 제품을 만들고 관리 하며 분석 보고서를 볼 수 있습니다. Microsoft 365 앱의 경우 파트너 센터 내에서 Office 스토어 프로그램을 활성화 해야 합니다.

자세한 내용은 [상용 Marketplace 시작](https://docs.microsoft.com/azure/marketplace/) 및 [Microsoft 상용 Marketplace를 통해 Office 365 추가 기능을 수익 창출](https://docs.microsoft.com/office/dev/store/monetize-addins-through-microsoft-commercial-marketplace)를 참조 하세요.

### <a name="how-do-i-get-support-assistance-for-the-commercial-marketplace"></a>상업적 marketplace에 대 한 지원 지원을 받을 어떻게 할까요? 있나요?

Marketplace 게시자 지원 팀에 문의 하려면 파트너 센터 내에서 [지원 티켓을 제출할](https://partner.microsoft.com/support/v2/?stage=1) 수 있습니다. 

[활성 커뮤니티 포럼에 참여](https://www.microsoftpartnercommunity.com/t5/Microsoft-AppSource-and-Azure/bd-p/2222) 하 여 모범 사례 및 공유 정보에 대해 알아볼 수도 있습니다.

### <a name="what-analytics-are-available-to-my-organization-from-the-commercial-marketplace"></a>상업적 marketplace에서 조직에 사용할 수 있는 분석은 무엇 인가요?

Microsoft는 상업적 marketplace 내에서 귀하의 제품에 대 한 보고를 제공 합니다. 고객, 주문, 매장 참여 등의 데이터에 액세스 하려면 [파트너 센터에서 상업적 marketplace에 대 한 분석](https://docs.microsoft.com/azure/marketplace/partner-center-portal/analytics)으로 이동 합니다.

### <a name="what-is-microsofts-relationship-with-my-customers"></a>내 고객과의 Microsoft 관계는 무엇 인가요?

상업적 marketplace는 에이전시 모델을 사용 하 여 작동 합니다. 고객은 EA (Microsoft 클라우드 계약 또는 기업계약)를 통해 microsoft와 함께 구매 계약에 서명 합니다. Microsoft는 적용 가능한 지역에서 청구, 컬렉션, 세금 지불 및 게시자 지급을 담당 하 고 있습니다. 게시자는 가격을 설정 하 고, 사용 약관을 설정 하 고, 기술 지원을 제공 합니다.

### <a name="what-terms-of-use-options-are-available-with-the-commercial-marketplace"></a>상업적 marketplace에서 사용할 수 있는 사용 약관은 무엇 인가요?

게시자는 고객이 구매 흐름의 일부로 수락 해야 하는 소프트웨어 솔루션에 대 한 사용 약관을 설정 합니다. 작업을 간소화 하 고 속도를 높이려면 사용 약관에 포함 하도록 선택할 수 있는 표준 계약을 제공 합니다. 이렇게 하면 고객이 표준 계약을 한 번 검토 하 고 법률 및 구매 팀에서 승인을 받은 다음 각 제품에 대 한 개정에 집중할 수 있습니다.

모든 고객에 게 적용 되는 변경 내용을 만들거나 특정 고객에 대 한 변경 작업을 수행할 수 있습니다. 텍스트 상자에 입력 하거나 웹 주소로 표시 되는 웹 사이트에서 호스팅될 수 있는 사용 약관을 완전히 사용자 지정할 수도 있습니다.

자세히 알아보려면 [Microsoft 상업적 marketplace에 대 한 표준 계약](https://docs.microsoft.com/azure/marketplace/standard-contract)을 참조 하세요.

### <a name="is-azure-marketplace-available-in-azure-government"></a>Azure Government에서 Azure Marketplace 사용할 수 있나요?

예, Azure Government Marketplace는 Azure Government 지역 내에서 사용할 수 있으며 Azure virtual machine 및 Azure 응용 프로그램 제품을 지원 합니다. 표준 게시 프로세스 중에 Azure Government Marketplace에서 제품을 사용할 수 있도록 하는 확인란 옵션이 있습니다.

### <a name="what-permissions-do-i-need-in-partner-center-to-manage-my-offers"></a>파트너 센터에서 내 제품을 관리 하는 데 필요한 권한은 무엇 인가요?

파트너 센터 액세스를 제어 하는 네 가지 관련 역할이 있습니다.

- *관리자*: 디렉터리에서 파트너 센터 사용자를 관리 하 고 제품을 만들고 업데이트할 수 있는 권한을 제공 합니다.
- *개발자*: 사용자가 제안을 만들고 업데이트할 수 있습니다.
- *재무 참여자*: 지급 보고를 볼 수 있는 권한을 제공 합니다.
- *Owner*: 지급 계정 및 세금 프로필을 설정할 수 있습니다. 게시자 계약을 수락한 사용자는 기본적으로 소유자가 됩니다.

새 사용자 계정을 만들거나 기본 디렉터리에 게스트 계정을 초대 하려면 Azure Active Directory (Azure AD) *전역 관리자* 권한이 필요 합니다. 파트너 센터는 id 모델에 Azure AD를 적용 하며 조직의 Azure AD 인스턴스는 IT 또는 보안 팀에서 관리 하 고 제어 하는 경우가 많습니다.

자세히 알아보려면 [사용자 역할 및 사용 권한 할당](https://docs.microsoft.com/partner-center/permissions-overview)을 참조 하세요.

### <a name="do-customers-need-a-subscription-for-microsoft-azure-cloud-services"></a>고객에 게 Microsoft Azure Cloud Services에 대 한 구독이 필요 한가요?

제품의 배달 모델에 따라 Azure Cloud Services에 대 한 구독이 필요할 수 있습니다. Azure 구독, Microsoft 365 응용 프로그램 또는 Dynamics 365 응용 프로그램에서 직접 사용 하는 솔루션에는 고객이 기존 구독을 보유 해야 합니다. 구독을 청구 메커니즘으로 사용 하기 때문에 Azure Marketplace 구매한 제품에는 Azure 구독이 필요 합니다.

### <a name="whos-responsible-for-supporting-customers"></a>고객 지원 담당자는 누구 인가요?

상업적 마켓플레이스 게시자는 해당 소프트웨어를 지원 하 고, Microsoft는 구매, 청구, 지불 및 구독 관리 환경을 지원할 책임이 있습니다.

### <a name="whats-the-process-of-validating-the-end-to-end-purchase-and-setup-flow"></a>종단 간 구매 및 설정 흐름의 유효성을 검사 하는 프로세스는 무엇 인가요?

게시 프로세스 중에 제품 미리 보기에 대 한 액세스 권한이 제공 됩니다. 액세스는 **미리 보기** 탭에서 지정 하는 사용자 에게만 제한 되며 다른 사용자가 해당 제품을 볼 수 없습니다. 미리 보기를 구매 하 고 프로세스를 테스트할 수 있지만 제품의 설정에 따라 전체 금액이 청구 됩니다.

매우 저렴 한 가격으로 구매를 완료 하려면 단독으로 볼 수 있는 비공개 요금제를 게시 하는 것이 좋습니다. $0.00 가격은 지원 되지만 유료 구독 처럼 전체 환경을 반영 하지는 않으므로 $1.00에 가격을 설정 하는 것이 좋습니다.

### <a name="what-are-leads-from-the-commercial-marketplace-and-what-should-i-do-with-them"></a>상업적 marketplace의 잠재 고객은 무엇 이며 어떻게 해야 하나요?

고객이 Microsoft AppSource 및 Azure Marketplace 상점에 제품을 사용 하는 경우 정보가 캡처됩니다. 고객에 게 충분 한 관심이 표시 되 면 잠재 고객이 만들어지고 선택한 시스템에 기회가 전송 됩니다. 상업적 marketplace로 성공한 게시자는 잠재 고객을 안내 하 고, 관계를 개발 하 고, 관심을 계산 하는 경우가 많습니다.

자세한 내용은 [marketplace 제품의 고객 리드](https://docs.microsoft.com/azure/marketplace/partner-center-portal/commercial-marketplace-get-customer-leads)를 참조 하세요.

### <a name="why-arent-my-changes-reflected-in-the-offer-on-microsoft-appsource-or-azure-marketplace"></a>변경 내용이 Microsoft AppSource 또는 Azure Marketplace의 제품에 반영 되지 않는 이유는 무엇 인가요?

파트너 센터 내에서 변경한 내용은 제품을 다시 게시 한 후에만 시스템 및 상업적 마켓플레이스 업데이트 됩니다. 변경 후 게시에 대 한 제품을 제출 했는지 확인 합니다.

## <a name="channels"></a>채널

### <a name="where-can-customers-find-my-commercial-marketplace-offers"></a>고객이 상용 marketplace 제품을 어디에서 찾을 수 있나요?

제품을 구성 하는 방법 및 고객의 작업 컨텍스트를 바탕으로 제품 내 환경, websites, 대리점 및 Microsoft 영업 팀을 통해 고객에 게 제공 됩니다. 소프트웨어가 Azure, Microsoft 365, Dynamics 365 또는 Power Platform 서비스와 직접 사용 되는 경우 고객은 제품 내에서 찾을 수 있습니다. Microsoft AppSource 및 Azure Marketplace은 제안 유형과 대상에 따라 분할 되며, 대리점은 CSP (클라우드 솔루션 공급자) 프로그램에 옵트인 한 경우에만 제품에 액세스할 수 있습니다.

### <a name="can-my-offer-appear-in-both-microsoft-appsource-and-azure-marketplace"></a>내 제품이 Microsoft AppSource와 Azure Marketplace 모두에 표시 되나요?

Microsoft AppSource은 비즈니스 및 업계 솔루션에 대 한 대상 이며 Azure Marketplace IT 및 개발자 솔루션을 위해 설계 되었습니다. 제공은 제품을 만들 때 선택한 범주와 산업을 기반으로 하는 각 상점에서 제공 됩니다. 드문 경우 지만 제품이 두 상점의 사용자와 관련 되는 경우 둘 다에 표시 될 수 있습니다.

### <a name="what-is-the-cloud-solution-provider-program"></a>클라우드 솔루션 공급자 프로그램 이란?

대리점의 글로벌 네트워크에는 클라우드 솔루션 공급자 프로그램에 참여 하는 수십 개의 조직이 포함 됩니다. CSP 프로그램을 통해 조직은 상용 marketplace에서 제품 및 제품을 재판매 수 있습니다.

자세한 내용은 [클라우드 솔루션 공급자](https://docs.microsoft.com/azure/marketplace/cloud-solution-providers)를 참조 하세요.

### <a name="how-can-i-find-resellers-to-establish-partnerships"></a>대리점을 어떻게 찾고 파트너 관계를 설정할 수 있나요?

[솔루션 공급자 디렉터리](https://www.microsoft.com/solution-providers/home)를 사용 하 여 위치, 제품, 서비스, 기술, 산업 또는 조직 이름을 기준으로 파트너를 검색할 수 있습니다.

### <a name="can-i-limit-the-resellers-who-are-allowed-to-sell-my-offer"></a>제품을 판매할 수 있는 대리점을 제한할 수 있나요?

예, 클라우드 솔루션 공급자 프로그램의 모든 대리점 또는 지정 된 사용자 에게만 제공 되는 제품을 만들 수 있습니다.

자세한 내용은 [클라우드 솔루션 공급자](https://docs.microsoft.com/azure/marketplace/cloud-solution-providers)를 참조 하세요.

### <a name="how-can-i-pay-margin-to-resellers"></a>대리점에 어떻게 요금을 지불할 수 있나요?

상업적 marketplace에는 대리점에 대 한 여백을 지정 하거나 직접 및 자동으로 지불 하는 기능이 없습니다. 그러나 상업적 marketplace 시스템 외부에서 계약, 판매 콘테스트 또는 여백을 만들 수 있습니다.

### <a name="how-does-geographic-availability-work-with-resellers"></a>대리점에서 지리적 가용성이 어떻게 작동 하나요?

제품 및 요금제의 가용성은 고객의 청구 위치에서 정의 됩니다. 대리점 먼저 판매 중인 고객을 선택 하 고 제품을 선택 합니다. 고객이 지원 되지 않는 위치에 있는 경우 제품을 사용할 수 없습니다.

## <a name="billing-capabilities"></a>요금 청구 기능

### <a name="whats-the-difference-between-transaction-and-monetization"></a>트랜잭션과 수익 화의 차이점은 무엇 인가요?

- 트랜잭션은 상품을 위한 money의 교환입니다. 제품은 가격이 있고 청구 및 컬렉션을 통해 고객이 지불 하는 경우 상업적 marketplace에서 불가능 간주 됩니다. 
- 수익 화은 상업 플랫폼 내부 또는 외부에서 발생할 수 있는 지적 재산에 대 한 비용을 부과 하는 기능입니다. 예를 들어 BYOL (사용자 라이선스 보유) 팀 앱은 불가능 SaaS 라이선스 서비스를 사용 하 여 수익 화할 수 있습니다.

### <a name="what-offer-types-are-transacted-through-microsoft"></a>Microsoft에서 어떤 제품 유형을 트랜잭션 하나요?

*지금 가져오기* 는 일부 배달 모델에 대해 Microsoft를 통해 제공 될 수 있습니다. 이 기능은 현재 다음과 같은 배달 모델에서 지원 됩니다.
- VM 이미지
- Azure 앱 (관리 되는 응용 프로그램)
- SaaS 앱

### <a name="what-billing-models-are-supported-in-the-commercial-marketplace"></a>상업적 marketplace에서 지원 되는 청구 모델은 무엇 인가요?

- VM 이미지는 시간별 사용량을 기준으로 요금이 청구 되 고 가격은 시간당 설정 됩니다.
- 관리 되는 응용 프로그램은 선택적 월별 고정 요금, 요금제 청구 및 앱에서 사용 하는 VM 이미지의 사용량 비용을 기준으로 요금이 청구 됩니다.
- SaaS 응용 프로그램은 월별 또는 연간 기준으로 단일 요금, 사용자 단위 또는 사용자 지정 계량을 기준으로 요금이 청구 됩니다.

### <a name="can-i-monetize-offer-types-that-are-not-transactable"></a>불가능 되지 않는 제품 유형을 수익 창출 수 있나요?

모든 제품 유형을 불가능 수 있는 것은 아니지만 수백만 명의 고객에 게 연락 하 여 지적 재산을 수익 창출 수 있습니다. 솔루션 내에서 라이선스 검사를 추가 하 여 라이선스에 대 한 지불을 수집할 수 있습니다. 판매 라이선스는 Microsoft 또는 별도로 수행할 수 있습니다.

### <a name="can-i-offer-a-software-trial-through-the-commercial-marketplace"></a>상용 marketplace를 통해 소프트웨어 평가판을 제공할 수 있나요?

예, 여러 가지 방법으로 무료 평가판을 제공할 수 있습니다.

- 선호 되는 방법은 유료 SaaS 제품 (한 달) 및 VM 이미지 제품 (1 ~ 3 개월)의 시작 부분에 무료 기간을 추가 하는 것입니다. 고객은 $0.00 요금을 받고 평가판 기간이 종료 되 면 구독이 자동으로 유료로 변경 됩니다. 그러면 계약 용어가 해당 지점에서 시작 됩니다.
- 고객이 상업적 marketplace 외부에서 평가판을 시작 하는 데 사용 하는 웹 주소로 고객이 원하는 SaaS 제품 목록을 만들 수 있습니다. 일부 제품 유형은 고객에 게 데모를 제공할 수 있도록 하는 *테스트 드라이브* 를 지원 합니다.

### <a name="who-pays-for-the-underlying-azure-consumption"></a>기본 Azure 사용량을 어떻게 지불 하나요?

고객의 Azure 구독에서 직접 솔루션을 사용 하는 경우 고객은 Azure 비용을 지불할 책임이 있습니다. SaaS 응용 프로그램의 경우와 같이 Azure 구독에서 솔루션을 사용 하는 경우 Azure 비용을 담당 하 게 됩니다. 고객의 SaaS 구독 비용으로이를 고려해 야 합니다.

### <a name="can-i-sell-professional-services-through-the-commercial-marketplace"></a>상업적 marketplace를 통해 전문 서비스를 판매할 수 있나요?

클라우드 서비스를 기반으로 구축 되거나 빌드된 소프트웨어는 상업적 marketplace를 통해 거래 될 수 있습니다. 전문 서비스는 아직 불가능 되지 않습니다.

## <a name="purchasing"></a>구매

### <a name="whats-required-to-purchase-solutions-from-microsoft-appsource-or-azure-marketplace"></a>Microsoft AppSource 또는 Azure Marketplace에서 솔루션을 구입 하려면 어떻게 해야 하나요?

Azure Marketplace에는 고객에 게 Azure Active Directory ID와 활성 Azure 구독이 있어야 합니다. 둘 중 하나 또는 한 가지 요구 사항만 충족 되는 경우 고객은 구매 흐름에서 해당 요구 사항을 만들라는 메시지를 표시 합니다.

Microsoft AppSource에는 고객에 게 Azure Active Directory ID가 있어야 합니다. ID가 존재 하지 않는 경우 고객은 구매 흐름에서 해당 ID를 만들라는 메시지를 표시 합니다.

### <a name="what-payment-methods-are-supported-with-the-commercial-marketplace"></a>상업적 marketplace에서 지원 되는 결제 방법은 무엇 인가요?

Microsoft AppSource은 신용 카드 지불을 지원 합니다. Azure Marketplace은 신용 카드 지불 또는 청구서를 지원 합니다. Azure Marketplace를 통해 구매한 제품은 고객의 Azure 청구서에 자동으로 추가 되며 해당 구독과 연결 된 결제 방법에 따라 요금이 청구 됩니다.

### <a name="can-customers-apply-azure-credits-or-sponsorship-toward-purchases"></a>고객이 구매에 Azure 크레딧 또는 후원을 적용할 수 있나요?

아니요, 선불 또는 후원 Azure를 사용 하 여 Azure Marketplace에서 파트너 솔루션을 구매할 수 없습니다.

### <a name="what-azure-subscription-permissions-are-required-to-complete-a-purchase"></a>구매를 완료 하는 데 필요한 Azure 구독 권한은 무엇 인가요?

사용자 계정에는 Azure 구독에 대 한 *소유자* 또는 *참가자* 권한이 있어야 합니다.

### <a name="in-what-countries-can-customers-purchase-from-microsoft-appsource-and-azure-marketplace"></a>고객이 Microsoft AppSource 및 Azure Marketplace에서 구매할 수 있는 국가는 무엇 인가요?

고객은 전 세계 141 곳의 상용 marketplace에서 솔루션을 받을 수 있습니다.

위치 목록을 검토 하려면 [상용 마켓플레이스 게시자에 대 한 세금 세부 정보](https://docs.microsoft.com/azure/marketplace/partner-center-portal/tax-details-paid-transactions)로 이동 합니다.

### <a name="what-currencies-are-supported-with-the-commercial-marketplace"></a>상용 marketplace에서 지원 되는 통화는 무엇 인가요?

다음 17 개의 통화로 트랜잭션을 처리할 수 있습니다.
- AUD
- BRL
- CAD
- CHF
- DKK
- EUR
- GBP
- INR
- JPY
- KRW
- NOK
- NZD
- RMB
- RUB
- SEK
- TWD
- USD

통화 목록을 검토 하려면 [상용 marketplace에 대 한 지리적 가용성 및 통화 지원](https://docs.microsoft.com/azure/marketplace/marketplace-geo-availability-currencies)으로 이동 합니다.

### <a name="can-customers-change-subscriptions-or-license-volumes-mid-term"></a>고객이 구독 또는 라이선스 볼륨을 변경할 수 있나요?

고객은 언제 든 지 사용자 당 SaaS 구독에 대 한 사용자 수를 늘리거나 줄일 수 있습니다. 라이선스 구독 조건은 볼륨 또는 계획 업그레이드에 관계 없이 동일 하 게 유지 됩니다. Api (응용 프로그래밍 인터페이스)를 통해 기존 구독에 대 한 계획 변경도 지원 됩니다. 게시자는 API 호출을 사용 하 여 고객의 구독 계획을 변경할 수 있습니다. Isv (독립 소프트웨어 공급 업체)는 업그레이드 및 다운 그레이드 상황을 위해 SaaS 솔루션 내에서 고객의 사용자 경험을 고려 하는 것이 좋습니다.

기존 구독에 대해 계획을 변경 하는 경우 해당 기간은 변경 되지 않은 상태로 유지 되 고 나머지 용어의 기간에 따라 비용이 계산 됩니다.

### <a name="what-could-block-a-customer-from-completing-a-purchase"></a>고객이 구매를 완료 하는 것을 차단할 수 있는 것은 무엇 인가요?

Microsoft AppSource 고객에 게는 Azure AD 인스턴스와 동일한 국가의 청구 주소가 있는 신용 카드가 있어야 하 고, 고객의 Azure AD 관리자는 시스템이 인증을 Microsoft AppSource 수 있도록 방지 해야 합니다.

Azure Marketplace의 경우 고객은 활성 Azure 구독 및 해당 구독에 대 한 *소유자* 또는 *참가자* 권한을 가진 사용자 계정이 있어야 합니다. 고객의 Azure 관리자는 Azure Policy이 사용 중인 제품과 충돌 하지 않는지, 그리고 비용 관리를 사용 하지 않도록 설정 하 여 Azure Marketplace를 사용 하지 않도록 설정 해야 합니다.

## <a name="collection-and-payouts"></a>Collection 및 지급

### <a name="when-are-customers-billed-for-their-purchases-in-the-commercial-marketplace"></a>고객은 상업적 marketplace에서 구매한 고객에 대 한 요금은 언제 청구 되나요?

고객은 일반적으로 이전 청구 주기에서 구매를 위해 매월 청구 됩니다. VM 이미지, SaaS 응용 프로그램 및 관리 되는 응용 프로그램과 같은 요금제 요금제를 사용 하는 소비 기반 제품은 사용 된 후 요금이 청구 됩니다. 구독 기반 요금제는 해당 기간의 시작 부분에서 청구 됩니다.

### <a name="how-often-will-i-receive-payouts-for-sales-through-the-commercial-marketplace"></a>상업적 marketplace를 통해 판매에 대 한 지급는 얼마나 자주 수신 하나요?

지급 매월 발생 합니다. 이전 달에 처리 된 컬렉션은 한 번에 지불 되며 판매에는 최소 조합 값 $50.00이 있어야 합니다.

신용 카드 구독과 구매는 매월 청구 되 고, Microsoft 고객 계약 또는 기업계약에 대 한 송장은 빈도가 떨어질 수 있습니다. 고객의 요금이 청구 된 시간부터 30 일의 지불을 제공 합니다. 지불을 받은 후 신용 카드 거래는 30 일의 크리스마스로 이동 합니다.

자세한 내용은 [지불 임계값, 메서드 및 시간 프레임](https://docs.microsoft.com/azure/marketplace/partner-center-portal/payment-thresholds-methods-timeframes)을 참조 하세요.

### <a name="in-what-countriesregions-are-taxes-managed-on-my-behalf"></a>사용자를 대신 하 여 세금을 관리 하는 국가/지역은 어디 인가요?

고객은 전 세계의 상용 marketplace에서 솔루션을 찾을 수 있으며 여러 위치에서 세금을 관리 합니다. 

전체 위치 목록을 검토 하려면 [상용 마켓플레이스 게시자에 대 한 세금 세부 정보](https://docs.microsoft.com/azure/marketplace/partner-center-portal/tax-details-paid-transactions)로 이동 합니다.

### <a name="where-can-i-find-payout-reporting-for-the-commercial-marketplace"></a>상업적 marketplace에 대 한 지급 보고는 어디서 찾을 수 있나요?

파트너 센터 내에서 *소유자* 또는 *재무 참가자* 권한이 있는 사용자는 지급 보고를 볼 수 있습니다. 이러한 사용자는 청구서를 포함 하는 손을 표시 하는 아이콘을 선택 하 여 오른쪽 위에 있는 보고서에 액세스할 수 있습니다.

자세한 내용은 [지급 보고](https://docs.microsoft.com/azure/marketplace/partner-center-portal/payout-summary)를 참조 하세요.

### <a name="how-do-i-configure-my-payout-method-and-destination"></a>지급 메서드 및 대상을 구성 하는 어떻게 할까요??

파트너 센터를 사용 하 여 *소유자* 권한이 있는 사용자는 지급 계정 및 세금 양식을 설정할 수 있습니다. 사용자는 오른쪽 위에 있는 **설정** 단추 (기어 아이콘)를 선택 하 여이 작업을 수행할 수 있습니다.

자세한 내용은 [지급 account and 세금 forms](https://docs.microsoft.com/azure/marketplace/partner-center-portal/set-up-your-payout-account-tax-forms)로 이동 하세요.

### <a name="are-returns-and-refunds-supported-with-the-commercial-marketplace"></a>상용 marketplace에서 반환 및 환불 지원 되나요?

환불를 수신 하기 위해 고객은 지정 된 시간 프레임 내에 구독을 취소할 수 있습니다. 솔루션이 사용 된 후 사용량에 따라 청구 되는 제안은 환불에 적합 하지 않습니다.
