# <a name="co-sell-requirements"></a>공동 판매 요구 사항

이 문서에서는 다양 한 공동 판매 상태 수준에 대 한 요구 사항을 제공 합니다. 공동 판매를 지 원하는 제품 형식의 최신 목록은  [상업용 marketplace 제품에 대 한 공동 판매 구성](commercial-marketplace-co-sell.md)을 참조 하세요. 공동 판매에 대 한 개요는 [Microsoft 영업 팀 및 파트너와 공동 판매 개요](marketplace-co-sell.md)를 참조 하세요.

다음 표에서는 가능한 모든 공동 판매 상태를 보여 줍니다.

| 상태 | 의견 |
| ------------ | ------------- |
| 공동 판매 준비 안 함 | [공동 판매 준비 상태에 대 한 최소 요구 사항이](#requirements-for-co-sell-ready-status) 충족 되지 않았습니다. |
| 공동 판매 준비 | [공동 판매 준비 상태에 대 한 모든 요구 사항이](#requirements-for-co-sell-ready-status) 충족 되었습니다. |
| Azure IP 공동 판매 incentivized | [이러한 추가 요구 사항](#requirements-for-IP-Co-sell-incentivized-status)외에도 공동 판매 준비 요구 사항이 충족 되었습니다. |
| Biz 앱 ISV 연결 프리미엄 동기  | 이 상태는 Dynamics 365 및 Power Apps 제품에 적용 되며, [이 상태에 대 한 모든 요구 사항이](#requirements-for-biz-apps-isv-connect-premium-incentive-status) 충족 되었음을 나타냅니다. |
|||

## <a name="requirements-for-co-sell-ready-status"></a>공동 판매 준비 상태에 대 한 요구 사항

공동 판매 준비 상태를 달성 하기 위한 제안의 경우 다음 요구 사항을 충족 해야 합니다.

**모든 파트너**:

- 파트너 센터에 MPN ID 및 활성 [상용 marketplace 계정이](./partner-center-portal/create-account.md)있어야 합니다.
- 파트너 센터에 전체 [비즈니스 프로필이](/partner-center/create-a-marketing-profile.md) 있는지 확인 합니다. 비즈니스 프로필은 자격을 갖춘 Microsoft 파트너로 서 비즈니스 요구 사항을 해결 하기 위해 고유한 솔루션과 전문 지식을 [찾고 있는 고객](/partner-center/referrals.md)에 게 비즈니스를 소개 하는 데 도움이 됩니다.
- **Microsoft와 공동 판매** 탭을 완료 하 고 제품을 상용 marketplace에 게시 합니다.
- 각 공동 판매 적격 지리 및 필수 자료 청구서에 대 한 영업 담당자를 제공 합니다.

**서비스 파트너**:

- _서비스 솔루션_ 유형의 제안의 경우 역량 영역에 활성 골드 역량이 있어야 합니다.
 
**Isv Business Applications**:

- Dynamics 365 고객 Engagement & PowerApps 및 Dynamics 365 재무 & Ops (Dynamics 365 business central 제외) 및 PowerApps 솔루션에는 ISV 연결 등록이 필요 합니다.

### <a name="complete-the-co-sell-with-microsoft-tab"></a>Microsoft 탭으로 공동 판매를 완료 합니다.

제품을 게시 하거나 업데이트할 때 [상업적 marketplace 제품에 대 한 공동 판매 구성](commercial-marketplace-co-sell.md)에 자세히 설명 된 대로 **Microsoft와 공동 판매** 탭에 필요한 모든 정보를 제공 합니다. 여기에는 다음 문서를 제공 하는 작업이 포함 됩니다.

- 솔루션/제안 원-호출기
- 솔루션/제안 피치 데크

이러한 문서를 만드는 데 도움이 되는 템플릿을 제공 합니다. Microsoft와 공동 판매 탭의 필수 및 선택적 정보에 대 한 자세한 내용은 [상용 marketplace 제품에 대 한 공동 판매 구성](commercial-marketplace-co-sell.md)을 참조 하세요.

### <a name="publish-your-offer-live"></a>제품을 라이브 게시

공동 판매 준비 상태를 충족 하려면 제품 또는 솔루션을 상용 marketplace 온라인 상점 중 하나 이상에 게시 해야 합니다. Azure Marketplace 또는 Microsoft AppSource. 상용 marketplace에 제품을 게시 하는 방법에 대 한 자세한 내용은 [제품 유형별 게시 가이드](publisher-guide-by-offer-type.md)를 참조 하세요. 이전에 상업적 marketplace에 제품을 게시 하지 않은 경우에는 [상업적 marketplace 계정이](./partner-center-portal/create-account.md)있는지 확인 하세요.

## <a name="requirements-for-ip-co-sell-incentivized-status"></a>IP 공동 판매 incentivized 상태에 대 한 요구 사항

Azure IP 공동 판매 incentivized 상태는 다음 제안 유형에 적용 됩니다.

- Azure 애플리케이션
- Azure 컨테이너
- Azure Virtual Machine
- IoT Edge 모듈
- SaaS(Software as a Service)

공동 판매 준비 상태를 달성 한 후에 Azure IP 공동 판매 incentivized 상태를 달성 하기 위한 세 가지 추가 요구 사항이 있습니다.

요구 사항 1-다음을 구현 합니다.

- _조직 수준_ 에서, 지난 12 개월 동안 Azure에서 사용 된 수익 임계값을 $10만 USD 이상 생성 합니다. 이는 Azure 솔루션을 조합 하 여 얻을 수 있습니다. 제품이 상업적 marketplace에서 불가능 경우 청구 되는 수익 임계값 $10만 USD를 충족 하 여이 요구 사항을 충족할 수 있습니다.

요구 사항 2-Azure 기반 솔루션에 대 한 Microsoft 기술 유효성 검사를 통과 합니다.
- 기술 유효성 검사는 제품의 인프라 중 50%가 Azure에서 반복 가능한 IP 코드를 사용 하는지 확인 해야 합니다. 상업적 marketplace의 불가능 Azure Vm 및 Azure 애플리케이션 솔루션은 기본적으로이 요구 사항을 충족 합니다.

요구 사항 3 – 참조 아키텍처 다이어그램을 제공 합니다.
- 검토를 위해 파트너 센터에서 공동 판매 문서로 참조 아키텍처 다이어그램을 업로드 합니다. 이 다이어그램을 만드는 방법에 대 한 지침은 [참조 아키텍처 다이어그램](reference-architecture-diagram.md)을 참조 하세요. 다이어그램을 업로드 하는 방법에 대 한 자세한 내용은 [상업용 marketplace 제품에 대 한 공동 판매 구성](commercial-marketplace-co-sell.md)을 참조 하세요.

## <a name="requirements-for-biz-apps-isv-connect-premium-incentive-status"></a>Biz 앱에 대 한 요구 사항 ISV 연결 프리미엄 동기 상태

이 상태는 Dynamics 365 또는 Power Apps에서 빌드된 IP 기반 솔루션, 앱 및 서비스에 적용 됩니다.

공동 판매 준비 상태 (위에 나열 됨)를 수행 하 여 Biz 앱 ISV 연결 표준 동기를 달성할 필요가 없습니다. 그러나 앱에서 공동 판매 준비 상태를 달성 한 후에는 지난 12 개월 동안의 수익 공유 및 공동 판매 결과/임계값을 기반으로 하는 Biz 앱 ISV 연결 프리미엄 동기 상태를 고려할 수 있습니다.

요구 사항- [ISV Connect](business-applications-isv-program.md) premium 계층에 활성 등록이 있어야 합니다.
