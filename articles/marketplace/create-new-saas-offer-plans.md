# <a name="how-to-create-plans-for-your-saas-offer"></a>SaaS 제품에 대 한 계획을 만드는 방법

Microsoft 상업적 marketplace를 통해 판매 되는 제품에는 하나 이상의 계획이 있어야 합니다. 동일한 제품 내에서 다양 한 옵션을 사용 하 여 다양 한 계획을 만들 수 있습니다. 이러한 계획 (Sku 라고도 함)은 버전, 수익 화 또는 서비스 계층의 측면에서 다를 수 있습니다. 요금제에 대 한 자세한 지침은 [상용 marketplace 제품에 대 한 계획 및 가격 책정](plans-pricing.md)을 참조 하세요.

> [!NOTE]
> 트랜잭션을 독립적으로 처리 하도록 선택 하면이 옵션이 표시 되지 않습니다. 대신 [SaaS 제품을 출시 하는 방법](create-new-saas-offer-marketing.md)으로 건너뛰십시오.

## <a name="create-a-plan"></a>계획 만들기

1. **계획 개요** 탭의 맨 위 근처에서 **+ 새 계획 만들기**를 선택 합니다.

1. 표시 되는 대화 상자에서 **계획 id** 상자에 고유한 계획 id를 입력 합니다. 최대 50의 소문자 영숫자, 대시 또는 밑줄을 사용 합니다. **만들기**를 선택한 후에는 계획 ID를 수정할 수 없습니다.

1. **계획 이름** 상자에이 계획에 대 한 고유한 이름을 입력 합니다. 최대 50 문자를 사용 합니다.

1. **만들기**를 선택합니다.

## <a name="define-the-plan-listing"></a>계획 목록 정의

**계획 목록** 탭에서 상업적 marketplace에 표시 하려는 계획 이름 및 설명을 정의할 수 있습니다.

1. **계획 이름** 상자에이 계획의 앞에서 제공한 이름이 표시 됩니다. 언제 든 지 변경할 수 있습니다. 이 이름은 상용 marketplace에서 제품의 소프트웨어 계획의 제목으로 표시 됩니다.

1. **계획 설명** 상자에서이 소프트웨어 계획의 고유한 정의와 제품 내 다른 요금제의 차이점을 설명 합니다. 이 설명에는 최대 500 자까지 사용할 수 있습니다.
1. 다음 탭을 계속 하기 전에 **초안 저장** 을 선택 합니다. **가격 책정 및 가용성**.

## <a name="define-markets-pricing-and-availability"></a>시장, 가격 책정 및 가용성 정의

모든 플랜은 하나 이상의 시장에서 사용할 수 있어야 합니다. **가격 책정 및 가용성** 탭에서이 요금제를 사용할 수 있는 시장, 원하는 수익 화 모델, 가격 및 청구 조건을 구성할 수 있습니다. 또한 계획이 모든 사용자에 게 표시 되도록 할지 아니면 특정 고객 에게만 표시 되도록 할지를 지정할 수 있습니다 (비공개 요금제 라고도 함).

1. **시장**에서 **시장 편집** 링크를 선택 합니다.
1. 표시 되는 대화 상자에서 요금제를 사용할 수 있도록 할 시장 위치를 선택 합니다. 최소 1 자에서 최대 141 시장을 선택 해야 합니다.

   > [!NOTE]
   > 이 대화 상자에는 Microsoft에서 판매 하 고 사용자를 대신 하 여 세금을 사용 하는 "세금 송금" 국가만 필터링 하는 옵션 및 검색 상자가 포함 되어 있습니다.

1. **저장**을 선택 하 여 대화 상자를 닫습니다.

## <a name="define-a-pricing-model"></a>가격 책정 모델 정의

가격 책정 모델을 각 계획에 연결 해야 합니다. 즉, _정액 요금_ 또는 _사용자 단위_입니다. 동일한 제안의 모든 요금제는 동일한 가격 책정 모델을 사용 해야 합니다. 예를 들어 제품에는 단일 요금제와 사용자 당 다른 요금제를 사용할 수 없습니다. 자세한 내용은 [SaaS 가격 책정 모델](plan-saas-offer.md#saas-pricing-models)을 참조 하세요.

> [!IMPORTANT]
> 제품을 게시 한 후에는 가격 책정 모델을 변경할 수 없습니다. 또한 동일한 제품의 모든 플랜은 동일한 가격 책정 모델을 공유해야 합니다.

### <a name="configure-flat-rate-pricing"></a>기본 요금 가격 책정 구성

1. **가격 책정 및 가용성** 탭의 **가격**에서 **Flat rate**를 선택 합니다.
1. **월별** 또는 **연간** 확인란을 선택 하거나 둘 다 선택 하 고 가격을 입력 합니다.

### <a name="add-a-custom-meter-dimension"></a>사용자 지정 측정기 차원 추가

이 옵션은 기본 요금 가격 책정을 선택한 경우에만 사용할 수 있습니다. 자세한 내용은 [상용 marketplace 계량 서비스를 사용 하 여 SaaS에 대 한 요금제 청구](./partner-center-portal/saas-metered-billing.md)를 참조 하세요.

1. **Marketplace 계량 서비스 차원**에서 **사용자 지정 측정기 차원 추가 (최대 30)** 링크를 선택 합니다.
1. **ID** 상자에 사용 이벤트를 내보내는 동안 변경할 수 없는 식별자 참조를 입력 합니다.
1. **표시 이름** 상자에 차원과 연결 된 표시 이름을 입력 합니다. 예를 들어 "보낸 문자 메시지"가 있습니다.
1. **측정 단위** 상자에 청구 단위에 대 한 설명을 입력 합니다. 예를 들어 "문자 단위 메시지" 또는 "100 전자 메일"을 사용할 수 있습니다.
1. 단가 **별 가격** 상자에 차원의 한 단위에 대 한 가격을 입력 합니다.
1. **기준에 포함 된 월간 수량** 상자에 월간 요금을 지불 하는 고객을 위해 매월 포함 되는 차원의 수량 (정수)을 입력 합니다. 무제한 수량을 설정 하려면 대신 확인란을 선택 합니다.
1. **연간 수량을 기준** 으로 계산 된 연간 요금을 지불 하는 고객을 위해 매월 포함 되는 차원의 수량 (정수)을 입력 합니다. 무제한 수량을 설정 하려면 대신 확인란을 선택 합니다.
1. 다른 사용자 지정 측정기 차원을 추가 하려면 **다른 차원 추가** 링크를 선택 하 고 1 ~ 7 단계를 반복 합니다.

### <a name="configure-per-user-pricing"></a>사용자 단위 가격 책정 구성

1. **가격 책정 및 가용성** 탭의 **가격**에서 **사용자별**를 선택 합니다.
2. 해당 하는 경우 **사용자 제한**에서이 계획에 대 한 사용자의 최소 및 최대 수를 지정 합니다.
3. **청구 기간**에서 월별 가격, 연간 가격 또는 둘 다를 지정 합니다.

### <a name="validate-custom-prices"></a>사용자 지정 가격의 유효성 검사

개별 시장에서 사용자 지정 가격을 설정 하려면 가격 책정 스프레드시트를 내보내고 수정한 후 가져옵니다. 이 가격의 유효성을 검사 하 고 이러한 설정을 소유 하 게 됩니다. 자세한 내용은 [사용자 지정 가격](plans-pricing.md#custom-prices)을 참조 하세요.

1. 가격 책정 데이터를 내보낼 수 있도록 먼저 가격 책정 변경 내용을 저장해야 합니다. **가격 책정 및 가용성** 탭의 아래쪽에 있는 **초안 저장**을 선택 합니다.
1. **가격 책정**에서 **가격 데이터 내보내기** 링크를 선택 합니다.
1. Microsoft Excel에서 exportedPrice.xlsx 파일을 엽니다.
1. 스프레드시트에서 가격 정보에 대 한 업데이트를 수행 하 고를 저장 합니다. CSV 파일.<br> Excel에서 편집을 사용 하도록 설정 해야 파일을 업데이트할 수 있습니다.
2. 가격 책정 **및 가용성** 탭의 **가격 책정**에서 **가격 책정 데이터 가져오기** 링크를 선택 합니다.
3. 표시 되는 대화 상자에서 **예**를 클릭 합니다.
4. 업데이트 한 exportedPrice.xlsx 파일을 선택한 다음 **열기**를 클릭 합니다.

### <a name="enable-a-free-trial"></a>무료 평가판 사용

제품의 각 요금제에 대 한 무료 평가판을 구성할 수 있습니다. 1 개월 무료 평가판을 허용 하려면이 확인란을 선택 합니다. Marketplace 계량 서비스를 사용 하는 요금제에는이 확인란을 사용할 수 없습니다. 자세한 내용은 [무료 평가판](plans-pricing.md#free-trials)을 참조 하세요.

> [!IMPORTANT]
> 불가능 제안을 무료 평가판으로 게시 한 후에는 해당 계획에 대해 사용 하지 않도록 설정할 수 없습니다. 계획을 다시 만들 필요가 없도록 제안을 게시 하기 전에이 설정이 올바른지 확인 합니다.

- **무료 평가판**에서 **1 개월 무료 평가판 허용** 확인란을 선택 합니다.

## <a name="choose-who-can-see-your-plan"></a>요금제를 볼 수 있는 사용자 선택

모든 사용자 또는 특정 대상에 게 표시 되도록 각 계획을 구성할 수 있습니다. 할당 한 각 테 넌 트 ID에 대 한 설명을 포함 하는 옵션과 함께 테 넌 트 Id를 사용 하 여 개인 계획에 대 한 액세스 권한을 부여 합니다. 을 사용 하 여 최대 10 개의 테 넌 트 Id를 수동으로 추가 하거나 최대 2만 개의 테 넌 트 Id를 추가할 수 있습니다. CSV 파일.

> [!NOTE]
> 비공개 요금제를 게시 하는 경우 나중에 공용으로 표시를 변경할 수 있습니다. 그러나 공개 계획을 게시 한 후에는 해당 표시 유형을 개인으로 변경할 수 없습니다.

### <a name="make-your-plan-public"></a>계획을 공개로 설정

1. **계획 표시 유형**에서 **공개** 상자를 선택 합니다.
1. **초안 저장**을 선택 하 고 탭의 왼쪽 **위에서 계획 개요 탭으로** 돌아가려면 **계획 개요** 를 선택 합니다.
1. 이 제품에 대 한 다른 계획을 만들려면 **계획 개요** 탭의 맨 위에 있는 **+ 새 계획 만들기**를 선택 합니다. 그런 다음 [계획 만들기](#create-a-plan) 섹션의 단계를 반복 합니다. 그렇지 않으면 [요금제 보기](#view-your-plans)로 이동 합니다.

### <a name="manually-add-tenant-ids-for-a-private-plan"></a>비공개 요금제에 대 한 테 넌 트 Id 수동 추가 

1. **계획 표시 유형**에서 **비공개 요금제** 상자를 선택 합니다.
1. 표시 되는 **테 넌 트 id** 상자에이 개인 계획에 대 한 액세스 권한을 부여 하려는 대상 그룹의 Azure AD 테 넌 트 id를 입력 합니다. 하나 이상의 테 넌 트 ID가 필요 합니다.
1. 필드 **설명** 상자에이 대상 그룹에 대 한 설명을 입력 합니다.
1. 다른 테 넌 트 ID를 추가 하려면 2 단계와 3 단계를 반복 합니다.
1. 테 넌 트 Id 추가가 완료 되 면 **초안 저장**을 선택한 다음 탭의 왼쪽 **위에서 계획 개요 탭으로** 돌아가려면 **계획 개요** 를 선택 합니다.
1. 이 제품에 대 한 다른 계획을 만들려면 **계획 개요** 탭의 맨 위에 있는 **+ 새 계획 만들기**를 선택 합니다. 그런 다음 [계획 만들기](#create-a-plan) 섹션의 단계를 반복 합니다. 그렇지 않으면 [요금제 보기](#view-your-plans)로 이동 합니다.

### <a name="use-a-csv-file-for-a-private-plan"></a>를 사용 합니다. 개인 계획에 대 한 CSV 파일

1. **계획 표시 유형**에서 **비공개 요금제** 상자를 선택 합니다.
2. **대상 그룹 (csv)** 링크를 선택 합니다.
3. 를 엽니다. CSV 파일을 추가 하 고 **id** 열에 개인 제품에 대 한 액세스 권한을 부여 하려는 Azure id를 추가 합니다.
4. 필요에 따라 **설명** 열에 각 대상 그룹에 대 한 설명을 입력 합니다.
5. Azure ID가 있는 각 행에 대해 **유형** 열에서 "TenantID"를 추가 합니다.
6. 을 저장 합니다. CSV 파일.
7. **가격 책정 및 가용성** 탭의 **계획 표시 유형**에서 **대상 그룹 (csv)** 링크를 선택 합니다.
8. 표시 되는 대화 상자에서 **예**를 선택 합니다.
9. 을 선택 합니다. CSV 파일을 선택 하 고 **열기**를 선택 합니다.
10. **초안 저장**을 선택 하 고 탭의 왼쪽 **위에서 계획 개요 탭으로** 돌아가려면 **계획 개요** 를 선택 합니다.
11. 이 제품에 대 한 다른 계획을 만들려면 **계획 개요** 탭의 맨 위에서 **+ 새 계획 만들기**를 선택 합니다. 그런 다음 [계획 만들기](#create-a-plan) 섹션의 단계를 반복 합니다. 그렇지 않고 계획 만들기를 완료 한 경우 다음 섹션인 **계획 보기**로 이동 합니다.

## <a name="view-your-plans"></a>요금제 보기

하나 이상의 계획을 만든 후 계획 **개요** 탭에서 계획 이름, 계획 ID, 가격 책정 모델, 가용성 (공용 또는 개인), 현재 게시 상태 및 사용 가능한 모든 작업을 볼 수 있습니다.

**계획 개요** 탭의 **작업** 열에서 사용할 수 있는 작업은 계획의 상태에 따라 달라 지 며 다음을 포함할 수 있습니다.

- 계획 상태가 **draft**인 경우 **작업** 열의 링크는 **초안 삭제**라고 표시 됩니다.
- 계획 상태가 **Live**인 경우 **작업** 열의 링크는 **계획 판매** 또는 **동기화 개인 대상 그룹**중 하나가 됩니다. **개인 대상 그룹 동기화** 링크는 제안에 대 한 다른 업데이트를 게시 하지 않고 개인 사용자에 대 한 변경 내용만 게시 합니다.
