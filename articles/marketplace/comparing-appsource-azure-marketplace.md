# <a name="comparing-appsource-and-azure-marketplace"></a>AppSource와 Azure Marketplace 비교  
상점들은 고유한 고객에게 요구 사항을 제공하며, 역할별로 대상 지정이 가능합니다. 사용자는 고객에 따라 적합한 솔루션 또는 서비스를 제공할 수 있습니다.  

## <a name="understanding-the-differences-between-the-storefronts"></a>상점 간의 차이점 이해  
상점을 선택하려면 먼저 자신의 제품에 대한 대상 그룹을 찾습니다. 두 대상을 모두 대상으로 지정하는 경우 한 번 게시로 두 상점에서 모두 판매할 수 있습니다.  
 
다음과 같은 각 상점의 추가적인 이점을 고려해야 합니다.   

### <a name="appsource"></a>AppSource  

| 이점 | 세부 정보 | 
|:--- |:--- | 
| ***청구 유연성*** | 평가판 체험을 프로비전합니다. 현재 상업용 게시 옵션을 제공하지 않습니다. 추가 투자 또는 변경 없이 현재 주문 및 청구 인프라를 사용할 수 있습니다. | 
| ***다른 파트너와의 용이한 연결*** | 특정 구현 시나리오에서 독립 소프트웨어 공급업체, 시스템 통합업체 및 관리되는 서비스 공급업체로 연결됩니다. 공동으로 새 고객에게 판매할 수 있습니다. | 
| ***Automation*** | 특정 구현 시나리오에서 독립 소프트웨어 공급업체, 시스템 통합업체 및 관리되는 서비스 공급업체로 연결됩니다. 공동으로 새 고객에게 판매할 수 있습니다. |  
| ***다중 클라우드 유형*** | 현재 Azure Stack, Azure Government 또는 지역별 클라우드를 지원하지 않습니다. | 
| ***고객의 상황에 맞는 제품 제공*** | Microsoft 제품(예: Dynamics 365, Power BI 및 Office 365)에 대한 인앱 환경을 통해 더 많은 고객에게 연결합니다. |  

### <a name="azure-marketplace"></a>Azure Marketplace  

| 이점 | 세부 정보 | 
|:--- |:--- | 
| ***청구 유연성*** | VM에 대한 종량제 청구 옵션에서는 Microsoft EA(기업 계약) 또는 웹 직접 판매 모델을 사용합니다. 가격 책정 옵션은 제품이 영구적으로 무료인 무료 계층 구독을 포함합니다. 또한 가격 책정 옵션은 지금 사용해 보기 구독을 포함합니다. 지금 사용해 보기 구독은 제한된 홍보 기간 동안에는 무료이고 홍보 기간이 지나면 VM이 유료 구독으로 전환됩니다. BYOL 활성화도 지원되는 옵션 중 하나입니다.<br />청구 옵션 및 다음 VM 제품의 경우 프로비전된 Azure 리소스는 모두 고객에게 직접 요금이 청구됩니다.<ul> <li>Azure 앱: 관리되는 앱</li> <li>Azure 앱: 솔루션 템플릿</li> </ul> | 
| ***다른 파트너와의 용이한 연결*** | 현재는 서비스 공급자 또는 배달 파트너를 자신의 제품에 연결할 수 없습니다. 이 기능은 2018년에 시작될 예정입니다. | 
| ***Automation*** | 현재는 서비스 공급자 또는 배달 파트너를 자신의 제품에 연결할 수 없습니다. 추가 기능 프로비전과 함께 자동화된 SaaS를 활용합니다. Azure 앱: 솔루션 템플릿을 사용하여 SaaS 기반 데이터 수집 및 배포 시나리오를 자동화합니다. |  
| ***다중 클라우드 유형*** | 다음과 같은 형식을 사용하여 공용 클라우드 및 온-프레미스 솔루션을 모두 게시합니다.<ul> <li>Azure Stack</li> <li>Azure Government</li> <li>중국 및 독일을 포함하는 지역별 클라우드</li> </ul> | 
| ***고객의 상황에 맞는 제품 제공*** | 상황별 검색을 위해 Azure Portal 내 환경(가상 머신 및 Azure 앱: 솔루션 템플릿)에서 사용자 솔루션을 사용하도록 할 수 있습니다. |
