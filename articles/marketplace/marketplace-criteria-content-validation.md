# <a name="azure-marketplace-listing-guidelines"></a>Azure Marketplace 목록 가이드라인

이 문서에는 Azure Marketplace에 나열 될 새로운 제품 및 서비스에 대 한 요구 사항 및 지침이 포함 되어 있습니다.

모든 제안은 "모든 제품에 대 한 요구 사항 나열" 섹션에 나열 된 요구 사항을 충족 해야 합니다.

다음을 비롯 한 특정 목록 형식에 대 한 추가 요구 사항 및 검사 목록이 제공 됩니다. 
* [평가판 제품 요구 사항](#trial-offer-requirements)
* [SaaS 앱 요구 사항](#saas-app-requirements)
* [컨테이너 제품 요구 사항](#container-offer-requirements)
* [컨설팅 제품 요구 사항](#consulting-offer-requirements)

## <a name="listing-requirements-for-all-offers"></a>모든 제품에 대 한 요구 사항 나열

|  | 목록 요소 | 기본 요구 사항 | 최적 요구 사항 |
|:--- |:--- |:--- |:--- |
| 1 | 잠재 대상 고객 | 선행 대상이 구성 되어 있습니다.| 단일 상업적 파트너 (OCP) 카탈로그에는 파트너 솔루션 탭에도 나열 된 잠재 고객 대상 CRM 정보가 있습니다. | 
| 2 | 제품 이름 | 솔루션 제품에 대해 간략하게 설명 합니다. 파트너의 웹 사이트에 있는 솔루션의 온라인 프로 모션을 찾습니다. | 키 검색 단어를 포함 합니다. |
| 3 | 로고 | 로고가 올바르게 표시 됩니다. | 로고가 올바르게 표시 되 고 Azure Portal에 있는 "주인공 이미지"를 포함 하는 "주인공 이미지"를 포함 합니다. |
| 4 |  제품 설명 | <ul><li> 2-3 단락을 포함 합니다.</li><li>솔루션 제공은 한눈에 쉽게 이해할 수 있습니다.</li><li>철자 및 문법 오류를 제외 합니다.</li><li>는 포괄적 이며 대상 대상 그룹, 사용자 유형 및 가치 (가치 제안)를 캡처합니다.</li><li>이해 하기 쉬운 짧은 문장을 포함 하는 단락 설명 폼에 있습니다.</li></ul> | <ul><li> 대상 업계에 대해서는 개요가 표시 됩니다 (관련 된 경우).</li><li>좋은 스타일 서식 지정. 각 단락 제목에는 다음에 나오는 내용을 요약 하 고 글머리 기호를 사용 하 여 주요 혜택을 강조 하는 단일 문장이 나 구가 있습니다. 이 목표는 독자가 쉽게 볼 수 있는 형식으로 제품을 이해 하 고 긴 단락을 읽을 필요가 없다는 것입니다.</li><li>각 단락 사이에 간격이 있습니다. 차량 브로슈어 처럼 읽혀집니다. 즉, 포괄적 이며 기술 전문 용어 없이 제품을 간단히 설명 합니다. | 
| 5 | 범주 및 산업 | <ul><li>범주는 솔루션 제품의 기능과 일치 합니다.</li><li>솔루션이 맞지 않는 범주로 확장 하지 마십시오.</li><li>최적의 산업 또는 모든 산업 (최적의 산업이 아닌 경우)을 선택 합니다. | <ul><li>최대 3 개 범주.</li><li>최대 3 개 산업. |
| 6 | 이미지 | <ul><li>이미지가 필요 하지 않지만 제공 되는 경우 고해상도 이미지로 표시 되어야 합니다.</li><li>이미지 요구 사항은 Cloud 파트너 포털 (CPP)에 나열 되어 있습니다.</li><li>스크린 샷에 포함 된 텍스트는 명확 하 고 이미지는 명확 하지 않습니다. | 솔루션 제공은 한눈에 쉽게 이해할 수 있습니다. |
| 7 | 동영상 | <ul><li>비디오가 필요 하지 않지만 제공 되는 경우 오류 없이 재생 해야 합니다.</li><li>제공 된 경우 마이그레이션 솔루션을 시연 하지 *않는 한* 경쟁 업체 회사를 참조 하지 않을 수 있습니다. |<ul><li>이상적인 길이는 3 분입니다.</li><li>솔루션 제품은 비디오 콘텐츠를 통해 쉽게 이해할 수 있습니다.</li><li>솔루션 기능 데모를 제공 합니다. |
| 8 | 상태 나열 (작업 호출) | <ul><li>다음 형식 중 하나로 레이블을 지정 해야 합니다. <ul><li>*나에 게 연락*</li><li>*평가판 가져오기**이제*/평가판 시험*드라이브* *시작*//</li><li>*지금 구입 하세요*.*Get It Now* /</li></ul></ul> | 고객은 다음 단계를 쉽게 파악할 수 있습니다. <ol><li>평가판을 사용해 보세요.</li><li>지금 구입 하세요.</li><li>개념 증명 (POC), 평가 또는 브리핑에 맞게 정렬 하려면 전자 메일 또는 전화 번호를 통해 문의 하세요.</li></ol> | 
| 9 | 솔루션 가격 책정 | 솔루션 가격 책정 탭/세부 정보가 있어야 하 고 가격은 파트너 솔루션 제품의 현지 통화에 있어야 합니다. | 고객 옵션을 제공 하기 위해 계층 가격 책정을 사용 하 여 여러 청구 옵션을 사용할 수 있습니다. |    
| 10 | 자세한 정보 | 아래쪽에 있는 링크 (설명에서는 왼쪽에 있는 Azure Marketplace 링크가 아님)는 솔루션에 대 한 자세한 정보를 제공 하 고 공개적으로 제공 되며 올바르게 표시 됩니다. |    파트너 홈 페이지 뿐만 아니라 특정 항목 (예: 파트너 사이트의 사양 페이지)에 대 한 링크입니다. | 
| 11 | 솔루션 지원 및 도움말 | 다음 중 하나 이상에 대한 링크: <ul><li>전화 번호</li><li>전자 메일 지원</li><li>채팅 에이전트</li><li>커뮤니티 포럼 |<ul><li>모든 지원 방법이 나열 됩니다.</li><li>유료 지원은 *평가판* 또는 *시험* 사용 기간 동안 무료로 제공 됩니다. | 
| 12 | 법적 정보 | 정책 또는 용어는 공용 URL을 통해 사용할 수 있습니다. |  |

| 3 | 시험 사용 | *예* 또는 *아니요* 를 선택 합니다. | 고객은 다음 단계를 쉽게 파악할 수 있습니다. <ol><li>평가판을 사용해 보세요.</li><li>지금 구입 하세요.</li><li>POC, 평가 또는 브리핑에 맞게 정렬 하려면 전자 메일 또는 전화 번호를 통해 문의 하세요.</li></ol> |
| 4 | Storefront 세부 정보: 제안 요약 | 앱의 검색 페이지에 표시 되 고 최대 100 자까지 표시 됩니다. | |
