# <a name="update-an-existing-offer-in-the-commercial-marketplace"></a>상업용 Marketplace에서 기존 제품 업데이트

파트너 센터의 [상업용 마켓플레이스 포털의](https://partner.microsoft.com/dashboard/commercial-marketplace/offers) **개요** 탭에서 기존 오퍼를 볼 수 있습니다.

현재 마켓플레이스에 있는 기존 오퍼를 업데이트하려면 다음 을 수행하세요.

1. 업데이트할 제품의 이름을 선택합니다. 오퍼의 상태는 **라이브,** 게시 **진행**중, **초안,** **주의 필요,** 또는 **마켓플레이스에서 판매되지 않음(이전에** 오퍼 판매를 중단하도록 선택한 경우)으로 나열될 수 있습니다. 선택되면 해당 **오퍼의 제안 개요** 페이지가 열립니다.
2. 업데이트할 영역에 대한 개요 카드 또는 왼쪽 탐색의 메뉴 항목에서 **업데이트를** 선택합니다. **오퍼 설정,** **속성,** **제안 목록,** **미리 보기,** **기술 구성,** **계획 개요**또는 **테스트 드라이브를**업데이트할 수 있습니다. 
3. 변경한 다음 **저장을**선택합니다. 모든 변경이 완료될 때까지 이 프로세스를 반복합니다.

## <a name="review-and-publish-an-updated-offer"></a>업데이트된 오퍼 검토 및 게시

업데이트된 쿠폰을 게시할 준비가 되면 모든 페이지에서 **게시를** 선택합니다. **검토 및 게시** 페이지가 열립니다. 이 페이지에서는

- 업데이트한 오퍼의 섹션에 대한 완료 상태를 확인합니다. 
    - **게시되지 않은 변경 내용**: 섹션이 업데이트되었으며 완료되었습니다. 필요한 모든 데이터가 제공되었으며 업데이트에 오류가 없습니다.
    - **불완전**: 섹션에 대한 업데이트로 인해 수정이 필요하거나 더 많은 정보가 제공되어야 하는 오류가 발생했습니다.
- 테스트가 원활하게 진행되도록 인증 테스트 팀에 추가 정보를 제공합니다.
- 제출을 선택하여 게시할 업데이트된 제안을 **제출합니다.**  업데이트된 제품의 미리 보기 버전이 검토및 사인오프할 수 있는 경우 이메일을 보내드립니다.

> [!IMPORTANT]
> 쿠폰이 사용 가능하면 쿠폰 미리 보기를 검토하고 **Go-live를** 선택하여 업데이트된 쿠폰을 의도한 대상(공개 또는 비공개)에 게시해야 합니다.

## <a name="update-a-plan-within-an-existing-offer"></a>기존 오퍼 내에서 요금제 업데이트

이미 게시한 기존 오퍼 내에서 플랜을 변경하려면 다음 단계

- 기존 **오퍼에** 대해 쿠폰 개요 페이지가 열려 있는 경우 변경할 플랜을 선택합니다. **계획 개요** 목록에서 계획에 액세스할 수 없는 경우 모든 **계획 보기를**선택합니다.
- 요금제 **이름,** 가격 책정 모델 또는 **가용성을**선택할 수 **있습니다.** *현재 플랜은 영어(미국)에서만 사용할 수 있습니다.*
- 요금제 이름, 설명, 가격 또는 잠재고객 가용성을 변경한 후 **저장을** 선택합니다. 
- 업데이트를 게시할 준비가 되면 **게시를** 선택합니다. **[검토 및 게시](#review-and-publish-an-updated-offer)** 페이지가 열리고 업데이트완료 상태를 제공합니다.

## <a name="compare-changes-to-marketplace-offers"></a>마켓플레이스 오퍼에 대한 변경 사항 비교

**[비교]** 를 사용하여 라이브로 만들기 전에 [게시된](#compare-changes-to-published-offer) 오퍼 또는 [미리 보기](#compare-changes-to-a-preview-offer) 로 변경한 내용을 감사할 수 있습니다.

>[!NOTE]
> 게시된 오퍼는 미리 보기 또는 라이브 상태로 성공적으로 게시된 오퍼입니다.

일반 감사 정보는 아래를 참조하십시오.

- 편집 프로세스 중 언제든지 **비교를** 사용할 수 있습니다.
- **비교** 페이지에서 필드를 선택하여 수정할 값으로 이동합니다.
- 업데이트를 게시할 준비가 되면 검토 **및 게시를**선택합니다.
- 업데이트되지 않은 모든 필드의 값을 보려면 모든 필드 필터를 **선택합니다.** **수정된 필드를**선택한 다음 아래 필터 중 하나를 선택하여 이러한 필드 내에서 필터를 수정할 수 있습니다.
    - **제거된 값** 필터는 게시한 필드를 표시하고 이제 완전히 제거합니다.
    - **추가된 값** 필터는 원래 게시하지 않았고 이제 추가중인 필드를 표시합니다.
    - **편집된 값** 필터는 게시된 필드를 표시하지만 이제 내용을 업데이트했습니다.

      >[!NOTE]
      > 이러한 필터 중 하나를 사용할 수 없는 경우 해당 형식을 업데이트하지 않았음을 나타냅니다.

- 업데이트되지 않은 값만 보려면 **변경되지 않은 필드** 필터를 선택합니다. 게시된 버전과 초안 버전에 대해 표시된 필드 값은 동일합니다.

  ![게시된 업데이트 또는 미리 보기 오퍼와 비교하기 위한 필터](./media/compare-changes-marketplace.png)

>[!NOTE]
> 다음 페이지는 현재 **비교를**지원하지 않습니다.
>- CSP 리셀러 고객
>- 테스트 드라이브 기술 구성
>- 테스트 드라이브 마켓플레이스 목록
>- 공동 판매
>- 추가 파일

### <a name="compare-changes-to-published-offer"></a>게시된 오퍼와 변경 사항 비교

게시된 오퍼의 변경 내용을 비교하려면 아래 지침을 따르십시오.

1. 페이지의 명령 모음에서 **비교를** 선택합니다. **비교** 페이지에서는 이 오퍼의 저장된 변경 내용과 게시된 마켓플레이스 오퍼의 버전을 나란히 제공합니다.
2. 오퍼의 특정 페이지에서 **비교에** 액세스하면 기본값에 해당 페이지의 변경 내용만 표시됩니다.
    - 모든 페이지의 변경 내용을 비교하려면 페이지 선택에서 페이지를 **변경하여 비교할 수 있습니다.**  
    - 모든 페이지에서 변경 내용을 오퍼와 비교하려면 **모든 페이지를**선택합니다.

기본적으로 **비교는** 라이브 마켓플레이스 오퍼에 대한 새로운 변경 사항을 대조합니다.

변경 사항이 적용될 수 있도록 업데이트한 후 쿠폰을 다시 게시해야 합니다.

### <a name="compare-changes-to-a-preview-offer"></a>변경 내용과 미리 보기 제안 비교

라이브 가 아닌 미리 보기에서 변경사항이 있는 경우 새로운 변경 사항을 미리 보기 마켓플레이스 오퍼와 비교할 수 있습니다.

아래 지침에 따라 새로운 변경 사항을 미리 보기 마켓플레이스 오퍼와 비교하세요.

1. 페이지의 명령 모음에서 **비교를** 선택합니다.
2. 드롭다운 **을** 선택하고 라이브 **오퍼에서** **미리 보기 오퍼로**변경합니다.
3. **비교** 페이지에서는 변경 내용을 표시하는 나란히 버전을 제공합니다.

>[!NOTE]
>쿠폰이 아직 라이브로 제공되지 않았지만 미리 보기로 게시한 경우 라이브 오퍼와 비교할 수 있는 옵션이 없습니다.

변경 사항이 적용될 수 있도록 업데이트한 후 쿠폰을 다시 게시해야 합니다.

## <a name="delete-a-draft-offer"></a>초안 오퍼 삭제

오퍼 개요 페이지에서 초안 삭제를 선택하여 초안 오퍼(게시되지 않은 **제안)를 삭제할** 수 **있습니다.** 이전에 쿠폰을 게시한 경우에는 이 옵션을 사용할 수 없습니다.

초안 제안을 삭제하려는 경우 파트너 센터에서 더 이상 오퍼를 보거나 액세스할 수 없으며 **모든 혜택** 페이지가 열립니다.

## <a name="delete-a-draft-plan"></a>초안 계획 삭제

게시되지 않은 계획을 삭제하려면 **계획 개요** 페이지에서 **초안 삭제를** 선택합니다. 이전에 쿠폰을 게시한 경우에는 이 옵션을 사용할 수 없습니다.

임시 플랜을 삭제하려는 경우 파트너 센터에서 플랜이 더 이상 표시되거나 액세스할 수 없게 됩니다.

## <a name="cancel-publishing"></a>게시 취소

게시 진행 중인 상태로 쿠폰을 취소하려면 다음 을 **수행하십시오.**

- 쿠폰 이름을 선택하여 **제안 개요** 페이지를 엽니다.
- 페이지의 오른쪽 상단 모서리에서 **게시 취소를** 선택합니다.
- 오퍼가 게시되는 것을 중지할지 확인합니다.

나중에 오퍼를 게시하려면 게시 프로세스를 다시 시작해야 합니다.

> [!NOTE]
> 쿠폰이 아직 게시자 사인오프 단계로 진행되지 않은 경우에만 쿠폰이 게시되지 않도록 할 수 있습니다. 라이브 **로이동을** 선택하면 더 이상 게시를 취소할 수 없습니다.

## <a name="stop-selling-an-offer-or-plan"></a>오퍼 또는 플랜 판매 중지

이미 쿠폰을 게시한 후 쿠폰 판매를 중단하려면 **쿠폰 개요** 페이지에서 **판매 중지를** 선택합니다. 플랜 판매를 중단하려면 **플랜 개요** 페이지에서 **판매 중지를** 선택합니다. (플랜 판매를 중단하는 옵션은 쿠폰에 둘 이상의 플랜이 있는 경우에만 사용할 수 있습니다. 오퍼 내의 다른 플랜에 영향을 주지 않고 하나의 플랜 판매를 중단할 수 있습니다.

오퍼 또는 플랜 판매를 중단하려는 것으로 확인되면 몇 시간 내에 마켓플레이스에 더 이상 표시되지 않으며 새로운 고객이 다운로드할 수 없습니다.

이전에 오퍼 또는 플랜을 구입한 고객은 계속 사용할 수 있습니다. 다시 다운로드할 수 있지만 나중에 오퍼 또는 플랜을 업데이트하고 다시 게시하면 업데이트를 받지 않습니다. 

오퍼/플랜 판매 중지 요청이 완료된 후에도 파트너 센터의 상업용 마켓플레이스 포털에서 계속 볼 수 있습니다. 초안을 삭제하려면 **제안 개요** 또는 **계획 개요** 페이지에서 **초안 삭제를** 선택합니다. 

이 제안을 나열하거나 판매하거나 다시 계획하기로 결정한 경우 지침에 따라 [기존 제품을 업데이트하십시오.](#update-an-existing-offer-in-the-commercial-marketplace) 변경 한 후 제안을 **게시하거나** 계획을 다시 계획해야한다는 것을 잊지 마십시오.

## <a name="remove-offers-from-existing-customers"></a>기존 고객의 오퍼 제거

기존 고객의 오퍼를 제거하려면 [지원 요청을 기록합니다.](https://support.microsoft.com/supportforbusiness/productselection?sapId=48734891-ee9a-5d77-bf29-82bf8d8111ff) **문제 유형** 목록에서 특정 쿠폰 유형을 선택한 다음 **카테고리** 목록에서 게시된 **쿠폰 제거를** 선택합니다. 요청을 제출합니다. 지원 팀이 제안 제거 프로세스를 안내합니다.

## <a name="sync-private-plan-audiences"></a>비공개 요금제 대상 동기화

비공개 제한 대상만 사용할 수 있도록 구성된 플랜이 하나 이상 포함된 경우 쿠폰에 다른 변경 사항을 게시하지 않고 해당 비공개 요금제에 액세스할 수 있는 잠재고객만 업데이트할 수 있습니다. 

계획에 대한 비공개 대상을 업데이트하고 동기화하려면 다음을 수행합니다.

- + **ID 추가** 또는 **고객 가져오기(csv)** 버튼을 사용하여 하나 이상의 비공개 요금제에서 대상을 수정한 다음 변경 내용을 저장합니다.
- 계획 개요 페이지에서 **비공개 잠재고객 동기화를** **선택합니다.**

**동기화 비공개 대상은** 초안 오퍼에 대한 다른 업데이트를 게시하지 않고 개인 대상에게만 변경 내용을 게시합니다.
