# <a name="update-an-existing-offer-in-the-commercial-marketplace"></a>상용 Marketplace에서 기존 제품 업데이트

기존 제품을 볼 수 있습니다 합니다 **개요** 탭의 [상용 Marketplace 포털](https://partner.microsoft.com/dashboard/commercial-marketplace/offers) 파트너 센터에서.

Marketplace에서 현재 유지 되는 기존 제품 업데이트: 

1. 업데이트 하려는 제품의 이름을 선택 합니다. 제품의 상태를 표시 될 수도 있습니다 **Live**, **진행 중인 게시**를 **초안**, **주의가 필요한**, 또는 **하지 marketplace에서 판매** (하는 경우 이전에 선택한 제품을 판매 중지) 합니다. 선택 하면 합니다 **개요를 제공** 페이지에 제공 하는 열립니다.
2. 선택 **업데이트** 개요 카드 또는 왼쪽 탐색에서 업데이트 하려는 영역에 대 한 메뉴 항목입니다. 업데이트 하려는 경우는 **설치 프로그램을 제공**, **속성**를 **목록을 제공**, **미리 보기**, **기술 구성** , **계획 개요**, 또는 **시험**합니다. 
3. 내용을 변경 하 고 선택한 **저장할**합니다. 모든 변경 내용을 완료 될 때까지이 프로세스를 반복 합니다.

## <a name="review-and-publish-an-updated-offer"></a>검토 및 업데이트 된 제품을 게시

업데이트 된 제품을 게시할 준비가 때 선택할 **게시** 페이지에서. 합니다 **검토 및 게시** 페이지가 열립니다. 이 페이지에서 다음을 수행할 수 있습니다.


- 제품 업데이트 부분에 대 한 완료 상태를 보려면 
    - **변경 내용이 게시 되지 않은**: 섹션 업데이트 되었습니다 하 고 완료 됩니다. 필요한 모든 데이터를 제공한 및 도입 업데이트에서 없음 오류가 발생 했습니다.
    - **불완전**: 섹션에 대 한 업데이트를 수정 해야 하는 오류를 도입 하거나 제공 될 더 많은 정보가 필요 합니다.
- 인증 테스트 팀에 원활 하 게 이동 테스트 확인 하도록 요청 하는 추가 정보를 제공 합니다.
- 선택 하 여 게시에 대 한 업데이트 된 제품을 제출 **제출**합니다.  메일을 보내겠습니다 경우 업데이트 된 제품의 미리 보기 버전을 검토 하 고 승인을 사용할 수 있습니다.

**중요**: 사용 가능 하 고 선택 되 면 제품 미리 보기를 검토 해야 합니다 **go-live** 사용자 (public 또는 private)에 업데이트 된 제품을 게시 하려면.

## <a name="update-a-plan-within-an-existing-offer"></a>기존 제품 내에서 계획 업데이트

이미 게시 하는 기존 제품 내에서 계획을 변경 합니다.

- 사용 하 여 합니다 **개요를 제공** 기존 제품에 대 한 페이지를 변경 하려면 원하는 계획을 선택 합니다. 계획에서 액세스할 수 없는 경우는 **계획 개요** 목록에서 **모든 계획을 확인 하세요**합니다.
- 계획을 선택 하도록 선택할 수 있습니다 **이름**하십시오 **가격 책정 모델**, 또는 **가용성**합니다. *계획은 영어 (미국)에서 사용할 수 있는 현재*합니다.
- 선택 **저장할** 계획 이름, 설명, 가격, 또는 대상 가용성에 필요한 내용을 변경한 후입니다. 
- 선택 **게시** 업데이트를 게시할 준비가 되 면 합니다. 합니다 **[검토 하 고 게시](#review-and-publish-an-updated-offer)** 페이지가 열리고 업데이트에 대 한 완료 상태를 제공 합니다. 

## <a name="delete-a-draft-offer"></a>초안 제품 삭제

(게시 되지 않은 하나) 초안 제품을 선택 하 여 삭제할 수 있습니다 **초안 삭제** 에서 합니다 **개요를 제공** 페이지입니다. 제품을 이전에 게시 한 경우이 옵션을 사용할 수 없습니다.

초안 제품을 삭제할 것인지 확인 하 고 나면 제품 더 이상 표시 되지 않거나 액세스할 수 있는 파트너 센터에서 및 **모든 제품** 페이지가 열립니다.

## <a name="delete-a-draft-plan"></a>초안 계획 삭제

게시 되지 않은 계획을 삭제 하려면 선택 **초안 삭제** 에서 합니다 **계획 개요** 페이지입니다. 이 옵션은 제품을 이전에 게시 한 경우를 사용할 수 없습니다.

계획 초안을 삭제할 것임을 확인 한 후 계획을 더 이상 표시 되지 않거나 액세스할 수 있는 파트너 센터에서.

## <a name="cancel-publishing"></a>게시 취소

사용 하 여 제품을 취소 하는 **진행에서 게시** 상태:

- 제품 이름 열을 선택 합니다 **개요를 제공** 페이지입니다. 
- 선택 **게시 취소** 페이지의 오른쪽 위 모서리에서.
- 제품 게시 되지 않도록 중지 되도록 확인 합니다. 

나중에 제품을 게시 하려는 경우에 게시 프로세스를 다시 시작 해야 합니다.

> [!NOTE]
> 제품이 게시자 로그 오프 단계를 아직 처리 되지 않았습니다 하는 경우에 게시 되는 제품을 중지할 수 있습니다. 선택한 후 **라이브로 전환** 취소할 수는 없습니다 더 이상 게시 합니다.

## <a name="stop-selling-an-offer-or-plan"></a>판매 제품 또는 계획을 중지 합니다.

이미 게시 한 후 제품을 판매를 중지 하려면 선택 **판매 중지** 에서 합니다 **개요를 제공** 페이지입니다. 계획을 판매를 중지 하려면 선택 **판매 중지** 에서 합니다 **계획 개요** 페이지입니다. (중지 계획을 판매 하는 옵션은 제품에 둘 이상의 계획 해야 하는 경우 사용할 수 있습니다만. 중지 하도록 선택할 수 제품 내에서 다른 계획에 영향을 주지 않고 한 계획을 판매 합니다.)

계획을 제품 판매를 중지 하려는 확인 하 고 나면 몇 시간 이내 더 이상 표시 됩니다는 marketplace에서 및 신규 고객에서 다운로드할 수 없습니다. 

이전에 제품 또는 계획을 얻은 고객이 계속 사용할 수 있습니다. 다시 다운로드할 수 있지만 업데이트 및 다시 제품을 게시 하거나 나중에 계획 하는 경우 업데이트를 받지 못합니다. 

요금제/판매 중지 요청이 완료 된 후 상용 Marketplace 파트너 센터 포털에 여전히 볼 수 있습니다. 초안을 삭제 하려면 선택 **초안 삭제** 에서 합니다 **개요를 제공** 또는 **계획 개요** 페이지입니다. 

이 제품을 판매 하거나 계획을 다시 나열 하려는 경우 지침에 따라 [기존 제품 업데이트](#update-an-existing-offer-in-the-commercial-marketplace)합니다. 해야 하는 데 잊지 **게시** 제품 또는 변경한 후 다시 계획 합니다.

## <a name="remove-offers-from-existing-customers"></a>기존 고객의 제품 제거

기존 고객의 제품을 제거할 [지원 요청을 기록](https://support.microsoft.com/supportforbusiness/productselection?sapId=48734891-ee9a-5d77-bf29-82bf8d8111ff)합니다. 특정 제품 유형을 선택 합니다 **문제 유형** 나열 하 고 선택한 **게시 된 제품을 제거** 에 **범주** 목록입니다. 요청을 제출합니다. 지원 팀은 제품 제거 프로세스를 안내 합니다.

## <a name="sync-private-plan-audiences"></a>동기화 비공개 계획 대상

제품 개인 제한 된 대상에만 가능 하도록 구성 된 하나 이상의 계획에 포함 된 경우에 제품으로 다른 변경 내용을 게시 하지 않고 해당 비공개 계획에 액세스할 수 있는 사용자만을 업데이트할 수입니다. 

업데이트 하 여 계획의 개인 사용자를 동기화 합니다.

- 사용 하 여 하나 이상의 개인 계획에서 대상 그룹을 수정 합니다 + **추가 ID** 또는 **고객 (csv)를 가져옵니다** 단추 및 다음 변경 내용을 저장 합니다.
- 선택 **개인 사용자를 동기화** 에서 합니다 **계획 개요** 페이지입니다.

**개인 사용자를 동기화** 초안 제품으로 설정한 다른 업데이트를 게시 하지 않고 개인 청중에 변경 내용만 게시 됩니다.
