# <a name="how-to-create-a-commercial-marketplace-account-in-partner-center"></a>파트너 센터에서 상용 Marketplace 계정을 만드는 방법

제품을 게시 하려면 [Azure Marketplace](https://azuremarketplace.microsoft.com/) 또는 [AppSource](https://appsource.microsoft.com/) 를 통해 합니다 [상용 Marketplace 포털](https://partner.microsoft.com/dashboard/commercial-marketplace/offers), 파트너 센터 계정을 만들려면 해야 합니다.  

이 문서에서는 파트너 센터 계정을 만들어야 하는 방법을 다룰 것 방법을 비롯 하 여: 

- [파트너 센터 등록 페이지를 사용 하 여 등록](#register-for-an-account-using-the-enrollment-page)
- [작업 계정으로 로그인](#sign-in-with-a-work-account)
- [사용 약관에 동의](#agree-to-terms-and-conditions) 
- [게시자 프로필을 제공 합니다.](#provide-your-publisher-profile)

계정 설치한 경우 합니다 [클라우드 파트너 포털 (CPP)](https://cloudpartner.azure.com) 파트너 센터를 통해 마이그레이션된, 새 계정을 만들 필요가 없습니다. 참조 [게시자 CPP에서 마이그레이션된](#publishers-migrated-from-cpp)합니다. 

## <a name="prerequisites"></a>필수 구성 요소

파트너 센터 계정을 만들려면 했는지 확인 합니다.

- 회사를 대신 하 여 법적 계약서에 서명할 인증 기관
- 회사의 법적 회사 이름, 주소 및 기본 연락처 (가능 하면)

계정 만들기 프로세스 중에이 정보를 확인 됩니다.

## <a name="register-for-an-account-using-the-enrollment-page"></a>등록 페이지를 사용 하는 계정에 대 한 등록 

방문 합니다 [ **Microsoft 파트너 센터에 오신 것을 환영** ](https://partner.microsoft.com/dashboard/account/v3/enrollment/introduction/azureisv) 등록 페이지 및 등록 정보를 찾을 수 없으면 검토 합니다.

### <a name="sign-in-with-a-work-account"></a>작업 계정으로 로그인

파트너 센터 등록의 일환으로, 새 파트너 센터 계정에 회사의 작업 전자 메일 계정 도메인을 연결 하는 요청 합니다. 이러한 계정에 연결 하 여 회사 직원 수에 로그인 파트너 센터 작업 계정 사용자 이름과 암호를 사용 하 여 합니다.

여부를 확인 하는 방법에 대 한 도움말 회사에 이미 회사 계정, 새 작업 계정을 만드는 방법, 사용 하 여 파트너 센터를 방문 하는 여러 회사 계정을 설정 하는 방법 [회사 계정 및 파트너 센터 작업](./company-work-accounts.md)합니다. 

### <a name="agree-to-terms-and-conditions"></a>사용 약관에 동의

사용 약관을 포함 하 여 두 집합에 동의 해야 합니다는 [Microsoft 온라인 정기 가입 계약](https://go.microsoft.com/fwlink/?LinkId=870457) 하며 [Microsoft Marketplace 판매자 계약](https://go.microsoft.com/fwlink/?linkid=843476)합니다.


### <a name="provide-your-publisher-profile"></a>게시자 프로필을 제공 합니다.

게시자 프로필에 회사 이름 및 MPN ID 포함 아직 수행 하는 경우 조인 해야 합니다는 [Microsoft Partner Network](https://partner.microsoft.com/commercial)합니다. Microsoft 파트너 네트워크에 가입 하면 제공 됩니다 MPN ID를 사용 하 여 

지금은 게시자 ID를 선언 해야 합니다. 게시자 ID는 Azure Marketplace 및 AppSource에서 귀사를 고유 하 게 식별 합니다. 

게시자 프로필 정보를 확인 한 후에 사용 약관에 동의 하 고 선택 하 여 파트너 센터 계정에 만들 수 있습니다 **수락 하 고 계속**합니다. *이러한 조건에 동의 하기 위해 회사를 대신 하 여 작업을 수행할 권한이 있어야 합니다.*

파트너 센터 등록을 완료 했으므로 했습니다 및 이동 합니다 [상용 Marketplace 개요](./commercial-marketplace-overview.md) 페이지.


## <a name="publishers-migrated-from-cpp"></a>게시자는 CPP에서 마이그레이션

계정에서 마이그레이션된 경우 합니다 [클라우드 파트너 포털 (CPP)](https://cloudpartner.azure.com), 새 파트너 센터를 만들 필요가 없습니다 계정을 있지만 배너 알림 및 전자 메일을 통해 새 파트너 센터 계정에 대 한 사용자 지정된 링크를 받게 됩니다 기존 CPP 계정에 로그인 한 후 표시 합니다.

방문 하 여 계정에 반환할 수 있습니다이 사용자 지정된 링크를 방문 하 여 새 파트너 센터 계정에 설정한 후 합니다 [상용 Marketplace 대시보드](https://partner.microsoft.com/dashboard/commercial-marketplace/) 파트너 센터에서.

게시 계약 및 회사 프로필 정보를 함께 새 파트너 센터 계정에 마이그레이션할 수는 이전에 계정 지급 프로필 정보, 사용자 계정을 만들고 권한을 설정 하 고 활성 CPP 계정과 연결 된 제공 합니다. 

마이그레이션되면 계정의 CPP에서 파트너 센터에 파트너 센터 계정을 계정 업데이트, 사용자 관리, 권한 및 지급 관리에 사용할 마스터 계정이 됩니다. 이러한 계정 업데이트 CPP 포털은 결국 사용 중단 될 때까지 읽기 전용 CPP 계정으로 다시 자동으로 동기화 됩니다.
