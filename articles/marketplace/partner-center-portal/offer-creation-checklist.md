# <a name="saas-offer-creation-checklist"></a>SaaS 제품 만들기 검사 목록

SaaS 오퍼 생성 프로세스는 여러 페이지를 안내합니다. 각 페이지에 제공할 수 있는 세부 정보와 각 항목에 대해 자세히 알아볼 수 있는 링크가 있습니다.

제공하거나 지정해야 하는 항목은 아래에 명시되어 있습니다. 일부 영역은 선택 사항이거나 기본값이 제공되어 원하는 대로 변경할 수 있습니다. 여기에 나열된 순서대로 이러한 섹션에서 작업할 필요가 없습니다.

| **항목**    | **목적**  |
| :---------- | :-------------------|
| [**새로운 오퍼 모달**](#new-offer-modal) | 오퍼 ID 정보를 수집합니다.  |
| [오퍼 설정 페이지](#offer-setup-page) | 주요 기능을 사용하도록 선택하고 Microsoft를 통해 제품을 판매하는 방법을 선택할 수 있습니다.  |
| [속성 페이지](#properties-page) | 마켓플레이스에서 쿠폰을 그룹화하는 데 사용되는 카테고리 및 산업, 쿠폰을 지원하는 법적 계약 및 앱 버전을 정의합니다. |
| [오퍼 리스팅 페이지](#offer-listing-page) | 오퍼 및 마케팅 자산에 대한 설명을 포함하여 마켓플레이스에 표시할 쿠폰 세부 정보를 정의합니다. |
| [미리 보기 페이지](#preview-page) | 더 넓은 마켓플레이스 오디언스에게 쿠폰을 라이브로 게시하기 전에 쿠폰을 공개할 수 있는 제한된 미리 보기 잠재고객을 정의하세요. |
| [기술 구성 페이지 제공](#technical-configuration-page)  | Microsoft를 통해 제품을 판매하도록 선택한 경우에만 사용할 수 있습니다. 오퍼에 연결하는 데 사용되는 기술 세부 정보(URL 경로, 웹후크, 테넌트 ID 및 앱 ID)를 정의합니다. |
| [**새로운 계획 모달**](#plan-identity-modal) | 계획 ID 정보를 수집합니다.  |
| [계획 목록 페이지](#plan-listing-page)  | Microsoft를 통해 제품을 판매하도록 선택한 경우에만 사용할 수 있습니다. 마켓플레이스에서 계획을 나열하는 데 사용되는 세부 정보를 정의합니다.  |
| [요금제 & 가용성 페이지](#plan-pricing--availability-page)  | Microsoft를 통해 제품을 판매하도록 선택한 경우에만 사용할 수 있습니다.  오퍼의 각 요금제(버전)에 대한 비즈니스 특성(가격 모델), 잠재고객 및 시장 가용성을 수집합니다.  |
| [테스트 드라이브 목록 페이지](#test-drive-listing-page)  | 오퍼에 대한 테스트 드라이브를 제공하도록 선택한 경우에만 사용할 수 있습니다. 마켓플레이스에서 테스트 드라이브를 나열하는 데 사용되는 세부 정보를 정의합니다.  |
| 테스트 드라이브 기술 구성 페이지  | 오퍼에 대한 테스트 드라이브를 제공하도록 선택한 경우에만 사용할 수 있습니다. 고객이 제품을 구매하기 전에 제품을 시험해 볼 수 있도록 데모(또는 "테스트 드라이브")에 대한 기술 세부 정보를 정의합니다.  |
| [페이지 검토 및 게시](#review-and-publish-page)  | 게시할 변경 사항을 선택하고 각 페이지의 상태를 확인하고 인증 팀에 메모를 제공합니다.  |


## <a name="new-offer-modal"></a>새로운 오퍼 모달 

제공하라는 첫 번째 정보는 오퍼의 ID와 별칭입니다. 

| **필드 이름**    | **참고**   |  
| :---------------- | :-----------| 
| 제품 ID  | 필요한 것은 생성 후 변경할 수 없습니다. 최대 50자이며 소문자, 숫자 문자, 대시 또는 밑줄로만 이루어져야 합니다. |
| 오퍼 별칭  | 필수 사항입니다. |

## <a name="offer-setup-page"></a>오퍼 설정 페이지

오퍼 설정 페이지는 다양한 채널을 선택하고 모션을 판매할 수 있을 뿐만 아니라 테스트 드라이브 및 고객 잠재 고객과 같은 주요 기능의 사용을 선언할 수 있습니다. 

| **필드 이름**    | **참고**   | 
| :---------------- | :-----------|  
| 당신은 마이크로 소프트를 통해 판매하시겠습니까?  | 필수 사항입니다. 기본값: Yes |
| 잠재 고객이 오퍼 리스팅과 상호 작용하기를 어떻게 원하십니까? (클릭 유도문한)  | Microsoft를 통해 판매하지 않는 경우 필요합니다. 기본값: 무료 평가판, 옵션: "지금 받기", "무료 평가판", "저에게 연락". |
| 체험 URL  | 고객이 오퍼 리스팅과 상호 작용해야 하는 방식으로 "무료 평가판"을 선택한 경우 필요합니다. |
| 쿠폰 URL  | 고객이 오퍼 리스팅과 상호 작용하는 방식으로 "Get It Now"를 선택한 경우 필요합니다. |
| 채널  | (선택 사항) 기본값: CSP(리셀러) 채널에 옵트인한 것이 아닙니다.  |
| 시험 사용 | (선택 사항) 기본값: 테스트 드라이브를 사용할 수 없습니다.  |
| 시험 사용 유형 | 테스트 드라이브를 사용하도록 설정한 경우 필요합니다. 기본값: 선택되지 않았습니다. 옵션: Azure 리소스 관리자, 비즈니스 센트럴용 Dynamics 365, 고객 참여를 위한 Dynamics 365, 운영용 Dynamics 365, 논리 앱, Power BI.  |
| 잠재 고객 관리 - CRM 시스템에 연결 | Microsoft를 통해 판매하는 경우 또는 목록제공을 "저에게 연락"하는 경우 필요합니다. 기본값: CRM 시스템이 연결되지 않았습니다. CRM 옵션: Azure 테이블, Azure Blob, Dynamics CRM 온라인, HTTP의 엔드포인트, Marketo, Salesforce  |

## <a name="properties-page"></a>속성 페이지

속성 페이지는 마켓플레이스에서 쿠폰을 그룹화하는 데 사용되는 카테고리 및 산업, 쿠폰을 지원하는 법적 계약 및 앱 버전을 정의하는 위치입니다. 이 페이지에서 오퍼에 대한 완전하고 정확한 세부 정보를 제공하여 적절하게 표시되고 올바른 고객 세트에 제공되도록 해야 합니다. 

| **필드 이름**    | **참고**   | 
| :---------------- | :-----------|  
| 범주 및 하위 범주 | 1및 최대 3. 기본값: 선택되지 않았습니다. |
| 산업 및 하위 산업 | (선택 사항) 각 L1 산업 내 최대 2L1 산업 및 최대 2개의 하위 산업, 기본값: 선택 없음 |
| 앱 버전  | (선택 사항) 기본값: 없음. |
| 표준 계약 사용  | (선택 사항) 기본값: 선택되지 않았습니다.  | |
| 사용 약관  | 표준 계약을 선택하지 않은 경우 필요합니다.  |

## <a name="offer-listing-page"></a>오퍼 리스팅 페이지

목록 페이지는 마켓플레이스에서 쿠폰 목록을 볼 때 고객이 볼 수 있는 텍스트와 이미지를 제공하는 곳입니다. 

| **필드 이름**    | **참고**   |
| :---------------- | :-----------| 
| 이름  | 필수, 최대 50 문자. |
| 요약  | 필수, 최대 100 chars. | 
| 설명  | 필수, 최대 3000 문자. |
| 시작하기 지침  | 필수, 최대 3000 문자. |
| 시작하기 지침  | 필수, 최대 3000 문자. |
| 검색 키워드  | 선택 사항, 권장, 최대 3 키워드. |
| 개인정보 처리방침 URL  | 필수 사항입니다. |
| CSP 프로그램 마케팅 자료 URL  | (선택 사항) |
| 유용한 링크 제목 + URL  | (선택 사항) |
| 문서 제목 + 파일 지원  | 필요한, 최소 1 및 최대 3. PDF 파일 형식이어야 합니다. |
| 스크린샷  | 필요한, 최소 1 스크린 샷 과 최대 5; 4 개 이상 권장 됩니다. PNG 형식의 1280 X 720이어야 합니다. |
| 스토어 로고(소형, 중형, 대형, 와이드, 히어로)  | 소형(48 X 48) 및 대형(216 X 216)이 필요합니다. 다른 크기는 선택 사항이지만 권장 : 중간 (90 x 90), 와이드 (255 x 115), 영웅 (815 x 290). PNG 형식이어야 합니다. |
| 동영상 이름 + URL + 썸네일  | 선택 사항, 권장, 최대 4 비디오. 썸네일PNG 형식의 1280 x 720이어야 합니다. 동영상은 YouTube 또는 Vimeo에서 호스팅되어야 합니다. |
| 연락처(CSP 프로그램, 엔지니어링, 지원)  | 엔지니어링 및 지원 담당자가 필요합니다(이름, 이메일 및 전화 번호); CSP 프로그램 연락처는 선택 사항이지만 권장됩니다. |
| 지원 URL  | 필수 사항입니다. |

## <a name="preview-page"></a>미리 보기 페이지

미리 보기 페이지는 쿠폰이 게재되기 전에 모든 요구 사항을 충족하는지 확인하기 위해 쿠폰 미리 보기에 액세스할 수 있도록 잠재고객을 지정하는 위치입니다. 

| **필드 이름**    | **참고**   | 
| :---------------- | :-----------| 
| AAD/MSA 이메일 + 설명 | 수동으로 입력한 경우 최소 1, 최대 10개, CSV 파일을 업로드하는 경우 최대 20개까지 필요합니다. |

## <a name="technical-configuration-page"></a>기술 구성 페이지 

기술 구성 페이지는 Microsoft에서 오퍼에 연결하는 데 사용하는 기술 세부 정보를 지정하는 위치입니다. Microsoft를 통해 판매하지 않기로 결정한 경우 이 페이지는 표시되지 않습니다.

| **필드 이름**    | **참고**   |  
| :---------------- | :-----------| 
| 방문 페이지 URL | 마이크로소프트를 통해 판매 하는 경우 필요 합니다. |
| 연결 웹후크 | 마이크로소프트를 통해 판매 하는 경우 필요 합니다. |
| Azure AD 테넌트 ID | 마이크로소프트를 통해 판매 하는 경우 필요 합니다. |
| Azure AD 앱 ID | 마이크로소프트를 통해 판매 하는 경우 필요 합니다. |

## <a name="plan-identity-modal"></a>아이덴티티 모달 계획

제공해야 하는 첫 번째 정보는 플랜의 이름과 ID입니다. Microsoft를 통해 판매하지 않기로 결정한 경우 이 페이지는 표시되지 않습니다.

| **필드 이름**    | **참고**   |  
| :---------------- | :-----------| 
| 플랜 ID  | 마이크로소프트를 통해 판매 하는 경우 필요 합니다. 생성 후에는 변경할 수 없습니다. 최대 50자이며 소문자, 숫자 문자, 대시 또는 밑줄로만 이루어져야 합니다. |
| 플랜 이름  | 마이크로소프트를 통해 판매 하는 경우 필요 합니다. 오퍼의 모든 플랜에서 고유해야 합니다. 최대 50자. |

## <a name="plan-listing-page"></a>계획 목록 페이지

계획 목록 페이지는 고객이 마켓플레이스에서 요금제를 볼 때 볼 수 있는 텍스트를 제공하는 곳입니다. Microsoft를 통해 판매하지 않기로 결정한 경우 이 페이지는 표시되지 않습니다.

| **필드 이름**    | **참고**   |  
| :---------------- | :-----------| 
| 계획 설명   | 마이크로소프트를 통해 판매 하는 경우 필요 합니다. 최대 500자. | |

## <a name="plan-pricing--availability-page"></a>요금제 & 가용성 페이지

요금제 가격 및 가용성 페이지는 제품의 각 요금제(버전)에 대한 비즈니스 특성, 잠재고객 및 시장 가용성을 정의하는 곳입니다. Microsoft를 통해 판매하지 않기로 결정한 경우 이 페이지는 표시되지 않습니다.

| **필드 이름**    | **참고**   | 
| :---------------- | :-----------| 
| 시장 가용성  | 필요한, 최소 1 및 최대 141. |
| 가격 책정 모델  | 필수 사항입니다. 기본값: 고정 요금. 옵션: 사용자당 고정 요금. |
| 최소 및 최대 좌석 수  | 좌석 기반 가격 책정 모델을 선택한 경우에만 사용할 수 있습니다. |
| 청구 기간  | 필수 사항입니다. 기본값: 월별. 옵션: 월간, 연간. |
| Price  | 월별 청구 기간을 선택한 경우 매월 USD가 필요합니다. 또는 연간 청구 기간을 선택한 경우 연간 USD. |
| 잠재고객 계획  | (선택 사항) 기본값: 공용 요금제. 옵션: 공용, 테넌트 ID별 비공개 |
| 제한된 계획 대상(테넌트 ID + 설명)  | 개인 플랜을 선택한 경우 필요합니다. 수동으로 입력하면 최소 1및 최대 10테넌트 아이디를 입력합니다. 최대 20000 경우 CSV 파일 가져오기. |

## <a name="test-drive-listing-page"></a>테스트 드라이브 목록 페이지

오퍼에 대한 테스트 드라이브를 제공하도록 선택한 경우에만 사용할 수 있습니다. 마켓플레이스에서 테스트 드라이브를 나열하는 데 사용되는 세부 정보를 정의합니다.

| **필드 이름**    | **참고**   | 
| :---------------- | :-----------| 
| 설명  | 필수 사항입니다. |
| 사용자 설명서 이름 + 파일  | 필수, 최대 1 문서. PDF 형식이어야합니다. |
| 비디오 이름, URL + 썸네일  | 선택 사항, 권장. 썸네일은 JPGP 또는 PNG 형식의 533 x 324이어야 합니다. 동영상은 YouTube 또는 Vimeo에서 호스팅되어야 합니다. |
