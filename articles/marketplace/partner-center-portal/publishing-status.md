# <a name="check-the-publishing-status-of-your-commercial-marketplace-offer"></a>상업용 마켓플레이스 오퍼의 게시 상태 확인

파트너 센터의 [상업용 마켓플레이스 포털의](https://partner.microsoft.com/dashboard/commercial-marketplace/offers) **오퍼 개요** 탭에서 현재 **게시 상태를** 볼 수 있습니다.

각 제품에 대해 다음 상태 표시기 중 하나가 표시되어야 합니다.

| **상태**    | **설명**  |
| :---------- | :-------------------|
| **초안** | 오퍼가 만들어졌지만 게시되지 않습니다. |
| **게시 진행 중** | 제안/계획은 게시 프로세스의 단계를 통해 그 방법을 작동하고 있습니다. |
| **주의가 필요함** | Microsoft 또는 게시 단계에서 인증 중에 중요한 문제가 발견되었습니다. |
| **미리 보기** | 제안은 Microsoft에서 인증했으며 이제 게시자가 최종 확인을 기다리고 있습니다. 라이브로 오퍼를 만들려면 라이브로 이동을 선택합니다. |
| **라이브** | 오퍼는 마켓플레이스에 생중계되며 고객이 보고 획득할 수 있습니다. |
| **보류 중인 정지 판매** | 게시자가 오퍼 또는 플랜에서 "판매 중지"를 선택했지만 작업이 아직 완료되지 않았습니다. |
| **마켓플레이스에서 사용할 수 없음** | 마켓플레이스에서 이전에 게시된 제안/계획이 삭제되었습니다. |

## <a name="automated-validation"></a>자동 유효성 검사

게시 프로세스의 첫 번째 단계는 자동화된 유효성 검사 집합입니다. 각 유효성 검사 단계는 오퍼를 만들 때 사용하도록 선택한 기능에 해당합니다. 이 기능을 사용하지 않은 경우 유효성 검사는 다음 게시 단계로 건너뜁니다. 게시 상태가 승인되기 전에 각 유효성 검사를 완료해야 합니다.

- **오퍼 구매 흐름 설정(<10분)**

이 단계에서는 Azure 포털을 통해 고객이 구매할 때 제품을 이행할 수 있는지 확인합니다. 이 단계는 Microsoft를 통해 판매되는 오퍼에만 적용됩니다.

- **테스트 드라이브 데이터 검증(~5분)**

이 단계에서는 오퍼의 테스트 드라이브 기술 구성 섹션에서 제공한 데이터의 유효성을 검사합니다. 테스트 드라이브 기능은 테스트 및 승인됩니다. 이 단계는 테스트 드라이브를 사용하도록 설정한 오퍼에만 적용됩니다.

- **테스트 드라이브 프로비저닝(~30분)**

이 단계에서는 이전 단계에서 테스트 드라이브의 데이터 및 기능을 검증한 후 고객이 사용할 수 있도록 테스트 드라이브의 인스턴스를 배포하고 복제합니다.  이 단계는 테스트 드라이브를 사용하도록 설정한 오퍼에만 적용됩니다.

- **리드 관리 검증 및 등록(<15분)**

이 단계에서는 잠재 고객 관리 시스템이 제안 설정에 제공된 세부 정보를 기반으로 고객 잠재 고객을 받을 수 있는지 확인합니다. 이 단계는 잠재 고객 관리를 사용하도록 설정한 오퍼에만 적용됩니다.

## <a name="certification"></a>인증

게시되기 전에 파트너 센터의 상업 시장에 제출된 제안은 인증을 받아야 합니다. 제출된 오퍼는 엄격한 테스트를 거치며, 일부는 자동화된 및 기타 설명서(Azure [Marketplace 인증 정책에](https://docs.microsoft.com/legal/marketplace/general-policies)대한 확인 포함)를 거칩니다. 오퍼 제출은 게시 흐름의 다음 단계로 진행하기 전에 인증을 받을 자격이 있음을 표시해야 합니다.

### <a name="types-of-validation-that-take-place-during-certification"></a>인증 중에 수행되는 유효성 검사 유형

제출된 각 오퍼에 대한 인증 프로세스에는 세 가지 수준의 유효성 검사가 포함되어 있습니다.

- 게시자 비즈니스 자격
- 내용 유효성 검사
- 기술 검증

#### <a name="publisher-business-eligibility"></a>게시자 비즈니스 자격

각 오퍼 유형은 게시자가 충족해야 하는 기본 자격 기준 집합을 검사합니다. 자격 기준에는 게시자의 MPN 상태, 보유 역량, 역량 수준 등이 포함될 수 있습니다.

#### <a name="content-validation"></a>내용 유효성 검사

콘텐츠 유효성 검사 중에 쿠폰을 만들 때 입력한 정보가 품질 및 관련성을 검사합니다. 이러한 검사는 마켓플레이스 목록 세부 정보, 가격, 가용성, 관련 요금제 등에 대한 출품작을 검토합니다. Azure 마켓플레이스 및/또는 AppSource 목록 기준을 충족하기 위해 제안에 다음이 포함되어 있는지 확인합니다.

- 오퍼를 정확하게 설명하는 제목;
- 철저한 개요와 가치 제안을 제공하는 잘 작성 된 설명;
- 품질 스크린 샷 및 동반 비디오; 및
- 제안이 Microsoft 플랫폼 및 도구를 활용하는 방법에 대한 설명입니다.

[일반 목록 정책을](https://docs.microsoft.com/legal/marketplace/certification-policies#100-general)읽고 콘텐츠 유효성 검사 기준에 대해 자세히 알아봅니다.

#### <a name="technical-validation"></a>기술 검증

기술 유효성 검사 중에 오퍼(패키지 또는 바이너리)는 다음과 같은 검사를 거칩니다.
- 맬웨어 검색
- 모니터링되는 네트워크 호출
- 패키지 분석
- 오퍼의 실제 기능 철저한 스캔

이 오퍼는 견고한지 확인하기 위해 다양한 플랫폼과 버전에서 테스트됩니다.

이 문서의 기술 구성 섹션에서 오퍼에 필요한 특정 구성 세부 정보를 검토합니다.

### <a name="certification-failure-report"></a>인증 실패 보고서

검토가 완료되면 오퍼가 인증을 통과한 후 게시 프로세스의 다음 단계로 이동합니다. 오퍼가 리스팅, 기술 또는 정책 검사에 실패했거나 해당 유형의 제안을 제출할 자격이 없는 경우 인증 실패 보고서가 생성되어 이메일로 전송됩니다.

이 보고서에는 실패한 정책에 대한 설명과 검토 노트가 포함되어 있습니다. 이 이메일 보고서를 검토하고, 문제를 해결하고, 필요한 경우 쿠폰을 업데이트하고, 파트너 센터의 [상용 마켓플레이스 포털을](https://partner.microsoft.com/dashboard/commercial-marketplace/offers) 사용하여 오퍼를 다시 제출합니다. (인증을 통과할 때까지 필요한 만큼 쿠폰을 다시 제출할 수 있습니다.)

## <a name="preview-creation"></a>미리 보기 만들기

미리 **보기 만들기** 단계에서 는 쿠폰의 미리 보기 섹션에 지정한 대상만 액세스할 수 있는 쿠폰 버전을 만듭니다.

>[!Note]
> 이 단계를 사용하여 조직 외부의 사람들에게 오퍼에 대한 가시성을 부여하지 마십시오. 대신 **비공개 쿠폰** 옵션을 사용합니다. 이 시점에서 오퍼링은 완전히 테스트되고 검증되지 않았으며 외부 배포에 대한 준비가 되지 않았습니다.

## <a name="publisher-approval"></a>게시자 승인

이 단계에서는 최종 게시 단계 전에 제안 미리 보기를 검토하고 승인하라는 요청과 함께 이메일로 전송됩니다.

Microsoft를 통해 제품을 판매하도록 선택한 경우 이 미리 보기 승인 단계에서 제품 요구 사항을 충족하는지 확인하기 위해 제품의 획득 및 배포를 테스트할 수 있습니다. 공개 마켓플레이스에서는 아직 쿠폰을 사용할 수 없습니다. 이 미리 보기를 테스트하고 승인한 후에는 [**오퍼 개요**](https://partner.microsoft.com/dashboard/commercial-marketplace/overview) 대시보드에서 **라이브 이동을** 선택해야 합니다.

이 미리 보기 단계에서 오퍼를 변경하려면 편집하고 다시 제출하여 새 미리 보기를 게시할 수 있습니다. 자세한 내용은 [기존 마켓플레이스 업데이트 문서를](#update-existing-marketplace-offers) 참조하세요.

쿠폰이 이미 라이브 중이고 마켓플레이스에서 공개되는 경우 [**오퍼 개요**](https://partner.microsoft.com/dashboard/commercial-marketplace/overview) 대시보드에서 **라이브** 로 이동을 선택할 때까지 업데이트한 내용은 공개되지 않습니다.

### <a name="publish-offer-to-the-public"></a>대중에게 제안 게시

파트너 센터에 로그인하고 오퍼에 액세스합니다. **쿠폰 개요** 페이지로 리디렉션됩니다. 이 페이지의 상단에 라이브 이동 **에**대한 옵션이 표시됩니다. **라이브 로이동을** 선택하면 확인 후 오퍼가 대중에게 공개되기 시작합니다. 오퍼가 라이브로 제공되면 이메일 알림을 받게 됩니다.

## <a name="publish"></a>게시

이제 마켓플레이스에서 사용할 수 있도록 오퍼와 함께 **라이브로 이동하도록** 선택되었으므로 라이브 오퍼가 제안의 미리 보기 버전과 마찬가지로 구성되었는지 확인하기 위해 일련의 최종 유효성 검사 검사가 진행됩니다.

- **오퍼 구매 흐름 설정(>10분)**

이 단계에서는 Azure 포털을 통해 고객이 구매할 때 제품을 이행할 수 있는지 확인합니다. 이 단계는 Microsoft를 통해 판매되는 오퍼에만 적용됩니다.

- **테스트 드라이브 데이터 검증(~5분)**

이 단계에서는 오퍼의 테스트 드라이브 기술 구성 섹션에서 제공한 데이터의 유효성을 검사합니다. 테스트 드라이브 기능은 테스트 및 승인됩니다. 이 단계는 테스트 드라이브를 사용하도록 설정한 오퍼에만 적용됩니다.

- **테스트 드라이브 프로비저닝(~30분)**

이 단계에서는 고객이 사용할 수 있도록 테스트 드라이브의 인스턴스를 배포하고 복제합니다.  이 단계는 테스트 드라이브를 사용하도록 설정한 오퍼에만 적용됩니다.

- **리드 관리 검증 및 등록(>15분)**

이 단계에서는 잠재 고객 관리 시스템이 제안 설정에 제공된 세부 정보를 기반으로 고객 잠재 고객을 받을 수 있는지 확인합니다. 이 단계는 잠재 고객 관리를 사용하도록 설정한 오퍼에만 적용됩니다.

- **최종 게시(>30분)**

이 단계에서는 귀하의 제안이 마켓플레이스에서 공개적으로 제공되도록 합니다.
