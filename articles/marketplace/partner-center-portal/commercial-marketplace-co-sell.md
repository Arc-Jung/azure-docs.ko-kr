# <a name="co-sell-option-in-partner-center"></a>파트너 센터의 공동 판매 옵션

공동 판매 탭에 대한 정보를 제공하는 것은 전적으로 선택 사항입니다. 공동 판매 준비 및 IP 공동 판매 준비 상태를 달성하는 데 필요합니다. 제공한 정보는 Microsoft 영업 팀에서 고객의 요구에 맞는 솔루션을 평가할 때 솔루션에 대해 자세히 알아보는 데 사용됩니다. 고객에게직접 사용할 수 없습니다. 자세한 내용은 Microsoft 를 [통한 판매를](https://docs.microsoft.com/azure/marketplace/partner-center-portal/create-new-saas-offer#sell-through-microsoft)참조하십시오.

파트너 센터의 공동 판매 옵션은 현재 일부 오퍼 유형에 사용할 수 있습니다. 추가 혜택 유형에 대한 지원은 시간이 지남에 따라 제공됩니다.

| **오퍼 유형** | **파트너 센터에서 공동 판매 페이지 사용 가능** |
|    -----------------------    |    -----------------------------  |
| 서비스로서의 소프트웨어(SaaS) |     2020년 2월 경 일반 공급 가능      |
| PowerApps를 & 고객 참여를 위한 Dynamics 365      |     지금 사용 가능      |
| Dynamics 365 for Operations     |     지금 사용 가능       |
| Dynamics 365 Business Central      |  2020년 2월 경 일반 공급 가능     |
| Azure 애플리케이션     |     2020년 2월 경 일반 공급 가능     |
| Azure Virtual Machine      |     공지 예정       |
| Azure 컨테이너     |     공지 예정       |
| 컨설팅 서비스      |     공지 예정       |
| IoT 에지 모듈      |     공지 예정       |
| 전원 BI 서비스 앱     |     공지 예정       |
|  |

## <a name="listing"></a>나열

공동 판매 목록은 Microsoft 영업 팀이 더 많은 고객에게 솔루션을 마케팅하는 데 도움이 됩니다.

- 오퍼가 빌드, 확장 또는 통합되는 Microsoft 플랫폼을 선택합니다.
- 오퍼가 타겟팅하는 시장 세그먼트를 선택합니다.
- 오퍼의 [솔루션 유형을](#solution-type) 선택합니다.
- 최대 [3개의 솔루션 영역과](#solution-areas) 하위 영역을 선택합니다.

### <a name="solution-type"></a>솔루션 형식

솔루션 유형은 오퍼가 해결하도록 설계된 시나리오를 정의하는 데 도움이 됩니다.

| **솔루션 유형**    | **설명**  |
| :------------------- | :-------------------|
| 장치(하드웨어) | 장치 제조업체에서 하드웨어를 구축하거나 판매하는 오퍼입니다. |
| IP (응용 프로그램) | 고객의 사용을 위해 라이선스가 부여된 앱 또는 기타 저작권이 있는 자료. 예를 들어 온-프레미스에서 라이선스를 부여받고 설치할 수 있는 CRM 프로그램입니다. |
| 서비스 | 컨설턴트를 통해 전달되는 특정 일회성 프로젝트에 대한 실무 전문 지식. 예: 클라이언트에 대한 고객 데이터베이스 설정(클라이언트가 배달 후 데이터베이스 운영을 담당할 경우). |
| 관리 형 서비스 | 클라우드 기반 프로젝트에 대한 실무 전문 지식은 일반적으로 지속적으로 진행됩니다. 예: 관리되는 서비스 공급자가 제공하는 지속적인 관리를 통해 온라인 데이터베이스를 실행하기 위한 플랫폼 및 도구를 제공합니다. |

### <a name="solution-areas"></a>솔루션 영역

솔루션 영역은 솔루션을 추가로 정의하는 데 도움이 됩니다.

오퍼에 대해 최대 3개의 솔루션 영역을 추가할 수 있습니다. 각 솔루션 영역에 대해 최대 2개의 솔루션 하위 영역을 선택할 수 있습니다. 여러 하위 범주를 `Ctrl` 선택하려면 Windows에서 키 또는 명령 키(Mac OS)를 사용합니다.

 |

## <a name="contacts"></a>연락처

연락처 정보를 통해 Microsoft 영업 팀 및 채널 리셀러는 조직의 적절한 리소스에서 추가 정보를 요청할 수 있습니다. 연락처 정보는 모든 Microsoft 영업 팀에서 사용할 수 있습니다. CSP 프로그램에서 쿠폰을 사용할 수 있도록 선택한 경우 채널 리셀러도 이 연락처 정보를 사용할 수 있습니다.

이 문서에서는 파트너의 다양한 역할과 파트너 중에서 선택하는 방법에 대한 지침을 제공합니다. 파트너 센터에서 CSV 템플릿을 다운로드하고, 각 행에 연락처에 대한 정보를 입력하고, 이 CSV 파일을 가져와 연락처를 제공할 수 있습니다. CSV 파일을 가져오면 기존 연락처가 덮어씁니다.

또한 제품의 기존 연락처 목록을 내보낸 다음 해당 파일을 변경할 수도 있습니다.

* *이름(필수)*: 연락처 이름입니다.
* *이메일 주소(필수)*: 연락처의 이메일 주소입니다.
* *직칭(필수)*: 직칭.
* *역할(필수)*: 템플릿에서 이 필드에 다음 역할 중 하나를 사용합니다.

    | **Role**    | **설명**  |
    | :------------------- | :-------------------|
    | *:::no-loc text="Partner marketing":::* | 이 역할은 솔루션을 마케팅하고 Microsoft 영업 팀 및 채널 리셀러와의 마케팅 노력에 중점을 둡니다. 마케팅 참여에 대한 주요 연락 지점및 제품 설명, 이미지 및 동영상과 같은 리스팅 콘텐츠를 제공합니다. |
    | *:::no-loc text="Partner sales":::* | 이 역할은 솔루션을 판매하고 Microsoft 영업 팀 및 채널 리셀러와 판매에 협력하는 데 중점을 둡니다. 공동 판매 솔루션의 경우 공동 판매 준비를 원하는 지역당 파트너 영업 담당자가 하나 이상 있음을 나타냅니다. 동일한 파트너 영업 담당자가 여러 지역을 포함할 수 있습니다. |
    | *:::no-loc text="Partner technical sales":::* | 영업 주기, 판매 후 통합 및 배포 기간 동안 기술 아키텍처 및 배포 고려 사항을 지원합니다. |
    | *:::no-loc text="Partner customer success manager":::* | 일반적으로 고객이 솔루션을 최대한 지원하고 고객의 조직 내에서 입지를 확대할 수 있도록 배포 후 고객을 지원합니다. |

* *국가/지역(필수):*  템플릿을 작성할 때 [나열된 두 문자 국가/지역 코드를](./commercial-marketplace-co-sell-countries.md)사용합니다. 연락처가 모든 국가/지역을 포함하는 경우 세 문자 코드 "OOO"를 사용합니다. 연락처가 둘 이상의 국가/지역을 포함하는 경우 쉼표로 구분된 두 문자 코드를 각각 입력합니다(예: 따옴표 없이 "US, CA, FR"을 템플릿에 입력).

    국가/지역은 각 연락처의 영역을 반영해야 합니다. Microsoft 영업 팀과 채널 리셀러는 이 정보를 사용하여 정보를 요청하거나 특정 국가/지역 내의 판매에 대한 공동 작업을 합니다.

* *주/지방(선택 사항):*  템플릿을 작성할 때 [주/지방 테이블에](./commercial-marketplace-co-sell-states.md)나열된 XX-XX 형식을 사용합니다.
