# <a name="dynamics-365-for-finance-and-operations-offer"></a>Dynamics 365 for Finance and Operations 제품

<table> <tr> <td>이 섹션에서는 Dynamics 365 for Finance and Operations 솔루션을 Microsoft <a href="https://appsource.microsoft.com">AppSource Marketplace</a>에 게시하는 방법에 대해 설명합니다. Microsoft <a href="https://dynamics.microsoft.com/finance-and-operations">Dynamics 365 for Finance and Operations</a>는 고급 재무, 운영, 제조 및 공급망 관리를 지원하는 ERP(전사적 자원 관리)입니다. </td> <td><img src="./media/publish_d365_new_offer/dynfinance-icon1.png"  alt="Dynamics 365 for Finance and Operations icon" /></td> </tr> </table>


## <a name="publishing-process"></a>게시 프로세스

다음 단계를 사용하여 Dynamics 365 for Finance and Operations 앱을 개발하고 이 앱을 AppSource에 게시합니다.

1. 필수 조건으로 [인증된 개발자 센터](https://dev.windows.com/en-us/registration?accountProgram=Azure)가 되어야 합니다.
2. 코드를 업그레이드합니다.
3. [LCS](https://docs.microsoft.com/dynamics365/unified-operations/dev-itpro/lifecycle-services/lcs)(Lifecycle Services)를 사용하여 데이터를 빌드, 패키지 및 배포합니다.
4. LCS에서 구현 방법을 만듭니다.
5. LCS에서 비즈니스 프로세스를 정의합니다.
6. Cloud 파트너 포털에서 마케팅 콘텐츠를 만듭니다.
7.  팀에서 앱의 유효성 검사를 완료합니다. 지침은 [재무 및 운영에 대한 애플리케이션의 유효성 검사](https://docs.microsoft.com/dynamics365/operations/dev-itpro/lcs-solutions/app-validation-lcs-solutions)를 참조하세요.
8. Cloud 파트너 포털에서 제품을 테스트하고 인증합니다.
9.  AppSource에 게시하고 라이브로 전환합니다.

이 섹션의 후속 문서에서는 이러한 단계를 더 자세히 설명합니다.
