# <a name="azure-saas-application-offer"></a>Azure SaaS 애플리케이션 제품

<table> <tr> <td>이 섹션에서는 SaaS(software-as-a-service) 애플리케이션을 Microsoft <a href="https://azuremarketplace.microsoft.com/marketplace/">Azure Marketplace</a> 또는 <a href="https://appsource.microsoft.com">Microsoft AppSource</a>에 게시하는 방법을 설명합니다. 자신의 Azure 구독에 솔루션을 배포하고, 애플리케이션을 테스트하기 위해 디자인하고 관리하는 인터페이스를 통해 고객이 로그인하는 경우 SaaS 애플리케이션을 사용합니다. <a href="https://azure.microsoft.com/services/active-directory">AAD</a>(Azure Active Directory)를 사용하여 기존 평가판 환경을 활용합니다. </td> <td><img src="./media/cloud-partner-portal-publish-web-app/saasapp-icon1.png"  alt="Azure SaaS application icon" /></td> </tr> </table>


## <a name="publishing-benefits"></a>게시 이점

Microsoft Marketplace에 애플리케이션을 나열하는 이점 중 일부는 다음과 같습니다.

- Office 365 및 Dynamics 365에서 Azure Active Directory 1억 사용자를 연결합니다.
- 영업 팀 확장: 전 세계의 비즈니스 사용자를 연결하고, 최종 사용자를 참여시키고 잠재 고객을 생성할 수 있고 여러 산업에서 새로운 고객과 대화를 시작하는 판매 채널을 확보합니다.

- 실행 가능한 인사이트 획득: 앱이 AppSource에서 작동되는 방법, 잘 작동하는지 및 판매 프로시저를 더욱 개선하는 방법에 대한 인사이트를 공유합니다.


## <a name="publishing-process"></a>게시 프로세스

Microsoft Azure Marketplace 또는 Microsoft AppSource에서 앱을 만들고 게시하려면 이 섹션에서는 다음 단계를 안내합니다.

1. [개발자 센터 등록](https://dev.windows.com/en-us/registration?accountProgram=Azure) *또는* [Microsoft 파트너 네트워크 등록](https://partner.microsoft.com/)과 같은 파트너 등록을 포함하여 필수 조건을 충족하는지 확인합니다.
2.  마켓플레이스 제품 및 계획을 정의합니다.
3.  마케팅 콘텐츠 및 가격 책정을 제공합니다.
4.  [Cloud 파트너 포털](https://cloudpartner.azure.com)에서 제품을 테스트하고 인증합니다.
5.  제품을 라이브로 전환합니다.
