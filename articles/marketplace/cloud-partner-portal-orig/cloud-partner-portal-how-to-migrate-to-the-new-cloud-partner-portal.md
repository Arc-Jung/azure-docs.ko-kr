# <a name="how-to-migrate-to-the-new-cloud-partner-portal"></a>새 클라우드 파트너 포털로 마이그레이션하는 방법

Cloud 파트너 포털로 마이그레이션하는 것은 중요한 단계입니다. 게시 프로세스의 피드백을 반영하여 포털에 신중한 투자를 했습니다. 이러한 투자는 게시 환경을 쉽게 사용하고 개선하는 데 중점을 두고 있습니다. 

-   조직 ID 로그인 지원
-   실시간 필드 유효성 검사로 오류 포착
-   게시 워크플로 간소화

제품을 쉽게 마이그레이션할 수 있으므로 향상된 게시 환경을 이용할 준비가 되면 다음 마이그레이션 단계를 검토하세요.

다음 마이그레이션 스테이지를 검토하여 성공적으로 마이그레이션하세요.



## <a name="decide-youre-ready-to-migrate"></a>마이그레이션할 준비가 되었는지 결정

제품을 Cloud 파트너 포털로 마이그레이션하기 전에 어떤 일이 수행될지 충분히 이해해야 합니다.

**내 모든 제품을 마이그레이션하나요?**

현재 모든 Virtual Machine 제품을 마이그레이션하고 있습니다. 초안, 스테이징된 또는 나열된 Virtual Machine 제품이 각각 마이그레이션됩니다.

다른 제품 유형의 경우(솔루션 템플릿 및 시험 사용) 당분간은 이전 포털에서 계속 작동합니다.

**내 라이브 제품은 계속 작동하나요?**

예, 이 마이그레이션은 라이브 제품에는 영향을 주지 않습니다. 또한 마이그레이션이 완료되면 새로운 포털에서 제품을 즉시 업데이트할 수 있도록 게시자 계정 정보가 미리 채워져 있습니다. 스테이징된 제품은 초안이 되며 새 포털에서 게시해야 합니다.

**마이그레이션할 때 내 제품의 가격 책정을 변경할 수 있나요?**

아니요. 이 마이그레이션에서 가격 변경을 지원하지 않으므로 모든 라이브 제품에 대한 가격 책정은 동일해야 합니다.

**내 제품이 준비/게시 상태라면 어떻게 하나요?**

제품이 현재 게시 중이거나 이전 포털에서 준비된 경우 마이그레이션이 실패합니다. 마이그레이션 시간을 선택할 때 제품이 게시됨/스테이징됨 상태인지 확인해야 합니다.

**새 역할 기반 액세스는 어떻게 작동하나요?**

새 포털에서 관리자는 게시 프로필의 소유자가 되며 모든 공동 관리자는 참가자가 됩니다. 게시 프로필의 내용을 편집하려면 새 포털로 이동해야 합니다.

사용자를 관리하고 이전 포털에서 제품에 대해 작업할 누군가를 추가하려는 경우 이전 포털에서 공동 관리자로 계속 추가합니다. 새 포털에서 제품 관련 작업을 수행할 다른 사용자를 추가하려면 새 포털에서 contributor로 추가합니다.

**마이그레이션하면 최종인가요?**

제품을 마이그레이션할 때 해당 제품의 모든 이전 포털 작업을 읽기 전용으로 표시하고 이전 포털에서 만들기를 차단합니다. 새 포털에서만 중단한 부분부터 마이그레이션된 모든 제품을 계속 편집 및 작업할 수 있습니다.

## <a name="submit-a-request-to-migrate"></a>마이그레이션할 요청 제출

마이그레이션을 계획하기로 했다면 게시자 이름 및 ID를 사용하여 Cloud 파트너 포털의 학습 페이지에서 마이그레이션하도록 제출합니다.

![게시 흐름 프로세스 다이어그램](./media/cloud-partner-portal-how-to-migrate-to-the-new-cloud-partner-portal/migrate.png)

## <a name="confirm-a-date-to-migrate"></a>마이그레이션할 날짜 확인

매주 화요일 오전 10시부터 오후 1시(PST)까지 제품을 마이그레이션하고 게시 계정 소유자로부터 적어도 24시간 이전에 전송 시간을 확인하도록 요청합니다. 워크플로에 방해가 되지 않도록 다른 시간에 마이그레이션해야 하는 경우 최적의 시간을 찾기 위한 작업을 수행합니다.

마이그레이션이 완료되었음을 알리고 새 포털에서 계속 작업할 수 있음을 알리는 전자 메일 알림이 필요할 수 있습니다.

## <a name="settle-in-and-enjoy"></a>안정적으로 즐기기

마이그레이션 후에는 새 포털에서 작업하는 것이 좋습니다. 이제부터 이전 포털에서는 제품이 읽기 전용이며 향후 모든 생성 및 편집은 새 포털에서 수행해야 합니다.

**마이그레이션에서 잘못된 것이 있는 경우 어떻게 해야 하나요?**
 
마이그레이션에 실패하면 변경 내용을 되돌리고 마이그레이션할 다른 시간을 다시 예약합니다. 마이그레이션을 완료한 후에도 질문이 있는 경우 피드백 도구를 통해 자유롭게 알려 주세요.
