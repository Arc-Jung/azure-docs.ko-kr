<a name="update-an-existing-offer-for-azure-marketplace"></a>Azure Marketplace에 대한 기존 제품 업데이트
==============================================

제안을 라이브로 전환한 후 다양한 종류의 업데이트를 적용할 수 있습니다.   [Cloud 파트너 포털](https://cloudpartner.azure.com/)에는 변경이 올바른지 확인하기 위해 제안 편집을 지원하는 다음을 비롯한 몇 가지 기능을 제공합니다.

-  기존 SKU에 새 VM(가상 머신) 이미지 버전 추가
-  SKU가 제공되는 지역 변경
-  새 SKU 추가
-  제안 또는 SKU에 대한 Marketplace 메타데이터 업데이트 
-  종량제 제안 가격 업데이트
-  기능 비교
-  기능 내역

제안 또는 SKU를 수정한 후 변경 내용이 라이브로 전환되려면 다시 게시해야 합니다.
이 문서에서는 VM 제품을 업데이트하는 다양한 측면을 안내합니다.


<a name="unpermitted-changes-to-vm-offersku"></a>VM 제품/SKU에 대해 허용되지 않는 변경
-----------------------------------

Azure Marketplace에서 제안이 라이브로 전환된 후 수정할 수 없는 몇 가지 VM 제안 또는 SKU 특성이 있습니다.

-  해당 제품의 제품 ID 및 게시자 ID
-  기존 SKU의 SKU ID
-  기존 SKU의 데이터 디스크 수
-  기존 SKU에 대한 청구/라이선스 모델 변경


<a name="updating-the-vm-image-version-for-a-sku"></a>SKU에 대한 VM 이미지 버전 업데이트
---------------------------------------

제품의 SKU에 대한 VM 이미지가 추가 기능, 보안 패치 등으로 업데이트되었습니다. 이러한 시나리오에서는 SKU가 참조하는 VM 이미지를 업데이트하려고 할 수 있습니다. VM 이미지를 업데이트하려면 다음 단계를 따릅니다. 

1.  [클라우드 파트너 포털](https://cloudpartner.azure.com/)에 로그인합니다.
2.  **모든 제안** 아래에서 업데이트할 제안을 찾습니다.
3.  **SKU** 양식 아래에서 VM 이미지를 업데이트하려는 SKU를 클릭합니다.
4.  **디스크 버전** 아래에서 **+새 디스크 버전**을 클릭하여 새 VM 이미지를 추가합니다.
5.  새 VM 이미지 **디스크 버전**을 제공합니다. 디스크 버전은 [의미 체계 버전](http://semver.org/) 형식을 따라야 합니다. 버전은 X.Y.Z 형식이며, X, Y 및 Z는 정수여야 합니다. 제공하는 새 버전이 이전 버전보다 큰지 확인합니다. 그렇지 않으면 다시 게시할 때 Azure Portal 또는 Azure Marketplace에 새 버전 번호가 표시되지 않습니다.
6.  **OS VHD URL**에 운영 체제 VHD에 대해 만들어진 SAS(공유 액세스 서명) URI를 입력합니다. SKU의 다양한 버전 간에는 데이터 디스크 수를 변경할 수 없습니다. 이전 버전에 데이터 디스크가 구성된 경우 새 버전에도 데이터 디스크가 구성되어 있어야 합니다.
7.  **게시**를 클릭하고 게시 워크플로를 시작하여 새 VM 버전을 Azure Marketplace 및 Azure Portal에 라이브로 전환합니다.


<a name="changing-regions-a-sku-is-available-in"></a>SKU가 제공되는 지역 변경
--------------------------------------

시간이 지남에 따라 더 많은 지역에서 제품/SKU를 사용할 수 있도록 하고 싶을 수 있습니다.
또는 특정 지역에서 제품/SKU 지원을 중지하고 싶을 수도 있습니다.
이러한 변경 내용을 구현하려면 다음 단계를 수행합니다.

1.  [클라우드 파트너 포털](https://cloudpartner.azure.com/)에 로그인합니다.
2.  **모든 제안** 아래에서 업데이트할 제안을 찾습니다.
3.  **SKU** 양식 아래에서 지역 가용성을 업데이트하려는 SKU를 클릭합니다.
4.  **국가/지역 가용성** 필드 아래에서 **국가 선택** 단추를 클릭합니다.
5.  지역 가용성 팝업에서 이 SKU에 대해 원하는 지역을 추가/제거합니다.
6.  **게시**를 클릭하고 게시 워크플로를 시작하여 SKU 지역 가용성을 업데이트합니다.

새 지역에서 SKU를 사용할 수 있게 되면 **가격 데이터 내보내기** 기능을 통해 특정 지역의 가격 책정을 지정할 수 있습니다. 이전에 한 번 사용할 수 있었던 지역을 다시 추가하는 경우 가격 책정 변경이 허용되지 않으므로 가격을 업데이트할 수 없습니다.


<a name="adding-a-new-sku"></a>새 SKU 추가
----------------

기존 제안에 대해 새 SKU를 사용할 수 있도록 하려면 다음 단계를 따릅니다.

1.  [클라우드 파트너 포털](https://cloudpartner.azure.com/)에 로그인합니다.
2.  **모든 제안** 아래에서 업데이트할 제안을 찾습니다.
3.  **SKU** 양식 아래에서 **새 SKU 추가**를 클릭하고 팝업에서 **SKU ID**를 제공합니다.
4.  [Azure Marketplace에 가상 머신 게시](./cloud-partner-portal-publish-virtual-machine.md)에 설명된 나머지 단계를 따릅니다.
5.  **게시**를 클릭하고 게시 워크플로를 시작하여 새 SKU를 라이브로 전환합니다.


<a name="updating-offer-marketplace-metadata"></a>제품 Marketplace 메타데이터 업데이트
------------------------------------

회사 로고를 업데이트하는 등 제품과 관련된 Marketplace 메타데이터를 업데이트해야 하는 시나리오가 있을 수 있습니다. 다음 단계를 사용하여 제안의 메타데이터를 업데이트합니다.

1.  [클라우드 파트너 포털](https://cloudpartner.azure.com/)에 로그인합니다.
2.  **모든 제안** 아래에서 업데이트할 제안을 찾습니다.
3.  **Marketplace** 양식으로 이동한 후 여기에 있는 지침에 따라 변경합니다.
4.  **게시**를 클릭하고 게시 워크플로를 시작하여 변경 내용을 라이브로 전환합니다.


<a name="updating-pricing-on-published-offers"></a>게시된 제안 가격 업데이트
------------------------------------

종량제 제안이 게시되면 기존 SKU의 가격을 늘릴 수 없지만, 동일한 제안 아래에 새 SKU를 만들고, 이전 SKU를 삭제한 후 제안을 다시 게시할 수 있습니다. 또한 이미 게시된 제안의 가격을 낮출 수도 있습니다. 제안 가격을 낮추려면

1.  가격을 낮추려는 SKU를 클릭합니다.
2.  1x1 GUI에서 가격을 설정한 경우 UI에서 직접 가격을 변경할 수 있습니다. 스프레드시트 가져오기/내보내기를 통해 가격을 설정하는 경우 가져오기/내보내기 기능을 통해서만 가격을 낮출 수 있습니다.
3.  **저장**을 클릭합니다.
4.  제안을 다시 게시하고 라이브로 전환합니다.

웹 사이트에 제품이 완전히 라이브되면 새 고객에게 가격이 표시되고, 모든 새 고객은 감소된 새 가격을 지불하게 됩니다.

기존 고객의 경우 가격 절감이 유효해진 청구 주기 시작 기간부터 가격 감소가 소급해서 적용됩니다.
가격 감소가 발생한 주기 동안 이미 요금이 청구되었으면 다음 청구 주기 동안 감소된 가격이 환불됩니다.


<a name="simplified-currency-pricing"></a>간소화된 통화 가격 책정

<a name="compare-feature"></a>기능 비교
---------------

이미 게시된 제안을 변경하는 경우 *비교* 기능을 사용하여 수행된 변경을 감사할 수 있습니다. 비교를 사용하려면

1.  편집 프로세스 중 언제든지 제안에 대해 **비교** 단추를 클릭할 수 있습니다.

2.  마케팅 자산 및 메타데이터 버전을 함께 비교합니다.


<a name="history-of-publishing-actions"></a>게시 작업의 기록
-----------------------------

모든 이전 게시 작업을 보려면 Cloud 파트너 포털의 왼쪽 탐색 창에서 **기록**을 클릭합니다. 여기에서 Azure Marketplace 제안의 수명 동안 수행된 타임스탬프 지정 작업을 확인할 수 있습니다.
