# <a name="edit-storage-targets"></a>스토리지 대상 편집

캐시의 저장소 대상 페이지에서 저장소 **대상을** 제거하거나 수정할 수 있습니다.

## <a name="remove-a-storage-target"></a>스토리지 대상 제거

저장소 대상을 제거하려면 목록에서 선택하여 **삭제** 단추를 클릭합니다.

이 작업은 이 Azure HPC 캐시 시스템과의 저장소 대상 연결을 제거하지만 백 엔드 저장소 시스템은 변경되지 않습니다. 예를 들어 Azure Blob 저장소 컨테이너를 사용한 경우 캐시에서 컨테이너를 삭제한 후에도 컨테이너와 해당 내용은 계속 존재합니다. 컨테이너를 다른 Azure HPC 캐시에 추가하거나 이 캐시에 다시 추가하거나 Azure 포털에서 삭제할 수 있습니다.

캐시에 저장된 모든 파일 변경 내용은 저장소 대상이 제거되기 전에 백 엔드 저장소 시스템에 기록됩니다. 이 프로세스는 많은 변경된 데이터가 캐시에 있는 경우 한 시간 이상 걸릴 수 있습니다.

## <a name="update-storage-targets"></a>스토리지 대상 업데이트

저장소 대상을 편집하여 일부 속성을 수정할 수 있습니다. 다른 속성은 다양한 유형의 저장소에 대해 편집할 수 있습니다.

* Blob 저장소 대상의 경우 네임스페이스 경로를 변경할 수 있습니다.

* NFS 저장소 대상의 경우 다음 속성을 변경할 수 있습니다.

  * 네임스페이스 경로
  * 사용 모델
  * 내보내기
  * 하위 디렉터리 내보내기

저장소 대상의 이름, 유형 또는 백 엔드 저장소 시스템(Blob 컨테이너 또는 NFS 호스트 이름/IP 주소)은 편집할 수 없습니다. 이러한 속성을 변경해야 하는 경우 저장소 대상을 삭제하고 새 값으로 대체값을 만듭니다.

저장소 대상을 수정하려면 저장소 대상 이름을 클릭하여 세부 정보 페이지를 엽니다. 페이지의 일부 필드는 편집할 수 있습니다.

![NFS 저장소 대상에 대한 편집 페이지의 스크린샷](media/hpc-cache-edit-storage-nfs.png)

## <a name="update-an-nfs-storage-target"></a>NFS 스토리지 대상 업데이트

NFS 저장소 대상의 경우 여러 속성을 업데이트할 수 있습니다. (위의 스크린샷을 참조하여 편집 페이지를 편집하십시오.)

* **사용 모델** - 사용 모델은 캐시가 데이터를 유지하는 방식에 영향을 미칩니다. 자세한 내용은 [사용 모델 선택을](hpc-cache-add-storage.md#choose-a-usage-model) 참조하십시오.
* **가상 네임스페이스 경로** - 클라이언트가 이 저장소 대상을 탑재하는 데 사용하는 경로입니다. 자세한 내용은 [집계된 네임스페이스 계획을](hpc-cache-namespace.md) 읽어보십시오.
* **NFS 내보내기 경로** - 이 네임스페이스 경로에 사용할 저장소 시스템 내보내기입니다.
* **하위 디렉터리 경로** - 내보내기 아래의 하위 디렉터리로 이 네임스페이스 경로와 연결할 수 있습니다. 하위 디렉터리를 지정할 필요가 없는 경우 이 필드를 비워 둡니다.

각 네임스페이스 경로에는 내보내기와 하위 디렉터리가 고유하게 결합되어 있어야 합니다. 즉, 백 엔드 저장소 시스템에서 동일한 디렉터리로 두 개의 서로 다른 클라이언트 대면 경로를 만들 수 없습니다.

변경한 후 **확인을** 클릭하여 저장소 대상을 업데이트하거나 **취소를** 클릭하여 변경 내용을 삭제합니다.

## <a name="update-an-azure-blob-storage-target"></a>Azure Blob 저장소 대상 업데이트

Blob 저장소 대상에 대한 세부 정보 페이지를 사용하면 가상 네임스페이스 경로를 수정할 수 있습니다.

![Blob 저장소 대상에 대한 편집 페이지의 스크린샷](media/hpc-cache-edit-storage-blob.png)

완료되면 **확인을** 클릭하여 저장소 대상을 업데이트하거나 **취소를** 클릭하여 변경 내용을 삭제합니다.
