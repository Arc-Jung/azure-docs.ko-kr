<h1>Azure Active Directory 정의</h1>

Azure Active Directory는 클라우드에서 ID 및 액세스 관리 기능을 제공하는 서비스입니다. Active Directory가 온-프레미스 ID 관리를 위해 Windows Server 운영 체제를 통해 고객에게 제공되는 서비스인 것과 거의 마찬가지로 Azure AD(Azure Active Directory)는 클라우드 기반 ID 관리를 위해 Azure를 통해 제공되는 서비스입니다. [자세한 정보][1]

조직의 클라우드 디렉터리이기 때문에 대상 사용자, 클라우드에 보관할 정보, 정보를 사용하거나 관리할 수 있는 사람 및 정보에 액세스할 수 있는 응용 프로그램 또는 서비스를 직접 결정합니다.

Azure AD를 사용하는 경우 조직 정보의 개인 정보 보호 및 보안에 대한 요구 사항을 준수하는 동시에 고확장성, 고가용성 및 통합된 재해 복구로 클라우드에서 Active Directory를 실행하는 것은 Microsoft의 책임입니다.

<h3>온-프레미스 Active Directory와 통합</h3>


Azure AD는 조직의 독립 실행형 클라우드 디렉터리로 사용될 수 있지만 기존 온-프레미스 Active Directory와 Azure AD를 통합할 수도 있습니다. 통합 기능에는 디렉터리 동기화 및 Single Sign-On이 포함되며, 향상된 관리자 및 최종 사용자 환경을 제공하기 위해 기존 온-프레미스 ID의 도달 범위를 클라우드로 확장합니다. [자세한 정보][2]

<h3>응용 프로그램과 통합</h3>


응용 프로그램 개발자는 Azure AD와 해당 응용 프로그램을 통합하여 사용자에게 Single Sign-On 기능을 제공할 수 있습니다. 이렇게 하면 클라우드에 엔터프라이즈 응용 프로그램을 호스트하고 회사 자격 증명을 사용하여 사용자를 쉽게 인증할 수 있습니다. 뿐만 아니라 SaaS(Software as a Service) 공급자는 Azure AD 조직의 사용자가 서비스에 인증할 때 인증을 간소화할 수 있습니다. 또한 개발자는 사용자 또는 그룹과 같은 엔터티를 관리하기 위해 그래프 API를 사용하여 디렉터리 데이터를 쿼리할 수 있습니다.
