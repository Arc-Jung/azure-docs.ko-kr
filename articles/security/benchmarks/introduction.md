# <a name="azure-security-benchmark-introduction"></a>Azure 보안 벤치마크 소개

온-프레미스 컴퓨팅에 대한 수년간 또는 수십 년의 경험이 있을 수 있습니다. 이러한 배포를 보호하는 방법을 알고 있습니다. 그러나 클라우드는 다릅니다. 클라우드 배포가 안전한지 어떻게 알 수 있습니까? 온-프레미스 시스템과 클라우드 배포에 대한 보안 관행의 차이점은 무엇입니까?

클라우드 보안에 사용할 수 있는 많은 백서, 모범 사례, 참조 아키텍처, 웹 지침, 오픈 소스 도구, 상용 솔루션, 인텔리전스 피드 등이 있습니다. 어떤 옵션을 사용해야 합니까? 클라우드에서 허용 가능한 수준의 보안을 얻으려면 어떻게 해야 할까요? 

클라우드 배포를 보호하는 가장 좋은 방법 중 하나는 클라우드 보안 벤치마크 권장 사항에 집중하는 것입니다. 모든 서비스를 보호하기 위한 벤치마크 권장 사항은 사이버 보안 위험과 이를 관리하는 방법에 대한 기본적인 이해에서 시작됩니다. 그런 다음 클라우드 서비스 공급자의 벤치마크 보안 권장 사항을 채택하여 환경에서 특정 보안 구성 설정을 선택하는 데 도움을 주어 이러한 이해를 사용할 수 있습니다. 

Azure 보안 벤치마크에는 Azure에서 사용하는 대부분의 서비스를 보호하는 데 사용할 수 있는 영향 이 큰 보안 권장 사항 모음이 포함되어 있습니다. 이러한 권장 사항은 대부분의 Azure 서비스에 적용할 수 있으므로 "일반" 또는 "조직"으로 생각할 수 있습니다. 그런 다음 Azure 보안 벤치마크 권장 사항은 각 Azure 서비스에 대해 사용자 지정되며 이 사용자 지정 지침은 서비스 권장 사항 문서에 포함되어 있습니다. 

Azure 보안 벤치마크 설명서는 보안 제어 및 서비스 권장 사항을 지정합니다.

- **보안 제어**: Azure 보안 벤치마크 권장 사항은 보안 컨트롤에 의해 분류됩니다. 보안 제어는 네트워크 보안 및 데이터 보호와 같은 고급 공급업체에 구애받지 않는 보안 요구 사항을 나타냅니다. 각 보안 컨트롤에는 이러한 권장 사항을 구현하는 데 도움이 되는 일련의 보안 권장 사항 및 지침이 있습니다. 
- **서비스 권장 사항:** 사용 가능한 경우 Azure 서비스에 대한 벤치마크 권장 사항에는 해당 서비스에 맞게 특별히 조정된 Azure 보안 벤치마크 권장 사항이 포함됩니다. 

"제어", "벤치마크" 및 "기준선"이라는 용어는 Azure 보안 벤치마크 설명서에서 자주 사용되며 Azure에서 이러한 용어를 사용하는 방법을 이해하는 것이 중요합니다. 

| 용어 | Description | 예제 |
|--|--|--|
| 제어 | **컨트롤은** 해결해야 하는 기능 또는 활동에 대한 상위 수준 설명이며 기술 또는 구현에 만국한되지 않습니다. | 데이터 보호는 보안 제어 중 하나입니다. 이 컨트롤에는 데이터를 보호하기 위해 해결해야 하는 특정 작업이 포함되어 있습니다. |
| 벤치마크 | **벤치마크에는** Azure와 같은 특정 기술에 대한 보안 권장 사항이 포함되어 있습니다. 권장 사항은 속한 컨트롤에 의해 분류됩니다. | Azure 보안 벤치마크는 Azure 플랫폼과 관련된 보안 권장 사항으로 구성됩니다.  |
| 기초 | **기준은** 조직의 보안 요구 사항입니다. 보안 요구 사항은 벤치마크 권장 사항을 기반으로 합니다. 각 조직은 기준에 포함할 벤치마크 권장 사항을 결정합니다. | Contoso 회사는 Azure 보안 벤치마크에서 특정 권장 사항을 요구하도록 선택하여 보안 기준을 만듭니다. |
