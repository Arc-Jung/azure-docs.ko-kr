# <a name="overview-of-azure-security-baselines"></a>Azure 보안 기준 개요

Azure 보안 기준은 향상된 툴링, 추적 및 보안 기능을 통해 제품의 보안을 강화하고 환경을 보호할 때 일관된 환경을 제공하는 데 도움이 됩니다.

Azure 서비스 보안 기준은 클라우드 중심 제어 영역에 중점을 둡니다. 이러한 컨트롤은 CIS(인터넷 보안 센터)에서 설명하는 것과 같이 잘 알려진 보안 벤치마크와 일치합니다. 기본 기준은 [Azure 보안 벤치마크에](overview.md)나열된 제어 영역에 대한 지침을 제공합니다.

각 권장 사항에는 다음 정보가 포함됩니다.
- **Azure ID**: 권장 사항에 해당하는 Azure 보안 벤치마크 ID입니다.
- **권장 사항**: Azure ID 바로 다음에 따라 권장 사항은 컨트롤에 대한 상위 수준 설명을 제공합니다.
- **지침**: 권장 사항에 대한 근거 및 구현 방법에 대한 지침에 대한 링크입니다. Azure 보안 센터에서 권장 사항을 지원하는 경우 해당 정보도 나열됩니다.
- **책임**: 제어 구현에 대한 책임은 누구입니까? 가능한 시나리오는 고객 책임, Microsoft 책임 또는 공동 책임입니다.
- **Azure 보안 센터 모니터링**: 참조 링크가 있는 Azure 보안 센터에서 컨트롤을 모니터링하는지 여부입니다.

이 특정 서비스에 적용되지 않는 권장 사항을 포함한 모든 권장 사항은 Azure Security 벤치마크가 각 서비스와 어떻게 관련되는지에 대한 전체 그림을 제공하기 위해 기준에 포함됩니다. 여러 가지 이유로 적용되지 않는 컨트롤(예: IaaS/계산 중심 컨트롤(예: OS 구성 관리 관련 컨트롤)이 PaaS 서비스에 적용되지 않을 수 있습니다.
