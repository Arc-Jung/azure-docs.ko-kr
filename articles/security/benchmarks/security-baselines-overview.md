# <a name="overview-of-azure-security-baselines"></a>Azure 보안 기준 개요

Azure 보안 기준은 향상 된 도구, 추적 및 보안 기능을 통해 제품의 보안을 강화 하 고 환경을 보호할 때 일관 된 환경을 제공 하는 데 도움이 됩니다.

Azure 서비스 보안 기준은 클라우드 중심 제어 영역에 중점을 둡니다. 이러한 컨트롤은 CI (Internet Security)의 중앙에서 설명 하는 것과 같은 잘 알려진 보안 벤치 마크와 일치 합니다. Microsoft 기준은 [Azure 보안 벤치 마크](overview.md)에 나열 된 제어 영역에 대 한 지침을 제공 합니다.

각 권장 사항에는 다음 정보가 포함 됩니다.
- **AZURE ID**: 권장 사항에 해당 하는 Azure 보안 벤치 마크 ID입니다.
- **권장 사항**: Azure ID 바로 뒤에 나오는 권장 사항에 따라 컨트롤에 대 한 개략적인 설명이 제공 됩니다.
- **지침**: 권장 사항 및 구현 방법에 대 한 지침을 제공 하는 링크를 제공 합니다. Azure Security Center에서 권장 사항을 지원 하면 해당 정보도 나열 됩니다.
- **책임**: 컨트롤의 구현을 담당 하는 사용자입니다. 가능한 시나리오는 고객 책임, Microsoft 책임 또는 공유 책임입니다.
- **Azure Security Center 모니터링**: Azure Security Center에 의해 컨트롤이 모니터링 되는지 여부와 참조 링크를 포함 합니다.

이 특정 서비스에 적용 되지 않는 권장 사항을 비롯 한 모든 권장 사항은 Azure 보안 벤치 마크가 각 서비스와 어떻게 관련 되는지에 대 한 전체 개요를 제공 하는 기준에 포함 되어 있습니다. 여러 가지 이유로 적용 되지 않는 컨트롤이 있을 수 있습니다. 예를 들어, IaaS/계산 중심 컨트롤 (예: OS 구성 관리와 관련 된 컨트롤)은 PaaS 서비스에 적용 되지 않을 수 있습니다.
