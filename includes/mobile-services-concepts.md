## <a name="what-is"></a>모바일 서비스란

모바일 서비스는 Azure를 사용하여 고성능 모바일 앱을 손쉽게 디자인할 수 있는 Azure 서비스로, 사용자 앱에서 백엔드 기능을 사용할 수 있도록 하는 다양한 Azure 서비스를 통합합니다. 모바일 서비스에서 사용자 앱을 지원하기 위해 제공하는 Azure 기능은 다음과 같습니다. 

+ 앱 데이터를 저장하는 테이블의 손쉬운 프로비저닝 및 관리 
+ 사용자 앱에 푸시 알림을 전달하는 알림 서비스와 통합
+ 잘 알려진 ID 공급자와 인증 통합
+ 테이블 액세스 권한 부여를 위해 세분화된 제어
+ 서버상의 사용자 지정 비즈니스 논리
+ 타사 클라우드 서비스와 통합
+ 모바일 서비스 인스턴스 확장 기능 지원
+ 서비스 모니터링 및 로깅

모바일 서비스는 각 지원 플랫폼용 클라이언트 라이브러리를 제공하므로 이를 사용하면 사용자 플랫폼에서 손쉽게 앱을 개발할 수 있습니다.

## <a name="concepts"> </a>개념

다음은 모바일 서비스의 주요 기능 및 중요 개념입니다.

<!--![1][1]-->

+ **.NET 백 엔드:** 이러한 종류의 모바일 서비스는 ASP.NET Web API 프로젝트로 구현됩니다. 백 엔드 코드는 WebAPI 컨트롤러로 구현됩니다.
+ **응용 프로그램 키:** 모바일 서비스에 의해 생성된 고유한 값을 함께 배포된 응용 프로그램 및 클라이언트에서 생성된 요청에 표시 합니다. 임의의 클라이언트에서 모바일 서비스에 대한 액세스를 제한 하기에 유용하지만 이 키는 안전하지 않으며 응용 프로그램의 사용자를 인증 사용할 수 없습니다.
+ **인증 토큰:** 사용자가 인증 후 모바일 서비스에 의해 생성되는 액세스 토큰입니다.
+ **ID 공급자:** Facebook, Twitter, Google, Microsoft 계정 또는 Azure Active Directory와 같은 모바일 서비스 사용자를 인증하는 데 사용되는 외부 서비스입니다.
+ **JavaScript 백 엔드:** 이러한 종류의 모바일 서비스는 node.js 서비스로 구현됩니다. 백 엔드 코드는 JavaScript로 기록됩니다.
+ **권한:** 테이블 작업 또는 사용자 지정 API 호출에 필요한 최소 인증 수준입니다.
+ **푸시 알림:** 등록된 장치 또는 Azure 알림 허브를 사용하여 사용자에게 전송되는 서비스 시작 메시지입니다.
+ **배율:** 추가로 소모, 더 많은 처리 능력, 성능 및 저장소에 대한 앱의 인기가 높아지면 추가하는 기능입니다.
+ **예약된 작업:** 미리 결정된 일정 또는 요청 시에 실행되는 사용자 지정 코드가 있습니다.
+ **서버 스크립트:** JavaScript 백 엔드 모바일 서비스의 사용자 지정 비즈니스 논리 코드입니다. 이 코드는 테이블 작업(읽기, 삽입, 업데이트, 삭제), 사용자 지정 API 또는 예약된 작업에 등록됩니다.
+ **테이블:** 사용자 데이터는 SQL 데이터베이스의 테이블에 저장됩니다. 관리 포털에서 이러한 테이블을 만들 수 있습니다.
