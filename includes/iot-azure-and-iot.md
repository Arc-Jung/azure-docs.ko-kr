# Azure 및 IoT

Microsoft Azure 및 IoT(사물 인터넷)를 시작합니다. 이 문서에서는 Azure 서비스를 사용 하여 배포할 수 있는 IoT 솔루션의 공통된 특성을 설명하는 일반적인 IoT 솔루션 아키텍처를 소개합니다. 일반적인 IoT 솔루션은 수백만 개가 될 수 있는 장치들 간의 안전한 양방향 통신과 장치-클라우드 이벤트 스트림에 대해 파악하기 위해 자동화된 예측 분석 등을 사용하는 솔루션 백 엔드가 필요합니다.

### 장치 연결

일반적인 IoT 시나리오에서 장치는 저장소 및 처리를 위한 클라우드 끝점에 온도 판독값과 같은 원격 분석을 보냅니다. 또한 장치는 클라우드 끝점에서 메시지를 읽어 수신하고 클라우드-장치 명령에 응답할 수 있습니다. 예를 들어 장치는 데이터를 샘플링하는 빈도를 변경하도록 지시하는 명령을 검색할 수도 있습니다.

IoT 프로젝트의 가장 큰 당면 과제 중 하나는 장치를 안정적이고 안전하게 솔루션 백 엔드에 연결하는 방법입니다. 일반적으로 IoT 장치는 브라우저 및 모바일 앱 등 다른 클라이언트에 비해 다른 특징을 가집니다. IoT 장치:

- 운영자가 없는 내장 시스템인 경우가 많습니다.
- 물리적 액세스에 많은 비용이 드는 원격 위치에 있을 수 있습니다.
- 솔루션 백 엔드를 통해서만 연결 가능합니다.
- 전원 및 리소스 처리 제한이 있을 수 있습니다.
- 일시적이거나 느리거나 비용이 많이 드는 네트워크 연결이 있을 수 있습니다.
- 독점, 사용자 지정 또는 업계 특정 응용 프로그램 프로토콜을 사용해야 할 수 있습니다.
- 널리 사용되는 다양한 하드웨어 및 소프트웨어 플랫폼을 사용하여 만들 수 있습니다.

위의 요구 사항 외에도 모든 IoT 솔루션은 크기 조정, 보안 및 안정성을 제공해야 합니다. 연결 요구 사항의 결과 집합은 웹 컨테이너 및 메시징 브로커와 같은 기존 기술을 사용하여 구현하는데 어렵고 많은 시간이 소요됩니다.

장치는 클라우드 게이트웨이 끝점와 직접 통신하거나 장치가 클라우드 게이트웨이가 지원하는 통신 프로토콜을 사용할 수 없는 경우, 프로토콜을 변환하는 중간 게이트웨이를 통해 연결할 수 있습니다.

### 데이터 처리 및 분석

클라우드에서 IoT 솔루션 백 엔드는 다음을 수행합니다.

- 사용 중인 장치로부터 대규모로 원격 분석을 수신하고 해당 데이터를 처리하고 저장하는 방법을 결정합니다. 
- 명령을 클라우드에서 특정 장치로 보내는데 사용할 수도 있습니다.
- 장치를 프로비전하고 인프라 연결을 허용할 장치를 제어하도록 하는 장치 등록 기능을 제공합니다.
- 장치 상태를 추적하고 해당 작업을 모니터링할 수 있습니다.

IoT 솔루션은 자동 피드백 루프를 포함할 수 있습니다. 예를 들어 백 엔드의 기계 학습 모듈은 특정 장치의 온도가 정상 작동 수준보다 높다는 원격 분석으로부터 식별한 다음 정정 작업을 수행할 수 있도록 장치에 명령을 보낼 수 있습니다.
