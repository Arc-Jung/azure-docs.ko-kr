# Microsoft Azure 및 IoT(사물 인터넷)

일반적인 IoT 솔루션은 장치 간에 안전한, 양방향 통신, 수 백만까지 가능한 번호 및 응용 프로그램 백 엔드를 필요로 합니다. 응용 프로그램 백 엔드 기능의 예제는 장치-클라우드 이벤트 처리를 포함하여 자동화된 예측 분석을 사용하여 통찰력을 발견할 수 있습니다.

여러 언어 및 언어 하드웨어 플랫폼을 지원하는 Microsoft는 클라이언트 응용 프로그램을 개발하여 IoT 장치에서 실행되는 라이브러리 집합을 제공합니다. IoT 백 엔드 응용 프로그램을 구현하려면 여러 Azure 서비스를 결합하거나 [Azure IoT Suite][]에서 사용할 수 있는 미리 구성된 솔루션 중 하나를 사용할 수 있습니다. Azure에서 이 IoT 인프라를 사용하는 방법을 더 잘 이해하려면 일반적인 IoT 솔루션 아키텍처를 고려하는 것이 유용합니다.

## IoT 솔루션 아키텍처

다음의 다이어그램에서는 일반적인 IoT 솔루션 아키텍처를 보여줍니다. 특정 Azure 서비스의 이름을 포함하지 않지만 제네릭 IoT 솔루션 아키텍처의 주요 요소를 설명합니다. 다음 섹션에서는 이 솔루션의 요소에 대한 자세한 정보를 제공합니다.

### 장치 연결

일반적인 IoT 시나리오에서 장치는 저장소 및 처리용 클라우드 끝점의 온도 판독값과 같은 장치-클라우드 원격 분석 데이터를 보냅니다. 또한 장치는 클라우드 끝점에서 메시지를 읽어 수신하고 클라우드-장치 명령에 응답할 수 있습니다. 예를 들어 장치는 데이터를 샘플링하는 빈도를 변경하도록 지시하는 명령을 검색할 수도 있습니다.

IoT 솔루션의 장치 또는 데이터 소스는 간단한 네트워크에 연결된 센서에서 강력한, 독립 실행형, 계산 장치까지 다양합니다. 장치는 처리 기능, 메모리, 통신 대역폭 및 통신 프로토콜 지원이 제한될 수 있습니다.

장치는 AMQP 또는 HTTP와 같은 통신 프로토콜을 사용하여 또는 같은 프로토콜 변환과 같은 서비스를 제공하는 필드 게이트웨이와 같은 일부 중간자를 통해 클라우드 게이트웨이의 끝점과 직접 통신할 수 있습니다.

### 데이터 처리 및 분석

클라우드에서 스트림 이벤트 프로세서는 장치-클라우드 메시지를 규모별로 받고 해당 메시지를 처리하고 저장하는 방법을 결정합니다. 연결된 장치에 솔루션을 사용하면 특정 장치에 명령의 형태로 클라우드-장치 데이터를 보낼 수 있습니다. IoT 솔루션으로 장치 등록을 사용하면 장치를 프로비전하고 어떤 장치를 인프라에 연결할지 제어할 수 있습니다. 백 엔드를 사용하면 장치의 상태를 추적하고 해당 작업을 모니터링할 수 있습니다.

IoT 솔루션은 자동 피드백 루프를 포함할 수 있습니다. 예를 들어 기계 학습 모듈은 특정 장치의 온도가 정상 작동 수준 보다 높은 장치-클라우드 원격 분석 데이터에서 식별하고 올바른 조치를 취할 수 있도록 장치에 명령을 보낼 수 있습니다.

### 프레젠테이션

IoT 솔루션 다수는 해당 장치에서 수집된 데이터를 사용자에게 보여주고 분석할 수 있도록 합니다. 이러한 보기는 대시보드 또는 BI 보고서의 형태일 수 있습니다.
