### <a name="policy-for-users-in-your-directory"></a>정책: 디렉터리의 사용자에 대 한

이 액세스 패키지를 요청할 수 있는 디렉터리의 사용자 및 그룹에 정책을 사용 하려는 경우 다음이 단계를 수행 합니다.

1. 에 **액세스를 요청할 수 있는 사용자** 섹션에서 **디렉터리의 사용자에 대 한**합니다.

1. 에 **사용자 및 그룹 선택** 섹션에서 클릭 **사용자 및 그룹 추가**합니다.

1. 선택한 사용자 및 그룹 창에서 사용자 및 추가 하려는 그룹을 선택 합니다.

    ![패키지 액세스-정책 선택 사용자 및 그룹](./media/active-directory-entitlement-management-policy/policy-select-users-groups.png)

1. 클릭 **선택** 사용자 및 그룹을 추가 합니다.

1. 아래로 이동 합니다 [정책: 요청](#policy-request) 섹션입니다.

### <a name="policy-for-users-not-in-your-directory"></a>정책: 디렉터리에 없는 사용자에 대 한

이 액세스 패키지를 요청할 수 있는 디렉터리에 없는 사용자에 대해 정책을 사용 하려는 경우 다음이 단계를 수행 합니다. 디렉터리에서 허용 하도록 구성 해야 합니다 **조직 관계 공동 작업 제한** 설정 합니다.

> [!NOTE]
> 아직 해당 요청이 승인 되거나 자동으로 승인 된 디렉터리에 게스트 사용자 계정은 사용자에 대해 생성 됩니다. 게스트 초대 될 수 있지만 초대 전자 메일을 받지 않습니다. 대신, 해당 액세스 패키지 할당 전달 될 때 이메일을 받게 됩니다. 기본적으로 게스트 사용자가 더 이상 해당 이상에 모든 액세스 패키지 할당을 해당 마지막 할당 만료 되었거나 취소 되었습니다 때문에 게스트 사용자 계정 로그인에서 차단 되며 삭제 액세스 패키지 할당이 없습니다. 있는 경우에 디렉터리에 무기한으로 유지 하는 게스트 사용자가 하려는 경우 권한 관리 구성에 대 한 설정을 변경할 수 있습니다.

1. 에 **액세스를 요청할 수 있는 사용자** 섹션에서 **디렉터리에 없는 사용자에 대 한**합니다.

1. 에 **선택 외부 Azure AD 디렉터리** 섹션에서 클릭 **디렉터리 추가**합니다.

1. 도메인 이름 및 외부 검색 입력 Azure AD 디렉터리입니다.

1. 제공 된 디렉터리 이름으로 초기 도메인 올바른 디렉터리 인지 확인 합니다.

    > [!NOTE]
    > 디렉터리의 모든 사용자가이 액세스 패키지를 요청할 수 됩니다. 디렉터리, 뿐 아니라 검색에 사용 되는 도메인에 연결 된 모든 하위 도메인에서 사용자가 포함 됩니다.

    ![패키지 액세스-정책 선택 디렉터리](./media/active-directory-entitlement-management-policy/policy-select-directories.png)

1. 클릭 **추가** 디렉터리를 추가 합니다.

1. 디렉터리를 더 추가 하려면이 단계를 반복 합니다.

1. 모든 디렉터리를 추가한 후 원하는 정책에 포함할 **선택**합니다.

1. 아래로 이동 합니다 [정책: 요청](#policy-request) 섹션입니다.

### <a name="policy-none-administrator-direct-assignments-only"></a>정책: 없음 (관리자가 직접 할당만)

정책을 액세스 요청을 무시 하 고 관리자가 직접 액세스 패키지에 특정 사용자를 할당 하도록 허용 하려는 경우 다음이 단계를 수행 합니다. 사용자는 액세스 패키지 요청할 필요가 없습니다. 만료 설정을 설정할 수 있습니다 하지만 요청 설정이 없습니다.

1. 에 **액세스를 요청할 수 있는 사용자** 섹션에서 **없음 (직접 할당만 관리자**합니다.

    액세스 패키지를 만든 후에 직접 액세스 패키지에 특정 내부 및 외부 사용자를 할당할 수 있습니다. 외부 사용자를 지정 하는 경우 게스트 사용자 계정은 디렉터리에 만들어집니다.

1. 아래로 이동 합니다 [정책: 만료](#policy-expiration) 섹션입니다.

### <a name="policy-request"></a>정책: 요청

요청 섹션에서 사용자 액세스 패키지를 요청 하는 경우 승인 설정을 지정 합니다.

1. 선택한 사용자의 요청에 대 한 승인을 설정 합니다 **승인을** 토글을 **예**합니다. 요청을 자동으로 승인 하도록로 설정/해제 **No**합니다.

1. 승인이 필요한 경우는 **승인자 선택** 섹션에서 **승인자 추가**합니다.

1. 승인자 선택 창에서 하나 이상의 사용자 및/또는 그룹 승인자를 선택 합니다.

    선택한 승인자 중 하나만 요청을 승인 해야 합니다. 모든 승인자의 승인이 필요 하지 않습니다. 승인 결정 사항을 기반으로 어떤 승인자가 요청을 먼저 검토 합니다.

    ![패키지 액세스-정책 선택 승인자](./media/active-directory-entitlement-management-policy/policy-select-approvers.png)

1. 클릭 **선택** 승인자를 추가 합니다.

1. 클릭 **Show advanced 요청 설정** 추가 설정을 표시 합니다.

    ![패키지 액세스-정책 선택 디렉터리](./media/active-directory-entitlement-management-policy/policy-advanced-request.png)

1. 패키지 액세스를 요청 하려면 근거를 제공 하는 사용자를 요구 하려면 설정 **근거 필요** 하 **예**합니다.

1. 설정 액세스 패키지에 대 한 요청을 승인 하는 근거를 제공 하는 승인자를 요구 하려면 **승인자 근거 필요** 하 **예**합니다.

1. 에 **승인 요청 제한 시간 (일)** 상자에서 승인자가 요청을 검토 해야 하는 시간을 지정 합니다. 승인자가 없습니다.이 기간 (일)에서에서 검토를 요청에 만료 되 고 사용자 액세스 패키지에 대 한 다른 요청을 제출 해야 합니다.

### <a name="policy-expiration"></a>정책: 만료

만료 섹션에서는 액세스 패키지에는 사용자의 할당 만료 될 때 지정 합니다.

1. 에 **만료** 으로 설정 **액세스 패키지가 만료 된** 에 **날짜**를 **기간 (일)** , 또는 **안 함**.

    에 대 한 **날짜에**, 나중에 만료 날짜를 선택 합니다.

    에 대 한 **기간 (일)** , 0과 3660 일 사이의 숫자를 지정 합니다.

    선택에 따라, 특정 날짜는 특정 일 수가 승인 된 후에 만료 됩니다 액세스 패키지에는 사용자의 할당 또는 사용 하지 않습니다.

1. 클릭 **Show advanced 만료 설정을** 추가 설정을 표시 합니다.

1. 해당 할당을 확장할 수 있도록 설정 **사용자가 액세스를 확장할 수 있도록** 하 **예**합니다.

    확장 정책에서 허용 하는 경우 사용자 메일이 14 일도 1 일에 만료 되도록 해당 액세스 패키지 할당을 설정 하기 전에 묻는 할당을 확장 하 고 있습니다.

    ![패키지 액세스 정책 만료 설정](./media/active-directory-entitlement-management-policy/policy-expiration.png)
