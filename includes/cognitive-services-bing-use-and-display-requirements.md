이러한 사용 및 표시 요구 사항은 Bing 웹 검색, 이미지 검색, 비디오 검색, 뉴스 검색, Bing Spell Check 및 Bing Autosuggest API에 대한 호출을 통해 사용할 수 있는 콘텐츠 및 관련 정보(예: 관계, 메타데이터 및 기타 신호)의 구현에 적용됩니다. 이러한 요구 사항과 관련된 구현 세부 정보는 특정 기능 및 결과에 대한 설명서에서 확인할 수 있습니다.

## <a name="1--bing-spell-check-api-and-bing-autosuggest-api--you-must-not"></a>1.  Bing Spell Check API 및 Bing Autosuggest API -  금지 사항은 다음과 같습니다.

- Bing Spell Check API 또는 Bing Autosuggest API에서 받은 모든 데이터는 복사, 저장 또는 캐시하지 않습니다. 사용자 또는 제3자가 제공할 수 있는 새 서비스 또는 기존 서비스를 학습, 평가 또는 개선하기 위해 기계 학습 또는 이와 유사한 알고리즘 작업의 일부로 Bing Spell Check API 또는 Bing Autosuggest API에서 받은 데이터는 사용하지 않습니다.

- 사용자 또는 제3자가 제공할 수 있는 새 서비스 또는 기존 서비스를 학습, 평가 또는 개선하기 위해 기계 학습 또는 이와 유사한 알고리즘 작업의 일부로 Bing Spell Check API 또는 Bing Autosuggest API에서 받은 데이터는 사용하지 않습니다.

## <a name="2--bing-web-search-image-search-news-search-and-video-search-apis-the-search-apis"></a>2.  BING 웹 검색 API, 이미지 검색 API, 뉴스 검색 API 및 비디오 검색 API("검색 API")

**정의** - 이러한 사용 및 표시 요구 사항과 관련된 섹션 2~7에 적용되는 정의는 다음과 같습니다.

- "답변"은 응답에서 반환된 결과의 범주를 의미합니다. 예를 들어 Bing 웹 검색 API의 응답에는 웹 페이지 결과, 이미지, 비디오 및 뉴스 범주의 답변이 포함될 수 있습니다.
- "응답"은 한 번의 검색 API 호출에 대응하여 받은 모든 답변 및 관련 데이터를 의미합니다.
- "결과"는 답변에 있는 정보 항목을 의미합니다.  예를 들어 하나의 뉴스 기사와 연결된 일단의 데이터는 뉴스 답변의 결과입니다.

**인터넷 검색 환경** - 응답으로 반환된 모든 데이터는 인터넷 검색 환경에서만 사용할 수 있습니다. 인터넷 검색 환경은 해당되는 경우 다음과 같이 표시되는 콘텐츠를 의미합니다.

- 최종 사용자의 직접적 쿼리 또는 사용자의 검색 관심과 의도(예: 사용자 지정 검색 쿼리)의 다른 표시에 관련되고 빠르게 반응합니다. 

- 사용자가 데이터 원본을 찾고 탐색할 수 있습니다(예: 제공된 URL이 하이퍼링크로 구현되어 콘텐츠 또는 특성이 데이터와 함께 눈에 띄게 표시되는 클릭 가능한 링크인 경우). 

- 최종 사용자가 선택할 수 있는 여러 결과를 포함합니다(예: 뉴스 답변에서 여러 결과가 표시되거나 이보다 적은 수의 결과가 모두 반환되는 경우). 

- 검색 목적에 적합한 양으로 제한됩니다(예: 이미지 썸네일이 사용자의 디스플레이 크기에 비례한 크기로 되는 경우). 

- 최종 사용자에게 콘텐츠가 인터넷 검색 결과임을 알리는 가시적 표시를 포함합니다(예: "웹에서"라는 내용이 있는 표현). 그리고

- 검색 API에서 받은 데이터를 사용함으로써 관련 법률이나 제3자의 권리를 침해하지 않도록 적절한 조치의 다른 조합을 포함합니다.  적절한 조치에 대해 법적 자문이 필요하면 법률 고문과 상담하십시오.

인터넷 검색 환경 요구 사항에 대한 유일한 예외는 아래 섹션 7, 비표시 URL 검색에서 설명하는 URL 검색의 경우입니다.

**일반** - 금지 사항은 다음과 같습니다. 

- 응답 데이터는 복사, 저장 또는 캐시하지 않습니다(예외: 아래 "서비스 연속성" 섹션에서 허용하는 범위 내 보존). 

- 결과 콘텐츠는 수정하지 않습니다(예외: 다른 요구 사항을 위반하지 않는 방식의 서식 재지정). 

- 결과 콘텐츠와 관련된 특성과 URL은 생략하지 않습니다.

- 순서 또는 순위를 제공할 때 답변에 표시된 결과 순서는 변경하지 않습니다(누락 포함).

- 최종 사용자가 다른 콘텐츠를 응답의 일부로 믿도록 유도하는 방식으로 응답의 일부에 다른 콘텐츠를 표시하지 않습니다. 

- Microsoft에서 제공하지 않는 광고는 응답의 일부를 표시하는 페이지에 표시하지 않습니다.

- 사용자 또는 제3자가 제공할 수 있는 새 서비스 또는 기존 서비스를 학습, 평가 또는 개선하기 위해 기계 학습 또는 이와 유사한 알고리즘 작업의 일부로 검색 API에서 받은 데이터는 사용하지 않습니다.

## <a name="3-advertising"></a>3. 광고
광고(Microsoft 또는 다른 공급자 제공)는 (i) 이미지 검색, 뉴스 검색 또는 비디오 검색 API의 응답, 또는 (ii) 다른 검색 API의 이미지, 뉴스 및/또는 비디오 결과에 주로(또는 전적으로) 필터링되거나 제한되는 응답으로 표시하면 안됩니다.

## <a name="4-branding"></a>4. 브랜딩
Microsoft에서 특정 용도에 대해 서면으로 달리 명시하지 않는 한 https://go.microsoft.com/fwlink/?linkid=833278에서 설명한 대로 Microsoft에 표시되는 각 응답(또는 응답 부분)의 특성을 지정할 수 있습니다.


## <a name="5--transferring-responses"></a>5.  응답 전송
사용자가 메시지 앱 또는 소셜 미디어 게시와 같이 다른 사용자에게 응답을 전송할 수 있도록 하는 경우 다음 사항이 적용됩니다.

* 전송되는 응답에 적용되는 작업은 다음과 같습니다.
  * 전송하는 사용자에게 표시된 콘텐츠에서 수정되지 않은 콘텐츠로 구성해야 합니다(서식 변경 허용됨).
  * 메타데이터 형식의 데이터는 포함할 수 없습니다.
  * Bing에서 제공하는 인터넷 검색 환경(예: "Bing 제공", "이 이미지에 대해 Bing에서 자세히 알아보기" 또는 "이 이미지에 대해 Bing에서 자세히 탐색") 또는 Bing 로고를 통해 응답을 얻었음을 나타내는 언어를 표시해야 합니다.
  * 응답을 생성하는 데 사용된 전체 쿼리를 눈에 띄게 표시해야 합니다. 그리고
  * 직접적으로 또는 bing.com이나 m.bing.com을 통해 응답의 기본 원본에 대해 눈에 띄는 링크 또는 이와 유사한 특성을 포함해야 합니다.

* 응답 전송은 자동화할 수 없습니다.  전송은 응답을 전송하려는 의도를 명확하게 나타내는 사용자 작업으로 시작해야 합니다.
* 사용자는 전송할 사용자의 쿼리 결과로 얻은 응답만 전송할 수 있습니다.

##<a name="6-continuity-of-service"></a>6. 서비스 연속성
응답 데이터는 복사, 저장 또는 캐시하면 안됩니다. 그러나 서비스 액세스 및 데이터 렌더링의 연속성을 사용하려면 다음과 같은 경우에만 결과를 유지할 수 있습니다.

**장치** -  다음과 같은 목적으로만 보관된 결과를 사용할 수 있다면 (i) 쿼리 시간으로부터 24시간 미만 동안 또는 (ii) 최종 사용자가 업데이트된 결과에 대해 다른 쿼리를 제출할 때까지 최종 사용자가 해당 결과를 장치에서 유지할 수 있습니다.

- 최종 사용자가 해당 장치의 해당 최종 사용자에게 이전에 반환된 결과에 액세스할 수 있도록 합니다(예: 서비스 중단의 경우). 또는

- 해당 최종 사용자의 신호를 기반으로 최종 사용자의 요구를 예상하여 개인 설정된 자동 관리 쿼리에 대해 반환된 결과를 저장합니다(예: 예상되는 서비스 중단의 경우).

**서버** -  다음과 같은 목적으로만 보관된 결과만 제어하고 표시하는 서버에서 단일 최종 사용자 특정의 결과를 안전하게 유지할 수 있습니다.

- (i) 최종 사용자의 초기 쿼리 시간으로부터 21일 이상 결과를 유지할 수 없고 (ii) 최종 사용자의 새 쿼리 또는 반복 쿼리에 대한 응답으로 결과를 표시할 수 없다면, 최종 사용자가 솔루션에서 해당 사용자에게 이전에 반환된 결과의 기록 보고서를 액세스할 수 있도록 합니다. 또는

- (i) 쿼리 시간으로부터 24시간 미만 동안 또는 (ii) 최종 사용자가 업데이트된 결과에 대해 다른 쿼리를 제출할 때까지 해당 최종 사용자의 신호를 기반으로 최종 사용자의 요구를 예상하여 개인 설정된 자동 관리 쿼리에 대해 반환된 결과를 저장합니다.

보관할 때마다 특정 사용자에 대한 결과를 다른 사용자에 대한 결과와 혼합할 수 없습니다. 즉, 각 사용자의 결과를 개별적으로 유지하고 전달해야 합니다.

**일반** - 보관된 결과를 모두 표시하기 위해 수행해야 하는 작업은 다음과 같습니다.

- 쿼리가 전송된 시간에 대한 명확하고 눈에 잘 띄는 통지를 포함합니다.

- 업데이트된 결과를 다시 쿼리하고 얻기 위해 사용자에게 단추 또는 이와 유사한 수단을 제시합니다. 

- 결과 표시에서 Bing 브랜딩을 유지합니다. 그리고

- 지정된 시간 범위 내에 저장된 결과를 삭제합니다(필요한 경우 새 쿼리로 새로 고침).


##<a name="7--non-display-url-discovery"></a>7.  비표시 URL 검색
사용자 또는 고객의 쿼리에 응답하는 정보 출처의 URL을 검색하기 위한 용도로만 비인터넷 검색 환경에서 검색 응답을 사용할 수 있습니다. (i) 특정 쿼리에 대한 응답으로 해당 사용자 또는 고객에게만 제공하는 보고서 또는 이와 유사한 응답으로 이러한 URL을 복사할 수 있으며, (ii) 쿼리와 관련하여 중요하고 가치 있는 추가 콘텐츠를 포함할 수 있습니다. 이러한 사용 및 표시 요구 사항과 관련된 섹션 2~6의 요구 사항은 다음을 제외하고는 비표시 사용에 적용되지 않습니다. 

* 위에 설명한 제한된 URL 복사 이외의 검색 응답에서는 데이터 또는 콘텐츠를 캐시, 복사 또는 저장하거나 파생시키면 안됩니다.
* 검색 API에서 받은 데이터(URL 포함) 사용이 관련 법률이나 제3자의 권리를 침해하지 않는지 확인해야 합니다. 그리고
* 사용자 또는 제3자가 제공할 수 있는 서비스를 학습, 평가 또는 개선하기 위해 기계 학습 또는 이와 유사한 알고리즘 작업의 일부로 검색 API에서 받은 데이터(URL 포함)는 사용하지 않습니다.
