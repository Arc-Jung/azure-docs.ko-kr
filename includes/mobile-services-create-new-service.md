다음 단계에 따라 새 모바일 서비스를 만드세요.

1.  [관리 포털][관리 포털]에 로그인합니다.

2.  탐색 창 맨 아래쪽에서 **+새로 만들기**를 클릭합니다.

    ![][0]

3.  **계산**, **모바일 서비스**를 확장한 후 **만들기**를 클릭합니다.

    ![][1]

    **새 모바일 서비스** 대화 상자가 표시됩니다.

4.  **Create a mobile service** 페이지에서 **Create a free 20 MB SQL Database**, **Node.js** 런타임을 차례로 선택한 후 **URL** 입력란에 새 모바일 서비스를 위한 하위 도메인 이름을 입력하고 이름 확인을 기다립니다. 이름 확인이 완료되면 오른쪽 화살표 단추를 클릭하여 다음 페이지로 이동합니다.

    ![][2]

    **데이터베이스 설정 지정** 페이지가 표시됩니다.

    <div class="dev-callout">

    **참고**
    이 자습서에는 새 SQL 데이터베이스 인스턴스와 서버를 만드는 과정이 포함되어 있으며, 이 새 데이터베이스는 다른 SQL 데이터베이스 인스턴스처럼 다시 사용하고 관리할 수 있습니다. 새 모바일 서비스로 같은 지역에 데이터베이스를 이미 가지고 있다면 대신 **기존 데이터베이스 사용**을 선택할 수 있습니다. 다른 지역에 있는 데이터베이스는 추가 대역폭 비용과 대기 시간이 높으므로 사용을 권장하지 않습니다.

    </div>

5.  **이름**에 새 데이터베이스 이름을 입력한 후 **로그인 이름**에 새 SQL 데이터베이스 서버의 관리자 로그인 이름을 입력하고, 암호를 입력한 후 확인하고, 확인 단추를 클릭하여 완료합니다.

    ![][3]

    <div class="dev-callout">

    **참고**
    입력한 암호가 최소 요구 사항을 충족하지 못하거나 실제 암호와 일치하지 않으면 경고가 표시됩니다.
    지정한 관리자 로그인 이름과 암호를 적어 놓을 것을 권장합니다. SQL 데이터베이스 인스턴스나 서버를 다시 사용할 때 이 정보가 필요합니다.

    </div>

이제 모바일 앱에 사용할 수 있는 새 모바일 서비스를 만들었습니다.
