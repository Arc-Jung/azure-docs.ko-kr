1. Azure 포털에서 **새로 만들기** **>** **네트워킹** **>** **로컬 네트워크 게이트웨이**로 이동합니다.

	![로컬 네트워크 게이트웨이 만들기](./media/vpn-gateway-add-lng-rm-portal-include/addlng250.png)

2. **로컬 네트워크 게이트웨이 블레이드 만들기**에서 로컬 네트워크 게이트웨이 개체의 **이름**을 지정합니다.
 
3. 게이트웨이에 대한 **IP 주소**를 지정합니다. 연결하려는 외부 VPN 장치의 IP 주소입니다. NAT 뒤에 있을 수 없고 Azure에서 연결될 수 있어야 합니다.

4. **주소 공간**은 로컬(일반적으로 온-프레미스) 네트워크에서 주소 범위를 가리킵니다. 주소 공간 범위를 여러 개 추가할 수 있습니다. 여기에 입력한 범위는 게이트웨이를 통해 통신하는 가상 네트워크에 사용 중인 주소 공간 범위와 겹칠 수 없습니다. Azure 가상 네트워크 주소 공간 뿐만 아니라 온-프레미스 구성을 사용하여 조정해야 합니다.
 
5. **구독**의 경우 올바른 구독이 표시되어 있는지 확인합니다.

6. **리소스 그룹**의 경우 사용하려는 리소스 그룹을 선택합니다. 새 리소스 그룹을 만들거나 이미 만든 리소스 그룹을 선택할 수 있습니다. 새 리소스 그룹을 만들려면 상자에 이름을 입력합니다. 이미 만든 리소스 그룹을 선택하려면 **리소스 그룹**을 클릭하여 **리소스 그룹** 블레이드를 열고 사용 하려는 리소스 그룹을 선택합니다.

7. **위치**에 새 로컬 네트워크 게이트웨이를 만드는 경우 가상 네트워크 게이트웨이와 동일한 위치를 사용할 수 있습니다. 그러나 필수적이지 않습니다. 로컬 네트워크 게이트웨이는 다른 위치일 수 있습니다.

8. 대시보드에서 쉽게 이 로컬 네트워크 게이트웨이를 찾으려는 경우 "대시보드에 고정"을 선택된 상태로 둡니다.

9. **만들기**를 클릭하여 로컬 네트워크 게이트웨이를 만듭니다. "로컬 네트워크 게이트웨이 배포"가 대시보드에 표시됩니다.

10. 로컬 네트워크 게이트웨이를 만들 경우 볼 수 있도록 포털에서 열립니다.
