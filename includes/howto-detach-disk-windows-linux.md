## <span id="concepts"></span> </a>개념

Azure의 가상 컴퓨터는 운영 체제 디스크, 로컬 임시 디스크, 선택적 데이터 디스크 등 다양한 유형의 디스크를 사용합니다. 가상 컴퓨터에 데이터 디스크를 연결하여 응용 프로그램 데이터를 저장할 수 있습니다. 데이터 디스크는 고유한 컴퓨터를 사용하여 로컬에서 또는 Azure를 사용하여 클라우드에 만들 수 있는 VHD(가상 하드 디스크)입니다.

원할 때마다 데이터 디스크를 연결하고 분리할 수 있지만, 컴퓨터 크기에 따라 가상 컴퓨터에 연결할 수 있는 디스크 수가 제한됩니다.

가상 컴퓨터에 연결된 데이터 디스크가 더 이상 필요하지 않은 경우 쉽게 분리할 수 있습니다. 이 프로세스는 저장소에서 디스크를 삭제하지 않습니다. 디스크의 기존 데이터를 다시 사용하려는 경우 쉽게 동일한 가상 컴퓨터에 디스크를 다시 연결하거나 새 가상 컴퓨터에 연결할 수 있습니다.

데이터 디스크 사용에 대한 자세한 내용은 [디스크 및 이미지 관리][]를 참조하세요.

## <span id="finddisks"></span> </a>방법: 가상 컴퓨터에 연결된 디스크 찾기

가상 컴퓨터의 디스크 페이지나 대시보드를 사용하여 가상 컴퓨터에 연결된 디스크를 찾을 수 있습니다.

### 대시보드를 사용하여 연결된 디스크에 대한 정보 찾기

1.  아직 로그인하지 않은 경우 [Azure 관리 포털][]에 로그인합니다.

2.  **가상 컴퓨터**를 클릭하고 해당 가상 컴퓨터를 선택합니다.

3.  **대시보드**를 클릭합니다. 가상 컴퓨터의 대시보드에서 연결된 디스크 수와 디스크 이름을 찾을 수 있습니다. 다음 예제에서는 가상 컴퓨터에 연결된 데이터 디스크 한 개를 보여 줍니다.

    ![데이터 디스크 찾기][]

**참고:** 적어도 하나의 디스크가 모든 가상 컴퓨터에 연결되어 있습니다. 가상 컴퓨터를 삭제하지 않으면 분리할 수 없는 운영 체제 디스크가 각 가상 컴퓨터에 연결되어 있습니다. 로컬 임시 디스크는 영구적이지 않으므로 디스크 섹션에 표시되지 않습니다.

### 가상 컴퓨터의 디스크 페이지를 사용하여 연결된 디스크에 대한 정보 찾기

1.  아직 로그인하지 않은 경우 [Azure 관리 포털][]에 로그인합니다.

2.  **Virtual Machines**를 클릭한 후 **Disks**를 클릭합니다. 이 페이지에는 가상 컴퓨터에 사용할 수 있는 모든 디스크 목록 및 가상 컴퓨터에서 사용 중인 디스크가 표시됩니다. 이 목록은 운영 체제 디스크와 데이터 디스크가 결합된 목록입니다. 가상 컴퓨터에 연결된 두 유형의 디스크를 구분하려면 대시보드를 사용합니다.

    **참고:** 가상 컴퓨터에 새 데이터 디스크를 연결하는 경우 디스크에 사용되는 .vhd 파일에 이름을 할당할 수 있지만 디스크 이름은 Azure에서 할당합니다. 이름은 클라우드 서비스 이름, 가상 컴퓨터 이름 및 숫자 식별자로 구성됩니다.

## <span id="detachdisk"></span> </a>방법: 데이터 디스크 분리

분리하려는 디스크의 이름을 찾은 후 다음 단계에 따라 가상 컴퓨터에서 디스크를 분리할 수 있습니다.

1.  아직 로그인하지 않은 경우 Azure 관리 포털에 로그인합니다.

2.  **Virtual Machines**를 클릭하고 분리하려는 데이터 디스크가 있는 가상 컴퓨터를 선택한 후 **Detach The Disk**를 클릭합니다.

3.  데이터 디스크를 선택하고 확인 표시를 클릭하여 분리합니다.

    ![디스크 분리 세부 정보][]

    디스크가 저장소에 유지되지만 더 이상 가상 컴퓨터에 연결되어 있지 않습니다.

이제 동일한 가상 컴퓨터나 새 컴퓨터에 디스크를 다시 연결할 수 있습니다. 지침은 [가상 컴퓨터에 데이터 디스크를 연결하는 방법][]을 참조하세요.
