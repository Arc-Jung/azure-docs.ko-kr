디스크 버스팅은 프리미엄 SSD에 지원됩니다. 버스팅은 프리미엄 SSD 디스크 크기 <= 512 GiB (P20 이하)에서 지원됩니다. 이러한 디스크 크기는 최상의 노력으로 버스팅을 지원하고 크레딧 시스템을 사용하여 버스팅을 관리합니다. 디스크 트래픽이 디스크 크기에 대한 프로비저닝된 성능 목표보다 낮을 때마다 버스트 버킷에 크레딧이 누적되고 트래픽이 대상을 초과하여 버스트될 때 크레딧을 사용합니다. 디스크 트래픽은 프로비저닝된 대상의 IOPS 및 대역폭에 대해 추적됩니다. 디스크 버스팅은 IOPS 또는 처리량에 대한 VM(가상 머신) 크기 제한을 우회하지 않습니다.

디스크 버스팅은 디스크를 지원하는 디스크 크기의 새 배포에서 기본적으로 활성화됩니다. 기존 디스크 크기는 디스크 버스팅을 지원하는 경우 다음 방법 중 하나를 통해 버스팅을 활성화할 수 있습니다.

- 디스크를 분리하고 다시 연결합니다.
- VM을 중지하고 시작합니다.

## <a name="burst-states"></a>버스트 상태

모든 버스트 적용 디스크 크기는 디스크가 가상 컴퓨터에 연결될 때 전체 버스트 크레딧 버킷으로 시작됩니다. 버스팅의 최대 지속 시간은 버스트 크레딧 버킷의 크기에 따라 결정됩니다. 크레딧 버킷 크기까지만 사용하지 않은 크레딧을 누적할 수 있습니다. 언제든지 디스크 버스트 크레딧 버킷은 다음 세 가지 상태 중 하나에 있을 수 있습니다. 

- 디스크 트래픽이 프로비저닝된 성능 대상보다 적게 사용하는 경우 발생합니다. 디스크 트래픽이 IOPS 또는 대역폭 대상 또는 둘 다 를 초과하는 경우 크레딧을 누적할 수 있습니다. 전체 디스크 대역폭을 소비할 때에도 IO 크레딧을 누적할 수 있습니다.  

- 디스크 트래픽이 프로비저닝된 성능 대상 보다 더 많은 것을 사용하는 경우 감소합니다. 버스트 트래픽은 IOPS 또는 대역폭의 크레딧을 독립적으로 사용합니다. 

- 디스크 트래픽이 프로비저닝된 성능 대상에 정확히 있는 경우 상수로 유지됩니다. 

버스트 사양과 함께 버스트 지원을 제공하는 디스크 크기는 아래 표에 요약되어 있습니다.

## <a name="example-scenarios"></a>예제 시나리오

이 작동 방식을 더 잘 설명하려면 다음과 같은 몇 가지 예제 시나리오를 소개합니다.

- 디스크 버스팅의 이점을 누릴 수 있는 한 가지 일반적인 시나리오는 OS 디스크에서 VM 부팅 및 응용 프로그램 시작이 더 빠르다는 것입니다. 예를 들어 8 GiB OS 이미지가 있는 Linux VM을 예로 들어 보겠습니다. P2 디스크를 OS 디스크로 사용하는 경우 프로비저닝된 대상은 120 IOPS 및 25MBps입니다. VM이 시작되면 부팅 파일을 로드하는 OS 디스크에 읽기 스파이크가 표시됩니다. 버스팅이 도입되면 최대 3500 IOPS 및 170MBps의 최대 버스트 속도로 읽을 수 있어 로드 시간을 6배 이상 단축할 수 있습니다. VM 부팅 후 응용 프로그램에서 대부분의 데이터 작업이 연결된 데이터 디스크에 대해 사용되므로 OS 디스크의 트래픽 수준은 일반적으로 낮습니다. 트래픽이 프로비저닝된 목표보다 낮으면 크레딧이 누적됩니다.

- 원격 가상 데스크톱 환경을 호스팅하는 경우 활성 사용자가 AutoCAD와 같은 응용 프로그램을 시작할 때마다 OS 디스크에 대한 트래픽 읽기가 크게 증가합니다. 이 경우 버스트 트래픽은 누적된 크레딧을 소비하여 프로비저닝된 대상을 넘어 훨씬 빠르게 응용 프로그램을 시작할 수 있습니다.

- P1 디스크에는 120IOPS 및 25MBps의 프로비전된 대상이 있습니다. 디스크의 실제 트래픽이 지난 1초 간격에서 100 IOPS 및 20MBps인 경우 사용하지 않은 20개의 IO와 5MB는 디스크의 버스트 버킷에 적립됩니다. 버스트 버킷의 크레딧은 트래픽이 프로비저닝된 대상을 초과할 때 최대 버스트 한도까지 나중에 사용할 수 있습니다. 최대 버스트 제한은 사용할 버스트 크레딧이 있는 경우에도 디스크 트래픽의 한도를 정의합니다. 이 경우 크레딧 버킷에 10,000개의 IO가 있더라도 P1 디스크는 초당 최대 3,500 IO 버스트 이상을 발행할 수 없습니다.
