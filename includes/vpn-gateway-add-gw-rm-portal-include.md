1. 포털에서 **새로 만들기**로 이동한 다음 **네트워킹**을 클릭합니다. 목록에서 **가상 네트워크 게이트웨이**를 선택합니다.

	![게이트웨이](./media/vpn-gateway-add-gw-rm-portal-include/creategw250.png)

2. **가상 네트워크 게이트웨이 만들기** 블레이드 **이름** 필드에 게이트웨이의 이름을 지정합니다. 이는 게이트웨이 서브넷 이름 지정과 동일하지는 않습니다. 게이트웨이 개체의 이름입니다.
 
3. 다음으로 이 게이트웨이를 배포하려는 가상 네트워크를 선택합니다. 화살표를 클릭하여 **가상 네트워크 선택** 블레이드를 연 다음 VNet을 클릭합니다. VNet을 목록에 표시하기 위해서는 유효한 게이트웨이 서브넷이 이미 있어야 합니다.

4. 공용 IP 주소를 선택합니다. 화살표를 클릭하여 **공용 IP 주소 블레이드 선택**을 엽니다. **새로 만들기**를 클릭하여 **공용 IP 주소 블레이드 만들기**를 엽니다. 공용 IP 주소의 이름을 입력합니다. 이는 IP 주소를 요청하는 것이 아닙니다. IP 주소는 동적으로 할당됩니다. 대신, 이는 주소가 할당될 IP 주소 개체의 이름입니다. **확인**을 클릭하여 변경 내용을 저장합니다.

5. **게이트웨이 유형**의 경우 구성에 대해 지정된 게이트웨이 유형을 선택합니다.

6. **VPN 유형**의 경우 구성에 대해 지정된 VPN 유형을 선택합니다.

7. **구독**의 경우 올바른 구독을 선택하는지 확인합니다.

8. **리소스 그룹**의 경우 리소스 그룹은 선택한 가상 네트워크에 의해 결정됩니다.

9. **위치**의 경우 리소스 그룹과 VNet이 모두 있는 위치를 표시하는지 확인합니다.

10. 게이트웨이를 대시보드에 표시하려는 경우 **대시보드에 고정**을 선택할 수 있습니다. **만들기**를 클릭하여 게이트웨이 만들도록 합니다. 대시보드에 "가상 네트워크 게이트웨이 배포" 타일이 표시됩니다. 하나의 게이트웨이를 만드는 데 최대 45분이 걸릴 수 있습니다. 백그라운드에서 많은 시간이 진행됩니다. 완료 상태를 확인하기 위해 포털 페이지를 새로 고쳐야 할 수 있습니다.

	
	![게이트웨이](./media/vpn-gateway-add-gw-rm-portal-include/deployvnetgw150.png)

11. 게이트웨이를 만든 후 포털에서 가상 네트워크를 살펴보면 게이트웨이에 할당된 IP 주소를 볼 수 있습니다. 게이트웨이가 연결된 장치로 표시됩니다. 연결된 장치(가상 네트워크 게이트웨이)를 클릭하여 자세한 내용을 볼 수 있습니다.
