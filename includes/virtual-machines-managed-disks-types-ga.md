## <a name="premium-ssd"></a>프리미엄 SSD

Azure 프리미엄 SSD는 IO(입출력) 집약적 워크로드가 있는 VM(가상 머신)에 대기 시간이 짧은 고성능 디스크를 지원합니다. 프리미엄 스토리지 디스크의 속도와 성능을 활용하기 위해 기존 VM 디스크를 프리미엄 SSD로 마이그레이션할 수 있습니다. 프리미엄 SSD는 중요 업무용 프로덕션 애플리케이션에 적합합니다. 프리미엄 SSD는 프리미엄 스토리지 호환 VM 시리즈와 함께 사용할 수 있습니다.

프리미엄 저장소 호환 크기를 포함하여 Azure for Windows의 개별 VM 유형 및 크기에 대한 자세한 내용은 [Windows VM 크기를](../articles/virtual-machines/windows/sizes.md)참조하십시오. 프리미엄 스토리지 호환 크기를 포함하여 Linux용 Azure의 개별 VM 유형 및 크기에 대해 자세히 알아보려면 [Linux VM 크기를](../articles/virtual-machines/linux/sizes.md)참조하십시오. 이러한 문서 중 하나에서 각 개별 VM 크기 문서를 확인하여 프리미엄 저장소 호환 되는지 확인해야 합니다.

### <a name="disk-size"></a>디스크 크기
[!INCLUDE [disk-storage-premium-ssd-sizes](disk-storage-premium-ssd-sizes.md)]

Premium Storage 디스크를 프로비전하면 표준 스토리지와 달리, 해당 디스크의 용량, IOPS 및 처리량이 보장됩니다. 예를 들어 P50 디스크를 만들면 Azure에서 해당 디스크에 스토리지 용량 4,095GB, 7,500 IOPS, 250MB/초 처리량이 프로비전됩니다. 애플리케이션에서 용량 및 성능의 전체 또는 일부를 사용할 수 있습니다. 프리미엄 SSD 디스크는 이전 표에 설명된 낮은 한 자릿수 밀리초 대기 시간 및 대상 IOPS 및 처리량을 99.9%로 제공하도록 설계되었습니다.

## <a name="bursting"></a>파열

P30보다 작은 프리미엄 SSD 크기는 이제 디스크 버스팅을 제공하며 디스크당 IOPS를 최대 3,500Mbps, 대역폭은 최대 170Mbps까지 버스트할 수 있습니다. 버스팅은 자동화되어 신용 시스템을 기반으로 작동합니다. 디스크 트래픽이 프로비저닝된 성능 목표보다 낮을 때 크레딧이 버스트 버킷에 자동으로 누적되고 트래픽이 대상을 초과하여 최대 버스트 한도까지 버스트될 때 크레딧이 자동으로 소비됩니다. 최대 버스트 제한은 사용할 버스트 크레딧이 있는 경우에도 디스크 IOPS & 대역폭의 천장을 정의합니다. 디스크 버스팅은 IO 패턴의 예측할 수 없는 변경에 대한 허용 오차를 더 잘 제공합니다. 당신은 가장 뾰족한 트래픽OS 디스크 부팅 및 응용 프로그램에 사용할 수 있습니다.    

기본적으로 해당 디스크 크기의 새 배포에서 사용자 작업이 필요하지 않은 디스크 버스팅 지원이 활성화됩니다. 적용 가능한 크기의 기존 디스크의 경우 디스크 분리 및 다시 연결또는 연결된 VM을 중지하고 다시 시작하는 두 가지 옵션 중 하나를 사용하여 버스팅을 활성화할 수 있습니다. 디스크가 최대 버스트 제한 30분에서 최대 지속 시간을 지원하는 가상 컴퓨터에 연결된 경우 모든 버스트 적용 디스크 크기는 전체 버스트 크레딧 버킷으로 시작됩니다. Azure Disks에서 버스트 작업 방법에 대한 자세한 내용은 [프리미엄 SSD 버스팅](../articles/virtual-machines/linux/disk-bursting.md)을 참조하십시오. 

### <a name="transactions"></a>의

프리미엄 SSD의 경우 처리량 256KiB 이하의 각 I/O 작업은 단일 I/O 작업으로 간주됩니다. 처리량의 256KiB보다 큰 I/O 작업은 크기 256 KiB의 다중 I/O로 간주됩니다.

## <a name="standard-ssd"></a>표준 SSD

Azure 표준 SSD는 더 낮은 IOPS 수준에서 일관된 성능이 필요한 워크로드에 최적화된 비용 효율적 스토리지 옵션입니다. 특히 표준 SSD는 온-프레미스 HDD 솔루션에서 실행 중인 다양한 워크로드에 문제가 발생해서 클라우드로 전환하고자 하는 경우에 적합한 입문용 환경을 제공합니다. 표준 HDD에 비해 표준 SSD는 더 나은 가용성, 일관성, 안정성 및 대기 시간을 제공합니다. 표준 SSD는 웹 서버, 낮은 IOPS 애플리케이션 서버, 가끔 사용되는 엔터프라이즈 애플리케이션 및 개발/테스트 워크로드에 적합합니다. 표준 HDD와 마찬가지로 표준 SSD는 모든 Azure VM에서 사용할 수 있습니다.

### <a name="disk-size"></a>디스크 크기
[!INCLUDE [disk-storage-standard-ssd-sizes](disk-storage-standard-ssd-sizes.md)]

표준 SSD는 한 자리 수 밀리초 대기 시간 및 IOPS 및 처리량을 이전 표에 설명된 99%까지 제공하도록 설계되었습니다. 실제 IOPS 및 처리량은 트래픽 패턴에 따라 달라질 수 있습니다. 표준 SSD는 HDD 디스크보다 더 일관적인 성능과 더 짧은 대기 시간을 제공합니다.

### <a name="transactions"></a>의

표준 SSD의 경우 처리량의 256KiB 이하의 각 I/O 작업은 단일 I/O 작업으로 간주됩니다. 처리량의 256KiB보다 큰 I/O 작업은 크기 256 KiB의 다중 I/O로 간주됩니다. 이러한 트랜잭션은 청구에 영향을 미칩니다.

## <a name="standard-hdd"></a>표준 HDD

Azure 표준 HDD는 대기 시간에 민감하지 않은 워크로드를 실행하는 VM에 안정적이고 저렴한 디스크를 지원합니다. 표준 스토리지를 사용하는 경우 데이터는 HDD(하드 디스크 드라이브)에 저장됩니다. 표준 HDD 디스크의 대기 시간, IOPS 및 처리량은 SSD 기반 디스크에 비해 더 광범위하게 다를 수 있습니다. 표준 HDD 디스크는 대부분의 IO 작업에 대해 10ms 미만의 쓰기 대기 시간을 제공하고 20ms 미만의 대기 시간을 읽도록 설계되었지만 실제 성능은 IO 크기 및 워크로드 패턴에 따라 다를 수 있습니다. VM으로 작업할 때는 개발/테스트 시나리오와 덜 중요한 워크로드에 표준 HDD 디스크를 사용할 수 있습니다. 표준 HDD는 모든 Azure 리전에서 사용할 수 있으며 모든 Azure VM에서 사용할 수 있습니다.

## <a name="billing"></a>결제

관리 디스크를 사용할 때 적용되는 청구 고려 사항은 다음과 같습니다.

- 디스크 유형
- 관리 디스크 크기
- 스냅샷
- 아웃바운드 데이터 전송
- 트랜잭션 수

**관리 디스크 크기:** 관리 디스크의 요금은 프로비저닝된 크기에 따라 청구됩니다. Azure에서 프로비저닝된 크기(반올림)는 가장 가깝게 제안되는 디스크 크기에 매핑됩니다. 제안되는 디스크 크기에 대한 자세한 내용은 이전 표를 참조하세요. 각 디스크는 지원되는 프로비저닝된 디스크 크기 제안에 매핑되고 이에 따라 요금이 청구됩니다. 예를 들어 200GiB 표준 SSD를 프로비저닝한 경우 E15(256GiB)의 디스크 크기 제안에 매핑됩니다. 프로비전된 디스크에 대한 청구는 Premium Storage 제품의 월별 가격을 사용하여 시간당 비례합니다. 예를 들어 E10 디스크를 프로비전하고 20시간 후 삭제한 경우 20시간에 비례하여 E10 제품에 대해 청구됩니다. 이는 디스크에 기록되는 실제 데이터 양에 관계없이 적용됩니다.

**스냅샷**: 스냅샷은 사용된 크기에 따라 청구됩니다. 예를 들어 프로비저닝된 용량이 64GiB이고 실제 사용된 데이터 크기가 10GiB인 관리 디스크의 스냅샷을 만들면 사용된 10GiB의 데이터 크기에 대해서만 스냅샷 요금이 청구됩니다.
