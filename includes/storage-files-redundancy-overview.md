Azure 파일의 데이터를 데이터 손실 또는 손상으로부터 보호하기 위해 모든 Azure 파일 공유는 기록될 때 각 파일의 여러 복사본을 저장합니다. 워크로드의 요구 사항에 따라 추가 중복 도를 선택할 수 있습니다. Azure Files는 현재 다음과 같은 데이터 중복 옵션을 지원합니다.

- **로컬 중복**: LRS라고도 하는 로컬 중복 저장소는 모든 파일이 Azure 저장소 클러스터 내에 세 번 저장된다는 것을 의미합니다. 이렇게 하면 잘못된 디스크 드라이브와 같은 하드웨어 오류로 인한 데이터 손실을 방지할 수 있습니다.
- **영역 중복**: ZRS라고도 하는 영역 중복 저장소는 모든 파일이 세 개의 서로 다른 Azure 저장소 클러스터에 대해 세 번 저장된다는 것을 의미합니다. 세 개의 서로 다른 Azure 저장소 클러스터는 로컬 중복 저장소와 마찬가지로 각 파일을 세 번 저장하며 서로 다른 Azure *가용성 영역에서*물리적으로 격리됩니다. 가용성 영역은 Azure 지역 내의 고유한 물리적 위치입니다. 각 영역은 독립된 전원, 냉각 및 네트워킹을 갖춘 하나 이상의 데이터 센터로 구성됩니다. 저장소에 대한 쓰기는 세 가지 가용성 영역 모두의 저장소 클러스터에 기록될 때까지 허용되지 않습니다. 
- **지리적 중복**: GRS라고도 하는 지리적 중복 저장소는 파일이 기본 지역의 Azure 저장소 클러스터 내에 세 번 저장된다는 점에서 로컬 중복 저장소와 같습니다. 그런 다음 모든 쓰기가 Microsoft 정의 보조 지역으로 비동기적으로 복제됩니다. 지리적 중복 저장소는 두 Azure 지역 간에 분산된 데이터의 복사본 6개를 제공합니다. 자연 재해 또는 기타 유사한 이벤트로 인해 Azure 지역이 영구적으로 손실되는 등의 큰 재해가 발생할 경우 Microsoft는 장애 조치(failover)를 수행하여 실제로 보조 데이터베이스가 기본이 되어 모든 작업에 서비스를 제공합니다. 주 영역과 보조 지역 간의 복제는 비동기이므로 큰 재해가 발생할 경우 아직 보조 지역으로 복제되지 않은 데이터가 손실됩니다. 지역 중복 저장소 계정의 수동 장애 조치(failover)를 수행할 수도 있습니다.
- **지리적 영역 중복**: GZRS라고도 하는 지리적 영역 중복 저장소는 파일이 기본 영역의 3개의 고유한 저장소 클러스터에 걸쳐 9번 저장된다는 점에서 영역 중복 저장소와 같습니다. 그런 다음 모든 쓰기가 Microsoft 정의 보조 지역으로 비동기적으로 복제됩니다. 지리적 영역 중복 저장소에 대한 장애 조치 프로세스는 지리적 중복 저장소와 동일하게 작동합니다.

표준 Azure 파일 공유는 네 가지 중복 유형을 모두 지원하지만 프리미엄 Azure 파일은 로컬 중복 및 영역 중복 저장소만 지원합니다.

범용 버전 2(GPv2) 저장소 계정은 Azure Files에서 지원되지 않는 두 가지 추가 중복 옵션을 제공합니다: RA-GRS라고도 하는 액세스 가능한 지리적 중복 저장소 읽기 및 액세스 가능한 지리적 영역 중복 저장소를 읽는 경우가 많습니다. RA-GZRS라고 합니다. 이러한 옵션을 설정하여 저장소 계정에 Azure 파일 공유를 프로비전할 수 있지만 Azure Files는 보조 지역의 읽기를 지원하지 않습니다. 읽을 수 있는 지리적 또는 지리적 영역 중복 저장소 계정에 배포된 Azure 파일 공유는 각각 지리적 중복 또는 지역 영역 중복 저장소로 청구됩니다.
