데이터 손실 또는 손상 으로부터 Azure 파일 공유의 데이터를 보호 하기 위해 모든 Azure 파일 공유는 작성 될 때 각 파일의 여러 복사본을 저장 합니다. 워크 로드의 요구 사항에 따라 추가 수준의 중복성을 선택할 수 있습니다. 현재 Azure Files는 다음과 같은 데이터 중복 옵션을 지원 합니다.

- **로컬 중복**: LRS 라고도 하는 로컬 중복 저장소는 모든 파일이 Azure storage 클러스터 내에서 3 번 저장 됨을 의미 합니다. 이는 잘못 된 디스크 드라이브와 같은 하드웨어 오류로 인 한 데이터 손실을 방지 합니다.
- **영역 중복**: ZRS 라고도 하는 영역 중복 저장소는 세 개의 고유한 Azure storage 클러스터에 모든 파일이 3 번 저장 됨을 의미 합니다. 세 개의 고유한 Azure storage 클러스터는 각각 로컬 중복 저장소와 마찬가지로 파일을 세 번 저장 하며, 서로 다른 Azure *가용성 영역*에 물리적으로 격리 되어 있습니다. 가용성 영역은 Azure 지역 내에서 고유한 물리적 위치입니다. 각 영역은 독립된 전원, 냉각 및 네트워킹을 갖춘 하나 이상의 데이터 센터로 구성됩니다. 저장소에 대 한 쓰기는 세 개의 모든 가용성 영역에서 저장소 클러스터에 기록 될 때까지 허용 되지 않습니다. 
- **지역 중복**: 지역 중복 저장소 (종종 GRS 라고도 함)는 로컬 중복 저장소와 비슷하며,이 파일은 주 지역의 Azure storage 클러스터 내에서 3 번 저장 됩니다. 그러면 모든 쓰기가 Microsoft에서 정의한 보조 지역에 비동기식으로 복제 됩니다. 지역 중복 저장소는 두 Azure 지역 간에 분산 된 데이터의 복사본 6 개를 제공 합니다. 자연 재해 또는 기타 유사한 이벤트로 인 한 Azure 지역의 영구 손실과 같은 심각한 재해가 발생 하는 경우 Microsoft는 장애 조치를 수행 하 여 모든 작업을 수행 하는 보조 복제본이 주 복제본이 되도록 장애 조치 (failover)를 수행 합니다. 주 지역과 보조 지역 간 복제는 비동기적 이므로, 심각한 재해가 발생 하는 경우 보조 지역에 아직 복제 되지 않은 데이터는 손실 됩니다. 지역 중복 저장소 계정의 수동 장애 조치 (failover)를 수행할 수도 있습니다.
- **지역 중복**: 지리적 영역 중복 저장소 (종종 GZRS 라고도 함)는 주 지역에서 3 개의 개별 저장소 클러스터에 걸쳐 파일이 9 번 저장 되는 영역 중복 저장소와 비슷합니다. 그러면 모든 쓰기가 Microsoft에서 정의한 보조 지역에 비동기식으로 복제 됩니다. 지역 중복 저장소에 대 한 장애 조치 (failover) 프로세스는 지역 중복 저장소에 대해 수행 하는 것과 동일 하 게 작동 합니다.

표준 Azure 파일 공유는 4 가지 중복성 유형을 모두 지원 하 고, 프리미엄 Azure 파일 공유는 로컬 중복 및 영역 중복 저장소만 지원 합니다.

GPv2 (범용 버전 2) 저장소 계정은 Azure Files에서 지원 되지 않는 두 가지 추가 중복성 옵션을 제공 합니다. 액세스 가능한 지역 중복 저장소 읽기 (일반적으로 RA-GRS 라고도 함) 및 액세스 가능한 지역 중복 저장소 읽기 (자주) RA-GZRS 라고 합니다. 이러한 옵션 집합을 사용 하 여 저장소 계정에서 Azure 파일 공유를 프로 비전 할 수 있지만 보조 지역에서 읽기를 지원 하지 Azure Files. 읽기 액세스 가능 지역 또는 지역 영역 중복 저장소 계정에 배포 된 Azure 파일 공유는 각각 지역 중복 또는 지역 중복 저장소로 청구 됩니다.
