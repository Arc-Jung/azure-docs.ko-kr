## 테이블 서비스 정의

Azure 테이블 저장소 서비스는 많은 양의 
구조화된 데이터를 저장합니다. 서비스는
Azure 클라우드 내부 및 외부에서 인증된 호출을 수락하는 NoSQL 데이터 저장소입니다. Azure
테이블은 구조화된 비관계형 데이터를 저장하는 데 적합합니다. 일반적인
테이블 서비스의 사용은 다음과 같습니다.

-   웹 규모 응용 프로그램을 처리할 수 있는 구조화된 데이터 TB 저장응용 프로그램
-   복잡한 조인, 외래 키 또는 저장된 절차가 필요하지 않은 데이터 집합을 저장하거나 빠른 액세스를 위해 역 정규화할 수 있습니다.
-   클러스터형 인덱스를 사용하여 신속하게 데이터 쿼리
-   WCF에서 OData 프로토콜 및 LINQ 쿼리를 사용하여 데이터에 액세스 데이터 서비스 .NET 라이브러리

테이블 서비스를 사용하여 구조화된 쿼리 집합, 비관계형 데이터를 저장할 수 있으며 테이블은 요청대로
확장됩니다.

## 테이블 서비스 개념

테이블 서비스에는 다음 구성 요소가 포함됩니다.

![Table1][Table1]

-   **URL 형식:** 다음 주소 형식을 사용한 계정의 코드 주소
    테이블:   
    http://`<storage account>`.table.core.windows.net/`<table>`  
      
    OData 프로토콜과 함께 이 주소를 사용하여 Azure 테이블 주소를 직접 지정할 수
    있습니다. 자세한 내용은 [OData.org][]를 참조하세요.

-   **저장소 계정:** Azure 저장소에 대한 모든 액세스는
    저장소 계정을 통해 수행됩니다. 저장소 계정 용량에 대한 자세한 내용은 [Azure 저장소 확장성 및 성능 목표](http://msdn.microsoft.com/library/azure/dn249410.aspx)(영문)를 참조하세요.

-   **테이블**: 테이블은 엔터티 컬렉션입니다. 테이블은 엔티티에 스키마를 적용하지 않으며
    단일 테이블에 다른 속성 집합이 있는 엔티티를 포함할 수 있음을
    의미합니다. 저장소 계정에 
	포함할 수 있는 테이블 수는 
    저장소 계정 용량 제한으로만 제한됩니다.

-   **엔터티**: 엔터티는 데이터베이스 행과 유사한 속성
    집합입니다. 엔터티의 크기는 최대 1MB일 수 있습니다.

-   **속성**: 속성은 이름 값 쌍입니다. 각 엔터티에는
    데이터를 저장할 최대 252개까지의 속성이 포함됩니다. 또한 각 엔터티에는
    파티션 키, 행 키 및 타임스탬프를 지정하는 3개의 시스템
    속성이 있습니다. 파티션 키가 동일한 엔터티는 보다 신속하게 쿼리할 수 있으며
    원자성 작업에서 삽입/업데이트됩니다. 엔터티의 행
    키는 파티션 내 고유 식별자입니다.
