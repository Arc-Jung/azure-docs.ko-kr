## <a name="for-users-in-your-directory"></a>디렉터리에 있는 사용자의 경우

디렉터리에 있는 사용자가 이 액세스 패키지를 요청할 수 있도록 하려면 다음 단계를 따릅니다. 요청 정책을 정의할 때 개별 사용자 또는 더 일반적으로 사용자 그룹을 지정할 수 있습니다. 예를 들어 조직에 이미 **모든 직원과**같은 그룹이 있을 수 있습니다.  액세스를 요청할 수 있는 사용자에 대 한 정책에 해당 그룹이 추가 된 경우 해당 그룹의 모든 구성원 액세스를 요청할 수 있습니다.

1. **Users who can request access**(액세스를 요청할 수 있는 사용자) 섹션에서 **For users in your directory**(디렉터리에 있는 사용자의 경우) 클릭합니다.

    이 옵션을 선택하면 디렉터리에서 이 액세스 패키지를 요청할 수 있는 사람을 더 구체화하기 위한 새 옵션이 나타납니다.

    ![액세스 패키지 - 요청 - 디렉터리에 있는 사용자용](./media/active-directory-entitlement-management-request-policy/for-users-in-your-directory.png)

1. 다음 옵션 중 하나를 선택합니다.

    |  |  |
    | --- | --- |
    | **특정 사용자 및 그룹** | 디렉터리에서 이 액세스 패키지를 요청할 수 있도록 지정한 사용자 및 그룹만 하려면 이 옵션을 선택합니다. |
    | **모든 회원(게스트 제외)** | 디렉터리내의 모든 구성원 사용자가 이 액세스 패키지를 요청할 수 있도록 하려면 이 옵션을 선택합니다. 이 옵션에는 디렉터리에 초대했을 수 있는 게스트 사용자가 포함되지 않습니다. |
    | **모든 사용자(게스트 포함)** | 디렉터리내의 모든 멤버 사용자와 게스트 사용자가 이 액세스 패키지를 요청할 수 있도록 하려면 이 옵션을 선택합니다. |

    게스트 사용자는 [Azure AD B2B를](../articles/active-directory/b2b/what-is-b2b.md)사용하여 디렉터리에 초대된 외부 사용자를 참조합니다. 멤버 사용자와 게스트 사용자 간의 차이점에 대한 자세한 내용은 [Azure Active Directory의 기본 사용자 사용 권한은 무엇입니까?](../articles/active-directory/fundamentals/users-default-permissions.md)

1. **특정 사용자 및 그룹을**선택한 경우 , 사용자 및 그룹 **추가를**클릭합니다.

1. 사용자 및 그룹 선택 창에서 추가할 사용자 및 그룹을 선택합니다.

    ![액세스 패키지 - 요청 - 사용자 및 그룹 선택](./media/active-directory-entitlement-management-request-policy/select-users-groups.png)

1. 사용자 및 그룹을 추가하려면 **선택을** 클릭합니다.

1. [승인](#approval) 섹션으로 건너뜁니다.

## <a name="for-users-not-in-your-directory"></a>디렉터리에 없는 사용자의 경우

 **디렉터리에 없는 사용자는** 다른 Azure AD 디렉터리 또는 도메인에 있는 사용자를 참조합니다. 이러한 사용자는 아직 디렉터리로 초대되지 않았을 수 있습니다. Azure AD 디렉터리 공동 **작업 제한에서**초대를 허용하도록 구성해야 합니다. 자세한 내용은 [B2B 외부 공동 작업 활성화를 참조하고 게스트를 초대할 수 있는 사용자를 관리합니다.](../articles/active-directory/b2b/delegate-invitations.md)

> [!NOTE]
> 요청이 승인되거나 자동 승인된 디렉터리에서 아직 사용자를 위해 게스트 사용자 계정이 만들어집니다. 게스트가 초대되지만 초대 이메일은 받지 않습니다. 대신 액세스 패키지 할당이 배달되면 전자 메일을 받게 됩니다. 기본적으로 나중에 해당 게스트 사용자에게 액세스 패키지 할당이 더 이상 없을 때 마지막 할당이 만료되었거나 취소되었기 때문에 해당 게스트 사용자 계정은 로그인이 차단되고 이후에 삭제됩니다. 게스트 사용자가 액세스 패키지 할당이 없는 경우에도 디렉터리에 무기한 유지하도록 하려면 권한 부여 관리 구성에 대한 설정을 변경할 수 있습니다. 게스트 사용자 개체에 대한 자세한 내용은 [Azure Active Directory B2B 공동 작업 사용자의 속성을](../articles/active-directory/b2b/user-properties.md)참조하십시오.

디렉터리에 없는 사용자가 이 액세스 패키지를 요청하도록 하려면 다음 단계를 따르십시오.

1. 액세스 섹션을 **요청할 수 있는 사용자** 섹션에서 디렉터리에 없는 사용자 에 대해 **클릭합니다.**

    이 옵션을 선택하면 새 옵션이 나타납니다.

    ![액세스 패키지 - 요청 - 디렉터리에 없는 사용자의 경우](./media/active-directory-entitlement-management-request-policy/for-users-not-in-your-directory.png)

1. 다음 옵션 중 하나를 선택합니다.

    |  |  |
    | --- | --- |
    | **연결된 특정 조직** | 관리자가 이전에 추가한 조직 목록에서 선택하려면 이 옵션을 선택합니다. 선택한 조직의 모든 사용자가 이 액세스 패키지를 요청할 수 있습니다. |
    | **연결된 모든 조직** | 연결된 모든 조직의 모든 사용자가 이 액세스 패키지를 요청할 수 있는 경우 이 옵션을 선택합니다. |
    | **모든 사용자(연결된 모든 조직 + 새 외부 사용자)** | 연결된 모든 조직의 모든 사용자가 이 액세스 패키지를 요청할 수 있고 B2B에서 목록 허용 또는 거부 설정이 새 외부 사용자에게 우선되어야 하는 경우 이 옵션을 선택합니다. |

    연결된 조직은 연결된 외부 Azure AD 디렉터리 또는 도메인과 관계를 맺고 있습니다.

1. **연결된 특정 조직을**선택한 경우 **디렉터리 추가를** 클릭하여 관리자가 이전에 추가한 연결된 조직 목록에서 선택합니다.

1. 이름 또는 도메인 이름을 입력하여 이전에 연결된 조직을 검색합니다.

    ![액세스 패키지 - 요청 - 디렉터리 선택](./media/active-directory-entitlement-management-request-policy/select-directories.png)

    공동 작업하려는 조직이 목록에 없는 경우 관리자에게 연결된 조직으로 추가하도록 요청할 수 있습니다. 자세한 내용은 [연결된 조직 추가를](../articles/active-directory/governance/entitlement-management-organization.md)참조하십시오.

1. 연결된 모든 조직을 선택한 후 **선택 을**클릭합니다.

    > [!NOTE]
    > 선택한 연결된 조직의 모든 사용자가 이 액세스 패키지를 요청할 수 있습니다. 여기에는 Azure B2B 허용 또는 거부 목록에 의해 해당 도메인이 차단되지 않는 한 조직과 연결된 모든 하위 도메인의 Azure AD 사용자가 포함됩니다. 자세한 내용은 [특정 조직의 B2B 사용자 초대 허용 또는 차단](../articles/active-directory/b2b/allow-deny-list.md)을 참조하세요.

1. [승인](#approval) 섹션으로 건너뜁니다.

## <a name="none-administrator-direct-assignments-only"></a>없음(관리자 직접 할당만)

액세스 요청을 우회하고 관리자가 이 액세스 패키지에 특정 사용자를 직접 할당할 수 있도록 하려면 다음 단계를 따릅니다. 사용자는 액세스 패키지를 요청할 필요가 없습니다. 수명 주기 설정을 설정할 수 있지만 요청 설정은 없습니다.

1. 액세스 섹션을 **요청할 수 있는 사용자** 섹션에서 **없음(관리자 직접 할당만)을 클릭합니다.**

    ![액세스 패키지 - 요청 - 관리자 직접 할당만 없음](./media/active-directory-entitlement-management-request-policy/none-admin-direct-assignments-only.png)

    액세스 패키지를 만든 후 액세스 패키지에 특정 내부 및 외부 사용자를 직접 할당할 수 있습니다. 외부 사용자를 지정하면 디렉터리에서 게스트 사용자 계정이 만들어집니다. 사용자를 직접 할당하는 것에 대한 자세한 내용은 [액세스 패키지에 대한 할당 보기, 추가 및 제거를](../articles/active-directory/governance/entitlement-management-access-package-assignments.md)참조하십시오.

1. [요청 활성화](#enable-requests) 섹션으로 건너뜁니다.

## <a name="approval"></a>승인

승인 섹션에서 사용자가 이 액세스 패키지를 요청할 때 승인이 필요한지 여부를 지정합니다. 승인 설정은 다음과 같은 방식으로 작동합니다.

- 선택한 승인자 또는 대체 승인자 중 한 명만 단일 단계 승인 요청을 승인해야 합니다. 
- 각 단계에서 선택한 승인자 중 한 명만 2단계 승인 요청을 승인해야 합니다.
- 승인자는 정책이 액세스를 관리하는 사람에 따라 관리자, 내부 스폰서 또는 외부 스폰서가 될 수 있습니다.
- 선택한 모든 승인자로부터 의한 승인은 단일 또는 2단계 승인에 필요하지 않습니다.
- 승인 결정은 요청을 먼저 검토하는 승인자를 기반으로 합니다.

요청 정책에 승인자를 추가하는 방법에 대한 데모를 보려면 다음 비디오를 시청하십시오.

>[!VIDEO https://www.microsoft.com/videoplayer/embed/RE4cZfg]

요청 정책에 다단계 승인을 추가하는 방법에 대한 설명은 다음 비디오를 시청하세요.

>[!VIDEO https://www.microsoft.com/videoplayer/embed/RE4d1Jw]

다음 단계에 따라 액세스 패키지에 대한 요청에 대한 승인 설정을 지정합니다.

1. 선택한 사용자의 요청에 대한 승인이 필요하려면 **승인 필요** 를 **예로**설정합니다. 또는 요청을 자동으로 승인하려면 토글을 **아니오로 설정합니다.**

1. 사용자가 액세스 패키지를 요청할 수 있는 근거를 제공하도록 요구하려면 **요청자 요청 자리** 맞추기를 **예로**설정합니다.
    
1. 이제 요청에 단일 또는 2단계 승인이 필요한지 여부를 결정합니다. 단일 **스테이지** 승인을 위해 스테이지 전환 수를 **1로** 설정하거나 2단계 승인을 위해 토글을 **2로** 설정합니다.

    ![액세스 패키지 - 요청 - 승인 설정](./media/active-directory-entitlement-management-request-policy/approval.png)

필요한 단계 수를 선택한 후 승인자를 추가하려면 다음 단계를 따르세요. 

### <a name="single-stage-approval"></a>단단계 승인

1. 첫 **번째 승인자**추가 :
    
    정책이 디렉터리에서 사용자에 대한 액세스를 제어하도록 설정된 경우 **관리자를 승인자로**선택할 수 있습니다. 또는 드롭다운 메뉴에서 특정 승인자 선택을 선택한 후 **승인자 추가를** 클릭하여 특정 사용자를 추가합니다.
    
    ![액세스 패키지 - 요청 - 디렉터리 사용자용 - 첫 번째 승인자](./media/active-directory-entitlement-management-request-policy/approval-single-stage-first-approver-manager.png)

    이 정책이 디렉터리에 없는 사용자의 액세스를 제어하도록 설정된 경우 **외부 스폰서** 또는 **내부 스폰서를**선택할 수 있습니다. 또는 특정 승인자 선택에서 승인자 또는 그룹 **추가를** 클릭하여 특정 사용자를 추가합니다.
    
    ![액세스 패키지 - 요청 - 디렉터리 를 벗어난 사용자 - 첫 번째 승인자](./media/active-directory-entitlement-management-request-policy/out-directory-first-approver.png)
    
1. **Manager를** 첫 번째 승인자로 선택한 경우 **대체 추가를** 클릭하여 디렉터리에서 하나 이상의 사용자 또는 그룹을 선택하여 대체 승인자로 지정합니다. 권한 부여 관리에서 액세스를 요청하는 사용자의 관리자를 찾을 수 없는 경우 대체 승인자가 요청을 받습니다.

    관리자는 **Manager** 특성을 사용하여 권한 관리에서 찾을 수 있습니다. 특성은 Azure AD의 사용자 프로필에 있습니다. 자세한 내용은 [Azure Active Directory](../articles/active-directory/fundamentals/active-directory-users-profile-azure-portal.md)를 사용하여 사용자의 프로필 정보 추가 또는 업데이트를 참조하세요.

1. **특정 승인자 선택을**선택한 경우 **승인자 추가를** 클릭하여 디렉터리에서 승인자가 될 사용자 또는 그룹을 하나 이상 선택합니다.

1. 결정 의 상자에서 **몇 일 동안 만들어야합니까?**, 승인자가 이 액세스 패키지에 대한 요청을 검토해야 하는 일 수를 지정합니다.

    이 기간 내에 요청이 승인되지 않으면 자동으로 거부됩니다. 사용자는 액세스 패키지에 대한 다른 요청을 제출해야 합니다.

1. 승인자가 자신의 결정에 대한 정당성을 제공하도록 요구하려면 승인자 정당화를 **예로**설정합니다.

    근거는 다른 승인자와 요청자가 볼 수 있습니다.

### <a name="2-stage-approval-preview"></a>2단계 승인(미리 보기)

2단계 승인을 선택한 경우 두 번째 승인자를 추가해야 합니다.

1. 두 **번째 승인자**추가 : 
    
    사용자가 디렉터리에 있는 경우 특정 승인자 선택에서 **승인자 추가를** 클릭하여 특정 사용자를 두 번째 승인자로 추가합니다.

    ![액세스 패키지 - 요청 - 디렉터리 사용자 - 두 번째 승인자](./media/active-directory-entitlement-management-request-policy/in-directory-second-approver.png)

    사용자가 디렉터리에 없는 경우 내부 **스폰서** 또는 **외부 스폰서를** 두 번째 승인자로 선택합니다. 승인자를 선택한 후 대체 승인자를 추가합니다.

    ![액세스 패키지 - 요청 - 디렉터리 를 벗어난 사용자 - 두 번째 승인자](./media/active-directory-entitlement-management-request-policy/out-directory-second-approver.png) 

1. 두 번째 승인자가 결정에 따라 상자에 요청을 승인해야 하는 일 수를 **지정하려면 며칠 이내에 해야 합니까?**. 

1. 승인자 필요 를 **예** 또는 **아니요로**전환하도록 설정합니다.

### <a name="alternate-approvers"></a>대체 승인자

요청을 승인할 수 있는 첫 번째 및 두 번째 승인자를 지정하는 것과 유사하게 대체 승인자를 지정할 수 있습니다. 대체 승인자가 있으면 요청이 만료되기 전에 승인 또는 거부됩니다(시간 시간). 2단계 승인을 위해 대체 승인자를 첫 번째 승인자와 두 번째 승인자를 나열할 수 있습니다. 

첫 번째 또는 두 번째 승인자가 요청을 승인하거나 거부할 수 없는 경우 대체 승인자를 지정하면 보류 중인 요청이 정책 설정 중에 지정한 전달 일정에 따라 대체 승인자에게 전달됩니다. 보류 중인 요청을 승인하거나 거부하는 전자 메일을 받습니다.

요청이 대체 승인자에게 전달된 후에도 첫 번째 또는 두 번째 승인자는 요청을 승인하거나 거부할 수 있습니다. 대체 승인자는 동일한 내 액세스 사이트를 사용하여 보류 중인 요청을 승인하거나 거부합니다.

승인자 및 대체 승인자가 될 사람 또는 그룹을 나열할 수 있습니다. 첫 번째, 두 번째 및 대체 승인자가 될 다른 사람 집합을 나열해야 합니다.
예를 들어 Alice와 Bob을 첫 번째 승인자로 나열한 경우 캐롤과 데이브를 대체 승인자로 나열합니다. 다음 단계를 사용하여 액세스 패키지에 대체 승인자를 추가합니다.

1. 첫 번째 승인자, 두 번째 승인자 또는 둘 다에서 **고급 요청 설정 표시를**클릭합니다.

    ![액세스 패키지 - 정책 - 고급 요청 설정 표시](./media/active-directory-entitlement-management-request-policy/alternate-approvers-click-advanced-request.png)

1. 설정 **작업이 수행되지 않으면 대체 승인자로 전달합니까?** **Yes**

1. **대체 승인자 추가를** 클릭하고 목록에서 대체 승인자를 선택합니다.

    ![액세스 패키지 - 정책 - 대체 승인자 추가](./media/active-directory-entitlement-management-request-policy/alternate-approvers-add.png)

1. 승인자가 요청을 승인하거나 거부해야 하는 일 수를 입력한 **후 대체 승인자로** 이동합니다. 요청 기간 전에 승인을 거부하지 않은 경우 요청이 만료(시간 시간)되고 사용자는 액세스 패키지에 대한 다른 요청을 제출해야 합니다. 

    요청 기간은 반감기에 도달한 후 하루 만에 대체 승인자에 요청이 전달될 수 있습니다. 이 예제에서 요청 기간은 14일입니다. 따라서 요청 기간은 7일째에 반감기에 도달합니다. 따라서 요청은 8일 이전에 전달할 수 없습니다. 또한 요청 기간의 마지막 날에는 요청을 전달할 수 없습니다. 따라서 이 예제에서는 요청을 전달할 수 있는 최신 요청이 13일째입니다.

## <a name="enable-requests"></a>요청 사용

1. 요청 정책의 사용자가 액세스 패키지를 즉시 사용할 수 있도록 하려면 사용 전환 을 **예로**이동합니다.

    액세스 패키지 만들기를 완료한 후에는 언제든지 나중에 활성화할 수 있습니다.

    **없음(관리자 직접 할당만 해당)을** 선택하고 사용 권한을 **아니요로**설정한 경우 관리자는 이 액세스 패키지를 직접 할당할 수 없습니다.

    ![액세스 패키지 - 정책 - 정책 설정 사용](./media/active-directory-entitlement-management-request-policy/enable-requests.png)

1. **다음**을 클릭합니다.
