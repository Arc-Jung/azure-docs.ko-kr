### <a name="timeout-values"></a>시간 제한 값
#### <a name="client-to-front-door"></a>클라이언트-Front Door
* Front Door에는 유휴 TCP 연결 시간 제한(61초)이 있습니다.

#### <a name="front-door-to-application-back-end"></a>애플리케이션 백 엔드에 대한 전면 도어
* 응답이 청크 응답인 경우 첫 번째 청크가 수신되면 200이 반환됩니다.
* HTTP 요청이 백 엔드로 전달되면 프런트 도어는 백 엔드에서 첫 번째 패킷을 30초 동안 기다립니다. 그런 다음 클라이언트에 503 오류를 반환합니다. 이 값은 API의 필드 sendRecvTimeoutSeconds를 통해 구성할 수 있습니다.
    * 캐싱 시나리오의 경우 이 시간 초과는 구성할 수 없으므로 요청이 캐시되고 Front Door 또는 백 엔드에서 첫 번째 패킷에 30초 이상 걸리는 경우 504 오류가 클라이언트에 반환됩니다. 
* 백 엔드에서 첫 번째 패킷을 받은 후 정문은 유휴 시간 에서 30초 동안 기다립니다. 그런 다음 클라이언트에 503 오류를 반환합니다. 이 시간 시간 값은 구성할 수 없습니다.
* 백 엔드 TCP 세션 시간 시간에 정문은 90초입니다.

### <a name="upload-and-download-data-limit"></a>데이터 한도 업로드 및 다운로드

|  | 청크 전송 인코딩(CTE) | HTTP 청킹 없이 |
| ---- | ------- | ------- |
| **다운로드** | 다운로드 크기에는 제한이 없습니다. | 다운로드 크기에는 제한이 없습니다. |
| **업로드** |    각 CTE 업로드가 2GB 미만인 한 제한은 없습니다. | 크기는 2GB보다 클 수 없습니다. |

### <a name="other-limits"></a>기타 제한
* 최대 URL 크기 - 8,192바이트 - 원시 URL의 최대 길이를 지정합니다(구성표 + 호스트 이름 + 포트 + 경로 + URL의 쿼리 문자열)
* 최대 쿼리 문자열 크기 - 4,096바이트 - 쿼리 문자열의 최대 길이를 바이트로 지정합니다.
* 상태 프로브 URL의 최대 HTTP 응답 헤더 크기 - 4,096바이트 - 상태 프로브의 모든 응답 헤더의 최대 길이를 지정했습니다.
