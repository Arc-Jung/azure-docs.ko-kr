

다음 단계에서는 Azure에 새 모바일 서비스를 만들고 앱을 이 새 서비스에 연결하는 코드를 프로젝트에 추가합니다. Visual Studio 2013은 사용자 대신 Azure에 연결하여 사용자가 입력하는 자격 증명을 사용함으로써 새 모바일 서비스를 만듭니다. 모바일 서비스를 만들 때에는 모바일 서비스가 앱 데이터를 저장하기 위해 사용할 Azure SQL 데이터베이스를 지정해야 합니다.

1. Visual Studio 2013에서 솔루션 탐색기를 열고 Windows 스토어 앱 프로젝트를 마우스 오른쪽 단추로 클릭한 후 **추가**를 클릭하고 **연결된 서비스...**를 클릭합니다. 

2. 서비스 관리자 대화 상자에서 **서비스 만들기...**를 클릭하고 모바일 서비스 만들기 대화 상자의 **구독**에서 **&lt;Manage...&gt;**를 선택합니다.

	![서비스 관리 구독 만들기](./media/mobile-services-dotnet-backend-create-new-service-vs2013/mobile-create-service-from-vs2013.png)

3. Microsoft Azure 구독 관리에서 **로그인...**을 클릭하여 필요에 따라 Azure 계정에 로그인하고 사용 가능한 구독을 선택한 후에 **닫기**를 클릭합니다.

	사용자 구독에 하나 이상의 모바일 서비스가 있는 경우 해당 서비스 이름이 나타납니다.

5. **모바일 서비스 만들기** 대화 상자로 돌아가서 **구독**, **런타임**의 **.NET Framework** 백 엔드 그리고 모바일 서비스의 **지역**을 선택한 후에 모바일 서비스 **이름**을 입력합니다.

	>[AZURE.NOTE]모바일 서비스 이름은 고유해야 합니다. 제공한 이름을 사용할 수 없는 경우 **Name** 옆에 빨간 X가 표시됩니다.

6. **데이터베이스**에서 **&lt;무료 SQL 데이터베이스 만들기&gt;**를 선택하고 **서버 사용자 이름**, **서버 암호** 및 **서버 암호 확인**을 입력한 후 **만들기**를 클릭합니다.

  	![VS 2013에서 새 모바일 서비스 만들기](./media/mobile-services-dotnet-backend-create-new-service-vs2013/mobile-create-service-from-vs2013-2.png)

	> [AZURE.NOTE]이 자습서에는 새 무료 SQL 데이터베이스 인스턴스와 서버를 만드는 과정이 포함되어 있으며, 이 새 데이터베이스는 다른 SQL 데이터베이스 인스턴스처럼 다시 사용하고 관리할 수 있습니다. 무료 데이터베이스 인스턴스는 한 개만 사용할 수 있습니다. 새 모바일 서비스로 같은 지역에 데이터베이스를 이미 가지고 있다면 기존 데이터베이스를 대신 선택할 수 있습니다. 기존 데이터베이스를 선택하는 경우에는 정확한 로그인 자격 증명을 제공해야 합니다. 정확하지 않은 자격 증명을 제공하면 모바일 서비스가 비정상 상태로 생성됩니다.

7. 모바일 서비스를 만든 후 서비스 관리자의 목록에서 새로 만든 모바일 서비스를 선택하고 **확인**을 클릭합니다.
 
   	마법사가 완료되면 모바일 서비스 프로젝트가 솔루션에 추가되고 필요한 NuGet 패키지가 설치되며, 모바일 서비스 클라이언트 라이브러리에 대한 참조가 프로젝트에 추가되고 프로젝트 소스 코드가 업데이트됩니다.
