<sup>1</sup>웹사이트 및 저장소 할당량은 별도로 언급하지 않는 한 웹호스팅 계획별로 지정됩니다.

<sup>2</sup>기본 및 표준 사이트 전용된 컴퓨터에서 실행합니다. 이러한 컴퓨터에서 호스팅할 수 있는 사이트의 실제 수는 사이트의 활동, 컴퓨터 인스턴스 및 해당 리소스 사용률의 크기에 따라 달라집니다.

<sup>3</sup>무료 계층에서 제외하고 웹 호스팅 계획에 대한 제한을 리소스 그룹별로 지정합니다. 리소스 그룹 만들기 및 관리에 대한 자세한 내용은 [리소스 그룹을 사용하여 Azure 리소스][useresourcegroups]관리를 참조하세요.

<sup>4</sup>전용 인스턴스는 작은 크기, 중간 크기 또는 큰 크기로 만들 수 있습니다. 자세한 내용은 [Azure 웹사이트 가격 책정 페이지][websitespricing]를 참조하세요. 지원 요청을 열어서 추가 인스턴스를 사용할 수 있습니다.

<sup>5</sup>모든 웹 사이트를 사이트 콘텐츠에 대한 동일한 저장소 리소스와 공유합니다. 따라서 저장 용량 제한은 웹 사이트 단위가 아닌 모든 사이트에서의 총 크기입니다. 저장소 제한은 지원 요청을 열어 늘릴 수 있습니다.

<sup>6</sup>기본 및 표준 계층에 CPU 또는 메모리에 대한 할당량은 없지만, 이러한 리소스는 전용 인스턴스의(인스턴스 크기 및 인스턴스 수) 실제 리소스에 의해 제한됩니다.

<sup>7</sup>웹 호스팅 계획의 모든 무료 사이트 간에 공유되는 무료 계층에 대한 메모리 제한입니다. 각 사이트의 공유 계층에 대한 메모리 제한입니다.

<sup>8</sup>이 값은 웹 사이트 인스턴스 당 동시 웹 소켓 연결을 나타냅니다. 예를 들어, 기본 웹 사이트를 두 인스턴스로 배율 조정 하는 경우에 700 동시 연결(350 x 2)을 해야 효과적입니다.

<sup>9</sup>기본 계층의 경우 표준 [SSL 가격][sslpricing]을 적용합니다.

<sup>10</sup>웹 사이트 인스턴스 내에서 백그라운드 작업으로 지속적으로, 일정에 따라 또는 요청 시 사용자 지정 실행 파일 및/또는 스크립트를 실행합니다. Always On은 연속 Webjob 실행을 위해 필요합니다. Azure 스케줄러 무료 또는 표준은 예약 된 Webjob을 위해 필요합니다.
